```python
def early_pqc_research():
    """The development of post-quantum cryptographic approaches."""
    print("\nEARLY POST-QUANTUM CRYPTOGRAPHY RESEARCH")
    print("=" * 70)

    print("""
    ALTERNATIVE MATHEMATICAL FOUNDATIONS:
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    Researchers began exploring mathematical problems believed
    to be hard even for quantum computers:

    1. LATTICE-BASED CRYPTOGRAPHY (1996-present)
       ┌─────────────────────────────────────────────────────────┐
       │ Founded on hard lattice problems:                       │
       │ • Shortest Vector Problem (SVP)                         │
       │ • Learning With Errors (LWE) - Regev, 2005              │
       │ • Ring-LWE - Lyubashevsky, Peikert, Regev, 2010        │
       │                                                         │
       │ Advantages: Efficient, versatile, well-studied          │
       └─────────────────────────────────────────────────────────┘

    2. HASH-BASED SIGNATURES (1979-present)
       ┌─────────────────────────────────────────────────────────┐
       │ Security relies only on hash function properties:       │
       │ • Lamport signatures (1979)                             │
       │ • Merkle signature scheme (1989)                        │
       │ • XMSS, SPHINCS+ (2010s)                                │
       │                                                         │
       │ Advantages: Conservative security assumptions           │
       └─────────────────────────────────────────────────────────┘

    3. CODE-BASED CRYPTOGRAPHY (1978-present)
       ┌─────────────────────────────────────────────────────────┐
       │ Based on error-correcting codes:                        │
       │ • McEliece cryptosystem (1978) - oldest PQC scheme!     │
       │ • Niederreiter variant (1986)                           │
       │ • Modern variants: BIKE, Classic McEliece               │
       │                                                         │
       │ Advantages: 45+ years of cryptanalysis                  │
       └─────────────────────────────────────────────────────────┘

    4. MULTIVARIATE CRYPTOGRAPHY (1980s-present)
       ┌─────────────────────────────────────────────────────────┐
       │ Based on solving systems of multivariate equations:     │
       │ • Oil-Vinegar schemes                                   │
       │ • Rainbow (broken 2022)                                 │
       │ • GeMSS                                                 │
       │                                                         │
       │ Status: Many schemes broken, limited current use        │
       └─────────────────────────────────────────────────────────┘

    5. ISOGENY-BASED CRYPTOGRAPHY (2006-present)
       ┌─────────────────────────────────────────────────────────┐
       │ Based on maps between elliptic curves:                  │
       │ • SIDH/SIKE (broken 2022)                               │
       │ • CSIDH (under study)                                   │
       │                                                         │
       │ Status: Active research after SIKE break                │
       └─────────────────────────────────────────────────────────┘
    """)
```

```python
    ━━━━━━━━━━━━━━━━━━

    On December 20, 2016, NIST issued a formal call for proposals
    for quantum-resistant public-key cryptographic algorithms.

    ┌─────────────────────────────────────────────────────────────────┐
    │                                                                 │
    │  "NIST is initiating a process to solicit, evaluate, and       │
    │   standardize one or more quantum-resistant public-key         │
    │   cryptographic algorithms."                                   │
    │                                                                 │
    │                        - NIST, December 2016                   │
    │                                                                 │
    └─────────────────────────────────────────────────────────────────┘


    WHAT NIST REQUESTED:
    ━━━━━━━━━━━━━━━━━━━━━

    Two categories of algorithms:

    1. PUBLIC-KEY ENCRYPTION / KEY ENCAPSULATION MECHANISMS (KEMs)
       • Replace RSA encryption
       • Replace Diffie-Hellman key exchange
       • Replace ECDH

    2. DIGITAL SIGNATURE ALGORITHMS
       • Replace RSA signatures
       • Replace DSA
       • Replace ECDSA


    SUBMISSION REQUIREMENTS:
    ━━━━━━━━━━━━━━━━━━━━━━━━━

    ┌─────────────────────────────────────────────────────────────────┐

    │ • Mathematical description                                     │
    │ • Security analysis and proofs                                 │
    │ • Performance characteristics                                  │
    │                                                                 │
    │ REFERENCE IMPLEMENTATION                                       │
    │ • Working code in C                                            │
    │ • Known Answer Tests (KATs)                                    │
    │                                                                 │
    │ INTELLECTUAL PROPERTY                                          │
    │ • Royalty-free licensing commitment                            │
    │ • Worldwide availability                                       │
    │                                                                 │
    │ DEADLINE: November 30, 2017                                    │
    └─────────────────────────────────────────────────────────────────┘


    WHY AN OPEN COMPETITION?
    ━━━━━━━━━━━━━━━━━━━━━━━━━

    NIST learned from successful past competitions:

    """)
```

## Summary

The NIST Post-Quantum Cryptography standardization initiative represents one of the most important cryptographic efforts in history:

1. **Origins**: Shor's 1994 algorithm revealed that quantum computers would break current public-key cryptography
2. **NIST's Response**: Open competition launched in 2016, following successful AES and SHA-3 models
3. **Global Participation**: 82 submissions from researchers worldwide
4. **Rigorous Evaluation**: Three rounds over 8 years, with continuous cryptanalysis
5. **Final Standards**: FIPS 203 (ML-KEM), FIPS 204 (ML-DSA), FIPS 205 (SLH-DSA) published August 2024

## Key Takeaways

- The quantum threat to cryptography has been known since 1994
- NIST's open competition model produces trusted, well-analyzed standards
- Post-quantum cryptography uses mathematical problems believed hard for quantum computers
- The SIKE break demonstrates why diversity and conservative security margins matter
- Organizations should begin migration planning now
