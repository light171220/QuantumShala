# SLH-DSA Parameter Sets

SLH-DSA (FIPS 205) defines 12 parameter sets offering different trade-offs between security level, signature size, and signing speed. Understanding these parameters is essential for selecting the right variant for your application.

## Parameter Set Overview

├─────────────────────────────────────────────────────────────────┤
│                                                                 │

│                                                                 │
│   HASH:     SHA2 or SHAKE                                      │
│   SECURITY: 128, 192, or 256 (NIST security category)         │
│   VARIANT:  f (fast) or s (small)                              │

## Fast vs Small Variants

```
┌─────────────────────────────────────────────────────────────────┐
│              FAST (f) VS SMALL (s) VARIANTS                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Fast Variants (f):                                            │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Optimized for: Signing speed                         │  │
│   │                                                         │  │
│   │   Characteristics:                                      │  │
│   │   • More hypertree layers (d=17-22)                    │  │
│   │   • Shorter trees per layer (h/d ≈ 3)                  │  │
│   │   • More FORS trees with shorter height                │  │
│   │   • Larger signatures (17-50 KB)                       │  │
│   │   • Faster signing (~10-50 ms)                         │  │
│   │                                                         │  │
│   │   Best for:                                             │  │
│   │   • Interactive applications                           │  │
│   │   • Frequent signing with bandwidth tolerance          │  │
│   │   • Development/testing environments                   │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Small Variants (s):                                           │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Optimized for: Signature size                        │  │
│   │                                                         │  │
│   │   Characteristics:                                      │  │
│   │   • Fewer hypertree layers (d=7-8)                     │  │
│   │   • Taller trees per layer (h/d ≈ 8-9)                 │  │
│   │   • Fewer FORS trees with taller height               │  │
│   │   • Smaller signatures (7-30 KB)                       │  │
│   │   • Slower signing (~200-500 ms)                       │  │
│   │                                                         │  │
│   │   Best for:                                             │  │
│   │   • Bandwidth-constrained environments                 │  │
│   │   • Infrequent signing (CA certificates)              │  │
│   │   • Long-term archival signatures                      │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Comparison (128-bit security):                               │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │              │   128f        │    128s                 │  │
│   │   Signature  │   17,088 B    │    7,856 B              │  │
│   │   Sign time  │   ~10 ms      │    ~200 ms              │  │
│   │   Verify     │   ~0.5 ms     │    ~5 ms                │  │
│   │   Ratio      │   2.2× larger │    20× slower           │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## SHA-2 vs SHAKE Instantiations

```
┌─────────────────────────────────────────────────────────────────┐
│              SHA-2 VS SHAKE HASH FUNCTIONS                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Both provide equivalent security - choice is about:          │
│   • Performance characteristics                                 │
│   • Hardware availability                                       │
│   • Compliance requirements                                     │
│                                                                 │
│   SHA-2 Variants (SHA-256, SHA-512):                           │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Advantages:                                           │  │
│   │   • Widely deployed hardware acceleration              │  │
│   │   • Extensive implementation experience                │  │
│   │   • Intel SHA-NI, ARM SHA2 instructions               │  │
│   │   • Familiar to implementers                           │  │
│   │                                                         │  │
│   │   Considerations:                                       │  │
│   │   • Fixed output sizes (256, 512 bits)                │  │
│   │   • May need HMAC construction for keyed hashing      │  │
│   │   • Uses SHA-256 for 128-bit, SHA-512 for 192/256    │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   SHAKE Variants (SHAKE256):                                   │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Advantages:                                           │  │
│   │   • Extendable output function (XOF)                   │  │
│   │   • Native variable-length output                      │  │
│   │   • Simpler domain separation                          │  │
│   │   • Part of SHA-3 family                               │  │
│   │                                                         │  │
│   │   Considerations:                                       │  │
│   │   • Less hardware support currently                    │  │
│   │   • Uses SHAKE128 for 128-bit, SHAKE256 for 192/256   │  │
│   │   • May be faster in software-only implementations    │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Internal Function Usage:                                      │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   SLH-DSA requires these hash functions:               │  │
│   │                                                         │  │
│   │   • H: General hash function                           │  │
│   │   • PRF: Pseudorandom function                         │  │
│   │   • PRFmsg: PRF for message randomization              │  │
│   │   • Hmsg: Hash for message digest                      │  │
│   │   • F: Tweakable hash (WOTS+ chains)                   │  │
│   │   • T: Tweakable hash (tree nodes)                     │  │
│   │                                                         │  │
│   │   SHA-2: Uses HMAC-based constructions                 │  │
│   │   SHAKE: Uses native XOF with domain separation        │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Parameter Selection Guide

```python
from dataclasses import dataclass
from typing import Optional

@dataclass
class SLHDSAParams:
    """SLH-DSA parameter set information."""
    name: str
    security_level: int  # 1, 3, or 5
    variant: str  # 'f' or 's'
    hash_family: str  # 'SHA2' or 'SHAKE'
    n: int  # Security parameter
    h: int  # Total tree height
    d: int  # Layers
    a: int  # FORS tree height
    k: int  # FORS trees
    pk_size: int  # Public key bytes
    sk_size: int  # Secret key bytes
    sig_size: int  # Signature bytes
    sign_time_ms: float  # Approximate signing time
    verify_time_ms: float  # Approximate verification time

# All 12 parameter sets
SLH_DSA_PARAMS = {
    'SLH-DSA-SHA2-128f': SLHDSAParams(
        'SLH-DSA-SHA2-128f', 1, 'f', 'SHA2',
        n=16, h=66, d=22, a=6, k=33,
        pk_size=32, sk_size=64, sig_size=17088,
        sign_time_ms=8, verify_time_ms=0.4
    ),
    'SLH-DSA-SHA2-128s': SLHDSAParams(
        'SLH-DSA-SHA2-128s', 1, 's', 'SHA2',
        n=16, h=63, d=7, a=12, k=14,
        pk_size=32, sk_size=64, sig_size=7856,
        sign_time_ms=180, verify_time_ms=3
    ),
    'SLH-DSA-SHA2-192f': SLHDSAParams(
        'SLH-DSA-SHA2-192f', 3, 'f', 'SHA2',
        n=24, h=66, d=22, a=8, k=33,
        pk_size=48, sk_size=96, sig_size=35664,
        sign_time_ms=15, verify_time_ms=0.7
    ),
    'SLH-DSA-SHA2-192s': SLHDSAParams(
        'SLH-DSA-SHA2-192s', 3, 's', 'SHA2',
        n=24, h=63, d=7, a=14, k=17,
        pk_size=48, sk_size=96, sig_size=16224,
        sign_time_ms=300, verify_time_ms=5
    ),
    'SLH-DSA-SHA2-256f': SLHDSAParams(
        'SLH-DSA-SHA2-256f', 5, 'f', 'SHA2',
        n=32, h=68, d=17, a=9, k=35,
        pk_size=64, sk_size=128, sig_size=49856,
        sign_time_ms=30, verify_time_ms=1.2
    ),
    'SLH-DSA-SHA2-256s': SLHDSAParams(
        'SLH-DSA-SHA2-256s', 5, 's', 'SHA2',
        n=32, h=64, d=8, a=14, k=22,
        pk_size=64, sk_size=128, sig_size=29792,
        sign_time_ms=400, verify_time_ms=7
    ),
}

# Add SHAKE variants (same sizes as SHA2)
for sha2_name, params in list(SLH_DSA_PARAMS.items()):
    shake_name = sha2_name.replace('SHA2', 'SHAKE')
    SLH_DSA_PARAMS[shake_name] = SLHDSAParams(
        shake_name, params.security_level, params.variant, 'SHAKE',
        params.n, params.h, params.d, params.a, params.k,
        params.pk_size, params.sk_size, params.sig_size,
        params.sign_time_ms, params.verify_time_ms
    )

def recommend_params(
    min_security: int = 1,
    max_sig_kb: Optional[float] = None,
    max_sign_ms: Optional[float] = None,
    prefer_sha2: bool = True,
    use_case: str = 'general'
) -> list:
    """
    Recommend SLH-DSA parameter sets based on requirements.

    Args:
        min_security: Minimum NIST category (1, 3, or 5)
        max_sig_kb: Maximum signature size in KB
        max_sign_ms: Maximum signing time in ms
        prefer_sha2: Prefer SHA-2 over SHAKE
        use_case: 'general', 'root_ca', 'firmware', 'frequent'

    Returns:
        List of suitable parameter set names, best first
    """
    candidates = []

    for name, params in SLH_DSA_PARAMS.items():
        # Check security level
        if params.security_level < min_security:
            continue

        # Check signature size
        if max_sig_kb and params.sig_size > max_sig_kb * 1024:
            continue

        # Check signing time
        if max_sign_ms and params.sign_time_ms > max_sign_ms:
            continue

        candidates.append((name, params))

    # Sort by use case priority
    def score(item):
        name, params = item
        s = 0

        # Prefer requested hash family
        if prefer_sha2 and params.hash_family == 'SHA2':
            s += 100
        elif not prefer_sha2 and params.hash_family == 'SHAKE':
            s += 100

        # Use case specific scoring
        if use_case == 'root_ca':
            # Prefer small signatures, slow OK
            s -= params.sig_size / 1000
            s += 10 if params.variant == 's' else 0
        elif use_case == 'firmware':
            # Prefer small signatures for distribution
            s -= params.sig_size / 1000
        elif use_case == 'frequent':
            # Prefer fast signing
            s -= params.sign_time_ms
            s += 50 if params.variant == 'f' else 0
        else:  # general
            # Balance size and speed
            s -= params.sig_size / 2000
            s -= params.sign_time_ms / 10

        return -s  # Negative for descending sort

    candidates.sort(key=score)
    return [name for name, _ in candidates]


# Example usage
print("Parameter Selection Examples:\n")

# Root CA certificate
ca_params = recommend_params(
    min_security=5,
    use_case='root_ca',
    prefer_sha2=True
)
print(f"Root CA (highest security, size OK):")
print(f"  Recommended: {ca_params[0]}")
print(f"  Signature: {SLH_DSA_PARAMS[ca_params[0]].sig_size:,} bytes")

# Firmware signing
fw_params = recommend_params(
    min_security=3,
    max_sig_kb=20,
    use_case='firmware'
)
print(f"\nFirmware Signing (moderate security, <20KB sig):")
print(f"  Recommended: {fw_params[0]}")
print(f"  Signature: {SLH_DSA_PARAMS[fw_params[0]].sig_size:,} bytes")

# Interactive application
app_params = recommend_params(
    min_security=1,
    max_sign_ms=50,
    use_case='frequent'
)
print(f"\nInteractive App (fast signing needed):")
print(f"  Recommended: {app_params[0]}")
print(f"  Sign time: {SLH_DSA_PARAMS[app_params[0]].sign_time_ms} ms")
```

## Summary

SLH-DSA parameter selection key points:

**12 Parameter Sets:**
- 3 security levels (128, 192, 256 bits)
- 2 variants each (fast/small)
- 2 hash families (SHA-2/SHAKE)

**Fast (f) vs Small (s):**
- Fast: More layers, faster signing, larger signatures
- Small: Fewer layers, slower signing, smaller signatures
- Speed difference: ~20× between variants
- Size difference: ~2× between variants

**SHA-2 vs SHAKE:**
- Equivalent security
- SHA-2: More hardware acceleration currently
- SHAKE: Native XOF, simpler construction
- Choose based on infrastructure and compliance

**Selection Guidelines:**
- Root CAs: 256s (max security, size OK)
- Intermediate CAs: 192s (balanced)
- Firmware: 128s (small distribution size)
- High-volume: 128f (if SLH-DSA required)
- Consider ML-DSA for frequent signing unless hash-only security needed
