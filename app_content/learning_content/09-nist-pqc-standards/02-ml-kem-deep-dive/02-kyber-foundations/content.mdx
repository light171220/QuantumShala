import numpy as np

def lattice_introduction():
    """Understanding lattices as mathematical structures."""
    print("INTRODUCTION TO LATTICES")
    print("=" * 70)

    print("""
    WHAT IS A LATTICE?
    ━━━━━━━━━━━━━━━━━━━

    A lattice is a regular arrangement of points in n-dimensional space.

    ┌─────────────────────────────────────────────────────────────────┐
    │                                                                 │
    │  FORMAL DEFINITION:                                            │
    │                                                                 │
    │  Given linearly independent vectors b₁, b₂, ..., bₙ ∈ ℝⁿ      │
    │  (called a basis), the lattice L is:                           │
    │                                                                 │
    │     L = { Σ aᵢbᵢ : aᵢ ∈ ℤ }                                    │
    │                                                                 │
    │  All integer linear combinations of the basis vectors.         │
    │                                                                 │
    └─────────────────────────────────────────────────────────────────┘


    2D LATTICE EXAMPLE:
    ━━━━━━━━━━━━━━━━━━━━

    Basis vectors: b₁ = (1, 0), b₂ = (0, 1)

         ●   ●   ●   ●   ●
           ↑
         ●   ●   ●   ●   ●
           │ b₂
         ●   ●   ●───●   ●
                 └─→
         ●   ●   ●  b₁ ●   ●

         ●   ●   ●   ●   ●

    Every dot is a lattice point: a·b₁ + b·b₂ for integers a, b


    WHY LATTICES FOR CRYPTOGRAPHY?
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    Certain lattice problems are believed to be hard:

    1. SHORTEST VECTOR PROBLEM (SVP)
       Find the shortest non-zero vector in the lattice.

    2. CLOSEST VECTOR PROBLEM (CVP)
       Given a target point, find the closest lattice point.

    3. LEARNING WITH ERRORS (LWE)
       Distinguish noisy linear equations from random.

    These problems appear hard even for QUANTUM COMPUTERS!
    """)

lattice_introduction()

## 2. The Learning With Errors Problem

```python
def lwe_problem():
    """Understanding the LWE problem."""
    print("\nTHE LEARNING WITH ERRORS (LWE) PROBLEM")
    print("=" * 70)

    print("""
    LWE SETUP:
    ━━━━━━━━━━━

    Choose parameters:
    • n: dimension (security parameter)
    • q: modulus (prime number)
    • χ: error distribution (small values, e.g., Gaussian)


    LWE PROBLEM STATEMENT:
    ━━━━━━━━━━━━━━━━━━━━━━━

    ┌─────────────────────────────────────────────────────────────────┐
    │                                                                 │
    │  SECRET: s ∈ Zqⁿ (secret vector)                               │
    │                                                                 │
    │  GIVEN: Multiple samples (aᵢ, bᵢ) where:                       │
    │                                                                 │
    │         bᵢ = ⟨aᵢ, s⟩ + eᵢ  (mod q)                            │
    │                                                                 │
    │         aᵢ ← random from Zqⁿ                                   │
    │         eᵢ ← small error from χ                                │
    │                                                                 │
    │  GOAL: Find s                                                  │
    │                                                                 │
    └─────────────────────────────────────────────────────────────────┘


    WHY IS THIS HARD?
    ━━━━━━━━━━━━━━━━━━

    ┌─────────────────────────────────────────────────────────────────┐
    │                                                                 │
    │  WITHOUT ERRORS (linear algebra):                              │
    │                                                                 │
    │     b = As    →    s = A⁻¹b                                   │
    │                                                                 │
    │     EASY! Gaussian elimination solves it.                      │
    │                                                                 │
    │  ─────────────────────────────────────────────────────────────  │
    │                                                                 │
    │  WITH ERRORS:                                                  │
    │                                                                 │
    │     b = As + e    →    s = ???                                │
    │                                                                 │
    │     HARD! Small errors scramble the solution.                  │
    │     No efficient classical or quantum algorithm known!         │
    │                                                                 │
    └─────────────────────────────────────────────────────────────────┘


    DECISION VS SEARCH LWE:
    ━━━━━━━━━━━━━━━━━━━━━━━━

    Search-LWE: Given (A, b = As + e), find s
    Decision-LWE: Distinguish (A, As + e) from (A, random)

    These are equivalent in hardness (reductions exist).
    """)

    # Demonstrate LWE
    print("\n    CONCRETE EXAMPLE (small parameters for illustration):")
    print("    " + "─" * 58)

    n, q = 3, 23  # Small for demonstration
    np.random.seed(42)

    # Secret
    s = np.array([5, 12, 3])
    print(f"\n    Secret s = {s}")

    # Generate samples
    print("\n    LWE samples (a, b = <a,s> + e mod q):")
    for i in range(3):
        a = np.random.randint(0, q, n)
        e = np.random.randint(-1, 2)  # Small error
        b = (np.dot(a, s) + e) % q
        print(f"    a{i+1} = {a}, b{i+1} = {b}, error = {e}")

lwe_problem()
```

## 3. From LWE to Ring-LWE to Module-LWE

```python
def lwe_variants():
    """Evolution of LWE variants."""
    print("\nFROM LWE TO RING-LWE TO MODULE-LWE")
    print("=" * 70)

    print("""
    THE EFFICIENCY PROBLEM:
    ━━━━━━━━━━━━━━━━━━━━━━━━

    Standard LWE with dimension n requires:
    • Public key: O(n²) elements
    • Operations: O(n²) multiplications

    For n = 1024 (needed for security): TOO LARGE and SLOW!


    SOLUTION 1: RING-LWE (2010)
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    ┌─────────────────────────────────────────────────────────────────┐
    │                                                                 │
    │  Replace Zqⁿ with a polynomial ring:                           │
    │                                                                 │
    │     Rq = Zq[X] / (Xⁿ + 1)                                      │
    │                                                                 │
    │  Elements are polynomials of degree < n with coefficients mod q│
    │                                                                 │
    │  Example (n=4, q=17):                                          │
    │     a(X) = 3 + 5X + 2X² + 11X³                                 │
    │                                                                 │
    │  Benefits:                                                     │
    │  • Public key: O(n) elements                                   │
    │  • Operations: O(n log n) via FFT/NTT                          │
    │                                                                 │
    └─────────────────────────────────────────────────────────────────┘


    SOLUTION 2: MODULE-LWE (ML-KEM uses this)
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    ┌─────────────────────────────────────────────────────────────────┐
    │                                                                 │
    │  Generalize Ring-LWE with a module structure:                  │
    │                                                                 │
    │     Rqᵏ = k-dimensional vectors over Rq                        │
    │                                                                 │
    │  Matrix A is k×k of ring elements (polynomials)                │
    │                                                                 │
    │  ML-KEM parameters:                                            │
    │  • ML-KEM-512:  k = 2                                          │
    │  • ML-KEM-768:  k = 3                                          │
    │  • ML-KEM-1024: k = 4                                          │
    │                                                                 │
    │  Benefits:                                                     │
    │  • Flexible security levels (change k)                         │
    │  • Good performance (still uses ring structure)                │
    │  • Security between Ring-LWE and LWE                           │
    │                                                                 │
    └─────────────────────────────────────────────────────────────────┘


    COMPARISON:
    ━━━━━━━━━━━━

    ┌─────────────┬──────────────┬──────────────┬──────────────────┐
    │ Variant     │ Structure    │ Key Size     │ Security         │
    ├─────────────┼──────────────┼──────────────┼──────────────────┤
    │ LWE         │ None         │ O(n²)        │ Strongest        │
    │ Ring-LWE    │ Single ring  │ O(n)         │ Additional       │
    │             │              │              │ structure risks  │
    │ Module-LWE  │ k×k rings    │ O(k²n)       │ Balanced         │
    └─────────────┴──────────────┴──────────────┴──────────────────┘

    Module-LWE provides the best tradeoff for practical cryptography.
    """)

lwe_variants()
```

## 4. The Kyber/ML-KEM Construction

```python
def kyber_construction():
    """How Kyber is built from Module-LWE."""
    print("\nTHE KYBER/ML-KEM CONSTRUCTION")
    print("=" * 70)

    print("""
    BUILDING BLOCKS:
    ━━━━━━━━━━━━━━━━━

    Ring: Rq = Zq[X] / (X²⁵⁶ + 1), where q = 3329

    ┌─────────────────────────────────────────────────────────────────┐
    │                                                                 │
    │  Why these parameters?                                         │
    │                                                                 │
    │  n = 256: Power of 2 enables fast NTT                          │
    │  q = 3329: Prime, q ≡ 1 (mod 256) for NTT                      │
    │  X²⁵⁶ + 1: Cyclotomic polynomial, nice structure               │
    │                                                                 │
    └─────────────────────────────────────────────────────────────────┘


    KEY GENERATION (simplified):
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    ┌─────────────────────────────────────────────────────────────────┐
    │                                                                 │
    │  1. Generate random seed ρ                                     │
    │                                                                 │
    │  2. Expand ρ to get matrix A ∈ Rqᵏˣᵏ                           │
    │     (A is public, derived deterministically from ρ)            │
    │                                                                 │
    │  3. Sample secret s ∈ Rqᵏ with small coefficients              │
    │                                                                 │
    │  4. Sample error e ∈ Rqᵏ with small coefficients               │
    │                                                                 │
    │  5. Compute t = A·s + e                                        │
    │                                                                 │
    │  Public Key:  pk = (ρ, t)                                      │
    │  Secret Key:  sk = s                                           │
    │                                                                 │
    └─────────────────────────────────────────────────────────────────┘


    ENCAPSULATION (simplified):
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    ┌─────────────────────────────────────────────────────────────────┐
    │                                                                 │
    │  Input: Public key pk = (ρ, t)                                 │
    │                                                                 │
    │  1. Regenerate A from ρ                                        │
    │                                                                 │
    │  2. Sample random r ∈ Rqᵏ with small coefficients              │
    │                                                                 │
    │  3. Sample errors e₁ ∈ Rqᵏ, e₂ ∈ Rq                            │
    │                                                                 │
    │  4. Compute:                                                   │
    │     u = Aᵀ·r + e₁                                              │
    │     v = tᵀ·r + e₂ + encode(message)                           │
    │                                                                 │
    │  5. Ciphertext: c = (compress(u), compress(v))                 │
    │                                                                 │
    │  6. Shared secret: K = Hash(m || Hash(c))                      │
    │                                                                 │
    └─────────────────────────────────────────────────────────────────┘


    DECAPSULATION (simplified):
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    ┌─────────────────────────────────────────────────────────────────┐
    │                                                                 │
    │  Input: Secret key sk = s, Ciphertext c = (u, v)               │
    │                                                                 │
    │  1. Decompress u and v                                         │
    │                                                                 │
    │  2. Compute: m' = decode(v - sᵀ·u)                            │
    │                                                                 │
    │     Why this works:                                            │
    │     v - sᵀ·u = tᵀ·r + e₂ + m - sᵀ·(Aᵀ·r + e₁)                │
    │              = (A·s + e)ᵀ·r + e₂ + m - sᵀ·Aᵀ·r - sᵀ·e₁       │
    │              = sᵀ·Aᵀ·r + eᵀ·r + e₂ + m - sᵀ·Aᵀ·r - sᵀ·e₁     │
    │              = eᵀ·r + e₂ - sᵀ·e₁ + m                          │
    │              ≈ m  (errors are small!)                          │
    │                                                                 │
    │  3. Re-encapsulate with m' and verify c matches                │
    │                                                                 │
    │  4. K = Hash(m' || Hash(c))                                    │
    │                                                                 │
    └─────────────────────────────────────────────────────────────────┘
    """)

kyber_construction()
```

def security_analysis():
    """Analyzing ML-KEM's security."""
    print("\nSECURITY ANALYSIS")
    print("=" * 70)

    print("""
    SECURITY ASSUMPTIONS:
    ━━━━━━━━━━━━━━━━━━━━━━

    ML-KEM's security reduces to:

    1. MODULE-LWE HARDNESS
       ┌─────────────────────────────────────────────────────────────┐
       │ Given (A, As + e), cannot distinguish from random          │
       │                                                             │
       │ Best known attacks:                                         │
       │ • Classical: Lattice reduction (BKZ algorithm)              │
       │ • Quantum: Grover-enhanced lattice sieving                  │
       │                                                             │
       │ Neither provides efficient attack for chosen parameters     │
       └─────────────────────────────────────────────────────────────┘

    2. HASH FUNCTION SECURITY (SHA3/SHAKE)
       ┌─────────────────────────────────────────────────────────────┐
       │ Random oracle assumption in security proofs                 │
       │ SHA3 family believed quantum-resistant                      │
       └─────────────────────────────────────────────────────────────┘


    CONCRETE SECURITY ESTIMATES:
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    ┌─────────────────────────────────────────────────────────────────┐
    │ Parameter Set  │ Classical Security │ Quantum Security         │
    ├────────────────┼────────────────────┼──────────────────────────┤
    │ ML-KEM-512     │ ~118 bits          │ ~107 bits                │
    │ ML-KEM-768     │ ~183 bits          │ ~164 bits                │
    │ ML-KEM-1024    │ ~256 bits          │ ~230 bits                │
    └────────────────┴────────────────────┴──────────────────────────┘

    These exceed the target security categories (128, 192, 256 bits).


    POTENTIAL WEAKNESSES CONSIDERED:
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    1. STRUCTURAL ATTACKS
       • Ring/module structure might enable special attacks
       • Extensive cryptanalysis found no practical exploits
       • Module-LWE less structured than Ring-LWE

    2. IMPLEMENTATION ATTACKS
       • Side-channels: Constant-time implementations exist
       • Fault attacks: Verification step provides protection

    3. PARAMETER MARGIN
       • Parameters chosen with conservative margins
       • Some buffer against future cryptanalysis advances
    """)

security_analysis()

## 6. Number Theoretic Transform (NTT)

```python
def ntt_explanation():
    """The NTT optimization in ML-KEM."""
    print("\nNUMBER THEORETIC TRANSFORM (NTT)")
    print("=" * 70)

    print("""
    WHY NTT?
    ━━━━━━━━━

    Polynomial multiplication is the core operation in ML-KEM.

    ┌─────────────────────────────────────────────────────────────────┐
    │                                                                 │
    │  Naive multiplication: O(n²) operations                        │
    │                                                                 │
    │  With NTT: O(n log n) operations                               │
    │                                                                 │
    │  For n = 256: ~100x speedup!                                   │
    │                                                                 │
    └─────────────────────────────────────────────────────────────────┘


    WHAT IS NTT?
    ━━━━━━━━━━━━━

    NTT is the "integer version" of FFT (Fast Fourier Transform):

    ┌─────────────────────────────────────────────────────────────────┐
    │                                                                 │
    │  FFT: Uses complex roots of unity (e^(2πi/n))                  │
    │       Works over complex numbers                                │
    │                                                                 │
    │  NTT: Uses modular roots of unity (ω where ωⁿ ≡ 1 mod q)      │
    │       Works over integers mod q                                 │
    │                                                                 │
    │  Same algorithm structure, different number system!            │
    │                                                                 │
    └─────────────────────────────────────────────────────────────────┘


    NTT FOR MULTIPLICATION:
    ━━━━━━━━━━━━━━━━━━━━━━━━

    To multiply polynomials a(X) · b(X):

    ┌─────────────────────────────────────────────────────────────────┐
    │                                                                 │
    │  1. Transform: â = NTT(a), b̂ = NTT(b)                          │
    │     O(n log n) each                                            │
    │                                                                 │
    │  2. Point-wise multiply: ĉᵢ = âᵢ · b̂ᵢ                          │
    │     O(n) operations                                            │
    │                                                                 │
    │  3. Inverse transform: c = NTT⁻¹(ĉ)                            │
    │     O(n log n)                                                 │
    │                                                                 │
    │  Total: O(n log n) instead of O(n²)                            │
    │                                                                 │
    └─────────────────────────────────────────────────────────────────┘


    ML-KEM NTT DETAILS:
    ━━━━━━━━━━━━━━━━━━━━

    • n = 256, q = 3329
    • Primitive 512th root of unity: ω = 17
    • Uses specialized "negacyclic" NTT for X²⁵⁶ + 1
    • Most operations stay in NTT domain for efficiency
    """)

ntt_explanation()
```

## Summary

ML-KEM's mathematical foundation consists of:

1. **Lattices**: Regular point structures with hard computational problems
2. **LWE Problem**: Adding small errors makes linear equations hard to solve
3. **Module-LWE**: Balanced structure between efficiency and security
4. **Kyber Construction**: Encryption scheme built on Module-LWE
5. **NTT Optimization**: Fast polynomial multiplication for efficiency

## Key Takeaways

- ML-KEM security relies on the hardness of Module-LWE
- No efficient classical or quantum algorithm breaks Module-LWE
- The ring structure enables efficient implementation via NTT
- Security proofs reduce ML-KEM to Module-LWE hardness
- Parameters chosen with conservative margins against future attacks
