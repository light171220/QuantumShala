## 1. Common Parameters Across All Variants

```python
def common_parameters():
    """Parameters shared by all ML-KEM variants."""
    print("COMMON PARAMETERS ACROSS ALL ML-KEM VARIANTS")
    print("=" * 70)

    print("""
    FIXED PARAMETERS:
    ━━━━━━━━━━━━━━━━━━

    ┌─────────────────────────────────────────────────────────────────┐
    │                                                                 │
    │  n = 256    Polynomial degree                                  │
    │             (coefficients per polynomial)                      │
    │                                                                 │
    │  q = 3329   Modulus for coefficient arithmetic                │
    │             (prime, q ≡ 1 mod 256)                             │
    │                                                                 │
    │  Ring: Rq = Zq[X] / (X²⁵⁶ + 1)                                 │
    │                                                                 │
    └─────────────────────────────────────────────────────────────────┘


    WHY THESE VALUES?
    ━━━━━━━━━━━━━━━━━━

    n = 256:
    • Power of 2 enables efficient NTT
    • Large enough for security
    • Small enough for efficiency

    q = 3329:
    • Prime number (required for field arithmetic)
    • q ≡ 1 (mod 512) enables 512-point NTT
    • q ≡ 1 (mod 256) enables negacyclic NTT
    • Small enough for efficient computation
    • Large enough to avoid decryption failures

    X²⁵⁶ + 1:
    • Cyclotomic polynomial
    • Enables negacyclic NTT
    • Ring structure provides security and efficiency


    SHARED CONSTANTS:
    ━━━━━━━━━━━━━━━━━━

    ┌─────────────────────────────────────────────────────────────────┐
    │ Constant           │ Value  │ Purpose                          │
    ├────────────────────┼────────┼──────────────────────────────────┤
    │ SHAKE128           │ XOF    │ Matrix A expansion               │
    │ SHAKE256           │ XOF    │ Hash functions G, H, J           │
    │ SHA3-256           │ Hash   │ Key derivation                   │
    │ SHA3-512           │ Hash   │ Internal hashing                 │
    │ Shared secret size │ 32 B   │ Output K in all variants         │
    └────────────────────┴────────┴──────────────────────────────────┘
    """)

common_parameters()
```

```python
    SECURITY ESTIMATE:
    ━━━━━━━━━━━━━━━━━━━

    Quantum:   ~107 bits

    Meets NIST Category 1 (≥ AES-128 security)
    """)
```

## Summary

ML-KEM provides three parameter sets targeting different security levels:

1. **ML-KEM-512**: Category 1 (AES-128), k=2, smallest sizes
2. **ML-KEM-768**: Category 3 (AES-192), k=3, recommended default
3. **ML-KEM-1024**: Category 5 (AES-256), k=4, maximum security

## Key Takeaways

- All variants share n=256, q=3329, and produce 32-byte shared secrets
- Size increases roughly linearly with k (module dimension)
- Performance differences are small in absolute terms
- ML-KEM-768 is the recommended default for most applications
- Choose based on data sensitivity, protection lifetime, and constraints
