# PQC Interoperability Overview

Deploying post-quantum cryptography requires careful consideration of interoperability - ensuring new PQC systems work with existing infrastructure while providing a smooth transition path.

## Hybrid Approach Overview

### Why Hybrid?

```
┌─────────────────────────────────────────────────────────────────┐
│              The Case for Hybrid Cryptography                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  HYBRID = Classical + PQC combined                              │
│                                                                  │
│  Security Goal:                                                  │
│  ┌────────────────────────────────────────────────────────────┐│
│  │  System is secure if EITHER algorithm remains unbroken     ││
│  │                                                             ││
│  │  If classical broken → PQC protects                        ││
│  │  If PQC broken      → Classical protects                   ││
│  └────────────────────────────────────────────────────────────┘│
│                                                                  │
│  WHY NOT PURE PQC?                                              │
│  1. PQC algorithms are newer, less analyzed                    │
│  2. Classical algorithms have decades of analysis              │
│  3. Unforeseen cryptanalytic breaks possible                   │
│  4. Defense in depth is prudent                                │
│                                                                  │
│  WHY NOT STAY CLASSICAL?                                        │
│  1. Quantum computers will break RSA/ECDSA                     │
│  2. "Harvest now, decrypt later" threat                        │
│  3. Some data needs decades of protection                      │
│  4. Transition takes years - start now                         │
│                                                                  │
│  HYBRID MODES                                                   │
│  ├─ Key Exchange: X25519 + ML-KEM                              │
│  ├─ Signatures: ECDSA + ML-DSA (composite)                     │
│  └─ Encryption: AES(shared_secret_1 || shared_secret_2)       │
└─────────────────────────────────────────────────────────────────┘
```

### Hybrid Implementation Patterns

```python
"""
Hybrid cryptography patterns
"""

from dataclasses import dataclass
from typing import Tuple
import hashlib

@dataclass
class HybridKeyPair:
    """Combined classical + PQC key pair."""
    classical_public: bytes
    classical_private: bytes
    pqc_public: bytes
    pqc_private: bytes

class HybridKEM:
    """
    Hybrid Key Encapsulation combining X25519 + ML-KEM.

    Security: Secure if either X25519 OR ML-KEM is secure.
    """

    def __init__(self, classical_kem, pqc_kem):
        self.classical = classical_kem
        self.pqc = pqc_kem

    def keygen(self) -> HybridKeyPair:
        """Generate hybrid key pair."""
        c_pk, c_sk = self.classical.keygen()
        p_pk, p_sk = self.pqc.keygen()

        return HybridKeyPair(
            classical_public=c_pk,
            classical_private=c_sk,
            pqc_public=p_pk,
            pqc_private=p_sk
        )

    def encapsulate(self, hybrid_pk: HybridKeyPair) -> Tuple[bytes, bytes]:
        """
        Encapsulate to produce ciphertext and shared secret.

        Returns:
            (combined_ciphertext, combined_shared_secret)
        """
        # Encapsulate with both
        c_ct, c_ss = self.classical.encapsulate(hybrid_pk.classical_public)
        p_ct, p_ss = self.pqc.encapsulate(hybrid_pk.pqc_public)

        # Combine ciphertexts
        combined_ct = c_ct + p_ct

        # Combine shared secrets using KDF
        combined_ss = self._combine_secrets(c_ss, p_ss)

        return combined_ct, combined_ss

    def decapsulate(self, hybrid_sk: HybridKeyPair,
                    combined_ct: bytes) -> bytes:
        """
        Decapsulate to recover shared secret.
        """
        # Split ciphertext
        c_ct_len = len(self.classical.ciphertext_size)
        c_ct = combined_ct[:c_ct_len]
        p_ct = combined_ct[c_ct_len:]

        # Decapsulate both
        c_ss = self.classical.decapsulate(hybrid_sk.classical_private, c_ct)
        p_ss = self.pqc.decapsulate(hybrid_sk.pqc_private, p_ct)

        # Combine shared secrets
        return self._combine_secrets(c_ss, p_ss)

    def _combine_secrets(self, ss1: bytes, ss2: bytes) -> bytes:
        """
        Combine shared secrets using HKDF.

        Important: Concatenation must be in consistent order.
        """
        # Using simple SHA-256 for illustration
        # In production, use HKDF-SHA256
        combined = hashlib.sha256(ss1 + ss2).digest()
        return combined

# Example structure
print("Hybrid KEM Structure")
print("=" * 50)
print("""
Encapsulation:
  Classical CT (32 bytes) + PQC CT (1088 bytes) = Combined CT
  Classical SS + PQC SS → HKDF → Combined SS (32 bytes)

Decapsulation:
  Split Combined CT → Classical CT + PQC CT
  Decapsulate each → Classical SS + PQC SS
  Combine → Same Combined SS
""")
```

## Transition Strategies

│  ├─ Test in non-production environments                        │

## Summary

PQC interoperability requires:

1. **Understanding Challenges**: Size, performance, and compatibility
2. **Hybrid Approach**: Combine classical and PQC for defense in depth
3. **Phased Migration**: Gradual transition with fallback options
4. **Standards Adoption**: Follow NIST, IETF, and industry standards
5. **Testing**: Extensive interoperability testing between implementations

The transition to PQC is a multi-year process requiring careful planning and coordination.
