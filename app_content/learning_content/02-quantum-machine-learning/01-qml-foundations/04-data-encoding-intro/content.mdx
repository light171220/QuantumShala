## 1. Why Data Encoding Matters

Before quantum circuits can process classical data, it must be encoded into quantum states.

<Definition term="Quantum Data Encoding">
The process of mapping classical data (numbers, vectors, images) into quantum states that can be manipulated by quantum gates. Also called "quantum embedding" or "feature mapping."
</Definition>

```python

print()

considerations = {
    "Expressivity": "Can the encoding represent the data structure?",
    "Efficiency": "How many qubits and gates are needed?",
    "Trainability": "Does it lead to barren plateaus?",
    "Inductive Bias": "Does it match the problem structure?"
}

print("Key Considerations:")
print("-" * 60)
for name, question in considerations.items():
    print(f"  {name}: {question}")
```

<Warning>
The choice of data encoding fundamentally affects what a quantum model can learn. A poor encoding can make simple problems impossible to solve, while a good encoding can make complex problems tractable.
</Warning>

## 2. Basis Encoding

The simplest encoding: store classical bits directly in computational basis states.

<Definition term="Basis Encoding">
Encoding classical binary strings directly into computational basis states. For a binary string b = b₁b₂...bₙ, the encoded state is |b₁b₂...bₙ⟩.
</Definition>

```python
import numpy as np

def basis_encode(integer, n_qubits):
    """
    Encode an integer into a computational basis state.

    Args:
        integer: Non-negative integer to encode
        n_qubits: Number of qubits (determines max value 2^n - 1)

    Returns:
        Quantum state vector
    """
    if integer >= 2 ** n_qubits:
        raise ValueError(f"Integer {integer} too large for {n_qubits} qubits")

    # Create state vector with 1 at position 'integer'
    state = np.zeros(2 ** n_qubits)
    state[integer] = 1.0

    return state

def basis_encode_binary(binary_string):
    """
    Encode a binary string into a computational basis state.

    Args:
        binary_string: String of 0s and 1s

    Returns:
        Quantum state vector
    """
    n_qubits = len(binary_string)
    integer = int(binary_string, 2)
    return basis_encode(integer, n_qubits)

# Examples
print("Basis Encoding Examples:")
print("=" * 50)

# Encode integer 5 with 4 qubits (binary: 0101)
state = basis_encode(5, 4)
print(f"Integer 5 with 4 qubits: |0101⟩")
print(f"State vector: {state}")
print(f"Non-zero at index 5: {np.where(state != 0)[0]}")

# Encode binary string
state2 = basis_encode_binary("110")
print(f"\nBinary '110': |110⟩")
print(f"State vector: {state2}")
```

## 3. Amplitude Encoding

Encode data as amplitudes of a quantum state, achieving exponential compression.

<Definition term="Amplitude Encoding">
Encoding a normalized vector x ∈ ℝᴺ as amplitudes of a log₂(N)-qubit quantum state: |ψ(x)⟩ = Σᵢ xᵢ|i⟩ where Σᵢ|xᵢ|² = 1.
</Definition>

<Formula>
$$|\psi(x)\rangle = \sum_{i=0}^{N-1} x_i |i\rangle, \quad \text{where } \sum_{i=0}^{N-1} |x_i|^2 = 1$$
</Formula>

```python
import numpy as np

def amplitude_encode(data):
    """
    Encode a data vector as quantum amplitudes.

    Args:
        data: Array of values (will be normalized)

    Returns:
        Normalized quantum state vector, number of qubits needed
    """
    # Pad to power of 2 if necessary
    n = len(data)
    n_qubits = int(np.ceil(np.log2(n)))
    padded_size = 2 ** n_qubits

    padded_data = np.zeros(padded_size)
    padded_data[:n] = data

    # Normalize
    norm = np.linalg.norm(padded_data)
    if norm == 0:
        raise ValueError("Cannot encode zero vector")

    state = padded_data / norm

    return state, n_qubits

# Example: Encode a 4D vector
data = np.array([1.0, 2.0, 3.0, 4.0])
state, n_qubits = amplitude_encode(data)

print("Amplitude Encoding Example:")
print("=" * 50)
print(f"Original data: {data}")
print(f"Normalized state: {np.round(state, 4)}")
print(f"Qubits needed: {n_qubits}")
print(f"Compression: {len(data)} values → {n_qubits} qubits")

# Verify normalization
print(f"\nNormalization check: |ψ|² = {np.sum(state**2):.6f}")

# Larger example
big_data = np.random.randn(1024)
state_big, n_big = amplitude_encode(big_data)
print(f"\n1024 values encoded in just {n_big} qubits!")
```

<Tip>
Amplitude encoding achieves exponential compression: N data points encoded in log₂(N) qubits. However, preparing an arbitrary amplitude-encoded state generally requires O(N) gates, potentially negating the space advantage.
</Tip>

## 4. Angle Encoding

Encode data values as rotation angles of quantum gates.

<Definition term="Angle Encoding">
Encoding data values as parameters of rotation gates. Each feature xᵢ becomes the angle of a rotation gate applied to a qubit.
</Definition>

```python
import numpy as np

def angle_encode(data, encoding_type='ry'):
    """
    Encode data as rotation angles.

    Args:
        data: Array of values
        encoding_type: 'ry', 'rz', or 'rx'

    Returns:
        List of quantum states (one qubit per data point)
    """
    states = []

    for x in data:
        # Scale to [0, π] for meaningful rotation
        theta = x  # Assume already in appropriate range

        if encoding_type == 'ry':
            # Ry(θ)|0⟩ = cos(θ/2)|0⟩ + sin(θ/2)|1⟩
            state = np.array([np.cos(theta/2), np.sin(theta/2)])
        elif encoding_type == 'rz':
            # Rz(θ)|+⟩ encodes phase
            state = np.array([1, np.exp(1j * theta)]) / np.sqrt(2)
        elif encoding_type == 'rx':
            state = np.array([np.cos(theta/2), -1j * np.sin(theta/2)])

        states.append(state)

    return states

# Example
data = np.array([0, np.pi/4, np.pi/2, np.pi])
states = angle_encode(data, 'ry')

print("Angle Encoding Example (Ry):")
print("=" * 50)
print("For Ry(θ)|0⟩ = cos(θ/2)|0⟩ + sin(θ/2)|1⟩")
print()

for i, (x, state) in enumerate(zip(data, states)):
    print(f"x = {x:.4f}: |ψ⟩ = {np.round(state, 4)}")
    prob_0 = np.abs(state[0])**2
    print(f"          P(|0⟩) = {prob_0:.4f}, P(|1⟩) = {1-prob_0:.4f}")
```

### Feature Maps with Angle Encoding

```python
import numpy as np

def feature_map_circuit(x, reps=1):
    """
    Create a feature map using angle encoding with entanglement.

    This is a conceptual representation - actual implementation
    would use a quantum framework.

    Args:
        x: Feature vector
        reps: Number of repetitions (layers)

    Returns:
        Description of circuit structure
    """
    n_features = len(x)

    circuit_desc = []

    for rep in range(reps):
        # Rotation layer
        circuit_desc.append(f"Layer {rep + 1}:")
        circuit_desc.append(f"  Ry(x) on all qubits: {[f'Ry({v:.2f})' for v in x]}")

        # Entangling layer (if more than 1 qubit)
        if n_features > 1:
            entangle = [f"CNOT({i},{(i+1) % n_features})" for i in range(n_features)]
            circuit_desc.append(f"  Entanglement: {entangle}")

    return circuit_desc

# Example feature map
x = np.array([0.5, 1.2, 0.8])
circuit = feature_map_circuit(x, reps=2)

print("Feature Map Circuit Structure:")
print("=" * 50)
for line in circuit:
    print(line)

print("\n✓ This creates a 3-qubit state encoding 3 features")
print("  with entanglement for expressivity.")
```

## 5. Comparing Encoding Methods

```python
import numpy as np

def compare_encodings():
    """Compare different encoding strategies."""

    comparison = {

            "expressivity": "High",
            "use_case": "Dense data, quantum ML algorithms"

        "Angle Encoding": {
            "qubits_for_N": "N qubits for N features",
            "gates": "O(N × layers)",
            "data_type": "Continuous features",

            "use_case": "VQC, quantum kernels"

compare_encodings()
```

<Tip>
For near-term QML, angle encoding is most common because:
1. Simple to implement with rotation gates
2. Number of qubits matches features (manageable for small datasets)
3. Easy to add entanglement for expressivity
4. Compatible with variational training
</Tip>

## 6. Practical Implementation

Let's implement a complete encoding class:

```python
import numpy as np

class QuantumEncoder:
    """A class implementing various quantum encoding strategies."""

    def __init__(self, method='angle', n_qubits=None):
        self.method = method
        self.n_qubits = n_qubits

    def encode(self, data):
        """Encode classical data into quantum state."""
        if self.method == 'basis':
            return self._basis_encode(data)
        elif self.method == 'amplitude':
            return self._amplitude_encode(data)
        elif self.method == 'angle':
            return self._angle_encode(data)
        else:
            raise ValueError(f"Unknown method: {self.method}")

    def _basis_encode(self, integer):
        """Basis encoding for integers."""
        n = self.n_qubits or int(np.ceil(np.log2(integer + 1)))
        state = np.zeros(2 ** n)
        state[int(integer)] = 1.0
        return state

    def _amplitude_encode(self, data):
        """Amplitude encoding for vectors."""
        data = np.array(data)
        n = len(data)
        n_qubits = int(np.ceil(np.log2(n)))

        # Pad and normalize
        padded = np.zeros(2 ** n_qubits)
        padded[:n] = data
        return padded / np.linalg.norm(padded)

    def _angle_encode(self, data):
        """Angle encoding using Ry rotations."""
        data = np.array(data)
        n = len(data)

        # Create tensor product state
        state = np.array([1.0])
        for x in data:
            qubit = np.array([np.cos(x/2), np.sin(x/2)])
            state = np.kron(state, qubit)

        return state

    def info(self, data):
        """Get encoding information."""
        data = np.array(data) if hasattr(data, '__len__') else np.array([data])
        n = len(data) if hasattr(data, '__len__') else 1

        info = {
            'basis': {
                'qubits': int(np.ceil(np.log2(max(data) + 1))) if self.method == 'basis' else 'N/A',
                'gates': 'O(log N)'
            },
            'amplitude': {
                'qubits': int(np.ceil(np.log2(n))),
                'gates': f'O({n})'
            },
            'angle': {
                'qubits': n,
                'gates': f'O({n})'
            }
        }
        return info[self.method]

# Demo
print("QuantumEncoder Demo:")
print("=" * 50)

# Amplitude encoding
encoder_amp = QuantumEncoder('amplitude')
data = [1, 2, 3, 4]
state = encoder_amp.encode(data)
print(f"\nAmplitude encoding of {data}:")
print(f"State: {np.round(state, 4)}")
print(f"Info: {encoder_amp.info(data)}")

# Angle encoding
encoder_angle = QuantumEncoder('angle')
data = [np.pi/4, np.pi/2]
state = encoder_angle.encode(data)
print(f"\nAngle encoding of {data}:")
print(f"State (4D): {np.round(state, 4)}")
print(f"Info: {encoder_angle.info(data)}")
```

## Summary

In this lesson, you learned about quantum data encoding:

1. **Why Encoding Matters**: The bridge between classical data and quantum processing
2. **Basis Encoding**: Direct binary representation, simple but limited
3. **Amplitude Encoding**: Exponential compression, but costly preparation
4. **Angle Encoding**: Feature values as rotation angles, practical for NISQ
5. **Trade-offs**: Qubits, gates, expressivity, and use cases
6. **Implementation**: Practical encoding class with multiple methods

<Tip>
For most near-term QML applications, start with angle encoding. It's intuitive, easy to implement, and scales naturally with feature dimension. Save amplitude encoding for when you have efficient preparation methods.
</Tip>
