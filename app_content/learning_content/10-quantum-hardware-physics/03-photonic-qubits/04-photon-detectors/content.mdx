## 1. Introduction to Single-Photon Detection

Single-photon detection is essential for photonic quantum computing, enabling measurement-based operations and heralding signals.

<Definition term="Single-Photon Detector">
A single-photon detector is a device that produces a measurable electrical signal upon absorbing a single photon, with minimal false triggers (dark counts) and high efficiency.
</Definition>

### Key Performance Metrics

| Metric | Definition | Importance |
|--------|------------|------------|
| Detection efficiency | P(click given photon) | Higher is better |
| Dark count rate | False clicks per second | Lower is better |
| Timing jitter | Uncertainty in detection time | Lower is better |
| Dead time | Recovery time after detection | Lower is better |
| Maximum count rate | Clicks per second capacity | Higher is better |

## 2. Single-Photon Avalanche Diodes (SPADs)

SPADs are semiconductor devices that use avalanche multiplication to detect single photons.

### Operating Principle

A SPAD operates in **Geiger mode** - biased above the breakdown voltage:

<Formula>
$$V_{bias} = V_{breakdown} + V_{excess}$$
</Formula>

When a photon creates an electron-hole pair, the carriers trigger an avalanche of impact ionization, producing a macroscopic current.

<Definition term="Geiger Mode">
In Geiger mode, a photodiode is biased above breakdown so that a single photon-generated carrier triggers a self-sustaining avalanche that must be actively quenched.
</Definition>

### SPAD Structure

Typical SPAD layers:
1. **Absorption region**: Where photons create carriers (Si, InGaAs)
2. **Multiplication region**: High-field region for avalanche
3. **Guard ring**: Prevents edge breakdown

### Silicon SPADs

| Parameter | Typical Value |
|-----------|---------------|
| Wavelength range | 400-1000 nm |
| Peak efficiency | 50-70% at 700 nm |
| Dark count rate | 25-1000 Hz |
| Timing jitter | 50-350 ps |
| Dead time | 20-100 ns |

```python
import numpy as np
import matplotlib.pyplot as plt

def spad_efficiency(wavelength_nm: np.ndarray, peak_wavelength: float = 700,
                    peak_efficiency: float = 0.6) -> np.ndarray:
    """
    Model SPAD detection efficiency vs wavelength.

    Args:
        wavelength_nm: Wavelength array in nm
        peak_wavelength: Wavelength of peak efficiency
        peak_efficiency: Maximum efficiency

    Returns:
        Detection efficiency array
    """
    # Simple model: absorption in Si falls off at long wavelengths
    # and reflection/transmission losses at short wavelengths
    efficiency = peak_efficiency * np.exp(-((wavelength_nm - peak_wavelength)**2) / (2 * 150**2))
    # Silicon bandgap cutoff around 1100 nm
    efficiency *= 1 / (1 + np.exp((wavelength_nm - 1050) / 30))
    return np.clip(efficiency, 0, 1)

# Plot SPAD efficiency curve
wavelengths = np.linspace(400, 1100, 200)
efficiency = spad_efficiency(wavelengths)

plt.figure(figsize=(10, 5))
plt.plot(wavelengths, efficiency * 100, 'b-', linewidth=2)
plt.xlabel('Wavelength (nm)')
plt.ylabel('Detection Efficiency (%)')
plt.title('Silicon SPAD Detection Efficiency')
plt.grid(True)
plt.axvline(x=850, color='r', linestyle='--', label='Typical QKD (850 nm)')
plt.legend()
plt.show()
```

### InGaAs SPADs for Telecom

For 1310/1550 nm telecom wavelengths, InGaAs/InP SPADs are used:

| Parameter | Typical Value |
|-----------|---------------|
| Wavelength range | 900-1700 nm |
| Efficiency | 10-25% |
| Dark count rate | 1-50 kHz (gated) |
| Afterpulsing | Significant issue |

<Warning>
InGaAs SPADs have higher dark counts and afterpulsing compared to silicon. They typically require gated operation or specialized quenching to mitigate these issues.
</Warning>

## 3. Superconducting Nanowire Single-Photon Detectors (SNSPDs)

SNSPDs offer superior performance for demanding quantum computing applications.

<Definition term="SNSPD">
An SNSPD is a superconducting thin-film device that detects single photons when absorbed energy creates a resistive hotspot that disrupts the superconducting state, generating a voltage pulse.
</Definition>

### Operating Principle

1. A thin superconducting nanowire (NbN, NbTiN, WSi) carries a bias current just below critical
2. Photon absorption creates a localized normal (resistive) region
3. Current diverts around hotspot, forming a resistive belt
4. Voltage pulse detected by readout electronics
5. Hotspot cools, superconductivity recovers

<Formula>
$$I_{bias} \approx 0.8-0.95 \times I_c$$
</Formula>

### SNSPD Performance

| Parameter | Typical Value | Best Demonstrated |
|-----------|---------------|-------------------|
| System efficiency | 80-95% | >98% |
| Dark count rate | <10 Hz | <0.1 Hz |
| Timing jitter | 15-50 ps | 3 ps |
| Dead time | 5-50 ns | <1 ns |
| Operating temp | 0.8-4 K | Up to 4 K |

```python
import numpy as np
import matplotlib.pyplot as plt

def snspd_detection_probability(energy_eV: float, gap_energy_eV: float = 0.001,
                                 bias_ratio: float = 0.9) -> float:
    """
    Model SNSPD detection probability.

    Args:
        energy_eV: Photon energy in eV
        gap_energy_eV: Superconducting gap energy (~1 meV for NbN)
        bias_ratio: I_bias / I_critical

    Returns:
        Detection probability
    """
    # Simplified model: detection requires breaking Cooper pairs
    # and creating sufficient quasiparticles
    if energy_eV < gap_energy_eV:
        return 0.0

    # Higher energy and higher bias both improve detection
    n_quasiparticles = energy_eV / gap_energy_eV
    threshold = (1 - bias_ratio) * 100  # Approximate threshold

    if n_quasiparticles > threshold:
        return 1.0 - np.exp(-(n_quasiparticles - threshold) / 10)
    else:
        return 0.5 * (n_quasiparticles / threshold) ** 2

# Calculate efficiency vs wavelength
wavelengths_nm = np.linspace(400, 2000, 200)
energies_eV = 1240 / wavelengths_nm  # E = hc/lambda

efficiencies = [snspd_detection_probability(E, 0.001, 0.9) for E in energies_eV]

plt.figure(figsize=(10, 5))
plt.plot(wavelengths_nm, np.array(efficiencies) * 100, 'b-', linewidth=2)
plt.xlabel('Wavelength (nm)')
plt.ylabel('Detection Efficiency (%)')
plt.title('SNSPD Detection Efficiency (Simplified Model)')
plt.grid(True)
plt.axvline(x=1550, color='r', linestyle='--', label='Telecom C-band')
plt.legend()
plt.show()
```

### SNSPD Design Considerations

**Nanowire geometry affects performance:**
- **Width**: Narrower = higher efficiency but lower critical current
- **Thickness**: Thinner = higher efficiency but lower yield
- **Fill factor**: Higher = better absorption
- **Material**: NbN (higher Tc) vs WSi (better IR response)

## 4. Detector Comparison

| Property | Si SPAD | InGaAs SPAD | SNSPD |
|----------|---------|-------------|-------|
| Best wavelength | 700 nm | 1550 nm | Broadband |
| Max efficiency | 70% | 25% | >98% |
| Dark counts | 25 Hz | 10 kHz | <10 Hz |
| Jitter | 50 ps | 100 ps | 15 ps |
| Temperature | Room temp | -40 to 0 C | 0.8-4 K |
| Cost | Low | Medium | High |

<Tip>
For photonic quantum computing requiring high efficiency and low noise, SNSPDs are the detector of choice despite the cryogenic requirements. The overhead is justified by dramatically improved gate success rates.
</Tip>

## 5. Photon Number Resolution

Standard detectors are "click" detectors - they cannot distinguish 1 photon from 2 or more. Photon-number-resolving (PNR) detectors provide this capability.

<Definition term="Photon Number Resolution">
A photon-number-resolving detector can distinguish the number of photons in an incident pulse, producing different output signals for n=0, 1, 2, 3... photons.
</Definition>

### Approaches to PNR Detection

#### 1. Transition Edge Sensors (TES)
Operate at the superconducting transition temperature where resistance is highly sensitive to energy deposition.

<Formula>
$$\Delta T \propto n \cdot E_{photon}$$
</Formula>

- **Resolution**: Can resolve 0-20+ photons
- **Efficiency**: >95%
- **Timing**: Poor (~100 ns resolution)
- **Temperature**: ~100 mK

#### 2. SNSPD Arrays
Multiple SNSPDs with time-division or spatial multiplexing:

```python
import numpy as np
from scipy.special import comb

def snspd_array_pnr(n_pixels: int, n_photons: int,
                    single_pixel_efficiency: float) -> dict:
    """
    Calculate photon number discrimination with SNSPD array.

    Args:
        n_pixels: Number of detector pixels
        n_photons: Number of incident photons
        single_pixel_efficiency: Detection efficiency per pixel

    Returns:
        Dictionary of probabilities for each number of clicks
    """
    eta = single_pixel_efficiency
    N = n_pixels
    n = n_photons

    click_probs = {}
    for k in range(min(n, N) + 1):
        # Probability of exactly k clicks given n photons uniformly distributed
        # among N pixels
        if n <= N:
            # Each photon hits different pixel with high probability
            prob = comb(n, k, exact=True) * (eta ** k) * ((1 - eta) ** (n - k))
        else:
            # Multiple photons per pixel - more complex
            # Simplified: assume independent pixel responses
            prob = comb(N, k, exact=True) * (1 - (1 - eta/N) ** n) ** k * \
                   ((1 - eta/N) ** n) ** (N - k)
        click_probs[k] = prob

    return click_probs

# Test with 4-pixel array
for n_photons in range(5):
    probs = snspd_array_pnr(4, n_photons, 0.9)
    print(f"{n_photons} photons: {probs}")
```

## 6. Dark Counts and Noise

Dark counts are false detection events that limit performance.

### Sources of Dark Counts

| Source | SPAD | SNSPD |
|--------|------|-------|
| Thermal generation | Yes | No (cryogenic) |
| Tunneling | Yes | No |
| Background photons | Yes | Yes |
| Cosmic rays | Rare | Rare |
| Electrical noise | Yes | Yes |

### Dark Count Rate Impact

<Formula>
$$SNR = \frac{\eta \cdot R_{photon}}{\sqrt{\eta \cdot R_{photon} + R_{dark}}}$$
</Formula>

For heralded photon detection:

<Formula>
$$\text{Fidelity reduction} \approx \frac{R_{dark} \cdot \tau_{window}}{R_{herald}}$$
</Formula>

```python
import numpy as np

def detection_fidelity(herald_rate_Hz: float, dark_count_rate_Hz: float,
                       coincidence_window_ns: float) -> float:
    """
    Calculate fidelity reduction due to dark counts in heralded detection.

    Args:
        herald_rate_Hz: Rate of herald events
        dark_count_rate_Hz: Dark count rate of signal detector
        coincidence_window_ns: Coincidence time window in nanoseconds

    Returns:
        Approximate fidelity (1 = perfect, 0 = no correlation)
    """
    window_s = coincidence_window_ns * 1e-9
    accidental_rate = dark_count_rate_Hz * window_s

    # True coincidences vs accidentals
    fidelity = herald_rate_Hz / (herald_rate_Hz + accidental_rate * herald_rate_Hz)

    return fidelity

# Compare SPAD vs SNSPD for heralded detection
for detector, dcr in [("Si SPAD", 100), ("SNSPD", 1)]:
    f = detection_fidelity(1e5, dcr, 1)
    print(f"{detector} (DCR={dcr} Hz): Fidelity = {f:.6f}")
```

## 7. Timing and Synchronization

Precise timing is critical for photon interference experiments.

### Timing Jitter Effects

<Formula>
$$V_{HOM} = V_{intrinsic} \cdot e^{-(\sigma_t / \tau_{coherence})^2}$$
</Formula>

where $\sigma_t$ is the combined timing jitter and $\tau_{coherence}$ is the photon coherence time.

### Time Tagging

Modern systems use time-to-digital converters (TDCs) for photon arrival time recording:
- Resolution: 1-10 ps
- Dead time: 1-50 ns
- Multiple channels for coincidence detection

## 8. Detector Arrays and Imaging

## 9. Emerging Technologies

### Room-Temperature Single-Photon Detectors
- Graphene-based detectors

### Integrated Detectors
- Waveguide-integrated SNSPDs
- On-chip SPAD arrays

<Warning>
While room-temperature single-photon detectors are being developed, they currently cannot match SNSPD performance. For near-term photonic quantum computing, cryogenic detection remains necessary for high-fidelity operations.
</Warning>

## 10. Summary

Single-photon detectors are crucial enabling technology for photonic quantum computing:

1. **SPADs** offer room-temperature operation but limited efficiency and higher dark counts
2. **SNSPDs** provide near-unity efficiency and minimal noise but require cryogenics
3. **Timing jitter** affects multi-photon interference visibility
4. **Photon-number resolution** can be achieved with TES or detector arrays
5. **Dark counts** fundamentally limit detection fidelity

The choice of detector depends on application requirements: SNSPDs for high-performance quantum computing, SPADs for cost-sensitive or room-temperature applications.
