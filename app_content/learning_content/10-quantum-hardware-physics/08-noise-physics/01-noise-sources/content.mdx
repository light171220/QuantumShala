## 1. Introduction: The Noise Landscape

Quantum computers operate at the edge of physical possibility, where even the tiniest environmental disturbances can destroy the delicate quantum states that encode information. Understanding noise is not just academic - it is essential for building practical quantum computers.

<Definition term="Quantum Noise">
Quantum noise refers to any unwanted interaction between a qubit and its environment that causes loss of quantum information through energy relaxation (T1 processes), phase randomization (T2 processes), or both. Noise fundamentally limits the depth of quantum circuits that can be executed reliably.
</Definition>

### Why Noise Matters

The error rate per gate operation is directly related to noise levels:

<Formula>
$$\epsilon_{\text{gate}} \approx \frac{t_{\text{gate}}}{T_1} + \frac{t_{\text{gate}}}{T_2} + \epsilon_{\text{control}}$$
</Formula>

For fault-tolerant quantum computing, we need $\epsilon < 10^{-3}$ to $10^{-4}$, requiring exquisite control over noise.

## 2. Classification of Noise Sources

### By Physical Origin

<Tip>
Understanding the physical origin of noise helps identify mitigation strategies. Some noise sources can be eliminated through better engineering, while others require error correction.
</Tip>

| Category | Examples | Typical Spectrum | Mitigation |
|----------|----------|-----------------|------------|
| Electromagnetic | Johnson noise, amplifier noise | White to 1/f | Filtering, shielding |
| Mechanical | Vibrations, acoustic noise | Low frequency | Isolation, damping |
| Material | TLS defects, charge traps | 1/f | Material improvements |
| Thermal | Phonons, photons | Bose-Einstein | Cooling |
| Quantum | Vacuum fluctuations, shot noise | White | Fundamental limit |
| Cosmic | Cosmic rays, radioactivity | Poisson | Shielding, soft coding |

### By Effect on Qubit

#### Longitudinal (T1) Noise
Causes energy exchange between qubit and environment:

<Formula>
$$\Gamma_1 = \frac{1}{T_1} = \frac{1}{2} S_\perp(\omega_{01})$$
</Formula>

where $S_\perp(\omega)$ is the transverse noise power spectral density at the qubit frequency.

#### Transverse (T2) Noise
Causes phase randomization without energy exchange:

<Formula>
$$\Gamma_\phi = \frac{1}{T_\phi} = \frac{1}{2} S_z(0) + \text{higher order terms}$$
</Formula>

The total dephasing rate combines both:

<Formula>
$$\frac{1}{T_2} = \frac{1}{2T_1} + \frac{1}{T_\phi}$$
</Formula>

## 3. The Spin-Boson Model

The spin-boson model provides a powerful framework for understanding qubit-environment interactions.

### Hamiltonian

<Formula>
$$H = \frac{\hbar\omega_0}{2}\sigma_z + \sum_k \hbar\omega_k a_k^\dagger a_k + \sigma_z \sum_k \hbar g_k (a_k + a_k^\dagger)$$
</Formula>

where:
- First term: Qubit with frequency $\omega_0$
- Second term: Environment as a bath of harmonic oscillators
- Third term: Qubit-bath coupling

### Spectral Density

The bath properties are encoded in the spectral density:

<Formula>
$$J(\omega) = \pi \sum_k g_k^2 \delta(\omega - \omega_k)$$
</Formula>

Different spectral densities lead to different decoherence behaviors:

<Definition term="Ohmic Spectral Density">
An Ohmic bath has $J(\omega) \propto \omega$ at low frequencies, characteristic of resistive environments. Sub-ohmic ($J \propto \omega^s$, $s < 1$) and super-ohmic ($s > 1$) baths show different decoherence dynamics.
</Definition>

```python
import numpy as np
import matplotlib.pyplot as plt

def spectral_density(omega, alpha, omega_c, s=1):
    """
    Generalized spectral density for spin-boson model.

    Args:
        omega: Frequency array
        alpha: Coupling strength (dimensionless)
        omega_c: Cutoff frequency
        s: Ohmicity parameter (s=1 is Ohmic)

    Returns:
        J(omega) spectral density
    """
    return alpha * omega**s * np.exp(-omega / omega_c)

# Compare different bath types
omega = np.linspace(0.01, 10, 200)
omega_c = 5.0
alpha = 0.1

plt.figure(figsize=(10, 6))
for s, label in [(0.5, 'Sub-ohmic (s=0.5)'),
                 (1.0, 'Ohmic (s=1)'),
                 (2.0, 'Super-ohmic (s=2)')]:
    J = spectral_density(omega, alpha, omega_c, s)
    plt.plot(omega, J, label=label, linewidth=2)

plt.xlabel('Frequency (arb. units)')
plt.ylabel('J(omega)')
plt.title('Spectral Density for Different Bath Types')
plt.legend()
plt.grid(True, alpha=0.3)
plt.show()
```

## 4. Power Spectral Density and Decoherence

### From Noise Spectrum to Decoherence

The relationship between noise and decoherence is captured through the filter function formalism:

<Formula>
$$\langle \phi^2(t) \rangle = \int_0^\infty \frac{d\omega}{2\pi} S(\omega) \frac{|F(\omega, t)|^2}{\omega^2}$$
</Formula>

where $F(\omega, t)$ is the filter function determined by the control sequence.

### Free Induction Decay (Ramsey)

For free evolution (Ramsey experiment), the filter function is:

<Formula>
$$|F_{\text{Ramsey}}(\omega, t)|^2 = 4\sin^2(\omega t / 2)$$
</Formula>

This peaks at low frequencies, making Ramsey measurements sensitive to low-frequency noise.

### Hahn Echo

The spin echo filter function:

<Formula>
$$|F_{\text{echo}}(\omega, t)|^2 = 16\sin^4(\omega t / 4)$$
</Formula>

This suppresses low-frequency noise, revealing the high-frequency noise floor.

<Example>
Calculating T2 from a known noise spectrum:

```python
import numpy as np
from scipy.integrate import quad

def calculate_t2_from_noise(S_func, t_max=1e-3, method='ramsey'):
    """
    Calculate T2 from noise power spectral density.

    Args:
        S_func: Function S(omega) returning noise PSD
        t_max: Maximum time for calculation
        method: 'ramsey' or 'echo'

    Returns:
        T2 time in seconds
    """
    def filter_ramsey(omega, t):
        if omega == 0:
            return t**2
        return 4 * np.sin(omega * t / 2)**2 / omega**2

    def filter_echo(omega, t):
        if omega == 0:
            return 0
        return 16 * np.sin(omega * t / 4)**4 / omega**2

    filter_func = filter_ramsey if method == 'ramsey' else filter_echo

    def coherence_decay(t):
        """Calculate coherence at time t."""
        def integrand(omega):
            return S_func(omega) * filter_func(omega, t) / (2 * np.pi)

        # Integrate from small omega to large omega
        phase_variance, _ = quad(integrand, 1e-6, 1e9)
        return np.exp(-phase_variance / 2)

    # Find T2 where coherence drops to 1/e
    times = np.logspace(-9, np.log10(t_max), 100)
    for t in times:
        if coherence_decay(t) < 1/np.e:
            return t

    return t_max

# Example: 1/f noise with white noise floor
def noise_spectrum(omega):
    """1/f noise plus white noise floor."""
    A_1f = 1e6  # 1/f amplitude (rad^2/s)
    S_white = 1e-3  # White noise floor (rad^2/s)
    omega_ir = 1  # IR cutoff
    return A_1f / (np.abs(omega) + omega_ir) + S_white

# Calculate T2 for both methods
T2_ramsey = calculate_t2_from_noise(noise_spectrum, method='ramsey')
T2_echo = calculate_t2_from_noise(noise_spectrum, method='echo')

print(f"T2* (Ramsey): {T2_ramsey*1e6:.1f} microseconds")
print(f"T2 (Echo): {T2_echo*1e6:.1f} microseconds")
```
</Example>

## 5. Markovian vs Non-Markovian Noise

### Markovian Noise

<Definition term="Markovian Noise">
Markovian noise has no memory - the future evolution depends only on the present state, not on history. This occurs when the bath correlation time is much shorter than the system dynamics. Leads to exponential decay of coherence.
</Definition>

Characteristics:
- Exponential coherence decay: $C(t) = e^{-\Gamma t}$
- White noise spectrum at relevant frequencies
- Lindblad master equation applies

### Non-Markovian Noise

<Definition term="Non-Markovian Noise">
Non-Markovian noise has memory effects - the bath retains information about past qubit states and feeds it back. Common in solid-state systems with low-frequency noise. Leads to non-exponential decay and potential information backflow.
</Definition>

Characteristics:
- Non-exponential decay (often Gaussian at short times)
- Colored noise spectrum (1/f or other power laws)
- Requires more sophisticated master equations

```python
import numpy as np
import matplotlib.pyplot as plt

def markovian_decay(t, Gamma):
    """Exponential decay for Markovian noise."""
    return np.exp(-Gamma * t)

def non_markovian_decay(t, sigma, Gamma_inf):
    """
    Gaussian-exponential decay for non-Markovian noise.

    Short times: Gaussian decay (sigma dominates)
    Long times: Exponential decay (Gamma_inf dominates)
    """
    return np.exp(-0.5 * (sigma * t)**2) * np.exp(-Gamma_inf * t)

# Compare decay profiles
t = np.linspace(0, 50, 200)

plt.figure(figsize=(10, 6))
plt.plot(t, markovian_decay(t, 0.1), 'b-', linewidth=2,
         label='Markovian (exponential)')
plt.plot(t, non_markovian_decay(t, 0.2, 0.05), 'r--', linewidth=2,
         label='Non-Markovian (Gaussian + exp)')

plt.xlabel('Time (arb. units)')
plt.ylabel('Coherence')
plt.title('Markovian vs Non-Markovian Decoherence')
plt.legend()
plt.grid(True, alpha=0.3)
plt.yscale('log')
plt.ylim(1e-3, 1)
plt.show()

# Signature of non-Markovianity: Information backflow
print("Non-Markovian signature: coherence can temporarily increase")
print("(information flows back from environment to qubit)")
```

<Warning>
Most solid-state qubits experience non-Markovian noise due to 1/f flux noise, charge noise, or TLS defects. Using Markovian models (standard Lindblad) can significantly underestimate T2 at short times and overestimate it at long times.
</Warning>

## 7. Noise Spectroscopy Techniques

### Ramsey Decay Analysis

```python
import numpy as np
from scipy.optimize import curve_fit

def ramsey_decay_model(t, T2_star, omega_detuning, A, B):
    """
    Ramsey decay with oscillation.

    For Gaussian noise: decay is Gaussian
    For Lorentzian noise: decay is exponential
    """
    return A * np.exp(-(t/T2_star)**2) * np.cos(omega_detuning * t) + B

def extract_noise_from_ramsey(times, signal):
    """
    Extract T2* and detuning from Ramsey data.
    """
    # Initial guess
    p0 = [10e-6, 2*np.pi*100e3, 0.5, 0.5]

    popt, pcov = curve_fit(ramsey_decay_model, times, signal, p0=p0)

    T2_star, omega_det, A, B = popt
    errors = np.sqrt(np.diag(pcov))

    return {
        'T2_star': T2_star,
        'T2_star_error': errors[0],
        'detuning_Hz': omega_det / (2*np.pi),
        'amplitude': A,
        'offset': B
    }
```

### Dynamical Decoupling Spectroscopy

By varying the number and timing of refocusing pulses, we can probe different frequency ranges of the noise spectrum:

<Formula>
$$S(\omega_n) \approx \frac{\pi}{2t} \left(-\ln W_n(t)\right)$$
</Formula>

where $W_n(t)$ is the coherence after $n$ refocusing pulses and $\omega_n = n\pi/t$.

## 8. Summary

Understanding noise in quantum hardware requires:

1. **Classification**: Know the physical origins and spectral properties
2. **Modeling**: Use spin-boson model and filter function formalism
3. **Measurement**: Apply noise spectroscopy techniques
4. **Markovianity**: Distinguish between memory-less and memory-full noise
5. **Platform specifics**: Each technology has unique dominant noise sources

<Tip>
The noise landscape is constantly evolving as hardware improves. What limited qubits 5 years ago may not be the dominant source today. Always characterize your specific device!
</Tip>

Key relationships to remember:
- $1/T_1 \propto S_\perp(\omega_0)$ - Transverse noise at qubit frequency causes relaxation
- $1/T_\phi \propto S_z(\omega \to 0)$ - Low-frequency longitudinal noise causes dephasing
- $1/T_2 = 1/(2T_1) + 1/T_\phi$ - Total decoherence combines both processes
