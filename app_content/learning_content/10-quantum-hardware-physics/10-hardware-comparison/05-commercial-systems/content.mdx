## 1. The Commercial Quantum Landscape

The quantum computing industry has rapidly evolved from research labs to commercial offerings. Understanding the vendor landscape is essential for making informed hardware decisions.

<Definition term="Commercial Quantum Computer">
A quantum computing system available for commercial use, either through cloud access, dedicated instances, or on-premise installation. Commercial systems provide documented specifications, service level agreements, and customer support.
</Definition>

### Market Structure

| Segment | Vendors | Access Model |

**Technology:** Superconducting transmon qubits

**Current Systems (2024):**

| System | Qubits | QV | CLOPS | Topology |
|--------|--------|-----|-------|----------|
| IBM Heron | 133 | TBD | ~5,000 | Heavy-hex |
| IBM Eagle | 127 | 128 | ~3,000 | Heavy-hex |
| IBM Osprey | 433 | - | ~2,000 | Heavy-hex |
| IBM Condor | 1,121 | - | TBD | Heavy-hex |

### Access Models

```python
# IBM Quantum access via Qiskit
from qiskit_ibm_runtime import QiskitRuntimeService

# Initialize service
service = QiskitRuntimeService(channel="ibm_quantum")

# List available backends
backends = service.backends()
for backend in backends:
    config = backend.configuration()
    print(f"{backend.name}: {config.n_qubits} qubits")

backend = service.backend("ibm_brisbane")
```

**Pricing Tiers:**
- **Open Plan:** Free, limited access, queue-based
- **Pay-as-you-go:** ~$1.60 per second of QPU time
- **Premium:** Dedicated access, custom pricing

<Tip>
IBM offers the most extensive free tier for learning and research, making it an excellent starting point. Their Qiskit ecosystem provides comprehensive documentation and tutorials.
</Tip>

### Unique Features

- **Qiskit Runtime:** Optimized hybrid classical-quantum workloads
- **Error Mitigation:** Built-in ZNE and other techniques
- **Dynamic Circuits:** Mid-circuit measurement and feedforward

```python
# Google Cirq example (simulator)
import cirq

# Create qubits
qubits = cirq.LineQubit.range(3)

# Build circuit
circuit = cirq.Circuit([
```

## 4. IonQ

### Technology Overview

IonQ is the leading trapped-ion quantum computing company, offering the highest fidelity gates commercially available.

**Technology:** Trapped ytterbium ions

**Current Systems:**

| System | Qubits | Algorithmic Qubits | 2Q Fidelity |
|--------|--------|-------------------|-------------|
| IonQ Aria | 25 | 23 | 99.6% |
| IonQ Forte | 32 | 29 | 99.7% |

<Definition term="Algorithmic Qubits">
IonQ's metric combining qubit count and fidelity: $AQ = n \times \log_2(1/\epsilon)$ where n is qubit count and epsilon is the two-qubit error rate. Represents the effective number of perfect qubits.
</Definition>

### Access Models

- **Amazon Braket:** Pay-per-task pricing
- **Microsoft Azure Quantum:** Azure credits
- **Google Cloud:** Partnership access
- **Direct API:** Enterprise customers

```python
# IonQ via Amazon Braket
from braket.aws import AwsDevice
from braket.circuits import Circuit

# Select IonQ device
device = AwsDevice("arn:aws:braket:us-east-1::device/qpu/ionq/Aria-1")

# Create circuit
circuit = Circuit().h(0).cnot(0, 1).probability()

# Run
task = device.run(circuit, shots=1000)
result = task.result()
```

**Pricing (via Braket):**
- Per-task: $0.30
- Per-shot: $0.01 (Aria), $0.015 (Forte)

### Unique Features

- **All-to-all connectivity**
- **Highest gate fidelities**
- **Native multi-qubit gates (GZZ)**

## 5. Quantinuum

### Technology Overview

Quantinuum (Honeywell + Cambridge Quantum) offers the highest-quality trapped-ion systems.

**Technology:** Trapped ytterbium ions with QCCD architecture

**Current Systems:**

| System | Qubits | QV | 2Q Fidelity |
|--------|--------|-------|-------------|
| H1-1 | 20 | 2^20 | 99.8% |
| H1-2 | 20 | 2^20 | 99.8% |
| H2 | 32 | 2^16 | 99.5% |

### Access Models

- **Direct enterprise contracts**
- **Azure Quantum marketplace**
- **Research partnerships**

### Unique Features

- **QCCD architecture:** Multiple zones for parallel operations
- **Highest QV record** (1,048,576 demonstrated)
- **Real-time error correction**
- **Extensive middleware (TKET compiler)**

<Warning>
Quantinuum systems have lower CLOPS due to slower ion operations. Best suited for applications where fidelity matters more than speed.
</Warning>

## 6. Rigetti Computing

### Technology Overview

Rigetti offers superconducting qubits with a focus on hybrid quantum-classical computing.

**Technology:** Superconducting transmon qubits

**Current Systems:**

| System | Qubits | Topology | Access |
|--------|--------|----------|--------|
| Ankaa-2 | 84 | Square lattice | Cloud |
| Aspen-M-3 | 80 | Octagonal | Cloud |

### Access Models

- **Rigetti Quantum Cloud Services (QCS)**
- **Amazon Braket**
- **On-premise (QPU-as-a-service)**

```python
# Rigetti via pyQuil
from pyquil import get_qc, Program
from pyquil.gates import H, CNOT, MEASURE

# Get quantum computer
qc = get_qc("Ankaa-2")

# Build program
p = Program()
p += H(0)
p += CNOT(0, 1)
ro = p.declare("ro", "BIT", 2)
p += MEASURE(0, ro[0])
p += MEASURE(1, ro[1])

# Execute
executable = qc.compile(p)
results = qc.run(executable)
```

### Unique Features

- **Quantum intermediate representation (Quil)**
- **Hybrid cloud architecture**
- **Parametric compilation**

### Selecting a Provider

<Example>
**Decision Framework:**

1. **Define your use case**
   - Algorithm type (variational, simulation, optimization)
   - Required qubits
   - Fidelity requirements

2. **Evaluate constraints**
   - Budget (free tier vs. enterprise)
   - Timeline (queue times vs. dedicated)
   - Integration needs (cloud provider)

3. **Start small, then scale**
   - Use simulators first
   - Test on free tiers
   - Benchmark across providers

```python
def recommend_vendor(use_case, budget, qubits_needed, fidelity_req):
    """Simple vendor recommendation logic."""

    if budget == "free":
        return "IBM Quantum (Open Plan)"

    if fidelity_req > 0.995:
        if budget == "high":
            return "Quantinuum"
        return "IonQ"

    if use_case == "optimization":
        return "D-Wave or QuEra"

    if use_case == "variational" and qubits_needed > 50:
        return "IBM or Rigetti"

    return "Start with IBM Quantum"
```
</Example>

## 11. Summary

Key takeaways for commercial quantum computing:

1. **IBM** leads in access and qubit count for superconducting systems
2. **IonQ/Quantinuum** offer highest fidelities with trapped ions
3. **Rigetti** provides flexible hybrid cloud options
4. **Emerging vendors** (QuEra, Pasqal) scale rapidly with neutral atoms
5. **Choice depends** on application, budget, and quality requirements

<Tip>
The "best" quantum computer depends entirely on your use case. For learning, start with IBM's free tier. For production, evaluate multiple vendors against your specific requirements. Consider using multi-cloud strategies to access different capabilities.
</Tip>
