## 1. Introduction to Quantum Control Waveforms

The fidelity of quantum gates depends critically on the shape of control pulses. Ideal pulses must balance competing requirements: fast operation (to minimize decoherence), spectral purity (to avoid leakage), and smooth transitions (to maintain adiabaticity).

<Definition term="Control Waveform">
A control waveform is the time-dependent amplitude and phase of an electromagnetic pulse applied to a qubit. For microwave-driven transmon qubits, this typically involves modulating a carrier wave at the qubit frequency with carefully shaped envelope functions.
</Definition>

### The Pulse Design Challenge

<Formula>
$$\Omega(t) = A(t) e^{i\phi(t)}$$
</Formula>

where $A(t)$ is the envelope amplitude and $\phi(t)$ is the phase.

**Key trade-offs:**

| Fast Pulses | Slow Pulses |
|-------------|-------------|
| Less decoherence | More decoherence |
| Broader bandwidth | Narrower bandwidth |
| More leakage risk | Less leakage |
| Harder calibration | Easier calibration |

<Tip>
A general rule: gate time should be at least 4-5 times the inverse anharmonicity to avoid significant leakage. For a transmon with 250 MHz anharmonicity, minimum gate time is ~16-20 ns.
</Tip>

## 2. Gaussian Pulses

### Basic Gaussian Envelope

The Gaussian pulse is the starting point for most qubit control:

<Formula>
$$A(t) = A_0 \exp\left(-\frac{(t-t_0)^2}{2\sigma^2}\right)$$
</Formula>

where $\sigma$ is the standard deviation (pulse width parameter) and $t_0$ is the pulse center.

**Properties:**
- Smooth turn-on and turn-off
- Well-defined frequency content
- Gaussian spectrum (bandwidth ~ 1/sigma)

```python
import numpy as np
import matplotlib.pyplot as plt

def gaussian_pulse(t: np.ndarray, amplitude: float, sigma: float,
                   t0: float = None) -> np.ndarray:
    """
    Generate a Gaussian pulse envelope.

    Args:
        t: Time array in seconds
        amplitude: Peak amplitude (Rabi frequency)
        sigma: Standard deviation in seconds
        t0: Center time (defaults to middle of array)

    Returns:
        Pulse amplitude array
    """
    if t0 is None:
        t0 = (t[0] + t[-1]) / 2

    return amplitude * np.exp(-(t - t0)**2 / (2 * sigma**2))

# Example: Create a 20 ns Gaussian pulse
duration = 20e-9  # 20 ns
sample_rate = 2e9  # 2 GSPS
t = np.arange(0, duration, 1/sample_rate)
sigma = duration / 6  # ~3 sigma on each side

pulse = gaussian_pulse(t, amplitude=1.0, sigma=sigma)

# Calculate the rotation angle for a given Rabi frequency
omega_rabi = 25e6 * 2 * np.pi  # 25 MHz Rabi frequency
rotation_angle = omega_rabi * np.trapz(pulse, t)
print(f"Rotation angle: {np.degrees(rotation_angle):.1f} degrees")
```

### Truncated Gaussian

In practice, Gaussian pulses are truncated to finite duration:

<Formula>
$$A_{trunc}(t) = \begin{cases}
A_0 \exp\left(-\frac{(t-t_0)^2}{2\sigma^2}\right) & \text{if } |t-t_0| \leq n_\sigma \cdot \sigma \\
0 & \text{otherwise}
\end{cases}$$
</Formula>

<Warning>
Truncating a Gaussian creates spectral leakage. Using at least 4-sigma truncation (each side) keeps leakage below 0.01% of peak.
</Warning>

## 3. DRAG Pulses

### The Leakage Problem

Transmon qubits have a third level |2⟩ separated from |1⟩ by the anharmonicity $\alpha$. Fast pulses can excite this level:

<Formula>
$$P_{leak} \propto \left(\frac{\Omega}{\alpha}\right)^2$$
</Formula>

where $\Omega$ is the Rabi frequency and $\alpha$ is the anharmonicity.

<Definition term="DRAG (Derivative Removal by Adiabatic Gate)">
DRAG is a pulse shaping technique that adds a derivative component to the quadrature channel, canceling transitions to the |2⟩ state. It enables faster gates with reduced leakage.
</Definition>

### DRAG Implementation

The DRAG pulse modifies the standard Gaussian:

<Formula>
$$I(t) = A(t)$$
$$Q(t) = -\frac{\beta}{\alpha} \frac{dA}{dt}$$
</Formula>

where:
- $I(t)$ is the in-phase (cosine) component
- $Q(t)$ is the quadrature (sine) component
- $\beta$ is the DRAG coefficient (typically ~1)
- $\alpha$ is the qubit anharmonicity

```python
import numpy as np

def drag_pulse(t: np.ndarray, amplitude: float, sigma: float,
               anharmonicity: float, drag_coeff: float = 1.0,
               t0: float = None) -> tuple:
    """
    Generate a DRAG pulse with I and Q components.

    Args:
        t: Time array in seconds
        amplitude: Peak amplitude
        sigma: Gaussian width in seconds
        anharmonicity: Qubit anharmonicity in Hz (negative for transmon)
        drag_coeff: DRAG coefficient (beta), typically ~1
        t0: Center time

    Returns:
        (I, Q) tuple of pulse components
    """
    if t0 is None:
        t0 = (t[0] + t[-1]) / 2

    # Gaussian envelope
    gaussian = amplitude * np.exp(-(t - t0)**2 / (2 * sigma**2))

    # Gaussian derivative
    derivative = gaussian * (-(t - t0) / sigma**2)

    # DRAG components
    I = gaussian
    Q = -drag_coeff / (2 * np.pi * anharmonicity) * derivative

    return I, Q

# Example: DRAG pulse for a transmon
duration = 20e-9
t = np.linspace(0, duration, 100)
sigma = duration / 6
anharmonicity = -250e6  # -250 MHz

I, Q = drag_pulse(t, amplitude=1.0, sigma=sigma,
                  anharmonicity=anharmonicity, drag_coeff=1.0)

print(f"Peak I amplitude: {np.max(np.abs(I)):.3f}")
print(f"Peak Q amplitude: {np.max(np.abs(Q)):.3f}")
print(f"Q/I ratio at peak: {np.max(np.abs(Q))/np.max(np.abs(I))*100:.1f}%")
```

### Calibrating DRAG

The optimal DRAG coefficient $\beta$ must be calibrated experimentally:

1. **AllXY sequence**: Run pairs of gates (e.g., XI, IX, XX) and measure fidelity
2. **Scan beta**: Sweep DRAG coefficient from 0 to 2
3. **Minimize error**: Find beta that minimizes phase errors

<Example>
DRAG calibration analysis:

```python
import numpy as np

def simulate_drag_calibration(beta_values: np.ndarray,
                               true_beta: float = 1.0,
                               anharmonicity: float = -250e6,
                               gate_duration: float = 20e-9) -> dict:
    """
    Simulate DRAG calibration scan.

    Returns error vs beta, simulating over/under-compensation.
    """
    # Simple model: error from residual |2> population
    # Error minimized when beta = true_beta

    # Phase error from DRAG mismatch
    phase_errors = (beta_values - true_beta) * 0.1  # Simplified model

    # Leakage error
    leakage_errors = 0.01 * (1 + (beta_values - true_beta)**2)

    total_error = np.abs(phase_errors) + leakage_errors

    return {
        'beta': beta_values,
        'phase_error': phase_errors,
        'leakage_error': leakage_errors,
        'total_error': total_error,
        'optimal_beta': beta_values[np.argmin(total_error)]
    }

# Scan DRAG coefficient
betas = np.linspace(0, 2, 41)
result = simulate_drag_calibration(betas, true_beta=1.05)

print(f"Optimal beta found: {result['optimal_beta']:.2f}")
print(f"Minimum total error: {np.min(result['total_error'])*100:.3f}%")
```
</Example>

## 4. Spectral Analysis of Pulses

### Fourier Transform of Pulses

Understanding the frequency content of pulses is crucial for avoiding off-resonant excitations:

<Formula>
$$\tilde{A}(\omega) = \int_{-\infty}^{\infty} A(t) e^{-i\omega t} dt$$
</Formula>

For a Gaussian pulse:
<Formula>
$$\tilde{A}(\omega) = A_0 \sigma \sqrt{2\pi} e^{-\frac{\omega^2 \sigma^2}{2}}$$
</Formula>

The bandwidth (FWHM) is approximately:
<Formula>
$$\Delta f \approx \frac{0.44}{\sigma}$$
</Formula>

```python
import numpy as np
from scipy.fft import fft, fftfreq

def analyze_pulse_spectrum(pulse: np.ndarray, sample_rate: float) -> dict:
    """
    Analyze the frequency spectrum of a pulse.

    Args:
        pulse: Time-domain pulse samples
        sample_rate: Sample rate in Hz

    Returns:
        Spectral analysis results
    """
    n = len(pulse)
    spectrum = np.abs(fft(pulse))
    freqs = fftfreq(n, 1/sample_rate)

    # Find peak and FWHM
    peak_idx = np.argmax(spectrum[:n//2])
    peak_freq = freqs[peak_idx]
    half_max = spectrum[peak_idx] / 2

    # Find FWHM (approximate)
    above_half = spectrum[:n//2] > half_max
    fwhm_samples = np.sum(above_half)
    fwhm_hz = fwhm_samples * sample_rate / n

    return {
        'frequencies': freqs[:n//2],
        'spectrum': spectrum[:n//2],
        'peak_frequency': peak_freq,
        'fwhm_hz': fwhm_hz,
        'bandwidth_mhz': fwhm_hz / 1e6
    }

# Compare spectra of different pulse widths
sample_rate = 2e9
for sigma_ns in [2, 4, 8]:
    duration = sigma_ns * 6e-9  # 3 sigma each side
    t = np.arange(0, duration, 1/sample_rate)
    pulse = gaussian_pulse(t, 1.0, sigma_ns*1e-9)

    analysis = analyze_pulse_spectrum(pulse, sample_rate)
    expected_bw = 0.44 / (sigma_ns * 1e-9) / 1e6  # MHz

    print(f"Sigma={sigma_ns} ns: BW={analysis['bandwidth_mhz']:.1f} MHz "
          f"(expected ~{expected_bw:.1f} MHz)")
```

### Avoiding Spectral Crowding

In multi-qubit systems, pulse bandwidth must not overlap with neighboring qubit frequencies:

<Formula>
$$\Delta f_{pulse} < |\omega_i - \omega_j|$$
</Formula>

<Warning>
For qubits spaced 100 MHz apart, pulse bandwidth should be much less than 100 MHz, implying sigma > 4.4 ns and gate times > 25-30 ns.
</Warning>

## 5. Single-Qubit Gate Calibration

### Amplitude Calibration (Rabi)

The pulse amplitude determines the rotation angle:

<Formula>
$$\theta = \int_0^T \Omega(t) dt$$
</Formula>

For a pi pulse ($\theta = \pi$) with Gaussian envelope:
<Formula>
$$A_\pi = \frac{\pi}{\sqrt{2\pi}\sigma}$$
</Formula>

```python
def calibrate_pi_amplitude(sigma: float, sample_rate: float,
                            measured_angles: list = None) -> dict:
    """
    Calculate or calibrate pi pulse amplitude.

    Args:
        sigma: Pulse width in seconds
        sample_rate: DAC sample rate
        measured_angles: List of (amplitude, measured_angle) for fitting

    Returns:
        Calibration results
    """
    # Theoretical amplitude for pi rotation
    # Integral of Gaussian = A * sigma * sqrt(2*pi)
    # For rotation = pi: A * sigma * sqrt(2*pi) * Rabi_freq = pi
    # If Rabi_freq = 1 (normalized), A_pi = 1 / (sigma * sqrt(2*pi)) * pi

    # Actually, for a truncated Gaussian at 3-sigma each side:
    # Integral factor is approximately 0.997 * sigma * sqrt(2*pi)

    theoretical_A_pi = np.pi / (sigma * np.sqrt(2 * np.pi) * 0.997)

    # If we have measured data, fit it
    if measured_angles:
        amps = np.array([m[0] for m in measured_angles])
        angles = np.array([m[1] for m in measured_angles])
        # Linear fit: angle = amp * calibration_factor
        cal_factor = np.sum(amps * angles) / np.sum(amps**2)
        fitted_A_pi = np.pi / cal_factor
    else:
        fitted_A_pi = None

    return {
        'theoretical_A_pi': theoretical_A_pi,
        'fitted_A_pi': fitted_A_pi,
        'A_pi_over_2': theoretical_A_pi / 2
    }

# Example calibration
sigma = 20e-9 / 6  # 20 ns total duration
cal = calibrate_pi_amplitude(sigma, 2e9)
print(f"Theoretical pi amplitude: {cal['theoretical_A_pi']:.4f}")
print(f"Pi/2 amplitude: {cal['A_pi_over_2']:.4f}")
```

### Frequency Calibration

Qubit frequency drift requires regular recalibration:

**Ramsey Experiment:**
1. Apply X/2 pulse
2. Wait time $\tau$
3. Apply X/2 pulse
4. Measure

The oscillation frequency gives the detuning:
<Formula>
$$P(|1\rangle) = \frac{1}{2}[1 + e^{-\tau/T_2^*}\cos(\delta\omega \cdot \tau)]$$
</Formula>

## 6. Two-Qubit Gate Pulses

### Flux Pulses for Tunable Couplers

For tunable transmons, two-qubit gates often use flux pulses:

<Formula>
$$\Phi(t) = \Phi_0 \cdot A(t)$$
</Formula>

**Net-Zero Flux Pulses:**
To avoid long-term flux bias drift, use bipolar pulses:

```python
def net_zero_flux_pulse(t: np.ndarray, amplitude: float,
                         duration: float, rise_time: float) -> np.ndarray:
    """
    Generate a net-zero flux pulse (positive then negative).

    Args:
        t: Time array
        amplitude: Peak flux amplitude
        duration: Total pulse duration
        rise_time: Rise/fall time

    Returns:
        Flux waveform
    """
    half_duration = duration / 2

    pulse = np.zeros_like(t)

    for i, ti in enumerate(t):
        if ti < rise_time:
            # Rising edge of positive lobe
            pulse[i] = amplitude * np.sin(np.pi * ti / (2 * rise_time))**2
        elif ti < half_duration - rise_time:
            # Flat top of positive lobe
            pulse[i] = amplitude
        elif ti < half_duration + rise_time:
            # Transition to negative
            phase = (ti - half_duration + rise_time) / (2 * rise_time)
            pulse[i] = amplitude * np.cos(np.pi * phase)
        elif ti < duration - rise_time:
            # Flat bottom of negative lobe
            pulse[i] = -amplitude
        elif ti < duration:
            # Rising edge back to zero
            phase = (ti - duration + rise_time) / rise_time
            pulse[i] = -amplitude * (1 - np.sin(np.pi * phase / 2)**2)

    return pulse

# Example: 100 ns CZ gate flux pulse
duration = 100e-9
t = np.linspace(0, duration, 200)
flux = net_zero_flux_pulse(t, amplitude=0.3, duration=duration, rise_time=10e-9)

print(f"Net flux: {np.trapz(flux, t)*1e9:.4f} (should be ~0)")
```

### Cross-Resonance (CR) Pulses

For fixed-frequency qubits, cross-resonance gates drive one qubit at the frequency of another:

<Formula>
$$H_{CR} = \frac{\Omega}{2}(a^\dagger + a) \otimes \sigma_z^{target}$$
</Formula>

**CR Pulse Shaping:**
- Flat-top with smooth edges (cosine ramps)
- Echo sequence (CR - X_target - CR) to remove unwanted terms
- Typical duration: 200-500 ns

```python
def cr_pulse_with_echo(t: np.ndarray, amplitude: float,
                        duration: float, ramp_time: float) -> tuple:
    """
    Generate CR pulse pair with echo.

    Returns:
        (cr_drive, target_x_pulse) waveforms
    """
    half_duration = duration / 2

    cr_drive = np.zeros_like(t)
    target_x = np.zeros_like(t)

    for i, ti in enumerate(t):
        # First CR pulse
        if ti < half_duration:
            if ti < ramp_time:
                cr_drive[i] = amplitude * 0.5 * (1 - np.cos(np.pi * ti / ramp_time))
            elif ti < half_duration - ramp_time:
                cr_drive[i] = amplitude
            else:
                phase = (ti - half_duration + ramp_time) / ramp_time
                cr_drive[i] = amplitude * 0.5 * (1 + np.cos(np.pi * phase))

        # X pulse on target (at center)
        if abs(ti - half_duration) < 10e-9:  # 20 ns X pulse
            target_x[i] = 1.0

        # Second CR pulse (same shape)
        if ti >= half_duration:
            t_shifted = ti - half_duration
            if t_shifted < ramp_time:
                cr_drive[i] = amplitude * 0.5 * (1 - np.cos(np.pi * t_shifted / ramp_time))
            elif t_shifted < half_duration - ramp_time:
                cr_drive[i] = amplitude
            elif t_shifted < half_duration:
                phase = (t_shifted - half_duration + ramp_time) / ramp_time
                cr_drive[i] = amplitude * 0.5 * (1 + np.cos(np.pi * phase))

    return cr_drive, target_x

# Example
duration = 400e-9
t = np.linspace(0, duration, 800)
cr, x = cr_pulse_with_echo(t, amplitude=0.5, duration=duration, ramp_time=20e-9)
print(f"CR pulse generated: {len(t)} samples")
```

## 7. Pulse Optimization Techniques

### Gradient-Based Optimization

Optimal control theory can find pulse shapes that maximize fidelity:

<Formula>
$$\mathcal{F} = |\langle\psi_{target}|U(T)|\psi_0\rangle|^2$$
</Formula>

**GRAPE Algorithm:**
1. Initialize pulse guess
2. Forward propagate Schrodinger equation
3. Backward propagate with target state
4. Calculate gradient of fidelity w.r.t. pulse amplitudes
5. Update pulse
6. Repeat until convergence

```python
import numpy as np

                       target_unitary: np.ndarray,

        learning_rate: Optimization step size

    Returns:
        Updated pulse
    """
    n_steps = len(pulse)

    # Forward propagation (simplified)
    # U = prod_i exp(-i * H(t_i) * dt)

    # In practice, this involves computing the propagator
    # and its derivative with respect to each pulse amplitude
```

### Bandwidth-Limited Optimization

Constrain optimization to realistic pulse bandwidths:

<Formula>
$$A(t) = \sum_{k=0}^{K} c_k \cos(k\omega_0 t) + s_k \sin(k\omega_0 t)$$
</Formula>

where $\omega_0 = 2\pi/T$ and $K$ determines the bandwidth.

## 8. Practical Considerations

### DAC Resolution and Quantization

DAC bit depth limits pulse precision:

<Formula>
$$\Delta A = \frac{A_{max}}{2^{N_{bits}}}$$
</Formula>

For 16-bit DAC with 1V range: $\Delta A \approx 15$ μV

### Calibration Drift

Environmental changes cause parameter drift:

| Parameter | Typical Drift | Recalibration Frequency |
|-----------|---------------|------------------------|
| Qubit frequency | 100 kHz/hour | Every 10-30 min |
| Pi amplitude | 0.1%/hour | Every 1-2 hours |
| DRAG coefficient | 1%/day | Daily |
| Readout frequency | 10 kHz/hour | Every 30-60 min |

<Tip>
Automated calibration routines should run in the background during quantum computations, updating parameters as needed without interrupting the main experiment.
</Tip>

## 9. Summary

Quantum control waveform design requires balancing multiple constraints:

1. **Gaussian pulses**: Provide smooth, well-behaved baseline
2. **DRAG**: Suppresses leakage to non-computational states
3. **Spectral purity**: Avoids crosstalk to neighboring qubits
4. **Two-qubit gates**: Require specialized flux or CR pulses
5. **Optimization**: GRAPE and other methods find optimal shapes
6. **Calibration**: Regular updates maintain gate fidelity

High-fidelity quantum operations require careful attention to pulse design, combined with robust calibration procedures.
