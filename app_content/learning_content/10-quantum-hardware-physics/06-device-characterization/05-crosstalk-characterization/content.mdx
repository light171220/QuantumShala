## 1. Introduction to Crosstalk

Crosstalk refers to unwanted interactions between qubits that occur during quantum operations. As qubit counts increase, crosstalk becomes a major challenge for scaling quantum computers.

<Definition term="Crosstalk">
Crosstalk is any unintended coupling or interaction between qubits that causes errors during quantum operations. It can be static (always-on coupling) or dynamic (only during gate operations).
</Definition>

### Types of Crosstalk

| Type | Source | Effect | Mitigation |
|------|--------|--------|------------|
| ZZ coupling | Always-on capacitive coupling | Phase errors during idle | Echo sequences, tunable couplers |
| Control line crosstalk | Microwave leakage | Unintended rotations | Improved filtering, calibration |
| Frequency crowding | Spectral overlap | Gate errors | Careful frequency allocation |
| Flux crosstalk | Magnetic coupling | Frequency shifts | Flux compensation |

## 2. Static ZZ Coupling

### Physical Origin

In transmon qubits, residual ZZ coupling arises from the exchange interaction:

<Formula>
$$H_{ZZ} = \frac{\zeta}{4} Z_1 \otimes Z_2$$
</Formula>

where $\zeta$ is the ZZ coupling strength, typically expressed in MHz.

For capacitively coupled transmons:

<Formula>
$$\zeta \approx \frac{2 g^2 \alpha_1 \alpha_2}{(\omega_1 - \omega_2)(\omega_1 - \omega_2 + \alpha_1)(\omega_1 - \omega_2 - \alpha_2)}$$
</Formula>

where $g$ is the coupling strength and $\alpha_i$ are anharmonicities.

### Effects of ZZ Coupling

ZZ coupling causes state-dependent frequency shifts:

```
|00>: No shift
|01>: Shift by +zeta/2
|10>: Shift by +zeta/2
|11>: Shift by +zeta
```

This leads to phase accumulation during two-qubit idle periods.

## 3. Measuring ZZ Coupling

### Ramsey-Based ZZ Measurement

The most direct method uses conditional Ramsey experiments:

```python
def measure_zz_coupling(qubit1, qubit2, delays, num_shots=1000):
    """
    Measure ZZ coupling using conditional Ramsey.

    Protocol:
    1. Prepare qubit2 in |0> or |1>
    2. Run Ramsey on qubit1
    3. Compare Ramsey frequency in both cases
    4. Difference gives ZZ coupling strength
    """
    results = {}

    for q2_state in [0, 1]:
        ramsey_data = []

        for delay in delays:
            p1_count = 0

            for _ in range(num_shots):
                # Reset both qubits
                qubit1.reset()
                qubit2.reset()

                # Prepare qubit2 in target state
                if q2_state == 1:
                    qubit2.x()

                # Ramsey on qubit1
                qubit1.rx(np.pi/2)
                qubit1.delay(delay)
                qubit1.rx(np.pi/2)

                # Measure qubit1
                if qubit1.measure() == 1:
                    p1_count += 1

            ramsey_data.append(p1_count / num_shots)

        results[f'q2={q2_state}'] = np.array(ramsey_data)

    # Fit Ramsey fringes
    freq_q2_0 = fit_ramsey_frequency(delays, results['q2=0'])
    freq_q2_1 = fit_ramsey_frequency(delays, results['q2=1'])

    # ZZ coupling is the frequency difference
    zz_coupling_Hz = freq_q2_1 - freq_q2_0

    return {
        'zz_coupling_Hz': zz_coupling_Hz,
        'zz_coupling_kHz': zz_coupling_Hz / 1e3,
        'ramsey_data': results,
        'freq_q2_0': freq_q2_0,
        'freq_q2_1': freq_q2_1
    }
```

### Echo-Based ZZ Measurement

For more precision, use an echo sequence:

```python
def measure_zz_echo(qubit1, qubit2, total_times, num_shots=1000):
    """
    Measure ZZ coupling using echo sequence.

    More robust against single-qubit dephasing.
    """
    zz_phases = []

    for total_time in total_times:
        tau = total_time / 2

        # Phase accumulated with q2 flipped mid-sequence
        phase_accum = 0
        count = 0

        for _ in range(num_shots):
            qubit1.reset()
            qubit2.reset()

            # Prepare q2 in |1>
            qubit2.x()

            # Ramsey with echo on q1, flip q2 at midpoint
            qubit1.rx(np.pi/2)
            qubit1.delay(tau)

            # Flip q2 at midpoint
            qubit2.x()

            # Echo pulse on q1
            qubit1.rx(np.pi)
            qubit1.delay(tau)

            # Final rotation depends on measurement basis
            qubit1.ry(np.pi/2)  # Measure Y

            if qubit1.measure() == 1:
                count += 1

        # Phase is encoded in Y measurement
        p_y = count / num_shots
        phase = np.arcsin(2 * p_y - 1)
        zz_phases.append(phase)

    # ZZ coupling from phase vs time slope
    # Phase = zeta * total_time (for this sequence)
    slope, _ = np.polyfit(total_times, zz_phases, 1)
    zz_coupling_Hz = slope / (2 * np.pi)

    return {
        'zz_coupling_Hz': zz_coupling_Hz,
        'phases': zz_phases,
        'times': total_times
    }
```

## 4. Simultaneous Randomized Benchmarking

Simultaneous RB (covered in the RB lesson) is the gold standard for detecting operational crosstalk.

### Protocol Summary

```python
def simultaneous_rb_crosstalk(qubits, sequence_lengths):
    """
    Detect crosstalk via simultaneous RB.

    1. Run individual RB on each qubit
    2. Run simultaneous RB on all qubits
    3. Compare error rates

    Crosstalk = EPC_simultaneous - EPC_individual
    """
    # Individual RB
    individual_epc = {}
    for i, q in enumerate(qubits):
        result = run_rb(q, sequence_lengths)
        individual_epc[i] = result['epc']

    # Simultaneous RB
    simultaneous_epc = {}
    sim_result = run_simultaneous_rb(qubits, sequence_lengths)
    for i in range(len(qubits)):
        simultaneous_epc[i] = sim_result[i]['epc']

    # Calculate crosstalk
    crosstalk = {}
    for i in range(len(qubits)):
        crosstalk[i] = simultaneous_epc[i] - individual_epc[i]

    return {
        'individual_epc': individual_epc,
        'simultaneous_epc': simultaneous_epc,
        'crosstalk': crosstalk
    }
```

### Interpreting Results

<Tip>
If simultaneous EPC > individual EPC by more than measurement uncertainty, crosstalk is present. The difference quantifies the additional error from operating qubits simultaneously.
</Tip>

## 5. Addressability Characterization

### What is Addressability?

Addressability measures how well you can control one qubit without affecting others.

<Definition term="Addressability Error">
Addressability error is the unintended rotation of a spectator qubit when driving a target qubit. It arises from frequency crowding or control line crosstalk.
</Definition>

### Measuring Addressability

```python
def measure_addressability(target_qubit, spectator_qubits, num_gates=100):
    """
    Measure addressability by checking spectator qubit state
    after driving the target qubit.

    Protocol:
    1. Initialize all qubits in |0>
    2. Apply N gates to target qubit
    3. Check if spectator qubits remain in |0>
    """
    addressability_errors = {}

    for spectator in spectator_qubits:
        error_count = 0
        total_shots = 1000

        for _ in range(total_shots):
            # Reset all qubits
            target_qubit.reset()
            spectator.reset()

            # Apply many gates to target
            for _ in range(num_gates):
                target_qubit.x()  # or random Cliffords

            # Check spectator (should still be |0>)
            if spectator.measure() == 1:
                error_count += 1

        # Addressability error rate
        error_rate = error_count / total_shots
        addressability_errors[spectator.id] = error_rate / num_gates

    return addressability_errors
```

### Frequency Crowding Analysis

```python
def analyze_frequency_crowding(qubit_frequencies, anharmonicities, coupling_map):
    """
    Analyze potential frequency collisions.

    Check for:
    1. Direct frequency collisions (|f_i - f_j| < threshold)
    2. 1-2 transition collisions (|f_i - (f_j + alpha_j)| < threshold)
    3. Two-photon collisions
    """
    threshold_MHz = 50  # Minimum separation

    collisions = []

    n_qubits = len(qubit_frequencies)
    for i in range(n_qubits):
        for j in range(i+1, n_qubits):
            f_i = qubit_frequencies[i]
            f_j = qubit_frequencies[j]
            a_i = anharmonicities[i]
            a_j = anharmonicities[j]

            # Direct collision
            if abs(f_i - f_j) < threshold_MHz:
                collisions.append({
                    'type': 'direct',
                    'qubits': (i, j),
                    'separation_MHz': abs(f_i - f_j)
                })

            # 0-1 to 1-2 collision
            if abs(f_i - (f_j + a_j)) < threshold_MHz:
                collisions.append({
                    'type': '01-12',
                    'qubits': (i, j),
                    'separation_MHz': abs(f_i - (f_j + a_j))
                })

            # Two-photon collision
            if abs(2*f_i - (f_j + f_i)) < threshold_MHz:
                collisions.append({
                    'type': 'two-photon',
                    'qubits': (i, j),
                    'separation_MHz': abs(f_i - f_j)
                })

    return {
        'collisions': collisions,
        'num_collisions': len(collisions),
        'safe': len(collisions) == 0
    }
```

## 6. Control Line Crosstalk

### Characterizing Microwave Leakage

```python
def measure_control_crosstalk(target_line, spectator_qubits, amplitude_range):
    """
    Measure microwave crosstalk from control line.

    Apply pulse on target line and check for rotation
    of spectator qubits at their frequencies.
    """
    crosstalk_matrix = {}

    for spectator in spectator_qubits:
        crosstalk_amplitudes = []

        for amplitude in amplitude_range:
            # Reset spectator
            spectator.reset()

            # Apply pulse on target line at spectator frequency
            apply_pulse(target_line,
                       frequency=spectator.frequency,
                       amplitude=amplitude,
                       duration=100e-9)

            # Measure rotation of spectator
            # Use Ramsey-like sequence
            spectator.rx(np.pi/2)
            spectator.delay(10e-9)
            spectator.rx(np.pi/2)

            # Deviation from |0> indicates crosstalk
            p1 = measure_probability(spectator, state=1)
            crosstalk_amplitudes.append(p1)

        # Fit to extract crosstalk coefficient
        # p1 = A * amplitude^2 for small amplitudes
        coeff = fit_quadratic(amplitude_range, crosstalk_amplitudes)
        crosstalk_matrix[spectator.id] = coeff

    return crosstalk_matrix
```

## 7. Mitigation Strategies

### ZZ Coupling Mitigation

```python
def zz_echo_mitigation(circuit, zz_couplings, qubit_pairs):
    """
    Insert echo pulses to cancel ZZ coupling.

    For each idle period, add refocusing pulses
    to cancel accumulated ZZ phase.
    """
    mitigated_circuit = []

    for operation in circuit:
        if operation.type == 'delay':
            # For idle periods, add echo if coupled qubits are involved
            for q1, q2 in qubit_pairs:
                if q1 in operation.qubits or q2 in operation.qubits:
                    # Insert echo to refocus ZZ
                    half_delay = operation.duration / 2
                    mitigated_circuit.append(Delay(half_delay, operation.qubits))
                    mitigated_circuit.append(X(q1))  # Echo pulse
                    mitigated_circuit.append(Delay(half_delay, operation.qubits))
                    break
            else:
                mitigated_circuit.append(operation)
        else:
            mitigated_circuit.append(operation)

    return mitigated_circuit
```

### Frequency Optimization

```python
def optimize_frequencies(initial_freqs, couplings, constraints):
    """
    Optimize qubit frequencies to minimize crosstalk.

    Uses numerical optimization to find frequency allocation
    that maximizes minimum separation while respecting constraints.
    """
    from scipy.optimize import minimize

    def crosstalk_cost(freqs):
        # Penalize small frequency separations
        cost = 0
        for i in range(len(freqs)):
            for j in range(i+1, len(freqs)):
                if couplings[i][j] > 0:  # Only coupled qubits
                    separation = abs(freqs[i] - freqs[j])
                    cost += 1 / (separation + 1)  # Avoid division by zero
        return cost

    result = minimize(
        crosstalk_cost,
        initial_freqs,
        bounds=constraints,
        method='L-BFGS-B'
    )

    return result.x
```

## 8. Summary

Crosstalk characterization is essential for multi-qubit systems:

1. **ZZ Coupling**: Measure with conditional Ramsey; mitigate with echo or tunable couplers
2. **Simultaneous RB**: Gold standard for detecting operational crosstalk
3. **Addressability**: Check spectator qubits when driving targets
4. **Frequency Crowding**: Plan frequency allocation carefully

Key metrics:
- ZZ coupling: < 100 kHz for high-fidelity operations
- Addressability error: < 10^-4 per gate
- Simultaneous vs individual EPC difference: < 10% of individual EPC

<Warning>
Crosstalk often becomes the dominant error source when individual qubit fidelities are high. It must be characterized and mitigated for any multi-qubit algorithm.
</Warning>
