## 1. Introduction to Gate Set Tomography

Gate Set Tomography (GST) provides complete, self-consistent characterization of a set of quantum gates. Unlike standard process tomography, GST simultaneously characterizes gates, state preparation, and measurement without assuming any of them are perfect.

<Definition term="Gate Set Tomography">
Gate set tomography is a protocol that simultaneously estimates the process matrices of multiple gates along with state preparation and measurement (SPAM) operations, using only the assumption that quantum mechanics is correct. It produces self-consistent estimates without requiring calibrated reference operations.
</Definition>

### Why Gate Set Tomography?

| Approach | SPAM-Free | Complete Info | Scalable | Self-Consistent |
|----------|-----------|---------------|----------|-----------------|
| Process Tomography | No | Yes | No | No |
| Randomized Benchmarking | Yes | No | Yes | N/A |
| Gate Set Tomography | Yes | Yes | Moderate | Yes |

## 2. The GST Framework

### The Gate Set

A gate set consists of:

<Formula>
$$\mathcal{G} = \{\rho, M, G_1, G_2, ..., G_k\}$$
</Formula>

where:
- $\rho$: Native state preparation (density matrix)
- $M$: Native measurement (POVM elements)
- $G_i$: Gate operations (process matrices)

### Superoperator Representation

In GST, quantum operations are represented as superoperators acting on vectorized density matrices:

<Formula>
$$|\rho\rangle\rangle = \text{vec}(\rho) = \begin{pmatrix} \rho_{00} \\ \rho_{01} \\ \rho_{10} \\ \rho_{11} \end{pmatrix}$$
</Formula>

A gate G becomes a 4x4 (for single qubit) process matrix acting on this vector.

### The Gram Matrix

GST constructs experiments using fiducial sequences:

<Formula>
$$g_{ij} = \langle\langle M | F_i^\dagger G F_j |\rho\rangle\rangle$$
</Formula>

where $F_i$ and $F_j$ are preparation and measurement fiducials.

## 3. GST Protocol Components

### Fiducials

Fiducials are short gate sequences that prepare informationally complete states and measurements:

```python
# Standard single-qubit fiducials
preparation_fiducials = [
    [],           # |0> preparation
    ['Gx'],       # X|0> = |1>
    ['Gy'],       # Y|0>
    ['Gx', 'Gx'], # X^2|0> = |0>
    ['Gx', 'Gy'], # More states...
    ['Gy', 'Gx']
]

measurement_fiducials = [
    [],           # Z measurement
    ['Gx'],       # Measure after X
    ['Gy'],       # Measure after Y
    ['Gx', 'Gx'],
    ['Gx', 'Gy'],
    ['Gy', 'Gx']
]
```

### Germs

Germs are short sequences designed to amplify all possible gate errors:

<Definition term="Germ">
A germ is a short gate sequence that, when repeated many times, amplifies specific error modes. A complete set of germs amplifies all Hamiltonian and stochastic error parameters.
</Definition>

```python
# Example germ set for {Gx, Gy, Gi} gate set
germs = [
    ['Gx'],
    ['Gy'],
    ['Gi'],
    ['Gx', 'Gy'],
    ['Gx', 'Gx', 'Gy'],
    ['Gx', 'Gy', 'Gy'],
    ['Gx', 'Gx', 'Gi'],
    ['Gy', 'Gy', 'Gi'],
    ['Gx', 'Gy', 'Gi'],
    ['Gx', 'Gi', 'Gy'],
    ['Gx', 'Gi', 'Gi'],
    ['Gy', 'Gi', 'Gi'],
    ['Gx', 'Gx', 'Gy', 'Gx', 'Gy', 'Gy']
]
```

### Germ Powers

Each germ is repeated to create sequences of increasing length:

<Formula>
$$L = 1, 2, 4, 8, 16, ..., L_{max}$$
</Formula>

The maximum length $L_{max}$ is determined by coherence times.

## 4. Running GST

### Circuit Generation

```python
def generate_gst_circuits(fiducials_prep, fiducials_meas, germs, max_lengths):
    """
    Generate all GST circuits.

    Args:
        fiducials_prep: Preparation fiducial sequences
        fiducials_meas: Measurement fiducial sequences
        germs: Germ sequences
        max_lengths: List of germ repetition lengths

    Returns:
        List of circuit sequences
    """
    circuits = []

    for L in max_lengths:
        for germ in germs:
            for f_prep in fiducials_prep:
                for f_meas in fiducials_meas:
                    # Build circuit: prep_fiducial + germ^L + meas_fiducial
                    circuit = f_prep + germ * L + f_meas
                    circuits.append(circuit)

    return circuits

# Example circuit count
n_prep = 6
n_meas = 6
n_germs = 12
lengths = [1, 2, 4, 8, 16, 32, 64, 128]

n_circuits = n_prep * n_meas * n_germs * len(lengths)
print(f"Total GST circuits: {n_circuits}")  # 6*6*12*8 = 3456 circuits
```

### Data Collection

```python
def collect_gst_data(circuits, qubit, num_shots=1000):
    """
    Collect GST experimental data.

    Args:
        circuits: List of gate sequences
        qubit: Qubit object
        num_shots: Measurements per circuit

    Returns:
        Dictionary mapping circuits to outcome counts
    """
    data = {}

    for circuit in circuits:
        counts = {'0': 0, '1': 0}

        for _ in range(num_shots):
            qubit.reset()

            # Apply circuit
            for gate in circuit:
                qubit.apply_gate(gate)

            # Measure
            result = qubit.measure()
            counts[str(result)] += 1

        data[tuple(circuit)] = counts

    return data
```

## 5. GST Analysis

### Maximum Likelihood Estimation

GST finds the gate set that maximizes the likelihood of observed data:

<Formula>
$$\mathcal{G}^* = \arg\max_{\mathcal{G}} \prod_{c} P(n_c | \mathcal{G})$$
</Formula>

where $n_c$ are the observed counts for circuit $c$.

### Gauge Optimization

GST results have gauge freedom - the same physical predictions can arise from different mathematical representations:

<Formula>
$$\mathcal{G}' = \{B\rho, MB^{-1}, BG_iB^{-1}\}$$
</Formula>

## 6. Interpreting GST Results

### Process Matrices

Each gate is characterized by its process matrix (Pauli transfer matrix):

<Formula>
$$G_{ij} = \frac{1}{d}\text{Tr}(P_i \cdot G(P_j))$$
</Formula>

where $P_i$ are Pauli operators and $G(\cdot)$ is the gate's action.

### Error Generators

Errors are analyzed via the error generator:

<Formula>
$$G = G_{target} \cdot e^{L}$$
</Formula>

where $L$ is the error generator with:

<Formula>
$$L = H + S + A$$
</Formula>

- $H$: Hamiltonian (coherent) errors
- $S$: Stochastic errors
- $A$: Affine (SPAM-like) errors

| Amplitude damping | Asymmetric S | T1 relaxation |

## 7. GST vs RB: When to Use Which

<Tip>
Use RB for quick benchmarking and monitoring. Use GST when you need detailed error information for calibration or error model development.
</Tip>

### Use Randomized Benchmarking When:
- You need a quick overall fidelity metric
- Monitoring gate quality over time
- Comparing different devices/configurations
- Resources (time, measurements) are limited

### Use Gate Set Tomography When:
- Diagnosing specific error mechanisms
- Developing error models for simulation
- Optimizing gate calibration
- Validating theoretical error predictions
- Need complete process matrix information

## 8. Practical Considerations

### Computational Resources

GST analysis involves optimizing over many parameters:
- Single qubit: ~40 parameters
- Two qubits: ~1000+ parameters

```python
def estimate_gst_resources(num_qubits, max_length, num_shots):
    """
    Estimate resources needed for GST.
    """
    if num_qubits == 1:
        n_params = 44  # Approximate
        n_circuits = 3456 * (max_length / 128)  # Scales with max_length
        n_measurements = n_circuits * num_shots
        compute_time_hours = 0.5  # Rough estimate

    elif num_qubits == 2:
        n_params = 1200  # Approximate
        n_circuits = 50000 * (max_length / 64)
        n_measurements = n_circuits * num_shots
        compute_time_hours = 24  # Can take much longer

    return {
        'parameters': n_params,
        'circuits': n_circuits,
        'total_measurements': n_measurements,
        'compute_time_hours': compute_time_hours
    }
```

## 9. Summary

Gate Set Tomography provides the most complete characterization of quantum gates:

1. **Self-consistent**: No assumptions about SPAM quality
2. **Complete**: Full process matrices for all gates
3. **Diagnostic**: Identifies specific error mechanisms
4. **Rigorous**: Based on maximum likelihood estimation

Key components:
- **Fiducials**: Create informationally complete state/measurement sets
- **Germs**: Amplify all error modes
- **Gauge optimization**: Match results to target gates

Trade-offs:
- More measurements than RB
- More computational analysis
- Provides much more detailed information

<Tip>
For routine characterization, use RB. When you need to understand WHY gates are failing or to develop accurate error models, use GST.
</Tip>
