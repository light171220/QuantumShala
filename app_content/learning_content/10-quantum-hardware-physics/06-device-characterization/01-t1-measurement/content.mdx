## 1. Introduction to Energy Relaxation

Energy relaxation, characterized by the time constant T1, is one of the most fundamental metrics for qubit quality. It describes how quickly a qubit in the excited state |1> spontaneously decays to the ground state |0>.

<Definition term="T1 (Energy Relaxation Time)">
T1 is the characteristic time constant for energy relaxation from the excited state to the ground state. After time T1, the probability of remaining in the excited state decreases to approximately 1/e (37%) of its initial value.
</Definition>

The T1 process is analogous to spontaneous emission in atomic physics but occurs through different mechanisms in solid-state qubits.

## 2. Physics of T1 Relaxation

### The Decay Process

When a qubit is prepared in the excited state |1>, it will spontaneously decay according to:

<Formula>
$$P_1(t) = P_1(0) \cdot e^{-t/T_1}$$
</Formula>

where $P_1(t)$ is the probability of measuring the qubit in state |1> at time $t$.

### Fermi's Golden Rule

The relaxation rate $\Gamma_1 = 1/T_1$ is determined by Fermi's Golden Rule:

<Formula>
$$\Gamma_1 = \frac{2\pi}{\hbar} |V_{01}|^2 \rho(E_{01})$$
</Formula>

where:
- $V_{01}$ is the matrix element coupling |1> to |0>
- $\rho(E_{01})$ is the density of states of the environment at the qubit frequency
- $E_{01} = \hbar\omega_{01}$ is the qubit transition energy

### Sources of T1 Relaxation

<Tip>
Understanding what limits T1 is crucial for improving qubit coherence. Different noise sources dominate in different frequency and temperature regimes.
</Tip>

#### 1. Purcell Effect
Decay through the readout resonator:

<Formula>
$$\Gamma_{\text{Purcell}} = \frac{g^2}{\Delta^2} \kappa$$
</Formula>

where:
- $g$ is the qubit-resonator coupling strength
- $\Delta = \omega_r - \omega_q$ is the qubit-resonator detuning
- $\kappa$ is the resonator decay rate

#### 2. Dielectric Loss
Energy dissipation in the substrate and interfaces:

<Formula>
$$\Gamma_{\text{dielectric}} = \omega_{01} \cdot p_{\text{surf}} \cdot \tan\delta$$
</Formula>

where $p_{\text{surf}}$ is the participation ratio of surface dielectrics and $\tan\delta$ is the loss tangent.

#### 3. Quasiparticle Tunneling
Non-equilibrium quasiparticles in the superconductor:

<Formula>
$$\Gamma_{\text{qp}} \propto x_{\text{qp}} \sqrt{\frac{2\Delta}{\hbar\omega_{01}}}$$
</Formula>

where $x_{\text{qp}}$ is the normalized quasiparticle density.

#### 4. Flux Noise (for Tunable Qubits)
Noise in the magnetic flux threading the SQUID loop:

<Formula>
$$\Gamma_{\text{flux}} = \left(\frac{\partial\omega_{01}}{\partial\Phi}\right)^2 S_\Phi(\omega_{01})$$
</Formula>

## 3. T1 Measurement Protocol

### Basic Pulse Sequence

The T1 measurement follows a simple protocol:

```
1. Initialize qubit in |0> (thermal relaxation or active reset)
2. Apply X gate (pi pulse) to excite to |1>
3. Wait for variable delay time t
4. Measure the qubit state
5. Repeat for statistics
6. Vary delay time t and fit exponential decay
```

<Example>
A typical T1 measurement sequence in Python:

```python
import numpy as np
from scipy.optimize import curve_fit

def t1_measurement_sequence(qubit, delays, num_shots=1000):
    """
    Perform T1 measurement on a qubit.

    Args:
        qubit: Qubit object with gate and measurement methods
        delays: Array of delay times in seconds
        num_shots: Number of measurement repetitions per delay

    Returns:
        Dictionary with delays, probabilities, and fitted T1
    """
    p1_values = []

    for delay in delays:
        # Count excited state measurements
        excited_counts = 0

        for _ in range(num_shots):
            # Reset to ground state
            qubit.reset()

            # Apply pi pulse to excite
            qubit.x()

            # Wait for delay time
            qubit.delay(delay)

            # Measure
            result = qubit.measure()
            if result == 1:
                excited_counts += 1

        # Calculate probability
        p1 = excited_counts / num_shots
        p1_values.append(p1)

    return np.array(delays), np.array(p1_values)

# Define exponential decay model
def exponential_decay(t, p1_0, t1, offset):
    """T1 decay model: P1(t) = P1(0) * exp(-t/T1) + offset"""
    return p1_0 * np.exp(-t / t1) + offset

# Example: Fit T1 from measurement data
delays = np.linspace(0, 500e-6, 50)  # 0 to 500 microseconds
# Simulated data with T1 = 100 us
true_t1 = 100e-6
p1_data = 0.95 * np.exp(-delays / true_t1) + 0.02 + 0.02 * np.random.randn(len(delays))

# Fit the data
popt, pcov = curve_fit(
    exponential_decay,
    delays,
    p1_data,
    p0=[0.95, 100e-6, 0.02],
    bounds=([0, 0, 0], [1, 1, 0.5])
)

fitted_t1 = popt[1]
t1_error = np.sqrt(pcov[1, 1])

print(f"Fitted T1: {fitted_t1 * 1e6:.2f} +/- {t1_error * 1e6:.2f} microseconds")
```
</Example>

### Measurement Considerations

<Warning>
Several factors can bias T1 measurements:
- **Readout errors**: Finite readout fidelity affects measured P1
- **State preparation errors**: Imperfect X gate doesn't fully excite
- **Thermal population**: Non-zero temperature gives residual |1> population
- **Heating during measurement**: Long sequences can heat the qubit
</Warning>

### Correcting for Readout Errors

The measured probabilities include readout errors:

<Formula>
$$\begin{pmatrix} P_0^{\text{meas}} \\ P_1^{\text{meas}} \end{pmatrix} =
\begin{pmatrix} 1-\epsilon_{0\to1} & \epsilon_{1\to0} \\ \epsilon_{0\to1} & 1-\epsilon_{1\to0} \end{pmatrix}
\begin{pmatrix} P_0^{\text{true}} \\ P_1^{\text{true}} \end{pmatrix}$$
</Formula>

Inverting this confusion matrix gives the corrected probabilities.

## 4. Exponential Fitting Techniques

### Simple Exponential Fit

For high-quality data, a simple exponential fit works well:

<Formula>
$$P_1(t) = A \cdot e^{-t/T_1} + B$$
</Formula>

Parameters:
- $A$: Initial excited state probability (ideally ~1)
- $T_1$: Energy relaxation time
- $B$: Offset (readout error + thermal population)

### Stretched Exponential (for Non-Markovian Noise)

When T1 fluctuates in time, use a stretched exponential:

<Formula>
$$P_1(t) = A \cdot e^{-(t/T_1)^\beta} + B$$
</Formula>

where $\beta < 1$ indicates T1 fluctuations (non-Markovian behavior).

```python
import numpy as np
from scipy.optimize import curve_fit

def stretched_exponential(t, A, t1, beta, B):
    """Stretched exponential for fluctuating T1"""
    return A * np.exp(-(t / t1)**beta) + B

def fit_t1_stretched(delays, p1_data):
    """
    Fit T1 with stretched exponential model.

    Returns T1, beta parameter, and uncertainties.
    """
    # Initial guesses
    p0 = [0.9, np.median(delays), 1.0, 0.05]
    bounds = ([0, 0, 0.1, 0], [1, np.max(delays)*10, 2, 0.5])

    popt, pcov = curve_fit(
        stretched_exponential,
        delays, p1_data,
        p0=p0, bounds=bounds
    )

    A, t1, beta, B = popt
    errors = np.sqrt(np.diag(pcov))

    return {
        'T1': t1,
        'T1_error': errors[1],
        'beta': beta,
        'beta_error': errors[2],
        'amplitude': A,
        'offset': B
    }

# Interpretation
# beta = 1: Standard exponential (Markovian)
# beta < 1: T1 fluctuations present (non-Markovian)
# beta > 1: Can indicate measurement artifacts
```

### Maximum Likelihood Estimation

For rigorous error estimation, use maximum likelihood:

```python
import numpy as np
from scipy.optimize import minimize

def negative_log_likelihood(params, delays, counts_1, counts_total):
    """
    Negative log-likelihood for binomial T1 data.

    Args:
        params: [A, T1, B] fitting parameters
        delays: Delay times
        counts_1: Number of |1> measurements at each delay
        counts_total: Total measurements at each delay
    """
    A, t1, B = params

    # Predicted probabilities
    p1_pred = A * np.exp(-delays / t1) + B

    # Clip to avoid log(0)
    p1_pred = np.clip(p1_pred, 1e-10, 1 - 1e-10)

    # Binomial log-likelihood
    ll = np.sum(
        counts_1 * np.log(p1_pred) +
        (counts_total - counts_1) * np.log(1 - p1_pred)
    )

    return -ll

def fit_t1_mle(delays, counts_1, counts_total):
    """Maximum likelihood T1 fit with proper error bars."""

    # Initial guess
    x0 = [0.9, np.median(delays), 0.05]
    bounds = [(0.1, 1.0), (delays[1], delays[-1]*5), (0, 0.3)]

    result = minimize(
        negative_log_likelihood,
        x0,
        args=(delays, counts_1, counts_total),
        bounds=bounds,
        method='L-BFGS-B'
    )

    # Estimate errors from Hessian
    from scipy.optimize import approx_fprime

    def hessian_diag(params):
        eps = 1e-8
        hess = []
        for i in range(len(params)):
            def f(p):
                params_copy = params.copy()
                params_copy[i] = p
                return negative_log_likelihood(params_copy, delays, counts_1, counts_total)

            h = (f(params[i] + eps) - 2*f(params[i]) + f(params[i] - eps)) / eps**2
            hess.append(h)
        return np.array(hess)

    hess = hessian_diag(result.x)
    errors = 1.0 / np.sqrt(np.abs(hess))

    return {
        'T1': result.x[1],
        'T1_error': errors[1],
        'A': result.x[0],
        'B': result.x[2]
    }
```

## 5. T1 Measurement Best Practices

### Choosing Delay Points

<Tip>
Optimal delay points are distributed logarithmically, with more points near T1:
</Tip>

```python
def optimal_delay_points(t1_estimate, num_points=50):
    """
    Generate optimal delay points for T1 measurement.

    Points are concentrated around T1 where sensitivity is highest.
    """
    # Span 0 to 5*T1 (captures 99.3% of decay)
    t_min = t1_estimate / 100
    t_max = t1_estimate * 5

    # Logarithmic spacing with extra points near T1
    log_points = np.logspace(
        np.log10(t_min),
        np.log10(t_max),
        num_points
    )

    return log_points
```

### Number of Shots

The statistical error in T1 scales as:

<Formula>
$$\sigma_{T_1} \propto \frac{T_1}{\sqrt{N \cdot M}}$$
</Formula>

where $N$ is shots per point and $M$ is number of delay points.

For 1% precision: Use ~1000 shots per point with ~50 delay points.

### Monitoring T1 Stability

```python
import numpy as np
import matplotlib.pyplot as plt
from datetime import datetime

class T1Monitor:
    """Monitor T1 stability over time."""

    def __init__(self):
        self.timestamps = []
        self.t1_values = []
        self.t1_errors = []

    def add_measurement(self, t1, t1_error):
        """Record a T1 measurement."""
        self.timestamps.append(datetime.now())
        self.t1_values.append(t1)
        self.t1_errors.append(t1_error)

    def get_statistics(self):
        """Calculate T1 statistics."""
        t1_array = np.array(self.t1_values)
        return {
            'mean': np.mean(t1_array),
            'std': np.std(t1_array),
            'min': np.min(t1_array),
            'max': np.max(t1_array),
            'cv': np.std(t1_array) / np.mean(t1_array)  # Coefficient of variation
        }

    def detect_drift(self, window=10):
        """Detect if T1 is drifting."""
        if len(self.t1_values) < window * 2:
            return None

        recent = np.mean(self.t1_values[-window:])
        previous = np.mean(self.t1_values[-2*window:-window])

        change_percent = (recent - previous) / previous * 100

        return {
            'drift_percent': change_percent,
            'significant': abs(change_percent) > 10
        }
```

## 6. Interpreting T1 Results

### T1 and Gate Fidelity

The T1-limited gate error for a gate of duration $t_g$ is:

<Formula>
$$\epsilon_{T_1} \approx \frac{t_g}{2 T_1}$$
</Formula>

<Example>
For T1 = 100 microseconds and gate time = 30 ns:

$$\epsilon_{T_1} = \frac{30 \times 10^{-9}}{2 \times 100 \times 10^{-6}} = 1.5 \times 10^{-4} = 0.015\%$$

This sets a fundamental limit on single-gate fidelity from T1 alone.
</Example>

### T1 Quality Factor

A useful figure of merit is the number of gate operations possible within T1:

<Formula>
$$Q_{T_1} = \frac{T_1}{t_{\text{gate}}}$$
</Formula>

Modern transmons achieve $Q_{T_1} > 3000$.

### Thermal Equilibrium

At finite temperature, the qubit reaches thermal equilibrium:

<Formula>
$$P_1^{\text{eq}} = \frac{1}{1 + e^{\hbar\omega_{01}/k_B T}}$$
</Formula>

At 20 mK with a 5 GHz qubit: $P_1^{\text{eq}} \approx 0.1\%$

## 7. Summary

T1 measurement is foundational for qubit characterization:

1. **Simple protocol**: X gate, variable delay, measurement
2. **Exponential fitting**: Extract T1 from decay curve
3. **Error analysis**: Use proper statistical methods
4. **Physical understanding**: Know what limits T1
5. **Quality metrics**: T1 determines gate error floor

Key values for state-of-the-art transmons:
- T1: 100-500 microseconds
- T1-limited gate error: < 0.01%
- T1 stability: < 10% variation over hours

<Tip>
Regular T1 monitoring is essential for quantum computer operation. T1 fluctuations can indicate cosmic ray impacts, two-level system activity, or changes in the electromagnetic environment.
</Tip>
