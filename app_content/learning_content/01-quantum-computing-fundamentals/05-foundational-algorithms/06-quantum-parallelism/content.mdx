## 1. What is Quantum Parallelism?

<Definition term="Quantum Parallelism">
The ability of a quantum computer to evaluate a function on all possible inputs simultaneously by applying it to a superposition state.
</Definition>

```python
import numpy as np

# Classical: evaluate f(x) for each x separately
def classical_evaluate(f, n):
    results = []
    for x in range(2**n):
        results.append(f(x))
    return results  # 2^n function calls

# Quantum: one application of Uf evaluates all
def quantum_evaluate_concept(f, n):
    """Conceptual - shows the idea."""
    # Create superposition of all inputs
    # |ψ⟩ = (1/√2^n) Σx |x⟩
    # Apply Uf once:
    # Uf|ψ⟩|0⟩ = (1/√2^n) Σx |x⟩|f(x)⟩
    print(f"Quantum: 1 application computes f on all {2**n} inputs!")

quantum_evaluate_concept(lambda x: x**2, 10)
```

## 2. The Limitation

While we compute f(x) for all x in one step, we can only extract ONE answer upon measurement!

```python
print("The catch:")
print("  - Compute: All 2^n values of f(x)")
print("  - Extract: Only ONE random x with f(x)")
print()
print("Solution: Use interference to make desired answer probable!")
```

## 3. Making Parallelism Useful

Quantum algorithms use interference to amplify useful information:

```python
print("Quantum Algorithm Recipe:")
print("1. Create superposition (parallelism)")
print("2. Apply oracle (compute on all inputs)")
print("3. Apply interference (amplify answers)")
print("4. Measure (extract amplified answer)")
```

## Summary

- Quantum parallelism evaluates f(x) for all x simultaneously
- Measurement collapses to one result
- Interference is key to extracting useful information
