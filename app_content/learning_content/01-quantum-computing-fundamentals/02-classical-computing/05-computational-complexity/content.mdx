## 1. Why Complexity Matters

Not all problems are equally hard to solve. Understanding complexity helps us:
- Choose efficient algorithms
- Identify problems computers can't solve quickly
- Find where quantum computers offer advantages

<Definition term="Computational Complexity">
The study of resources (time, space) required to solve computational problems as input size grows.
</Definition>

## 2. Big-O Notation

Big-O describes how algorithm runtime grows with input size n.

<Formula>
$$O(f(n))$$ means runtime grows at most proportionally to $f(n)$ for large $n$.
</Formula>

| Complexity | Name | Example |
|------------|------|---------|
| O(1) | Constant | Array access |
| O(log n) | Logarithmic | Binary search |
| O(n) | Linear | Linear search |
| O(n log n) | Linearithmic | Merge sort |
| O(n²) | Quadratic | Bubble sort |
| O(2ⁿ) | Exponential | Brute-force subset |

<Example>
```python
import time
import numpy as np

def measure_time(func, n):
    start = time.time()
    func(n)
    return time.time() - start

# O(1) - Constant
def constant_time(n):
    return 42

# O(n) - Linear
def linear_time(n):
    total = 0
    for i in range(n):
        total += i
    return total

# O(n²) - Quadratic
def quadratic_time(n):
    total = 0
    for i in range(n):
        for j in range(n):
            total += 1
    return total

# O(2^n) - Exponential
def exponential_time(n):
    if n <= 1:
        return 1
    return exponential_time(n-1) + exponential_time(n-2)

print("Runtime comparison:")
for n in [10, 100, 1000]:
    t_const = measure_time(constant_time, n)
    t_linear = measure_time(linear_time, n)
    t_quad = measure_time(quadratic_time, n) if n <= 1000 else "too slow"
    print(f"n={n}: O(1)={t_const:.6f}s, O(n)={t_linear:.6f}s, O(n²)={t_quad}")
```
</Example>

## 3. Complexity Class P

<Definition term="Class P">
P (Polynomial time) contains problems solvable in O(n^k) time for some constant k. These are considered "efficiently solvable."
</Definition>

**Examples in P:**
- Sorting: O(n log n)
- Shortest path: O(n²) with Dijkstra
- Matrix multiplication: O(n³) naive, O(n^2.37) best known
- Linear programming

## 4. Complexity Class NP

<Definition term="Class NP">
NP (Nondeterministic Polynomial) contains problems where a solution can be *verified* in polynomial time, even if *finding* the solution might be hard.
</Definition>

**Key insight:** P ⊆ NP (if you can solve it fast, you can verify it fast)

**Examples in NP:**
- **Factoring**: Hard to find factors, easy to verify: just multiply
- **SAT**: Hard to find satisfying assignment, easy to check
- **Traveling Salesman**: Hard to find shortest tour, easy to verify length

<Example>
```python
import random
import time

def is_factor(n, f):
    """Verify if f is a factor of n - O(1)"""
    return n % f == 0

def find_factor_naive(n):
    """Find a factor - O(√n) at best"""
    for i in range(2, int(n**0.5) + 1):
        if n % i == 0:
            return i
    return None

# Verification is easy
n = 1000000007 * 1000000009  # Product of two primes
print(f"n = {n}")

# Verification: instant
start = time.time()
is_valid = is_factor(n, 1000000007)
verify_time = time.time() - start
print(f"Verification time: {verify_time:.6f}s")

# Finding: takes longer
start = time.time()
factor = find_factor_naive(n)
find_time = time.time() - start
print(f"Finding factor time: {find_time:.4f}s")
print(f"Ratio: {find_time/verify_time:.0f}x slower to find than verify")
```
</Example>

## 5. NP-Complete Problems

<Definition term="NP-Complete">
The "hardest" problems in NP. If any NP-complete problem has a polynomial solution, then P = NP.
</Definition>

**Famous NP-complete problems:**
- **SAT**: Boolean satisfiability
- **3-SAT**: SAT with 3 literals per clause
- **Traveling Salesman** (decision version)
- **Graph Coloring**
- **Subset Sum**

## 6. The P vs NP Problem

The biggest open question in computer science:

<Formula>
$$\text{P} \stackrel{?}{=} \text{NP}$$
</Formula>

**If P = NP:** Many hard problems become easy!
**If P ≠ NP:** Some problems are inherently hard (most believe this)

**Why it matters:**
- Cryptography relies on P ≠ NP (factoring is hard)
- Many optimization problems would become tractable
- $1 million Millennium Prize!

## 7. Beyond NP: Harder Classes

| Class | Description | Example |
|-------|-------------|---------|
| P | Solvable in poly time | Sorting |
| NP | Verifiable in poly time | Factoring |
| PSPACE | Poly space | Game strategy |
| EXP | Exponential time | Chess (optimal) |
| BQP | Quantum poly time | Shor's algorithm |

<Example>
```python

classes = {

```
</Example>

## 8. Quantum Complexity: BQP

<Definition term="BQP">
Bounded-error Quantum Polynomial time - problems solvable by quantum computers in polynomial time with error < 1/3.
</Definition>

**Key relationships:**
- P ⊆ BQP (quantum can simulate classical)
- BQP ⊆ PSPACE (quantum can be simulated classically with exponential time but polynomial space)
- Unknown: Is NP ⊆ BQP? (Probably not!)

**Problems in BQP but (probably) not in P:**
- **Factoring** (Shor's algorithm)
- **Discrete log** (Shor's algorithm)
- **Simulation of quantum systems**

## 9. Quantum Speedups

| Problem | Classical | Quantum | Speedup |
|---------|-----------|---------|---------|
| Search | O(N) | O(√N) | Quadratic |
| Factoring | O(exp(n^⅓)) | O(n³) | Exponential |
| Simulation | O(2ⁿ) | O(n) | Exponential |
| Optimization | Varies | Varies | Problem-dependent |

<Warning>
Quantum computers don't solve NP-complete problems efficiently! Grover's search gives only quadratic speedup, not exponential.
</Warning>

## Summary

In this lesson, you learned:

1. **Big-O notation** describes algorithm efficiency
2. **Class P** = efficiently solvable problems
3. **Class NP** = efficiently verifiable problems
4. **P vs NP** is the biggest open problem in CS
5. **BQP** is the quantum complexity class
6. **Quantum speedups** are exponential for some problems (factoring), but only quadratic for search

<Tip>
Understanding complexity classes helps identify where quantum computing can help - exponential speedups exist for structured problems like factoring, but not for general NP-complete problems!
</Tip>
