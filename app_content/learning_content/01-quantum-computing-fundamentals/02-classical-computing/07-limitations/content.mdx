## 1. Physical Limits: Moore's Law and Beyond

<Definition term="Moore's Law">
The observation that transistor density doubles approximately every 2 years, leading to exponential growth in computing power. This trend is now slowing due to physical limits.
</Definition>

**The problem:** We're approaching atomic scales!

| Year | Transistor Size | Atoms Across |
|------|-----------------|--------------|
| 2000 | 130 nm | ~500 atoms |
| 2010 | 32 nm | ~120 atoms |
| 2020 | 5 nm | ~20 atoms |
| 2024 | 3 nm | ~12 atoms |

<Warning>
Below ~5nm, quantum effects like electron tunneling become significant, causing classical transistors to malfunction. This is actually one motivation for embracing quantum computing!
</Warning>

<Example>
```python
import numpy as np

def moores_law_projection(start_year=1970, start_transistors=2300):
    """Project transistor counts using Moore's Law."""
    years = np.arange(start_year, 2035, 2)
    transistors = start_transistors * (2 ** ((years - start_year) / 2))
    
    print("Moore's Law Projection:")
    print(f"{'Year':>6} | {'Transistors':>15} | {'Size (nm)':>10}")
    print("-" * 40)
    
    # Approximate size scaling (inverse relationship)
    base_size = 10000  # nm in 1970
    for y, t in zip(years, transistors):
        size = base_size / (t / start_transistors) ** 0.5
        size = max(size, 1)  # Physical limit
        print(f"{y:>6} | {t:>15,.0f} | {size:>10.1f}")

moores_law_projection()

# Physical limits
print("\n=== Physical Limits ===")
atom_diameter = 0.1  # nm (silicon)
print(f"Silicon atom diameter: {atom_diameter} nm")
print(f"At 3nm process: ~{3/atom_diameter:.0f} atoms across")
print(f"At 1nm process: ~{1/atom_diameter:.0f} atoms across")
print("Quantum tunneling becomes dominant below ~5nm!")
```
</Example>

## 2. Undecidable Problems

Some problems cannot be solved by ANY computer, regardless of speed!

<Definition term="Undecidability">
A problem is undecidable if no algorithm can always produce a correct yes/no answer in finite time.
</Definition>

**The Halting Problem:** Given a program and input, determine if it will halt or run forever.

<Example>
```python
def halting_paradox():
    """
    Proof that the halting problem is undecidable.
    
    Assume we have a function halts(program, input) that correctly
    determines if program(input) halts.
    
    Consider this paradoxical program:
    """
    
    def paradox(x):
        """
        If halts(paradox, paradox) returns True, loop forever.
        If halts(paradox, paradox) returns False, halt.
        """
        # This creates a contradiction!
        # If halts says we halt, we don't halt.
        # If halts says we don't halt, we do halt.
        pass
    
    print("The Halting Problem Paradox:")
    print("1. Assume halts(P, I) correctly determines if P(I) halts")
    print("2. Define paradox(x): loop forever if halts(paradox, paradox), else halt")
    print("3. What does halts(paradox, paradox) return?")
    print("   - If True: paradox loops, so halts was WRONG")
    print("   - If False: paradox halts, so halts was WRONG")
    print("4. Contradiction! No such halts() can exist.")

halting_paradox()
```
</Example>

**Other undecidable problems:**
- Rice's Theorem: Any non-trivial property of programs is undecidable
- Post Correspondence Problem
- Tiling problems

## 3. Intractable Problems

Some problems ARE solvable, but require exponential time.

<Definition term="Intractable">
A problem is intractable if the best known algorithm requires exponential time O(2^n) or worse.
</Definition>

<Example>
```python
import time
import itertools

def brute_force_tsp(cities: int) -> tuple:
    """
    Traveling Salesman Problem - brute force.
    Check all n! permutations.
    """
    # Generate random distances
    import random
    random.seed(42)
    dist = [[random.randint(1, 100) for _ in range(cities)] 
            for _ in range(cities)]
    
    start = time.time()
    
    best_length = float('inf')
    best_path = None
    count = 0
    
    for perm in itertools.permutations(range(cities)):
        count += 1
        length = sum(dist[perm[i]][perm[i+1]] for i in range(cities-1))
        length += dist[perm[-1]][perm[0]]  # Return to start
        if length < best_length:
            best_length = length
            best_path = perm
    
    elapsed = time.time() - start
    return best_path, best_length, count, elapsed

print("TSP Brute Force Complexity:")
print(f"{'Cities':>8} | {'Permutations':>15} | {'Time (s)':>12}")
print("-" * 42)

for n in [6, 7, 8, 9, 10]:
    path, length, count, elapsed = brute_force_tsp(n)
    print(f"{n:>8} | {count:>15,} | {elapsed:>12.4f}")

# Projection
print("\nProjection for larger inputs:")
import math
for n in [15, 20, 25, 30]:
    perms = math.factorial(n)
    # Assume 10^9 permutations/second
    seconds = perms / 1e9
    years = seconds / (365.25 * 24 * 3600)
    print(f"  {n} cities: {perms:.2e} permutations = {years:.2e} years")
```
</Example>

## 4. Limits of Parallelism

Can we just add more processors?

<Definition term="Amdahl's Law">
The speedup from parallelization is limited by the sequential portion of the program:
$$\text{Speedup} = \frac{1}{(1-P) + \frac{P}{N}}$$
where P is the parallelizable fraction and N is the number of processors.
</Definition>

<Example>
```python
def amdahls_law(parallel_fraction: float, num_processors: int) -> float:
    """Calculate speedup using Amdahl's Law."""
    sequential = 1 - parallel_fraction
    return 1 / (sequential + parallel_fraction / num_processors)

print("Amdahl's Law - Parallelism Limits")
print(f"{'Processors':>12} | {'95% parallel':>14} | {'99% parallel':>14}")
print("-" * 46)

for n in [2, 4, 8, 16, 64, 256, 1000, float('inf')]:
    s95 = amdahls_law(0.95, n)
    s99 = amdahls_law(0.99, n)
    n_str = "∞" if n == float('inf') else str(int(n))
    print(f"{n_str:>12} | {s95:>14.2f}x | {s99:>14.2f}x")

print("\nKey insight: Even with infinite processors:")
print(f"  95% parallel → max {amdahls_law(0.95, float('inf')):.1f}x speedup")
print(f"  99% parallel → max {amdahls_law(0.99, float('inf')):.1f}x speedup")
print("The sequential 1-5% becomes the bottleneck!")
```
</Example>

## 5. Quantum Simulation Barrier

Simulating quantum systems is exponentially hard for classical computers.

<Example>
```python
def quantum_simulation_cost(qubits: int) -> dict:
    """
    Calculate classical resources needed to simulate quantum systems.
    State vector: 2^n complex amplitudes
    Each amplitude: 16 bytes (complex128)
    """
    state_size = 2 ** qubits
    memory_bytes = state_size * 16
    
    # Convert to human-readable
    units = ['B', 'KB', 'MB', 'GB', 'TB', 'PB', 'EB', 'ZB']
    size = memory_bytes
    unit_idx = 0
    while size >= 1024 and unit_idx < len(units) - 1:
        size /= 1024
        unit_idx += 1
    
    return {
        'qubits': qubits,
        'amplitudes': state_size,
        'memory': f"{size:.1f} {units[unit_idx]}",
        'memory_bytes': memory_bytes
    }

print("Classical Resources to Simulate Quantum Systems:")
print(f"{'Qubits':>8} | {'State Vector Size':>20} | {'Memory Required':>15}")
print("-" * 52)

for n in [10, 20, 30, 40, 50, 60, 100]:
    result = quantum_simulation_cost(n)
    print(f"{n:>8} | {result['amplitudes']:>20,} | {result['memory']:>15}")

print("\nContext:")
print("• 40 qubits: Exceeds most supercomputer memory")
print("• 50 qubits: 'Quantum supremacy' threshold")
print("• 100 qubits: More states than atoms in universe!")
```
</Example>

<Warning>
This exponential scaling is why we NEED quantum computers to simulate quantum systems efficiently. A 100-qubit quantum computer can simulate what would take more memory than exists in the observable universe!
</Warning>

## 6. The Landauer Limit

Even energy has fundamental limits!

<Definition term="Landauer's Principle">
Erasing one bit of information requires a minimum energy of kT ln(2), where k is Boltzmann's constant and T is temperature.
</Definition>

<Example>
```python
import math

# Physical constants
k_B = 1.380649e-23  # Boltzmann constant (J/K)
T = 300  # Room temperature (K)

landauer_limit = k_B * T * math.log(2)

print("Landauer's Limit on Computation:")
print(f"At room temperature (300K):")
print(f"  Minimum energy per bit erasure: {landauer_limit:.4e} J")
print(f"  That's {landauer_limit * 6.242e18:.4f} meV")

# Compare to current technology
current_per_bit = 1e-15  # ~1 femtojoule (modern CPUs)
efficiency_ratio = current_per_bit / landauer_limit

print(f"\nCurrent CPUs use ~{current_per_bit:.0e} J/bit")
print(f"That's {efficiency_ratio:.0f}x the theoretical minimum")
print(f"Room for improvement: {math.log10(efficiency_ratio):.1f} orders of magnitude")

# Reversible computing note
print("\nNote: Landauer limit applies to IRREVERSIBLE operations")
print("Reversible computing (like quantum!) can theoretically avoid this limit!")
```
</Example>

## 7. Why These Limits Matter for Quantum Computing

| Classical Limitation | Quantum Solution |
|---------------------|------------------|
| Transistor size limits | Qubits use quantum effects |
| Exponential state spaces | Superposition handles naturally |
| Sequential algorithms | Quantum parallelism |
| Simulation costs | Native quantum simulation |
| Irreversible computation | Unitary (reversible) gates |

## Summary

Classical computing faces fundamental barriers:

1. **Physical limits**: Moore's Law ending, quantum tunneling at small scales
2. **Undecidability**: Some problems impossible to solve algorithmically
3. **Intractability**: Many problems require exponential time
4. **Parallelism limits**: Amdahl's Law caps speedup
5. **Simulation barrier**: Quantum systems exponentially hard to simulate
6. **Energy limits**: Landauer's principle bounds efficiency

<Tip>
These aren't just engineering challenges - they're fundamental limits! Quantum computing offers a new paradigm that works WITH quantum mechanics rather than fighting against it.
</Tip>
