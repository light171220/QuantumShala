## 1. The IPEA Concept

Standard QPE uses t counting qubits simultaneously. IPEA uses 1 counting qubit, measured t times with feedback.

<Definition term="Iterative QPE (IPEA)">
Extract phase bits one at a time, from least significant to most significant, using mid-circuit measurement and phase correction.
</Definition>

```python
import numpy as np

def ipea_step(phi, bit_position, previous_bits):
    """
    Simulate one step of iterative QPE.
    
    Returns: measured bit at this position
    """
    # Phase accumulated from previous bits
    correction = sum(b * 2**(-k-1) for k, b in enumerate(previous_bits))
    
    # Effective phase for this measurement
    effective_phi = (2**bit_position) * phi - correction
    
    # Probability of measuring |1⟩
    prob_1 = (1 + np.cos(2 * np.pi * effective_phi)) / 2
    
    # Simulate measurement
    measured = 1 if np.random.random() < prob_1 else 0
    return measured

# Example: estimate φ = 0.375 = 3/8
phi = 0.375
t = 5  # bits of precision

bits = []
for k in range(t):
    bit = ipea_step(phi, k, bits)
    bits.append(bit)

estimated = sum(b * 2**(-k-1) for k, b in enumerate(bits))
print(f"True φ = {phi}")
print(f"Bits: {bits}")
print(f"Estimated: {estimated}")
```

## 2. Space vs Time Tradeoff

| Method | Qubits | Measurements | Circuit Depth |
|--------|--------|--------------|---------------|
| Standard QPE | t + n | 1 | O(t × U_depth) |
| Iterative QPE | 1 + n | t | O(t × U_depth) |

IPEA trades space for time (multiple measurements vs multiple qubits).

## 3. When to Use IPEA

Use IPEA when:
- Qubits are scarce but measurements are cheap
- Mid-circuit measurement is available
- Circuit depth is a concern

Use standard QPE when:
- Parallel qubits are available
- Single-shot measurement is required

## Summary

- IPEA: 1 counting qubit, t measurements
- Uses classical feedback to correct phases
- Same precision as standard QPE
- Space-time tradeoff favors qubit-limited systems
