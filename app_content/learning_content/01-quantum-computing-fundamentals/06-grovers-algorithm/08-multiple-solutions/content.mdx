## 1. The Multi-Solution Setting

In many real problems, there are multiple valid solutions. For example:
- Database: multiple records matching a query
- SAT: multiple satisfying assignments
- Optimization: multiple optimal solutions

<Definition term="Multi-Solution Search">
Given a search space of size N with k marked items (solutions), find any one of the k solutions.
</Definition>

The good news: more solutions make the search **faster**, not harder!

```python
import numpy as np

def multi_solution_example():
    """Demonstrate multiple solutions in a search space."""
    N = 16  # 4 qubits
    solutions = [3, 7, 11]  # 3 valid solutions
    k = len(solutions)

    print(f"Search space size: N = {N}")
    print(f"Number of solutions: k = {k}")
    print(f"Solutions: {solutions}")
    print(f"Probability of random guess: {k/N:.4f} = {k}/{N}")

multi_solution_example()
```

## 2. Why Does the Formula Change?

Recall the geometry: the initial state |s> makes angle theta with the non-solution subspace, where:

<Formula>
$$\sin(\theta) = \sqrt{\frac{k}{N}}$$
</Formula>

With k solutions:
- The "solution subspace" is now k-dimensional
- The amplitude pointing toward solutions is sqrt(k/N), not 1/sqrt(N)
- Larger initial overlap means fewer iterations needed!

```python
import numpy as np

def compare_angles(N, k_values):
    """Compare initial angles for different k values."""
    print(f"Initial angles for N={N}:")
    print(f"{'k':>4} {'sin(theta)':>12} {'theta (deg)':>12} {'theta (rad)':>12}")
    print("-" * 45)

    for k in k_values:
        sin_theta = np.sqrt(k/N)
        theta = np.arcsin(sin_theta)
        print(f"{k:>4} {sin_theta:>12.6f} {np.degrees(theta):>12.2f} {theta:>12.6f}")

compare_angles(256, [1, 2, 4, 8, 16, 32])
```

## 3. Optimal Iterations for k Solutions

The optimal iteration count scales as:

<Formula>
$$r_{opt} \approx \frac{\pi}{4}\sqrt{\frac{N}{k}}$$
</Formula>

More precisely:

<Formula>
$$r_{opt} = \text{round}\left(\frac{\pi/2 - \theta}{2\theta}\right) \text{ where } \sin(\theta) = \sqrt{\frac{k}{N}}$$
</Formula>

```python
import numpy as np

def optimal_iterations_k(N, k):
    """Calculate optimal iterations for k solutions."""
    theta = np.arcsin(np.sqrt(k/N))
    r_exact = (np.pi/2 - theta) / (2 * theta)
    return int(round(r_exact)), theta

def compare_iterations(N):
    """Show how iterations decrease with more solutions."""
    print(f"Optimal iterations for N={N}:")
    print(f"{'k':>4} {'r_opt':>8} {'Speedup':>10} {'Factor':>10}")
    print("-" * 35)

    r_1, _ = optimal_iterations_k(N, 1)

    for k in [1, 2, 4, 8, 16, 32, 64]:
        if k > N/2:
            break
        r_opt, theta = optimal_iterations_k(N, k)
        speedup = r_1 / r_opt if r_opt > 0 else float('inf')
        factor = np.sqrt(k)
        print(f"{k:>4} {r_opt:>8} {speedup:>10.2f}x {factor:>10.2f}")

compare_iterations(256)
```

<Tip>
Doubling the number of solutions reduces iterations by a factor of sqrt(2) approximately equals 1.41.
</Tip>

## 4. Probability Distribution with k Solutions

When there are k solutions, each solution has approximately equal probability after measurement:

```python
import numpy as np

def multi_solution_grover(n_qubits, marked_items):
    """
    Run Grover's with multiple solutions.
    Returns final state and probabilities.
    """
    N = 2**n_qubits
    k = len(marked_items)

    # Calculate optimal iterations
    theta = np.arcsin(np.sqrt(k/N))
    r_opt = max(0, int(round((np.pi/2 - theta) / (2*theta))))

    # Initial state
    state = np.ones(N) / np.sqrt(N)

    # Oracle marking all solutions
    oracle = np.eye(N)
    for item in marked_items:
        oracle[item, item] = -1

    # Diffusion operator
    s = np.ones(N) / np.sqrt(N)
    diffusion = 2 * np.outer(s, s) - np.eye(N)

    # Apply iterations
    for _ in range(r_opt):
        state = diffusion @ (oracle @ state)

    # Calculate probabilities
    probs = np.abs(state)**2

    return state, probs, r_opt

# Example with 4 solutions
n = 4
N = 2**n
marked = [2, 5, 10, 13]

state, probs, r = multi_solution_grover(n, marked)

print(f"N={N}, k={len(marked)}, r_opt={r}")
print(f"\nProbability distribution:")
print(f"{'State':>6} {'Prob':>10} {'Type':>12}")
print("-" * 30)

for i in range(N):
    p = probs[i]
    if p > 0.01:
        typ = "SOLUTION" if i in marked else "non-solution"
        print(f"{i:>6} {p:>10.6f} {typ:>12}")

total_sol = sum(probs[i] for i in marked)
print(f"\nTotal solution probability: {total_sol:.6f}")
print(f"Per-solution probability: {total_sol/len(marked):.6f}")
```

## 5. Quantum Counting: Finding Unknown k

What if we don't know k? This is common in practice:
- How many database records match?
- How many SAT solutions exist?

<Definition term="Quantum Counting">
A quantum algorithm that estimates the number of solutions k using phase estimation on the Grover operator.
</Definition>

The key insight: the Grover operator G has eigenvalues e^(+/- 2i*theta), where sin(theta) = sqrt(k/N).

```python
import numpy as np

def quantum_counting_concept(N, k_actual):
    """
    Demonstrate the principle of quantum counting.
    (Simplified - actual implementation uses phase estimation)
    """
    theta = np.arcsin(np.sqrt(k_actual/N))

    # Grover eigenvalues
    eigenvalue_plus = np.exp(2j * theta)
    eigenvalue_minus = np.exp(-2j * theta)

    # Phase estimation would give us theta
    estimated_theta = theta  # (simulated perfect estimation)

    # From theta, recover k
    k_estimated = N * np.sin(estimated_theta)**2

    print(f"Quantum Counting (N={N}, actual k={k_actual}):")
    print(f"  True theta: {theta:.6f} rad = {np.degrees(theta):.2f} deg")
    print(f"  Eigenvalues: e^(+/- 2i*theta)")
    print(f"  |eigenvalue| = 1, phase = +/- {np.degrees(2*theta):.2f} deg")
    print(f"  Estimated k: {k_estimated:.2f}")

    return k_estimated

for k in [1, 4, 16]:
    quantum_counting_concept(256, k)
    print()
```

## 6. The Quantum Counting Algorithm

The full quantum counting algorithm:

1. Prepare Grover operator G as a controlled operation
2. Apply phase estimation to find eigenphases +/- 2*theta
3. Calculate k from theta: k = N * sin^2(theta)

```python
import numpy as np

def phase_estimation_simulation(N, k_true, precision_qubits):
    """
    Simulate phase estimation for quantum counting.

    Args:
        N: Search space size
        k_true: Actual number of solutions
        precision_qubits: Number of qubits for precision

    Returns:
        Estimated k value
    """
    theta_true = np.arcsin(np.sqrt(k_true/N))

    # Phase estimation gives theta with precision 1/2^m
    precision = 2**precision_qubits

    # Simulate phase estimation (in reality this is quantum)
    # The estimated phase is 2*theta (mod 2*pi)
    phase_true = 2 * theta_true

    # Discretize to precision
    phase_discrete = round(phase_true * precision / (2*np.pi)) * (2*np.pi) / precision

    # Add some simulated uncertainty
    theta_estimated = phase_discrete / 2

    # Recover k
    k_estimated = N * np.sin(theta_estimated)**2

    return k_estimated, theta_estimated, theta_true

# Test precision vs accuracy
N = 256
k_true = 10

print("Quantum Counting Precision Analysis:")
print(f"N={N}, k_true={k_true}")
print(f"{'Precision Qubits':>18} {'Estimated k':>12} {'Error':>10}")
print("-" * 45)

for m in range(4, 12):
    k_est, _, _ = phase_estimation_simulation(N, k_true, m)
    error = abs(k_est - k_true)
    print(f"{m:>18} {k_est:>12.2f} {error:>10.2f}")
```

## 7. Strategies for Unknown k

When k is unknown, several strategies exist:

### Strategy 1: Exponential Search

Try increasing iteration counts: 1, 2, 4, 8, ... until success.

```python
import numpy as np

def exponential_search_strategy(N, k_actual, max_attempts=20):
    """
    Exponential search when k is unknown.
    """
    print(f"Exponential search for N={N}, k={k_actual}")

    theta = np.arcsin(np.sqrt(k_actual/N))
    r_optimal = int(round((np.pi/2 - theta) / (2*theta)))

    print(f"True optimal iterations: {r_optimal}")
    print(f"\n{'Attempt':>8} {'Iterations':>12} {'Prob':>10} {'Success?':>10}")
    print("-" * 45)

    for attempt in range(max_attempts):
        r = 2**attempt  # 1, 2, 4, 8, 16, ...
        if r > 4 * r_optimal:
            break

        prob = np.sin((2*r + 1) * theta)**2
        success = prob > 0.5

        print(f"{attempt:>8} {r:>12} {prob:>10.4f} {'YES' if success else 'no':>10}")

        if success:
            print(f"\nFound good iteration count at attempt {attempt}")
            return r

    return None

exponential_search_strategy(256, 4)
```

### Strategy 2: Randomized Approach

Run with random iteration counts, measure, check solution.

```python
import numpy as np

def randomized_strategy(N, k_actual, num_trials=10):
    """
    Random iteration count strategy.
    """
    theta = np.arcsin(np.sqrt(k_actual/N))
    r_max = int(np.pi/2 / (2*theta) * 1.5)  # Upper bound guess

    print(f"Randomized search for N={N}, k={k_actual}")
    print(f"Trying random r in [1, {r_max}]")
    print(f"\n{'Trial':>6} {'r':>6} {'Prob':>10}")
    print("-" * 25)

    successes = 0
    for trial in range(num_trials):
        r = np.random.randint(1, r_max + 1)
        prob = np.sin((2*r + 1) * theta)**2
        print(f"{trial:>6} {r:>6} {prob:>10.4f}")
        if prob > 0.5:
            successes += 1

    print(f"\nSuccess rate: {successes}/{num_trials} = {successes/num_trials:.1%}")

np.random.seed(42)
randomized_strategy(256, 4)
```

### Strategy 3: Quantum Counting First

Use quantum counting to estimate k, then use optimal iterations.

```python
import numpy as np

def count_then_search(N, k_actual):
    """
    Two-phase strategy: count, then search.
    """
    print(f"Count-then-search for N={N}, k={k_actual}")

    # Phase 1: Quantum counting
    print("\nPhase 1: Quantum Counting")
    k_est, _, _ = phase_estimation_simulation(N, k_actual, 8)
    print(f"  Estimated k: {k_est:.2f}")

    # Phase 2: Grover search with estimated k
    print("\nPhase 2: Grover Search")
    theta_est = np.arcsin(np.sqrt(k_est/N))
    r_opt = max(0, int(round((np.pi/2 - theta_est) / (2*theta_est))))
    print(f"  Computed optimal iterations: {r_opt}")

    # Actual success probability with these iterations
    theta_actual = np.arcsin(np.sqrt(k_actual/N))
    prob = np.sin((2*r_opt + 1) * theta_actual)**2
    print(f"  Actual success probability: {prob:.4f}")

    return prob

count_then_search(256, 10)
```

## 8. Special Cases

### Case: Too Many Solutions (k > N/4)

When k > N/4, theta > pi/6, and the algorithm might only need 0 or 1 iterations!

```python
import numpy as np

def many_solutions_case(N):
    """Handle cases with many solutions."""
    print(f"Many solutions analysis for N={N}")
    print(f"{'k':>6} {'k/N':>8} {'r_opt':>8} {'Note':>20}")
    print("-" * 45)

    for k in [1, N//8, N//4, N//2, 3*N//4]:
        theta = np.arcsin(np.sqrt(k/N))
        r_exact = (np.pi/2 - theta) / (2*theta)
        r_opt = max(0, int(round(r_exact)))

        note = ""
        if r_opt == 0:
            note = "Measure immediately!"
        elif r_opt == 1:
            note = "Just one iteration"

        print(f"{k:>6} {k/N:>8.2%} {r_opt:>8} {note:>20}")

many_solutions_case(256)
```

### Case: k = N/4 (Quarter of Solutions)

At k = N/4, theta = pi/6 and optimal iterations = 1.

```python
import numpy as np

def quarter_solutions():
```

## 9. Complete Implementation

Here's a robust implementation handling all cases:

```python
import numpy as np

def robust_grover(n_qubits, marked_items, verbose=True):
    """
    Robust Grover's algorithm for multiple solutions.
    Handles edge cases and provides diagnostics.
    """
    N = 2**n_qubits
    k = len(marked_items)

    if k == 0:
        raise ValueError("No solutions specified")
    if k >= N:
        raise ValueError("Too many solutions (k >= N)")

    # Calculate parameters
    theta = np.arcsin(np.sqrt(k/N))
    r_opt = max(0, int(round((np.pi/2 - theta) / (2*theta))))

    # Special case: k > N/2, just measure
    if k > N/2:
        r_opt = 0

    # Run algorithm
    state = np.ones(N) / np.sqrt(N)

    oracle = np.eye(N)
    for item in marked_items:
        oracle[item, item] = -1

    s = np.ones(N) / np.sqrt(N)
    diffusion = 2 * np.outer(s, s) - np.eye(N)

    for _ in range(r_opt):
        state = diffusion @ (oracle @ state)

    # Results
    probs = np.abs(state)**2
    total_prob = sum(probs[item] for item in marked_items)

    if verbose:
        print(f"Robust Grover's Algorithm")
        print(f"  N = {N}, k = {k}")
        print(f"  theta = {np.degrees(theta):.2f} deg")
        print(f"  Optimal iterations: {r_opt}")
        print(f"  Total success probability: {total_prob:.6f}")
        print(f"  Per-solution probability: {total_prob/k:.6f}")

    return state, probs, total_prob

# Test various scenarios
print("=== Single solution ===")
robust_grover(4, [7])

print("\n=== Multiple solutions ===")
robust_grover(4, [2, 5, 9, 14])

print("\n=== Many solutions ===")
robust_grover(4, [0, 1, 2, 3, 4, 5, 6, 7, 8])
```

## Summary

In this lesson, you learned:

- Multiple solutions make Grover's algorithm faster: r_opt is proportional to sqrt(N/k)
- The formula changes because sin(theta) = sqrt(k/N) with k solutions
- Quantum counting uses phase estimation to find unknown k
- Several strategies exist for unknown k: exponential search, random, count-then-search
- Special cases (k > N/4) may require 0 or 1 iterations

## Key Formulas

| Quantity | Single Solution (k=1) | Multiple Solutions (k) |
|----------|----------------------|------------------------|
| sin(theta) | 1/sqrt(N) | sqrt(k/N) |
| Optimal r | (pi/4)*sqrt(N) | (pi/4)*sqrt(N/k) |
| Success prob | close to 1 | close to 1 (any solution) |
