# Physical Qubit Requirements

## Introduction

Building a quantum computer requires physical systems that can reliably represent and manipulate quantum information. The DiVincenzo criteria provide a checklist for evaluating potential qubit technologies.

## The DiVincenzo Criteria

```
DiVincenzo's Five Criteria (2000)
═════════════════════════════════

FOR QUANTUM COMPUTATION:

1. SCALABLE SYSTEM
   Well-characterized qubits
   Can add more qubits as needed

2. INITIALIZATION
   Ability to prepare a known initial state
   Usually |000...0⟩

3. LONG COHERENCE TIME
   T₂ >> gate time
   Qubits maintain quantum properties

4. UNIVERSAL GATE SET
   Single-qubit rotations
   Two-qubit entangling gate

5. QUBIT MEASUREMENT
   Readout of individual qubits
   High fidelity, fast


ADDITIONAL CRITERIA (for quantum networking):

6. INTERCONVERT STATIONARY/FLYING QUBITS
   Convert between storage and transmission

7. TRANSMIT FLYING QUBITS
   Faithfully transmit between locations
```

## Criterion 2: Initialization

```
State Initialization
════════════════════

REQUIREMENT:
    Prepare qubits in known state with high fidelity
    Typically |0⟩ for each qubit

METHODS:

1. OPTICAL PUMPING (atoms)
   Shine laser → atoms decay to ground state
   Fidelity: >99.9%

2. THERMALIZATION (superconducting)
   Cool to millikelvin temperatures
   Ground state thermally populated
   Fidelity: depends on T and ω

   P(ground) = 1 / (1 + e^(-ℏω/kT))

   For ω = 5 GHz, T = 15 mK:
   P(ground) ≈ 99.5%

3. MEASUREMENT + FEEDBACK
   Measure qubit state
   Apply correction if wrong
   Can achieve >99.9%

4. RESET PROTOCOLS
   Actively pump energy out
   Faster than waiting for decay
```

## Criterion 3: Coherence Times

```
Coherence: The Central Challenge
════════════════════════════════

T₁: ENERGY RELAXATION TIME
    |1⟩ → |0⟩ spontaneous decay
    Loses amplitude information
    "Amplitude damping"

T₂: DEPHASING TIME
    Phase randomization
    |+⟩ → mixed state
    Usually T₂ ≤ 2T₁

T₂*: INHOMOGENEOUS DEPHASING
    Includes slow fluctuations
    T₂* ≤ T₂


FIGURE OF MERIT:

    Quality = T₂ / t_gate

    How many gates before decoherence?

    ┌────────────────────────────────────────────┐
    │  Platform          T₂         t_gate   Q  │
    ├────────────────────────────────────────────┤
    │  Superconducting   ~100 μs    ~20 ns   5k │
    │  Trapped ions      ~1-10 s    ~1-10 μs 1M │
    │  NV centers        ~1 ms      ~10 ns  100k│
    │  Neutral atoms     ~1 s       ~1 μs    1M │
    │  Photonic          N/A        ~1 ns    ∞* │
    └────────────────────────────────────────────┘
    * Photons don't decohere, but are hard to store
```

## Criterion 5: Measurement

```
Qubit Readout
═════════════

REQUIREMENTS:
    - Single-shot measurement
    - High fidelity (distinguish |0⟩ from |1⟩)
    - Non-destructive (ideally)
    - Fast compared to T₁

READOUT METHODS:

1. FLUORESCENCE (atoms)
   Cycling transition: |1⟩ → bright, |0⟩ → dark
   Collect photons, count
   Fidelity: 99.9%+

2. DISPERSIVE READOUT (superconducting)
   Couple qubit to resonator
   State-dependent frequency shift
   Fidelity: 99%+

3. ELECTRON SHELVING (ions)
   Map qubit to bright/dark states
   Scatter many photons
   Very high fidelity


READOUT ERRORS:

    Confusion matrix:
    ┌─────────────────────────────┐
    │  Prepared │ Measured as    │
    │           │   0      1     │
    ├─────────────────────────────┤
    │    |0⟩    │  99.5%   0.5%  │
    │    |1⟩    │  1.0%   99.0%  │
    └─────────────────────────────┘

    Often asymmetric: |1⟩→|0⟩ during readout
```

## Decoherence Mechanisms

```
Sources of Decoherence
══════════════════════

ENERGY RELAXATION (T₁):
    - Spontaneous emission
    - Coupling to cold bath
    - Material losses

PURE DEPHASING (T_φ):
    1/T₂ = 1/(2T₁) + 1/T_φ

    Sources:
    - Fluctuating magnetic fields
    - Charge noise
    - Two-level defects
    - Thermal photons


PLATFORM-SPECIFIC:

Superconducting:
    - TLS defects in materials
    - Quasiparticle poisoning
    - Flux noise
    - Photon shot noise

Trapped ions:
    - Magnetic field fluctuations
    - Motional heating
    - Off-resonant scattering

Neutral atoms:
    - Trap intensity fluctuations
    - Collisions
    - Scattering from traps
```

## Trade-offs in Qubit Design

```
The Qubit Design Triangle
═════════════════════════

                Coherence
                   △
                  ╱ ╲
                 ╱   ╲
                ╱     ╲
               ╱       ╲
              ╱         ╲
         Speed ────────── Connectivity

TRADE-OFFS:

1. Coherence vs Speed:
   Weak coupling → long coherence, slow gates
   Strong coupling → fast gates, more noise

2. Coherence vs Connectivity:
   Isolated qubits → long coherence
   Many connections → more noise paths

3. Speed vs Connectivity:
   Local gates fast, long-range slow
   All-to-all needs more resources


PLATFORM POSITIONS:

    Superconducting: Fast, moderate coherence, limited connectivity
    Trapped ions: Slow, excellent coherence, good connectivity
    Neutral atoms: Medium, good coherence, reconfigurable
    Photonic: Fast, no memory, limited interaction
```

## Summary

Building physical qubits requires satisfying fundamental criteria:

1. **DiVincenzo criteria**: Roadmap for quantum hardware
2. **Coherence**: The central challenge (T₂ >> gate time)
3. **Initialization**: Start in known state (>99%)
4. **Gates**: Universal set with high fidelity
5. **Measurement**: Single-shot, high-fidelity readout

Different platforms make different trade-offs among these requirements.
