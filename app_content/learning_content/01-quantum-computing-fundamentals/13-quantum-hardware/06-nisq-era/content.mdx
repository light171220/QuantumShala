# The NISQ Era

## Introduction

We are currently in the NISQ era - Noisy Intermediate-Scale Quantum computing. Coined by John Preskill in 2018, this term describes today's quantum computers: powerful enough to do interesting things, but too error-prone for fault-tolerant algorithms.

## What Is NISQ?

```
NISQ Characteristics
════════════════════

N - NOISY
    Error rates: 0.1-1% per gate
    No full error correction
    Decoherence limits depth

I - INTERMEDIATE
    50-1000+ qubits
    Too many for classical simulation
    Too few for full fault tolerance

S - SCALE
    Meaningful computation possible
    But not arbitrary algorithms

Q - QUANTUM
    Genuine quantum effects
    Superposition, entanglement utilized


WHERE WE ARE (2024):
    ┌───────────────────────────────────────────┐
    │  Platform          Qubits    2Q Error    │
    ├───────────────────────────────────────────┤
    │  Superconducting   1000+     ~0.5%       │
    │  Trapped ions      30-50     ~0.2%       │
    │  Neutral atoms     1000+     ~0.5%       │
    │  Photonic          100+      ~1%         │
    └───────────────────────────────────────────┘
```

## NISQ Limitations

```python
def nisq_limitations():
    """
    Key limitations of NISQ devices.
    """
    print("NISQ Limitations")
    print("=" * 50)
    print("""
1. LIMITED CIRCUIT DEPTH
   Errors accumulate with depth
   Usable depth: 10-100 layers typically

   Fidelity ≈ F_gate^(N_gates)

   Example: 99% gates, 100 gates → 37% fidelity

2. NOISY MEASUREMENTS
   Readout errors: 1-5%
   Limits observable precision
   Need many shots for statistics

3. LIMITED CONNECTIVITY
   Not all qubits can interact
   SWAP gates add depth/errors
   Compilation overhead

4. COHERENCE CONSTRAINTS
   Must finish before decoherence
   Limits algorithm complexity

5. CALIBRATION DRIFT
   Parameters change over time
   Need frequent recalibration
   Reproducibility challenges


PRACTICAL CONSEQUENCES:
    ✗ Can't run Shor's algorithm (needs FT)
    ✗ Can't do arbitrary-depth circuits
    ✗ Results have significant noise
    ✓ Can do variational algorithms
    ✓ Can show quantum advantage (limited)
""")

nisq_limitations()
```

## Error Mitigation

```
Managing Errors Without Full QEC
════════════════════════════════

ERROR MITIGATION:
    Classical post-processing to reduce errors
    Works for expectation values
    No qubit overhead

TECHNIQUES:

1. ZERO-NOISE EXTRAPOLATION (ZNE):
   Run at multiple noise levels
   Extrapolate to zero noise

   Noise scale    1×    2×    3×
   Result        ───●───●───●───→ Zero noise
                              ↓
                          Estimate

2. PROBABILISTIC ERROR CANCELLATION (PEC):
   Characterize noise model
   Apply inverse operations probabilistically
   Sampling overhead

3. MEASUREMENT ERROR MITIGATION:
   Calibrate readout errors
   Apply inverse matrix to results

4. DYNAMICAL DECOUPLING:
   Insert identity sequences
   Average out slow noise


LIMITATIONS:
   ┌────────────────────────────────────────┐
   │  Mitigation       Error Correction    │
   ├────────────────────────────────────────┤
   │  No overhead      Large overhead      │
   │  Expectation only Full state         │
   │  Limited depth    Unlimited depth     │
   │  Now              Future              │
   └────────────────────────────────────────┘
```

## Summary

The NISQ era defines our current quantum computing landscape:

1. **Definition**: Noisy, 50-1000 qubits, intermediate scale
2. **Limitations**: Error accumulation, limited depth
3. **Algorithms**: Variational (VQE, QAOA), quantum sampling
4. **Mitigation**: ZNE, PEC, measurement error correction
5. **Future**: Path to fault tolerance over 5-15 years

Understanding NISQ constraints is essential for realistic quantum computing expectations.
