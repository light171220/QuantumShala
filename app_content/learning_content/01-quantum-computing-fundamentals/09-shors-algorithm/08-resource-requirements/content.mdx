# Resource Requirements

## Introduction

Factoring cryptographically relevant numbers (2048+ bits) requires substantial quantum resources. Understanding these requirements helps assess when quantum computers might threaten current cryptography and guides hardware development efforts.

## Qubit Requirements

```python
def qubit_requirements(key_bits):
    """
    Calculate qubit requirements for factoring an n-bit number.
    """
    n = key_bits

    # Basic algorithm requirements
    control_qubits = 2 * n
    target_qubits = n
    ancilla_arithmetic = 2 * n  # For modular arithmetic
    total_logical = control_qubits + target_qubits + ancilla_arithmetic

    print(f"""
Qubit Requirements for {n}-bit Factoring
════════════════════════════════════════

LOGICAL QUBITS (bare algorithm):
    Control register:     {control_qubits:,} qubits
    Target register:      {target_qubits:,} qubits
    Arithmetic ancilla:   {ancilla_arithmetic:,} qubits
    ─────────────────────────────────────
    Total logical:        {total_logical:,} qubits

OPTIMIZED IMPLEMENTATIONS:
    With qubit recycling:    ~{2*n:,} qubits
    Minimal (Beauregard):    ~{2*n + 3:,} qubits

PHYSICAL QUBITS (with error correction):
    For surface code with d=27:
    Physical per logical: ~2,000 qubits
    Total physical: ~{total_logical * 2000:,} qubits

    For surface code with d=13:
    Physical per logical: ~500 qubits
    Total physical: ~{total_logical * 500:,} qubits
    """)

    return total_logical

# Common key sizes
for bits in [512, 1024, 2048, 4096]:
    qubit_requirements(bits)
```

## Circuit Depth and Runtime

```python
def runtime_estimate(key_bits, gate_time_ns=100, error_rate=0.001):
    """
    Estimate runtime for factoring.
    """
    n = key_bits

    # Circuit depth (parallelized)
    depth = n ** 2  # Optimistic with maximum parallelization

    # Gate time in fault-tolerant implementation
    # Each logical gate takes multiple physical cycles
    ft_overhead = 1000  # Typical for surface code

    total_cycles = depth * ft_overhead
    time_ns = total_cycles * gate_time_ns
    time_seconds = time_ns / 1e9
    time_hours = time_seconds / 3600

    print(f"""
Runtime Estimate for {n}-bit Factoring
══════════════════════════════════════

CIRCUIT DEPTH:
    Optimized depth:     O(n²) = ~{n**2:,} layers
    With parallelization: May reduce by factor of n

FAULT-TOLERANT OVERHEAD:
    Each logical operation: ~{ft_overhead} physical cycles
    Total cycles: ~{total_cycles:,}

TIME ESTIMATES (at {gate_time_ns} ns/gate):
    Ideal (no overhead):     {depth * gate_time_ns / 1e9:.2f} seconds
    With FT overhead:        {time_seconds:,.0f} seconds
                           = {time_hours:,.1f} hours

PRACTICAL CONSIDERATIONS:
    QEC cycle time:          ~1 μs (superconducting)
    Syndrome decoding:       Real-time required
    Classical post-processing: Minutes
    """)

runtime_estimate(2048)
```

## Error Correction Overhead

```

WHY ERROR CORRECTION IS ESSENTIAL:

    Circuit depth for 2048-bit: ~10^8 gates
    Physical error rate: ~10^-3 per gate

    Without QEC:
    P(no errors) = (1 - 10^-3)^(10^8) ≈ 0

    We NEED fault-tolerant quantum computing!


SURFACE CODE PARAMETERS:

    Target logical error rate: p_L < 10^-15 (for full circuit)
    Physical error rate: p_phys = 10^-3

    Required code distance:
    p_L ≈ (p_phys/p_th)^((d+1)/2)

    For p_th ≈ 1%, d ≈ 27 needed
```

## Recent Optimizations

```python
def optimizations_summary():
    """
    Survey of techniques to reduce resource requirements.
    """
    print("""
Optimization Techniques
═══════════════════════

1. QUBIT-EFFICIENT CIRCUITS (Beauregard, 2003)
    Reduces to 2n+3 logical qubits
    Trades qubits for circuit depth
    Key: Sequential modular exponentiation

2. WINDOWED ARITHMETIC
    Precompute small lookup tables
    Reduce multiplications by factor of w
    Cost: Additional classical computation

3. APPROXIMATE QFT
    Truncate small-angle rotations
    Error < ε with O(n log(n/ε)) gates
    Often 10-100× fewer rotation gates

4. BETTER ERROR CORRECTION CODES
    LDPC codes may reduce overhead
    Fewer physical qubits per logical
    Active research area

5. ALGORITHMIC IMPROVEMENTS
    Regev's algorithm (2023): O(n^1.5) gates
    But requires more qubits

6. COMPILATION OPTIMIZATIONS
    Gate synthesis improvements
    Reduced T-count
    Better circuit layouts


IMPACT OF OPTIMIZATIONS:

    Original estimates (2000s): ~10^9 physical qubits
    Current estimates (2020s): ~10^6-10^7 physical qubits
    Future (with advances): ~10^5-10^6 physical qubits?

    Still a major engineering challenge!
    """)

optimizations_summary()
```

## Summary

Resource requirements for cryptographic Shor's algorithm:

1. **Qubits**: ~6,000 logical / ~10 million physical for RSA-2048
2. **Gates**: ~10^12 T gates with fault-tolerant implementation
3. **Runtime**: Hours to days with current projections
4. **Error correction**: Dominates the resource count
5. **Current gap**: ~10,000× in qubit count from today's machines

Shor's algorithm remains a long-term threat to RSA, motivating post-quantum cryptography development.
