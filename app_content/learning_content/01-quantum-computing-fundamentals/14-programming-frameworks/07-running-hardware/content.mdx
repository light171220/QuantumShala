## Transpilation for Hardware

```python
from qiskit import QuantumCircuit, transpile

# Ideal circuit
qc = QuantumCircuit(3)
qc.h(0)
qc.cx(0, 2)  # May not be directly connected!
qc.ccx(0, 1, 2)  # Toffoli needs decomposition

print("Original circuit:")
print(qc.draw())

# Transpile for specific hardware (conceptual)
"""
from qiskit_ibm_runtime import QiskitRuntimeService

service = QiskitRuntimeService()

# Transpile for hardware constraints
transpiled = transpile(qc, backend, optimization_level=3)

print(f"Original depth: {qc.depth()}")
print(f"Transpiled depth: {transpiled.depth()}")
print(f"Original gates: {qc.count_ops()}")
print(f"Transpiled gates: {transpiled.count_ops()}")
"""

print("""
Transpilation handles:
- Gate decomposition (Toffoli → 1q + 2q gates)
- Routing (SWAP for non-adjacent qubits)
- Optimization (reduce gate count)
- Scheduling (timing constraints)
""")
```

## Managing Jobs

# Job management patterns

"""

# Cancel job
job.cancel()

# Get result (blocks until done)
result = job.result()

## Interpreting Hardware Results

```
Hardware vs Simulator Results
═════════════════════════════

SIMULATOR (ideal):
    |00⟩: 50.0%
    |11⟩: 50.0%

HARDWARE (real):
    |00⟩: 47.3%
    |01⟩: 2.1%   ← Errors!
    |10⟩: 1.8%   ← Errors!
    |11⟩: 48.8%


SOURCES OF DISCREPANCY:
    1. Gate errors (~0.1-1% per gate)
    2. Measurement errors (~1-5%)
    3. Decoherence during execution
    4. Crosstalk between qubits
    5. State preparation errors


ERROR MITIGATION:
    - Measurement error mitigation
    - Zero-noise extrapolation
    - Probabilistic error cancellation
    - Dynamical decoupling


BEST PRACTICES:
    - Use error mitigation when available
    - Run multiple times for statistics
    - Compare to noisy simulation
    - Use highest-quality backends
```
