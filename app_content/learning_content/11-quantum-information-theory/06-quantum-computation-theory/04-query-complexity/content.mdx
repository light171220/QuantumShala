## 1. Introduction to Query Complexity

Query complexity measures computational cost by counting queries to an oracle, abstracting away other computational resources.

<Definition term="Query Complexity">
The query complexity Q(f) of computing function f is the minimum number of queries to an input oracle needed to determine f(x) for worst-case input x.
</Definition>

This framework is powerful because:
- It isolates the "information-theoretic" aspect of computation
- Lower bounds are often provable (unlike circuit complexity)
- Quantum speedups are most clearly demonstrated here

```python
import numpy as np

def oracle_query_model():
    """
    Explain the oracle query model.
    """
    print("Oracle Query Model")
    print("=" * 50)

    print("""
    Classical Oracle:
    - Input: query index i
    - Output: x_i (the i-th bit of input x)
    - Cost: 1 query per bit accessed

    Quantum Oracle (standard form):
    |i>|b> -> |i>|b XOR x_i>

    - Input: superposition of indices
    - Applies f to all indices simultaneously
    - Cost: 1 query per oracle call

    Key difference:
    - Classical: one answer per query
    - Quantum: coherent superposition of all answers
    """)

oracle_query_model()
```

## 2. Classical Query Complexity

### Deterministic and Randomized

<Definition term="Deterministic Query Complexity">
D(f) is the minimum queries needed by a deterministic algorithm to compute f correctly on all inputs.
</Definition>

<Definition term="Randomized Query Complexity">
R(f) is the minimum queries needed by a randomized algorithm to compute f correctly with probability at least 2/3 on all inputs.
</Definition>

```python
def classical_query_examples():

    """
    examples = {

            'D': 'n (must check all to confirm)',

            'D': 'n (every bit matters)',
            'R': 'n (randomization doesnt help)',
            'reason': 'Each bit independently affects output'
        },
        'MAJORITY (more 1s than 0s?)': {
            'D': 'n (must know exact count)',
            'R': 'n (same for worst case)',
            'reason': 'Close to threshold requires counting'
        },
        'Element distinctness': {
```

## 3. Quantum Query Complexity

### Definition

<Definition term="Quantum Query Complexity">
Q(f) is the minimum queries needed by a quantum algorithm to compute f correctly with probability at least 2/3 on all inputs.
</Definition>

The quantum oracle acts as:
$$O_x |i\rangle|b\rangle = |i\rangle|b \oplus x_i\rangle$$

Or in phase form:
$$O_x^{\pm} |i\rangle = (-1)^{x_i} |i\rangle$$

```python
import numpy as np

def quantum_oracle_phase(x, state):
    """
    Apply phase oracle: |i> -> (-1)^{x_i} |i>

    Args:
        x: Binary string (input to oracle)
        state: Quantum state vector

    Returns:
        State after oracle application
    """
    n = len(x)
    new_state = state.copy()

    for i in range(len(state)):
        if x[i % n] == '1':
            new_state[i] *= -1

    return new_state

def demonstrate_quantum_query():
    """
    Demonstrate quantum oracle query on superposition.
    """
    # 2-qubit example
    n = 2
    N = 2**n

    # Start with uniform superposition
    state = np.ones(N) / np.sqrt(N)
    print("Initial state (uniform superposition):")
    print(f"  |psi> = (1/2)(|00> + |01> + |10> + |11>)")

    # Apply oracle for x = "10"
    x = "1010"  # Repeat for 4-dim state
    state_after = quantum_oracle_phase(x[:N], state)

    print(f"\nAfter oracle query (x = '10'):")
    print(f"  |psi'> = (1/2)(|00> - |01> + |10> - |11>)")
    print(f"\nNotice: one query reveals info about ALL bits simultaneously!")

demonstrate_quantum_query()
```

## 4. Quantum Speedups in Query Complexity

### Grover's Search: Quadratic Speedup

<Definition term="Grover's Speedup">
For unstructured search in N items:
- Classical: Theta(N) queries
- Quantum: Theta(sqrt(N)) queries

This is a quadratic speedup, provably optimal.
</Definition>

```python
import numpy as np

def grover_analysis(n_bits):
    """
    Analyze Grover's algorithm query complexity.
    """
    N = 2**n_bits

    # Grover's iterations
    optimal_iterations = int(np.pi/4 * np.sqrt(N))

    # Success probability
    theta = np.arcsin(1/np.sqrt(N))
    prob_after_k = lambda k: np.sin((2*k + 1) * theta)**2

    results = {
        'n_bits': n_bits,
        'search_space': N,
        'classical_queries': N,  # Expected for random search
        'quantum_queries': optimal_iterations,
        'speedup': N / optimal_iterations,
        'success_probability': prob_after_k(optimal_iterations)
    }

    return results

print("Grover's Algorithm Query Analysis:")
print("=" * 60)
print(f"{'Bits':<6}{'Space':<12}{'Classical':<12}{'Quantum':<10}{'Speedup'}")
print("-" * 60)

for n in [4, 8, 12, 16, 20]:
    r = grover_analysis(n)
    print(f"{n:<6}{r['search_space']:<12}{r['classical_queries']:<12}"
          f"{r['quantum_queries']:<10}{r['speedup']:.1f}x")
```

### Simon's Problem: Exponential Speedup

<Definition term="Simon's Problem">
Given oracle access to f: {0,1}^n -> {0,1}^n with the promise that:
- f(x) = f(y) iff x XOR y in {0, s} for unknown s

Find s.

- Classical: Omega(2^{n/2}) queries (birthday bound)
- Quantum: O(n) queries
</Definition>

```python
def simon_query_comparison(n):
    """
    Compare classical vs quantum queries for Simon's problem.
    """
    # Classical: birthday paradox bound
    classical_queries = 2**(n/2) * np.sqrt(np.pi / 2)

    # Quantum: need n-1 linearly independent equations
    quantum_queries = 2 * n  # Conservative estimate

    return {
        'n': n,
        'classical': classical_queries,
        'quantum': quantum_queries,
        'ratio': classical_queries / quantum_queries
    }

print("\nSimon's Problem Query Complexity:")
print("=" * 60)
print(f"{'n':<6}{'Classical':<20}{'Quantum':<12}{'Speedup'}")
print("-" * 60)

for n in [10, 20, 30, 40, 50]:
    r = simon_query_comparison(n)
    print(f"{n:<6}{r['classical']:<20.0f}{r['quantum']:<12}{r['ratio']:.0f}x")

print("\nNote: Simon's problem shows EXPONENTIAL quantum speedup!")
```

## 5. Query Lower Bound Techniques

### The Polynomial Method

<Definition term="Polynomial Method">
For any quantum algorithm making T queries, the acceptance probability p(x) is a polynomial of degree at most 2T in the input bits. Therefore:

Q(f) >= deg(f) / 2

where deg(f) is the minimum degree of a polynomial representing f.
</Definition>

```python
def polynomial_method_examples():
    """
    Examples of polynomial method lower bounds.
    """
    examples = {
        'OR function': {
            'polynomial_degree': 'n',
            'lower_bound': 'n/2',
            'actual_quantum': 'O(sqrt(n))',
            'note': 'Polynomial method is loose here'
        },
        'PARITY function': {
            'polynomial_degree': 'n',
            'lower_bound': 'n/2',
            'actual_quantum': 'n/2 (tight!)',
            'note': 'Polynomial method is tight'
        },
        'MAJORITY function': {
            'polynomial_degree': 'n',
            'lower_bound': 'n/2',
            'actual_quantum': 'O(n)',
            'note': 'No quantum speedup'
        }
    }

    print("Polynomial Method Lower Bounds:")
    print("=" * 60)
    for func, details in examples.items():
        print(f"\n{func}:")
        for key, value in details.items():

polynomial_method_examples()
```

### The Adversary Method

<Definition term="Adversary Method">
The adversary bound uses the structure of "hard" input pairs to prove lower bounds:

Q(f) >= Gamma(f) / 2

where Gamma(f) is the adversary bound based on how well the algorithm must distinguish hard input pairs.
</Definition>

```python
def adversary_bound_or():
    """
    Calculate adversary lower bound for OR function.
    """
    print("Adversary Method for OR Function:")
    print("=" * 50)

    print("""
    Hard input pairs for OR:
    - 0^n (all zeros) must output 0
    - e_i (single 1 at position i) must output 1

    For each pair (0^n, e_i):
    - They differ in only 1 bit (position i)
    - Algorithm must query position i to distinguish

    Adversary argument:
    - There are n hard pairs
    - Each query can eliminate at most O(sqrt(n)) pairs
    - Therefore need Omega(sqrt(n)) queries

    This matches Grover's upper bound!
    """)

adversary_bound_or()
```
