## 1. Introduction to BQP

BQP (Bounded-error Quantum Polynomial time) is the fundamental complexity class capturing efficient quantum computation. It represents what quantum computers can solve efficiently with high probability.

<Definition term="BQP">
A decision problem L is in BQP if there exists a polynomial-time uniform family of quantum circuits {C_n} such that for all inputs x of length n:
- If x is in L: Pr[C_n accepts x] >= 2/3
- If x is not in L: Pr[C_n accepts x] <= 1/3
</Definition>

The key elements are:
1. **Polynomial-time**: Circuit size is polynomial in input length
2. **Uniform**: Circuits can be efficiently generated
3. **Bounded error**: Success probability bounded away from 1/2

```python
import numpy as np
from typing import Callable, Tuple

def bqp_machine(circuit_family: Callable[[int], 'QuantumCircuit'],
                input_x: str) -> bool:
    """
    Abstract BQP machine execution.

    Args:
        circuit_family: Function mapping n to circuit C_n
        input_x: Input string

    Returns:
        True if circuit accepts, False otherwise
    """
    n = len(input_x)

    # Get circuit for input length n
    circuit = circuit_family(n)

    # Encode input in quantum state
    # |x> = |x_1, x_2, ..., x_n, 0, 0, ..., 0> (ancillas)
    initial_state = encode_input(input_x)

    # Apply circuit
    final_state = circuit.apply(initial_state)

    # Measure designated output qubit
    prob_accept = measure_output_qubit(final_state)

    # Return True with probability prob_accept
    return np.random.random() < prob_accept

def verify_bqp_membership(algorithm, test_cases, num_trials=1000):
    """
    Empirically verify BQP acceptance criteria.

    BQP requires:
    - Yes instances: accept with prob >= 2/3
    - No instances: accept with prob <= 1/3
    """
    results = {}

    for instance, is_yes in test_cases:
        accepts = sum(algorithm(instance) for _ in range(num_trials))
        prob = accepts / num_trials

        if is_yes:
            valid = prob >= 0.6  # Allow some statistical error
            expected = ">= 2/3"
        else:
            valid = prob <= 0.4
            expected = "<= 1/3"

        results[instance] = {
            'prob': prob,
            'expected': expected,
            'valid': valid
        }

    return results
```

### The 2/3 Threshold

The choice of 2/3 (or 1/3) is somewhat arbitrary. Any constant bounded away from 1/2 gives the same class:

<Formula>
For any constants 1/2 < a < b <= 1, the class BQP_[a,b] equals BQP.
</Formula>

This is because of **probability amplification**:

```python
import numpy as np

def amplify_success_probability(base_prob: float,
                                target_prob: float,
                                n_trials: int = None) -> dict:
    """
    Calculate trials needed to amplify success probability.

    By running the algorithm k times and taking majority vote,
    we can boost probability exponentially.

    Args:
        base_prob: Original success probability (> 0.5)
        target_prob: Target success probability
        n_trials: Override automatic calculation

    Returns:
        Analysis of amplification
    """
    if base_prob <= 0.5:
        raise ValueError("Base probability must exceed 0.5")

    gap = base_prob - 0.5

    if n_trials is None:
        # Chernoff bound: k trials gives error exp(-2 * gap^2 * k)
        # For target_prob, need: 1 - exp(-2 * gap^2 * k) >= target_prob
        # k >= -log(1 - target_prob) / (2 * gap^2)
        n_trials = int(np.ceil(-np.log(1 - target_prob) / (2 * gap**2)))

    # Actual success probability with n_trials and majority vote
    from scipy.stats import binom
    majority = n_trials // 2 + 1
    actual_prob = 1 - binom.cdf(majority - 1, n_trials, base_prob)

    return {
        'base_probability': base_prob,
        'target_probability': target_prob,
        'trials_needed': n_trials,
        'actual_probability': actual_prob,
        'overhead': n_trials
    }

# Example: Amplifying 2/3 to various targets
print("Probability Amplification (base = 2/3):")
print("-" * 50)
for target in [0.9, 0.99, 0.999, 0.9999]:
    result = amplify_success_probability(2/3, target)
    print(f"Target {target}: {result['trials_needed']} trials "
          f"-> actual {result['actual_probability']:.6f}")
```

### Proving BQP ⊆ PSPACE

<Definition term="Theorem: BQP ⊆ PSPACE">
Every problem in BQP can be solved in polynomial space.
</Definition>

**Proof Sketch**:

```python
def simulate_quantum_circuit_in_pspace(circuit, input_x):
    """
    Simulate quantum circuit using polynomial space.

    Key insight: We don't need to store the full 2^n amplitude vector.
    Instead, we can compute individual amplitudes on demand.

    The amplitude <y|C|x> can be computed by:
    1. Summing over all computational paths from x to y
    2. Each path contributes a product of gate matrix elements
    3. The sum has exponentially many terms, but each term uses poly space
    """

    n = circuit.num_qubits
    m = circuit.num_gates

    def compute_amplitude(y: str, x: str) -> complex:
        """
        Compute <y|C|x> using Feynman path integral approach.

        Sum over all 2^m intermediate states (for m-gate circuit).
        Each term is a product of at most m matrix elements.
        """
        amplitude = 0j

        # Iterate over all intermediate basis states
        # This takes exponential TIME but only polynomial SPACE
        for path in iterate_paths(x, y, circuit):
            contribution = compute_path_amplitude(path, circuit)
            amplitude += contribution

        return amplitude

    # Compute acceptance probability
    prob_accept = 0.0
    for y in accepting_outputs(n):
        amp = compute_amplitude(y, input_x)
        prob_accept += abs(amp)**2

    return prob_accept

# The key insight:
print("""
Why BQP ⊆ PSPACE:

1. Acceptance probability = sum over accepting outputs y of |<y|C|x>|^2

2. Each amplitude <y|C|x> = sum over paths of product of matrix elements

3. Computing one amplitude:
   - Enumerate paths: exponential time
   - Space per path: O(n) for current state + O(m) for gate positions
   - Total space: polynomial

4. Can enumerate paths using only polynomial space (like counting)

5. Therefore: polynomial space suffices to compute acceptance probability
   to sufficient precision to distinguish 2/3 from 1/3.
""")
```

## 4. Error Reduction in BQP

### Standard Amplification

<Formula>
If a BQP algorithm has success probability p > 1/2, running it k times and taking the majority vote gives success probability:

$$1 - \exp(-2k(p - 1/2)^2)$$
</Formula>

```python
import numpy as np
from scipy.stats import binom

def majority_vote_analysis(p: float, k: int) -> dict:
    """
    Analyze majority voting for error reduction.

    Args:
        p: Success probability of single run
        k: Number of runs (should be odd)

    Returns:
        Analysis of amplified probability
    """
    if k % 2 == 0:
        k += 1  # Make odd for clean majority

    majority_threshold = k // 2 + 1

    # Probability of getting majority correct
    # P(at least majority_threshold successes out of k)
    prob_success = 1 - binom.cdf(majority_threshold - 1, k, p)

    # Error probability
    error_prob = 1 - prob_success

    # Chernoff bound
    gap = p - 0.5
    chernoff_bound = np.exp(-2 * k * gap**2)

    return {
        'single_run_prob': p,
        'num_runs': k,
        'amplified_prob': prob_success,
        'error_prob': error_prob,
        'chernoff_bound': chernoff_bound,
        'bits_of_confidence': -np.log2(error_prob) if error_prob > 0 else float('inf')
    }

# Demonstrate amplification
print("Error Reduction via Majority Voting:")
print("=" * 60)
print(f"{'Runs':<8}{'Success Prob':<15}{'Error Prob':<15}{'Confidence'}")
print("-" * 60)

for k in [1, 3, 7, 15, 31, 63, 127]:
    result = majority_vote_analysis(2/3, k)
    print(f"{k:<8}{result['amplified_prob']:<15.10f}"
          f"{result['error_prob']:<15.2e}"
          f"{result['bits_of_confidence']:.1f} bits")
```

### Achieving Negligible Error

<Definition term="BQP with Negligible Error">
For any polynomial p(n), we can reduce error probability to 1/2^{p(n)} using only O(p(n)) repetitions.
</Definition>

This is crucial for:
- Composition of quantum algorithms
- Cryptographic applications
- Fault-tolerant computation

### The Forrelation Problem

<Definition term="Forrelation">
Given oracle access to functions f, g: {0,1}^n -> {-1,+1}, distinguish:
- f and g are random
- f and g satisfy the Forrelation relation: sum_x f(x)H_ng(x) is large
</Definition>

This problem separates BQP from the polynomial hierarchy:

```python
import numpy as np

def forrelation_demonstration(n: int):
    """
    Demonstrate the Forrelation problem structure.

    Quantum: O(1) queries suffice
    Classical: Requires 2^{n/2} queries
    """
    N = 2**n

    # Hadamard matrix (normalized)
    def hadamard_transform(f):
        """Apply Hadamard transform to function values."""
        result = np.zeros(N)
        for x in range(N):
            for y in range(N):
                # H[x,y] = (-1)^{x.y} / sqrt(N)
                dot = bin(x & y).count('1') % 2
                result[x] += ((-1)**dot) * f[y]
        return result / np.sqrt(N)

    # Case 1: Random f and g
    f_random = np.random.choice([-1, 1], N)
    g_random = np.random.choice([-1, 1], N)

    Hg_random = hadamard_transform(g_random)
    correlation_random = np.abs(np.sum(f_random * Hg_random) / N)

    # Case 2: Forrelated f and g
    # g = H f (in the Boolean sense)
    f_forrelated = np.random.choice([-1, 1], N)
    g_forrelated = np.sign(hadamard_transform(f_forrelated))
    g_forrelated[g_forrelated == 0] = 1

    Hg_forrelated = hadamard_transform(g_forrelated)
    correlation_forrelated = np.abs(np.sum(f_forrelated * Hg_forrelated) / N)

    print(f"Forrelation Demo (n={n}, N={N}):")
    print(f"  Random case correlation: {correlation_random:.4f}")
    print(f"  Forrelated case correlation: {correlation_forrelated:.4f}")
    print(f"  Quantum queries to distinguish: O(1)")
    print(f"  Classical queries needed: Omega(2^{n//2}) = Omega({2**(n//2)})")

forrelation_demonstration(6)
```
