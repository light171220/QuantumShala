## 1. What is Quantum Supremacy?

<Definition term="Quantum Supremacy">
Quantum supremacy (or quantum computational advantage) is the demonstration that a quantum computer can perform a computational task that is practically impossible for any classical computer, regardless of whether the task has practical applications.
</Definition>

Key distinctions:


## 2. Sampling Problems

### The Sampling Framework

Unlike decision problems (yes/no), sampling problems require producing outputs from a specific probability distribution.

<Definition term="Sampling Problem">
A sampling problem is specified by a family of probability distributions {D_n} over {0,1}^m. The task is to produce samples from D_n given input of size n.
</Definition>

```python
import numpy as np
from typing import List, Callable

def random_circuit_distribution(n_qubits: int, depth: int,
                                 circuit_seed: int) -> np.ndarray:
    """
    Compute the output distribution of a random quantum circuit.

    For small sizes, we can compute this exactly.
    For large sizes (n > 50), this becomes intractable.

    Args:
        n_qubits: Number of qubits
        depth: Circuit depth
        circuit_seed: Seed for random circuit generation

    Returns:
        Probability distribution over 2^n outcomes
    """
    np.random.seed(circuit_seed)

    # Initialize state |0...0>
    dim = 2**n_qubits
    state = np.zeros(dim, dtype=complex)
    state[0] = 1.0

    # Apply random gates
    for layer in range(depth):
        # Single-qubit random gates on all qubits
        for qubit in range(n_qubits):
            # Random SU(2) gate
            theta, phi, lam = np.random.uniform(0, 2*np.pi, 3)
            gate = random_su2(theta, phi, lam)
            state = apply_single_qubit_gate(state, gate, qubit, n_qubits)

        # Two-qubit gates on alternating pairs
        for q in range(0, n_qubits - 1, 2):
            if (layer + q//2) % 2 == 0:
                state = apply_cz_gate(state, q, q+1, n_qubits)

    # Return probability distribution
    return np.abs(state)**2

def random_su2(theta, phi, lam):
    """Generate random SU(2) matrix."""
    return np.array([
        [np.cos(theta/2), -np.exp(1j*lam)*np.sin(theta/2)],
        [np.exp(1j*phi)*np.sin(theta/2), np.exp(1j*(phi+lam))*np.cos(theta/2)]
    ])

def apply_single_qubit_gate(state, gate, qubit, n_qubits):
    """Apply single-qubit gate to state vector."""
    dim = 2**n_qubits
    new_state = np.zeros(dim, dtype=complex)

    for i in range(dim):
        bit = (i >> (n_qubits - 1 - qubit)) & 1
        partner = i ^ (1 << (n_qubits - 1 - qubit))

        if bit == 0:
            new_state[i] += gate[0, 0] * state[i] + gate[0, 1] * state[partner]
        else:
            new_state[i] += gate[1, 0] * state[partner] + gate[1, 1] * state[i]

    return new_state

def apply_cz_gate(state, q1, q2, n_qubits):
    """Apply CZ gate to state vector."""
    new_state = state.copy()
    for i in range(2**n_qubits):
        b1 = (i >> (n_qubits - 1 - q1)) & 1
        b2 = (i >> (n_qubits - 1 - q2)) & 1
        if b1 == 1 and b2 == 1:
            new_state[i] *= -1
    return new_state

# Demo: Small circuit distribution
n, d = 4, 5
probs = random_circuit_distribution(n, d, seed=42)
print(f"Random circuit output distribution ({n} qubits, depth {d}):")
print(f"Number of outcomes: {len(probs)}")
print(f"Sum of probabilities: {probs.sum():.6f}")
print(f"Max probability: {probs.max():.6f}")
print(f"Min probability: {probs.min():.6f}")
```

### Random Circuit Sampling

<Definition term="Random Circuit Sampling">
The random circuit sampling problem asks to sample from the output distribution of a randomly chosen quantum circuit. The complexity assumption is that this distribution cannot be efficiently sampled classically.
</Definition>

The key insight:


## 3. Cross-Entropy Benchmarking

### Measuring Sampling Quality

How do we verify that a quantum computer is sampling from the correct distribution?

<Definition term="Cross-Entropy Benchmark (XEB)">
The linear cross-entropy benchmark measures the quality of samples by comparing them to the ideal output distribution:

$$\text{XEB} = 2^n \langle P(x) \rangle_{\text{samples}} - 1$$

where P(x) is the ideal probability of outcome x.
</Definition>

```python
import numpy as np
from scipy.stats import spearmanr

def cross_entropy_benchmark(samples: np.ndarray,
                            ideal_probs: np.ndarray) -> dict:
    """
    Compute cross-entropy benchmark score.

    XEB = 2^n * mean(P(x) for x in samples) - 1

    - XEB = 0 for uniform random samples
    - XEB = 1 for perfect sampling
    - XEB = fidelity for depolarizing noise

    Args:
        samples: Array of sample bitstrings (as integers)
        ideal_probs: Ideal probability distribution

    Returns:
        Dict with XEB score and related statistics
    """
    n_qubits = int(np.log2(len(ideal_probs)))
    n_samples = len(samples)

    # Probabilities of observed samples
    observed_probs = ideal_probs[samples]

    # XEB formula
    xeb = (2**n_qubits) * np.mean(observed_probs) - 1

    # Statistical analysis
    xeb_std = (2**n_qubits) * np.std(observed_probs) / np.sqrt(n_samples)

    # Expected values
    # For uniform: E[P(x)] = 1/2^n, so XEB = 0
    # For perfect: E[P(x)] = sum P(x)^2 = (1 + 1)/2^n for Porter-Thomas

    uniform_expectation = 0
    perfect_expectation = (2**n_qubits) * np.sum(ideal_probs**2) - 1

    return {
        'xeb_score': xeb,
        'xeb_std': xeb_std,
        'n_samples': n_samples,
        'n_qubits': n_qubits,
        'uniform_xeb': uniform_expectation,
        'perfect_xeb': perfect_expectation,
        'estimated_fidelity': xeb  # For depolarizing noise
    }

def generate_samples(probs: np.ndarray, n_samples: int,
                     noise_level: float = 0) -> np.ndarray:
    """
    Generate samples from distribution with optional noise.

    Noise model: depolarizing - mix with uniform distribution.
    """
    n_outcomes = len(probs)

    if noise_level == 0:
        # Perfect sampling
        return np.random.choice(n_outcomes, size=n_samples, p=probs)
    else:
        # Mix with uniform
        noisy_probs = (1 - noise_level) * probs + noise_level / n_outcomes
        return np.random.choice(n_outcomes, size=n_samples, p=noisy_probs)

# Demonstration
np.random.seed(42)
n_qubits = 6
n_samples = 10000

# Generate random circuit distribution
probs = random_circuit_distribution(n_qubits, depth=10, circuit_seed=123)

print("Cross-Entropy Benchmark Demo")
print("=" * 50)

for noise in [0, 0.1, 0.3, 0.5, 1.0]:
    samples = generate_samples(probs, n_samples, noise_level=noise)
    result = cross_entropy_benchmark(samples, probs)
    fidelity = 1 - noise
    print(f"\nNoise level: {noise:.1%} (fidelity: {fidelity:.1%})")
    print(f"  XEB score: {result['xeb_score']:.4f} +/- {result['xeb_std']:.4f}")
    print(f"  Expected (theory): {fidelity:.4f}")
```

### Porter-Thomas Distribution

<Definition term="Porter-Thomas Distribution">
For random quantum circuits, the output probabilities follow a Porter-Thomas distribution:

$$P(p) = N e^{-Np}$$

where N = 2^n is the Hilbert space dimension. This is an exponential distribution with mean 1/N.
</Definition>

```python
import numpy as np
import matplotlib.pyplot as plt

def verify_porter_thomas(probs: np.ndarray):
    """
    Verify that circuit output follows Porter-Thomas distribution.
    """
    N = len(probs)

    # Rescale probabilities
    scaled = N * probs

    # Theoretical Porter-Thomas: P(Np) = exp(-Np)
    x_theory = np.linspace(0, 6, 100)
    y_theory = np.exp(-x_theory)

    # Empirical histogram
    hist, bins = np.histogram(scaled, bins=50, density=True)
    bin_centers = (bins[:-1] + bins[1:]) / 2

    print("Porter-Thomas Distribution Verification:")
    print(f"  Number of outcomes: {N}")
    print(f"  Mean of N*p: {np.mean(scaled):.4f} (theory: 1.0)")
    print(f"  Std of N*p: {np.std(scaled):.4f} (theory: 1.0)")
    print(f"  Max of N*p: {np.max(scaled):.4f}")

    return {'scaled': scaled, 'x_theory': x_theory, 'y_theory': y_theory}

# Verify for random circuit
probs = random_circuit_distribution(8, depth=20, circuit_seed=42)
result = verify_porter_thomas(probs)
```

## 4. Complexity-Theoretic Foundations

### Hardness of Sampling

<Definition term="Sampling Hardness">
Under plausible complexity-theoretic assumptions, approximate random circuit sampling is classically intractable:

If there exists an efficient classical algorithm that samples from a distribution close to random circuit output, then the polynomial hierarchy collapses.
</Definition>

