## 1. The Source Coding Problem

Given a source that produces symbols from alphabet $\mathcal{X}$ with probability distribution $p(x)$, how efficiently can we represent these symbols?

<Definition term="Source Coding">
Source coding (data compression) is the process of representing source symbols using fewer bits on average, while allowing perfect reconstruction of the original data.
</Definition>

The fundamental question: **What is the minimum average number of bits per symbol needed?**

Shannon's answer: The entropy $H(X)$.

## 2. Shannon's Source Coding Theorem

<Definition term="Source Coding Theorem">
For a discrete memoryless source with entropy $H(X)$:

1. **Achievability**: There exists a coding scheme achieving average rate arbitrarily close to $H(X)$ bits per symbol.

2. **Converse**: No coding scheme can achieve average rate below $H(X)$ bits per symbol with vanishing error probability.
</Definition>

<Formula>
$$H(X) \leq \bar{L} < H(X) + 1$$
</Formula>

where $\bar{L}$ is the average codeword length for any uniquely decodable code.

```python
import numpy as np
from collections import Counter
import heapq

def theoretical_bounds(probabilities):
    """
    Calculate theoretical compression bounds.

    Returns entropy (lower bound) and entropy + 1 (upper bound for symbol-by-symbol coding).
    """
    probs = np.array(probabilities)
    probs = probs[probs > 0]
    entropy = -np.sum(probs * np.log2(probs))

    return {
        'entropy': entropy,
        'lower_bound': entropy,
        'upper_bound_symbol': entropy + 1,
        'max_compression_ratio': entropy / np.log2(len(probabilities))
    }

# Example: English letter frequencies (simplified)
probs = [0.12, 0.09, 0.08, 0.08, 0.07, 0.07, 0.06, 0.06,
         0.05, 0.05, 0.04, 0.04, 0.03, 0.03, 0.03, 0.02,
         0.02, 0.02, 0.01, 0.01, 0.01, 0.01, 0.005, 0.005, 0.003, 0.002]

bounds = theoretical_bounds(probs)
print(f"Entropy: {bounds['entropy']:.4f} bits/symbol")
print(f"vs. fixed-length: {np.log2(26):.4f} bits/symbol")
print(f"Compression ratio: {bounds['max_compression_ratio']:.2%}")
```

## 3. Prefix-Free Codes

<Definition term="Prefix-Free Code">
A code is prefix-free (or instantaneous) if no codeword is a prefix of another codeword. Such codes can be decoded symbol-by-symbol without lookahead.
</Definition>

### Kraft Inequality

For a prefix-free code with codeword lengths $l_1, l_2, \ldots, l_n$:

<Formula>
$$\sum_{i=1}^{n} 2^{-l_i} \leq 1$$
</Formula>

Conversely, for any lengths satisfying this inequality, a prefix-free code exists.

<Example>
Valid prefix-free code: {0, 10, 110, 111}
- "0" is not a prefix of "10", "110", or "111"
- "10" is not a prefix of "110" or "111"
- Check: $2^{-1} + 2^{-2} + 2^{-3} + 2^{-3} = 0.5 + 0.25 + 0.125 + 0.125 = 1$ ✓
</Example>

## 4. Huffman Coding

Huffman coding constructs an optimal prefix-free code for a given probability distribution.

<Definition term="Huffman Code">
A Huffman code is a prefix-free code that minimizes the average codeword length among all prefix-free codes for a given source distribution.
</Definition>

### Algorithm

1. Create a leaf node for each symbol with its probability
2. Repeat until one node remains:
   - Remove two nodes with lowest probabilities
   - Create new parent node with combined probability
   - Add parent back to the queue
3. Assign 0/1 to each edge, read codewords from root to leaves

```python
import heapq
from collections import defaultdict

class HuffmanNode:
    def __init__(self, symbol=None, prob=0):
        self.symbol = symbol
        self.prob = prob
        self.left = None
        self.right = None

    def __lt__(self, other):
        return self.prob < other.prob

def build_huffman_tree(symbols, probabilities):
    """Build Huffman tree from symbols and probabilities."""
    # Create leaf nodes
    heap = [HuffmanNode(s, p) for s, p in zip(symbols, probabilities)]
    heapq.heapify(heap)

    # Build tree
    while len(heap) > 1:
        left = heapq.heappop(heap)
        right = heapq.heappop(heap)

        parent = HuffmanNode(prob=left.prob + right.prob)
        parent.left = left
        parent.right = right

        heapq.heappush(heap, parent)

    return heap[0]

def get_huffman_codes(root, prefix="", codes=None):
    """Extract codes from Huffman tree."""
    if codes is None:
        codes = {}

    if root.symbol is not None:
        codes[root.symbol] = prefix if prefix else "0"
    else:
        if root.left:
            get_huffman_codes(root.left, prefix + "0", codes)
        if root.right:
            get_huffman_codes(root.right, prefix + "1", codes)

    return codes

def huffman_encode(symbols, probabilities):
    """Complete Huffman encoding analysis."""
    tree = build_huffman_tree(symbols, probabilities)
    codes = get_huffman_codes(tree)

    # Calculate average length
    avg_length = sum(p * len(codes[s]) for s, p in zip(symbols, probabilities))

    # Calculate entropy
    probs = np.array(probabilities)
    probs = probs[probs > 0]
    entropy = -np.sum(probs * np.log2(probs))

    return {
        'codes': codes,
        'average_length': avg_length,
        'entropy': entropy,
        'efficiency': entropy / avg_length
    }

# Example
symbols = ['A', 'B', 'C', 'D', 'E']
probs = [0.4, 0.2, 0.2, 0.1, 0.1]

result = huffman_encode(symbols, probs)
print("Huffman Codes:")
for s in symbols:
    print(f"  {s}: {result['codes'][s]} (p={probs[symbols.index(s)]})")
print(f"\nAverage length: {result['average_length']:.4f} bits")
print(f"Entropy: {result['entropy']:.4f} bits")
print(f"Efficiency: {result['efficiency']:.2%}")
```

## 5. Arithmetic Coding

Arithmetic coding achieves rates closer to entropy by encoding entire messages as intervals.

<Definition term="Arithmetic Coding">
Arithmetic coding represents a message as a subinterval of [0, 1), where the interval width equals the probability of the message.
</Definition>

### Key Idea

- Start with interval [0, 1)
- For each symbol, narrow the interval proportionally to symbol probabilities
- Final interval represents the entire message
- Output enough bits to uniquely identify the interval

```python
def arithmetic_encode_demo(message, symbol_probs):
    """
    Demonstrate arithmetic coding concept.

    Args:
        message: List of symbols
        symbol_probs: Dict mapping symbol to probability
    """
    # Build cumulative distribution
    symbols = sorted(symbol_probs.keys())
    cumulative = {}
    cumsum = 0
    for s in symbols:
        cumulative[s] = (cumsum, cumsum + symbol_probs[s])
        cumsum += symbol_probs[s]

    # Encode
    low, high = 0.0, 1.0

    print("Arithmetic Coding Process:")
    print(f"Initial interval: [{low}, {high})")

    for symbol in message:
        range_width = high - low
        sym_low, sym_high = cumulative[symbol]

        new_high = low + range_width * sym_high
        new_low = low + range_width * sym_low

        print(f"  Symbol '{symbol}': [{new_low:.6f}, {new_high:.6f})")

        low, high = new_low, new_high

    # Bits needed to specify this interval
    interval_width = high - low
    bits_needed = -np.log2(interval_width)

    print(f"\nFinal interval: [{low:.10f}, {high:.10f})")
    print(f"Interval width: {interval_width:.10f}")
    print(f"Bits needed: {bits_needed:.4f}")
    print(f"Bits per symbol: {bits_needed / len(message):.4f}")

    return low, high

# Example
probs = {'A': 0.5, 'B': 0.25, 'C': 0.25}
message = ['A', 'B', 'A', 'C']

arithmetic_encode_demo(message, probs)
```

### Advantages over Huffman

1. Can achieve rates arbitrarily close to entropy
2. Better for skewed distributions (probabilities near 0 or 1)
3. Easy to adapt to changing statistics

<Warning>
Huffman coding is optimal among symbol-by-symbol codes, but arithmetic coding can beat it by considering blocks of symbols implicitly.
</Warning>

## 6. Typical Sequences and AEP

<Definition term="Typical Sequence">
A sequence $x_1, \ldots, x_n$ is $\epsilon$-typical if:

$$2^{-n(H(X)+\epsilon)} \leq p(x_1, \ldots, x_n) \leq 2^{-n(H(X)-\epsilon)}$$
</Definition>

### Asymptotic Equipartition Property (AEP)

<Definition term="AEP">
For i.i.d. random variables $X_1, \ldots, X_n$:

$$-\frac{1}{n}\log p(X_1, \ldots, X_n) \to H(X)$$

in probability as $n \to \infty$.
</Definition>

**Implications:**
1. Most probability mass concentrates on ~$2^{nH(X)}$ typical sequences
2. Each typical sequence has probability ~$2^{-nH(X)}$
3. We only need $nH(X)$ bits to index typical sequences

```python
import numpy as np
from collections import Counter

def analyze_typical_sequences(prob_dist, n_length, n_samples=10000):
    """
    Empirically analyze typical sequences.

    Args:
        prob_dist: Dict mapping symbol to probability
        n_length: Sequence length
        n_samples: Number of sequences to generate
    """
    symbols = list(prob_dist.keys())
    probs = [prob_dist[s] for s in symbols]

    # Calculate entropy
    entropy = -sum(p * np.log2(p) for p in probs if p > 0)

    # Generate sequences
    sequences = []
    log_probs = []

    for _ in range(n_samples):
        seq = np.random.choice(symbols, size=n_length, p=probs)
        # Calculate log probability
        log_p = sum(np.log2(prob_dist[s]) for s in seq)
        sequences.append(tuple(seq))
        log_probs.append(log_p)

    log_probs = np.array(log_probs)

    # Per-symbol log probability
    per_symbol = -log_probs / n_length

    print(f"Sequence length: {n_length}")
    print(f"True entropy H(X): {entropy:.4f} bits")
    print(f"Mean of -log(p)/n: {np.mean(per_symbol):.4f} bits")
    print(f"Std of -log(p)/n: {np.std(per_symbol):.4f} bits")

    # Count typical sequences (epsilon = 0.1)
    epsilon = 0.1
    typical_mask = (per_symbol >= entropy - epsilon) & (per_symbol <= entropy + epsilon)
    frac_typical = np.mean(typical_mask)

    print(f"\nFraction of {epsilon}-typical sequences: {frac_typical:.2%}")
    print(f"Expected number of typical sequences: ~2^{n_length * entropy:.1f} = {2**(n_length * entropy):.0f}")
    print(f"Total possible sequences: {len(symbols)**n_length}")

# Example
prob_dist = {'A': 0.5, 'B': 0.3, 'C': 0.2}

for n in [10, 50, 100]:
    print(f"\n{'='*50}")
    analyze_typical_sequences(prob_dist, n)
```

## 7. Quantum Source Coding

Shannon's theorem extends to quantum sources:

<Definition term="Schumacher Compression">
For a quantum source producing states $|\psi_i\rangle$ with probabilities $p_i$, the optimal compression rate is the von Neumann entropy:

$$S(\rho) = -\text{Tr}(\rho \log_2 \rho)$$

where $\rho = \sum_i p_i |\psi_i\rangle\langle\psi_i|$.
</Definition>

<Tip>
Quantum compression is fundamentally different: we compress quantum states into fewer qubits while preserving quantum coherence. This enables quantum communication at rates below classical limits when quantum entanglement is available.
</Tip>

## 8. Practical Considerations

### Universal Compression

Universal codes work without knowing the source distribution:
- **Lempel-Ziv (LZ77, LZ78)**: Foundation of gzip, PNG
- **Burrows-Wheeler Transform**: Foundation of bzip2
- Asymptotically achieve entropy rate

### Rate-Distortion Theory

When lossy compression is allowed:

<Formula>
$$R(D) = \min_{p(\hat{x}|x): E[d(X,\hat{X})] \leq D} I(X;\hat{X})$$
</Formula>

The minimum rate to achieve distortion $D$.
