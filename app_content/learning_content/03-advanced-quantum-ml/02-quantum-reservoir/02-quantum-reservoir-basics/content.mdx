## 1. From Classical to Quantum Reservoirs

Quantum reservoir computing (QRC) replaces the classical reservoir with a quantum system, exploiting quantum dynamics for information processing.

<Definition term="Quantum Reservoir Computing">
Quantum reservoir computing uses the natural dynamics of a quantum system as the reservoir. Inputs are encoded into the quantum state, the system evolves under its Hamiltonian, and measurements provide the features for a classical readout layer.
</Definition>

### The Quantum Advantage Intuition

Classical reservoirs have N neurons → N features. Quantum reservoirs with N qubits have:
- 2^N dimensional Hilbert space
- Exponentially many observables to measure
- Quantum correlations (entanglement) as resources

<Formula>
$$|\psi(t)\rangle = e^{-iHt}|\psi(0)\rangle$$
</Formula>

The quantum evolution naturally provides the nonlinear, high-dimensional transformation needed for reservoir computing.

## 2. Quantum Reservoir Architecture

```
Classical Input → Encoding → Quantum Evolution → Measurement → Readout
    u(t)           ρ(t)        U(t)ρU†(t)        ⟨Oᵢ⟩         W_out
```

### Input Encoding

Map classical inputs to quantum operations:

```python
import numpy as np

def encode_input(u, encoding='rotation'):
    """
    Encode classical input into quantum operation.

    Args:
        u: Classical input value
        encoding: 'rotation', 'amplitude', or 'phase'

    Returns:
        2x2 unitary encoding the input
    """
    if encoding == 'rotation':
        # Rotation around Y-axis
        theta = u * np.pi  # Scale input to rotation angle
        return np.array([
            [np.cos(theta/2), -np.sin(theta/2)],
            [np.sin(theta/2), np.cos(theta/2)]
        ], dtype=complex)

    elif encoding == 'amplitude':
        # Encode in superposition amplitudes
        alpha = np.cos(u * np.pi / 2)
        beta = np.sin(u * np.pi / 2)
        # Returns state, not unitary
        return np.array([alpha, beta], dtype=complex)

    elif encoding == 'phase':
        # Encode in relative phase
        return np.array([
            [1, 0],
            [0, np.exp(1j * u * np.pi)]
        ], dtype=complex)
```

### Quantum Dynamics (Reservoir)

The quantum system evolves under a Hamiltonian:

<Formula>
$$H = \sum_i h_i \sigma_i + \sum_{i<j} J_{ij} \sigma_i \sigma_j$$
</Formula>

```python
import numpy as np
from scipy.linalg import expm

class QuantumReservoir:
    """Quantum reservoir implementation."""

    def __init__(self, n_qubits, coupling_strength=1.0):
        self.n_qubits = n_qubits
        self.dim = 2**n_qubits

        # Build Hamiltonian (random Ising-like)
        self.H = self._build_hamiltonian(coupling_strength)

        # Initial state
        self.rho = np.zeros((self.dim, self.dim), dtype=complex)
        self.rho[0, 0] = 1  # |00...0⟩

    def _build_hamiltonian(self, J):
        """Build random transverse-field Ising Hamiltonian."""
        H = np.zeros((self.dim, self.dim), dtype=complex)

        # Pauli matrices
        I = np.eye(2)
        X = np.array([[0, 1], [1, 0]])
        Z = np.array([[1, 0], [0, -1]])

        def kron_n(ops):
            result = ops[0]
            for op in ops[1:]:
                result = np.kron(result, op)
            return result

        # Transverse field terms
        for i in range(self.n_qubits):
            ops = [I] * self.n_qubits
            ops[i] = X
            H += np.random.uniform(0.5, 1.5) * kron_n(ops)

        # ZZ coupling terms
        for i in range(self.n_qubits - 1):
            ops = [I] * self.n_qubits
            ops[i] = Z
            ops[i + 1] = Z
            H += J * np.random.uniform(-1, 1) * kron_n(ops)

        return H

    def encode_and_evolve(self, u, dt=1.0):
        """Encode input and evolve the system."""
        # Encode input on first qubit
        U_enc = self._encoding_unitary(u)

        # Apply encoding
        self.rho = U_enc @ self.rho @ U_enc.conj().T

        # Time evolution
        U_evol = expm(-1j * self.H * dt)
        self.rho = U_evol @ self.rho @ U_evol.conj().T

        return self.rho

    def _encoding_unitary(self, u):
        """Create encoding unitary for input u."""
        theta = u * np.pi
        Ry = np.array([
            [np.cos(theta/2), -np.sin(theta/2)],
            [np.sin(theta/2), np.cos(theta/2)]
        ], dtype=complex)

        # Apply to first qubit
        I_rest = np.eye(2**(self.n_qubits - 1))
        return np.kron(Ry, I_rest)

    def measure_observables(self):
        """Measure expectation values of observables."""
        observables = []

        # Single-qubit observables
        I = np.eye(2)
        paulis = [
            np.array([[1, 0], [0, -1]]),  # Z
            np.array([[0, 1], [1, 0]]),   # X
            np.array([[0, -1j], [1j, 0]]) # Y
        ]

        for i in range(self.n_qubits):
            for P in paulis:
                ops = [I] * self.n_qubits
                ops[i] = P
                O = ops[0]
                for op in ops[1:]:
                    O = np.kron(O, op)
                exp_val = np.real(np.trace(O @ self.rho))
                observables.append(exp_val)

        return np.array(observables)
```

## 3. Measurement and Feature Extraction

Unlike classical reservoirs where the state is directly accessible, quantum reservoirs require measurement.

<Definition term="Observable Features">
In QRC, features are extracted by measuring expectation values of observables: ⟨O⟩ = Tr(ρO). Common choices include local Pauli operators and their products.
</Definition>

### Observable Selection

```python
def generate_observables(n_qubits, order=2):
    """
    Generate observable operators for measurement.

    Args:
        n_qubits: Number of qubits
        order: Maximum Pauli string length

    Returns:
        List of observable matrices
    """
    I = np.eye(2)
    paulis = {
        'I': I,
        'X': np.array([[0, 1], [1, 0]]),
        'Y': np.array([[0, -1j], [1j, 0]]),
        'Z': np.array([[1, 0], [0, -1]])
    }

    observables = []

    # Single-qubit observables (order 1)
    if order >= 1:
        for i in range(n_qubits):
            for name, P in paulis.items():
                if name == 'I':
                    continue
                ops = [I] * n_qubits
                ops[i] = P
                obs = ops[0]
                for op in ops[1:]:
                    obs = np.kron(obs, op)
                observables.append(obs)

    # Two-qubit observables (order 2)
    if order >= 2:
        for i in range(n_qubits):
            for j in range(i+1, n_qubits):
                for n1, P1 in paulis.items():
                    for n2, P2 in paulis.items():
                        if n1 == 'I' and n2 == 'I':
                            continue
                        ops = [I] * n_qubits
                        ops[i] = P1
                        ops[j] = P2
                        obs = ops[0]
                        for op in ops[1:]:
                            obs = np.kron(obs, op)
                        observables.append(obs)

    return observables
```

<Tip>
The number of observables grows polynomially with qubit count for fixed Pauli weight, but exponentially if measuring all possible observables. Choose observables based on the task requirements.
</Tip>

## 4. Quantum Information Processing Capacity

Quantum reservoirs can exhibit enhanced information processing capacity.

### Memory Capacity

<Formula>
$$MC = \sum_{k=1}^{\infty} \max_f \text{cor}^2(y(t), u(t-k))$$
</Formula>

For quantum reservoirs, memory capacity can exceed the number of qubits due to the exponentially large Hilbert space.

### Nonlinear Capacity

Quantum dynamics naturally provide nonlinearity through:
1. **State encoding**: Mapping classical → quantum
2. **Measurement**: Projective measurement is nonlinear
3. **Quantum evolution**: Unitary evolution in exponential space

```python
def measure_quantum_memory_capacity(qrc, max_delay=20, n_samples=500):
    """
    Measure memory capacity of quantum reservoir.

    Returns total memory capacity and per-delay values.
    """
    # Generate random input
    u = np.random.uniform(-1, 1, n_samples)

    # Collect features
    features = []
    for t in range(n_samples):
        qrc.encode_and_evolve(u[t])
        obs = qrc.measure_observables()
        features.append(obs)

    features = np.array(features)

    # Measure capacity for each delay
    mc_values = []
    washout = 50

    for delay in range(1, max_delay + 1):
        X = features[washout + delay:]
        y = u[washout:-delay]

        # Train linear readout
        reg = 1e-6
        XTX = X.T @ X + reg * np.eye(X.shape[1])
        W = np.linalg.solve(XTX, X.T @ y)

        # Evaluate
        y_pred = X @ W
        corr = np.corrcoef(y_pred, y)[0, 1]
        mc = corr**2 if not np.isnan(corr) else 0
        mc_values.append(mc)

    return sum(mc_values), mc_values
```

## 5. Entanglement as a Resource

Entanglement in the quantum reservoir can enhance computational capabilities.

<Definition term="Entanglement Entropy">
For a bipartition of the system into A and B, the entanglement entropy S(ρ_A) = -Tr(ρ_A log ρ_A) quantifies quantum correlations. Higher entanglement can enable richer feature representations.
</Definition>

```python
def compute_entanglement_entropy(rho, partition_size):
    """
    Compute entanglement entropy for a bipartition.

    Args:
        rho: Full density matrix
        partition_size: Number of qubits in subsystem A
    """
    dim = rho.shape[0]
    n_qubits = int(np.log2(dim))

    dim_A = 2**partition_size
    dim_B = 2**(n_qubits - partition_size)

    # Reshape and partial trace
    rho_reshaped = rho.reshape(dim_A, dim_B, dim_A, dim_B)
    rho_A = np.trace(rho_reshaped, axis1=1, axis2=3)

    # Compute von Neumann entropy
    eigenvalues = np.linalg.eigvalsh(rho_A)
    eigenvalues = eigenvalues[eigenvalues > 1e-12]

    entropy = -np.sum(eigenvalues * np.log2(eigenvalues))

    return entropy
```

## 6. Implementation Considerations

### Noise Effects

Real quantum systems experience decoherence:

```python
def apply_depolarizing_noise(rho, p):
    """
    Apply depolarizing noise to density matrix.

    ρ → (1-p)ρ + p(I/d)
    """
    d = rho.shape[0]
    return (1 - p) * rho + p * np.eye(d) / d

class NoisyQuantumReservoir(QuantumReservoir):
    """Quantum reservoir with noise."""

    def __init__(self, n_qubits, noise_strength=0.01):
        super().__init__(n_qubits)
        self.noise = noise_strength

    def encode_and_evolve(self, u, dt=1.0):
        rho = super().encode_and_evolve(u, dt)
        # Apply noise after evolution
        self.rho = apply_depolarizing_noise(self.rho, self.noise)
        return self.rho
```

## 7. Comparison: Quantum vs Classical

| Aspect | Classical RC | Quantum RC |
|--------|-------------|------------|
| State space | N neurons | 2^N Hilbert space |
| Features | N values | O(poly(N)) to O(2^N) |
| Nonlinearity | Activation (tanh) | Measurement |
| Correlations | Statistical | Entanglement |
| Training | Ridge regression | Ridge regression |
| Hardware | Classical | Quantum |

<Tip>
Quantum advantage in reservoir computing is most pronounced for tasks that benefit from the high-dimensional feature space and quantum correlations, such as chaotic time series prediction and certain classification tasks.
</Tip>
