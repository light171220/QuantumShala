## 1. Molecular Geometry Optimization

One of the most practical applications of VQE is finding equilibrium molecular geometries.

<Definition term="Geometry Optimization">
Geometry optimization finds the nuclear configuration that minimizes the total electronic energy. At equilibrium, the gradient of energy with respect to nuclear positions is zero.
</Definition>

### Born-Oppenheimer Approach

<Formula>
$$E_{total}(\mathbf{R}) = E_{electronic}(\mathbf{R}) + E_{nuclear}(\mathbf{R})$$
</Formula>

At each geometry, VQE solves the electronic structure problem.

```python
import numpy as np
from scipy.optimize import minimize

class QuantumGeometryOptimizer:
    """Optimize molecular geometry using VQE for energies."""

    def __init__(self, molecule_builder, vqe_solver):
        """
        Args:
            molecule_builder: Function(geometry) -> Hamiltonian
            vqe_solver: VQE instance for energy calculation
        """
        self.build_molecule = molecule_builder
        self.vqe = vqe_solver
        self.energy_history = []
        self.geometry_history = []

    def compute_energy(self, geometry):
        """Compute energy at given geometry using VQE."""
        # Build Hamiltonian for this geometry
        H = self.build_molecule(geometry)

        # Run VQE
        energy = self.vqe.run(H)

        self.energy_history.append(energy)
        self.geometry_history.append(geometry.copy())

        return energy

    def compute_gradient(self, geometry, delta=0.001):
        """
        Compute energy gradient using finite differences.

        More sophisticated: analytical gradients via response theory.
        """
        n_coords = len(geometry)
        gradient = np.zeros(n_coords)

        E0 = self.compute_energy(geometry)

        for i in range(n_coords):
            geometry_plus = geometry.copy()
            geometry_plus[i] += delta

            E_plus = self.compute_energy(geometry_plus)
            gradient[i] = (E_plus - E0) / delta

        return gradient

    def optimize(self, initial_geometry, method='BFGS'):
        """Run geometry optimization."""
        result = minimize(
            self.compute_energy,
            initial_geometry,
            method=method,
            options={'disp': True}
        )

        return {
            'optimized_geometry': result.x,
            'final_energy': result.fun,
            'converged': result.success,
            'n_evaluations': len(self.energy_history)
        }

# Example: H2 bond length optimization
def h2_hamiltonian(geometry):
```

### Quantum Analytical Gradients

For efficiency, compute gradients analytically using the Hellmann-Feynman theorem:

<Formula>
$$\frac{\partial E}{\partial R_i} = \langle\psi|\frac{\partial H}{\partial R_i}|\psi\rangle$$
</Formula>

<Tip>
Analytical gradients require measuring additional observables (∂H/∂R) but avoid the multiple VQE runs needed for finite differences. For molecules with many atoms, this is essential.
</Tip>

## 2. Reaction Pathway Calculation

Understanding chemical reactions requires mapping potential energy surfaces and finding transition states.

<Definition term="Minimum Energy Path">
The minimum energy path (MEP) connects reactants to products through the lowest-energy route. The highest point along this path is the transition state, determining the reaction rate.
</Definition>

### Nudged Elastic Band Method

```python
import numpy as np

class QuantumNEB:
    """Nudged Elastic Band for reaction pathways."""

    def __init__(self, vqe_solver, n_images=10, spring_constant=0.1):
        self.vqe = vqe_solver
        self.n_images = n_images
        self.k = spring_constant

    def initialize_path(self, reactant, product):
        """Linear interpolation between endpoints."""
        path = []
        for i in range(self.n_images):
            t = i / (self.n_images - 1)
            image = (1 - t) * reactant + t * product
            path.append(image)
        return np.array(path)

    def compute_path_energies(self, path):
        """Compute VQE energy for each image."""
        energies = []
        for image in path:
            H = self.build_hamiltonian(image)
            E = self.vqe.run(H)
            energies.append(E)
        return np.array(energies)

    def compute_forces(self, path, energies):
        """
        Compute NEB forces on each image.

        Force = -∇E_perpendicular + F_spring_parallel
        """
        forces = np.zeros_like(path)
        n = len(path)

        for i in range(1, n - 1):  # Skip endpoints
            # Tangent vector
            tau = path[i + 1] - path[i - 1]
            tau = tau / np.linalg.norm(tau)

            # Energy gradient (would use VQE gradient)
            grad_E = self.numerical_gradient(path[i])

            # Perpendicular force
            F_perp = grad_E - np.dot(grad_E, tau) * tau

            # Spring force (parallel to path)
            F_spring = self.k * (
                np.linalg.norm(path[i + 1] - path[i]) -
                np.linalg.norm(path[i] - path[i - 1])
            ) * tau

            forces[i] = -F_perp + F_spring

        return forces

    def optimize_path(self, reactant, product, n_steps=100):
        """Optimize the reaction path."""
        path = self.initialize_path(reactant, product)

        for step in range(n_steps):
            energies = self.compute_path_energies(path)
            forces = self.compute_forces(path, energies)

            # Update images (simple gradient descent)
            path[1:-1] += 0.01 * forces[1:-1]

            # Find transition state (maximum energy)
            ts_idx = np.argmax(energies)

            if step % 10 == 0:
                print(f"Step {step}: TS energy = {energies[ts_idx]:.4f}")

        return path, energies
```

## 3. Strongly Correlated Materials

Variational methods excel for strongly correlated systems where classical methods fail.

<Definition term="Strongly Correlated System">
A strongly correlated system has electronic interactions comparable to or larger than kinetic energy, causing mean-field theories to fail. Examples include transition metal oxides, cuprate superconductors, and Mott insulators.
</Definition>

### The Hubbard Model

<Formula>
$$H = -t\sum_{\langle i,j\rangle,\sigma} (c_{i\sigma}^\dagger c_{j\sigma} + h.c.) + U\sum_i n_{i\uparrow}n_{i\downarrow}$$
</Formula>

```python
import numpy as np
from itertools import product

class HubbardModelVQE:
    """VQE for the Hubbard model."""

    def __init__(self, n_sites, t=1.0, U=4.0, filling=0.5):
        """
        Args:
            n_sites: Number of lattice sites
            t: Hopping parameter
            U: On-site Coulomb repulsion
            filling: Electron filling fraction
        """
        self.n_sites = n_sites
        self.t = t
        self.U = U
        self.n_electrons = int(2 * n_sites * filling)

        self.n_qubits = 2 * n_sites  # spin up + spin down

    def build_hamiltonian(self, boundary='periodic'):
        """Build Hubbard Hamiltonian in qubit basis."""
        dim = 2**self.n_qubits
        H = np.zeros((dim, dim), dtype=complex)

        # Hopping terms
        for i in range(self.n_sites):
            j = (i + 1) % self.n_sites if boundary == 'periodic' else i + 1
            if j < self.n_sites:
                # Spin up hopping
                H += self._hopping_term(i, j, spin='up')
                # Spin down hopping
                H += self._hopping_term(i, j, spin='down')

        # On-site interaction
        for i in range(self.n_sites):
            H += self._interaction_term(i)

        return H

    def _hopping_term(self, i, j, spin):
        """Build hopping term between sites i and j."""
        # Map to qubit indices
        if spin == 'up':
            qi, qj = 2*i, 2*j
        else:
            qi, qj = 2*i + 1, 2*j + 1

        # c†_i c_j + c†_j c_i in Jordan-Wigner
        # Requires string of Z operators between i and j
        # Simplified: direct matrix construction
        return self._fermionic_hopping(qi, qj)

    def _interaction_term(self, site):
        """Build on-site interaction n_↑ n_↓."""
        q_up = 2 * site
        q_down = 2 * site + 1

        dim = 2**self.n_qubits
        term = np.zeros((dim, dim), dtype=complex)

        # n_↑ n_↓ = (1-Z_up)(1-Z_down)/4
        for state in range(dim):
            n_up = (state >> (self.n_qubits - 1 - q_up)) & 1
            n_down = (state >> (self.n_qubits - 1 - q_down)) & 1
            term[state, state] = self.U * n_up * n_down

        return term

    def compute_observables(self, state):
        """Compute physical observables from VQE state."""
        observables = {}

        # Double occupancy
        D = 0
        for i in range(self.n_sites):
            D += self._expectation(state, self._interaction_term(i))
        observables['double_occupancy'] = D / (self.n_sites * self.U)

        # Total spin
        # Sz = (N_up - N_down) / 2
        observables['Sz'] = self._compute_sz(state)

        return observables

# Example usage
hubbard = HubbardModelVQE(n_sites=4, t=1.0, U=4.0)
H = hubbard.build_hamiltonian()
print(f"Hubbard model: {hubbard.n_sites} sites, U/t = {hubbard.U/hubbard.t}")
print(f"Qubits needed: {hubbard.n_qubits}")
print(f"Hilbert space dimension: {2**hubbard.n_qubits}")
```

<Warning>
The Hubbard model at half-filling with U >> t exhibits a Mott insulating phase that is notoriously difficult for classical methods. This is a prime candidate for quantum advantage.
</Warning>

## 4. Hybrid Quantum-Classical Workflows

Practical quantum chemistry combines quantum and classical methods.

<Definition term="Hybrid Workflow">
A hybrid workflow uses classical computers for parts of the calculation where they excel (geometry optimization, basis set handling, integral computation) and quantum computers for the electronically challenging parts (strong correlation, excited states).
</Definition>

## 5. Resource Estimation

Understanding quantum resource requirements is crucial for practical applications.

### Qubit and Gate Counts

```python
def estimate_vqe_resources(n_orbitals, n_electrons, ansatz_type='UCCSD'):
    """
    Estimate quantum resources for VQE.

    Args:
        n_orbitals: Number of molecular orbitals
        n_electrons: Number of electrons
        ansatz_type: 'UCCSD', 'ADAPT', 'HEA'

    Returns:
        Resource estimates
    """
    n_qubits = 2 * n_orbitals  # Spin orbitals (JW encoding)

    if ansatz_type == 'UCCSD':
        n_occ = n_electrons // 2
        n_virt = n_orbitals - n_occ

        # Singles: n_occ * n_virt
        n_singles = n_occ * n_virt

        # Doubles: C(n_occ,2) * C(n_virt,2)
        n_doubles = (n_occ * (n_occ-1) // 2) * (n_virt * (n_virt-1) // 2)

        # Each double excitation: ~8 Pauli terms
        # Each Pauli exponential: ~O(n) CNOTs for JW string
        n_parameters = n_singles + n_doubles
        n_cnots = n_doubles * 8 * n_qubits

        circuit_depth = n_doubles * n_qubits  # Rough estimate

    elif ansatz_type == 'ADAPT':
        # Typically 10-100x fewer parameters than UCCSD
        n_parameters = (n_occ * n_virt + n_doubles) // 20
        n_cnots = n_parameters * n_qubits
        circuit_depth = n_parameters * 2

    elif ansatz_type == 'HEA':
        n_layers = 4  # Typical
        n_parameters = 2 * n_qubits * n_layers
        n_cnots = n_qubits * n_layers
        circuit_depth = 2 * n_layers

    return {
        'qubits': n_qubits,
        'parameters': n_parameters,
        'cnot_gates': n_cnots,
        'circuit_depth': circuit_depth,
        'ansatz': ansatz_type
    }

# Example: Caffeine molecule (C8H10N4O2)
# ~100 orbitals in minimal basis
resources = estimate_vqe_resources(
    n_orbitals=100,
    n_electrons=102,
    ansatz_type='UCCSD'
)

print("Caffeine VQE resource estimates:")
for key, value in resources.items():
    print(f"  {key}: {value}")
```

## 6. Quantum Advantage Assessment

<Definition term="Quantum Advantage">
Quantum advantage occurs when a quantum algorithm solves a problem faster, more accurately, or with fewer resources than the best classical alternative. For chemistry, this typically means accuracy beyond classical limits for strongly correlated systems.
</Definition>

### When to Expect Advantage

| System Type | Classical Difficulty | Quantum Potential |
|-------------|---------------------|-------------------|
| Single-reference | Low (CCSD(T) works) | Limited |
| Multi-reference | High | Promising |
| Strongly correlated | Very high | High |
| Large active spaces | Exponential | Polynomial |

```python
def assess_quantum_advantage(system_properties):
    """
    Assess potential for quantum advantage.

    Args:
        system_properties: Dict with system characteristics

    Returns:
        Assessment and recommendation
    """
    assessment = {
        'quantum_recommended': False,
        'reasons': [],
        'classical_alternatives': []
    }

    # Check for multi-reference character
    if system_properties.get('multi_reference_character', 0) > 0.1:
        assessment['reasons'].append('Strong static correlation')
        assessment['quantum_recommended'] = True

    # Check for active space size
    active_size = system_properties.get('active_electrons', 0) * \
                  system_properties.get('active_orbitals', 0)
    if active_size > 30:
        assessment['reasons'].append('Large active space (FCI intractable)')
        assessment['quantum_recommended'] = True
    else:
        assessment['classical_alternatives'].append('CASSCF/CASPT2')

    # Check for strong correlation ratio
    if system_properties.get('U_over_t', 0) > 4:
        assessment['reasons'].append('Strong correlation (U/t > 4)')
        assessment['quantum_recommended'] = True

    if not assessment['quantum_recommended']:
        assessment['reasons'].append('Classical methods likely sufficient')
        assessment['classical_alternatives'].extend(['CCSD(T)', 'DMRG'])

    return assessment
```

<Tip>
Focus quantum resources on the chemically interesting parts: reaction centers, transition metals, bond-breaking regions. Use classical methods for the rest through embedding.
</Tip>
