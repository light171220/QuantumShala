# Privacy in Quantum Federated Learning

While federated learning keeps raw data local, model updates can still leak sensitive information. This lesson explores privacy-preserving techniques adapted for quantum federated learning.

- Understand privacy threats in quantum FL
- Implement differential privacy for quantum parameters
- Apply gradient perturbation techniques
- Analyze privacy-utility trade-offs
- Design privacy-preserving quantum protocols
- Evaluate quantum-specific privacy risks

## 1. Privacy Threats in Federated Learning

Even without sharing raw data, model updates can reveal sensitive information about training data.

<Definition term="Membership Inference Attack">
An attack that determines whether a specific data point was used in training by analyzing model updates or predictions.
</Definition>

```python
import numpy as np

def demonstrate_gradient_leakage():
    """
    Show how gradients can leak information.
    """
    print("Gradient Leakage Demonstration:")
    print("=" * 50)

    # Simulated scenario: binary classification
    # Client has mostly class 0, few class 1

    # True data distribution (private!)
    n_class_0 = 95
    n_class_1 = 5
    total = n_class_0 + n_class_1

    # Gradient from training reveals class imbalance
    # For softmax output, gradient ≈ prediction - true_label
    np.random.seed(42)

    # Initial model predicts ~0.5 for both classes
    pred = 0.5

    # Gradient for class 0 samples: pred - 1 = -0.5
    # Gradient for class 1 samples: pred - 0 = 0.5

    avg_gradient = (n_class_0 * (pred - 1) + n_class_1 * (pred - 0)) / total
    print(f"\nTrue class ratio: {n_class_0}:{n_class_1}")
    print(f"Average gradient: {avg_gradient:.4f}")

    # Attacker can infer ratio from gradient!
    # avg_gradient = n0/total * (pred-1) + n1/total * (pred-0)
    # avg_gradient = (n0*(pred-1) + n1*pred) / total
    # avg_gradient = (n0*pred - n0 + n1*pred) / total
    # avg_gradient = pred - n0/total

    inferred_ratio = pred - avg_gradient
    print(f"\nAttacker infers class 0 ratio: {inferred_ratio:.2%}")
    print(f"True class 0 ratio: {n_class_0/total:.2%}")

    return avg_gradient

def privacy_risks_in_quantum_fl():
    """
    Identify quantum-specific privacy risks.
    """
    print("\n\nQuantum FL Privacy Risks:")
    print("=" * 50)

    risks = [
        {

            'description': 'Quantum parameters may encode data features',
            'severity': 'high'
        },
        {
            'risk': 'Gradient analysis',
            'description': 'Gradient magnitudes reveal data characteristics',
            'severity': 'high'
        },
        {
            'risk': 'Device fingerprinting',
            'description': 'Noise patterns identify specific quantum devices',
            'severity': 'medium'
        },
        {
            'risk': 'Timing attacks',
```

<Warning>
Gradient leakage is a real threat. Even aggregate gradients can reveal class distributions, outliers, and other sensitive data properties.
</Warning>

## 2. Differential Privacy Fundamentals

Differential privacy provides mathematical guarantees about privacy by adding calibrated noise.

<Formula>
(ε, δ)-Differential Privacy:
For any two neighboring datasets D and D' differing by one record:
$$P[M(D) \in S] \leq e^\epsilon \cdot P[M(D') \in S] + \delta$$
</Formula>

```python
import numpy as np

class DifferentialPrivacy:
    """
    Differential privacy mechanisms for federated learning.
    """

    def __init__(self, epsilon, delta=1e-5):
        """
        Args:
            epsilon: Privacy budget (smaller = more private)
            delta: Probability of privacy breach
        """
        self.epsilon = epsilon
        self.delta = delta

    def gaussian_mechanism(self, value, sensitivity):
        """
        Add Gaussian noise for (ε,δ)-DP.

        Args:
            value: True value to protect
            sensitivity: Maximum change from one record
        """
        # σ = sensitivity * sqrt(2 * ln(1.25/δ)) / ε
        sigma = sensitivity * np.sqrt(2 * np.log(1.25 / self.delta)) / self.epsilon
        noise = np.random.normal(0, sigma, size=np.shape(value))
        return value + noise, sigma

    def laplace_mechanism(self, value, sensitivity):
        """
        Add Laplace noise for ε-DP.
        """
        scale = sensitivity / self.epsilon
        noise = np.random.laplace(0, scale, size=np.shape(value))
        return value + noise, scale

    def clip_gradient(self, gradient, clip_norm):
        """
        Clip gradient to bound sensitivity.
        """
        norm = np.linalg.norm(gradient)
        if norm > clip_norm:
            gradient = gradient * (clip_norm / norm)
        return gradient

# Demonstration
print("Differential Privacy Mechanisms:")
print("=" * 50)

np.random.seed(42)

# Different privacy levels
epsilons = [0.1, 1.0, 10.0]
sensitivity = 1.0  # Maximum gradient norm
true_value = np.array([1.0, 2.0, 3.0])

print(f"\nTrue value: {true_value}")
print(f"Sensitivity: {sensitivity}")

for eps in epsilons:
    dp = DifferentialPrivacy(epsilon=eps)
    noisy, sigma = dp.gaussian_mechanism(true_value, sensitivity)

    print(f"\nε = {eps}:")
    print(f"  Noise σ = {sigma:.4f}")
    print(f"  Noisy value: {noisy}")
    print(f"  Error: {np.linalg.norm(noisy - true_value):.4f}")
```

<Tip>
Smaller ε means stronger privacy but more noise. Typical values range from 0.1 (strong privacy) to 10 (weak privacy). Values above 10 provide little privacy protection.
</Tip>

## 3. Differential Privacy for Quantum Parameters

Quantum parameters require special handling due to their periodic nature and bounded ranges.

```python
import numpy as np

class QuantumDP:
    """
    Differential privacy for quantum circuit parameters.
    """

    def __init__(self, epsilon, delta=1e-5):
        self.epsilon = epsilon
        self.delta = delta

    def wrapped_gaussian_noise(self, params, sensitivity, period=2*np.pi):
        """
        Add Gaussian noise with wrapping for periodic parameters.
        """
        sigma = sensitivity * np.sqrt(2 * np.log(1.25 / self.delta)) / self.epsilon

        noise = np.random.normal(0, sigma, size=params.shape)
        noisy_params = params + noise

        # Wrap to [0, period]
        noisy_params = noisy_params % period

        return noisy_params, sigma

    def von_mises_noise(self, params, kappa):
        """
        Add von Mises noise (circular Gaussian equivalent).

        Args:
            params: Angles in [0, 2π]
            kappa: Concentration parameter (higher = less noise)
        """
        # Sample from von Mises centered at each parameter
        noisy_params = np.array([
            np.random.vonmises(p, kappa) for p in params
        ])

        # Ensure in [0, 2π]
        noisy_params = noisy_params % (2 * np.pi)

        return noisy_params

    def privacy_amplification_by_sampling(self, epsilon, sampling_rate):
        """
        Compute amplified privacy when subsampling clients.

        Privacy is stronger when only a fraction of clients participate.
        """
        # Simplified amplification (for small sampling rates)
        amplified_epsilon = np.log(1 + sampling_rate * (np.exp(epsilon) - 1))
        return amplified_epsilon

# Demonstration
print("Quantum Differential Privacy:")
print("=" * 50)

np.random.seed(42)

# Original quantum parameters
params = np.array([0.1, np.pi/2, np.pi, 3*np.pi/2, 6.0])
print(f"\nOriginal parameters: {params}")

qdp = QuantumDP(epsilon=1.0)

# Wrapped Gaussian noise
noisy_wrapped, sigma = qdp.wrapped_gaussian_noise(params, sensitivity=0.5)
print(f"\nWrapped Gaussian (σ={sigma:.3f}):")
print(f"  Noisy: {noisy_wrapped}")

# Von Mises noise (circular equivalent)
noisy_vm = qdp.von_mises_noise(params, kappa=5.0)
print(f"\nVon Mises (κ=5.0):")
print(f"  Noisy: {noisy_vm}")

# Privacy amplification
print("\n\nPrivacy Amplification by Subsampling:")
print("-" * 40)
for rate in [0.1, 0.3, 0.5, 1.0]:
    amp_eps = qdp.privacy_amplification_by_sampling(1.0, rate)
    print(f"  Sampling rate {rate:.1f}: ε = {amp_eps:.4f}")
```

## 4. Gradient Clipping and Perturbation

Bounding gradients before adding noise controls the privacy-utility trade-off.

<Formula>
DP-SGD for Quantum FL:
$$\tilde{g} = \frac{1}{n}\sum_{i=1}^{n} \text{clip}(g_i, C) + \mathcal{N}(0, \sigma^2 C^2 I)$$
</Formula>

```python
import numpy as np

class DPQuantumFL:
    """
    Differentially private quantum federated learning.
    """

    def __init__(self, n_params, epsilon, delta, clip_norm):
        self.n_params = n_params
        self.epsilon = epsilon
        self.delta = delta
        self.clip_norm = clip_norm
        self.global_params = np.random.randn(n_params) * 0.1

    def clip_gradient(self, gradient):
        """Clip gradient to bound sensitivity."""
        norm = np.linalg.norm(gradient)
        if norm > self.clip_norm:
            return gradient * (self.clip_norm / norm), True
        return gradient, False

    def compute_noise_scale(self, n_clients):
        """
        Compute noise scale for target privacy.
        """
        # Sensitivity per client is clip_norm / n_clients
        sensitivity = self.clip_norm / n_clients
        sigma = sensitivity * np.sqrt(2 * np.log(1.25 / self.delta)) / self.epsilon
        return sigma

    def private_aggregate(self, client_gradients):
        """
        Aggregate gradients with DP guarantees.
        """
        n_clients = len(client_gradients)

        # Clip each gradient
        clipped_grads = []
        clip_fractions = []
        for grad in client_gradients:
            clipped, was_clipped = self.clip_gradient(grad)
            clipped_grads.append(clipped)
            clip_fractions.append(was_clipped)

        # Average clipped gradients
        avg_gradient = np.mean(clipped_grads, axis=0)

        # Add noise
        sigma = self.compute_noise_scale(n_clients)
        noise = np.random.normal(0, sigma, size=self.n_params)
        noisy_gradient = avg_gradient + noise

        return {
            'gradient': noisy_gradient,
            'noise_scale': sigma,
            'clip_fraction': np.mean(clip_fractions)
        }

    def update(self, client_gradients, learning_rate=0.1):
        """Apply private update."""
        result = self.private_aggregate(client_gradients)
        self.global_params -= learning_rate * result['gradient']
        return result

# Demonstration
print("DP Quantum Federated Learning:")
print("=" * 50)

np.random.seed(42)

# Setup
n_params = 10
n_clients = 5

dp_fl = DPQuantumFL(
    n_params=n_params,
    epsilon=1.0,
    delta=1e-5,
    clip_norm=1.0
)

# Simulate client gradients (some are outliers)
client_gradients = [
    np.random.randn(n_params) * 0.5,    # Normal
    np.random.randn(n_params) * 0.3,    # Normal
    np.random.randn(n_params) * 2.0,    # Outlier
    np.random.randn(n_params) * 0.4,    # Normal
    np.random.randn(n_params) * 3.0,    # Outlier
]

print("\nClient gradient norms:")
for i, g in enumerate(client_gradients):
    norm = np.linalg.norm(g)
    print(f"  Client {i}: {norm:.3f}" + (" (outlier)" if norm > 1.0 else ""))

result = dp_fl.update(client_gradients)

print(f"\nPrivate aggregation:")
print(f"  Noise scale: {result['noise_scale']:.4f}")
print(f"  Clipped fraction: {result['clip_fraction']:.1%}")
print(f"  Gradient norm: {np.linalg.norm(result['gradient']):.4f}")
```

<Warning>
Aggressive clipping can slow convergence but is necessary for privacy. Monitor the clipping fraction—if too high, increase the clip norm or reduce the learning rate.
</Warning>

## 5. Privacy Accounting

Track cumulative privacy loss across training rounds.

```python
import numpy as np

class PrivacyAccountant:
    """
    Track privacy budget consumption over training.
    """

    def __init__(self, total_epsilon, delta=1e-5):
        self.total_epsilon = total_epsilon
        self.delta = delta
        self.spent_epsilon = 0.0
        self.round_history = []

    def basic_composition(self, round_epsilon):
        """
        Basic composition: epsilons add up.
        """
        self.spent_epsilon += round_epsilon
        self.round_history.append({
            'round': len(self.round_history) + 1,
            'round_epsilon': round_epsilon,
            'cumulative': self.spent_epsilon,
            'remaining': self.total_epsilon - self.spent_epsilon
        })
        return self.spent_epsilon <= self.total_epsilon

    def advanced_composition(self, round_epsilon, n_rounds):
        """
        Advanced composition theorem (tighter bound).

        For k rounds with ε per round:
        Total ε ≤ √(2k ln(1/δ')) ε + k ε (e^ε - 1)
        """
        delta_prime = self.delta / 2
        term1 = np.sqrt(2 * n_rounds * np.log(1 / delta_prime)) * round_epsilon
        term2 = n_rounds * round_epsilon * (np.exp(round_epsilon) - 1)
        total = term1 + term2
        return total

    def budget_per_round(self, n_rounds, composition='advanced'):
        """
        Compute per-round epsilon for given total budget.
        """
        if composition == 'basic':
            return self.total_epsilon / n_rounds
        else:
            # Binary search for round epsilon
            low, high = 0.0, self.total_epsilon
            for _ in range(50):
                mid = (low + high) / 2
                total = self.advanced_composition(mid, n_rounds)
                if total < self.total_epsilon:
                    low = mid
                else:
                    high = mid
            return low

    def get_status(self):
        """Get current privacy status."""
        return {
            'total_budget': self.total_epsilon,
            'spent': self.spent_epsilon,
            'remaining': max(0, self.total_epsilon - self.spent_epsilon),
            'rounds_completed': len(self.round_history),
            'exhausted': self.spent_epsilon >= self.total_epsilon
        }

# Demonstration
print("Privacy Accounting:")
print("=" * 50)

accountant = PrivacyAccountant(total_epsilon=5.0, delta=1e-5)

# Compare composition methods
n_rounds = 100

basic_per_round = accountant.budget_per_round(n_rounds, 'basic')
advanced_per_round = accountant.budget_per_round(n_rounds, 'advanced')

print(f"\nFor {n_rounds} rounds with total ε = {accountant.total_epsilon}:")
print(f"  Basic composition: ε/round = {basic_per_round:.4f}")
print(f"  Advanced composition: ε/round = {advanced_per_round:.4f}")
print(f"  Improvement: {advanced_per_round/basic_per_round:.2f}x")

# Simulate training
print("\n\nSimulating training with basic composition:")
round_epsilon = 0.1

for r in range(1, 21):
    can_continue = accountant.basic_composition(round_epsilon)
    if r <= 5 or r >= 18:
        status = accountant.get_status()
        print(f"  Round {r}: spent={status['spent']:.2f}, "
              f"remaining={status['remaining']:.2f}")
    elif r == 6:
        print("  ...")

    if not can_continue:
        print(f"  Budget exhausted at round {r}!")
        break

status = accountant.get_status()
print(f"\nFinal status: {status}")
```

## 6. Privacy-Utility Trade-offs

Balancing privacy and model utility requires careful tuning.

```python
import numpy as np

def privacy_utility_analysis():
    """
    Analyze trade-offs between privacy and utility.
    """
    print("Privacy-Utility Trade-off Analysis:")
    print("=" * 50)

    np.random.seed(42)

    # Simulate training with different privacy levels
    epsilons = [0.1, 0.5, 1.0, 2.0, 5.0, 10.0, float('inf')]
    n_params = 50
    n_clients = 10
    n_rounds = 50
    clip_norm = 1.0

    # True gradient (what we'd get without privacy)
    true_gradient = np.random.randn(n_params) * 0.3

    results = []

    for epsilon in epsilons:
        if epsilon == float('inf'):
            # No privacy
            noisy_gradient = true_gradient
        else:
            # Add DP noise
            delta = 1e-5
            sensitivity = clip_norm / n_clients
            sigma = sensitivity * np.sqrt(2 * np.log(1.25 / delta)) / epsilon
            noise = np.random.normal(0, sigma, size=n_params)
            noisy_gradient = true_gradient + noise

        # Compute metrics
        mse = np.mean((noisy_gradient - true_gradient) ** 2)
        cosine_sim = np.dot(noisy_gradient, true_gradient) / (
            np.linalg.norm(noisy_gradient) * np.linalg.norm(true_gradient)
        )

        results.append({
            'epsilon': epsilon,
            'mse': mse,
            'cosine_similarity': cosine_sim
        })

        eps_str = f"ε={epsilon}" if epsilon != float('inf') else "No DP"
        print(f"\n{eps_str:12s}:")
        print(f"  MSE: {mse:.6f}")
        print(f"  Cosine similarity: {cosine_sim:.4f}")

    # Recommendations
    print("\n\nRecommendations:")
    print("-" * 40)
    print("• Strong privacy (ε ≤ 1): Significant utility loss")
    print("  Use for: Sensitive medical/financial data")
    print("• Moderate privacy (ε = 2-5): Balanced trade-off")
    print("  Use for: General applications")
    print("• Weak privacy (ε > 10): Minimal protection")
    print("  Use for: Non-sensitive data, auditing purposes")

    return results

def techniques_to_improve_tradeoff():
    """
    Techniques to get better privacy-utility trade-off.
    """
    print("\n\nTechniques for Better Trade-offs:")
    print("=" * 50)

    techniques = [
        {
            'name': 'Larger batch sizes',
            'description': 'Averaging more samples reduces noise impact',
            'privacy_cost': 'Same',
            'utility_gain': 'Moderate'
        },
        {
            'name': 'Privacy amplification by subsampling',
            'description': 'Random client selection amplifies privacy',
            'privacy_cost': 'Reduced',
            'utility_gain': 'Slight'
        },
        {
            'name': 'Gradient compression + DP',
            'description': 'Compress before adding noise',
            'privacy_cost': 'Same',
            'utility_gain': 'Moderate'
        },
        {
            'name': 'Local DP + shuffling',
            'description': 'Shuffle updates before aggregation',
            'privacy_cost': 'Reduced',
            'utility_gain': 'Good'
        },
        {
            'name': 'Adaptive clipping',
            'description': 'Tune clip norm based on gradient distribution',
            'privacy_cost': 'Same',
            'utility_gain': 'Good'
        }

    for t in techniques:
```

<Tip>
Start with moderate privacy (ε ≈ 2-5) and adjust based on observed model quality. Use privacy amplification techniques to get more utility for the same privacy level.
</Tip>

## Summary

Privacy in quantum federated learning requires multiple layers of protection:

1. **Threat Model**: Understand what gradients can reveal
2. **Differential Privacy**: Add calibrated noise for formal guarantees
3. **Quantum-Specific Handling**: Respect parameter periodicity
4. **Gradient Clipping**: Bound sensitivity before adding noise
5. **Privacy Accounting**: Track budget consumption over time
6. **Trade-off Management**: Balance privacy and utility

The combination of federated learning, differential privacy, and secure aggregation provides strong privacy guarantees for quantum ML.
