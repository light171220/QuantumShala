## 2. Portfolio Optimization

The Markowitz portfolio optimization problem is well-suited for quantum approaches.

<Formula>
Mean-variance portfolio optimization:
$$\min_w \left( \lambda w^T \Sigma w - (1-\lambda) \mu^T w \right)$$
subject to $\sum_i w_i = 1$, $w_i \in \{0, 1\}$
</Formula>

```python
import numpy as np

class QuantumPortfolioOptimizer:
    """
    Portfolio optimization using QAOA-style approach.
    """

    def __init__(self, returns, covariance, risk_factor=0.5):
        """
        Args:
            returns: Expected returns for each asset
            covariance: Covariance matrix of returns
            risk_factor: Weight between risk (1) and return (0)
        """
        self.returns = np.array(returns)
        self.covariance = np.array(covariance)
        self.risk_factor = risk_factor
        self.n_assets = len(returns)

    def classical_cost(self, selection):
        """
        Compute portfolio cost for binary selection.
        """
        w = np.array(selection, dtype=float)

        # Normalize to sum to 1 if any selected
        if np.sum(w) > 0:
            w = w / np.sum(w)

        # Risk (variance)
        risk = w @ self.covariance @ w

        # Return
        expected_return = self.returns @ w

        # Combined objective (minimize)
        cost = self.risk_factor * risk - (1 - self.risk_factor) * expected_return

        return {
            'cost': cost,
            'risk': risk,
            'return': expected_return,
            'weights': w
        }

    def build_qubo(self):
        """
        Build QUBO formulation for QAOA/quantum annealing.
        """
        n = self.n_assets

        # QUBO matrix Q where cost = x^T Q x
        Q = np.zeros((n, n))

        # Risk term: λ Σ_ij Σ_ij x_i x_j
        Q += self.risk_factor * self.covariance

        # Return term: -(1-λ) Σ_i μ_i x_i (diagonal)
        for i in range(n):
            Q[i, i] -= (1 - self.risk_factor) * self.returns[i]

        return Q

    def exhaustive_search(self):
        """
        Find optimal portfolio via exhaustive search.
        """
        best_cost = float('inf')
        best_selection = None

        for bits in range(1, 2**self.n_assets):  # Skip all-zeros
            selection = [int(b) for b in format(bits, f'0{self.n_assets}b')]
            result = self.classical_cost(selection)

            if result['cost'] < best_cost:
                best_cost = result['cost']
                best_selection = selection
                best_result = result

        return best_selection, best_result

# Demonstration
```

## 3. Quantum Amplitude Estimation for Risk

Quantum amplitude estimation can speed up Monte Carlo simulations for risk metrics.

<Definition term="Quantum Amplitude Estimation">
A quantum algorithm that estimates the probability of measuring a specific outcome, achieving quadratic speedup over classical Monte Carlo sampling.
</Definition>

```python
import numpy as np

def risk_metrics_overview():
    """
    Explain risk metrics and quantum approach.
    """
    print("Quantum Risk Analysis:")
    print("=" * 50)

    metrics = [
        {

            'definition': 'Maximum loss at given confidence level',
            'classical': 'Monte Carlo simulation',

            'speedup': 'Quadratic (√N samples)'
        },
        {
            'name': 'Conditional VaR (CVaR)',
            'definition': 'Expected loss beyond VaR',
            'classical': 'Monte Carlo with conditional expectation',
            'quantum': 'Modified amplitude estimation',
            'speedup': 'Quadratic'
        },
        {
            'name': 'Credit Risk',
            'definition': 'Probability of default',
            'classical': 'Credit scoring models',
            'quantum': 'Quantum ML for classification',
            'speedup': 'Problem-dependent'

    ]

    for m in metrics:
        print(f"\n{m['name']}:")
        print(f"  Definition: {m['definition']}")
        print(f"  Classical: {m['classical']}")
        print(f"  Quantum: {m['quantum']}")
        print(f"  Speedup: {m['speedup']}")

def simulate_var_calculation():
    """
    Simulate VaR calculation classically and explain quantum approach.
    """
    print("\n\nVaR Calculation Example:")
    print("=" * 50)

    np.random.seed(42)

    # Portfolio parameters
    portfolio_value = 1_000_000
    expected_return = 0.08
    volatility = 0.15
    confidence = 0.95

    # Classical Monte Carlo
    n_simulations = 10000
    returns = np.random.normal(expected_return, volatility, n_simulations)
    losses = -returns * portfolio_value

    var_classical = np.percentile(losses, confidence * 100)

    print(f"\nPortfolio: ${portfolio_value:,}")
    print(f"Expected return: {expected_return:.1%}")
    print(f"Volatility: {volatility:.1%}")

    print(f"\nClassical Monte Carlo ({n_simulations:,} simulations):")
    print(f"  VaR (95%): ${var_classical:,.0f}")

    # CVaR
    cvar = np.mean(losses[losses > var_classical])
    print(f"  CVaR (95%): ${cvar:,.0f}")

    # Quantum approach explanation
    print("\n\nQuantum Amplitude Estimation Approach:")
    print("-" * 40)
    print("1. Encode probability distribution as quantum state")
    print("2. Create operator A where P(loss > threshold) = |⟨1|A|0⟩|²")
    print("3. Use amplitude estimation to find probability")
    print(f"4. Speedup: {n_simulations} classical → {int(np.sqrt(n_simulations))} quantum queries")

risk_metrics_overview()
simulate_var_calculation()
```

<Warning>
Quantum amplitude estimation requires coherent operations. On NISQ devices, noise limits the practical speedup, often negating the theoretical quadratic advantage.
</Warning>

## 4. Option Pricing

Quantum computing can potentially accelerate derivative pricing.

```python
import numpy as np

def option_pricing_quantum():
    """
    Quantum approach to option pricing.
    """
    print("Quantum Option Pricing:")
    print("=" * 50)

    # Black-Scholes parameters
    S0 = 100      # Initial stock price
    K = 105       # Strike price
    r = 0.05      # Risk-free rate
    sigma = 0.2   # Volatility
    T = 1.0       # Time to maturity

    # Classical Monte Carlo pricing
    np.random.seed(42)
    n_paths = 10000

    # Simulate stock paths
    Z = np.random.randn(n_paths)
    ST = S0 * np.exp((r - 0.5 * sigma**2) * T + sigma * np.sqrt(T) * Z)

    # Call option payoff
    payoffs = np.maximum(ST - K, 0)
    call_price = np.exp(-r * T) * np.mean(payoffs)
    std_error = np.std(payoffs) / np.sqrt(n_paths) * np.exp(-r * T)

    print(f"\nEuropean Call Option:")
    print(f"  Spot: ${S0}, Strike: ${K}")
    print(f"  Rate: {r:.1%}, Vol: {sigma:.1%}, Maturity: {T}y")

    print(f"\nClassical Monte Carlo ({n_paths:,} paths):")
    print(f"  Price: ${call_price:.2f}")
    print(f"  Std Error: ${std_error:.4f}")

    # Quantum approach
    print("\n\nQuantum Monte Carlo Approach:")
    print("-" * 40)

    steps = [
        "1. Encode price distribution into quantum state",
        "2. Create payoff operator: |S⟩ → max(S-K, 0)|S⟩",
        "3. Use amplitude estimation for expected payoff",
        f"4. Theoretical speedup: {n_paths:,} → {int(np.sqrt(n_paths)):,} queries"
    ]

    for step in steps:
        print(f"  {step}")

    # Challenges
    print("\n\nChallenges for NISQ:")
    challenges = [
        "Loading probability distributions requires deep circuits",
        "Amplitude estimation needs many controlled operations",
        "Error rate limits practical speedup",
        "Classical methods are highly optimized"
    ]
    for c in challenges:
        print(f"  • {c}")

option_pricing_quantum()
```

## Summary

Quantum finance offers potential but requires realistic expectations:

1. **Applications**: Portfolio optimization, risk analysis, derivative pricing
2. **Current State**: Small demonstrations, no practical advantage yet
3. **QUBO Formulation**: Map portfolio problems to quantum optimization
4. **Amplitude Estimation**: Theoretical speedup for Monte Carlo
5. **Limitations**: Noise, depth, competition from optimized classical methods
6. **Strategy**: Build capabilities now, expect advantage in 5-10+ years

Finance will likely be an early adopter once quantum hardware matures.
