# Quantum Speedup Analysis

Understanding when quantum computers can actually outperform classical computers for Monte Carlo requires careful analysis of complexity, resources, and practical constraints.

<Definition term="Quantum Advantage">
The condition where a quantum algorithm solves a problem faster than any classical algorithm, considering all overheads including error correction, state preparation, and measurement. True quantum advantage requires the total quantum computation time to be less than the classical time.
</Definition>

## Theoretical Speedup

The fundamental speedup in quantum Monte Carlo comes from amplitude estimation.

<Formula expression="\\text{Speedup} = \\frac{O(1/\\epsilon^2)}{O(1/\\epsilon)} = O(1/\\epsilon)" description="Quadratic speedup in terms of target accuracy ε" />

```python
import numpy as np
import matplotlib.pyplot as plt

def theoretical_speedup_analysis():
    """Analyze theoretical quantum speedup for Monte Carlo."""

    # Accuracy levels
    epsilons = np.logspace(-1, -5, 50)

    # Classical complexity: O(σ²/ε²) where σ is standard deviation
    sigma = 1.0  # Normalized
    classical = sigma**2 / epsilons**2

    # Quantum complexity: O(1/ε) queries
    quantum = 1 / epsilons

    # Speedup
    speedup = classical / quantum

    print("Theoretical Speedup Analysis:")
    print("=" * 55)
    print(f"{'Accuracy ε':>12} {'Classical':>15} {'Quantum':>12} {'Speedup':>10}")
    print("-" * 52)

    for eps in [0.1, 0.01, 0.001, 0.0001, 0.00001]:
        c = int(sigma**2 / eps**2)
        q = int(1 / eps)
        s = c / q
        print(f"{eps:>12.5f} {c:>15,} {q:>12,} {s:>10.0f}x")

    return epsilons, speedup

eps, speedup = theoretical_speedup_analysis()
```

## State Preparation Bottleneck

State preparation can be the dominant cost, potentially negating the quantum speedup.

```python
def state_preparation_analysis():
    """Analyze state preparation costs for different distributions."""

    distributions = {
        'uniform': {
            'gates': lambda n: n,  # Just H gates
            'description': 'n Hadamard gates',
            'efficient': True
        },
        'normal': {
            'gates': lambda n: n**2,  # Grover-Rudolph for log-concave
            'description': 'O(n²) controlled rotations',
            'efficient': True
        },
        'lognormal': {
            'gates': lambda n: n**2,  # Transform of normal
            'description': 'O(n²) via log transformation',
            'efficient': True
        },
        'empirical': {
            'gates': lambda n: 2**n,  # General QRAM loading
            'description': 'O(2^n) for arbitrary distribution',
            'efficient': False
        },
        'mixture': {
            'gates': lambda n: n * 2**n,  # Multiple modes
            'description': 'Depends on number of components',
            'efficient': False
        }
    }

    n_qubits = [4, 6, 8, 10, 12]

    print("State Preparation Costs by Distribution:")
    print("=" * 60)

    for dist, details in distributions.items():
        print(f"\n{dist.upper()} ({details['description']}):")
        print(f"  Efficient: {details['efficient']}")
        print(f"  Gate counts:", end="")
        for n in n_qubits:
            gates = details['gates'](n)
            if gates > 1e9:
                print(f" n={n}:>10^9", end="")
            else:
                print(f" n={n}:{gates:,}", end="")
        print()

    # Impact on speedup
    print("\n" + "=" * 60)
    print("Impact on Quantum Advantage:")
    print("  - Efficient preparation (O(poly(n))): Speedup preserved")
    print("  - Exponential preparation (O(2^n)): Speedup negated")
    print("  - Financial applications typically use log-concave: OK")

    return distributions

state_preparation_analysis()
```

<Warning>
If state preparation requires O(2^n) gates, the quantum algorithm becomes exponentially slower than classical Monte Carlo. This is why efficient state preparation is crucial for quantum advantage.
</Warning>

## Error Correction Overhead

Fault-tolerant quantum computing adds significant overhead.

```python
def error_correction_overhead():
    """Analyze the impact of error correction on quantum speedup."""

    # Logical gate implementations
    # Assume surface code with code distance d

    def physical_qubits(logical_qubits, code_distance):
        """Physical qubits needed for surface code."""
        return logical_qubits * 2 * code_distance**2

    def t_gate_cost(code_distance):
        """Magic state distillation cost for T gate."""
        # Rough estimate: 10-100 physical T gates per logical T gate
        return 10 * code_distance

    # Error rates and required code distances
    print("Error Correction Overhead Analysis:")
    print("=" * 55)

    target_error_rates = [1e-3, 1e-6, 1e-9, 1e-12]
    physical_error_rate = 1e-3  # Assume 0.1% physical error rate

    for target in target_error_rates:
        # Code distance needed (rough approximation)
        d = max(3, int(np.ceil(-np.log10(target / physical_error_rate) * 2)))

        logical_q = 100  # Example: 100 logical qubits
        physical_q = physical_qubits(logical_q, d)
        t_cost = t_gate_cost(d)

        print(f"\nTarget error rate: {target:.0e}")
        print(f"  Required code distance: {d}")
        print(f"  Physical qubits (for {logical_q} logical): {physical_q:,}")
        print(f"  T-gate overhead: {t_cost}x")

    # Impact on total runtime
    print("\n" + "=" * 55)
    print("Net effect on quantum speedup:")
    print("  Theoretical speedup: O(1/ε)")
    print("  After error correction: Still O(1/ε), but with large constant")
    print("  Break-even requires: ε small enough to overcome constant factor")

    return True
```

## Break-Even Analysis

When does quantum Monte Carlo actually beat classical?

```python
def break_even_analysis():
    """Calculate break-even point for quantum advantage."""

    # Assumptions
    classical_ops_per_second = 1e9  # 1 GHz effective
    quantum_ops_per_second = 1e6   # 1 MHz (conservative with EC)
    ec_overhead = 100               # Error correction factor
    state_prep_overhead = 100       # State preparation gates per iteration

    # Classical cost: O(1/ε²) operations
    def classical_time(epsilon):
        return (1 / epsilon**2) / classical_ops_per_second

    # Quantum cost: O(1/ε) iterations × overhead per iteration
    def quantum_time(epsilon):
        iterations = 1 / epsilon
        ops_per_iteration = state_prep_overhead * ec_overhead
        total_ops = iterations * ops_per_iteration
        return total_ops / quantum_ops_per_second

    # Find break-even
    epsilons = np.logspace(-2, -6, 100)
    classical_times = [classical_time(e) for e in epsilons]
    quantum_times = [quantum_time(e) for e in epsilons]

    # Find crossover
    for i, (c, q, e) in enumerate(zip(classical_times, quantum_times, epsilons)):
        if q < c:
            break_even_eps = e
            break_even_classical = c
            break_even_quantum = q
            break

    print("Break-Even Analysis:")
    print("=" * 55)
    print(f"Assumptions:")
    print(f"  Classical speed: {classical_ops_per_second:.0e} ops/sec")
    print(f"  Quantum speed: {quantum_ops_per_second:.0e} ops/sec")
    print(f"  EC overhead: {ec_overhead}x")
    print(f"  State prep overhead: {state_prep_overhead} gates/iteration")

    print(f"\nBreak-even accuracy: ε ≈ {break_even_eps:.2e}")
    print(f"  Classical time at break-even: {break_even_classical:.2f} seconds")
    print(f"  Quantum time at break-even: {break_even_quantum:.2f} seconds")

    # Financial application examples
    print("\n" + "=" * 55)
    print("Financial Application Requirements:")

    applications = [
        ("Option pricing", 0.001, "1-10 seconds typical"),
        ("VaR (daily)", 0.01, "Minutes acceptable"),
        ("CVA computation", 0.0001, "Hours for large portfolios"),
        ("Real-time hedging", 0.01, "Milliseconds needed")
    ]

    for app, eps, classical_req in applications:
        c_time = classical_time(eps)
        q_time = quantum_time(eps)
        speedup = c_time / q_time
        print(f"\n{app} (ε = {eps}):")
        print(f"  Classical: {c_time:.2f} sec")
        print(f"  Quantum: {q_time:.2f} sec")
        print(f"  Speedup: {speedup:.1f}x")
        print(f"  Practical need: {classical_req}")

    return break_even_eps

break_even_analysis()
```

<Tip>
The break-even point depends heavily on hardware assumptions. As quantum hardware improves (higher gate fidelity, faster operations), the break-even accuracy will increase, making quantum advantage accessible for more applications.
</Tip>

## Summary

Quantum speedup analysis reveals:

| Factor | Impact on Advantage |
|--------|-------------------|
| Theoretical speedup | O(1/ε) - quadratic |
| State preparation | Can negate advantage if exponential |
| Error correction | 100-1000x overhead |
| Practical break-even | ε ~ 10^-4 to 10^-6 |

Key insights:

- Quadratic speedup is real but comes with significant overhead
- State preparation must be efficient (polynomial) for advantage
- Error correction adds substantial constant factors
- Break-even occurs at very high accuracy requirements
- Timeline: 2030-2040 for practical financial applications
