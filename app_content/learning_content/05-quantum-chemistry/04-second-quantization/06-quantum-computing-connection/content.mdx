## 1. The Mapping Problem

<Definition term="Fermion-to-Qubit Mapping">
A mathematical transformation that represents fermionic creation and annihilation operators as operators acting on qubits, preserving the anticommutation relations required for proper fermionic statistics.
</Definition>

```python
def mapping_problem():
    """Explain the fermion-to-qubit mapping problem."""

    print("=" * 60)

    print("\nThe Challenge:")
    print("  Fermions: {cŠĶĘ, c‚Ä†‚Īľ} = őīŠĶĘ‚Īľ (anticommutation)")
    print("  Qubits: [ŌÉŠĶĘ, ŌÉ‚Īľ] = 0 for i‚Č†j (commutation)")
    print()
    print("  How to represent fermions using qubits?")

    print("\nNaive Approach (Fails!):")
    print("-" * 45)
    print("  Try: c‚Ä†ŠĶĘ ‚Üí ŌÉ‚ĀļŠĶĘ = (XŠĶĘ - iYŠĶĘ)/2")
    print("       cŠĶĘ ‚Üí ŌÉ‚ĀĽŠĶĘ = (XŠĶĘ + iYŠĶĘ)/2")
    print()
    print("  Problem: ŌÉ‚Āļ‚āĀŌÉ‚Āļ‚āā = ŌÉ‚Āļ‚āāŌÉ‚Āļ‚āĀ (commute!)")

    mappings = [
```

## 2. Jordan-Wigner Transformation

<Formula>
$$c_j^\dagger = \left(\prod_{k<j} Z_k\right) \sigma_j^+ = Z_0 Z_1 \cdots Z_{j-1} \frac{X_j - iY_j}{2}$$
</Formula>

```python
import numpy as np

def jordan_wigner():
    """Explain the Jordan-Wigner transformation."""

    print("Jordan-Wigner Transformation")
    print("=" * 60)

    print("\nTransformation Rules:")
    print("-" * 45)
    print("  c‚Ä†‚Īľ = Z‚āÄZ‚āĀ...Z‚Īľ‚āč‚āĀ ‚äó ŌÉ‚Āļ‚Īľ ‚äó I ‚äó I ‚äó ...")
    print("  c‚Īľ = Z‚āÄZ‚āĀ...Z‚Īľ‚āč‚āĀ ‚äó ŌÉ‚ĀĽ‚Īľ ‚äó I ‚äó I ‚äó ...")
    print()
    print("  where ŌÉ¬Ī = (X ¬Ī iY)/2")

    print("\nNumber Operator:")
    print("  nŐā‚Īľ = c‚Ä†‚Īľc‚Īľ = (I - Z‚Īľ)/2")
    print()
    print("  No string needed! Z's cancel.")

    print("\nExamples (4 qubits):")
    print("-" * 45)
    print("  c‚Ä†‚āÄ = ŌÉ‚Āļ‚āÄ ‚äó I ‚äó I ‚äó I")
    print("  c‚Ä†‚āĀ = Z‚āÄ ‚äó ŌÉ‚Āļ‚āĀ ‚äó I ‚äó I")
    print("  c‚Ä†‚āā = Z‚āÄ ‚äó Z‚āĀ ‚äó ŌÉ‚Āļ‚āā ‚äó I")
    print("  c‚Ä†‚āÉ = Z‚āÄ ‚äó Z‚āĀ ‚äó Z‚āā ‚äó ŌÉ‚Āļ‚āÉ")

    print("\nWhy It Works:")
    print("-" * 45)
    print("  Consider c‚Ä†‚āĀc‚Ä†‚āā vs c‚Ä†‚āāc‚Ä†‚āĀ:")
    print()
    print("  c‚Ä†‚āĀc‚Ä†‚āā = (Z‚āÄ‚äóŌÉ‚Āļ‚āĀ‚äóI‚äóI)(Z‚āÄ‚äóZ‚āĀ‚äóŌÉ‚Āļ‚āā‚äóI)")
    print("        = I‚äóŌÉ‚Āļ‚āĀZ‚āĀ‚äóŌÉ‚Āļ‚āā‚äóI")
    print()
    print("  c‚Ä†‚āāc‚Ä†‚āĀ = (Z‚āÄ‚äóZ‚āĀ‚äóŌÉ‚Āļ‚āā‚äóI)(Z‚āÄ‚äóŌÉ‚Āļ‚āĀ‚äóI‚äóI)")
    print("        = I‚äóZ‚āĀŌÉ‚Āļ‚āĀ‚äóŌÉ‚Āļ‚āā‚äóI")
    print()
    print("  Since ŌÉ‚ĀļZ = -ZŌÉ‚Āļ, we get c‚Ä†‚āĀc‚Ä†‚āā = -c‚Ä†‚āāc‚Ä†‚āĀ ‚úď")

jordan_wigner()
```

## 3. Building the Qubit Hamiltonian

```python
def qubit_hamiltonian():
    """Build molecular Hamiltonian for quantum computer."""

    print("Building the Qubit Hamiltonian")
    print("=" * 60)

    print("\nFermionic Hamiltonian:")
    print("  ń§ = ő£‚āö‚āĎ h‚āö‚āĎ c‚Ä†‚āöcq + ¬Ĺő£‚āö‚āĎŠĶ£‚āõ (pq|rs) c‚Ä†‚āöc‚Ä†ŠĶ£c‚āõcq + E_nuc")

    print("\nStep 1: Apply Jordan-Wigner")
    print("-" * 45)
    print("  Replace each c‚Ä†, c with Pauli strings")

    print("\nStep 2: Expand Products")
    print("-" * 45)
    print("  Each term becomes sum of Pauli strings")
    print()
    print("  Example: c‚Ä†‚āÄc‚āĀ = (X‚āÄ-iY‚āÄ)(Z‚āÄ)(X‚āĀ+iY‚āĀ)/4")
    print("         = (X‚āÄZ‚āÄX‚āĀ + iX‚āÄZ‚āÄY‚āĀ - iY‚āÄZ‚āÄX‚āĀ + Y‚āÄZ‚āÄY‚āĀ)/4")
    print("         = (-iY‚āÄX‚āĀ + X‚āÄY‚āĀ - iY‚āÄiY‚āĀ - X‚āÄX‚āĀ)/4")
    print("         = (-Y‚āÄX‚āĀ + X‚āÄY‚āĀ)/4i + (-Y‚āÄY‚āĀ - X‚āÄX‚āĀ)/4")

    print("\nStep 3: Collect Terms")
    print("-" * 45)
    print("  ń§_qubit = ő£‚āĖ w‚āĖ P‚āĖ")
    print()
    print("  P‚āĖ = Pauli string (e.g., X‚āÄY‚āĀZ‚āāI‚āÉ)")
    print("  w‚āĖ = coefficient (real number)")

    print("\nResult: Hamiltonian as Weighted Pauli Sum")
    print("-" * 45)
    print("  ń§ = 0.5 III...I")
    print("    + 0.2 Z‚āÄI‚āĀI‚āā...")
    print("    + 0.1 X‚āÄY‚āĀZ‚āā...")
    print("    + ...")

qubit_hamiltonian()
```

<Tip>
The number of Pauli terms in the qubit Hamiltonian scales as O(N‚Āī) for N orbitals, coming from the two-electron integrals. However, many terms have the same Pauli string and can be combined.
</Tip>

## 5. Bravyi-Kitaev Transformation

<Definition term="Bravyi-Kitaev Transformation">
A fermion-to-qubit mapping that encodes occupation numbers in a binary tree structure, reducing the average weight of Pauli strings from O(N) to O(log N).
</Definition>

```python
def bravyi_kitaev():
    """Explain the Bravyi-Kitaev transformation."""

    print("Bravyi-Kitaev Transformation")
    print("=" * 60)

    print("\nIdea:")
    print("  Jordan-Wigner: qubit j stores occupation of orbital j")
    print("  Bravyi-Kitaev: qubit j stores PARITY of some orbitals")

    print("\nBinary Tree Structure:")
    print("-" * 45)
    print("  Qubits store partial sums of occupations")
    print("  Like a Fenwick tree (binary indexed tree)")
    print()
    print("  Example (8 orbitals):")
    print("    q‚āÄ = n‚āÄ")
    print("    q‚āĀ = n‚āÄ + n‚āĀ")
    print("    q‚āā = n‚āā")
    print("    q‚āÉ = n‚āÄ + n‚āĀ + n‚āā + n‚āÉ")
    print("    q‚āĄ = n‚āĄ")
    print("    ...")

    print("\nAdvantage:")
    print("-" * 45)
    print("  Jordan-Wigner: O(N) Pauli weight for c‚Ä†‚Īľ")
    print("  Bravyi-Kitaev: O(log N) Pauli weight")
    print()
    print("  Fewer gates needed for simulation!")

    print("\nComparison (16 orbitals):")
    print("-" * 45)
    print("  Jordan-Wigner:")
    print("    c‚Ä†‚āĀ‚āÖ requires Z‚āÄZ‚āĀZ‚āā...Z‚āĀ‚āĄ string")
    print("    Weight: 15")
    print()
    print("  Bravyi-Kitaev:")
    print("    c‚Ä†‚āĀ‚āÖ requires ~4 Pauli operators")
    print("    Weight: O(log 16) = 4")

    print("\nTrade-off:")
    print("  BK has more complex update rules")
    print("  But saves gates for large systems")

bravyi_kitaev()
```

<Warning>
Real quantum chemistry on quantum computers requires handling many practical issues: Trotterization errors, measurement shot noise, NISQ device noise, and the exponential cost of full tomography. Always benchmark classical methods first!

## Summary

1. **Fermion-to-qubit mapping** is needed because fermions anticommute
