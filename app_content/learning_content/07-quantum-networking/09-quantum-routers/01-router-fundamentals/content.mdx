# Quantum Router Fundamentals

## Introduction

Quantum routers are essential components for building large-scale quantum networks. Unlike classical routers that forward copied packets, quantum routers must handle quantum states that cannot be cloned, requiring fundamentally different approaches.

## Quantum vs Classical Routing

```
┌─────────────────────────────────────────────────────────────────────┐
│              CLASSICAL VS QUANTUM ROUTING                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  CLASSICAL ROUTING:                                                 │
│  ┌──────────────────────────────────────────────────────────────┐  │
│  │                                                              │  │
│  │  Packet ──→ [Router] ──→ Copy to buffer                     │  │
│  │                 │         ──→ Read destination               │  │
│  │                 │         ──→ Forward on correct port        │  │
│  │                 ▼                                            │  │
│  │  Properties: Store-and-forward, Copying allowed, No urgency  │  │
│  │                                                              │  │
│  └──────────────────────────────────────────────────────────────┘  │
│                                                                     │
│  QUANTUM ROUTING:                                                   │
│  ┌──────────────────────────────────────────────────────────────┐  │
│  │                                                              │  │
│  │  Qubit ──→ [Router] ──→ Cannot copy (no-cloning)            │  │
│  │                │         ──→ Cannot read without disturbing  │  │
│  │                │         ──→ Must preserve coherence          │  │
│  │                ▼                                             │  │
│  │  Properties: Cut-through, No copying, Time-critical          │  │
│  │                                                              │  │
│  └──────────────────────────────────────────────────────────────┘  │
│                                                                     │
│  KEY DIFFERENCES:                                                   │
│  • No buffering: Quantum states decohere in storage               │
│  • No inspection: Reading quantum data destroys it                │
│  • No amplification: Cannot boost weak quantum signals            │
│  • Entanglement: Routing may require pre-shared entanglement      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Quantum Router Architecture

```python
import numpy as np
from dataclasses import dataclass, field
from typing import List, Dict, Optional, Tuple
from enum import Enum

class RouterType(Enum):
    """Types of quantum routers."""
    SWITCHING = "switching"      # Direct optical switching
    TELEPORTATION = "teleportation"  # Teleportation-based
    HYBRID = "hybrid"            # Combination approach

class PortType(Enum):
    """Port types on quantum router."""
    QUANTUM = "quantum"          # Quantum channel port
    CLASSICAL = "classical"      # Classical control port
    HYBRID = "hybrid"           # Both quantum and classical

@dataclass
class RouterPort:
    """Quantum router port."""
    port_id: int
    port_type: PortType
    connected_to: Optional[str] = None
    wavelength: float = 1550.0   # nm
    loss_db: float = 0.5

@dataclass
class QuantumRouter:
    """Basic quantum router model."""
    router_id: str
    router_type: RouterType
    num_ports: int
    ports: List[RouterPort] = field(default_factory=list)
    memory_qubits: int = 0
    coherence_time_us: float = 100.0

    def __post_init__(self):
        if not self.ports:
            self.ports = [
                RouterPort(i, PortType.QUANTUM)
                for i in range(self.num_ports)
            ]

class QuantumRouterSimulator:
    """
    Simulate quantum router operations.
    """

    def __init__(self):
        self.routers: Dict[str, QuantumRouter] = {}
        self.routing_table: Dict[str, Dict[str, int]] = {}

    def add_router(self, router: QuantumRouter):
        """Add router to network."""
        self.routers[router.router_id] = router
        self.routing_table[router.router_id] = {}

    def connect_routers(
        self,
        router1_id: str,
        port1: int,
        router2_id: str,
        port2: int
    ):
        """Connect two routers."""
        r1 = self.routers.get(router1_id)
        r2 = self.routers.get(router2_id)

        if r1 and r2:
            r1.ports[port1].connected_to = f"{router2_id}:{port2}"
            r2.ports[port2].connected_to = f"{router1_id}:{port1}"

    def set_route(
        self,
        router_id: str,
        destination: str,
        output_port: int
    ):
        """Set routing entry."""
        if router_id in self.routing_table:
            self.routing_table[router_id][destination] = output_port

    def route_qubit(
        self,
        source: str,
        destination: str
    ) -> List[str]:
        """Route a qubit through the network."""
        path = [source]
        current = source

        while current != destination:
            if current not in self.routing_table:
                return []  # No route

            routes = self.routing_table[current]
            if destination not in routes:
                return []  # No route to destination

            output_port = routes[destination]
            router = self.routers[current]
            next_hop = router.ports[output_port].connected_to

            if not next_hop:
                return []  # Dead end

            next_router = next_hop.split(":")[0]
            path.append(next_router)
            current = next_router

            if len(path) > 100:  # Loop detection
                return []

        return path

    def print_topology(self):
        """Print network topology."""
        print("Quantum Router Network")
        print("=" * 50)

        for rid, router in self.routers.items():
            print(f"\nRouter: {rid} ({router.router_type.value})")
            print(f"  Ports: {router.num_ports}")
            print(f"  Memory qubits: {router.memory_qubits}")
            for port in router.ports:
                if port.connected_to:
                    print(f"  Port {port.port_id} → {port.connected_to}")

# Create sample network
print("Quantum Router Fundamentals")
print("=" * 50)

sim = QuantumRouterSimulator()

# Add routers
sim.add_router(QuantumRouter("R1", RouterType.SWITCHING, 4))
sim.add_router(QuantumRouter("R2", RouterType.SWITCHING, 4))
sim.add_router(QuantumRouter("R3", RouterType.TELEPORTATION, 4, memory_qubits=10))

# Connect routers
sim.connect_routers("R1", 0, "R2", 0)
sim.connect_routers("R2", 1, "R3", 0)

# Set routes
sim.set_route("R1", "R3", 0)
sim.set_route("R2", "R3", 1)

sim.print_topology()

# Route a qubit
path = sim.route_qubit("R1", "R3")
print(f"\nPath R1 → R3: {' → '.join(path)}")
```

```
│  2. STORE-AND-FORWARD MODE                                         │
│  ┌──────────────────────────────────────────────────────────────┐  │
│  │                                                              │  │
│  │  Input ──→ [Memory] ──→ Wait ──→ [Memory] ──→ Output        │  │
│  │                                                              │  │
│  │  • Enables buffering for congestion                         │  │
│  │  • Requires quantum memory                                   │  │
│  │  • Limited by coherence time                                │  │
│  │                                                              │  │
│  └──────────────────────────────────────────────────────────────┘  │
│                                                                     │
│  3. TELEPORTATION MODE                                             │
│  ┌──────────────────────────────────────────────────────────────┐  │
│  │                                                              │  │
│  │  Input + [Entangled Pair] ──→ Bell Meas ──→ Classical       │  │
│  │                                    ↓           ↓             │  │
│  │                              Corrections ──→ Output          │  │
│  │                                                              │  │
│  │  • Consumes entanglement                                    │  │
│  │  • Enables long-distance routing                            │  │
```

```python
class RouterOperationMode(Enum):
    """Router operation modes."""
    OPTICAL_SWITCH = "optical_switch"
    STORE_FORWARD = "store_forward"
    TELEPORTATION = "teleportation"

class AdvancedQuantumRouter:
    """
    Advanced quantum router with multiple operation modes.
    """

    def __init__(
        self,
        router_id: str,
        num_ports: int,
        memory_qubits: int = 0
    ):
        self.router_id = router_id
        self.num_ports = num_ports
        self.memory_qubits = memory_qubits
        self.memory_used = 0
        self.entanglement_pairs: Dict[Tuple[int, int], float] = {}
        self.switch_matrix = np.zeros((num_ports, num_ports))

    def configure_switch(self, input_port: int, output_port: int):
        """Configure optical switch matrix."""
        # Clear previous connections for this input
        self.switch_matrix[input_port, :] = 0
        self.switch_matrix[input_port, output_port] = 1

    def optical_switch(
        self,
        input_port: int,
        qubit_state: np.ndarray
    ) -> Tuple[int, np.ndarray, float]:
        """Route qubit through optical switch."""
        # Find output port
        output_port = np.argmax(self.switch_matrix[input_port])

        # Apply switch loss (simplified)
        switch_loss = 0.95  # 5% loss
        output_state = qubit_state * np.sqrt(switch_loss)

        return output_port, output_state, switch_loss

    def store_qubit(self, qubit_state: np.ndarray) -> Optional[int]:
        """Store qubit in quantum memory."""
        if self.memory_used >= self.memory_qubits:
            return None  # Memory full

        memory_slot = self.memory_used
        self.memory_used += 1
        # In reality, would store in actual quantum memory
        return memory_slot

    def retrieve_qubit(self, memory_slot: int) -> Optional[np.ndarray]:
        """Retrieve qubit from memory."""
        if memory_slot >= self.memory_used:
            return None

        # Simplified: return ideal state
        # Real implementation would have decoherence
        return np.array([1, 0], dtype=complex)

    def teleport_route(
        self,
        qubit_state: np.ndarray,
        output_port: int
    ) -> Dict:
        """Route via teleportation."""
        # Check for entanglement with output
        if output_port not in [p for (_, p), _ in self.entanglement_pairs.items()]:
            return {"success": False, "reason": "No entanglement available"}

        # Perform Bell measurement (simplified)
        bell_result = np.random.randint(0, 4)

        # Teleportation succeeds
        return {
            "success": True,
            "bell_result": bell_result,
            "output_port": output_port,
            "fidelity": 0.95  # Simplified
        }

    def route_decision(
        self,
        input_port: int,
        destination: str,
        routing_table: Dict[str, int]
    ) -> RouterOperationMode:
        """Decide which routing mode to use."""
        output_port = routing_table.get(destination)

        if output_port is None:
            return None

        # Direct optical switch if available
        if self.switch_matrix[input_port, output_port] == 1:
            return RouterOperationMode.OPTICAL_SWITCH

        # Check for entanglement
        if (input_port, output_port) in self.entanglement_pairs:
            return RouterOperationMode.TELEPORTATION

        # Fall back to store-and-forward if memory available
        if self.memory_used < self.memory_qubits:
            return RouterOperationMode.STORE_FORWARD

        return None

# Demonstrate router modes
```

## Summary

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ROUTER FUNDAMENTALS SUMMARY                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Key Differences from Classical:                                    │
│  • No cloning: Cannot copy qubits for routing decisions            │
│  • No buffering: Limited by decoherence time                       │
│  • No inspection: Cannot read quantum data                         │
│  • Entanglement: May consume entanglement for routing              │
│                                                                     │
│  Operation Modes:                                                   │
│  • Optical switching: Fastest, direct physical routing             │
│  • Store-and-forward: Uses quantum memory, limited time            │
│  • Teleportation: Consumes entanglement, enables long distance     │
│                                                                     │
│  Router Components:                                                 │
│  • Quantum plane: Physical quantum operations                      │
│  • Control plane: Routing decisions and resource management        │
│  • Classical plane: Protocol handling and network state            │
│                                                                     │
│  Challenges:                                                        │
│  • Maintaining coherence through routing                           │
│  • Managing limited quantum resources                              │
│  • Synchronizing with classical control                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

Quantum routers enable scalable quantum networks by intelligently directing quantum states while respecting quantum mechanical constraints.
