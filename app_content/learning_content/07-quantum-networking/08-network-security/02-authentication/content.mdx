# Authentication in Quantum Networks

## Introduction

While QKD provides information-theoretic security for key distribution, it requires an authenticated classical channel. Without authentication, a man-in-the-middle attack becomes trivial. This lesson explores authentication in quantum networks.

## Classical Authentication

```python
import numpy as np
import hashlib
from dataclasses import dataclass
from typing import List, Dict, Tuple

@dataclass
class AuthenticationScheme:
    """Authentication scheme specification."""
    name: str
    security_type: str
    key_consumption: str
    advantages: List[str]
    disadvantages: List[str]

class ClassicalAuthentication:
    """
    Classical authentication for QKD classical channel.
    """

    def __init__(self):
        self.schemes = self._define_schemes()

    def _define_schemes(self) -> Dict[str, AuthenticationScheme]:
        """Define authentication schemes."""
        return {
            "wegman_carter": AuthenticationScheme(
                name="Wegman-Carter (Universal Hashing)",
                security_type="Information-theoretic",
                key_consumption="O(log m) bits for m-bit message",
                advantages=[
                    "Information-theoretically secure",
                    "Efficient key consumption",
                    "Composable with QKD"
                ],
                disadvantages=[
                    "Requires initial shared key",
                    "Key consumed per authentication"
                ]
            ),
            "hmac": AuthenticationScheme(
                name="HMAC (Hash-based MAC)",
                security_type="Computational",
                key_consumption="Fixed key size",
                advantages=[
                    "Standard, widely implemented",
                    "Key can be reused",
                    "Fast computation"
                ],
                disadvantages=[
                    "Not information-theoretically secure",
                    "Vulnerable to quantum attacks (eventually)"
                ]
            ),
            "polynomial": AuthenticationScheme(
                name="Polynomial Authentication",
                security_type="Information-theoretic",
                key_consumption="Efficient (polynomial evaluation)",
                advantages=[
                    "Information-theoretically secure",
                    "Simple mathematics",
                    "Efficient"
                ],
                disadvantages=[
                    "Requires fresh key per message"
                ]
            )
        }

    def wegman_carter_auth(
        self,
        message: bytes,
        key: bytes,
        tag_length: int = 32
    ) -> bytes:
        """
        Simplified Wegman-Carter authentication.
        Uses hash as universal hash function (simplified).
        """
        combined = key + message
        tag = hashlib.sha256(combined).digest()[:tag_length]
        return tag

    def verify_auth(
        self,
        message: bytes,
        key: bytes,
        tag: bytes
    ) -> bool:
        """Verify authentication tag."""
        expected_tag = self.wegman_carter_auth(message, key, len(tag))
        return tag == expected_tag

    def print_schemes(self):
        """Print authentication schemes."""
        print("Classical Authentication Schemes for QKD")
        print("=" * 60)

        for scheme_id, scheme in self.schemes.items():
            print(f"\n{scheme.name}")
            print("-" * 50)
            print(f"  Security: {scheme.security_type}")
            print(f"  Key usage: {scheme.key_consumption}")
            print(f"  Advantages:")
            for a in scheme.advantages[:2]:
                print(f"    + {a}")
            print(f"  Disadvantages:")
            for d in scheme.disadvantages[:1]:
                print(f"    - {d}")

# Display schemes
auth = ClassicalAuthentication()
auth.print_schemes()
```

## Key Bootstrapping Problem

```
┌─────────────────────────────────────────────────────────────────────┐
│              KEY BOOTSTRAPPING PROBLEM                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Chicken-and-Egg Problem:                                          │
│  • QKD needs authenticated channel to work securely                │
│  • Information-theoretic authentication needs shared key           │
│  • Where does the first key come from?                             │
│                                                                     │
│  SOLUTIONS:                                                         │
│                                                                     │
│  1. PRE-SHARED KEY (Physical Distribution)                         │
│     ┌──────────────────────────────────────────────────────┐       │
│     │  Initial key distributed physically (courier, etc)   │       │
│     │  Used to bootstrap first QKD session                 │       │
│     │  New keys from QKD replace/extend initial key        │       │
│     └──────────────────────────────────────────────────────┘       │
│                                                                     │
│  2. HYBRID APPROACH (Computational then IT)                        │
│     ┌──────────────────────────────────────────────────────┐       │
│     │  Use computational authentication initially          │       │
│     │  QKD generates IT-secure keys                        │       │
│     │  Switch to IT authentication with QKD keys           │       │
│     └──────────────────────────────────────────────────────┘       │
│                                                                     │
│  3. TRUSTED THIRD PARTY                                            │
│     ┌──────────────────────────────────────────────────────┐       │
│     │  Key server distributes initial keys to all parties  │       │
│     │  Server is trusted during bootstrap phase only       │       │
│     └──────────────────────────────────────────────────────┘       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Authentication Key Management

```python
class AuthKeyManager:
    """
    Manage authentication keys for QKD.
    """

    def __init__(self, initial_key: bytes = None):
        if initial_key is None:
            # Simulate pre-shared key
            initial_key = bytes(range(32))
        self.key_pool = initial_key
        self.key_used = 0

    def consume_key(self, bits_needed: int) -> bytes:
        """Consume key material for authentication."""
        bytes_needed = (bits_needed + 7) // 8

        if self.key_used + bytes_needed > len(self.key_pool):
            raise ValueError("Insufficient key material")

        key = self.key_pool[self.key_used:self.key_used + bytes_needed]
        self.key_used += bytes_needed
        return key

    def replenish_key(self, new_key: bytes):
        """Add new key material from QKD."""
        remaining = self.key_pool[self.key_used:]
        self.key_pool = remaining + new_key
        self.key_used = 0

    def key_remaining(self) -> int:
        """Return remaining key bits."""
        return (len(self.key_pool) - self.key_used) * 8

    def messages_possible(self, avg_message_bits: int = 1000) -> int:
        """Estimate messages that can be authenticated."""
        # Wegman-Carter: ~log(m) key bits per m-bit message
        key_per_msg = int(np.log2(avg_message_bits)) + 32  # Plus security parameter
        return self.key_remaining() // key_per_msg

# Demonstrate key management
print("\n\nAuthentication Key Management")
print("=" * 50)

manager = AuthKeyManager(bytes(256))  # 2048 bits initial key

print(f"Initial key: {manager.key_remaining()} bits")
print(f"Messages possible: ~{manager.messages_possible()}")

# Simulate QKD session
for session in range(3):
    auth_key = manager.consume_key(64)
    print(f"\nSession {session+1}:")
    print(f"  Key consumed: 64 bits for authentication")
    print(f"  Remaining: {manager.key_remaining()} bits")

    # QKD generates new key
    manager.replenish_key(bytes(128))  # 1024 bits from QKD
    print(f"  After QKD: {manager.key_remaining()} bits")
```

```
│  • Hybrid: computational→IT transition                             │

│                                                                     │
│  Key Management:                                                    │
│  • Initial key consumed for authentication                         │

│  • Must maintain positive key balance                              │

```

Authentication is the critical link that makes QKD practical—without it, even perfect quantum security is useless.
