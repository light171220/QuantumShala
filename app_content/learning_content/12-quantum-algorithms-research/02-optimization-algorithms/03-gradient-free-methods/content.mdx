## 1. Why Gradient-Free?

Gradient-free methods are valuable when:
- Function evaluations are noisy (finite shots)
- Gradients are expensive to compute
- The landscape has discontinuities

## 2. COBYLA

Constrained Optimization BY Linear Approximation builds local linear models:

```python
from scipy.optimize import minimize
import numpy as np

def cobyla_optimize(fn, x0, maxiter=500):
    """Run COBYLA optimization."""
    result = minimize(fn, x0, method='COBYLA',
                     options={'maxiter': maxiter, 'rhobeg': 0.5})
    return result.x, result.fun, result.nfev
```

## 3. Nelder-Mead Simplex

```python
def nelder_mead_optimize(fn, x0, maxiter=500):
    """Run Nelder-Mead simplex optimization."""
    result = minimize(fn, x0, method='Nelder-Mead',
                     options={'maxiter': maxiter})
    return result.x, result.fun, result.nfev
```

## 4. SPSA for Noisy Optimization

```python
class SPSA:
    def __init__(self, a=0.1, c=0.1, A=10):
        self.a, self.c, self.A = a, c, A
        self.k = 0

    def step(self, fn, x):
        self.k += 1
        ak = self.a / (self.k + self.A)**0.602
        ck = self.c / self.k**0.101

        delta = 2 * np.random.randint(0, 2, len(x)) - 1
        grad_est = (fn(x + ck*delta) - fn(x - ck*delta)) / (2*ck*delta)

        return x - ak * grad_est
```

## Summary

Gradient-free methods trade gradient information for robustness:
1. **COBYLA**: Linear approximations, handles constraints
2. **Nelder-Mead**: Simplex-based, no derivatives needed
3. **SPSA**: Efficient for high dimensions with noise
