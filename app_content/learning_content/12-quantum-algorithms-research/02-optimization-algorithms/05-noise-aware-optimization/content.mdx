## 1. Sources of Noise in VQE

1. **Shot noise**: Finite measurement samples
2. **Gate errors**: Imperfect quantum operations
3. **Decoherence**: Loss of quantum information

```python
import numpy as np

def estimate_shot_noise(variance, num_shots):
    """Estimate standard error from finite shots."""
    return np.sqrt(variance / num_shots)

def noisy_energy_evaluation(true_energy, variance, num_shots):
    """Simulate noisy energy measurement."""
    noise = np.random.normal(0, np.sqrt(variance / num_shots))
    return true_energy + noise
```

## 2. Robust Gradient Estimation

```python
def averaged_gradient(grad_fn, params, num_averages=5):
    """Average gradients to reduce noise."""
    gradients = [grad_fn(params) for _ in range(num_averages)]
    return np.mean(gradients, axis=0)

def adaptive_shots(variance_history, base_shots=100, max_shots=10000):
    """Adapt shot count based on observed variance."""
    if len(variance_history) < 2:
        return base_shots
    recent_var = np.mean(variance_history[-5:])
    return min(max_shots, int(base_shots * (1 + 10 * recent_var)))
```

## 3. Noise-Resilient Optimizers

```python
class RobustAdam:
    """Adam with noise-aware learning rate adaptation."""
    def __init__(self, lr=0.01, noise_scale=0.1):
        self.lr = lr
        self.noise_scale = noise_scale
        self.m = self.v = None
        self.t = 0

    def step(self, params, grad, estimated_noise=0):
        if self.m is None:
            self.m = np.zeros_like(params)
            self.v = np.zeros_like(params)

        self.t += 1
        self.m = 0.9 * self.m + 0.1 * grad
        self.v = 0.999 * self.v + 0.001 * grad**2

        # Reduce learning rate when noise is high
        effective_lr = self.lr / (1 + self.noise_scale * estimated_noise)

        m_hat = self.m / (1 - 0.9**self.t)
        v_hat = self.v / (1 - 0.999**self.t)

        return params - effective_lr * m_hat / (np.sqrt(v_hat) + 1e-8)
```

## Summary

Noise-aware optimization adapts to quantum hardware limitations:
1. Account for shot noise in gradient estimates
2. Adapt shot allocation based on variance
3. Use noise-resilient optimizer configurations
