## 1. Classical Shortest Path Algorithms

The shortest path problem is fundamental in graph theory and has numerous applications in routing, navigation, and network optimization.

<Definition term="Single-Source Shortest Path">
Given a weighted graph $G = (V, E, w)$ and a source vertex $s$, find the shortest path from $s$ to all other vertices. The distance $d(s, v)$ is the minimum sum of edge weights along any path from $s$ to $v$.
</Definition>

<Formula>
$$d(s, v) = \min_{\text{path } P: s \to v} \sum_{e \in P} w(e)$$
</Formula>

```python
import numpy as np
import heapq
from typing import Dict, List, Tuple, Optional
from collections import defaultdict

def dijkstra(adj: Dict[int, List[Tuple[int, float]]], source: int) -> Dict[int, float]:
    """
    Dijkstra's algorithm for single-source shortest paths.

    Args:
        adj: Adjacency list {node: [(neighbor, weight), ...]}
        source: Source vertex

    Returns:
        Dictionary of shortest distances from source
    """
    distances = defaultdict(lambda: float('inf'))
    distances[source] = 0
    visited = set()
    heap = [(0, source)]

    while heap:
        d, u = heapq.heappop(heap)

        if u in visited:
            continue
        visited.add(u)

        for v, weight in adj.get(u, []):
            if v not in visited:
                new_dist = d + weight
                if new_dist < distances[v]:
                    distances[v] = new_dist
                    heapq.heappush(heap, (new_dist, v))

    return dict(distances)

def bellman_ford(n: int, edges: List[Tuple[int, int, float]], source: int) -> Optional[Dict[int, float]]:
    """
    Bellman-Ford algorithm (handles negative weights).

    Returns:
        Distances dict, or None if negative cycle exists
    """
    distances = {i: float('inf') for i in range(n)}
    distances[source] = 0

    for _ in range(n - 1):
        for u, v, w in edges:
            if distances[u] + w < distances[v]:
                distances[v] = distances[u] + w

    # Check for negative cycles
    for u, v, w in edges:
        if distances[u] + w < distances[v]:
            return None

    return distances

# Example graph
adj = {
    0: [(1, 4), (2, 1)],
    1: [(3, 1)],
    2: [(1, 2), (3, 5)],
    3: []
}

distances = dijkstra(adj, 0)
print("Shortest distances from vertex 0:")
for v, d in sorted(distances.items()):
    print(f"  To {v}: {d}")
```

## 2. Quantum Speedup Potential

Quantum algorithms can provide speedups for shortest path problems in certain scenarios.

<Tip>
The quantum speedup for shortest path depends heavily on graph structure. For sparse graphs with $m$ edges and $n$ vertices, quantum approaches can achieve $O(\sqrt{nm})$ compared to classical $O(m + n \log n)$ for Dijkstra.
</Tip>

| Problem | Classical | Quantum | Speedup |
|---------|-----------|---------|---------|
| SSSP (dense) | $O(n^2)$ | $O(n^{1.5})$ | $\sqrt{n}$ |
| SSSP (sparse) | $O(m + n\log n)$ | $O(\sqrt{nm}\log n)$ | Limited |
| APSP | $O(n^3)$ | $O(n^{2.5})$ | $\sqrt{n}$ |
| Negative edge detection | $O(nm)$ | $O(n\sqrt{m})$ | $\sqrt{m}$ |

```python
def complexity_comparison(n: int, m: int) -> Dict:
    """
    Compare classical and quantum complexities for shortest path.
    """
    return {
        'n_vertices': n,
        'm_edges': m,
        'dijkstra': m + n * np.log2(n),
        'bellman_ford': n * m,
        'quantum_sssp': np.sqrt(n * m) * np.log2(n),
        'classical_apsp': n**3,
        'quantum_apsp': n**2.5
    }

# Compare for different graph sizes
print("Complexity Comparison:")
print("=" * 60)
for n in [100, 1000, 10000]:
    m = n * 10  # Sparse graph
    comp = complexity_comparison(n, m)
    print(f"\nn={n}, m={m}:")
    print(f"  Dijkstra: {comp['dijkstra']:.2e}")
    print(f"  Quantum SSSP: {comp['quantum_sssp']:.2e}")
    print(f"  Speedup factor: {comp['dijkstra']/comp['quantum_sssp']:.2f}x")
```

## 3. Grover Search for Path Finding

Grover's algorithm can search for paths with specific properties.

<Definition term="Path Search Problem">
Given a graph $G$, source $s$, target $t$, and constraint $C$, find a path from $s$ to $t$ satisfying $C$. Grover search achieves $O(\sqrt{N})$ where $N$ is the number of candidate paths.
</Definition>

```python
from qiskit import QuantumCircuit, QuantumRegister
from qiskit.quantum_info import Statevector
import numpy as np

def encode_path_oracle(n_qubits: int, valid_paths: List[int]) -> QuantumCircuit:
    """
    Create oracle marking valid paths.

    Args:
        n_qubits: Number of qubits (2^n possible paths)
        valid_paths: List of valid path indices
    """
    qc = QuantumCircuit(n_qubits)

    for path_idx in valid_paths:
        # Convert to binary and apply controlled-Z
        binary = format(path_idx, f'0{n_qubits}b')

        # Apply X gates for 0 bits
        for i, bit in enumerate(reversed(binary)):
            if bit == '0':
                qc.x(i)

        # Multi-controlled Z
        if n_qubits == 1:
            qc.z(0)
        elif n_qubits == 2:
            qc.cz(0, 1)
        else:
            # MCZ implementation
            qc.h(n_qubits - 1)
            qc.mcx(list(range(n_qubits - 1)), n_qubits - 1)
            qc.h(n_qubits - 1)

        # Undo X gates
        for i, bit in enumerate(reversed(binary)):
            if bit == '0':
                qc.x(i)

    return qc

def grover_diffusion(n_qubits: int) -> QuantumCircuit:
    """Grover diffusion operator."""
    qc = QuantumCircuit(n_qubits)

    qc.h(range(n_qubits))
    qc.x(range(n_qubits))

    # Multi-controlled Z
    qc.h(n_qubits - 1)
    qc.mcx(list(range(n_qubits - 1)), n_qubits - 1)
    qc.h(n_qubits - 1)

    qc.x(range(n_qubits))
    qc.h(range(n_qubits))

    return qc

def grover_path_search(n_qubits: int, valid_paths: List[int],
                        n_iterations: int = None) -> np.ndarray:
    """
    Run Grover search for valid paths.

    Returns:
        Probability distribution over paths
    """
    N = 2**n_qubits
    M = len(valid_paths)

    if n_iterations is None:
        n_iterations = int(np.pi/4 * np.sqrt(N/M))

    qc = QuantumCircuit(n_qubits)
    qc.h(range(n_qubits))

    oracle = encode_path_oracle(n_qubits, valid_paths)
    diffusion = grover_diffusion(n_qubits)

    for _ in range(n_iterations):
        qc = qc.compose(oracle)
        qc = qc.compose(diffusion)

    sv = Statevector.from_instruction(qc)
    return np.abs(sv.data)**2

# Example: Search for shortest paths
n_qubits = 4  # 16 possible path encodings
valid_paths = [3, 7, 11]  # Paths with distance <= threshold

probs = grover_path_search(n_qubits, valid_paths)

print("Grover Path Search Results:")
print("Valid paths:", valid_paths)
print("Probabilities:")
for i, p in enumerate(probs):
    if p > 0.01:
        print(f"  Path {i}: {p:.4f}")
```

## 4. Quantum Walk-Based Shortest Path

Continuous-time quantum walks can detect distances through interference patterns.

<Formula>
$$|\psi(t)\rangle = e^{-iAt}|s\rangle$$
$$P_t(v) = |\langle v|e^{-iAt}|s\rangle|^2$$
</Formula>

The time at which probability peaks at vertex $v$ relates to the graph distance.

```python
from scipy.linalg import expm

def quantum_walk_distances(adj_matrix: np.ndarray, source: int,
                            max_time: float = 10.0,
                            time_steps: int = 100) -> Dict[int, float]:
    """
    Estimate distances using quantum walk dynamics.

    Returns:
        Dictionary of estimated distances based on first arrival times
    """
    n = adj_matrix.shape[0]

    # Initial state: localized at source
    psi_0 = np.zeros(n, dtype=complex)
    psi_0[source] = 1.0

    times = np.linspace(0.01, max_time, time_steps)
    first_arrival = {v: float('inf') for v in range(n)}
    first_arrival[source] = 0

    threshold = 0.1  # Probability threshold for "arrival"

    for t in times:
        U = expm(-1j * adj_matrix * t)
        psi_t = U @ psi_0
        probs = np.abs(psi_t)**2

        for v in range(n):
            if first_arrival[v] == float('inf') and probs[v] > threshold:
                first_arrival[v] = t

    return first_arrival

def compare_qw_classical_distances(adj_matrix: np.ndarray, source: int):
    """
    Compare quantum walk estimates with classical shortest paths.
    """
    n = adj_matrix.shape[0]

    # Convert to adjacency list
    adj_list = defaultdict(list)
    for i in range(n):
        for j in range(n):
            if adj_matrix[i, j] > 0:
                adj_list[i].append((j, 1))  # Unweighted

    classical = dijkstra(adj_list, source)
    quantum = quantum_walk_distances(adj_matrix, source)

    return {
        'classical': classical,
        'quantum_arrival': quantum
    }

# Example: Path graph
n = 6
adj_matrix = np.zeros((n, n))
for i in range(n - 1):
    adj_matrix[i, i + 1] = 1
    adj_matrix[i + 1, i] = 1

comparison = compare_qw_classical_distances(adj_matrix, 0)

print("\nDistance Comparison (Path Graph):")
print(f"{'Vertex':>8} {'Classical':>12} {'QW Arrival':>12}")
print("-" * 35)
for v in range(n):
    c_dist = comparison['classical'].get(v, float('inf'))
    q_time = comparison['quantum_arrival'].get(v, float('inf'))
    print(f"{v:8d} {c_dist:12.1f} {q_time:12.2f}")
```

## 5. Quantum Algorithm for All-Pairs Shortest Path

Quantum algorithms can speed up all-pairs shortest path computation.

<Definition term="All-Pairs Shortest Path (APSP)">
Compute the shortest path distance $d(u, v)$ for all pairs of vertices $u, v \in V$. Classical algorithms require $O(n^3)$ (Floyd-Warshall) or $O(n \cdot T_{SSSP})$.
</Definition>

```python
def floyd_warshall(weight_matrix: np.ndarray) -> np.ndarray:
    """
    Classical Floyd-Warshall for APSP.
    """
    n = weight_matrix.shape[0]
    dist = weight_matrix.copy()

    # Set diagonal to 0
    np.fill_diagonal(dist, 0)

    # Replace 0 (no edge) with infinity
    dist[dist == 0] = float('inf')
    np.fill_diagonal(dist, 0)

    for k in range(n):
        for i in range(n):
            for j in range(n):
                if dist[i, k] + dist[k, j] < dist[i, j]:
                    dist[i, j] = dist[i, k] + dist[k, j]

    return dist

def quantum_inspired_apsp(adj_matrix: np.ndarray) -> np.ndarray:
    """
    Quantum-inspired APSP using spectral methods.

    Uses eigendecomposition for distance estimation.
    """
    n = adj_matrix.shape[0]

    # Compute Laplacian
    D = np.diag(np.sum(adj_matrix, axis=1))
    L = D - adj_matrix

    # Eigendecomposition
    eigenvalues, eigenvectors = np.linalg.eigh(L)

    # Compute effective resistances (related to commute times)
    # R_eff(i,j) = sum_k (1/lambda_k)(phi_k(i) - phi_k(j))^2

    distances = np.zeros((n, n))

    for i in range(n):
        for j in range(i + 1, n):
            r_eff = 0
            for k in range(1, n):  # Skip zero eigenvalue
                if eigenvalues[k] > 1e-10:
                    diff = eigenvectors[i, k] - eigenvectors[j, k]
                    r_eff += diff**2 / eigenvalues[k]
            distances[i, j] = np.sqrt(r_eff)
            distances[j, i] = distances[i, j]

    return distances

# Example: Complete graph minus some edges
n = 5
adj = np.ones((n, n)) - np.eye(n)
adj[0, 4] = 0
adj[4, 0] = 0

fw_distances = floyd_warshall(adj.copy())
qi_distances = quantum_inspired_apsp(adj)

print("\nAPSP Comparison:")
print("Floyd-Warshall distances:")
print(fw_distances)
print("\nQuantum-inspired (spectral) distances:")
print(np.round(qi_distances, 2))
```

## 6. Hybrid Quantum-Classical Approaches

Practical shortest path algorithms often combine quantum and classical techniques.

```python
def hybrid_shortest_path(adj_matrix: np.ndarray, source: int,
                          target: int, method: str = 'grover_guided') -> Dict:
    """
    Hybrid quantum-classical shortest path finding.

    Args:
        adj_matrix: Graph adjacency matrix
        source: Source vertex
        target: Target vertex
        method: 'grover_guided', 'qw_informed', or 'classical'

    Returns:
        Path finding result
    """
    n = adj_matrix.shape[0]

    if method == 'classical':
        # Pure classical Dijkstra
        adj_list = defaultdict(list)
        for i in range(n):
            for j in range(n):
                if adj_matrix[i, j] > 0:
                    adj_list[i].append((j, adj_matrix[i, j]))

        distances = dijkstra(adj_list, source)
        return {
            'method': 'classical',
            'distance': distances.get(target, float('inf')),
            'complexity': 'O(m + n log n)'
        }

    elif method == 'qw_informed':
        # Use quantum walk to estimate promising directions
        qw_times = quantum_walk_distances(adj_matrix, source)

        # Prioritize vertices by quantum arrival time
        priority = sorted(range(n), key=lambda v: qw_times[v])

        # Classical search guided by quantum priorities
        distances = {source: 0}
        visited = set()

        for v in priority:
            if v in visited:
                continue
            visited.add(v)

            for u in range(n):
                if adj_matrix[v, u] > 0 and u not in visited:
                    new_dist = distances.get(v, float('inf')) + adj_matrix[v, u]
                    if new_dist < distances.get(u, float('inf')):
                        distances[u] = new_dist

        return {
            'method': 'qw_informed',
            'distance': distances.get(target, float('inf')),
            'complexity': 'O(sqrt(n) * n)'
        }

    elif method == 'grover_guided':
        # Use Grover to identify promising intermediate vertices
        # Then classical path reconstruction

        return {
            'method': 'grover_guided',
            'distance': 'Requires path oracle',
            'complexity': 'O(sqrt(paths))'
        }

    return {}

# Compare methods
n = 8
adj = np.zeros((n, n))
edges = [(0,1,1), (1,2,1), (2,3,1), (3,7,1),
         (0,4,2), (4,5,1), (5,6,1), (6,7,1)]
for i, j, w in edges:
    adj[i, j] = adj[j, i] = w

print("\nHybrid Shortest Path Comparison (0 -> 7):")
for method in ['classical', 'qw_informed']:
    result = hybrid_shortest_path(adj, 0, 7, method)
    print(f"  {result['method']}: distance = {result['distance']}")
```

## 7. Complexity Analysis and Limitations

<Warning>
Quantum speedups for shortest path are not universal. For sparse graphs where $m = O(n)$, classical Dijkstra achieves $O(n \log n)$, which can be competitive with quantum approaches that have large constant factors.
</Warning>

```python
def analyze_speedup_conditions(n: int, m: int) -> Dict:
    """
    Analyze when quantum speedup is beneficial.
    """
    classical_dijkstra = m + n * np.log2(n)
    classical_bellman_ford = n * m
    quantum_sssp = np.sqrt(n * m) * np.log2(n)

    # Account for quantum overhead (rough estimate)
    quantum_overhead = 1000  # Gate overhead factor
    quantum_practical = quantum_sssp * quantum_overhead

    return {
        'graph_density': m / (n * (n - 1) / 2),
        'dijkstra': classical_dijkstra,
        'quantum_theoretical': quantum_sssp,
        'quantum_practical': quantum_practical,
        'speedup_theoretical': classical_dijkstra / quantum_sssp,
        'speedup_practical': classical_dijkstra / quantum_practical,
        'quantum_beneficial': quantum_practical < classical_dijkstra
    }

print("\nSpeedup Analysis:")
print("=" * 70)
test_cases = [
    (100, 500),    # Sparse
    (100, 2000),   # Medium
    (100, 4500),   # Dense
    (1000, 5000),  # Large sparse
    (1000, 50000), # Large medium
]

for n, m in test_cases:
    analysis = analyze_speedup_conditions(n, m)
    beneficial = "Yes" if analysis['quantum_beneficial'] else "No"
    print(f"n={n:5d}, m={m:6d}, density={analysis['graph_density']:.2f}: "
          f"Speedup={analysis['speedup_theoretical']:.2f}x, "
          f"Practical={beneficial}")
```

## Summary

In this lesson, you learned:

1. **Classical algorithms**: Dijkstra ($O(m + n \log n)$) and Bellman-Ford ($O(nm)$) are the baselines
2. **Grover search**: Can find paths with specific properties in $O(\sqrt{N})$
3. **Quantum walks**: Continuous-time walks provide distance estimates through arrival times
4. **APSP speedup**: Quantum algorithms achieve $O(n^{2.5})$ vs classical $O(n^3)$
5. **Hybrid approaches**: Combining quantum and classical often outperforms pure approaches
6. **Practical considerations**: Quantum overhead can negate theoretical speedups for small graphs

Quantum shortest path algorithms are an active research area with potential for significant speedups on large, dense graphs.
