## 1. From Supremacy to Utility

<Definition term="Quantum Utility">
Practical quantum advantage where quantum computers provide value for real-world problems, not just artificial benchmarks designed to be hard for classical computers.
</Definition>

<KeyConcept title="Supremacy vs. Utility">
| Aspect | Quantum Supremacy | Quantum Utility |
|--------|------------------|-----------------|
| Goal | Prove classical hardness | Solve useful problems |
| Problems | Artificial benchmarks | Real applications |
| Verification | Statistical (XEB) | Domain-specific metrics |
| Timeline | Achieved (2019+) | Emerging (2024+) |
</KeyConcept>

```python
import numpy as np
from typing import Dict, List, Tuple, Optional
from dataclasses import dataclass
from enum import Enum

class AdvantageType(Enum):
    SUPREMACY = "computational_supremacy"
    UTILITY = "practical_utility"
    ECONOMIC = "economic_advantage"
    SCIENTIFIC = "scientific_discovery"

@dataclass
class QuantumAdvantageMetrics:
    """Metrics for evaluating quantum advantage claims."""
    problem_type: str
    quantum_time: float
    classical_time: float
    quantum_quality: float
    classical_quality: float
    resource_cost_ratio: float

    def speedup_factor(self) -> float:
        return self.classical_time / self.quantum_time

    def quality_ratio(self) -> float:
        return self.quantum_quality / self.classical_quality

    def has_advantage(self, threshold: float = 1.0) -> bool:
        """Check if quantum provides advantage considering quality and speed."""
        effective_speedup = self.speedup_factor() * self.quality_ratio()
        return effective_speedup > threshold

    def advantage_type(self) -> AdvantageType:
        if self.problem_type in ['rcs', 'boson_sampling']:
            return AdvantageType.SUPREMACY
        elif self.speedup_factor() > 100:
            return AdvantageType.ECONOMIC
        elif self.quality_ratio() > 1.5:
            return AdvantageType.SCIENTIFIC
        else:
            return AdvantageType.UTILITY
```

## 2. NISQ Era Limitations

<Definition term="NISQ (Noisy Intermediate-Scale Quantum)">
Current quantum devices with 50-1000+ qubits that lack full error correction, limiting circuit depth and algorithm complexity.
</Definition>

<Formula>
$$\text{Effective Circuit Depth} \approx \frac{T_2}{\tau_{\text{gate}}} \cdot \frac{1}{1 + n \cdot \epsilon}$$
</Formula>

where T_2 is coherence time, tau_gate is gate time, n is qubit count, and epsilon is error rate.

```python
@dataclass
class NISQDevice:
    """Model a NISQ quantum device."""
    n_qubits: int
    t1_us: float  # T1 relaxation time
    t2_us: float  # T2 dephasing time
    single_qubit_error: float
    two_qubit_error: float
    readout_error: float
    gate_time_ns: float

    def max_circuit_depth(self, target_fidelity: float = 0.1) -> int:
        """Estimate maximum useful circuit depth."""
        # Average error per layer (assuming n/2 two-qubit gates per layer)
        error_per_layer = (self.single_qubit_error * self.n_qubits +
                          self.two_qubit_error * self.n_qubits / 2)

        # Depth limited by accumulated error
        error_limited = int(np.log(target_fidelity) / np.log(1 - error_per_layer))

        # Depth limited by coherence
        layer_time_us = self.gate_time_ns * 1e-3 * 10  # ~10 gates per layer
        coherence_limited = int(self.t2_us / layer_time_us)

        return min(error_limited, coherence_limited)

    def expected_fidelity(self, depth: int, n_two_qubit: int) -> float:
        """Estimate circuit fidelity."""
        n_single = depth * self.n_qubits
        gate_fidelity = ((1 - self.single_qubit_error) ** n_single *
                         (1 - self.two_qubit_error) ** n_two_qubit)
        readout_fidelity = (1 - self.readout_error) ** self.n_qubits

        # Coherence decay
        total_time = depth * self.gate_time_ns * 1e-3 * 10
        coherence_fidelity = np.exp(-total_time / self.t2_us)

        return gate_fidelity * readout_fidelity * coherence_fidelity


class NISQAlgorithmDesigner:
    """Design algorithms for NISQ constraints."""

    def __init__(self, device: NISQDevice):
        self.device = device

    def optimize_ansatz_depth(self, target_expressibility: float,
                               min_fidelity: float = 0.1) -> dict:
        """Find optimal ansatz depth balancing expressibility and noise."""
        max_depth = self.device.max_circuit_depth(min_fidelity)

        results = []
        for depth in range(1, max_depth + 1):
            n_params = depth * self.device.n_qubits * 3  # Typical parameterization
            n_two_qubit = depth * (self.device.n_qubits - 1)

            expressibility = 1 - np.exp(-n_params / 50)  # Simplified model
            fidelity = self.device.expected_fidelity(depth, n_two_qubit)

            effective_quality = expressibility * fidelity

            results.append({
                'depth': depth,
                'expressibility': expressibility,
                'fidelity': fidelity,
                'effective_quality': effective_quality,
                'n_parameters': n_params
            })

        # Find optimal depth
        best = max(results, key=lambda x: x['effective_quality'])

        return {
            'optimal_depth': best['depth'],
            'analysis': results,
            'recommendation': best
        }
```

## 3. Error Mitigation Techniques

<Insight>
Error mitigation bridges the gap between noisy NISQ devices and fault-tolerant quantum computers by reducing errors without full error correction overhead.
</Insight>

```python
class ErrorMitigation:
    """Collection of error mitigation techniques."""

    @staticmethod
    def zero_noise_extrapolation(noise_scaled_results: Dict[float, float]) -> float:
        """
        Zero-Noise Extrapolation (ZNE): Run at multiple noise levels,
        extrapolate to zero noise.
        """
        noise_factors = np.array(list(noise_scaled_results.keys()))
        expectation_values = np.array(list(noise_scaled_results.values()))

        # Richardson extrapolation (polynomial fit)
        coeffs = np.polyfit(noise_factors, expectation_values, deg=len(noise_factors)-1)
        zero_noise_estimate = np.polyval(coeffs, 0)

        return zero_noise_estimate

    @staticmethod
    def probabilistic_error_cancellation(
        noisy_result: float,
        noise_model: Dict[str, float],
        n_samples: int = 1000
    ) -> Tuple[float, float]:
        """
        Probabilistic Error Cancellation (PEC): Decompose noise channel
        and cancel errors statistically.
        """
        # Simplified model: assume depolarizing noise
        p_error = noise_model.get('depolarizing', 0.01)

        # PEC overhead scales exponentially with circuit size
        gamma = 1 / (1 - p_error)  # Sampling overhead factor

        # Corrected estimate
        corrected = noisy_result * gamma
        variance = (gamma ** 2 - 1) / n_samples

        return corrected, np.sqrt(variance)

    @staticmethod
    def readout_error_mitigation(
        raw_counts: Dict[str, int],
        calibration_matrix: np.ndarray
    ) -> Dict[str, float]:
        """
        Readout Error Mitigation: Invert the calibration matrix
        to correct measurement errors.
        """
        n_qubits = int(np.log2(len(calibration_matrix)))
        n_states = 2 ** n_qubits

        # Build probability vector from counts
        total_shots = sum(raw_counts.values())
        raw_probs = np.zeros(n_states)
        for bitstring, count in raw_counts.items():
            idx = int(bitstring, 2)
            raw_probs[idx] = count / total_shots

        # Invert calibration matrix (use pseudo-inverse for stability)
        cal_inv = np.linalg.pinv(calibration_matrix)
        corrected_probs = cal_inv @ raw_probs

        # Clip to valid probabilities
        corrected_probs = np.clip(corrected_probs, 0, 1)
        corrected_probs /= corrected_probs.sum()

        return {format(i, f'0{n_qubits}b'): p
                for i, p in enumerate(corrected_probs) if p > 1e-10}

    @staticmethod
    def symmetry_verification(
        results: List[float],
        symmetry_check: callable
    ) -> List[float]:
        """
        Symmetry Verification: Post-select results that satisfy
        known symmetries of the problem.
        """
        verified = [r for r in results if symmetry_check(r)]
        return verified if verified else results  # Fallback to all results
```

## 4. Path to Fault Tolerance

<Definition term="Fault-Tolerant Quantum Computing">
Quantum computation with error rates suppressed to arbitrary precision through quantum error correction codes, enabling arbitrarily long computations.
</Definition>

<Formula>
$$\text{Logical Error Rate} \approx p_{\text{phys}}^{(d+1)/2} \cdot A^d$$
</Formula>

where p_phys is physical error rate, d is code distance, and A is a code-dependent constant.

```python
@dataclass
class FaultTolerantRequirements:
    """Estimate resources for fault-tolerant quantum computing."""

    target_logical_error: float
    physical_error_rate: float
    code_type: str = 'surface'

    def required_code_distance(self) -> int:
        """Calculate minimum code distance needed."""
        # Surface code threshold ~ 1%
        threshold = 0.01
        if self.physical_error_rate >= threshold:
            return -1  # Below threshold, FT not possible

        # d such that p^((d+1)/2) < target
        # (d+1)/2 > log(target) / log(p)
        d = 1
        while True:
            logical_error = self.physical_error_rate ** ((d + 1) / 2)
            if logical_error < self.target_logical_error:
                return d
            d += 2  # Surface code uses odd distances
            if d > 100:
                return d

    def physical_qubits_per_logical(self) -> int:
        """Estimate physical qubits needed per logical qubit."""
        d = self.required_code_distance()
        if d < 0:
            return -1

        if self.code_type == 'surface':
            return 2 * d * d  # Surface code: 2d^2 physical per logical
        elif self.code_type == 'color':
            return d * d  # Color code (approximate)
        else:
            return d * d

    def estimate_resources(self, n_logical_qubits: int,
                           circuit_depth: int) -> dict:
        """Estimate total resources for fault-tolerant computation."""
        d = self.required_code_distance()
        physical_per_logical = self.physical_qubits_per_logical()

        # Magic state distillation overhead
        t_gates_estimate = circuit_depth * n_logical_qubits * 0.1
        magic_state_qubits = int(t_gates_estimate * 10)  # Rough estimate

        total_physical = (n_logical_qubits * physical_per_logical +
                          magic_state_qubits)

        # Time overhead from error correction cycles
        time_overhead = d  # Syndrome extraction cycles

        return {
            'code_distance': d,
            'physical_per_logical': physical_per_logical,
            'total_physical_qubits': total_physical,
            'magic_state_factories': int(t_gates_estimate / 100) + 1,
            'time_overhead_factor': time_overhead,
            'estimated_physical_error_budget': self.physical_error_rate
        }


class QuantumResourceEstimator:
    """Estimate resources for quantum algorithms."""

    def __init__(self, algorithm: str, problem_size: int):
        self.algorithm = algorithm
        self.n = problem_size

    def logical_requirements(self) -> dict:
        """Estimate logical qubit and gate requirements."""
        estimates = {
            'shor_factoring': {
                'qubits': 2 * self.n + 3,
                'toffoli': self.n ** 3,
                'depth': self.n ** 2
            },
            'grover_search': {
                'qubits': self.n,
                'toffoli': self.n * int(np.sqrt(2 ** self.n)),
                'depth': int(np.sqrt(2 ** self.n))
            },
            'qpe_chemistry': {
                'qubits': self.n + 10,  # n orbitals + ancilla
                'toffoli': self.n ** 4,  # Rough estimate
                'depth': self.n ** 3
            },
            'qaoa_maxcut': {
                'qubits': self.n,
                'toffoli': 0,  # No Toffoli needed
                'depth': 100  # Typical p ~ 10-100 layers
            }
        }
        return estimates.get(self.algorithm, {'error': 'Unknown algorithm'})

    def nisq_feasibility(self, device: NISQDevice) -> dict:
        """Check if algorithm is feasible on NISQ device."""
        req = self.logical_requirements()
        if 'error' in req:
            return req

        max_depth = device.max_circuit_depth(0.1)
        feasible = (req['qubits'] <= device.n_qubits and
                    req['depth'] <= max_depth)

        return {
            'feasible': feasible,
            'qubits_ok': req['qubits'] <= device.n_qubits,
            'depth_ok': req['depth'] <= max_depth,
            'required_qubits': req['qubits'],
            'required_depth': req['depth'],
            'available_qubits': device.n_qubits,
            'available_depth': max_depth
        }
```

                          'problem_size' if not size_in_regime else 'none'

## Summary

<Summary>
1. **Supremacy vs. Utility**: Moving from proving quantum computers are fast to solving useful problems
2. **NISQ Limitations**: Noise and limited coherence restrict circuit depth and algorithm choice
3. **Error Mitigation**: Techniques like ZNE and PEC extend NISQ capabilities without full error correction
4. **Fault Tolerance Path**: Surface codes and other QEC schemes enable arbitrary-length computation with sufficient physical resources
5. **Application Domains**: Chemistry and physics simulation show clearest path to quantum advantage
</Summary>

<Warning>
Claims of quantum advantage should be scrutinized carefully. The classical algorithms landscape evolves rapidly, and today's quantum advantage may be challenged by tomorrow's classical improvements.
</Warning>
