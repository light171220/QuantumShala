## 1. What is Quantum Advantage?

**Quantum advantage** (or quantum supremacy) refers to the demonstration that a quantum computer can perform a computational task significantly faster than any classical computer. This concept lies at the heart of quantum computing's promise.

<Definition term="Quantum Advantage">
A quantum computational advantage exists for a problem P if there exists a quantum algorithm solving P that is asymptotically faster than the best possible classical algorithm, or if a quantum device can solve instances of P faster than any existing classical method.
</Definition>

### Types of Quantum Advantage

<Formula>
$$\text{Speedup Types:} \begin{cases}
\text{Polynomial} & T_Q = O(T_C^{1/k}) \text{ for } k > 1 \\
\text{Super-polynomial} & T_Q = O(\text{poly}(\log T_C)) \\
\text{Exponential} & T_Q = O(\log T_C)
\end{cases}$$
</Formula>

```python
import numpy as np
from typing import Tuple, List, Dict
from dataclasses import dataclass
from enum import Enum

class SpeedupType(Enum):
    """Classification of quantum speedup types."""
    POLYNOMIAL = "polynomial"
    SUPERPOLYNOMIAL = "super-polynomial"
    EXPONENTIAL = "exponential"
    CONSTANT = "constant"

@dataclass
class QuantumAdvantage:
    """Characterization of quantum advantage for a problem."""
    problem_name: str
    classical_complexity: str
    quantum_complexity: str
    speedup_type: SpeedupType
    proven: bool
    oracle_dependent: bool

    def speedup_factor(self, n: int) -> float:
        """Estimate speedup factor for problem size n."""
        classical = self._parse_complexity(self.classical_complexity, n)
        quantum = self._parse_complexity(self.quantum_complexity, n)
        return classical / quantum if quantum > 0 else float('inf')

    def _parse_complexity(self, complexity: str, n: int) -> float:
        """Parse complexity notation to numerical value."""
        complexity = complexity.upper()
        if 'O(2^N)' in complexity or 'O(EXP' in complexity:
            return 2 ** n
        elif 'O(N^3)' in complexity:
            return n ** 3
        elif 'O(N^2)' in complexity:
            return n ** 2
        elif 'O(N)' in complexity:
            return n
        elif 'O(SQRT(N))' in complexity:
            return np.sqrt(n)
        elif 'O(LOG(N))' in complexity:
            return np.log2(n)
        else:
            return n

# Known quantum advantages
KNOWN_ADVANTAGES = [
    QuantumAdvantage(
        "Integer Factoring", "O(exp(n^{1/3}))", "O(n^3)",
        SpeedupType.EXPONENTIAL, proven=True, oracle_dependent=False
    ),
    QuantumAdvantage(
        "Unstructured Search", "O(N)", "O(sqrt(N))",
        SpeedupType.POLYNOMIAL, proven=True, oracle_dependent=True
    ),
    QuantumAdvantage(
        "Simon's Problem", "O(2^{n/2})", "O(n)",
        SpeedupType.EXPONENTIAL, proven=True, oracle_dependent=True
    ),
]
```

## 2. Complexity-Theoretic Foundations

Understanding quantum advantage requires a deep dive into computational complexity theory.

### The BQP Complexity Class

<Definition term="BQP (Bounded-Error Quantum Polynomial Time)">
BQP is the class of decision problems solvable by a quantum computer in polynomial time with error probability at most 1/3. It represents the quantum analog of BPP for classical probabilistic computation.
</Definition>

<Formula>
$$\text{BQP} = \{L : \exists \text{ poly-time quantum algorithm } A \text{ such that}$$
$$\forall x : x \in L \Rightarrow \Pr[A(x) = 1] \geq 2/3, \quad x \notin L \Rightarrow \Pr[A(x) = 0] \geq 2/3\}$$
</Formula>

### Complexity Class Relationships

```python
def visualize_complexity_hierarchy() -> Dict[str, List[str]]:
    """Define known relationships between complexity classes."""
    hierarchy = {
        'P': ['BPP', 'BQP', 'NP', 'PSPACE'],
        'BPP': ['BQP', 'PSPACE'],
        'BQP': ['PP', 'PSPACE'],
        'NP': ['PSPACE', 'PP'],
        'PP': ['PSPACE'],
        'PSPACE': ['EXP'],
    }

    open_questions = [
        ('BQP', 'NP', 'unknown'),
        ('NP', 'BQP', 'unknown'),
        ('P', 'BQP', 'strict?'),
    ]

    return hierarchy, open_questions
```

## 3. The Quantum Supremacy Threshold

<Definition term="Quantum Supremacy">
Quantum supremacy is achieved when a quantum device performs a well-defined computational task that cannot be performed by any classical computer in a reasonable amount of time, regardless of the task's practical utility.
</Definition>

### Cross-Entropy Benchmarking

The cross-entropy benchmark (XEB) is the primary metric for verifying quantum computational advantage.

<Formula>
$$F_{XEB} = 2^n \langle p(x) \rangle_x - 1$$
</Formula>

where $p(x)$ is the ideal probability of bitstring $x$ and the average is over sampled bitstrings.

```python
def cross_entropy_benchmark(
    sampled_bitstrings: np.ndarray,
    ideal_probabilities: np.ndarray
) -> float:
    """
    Compute the linear cross-entropy benchmark (XEB).
    """
    n_qubits = int(np.log2(len(ideal_probabilities)))
    avg_prob = np.mean([ideal_probabilities[b] for b in sampled_bitstrings])
    xeb = (2 ** n_qubits) * avg_prob - 1
    return xeb
```

## 4. Verification of Quantum Computations

A crucial challenge is verifying that a quantum computer performed the claimed computation.

<Tip>
For quantum supremacy experiments, direct verification is impossible (by definition). Instead, we rely on indirect verification methods like XEB and spoofing analysis.
</Tip>

```python
class QuantumVerifier:
    """Methods for verifying quantum computational claims."""

    @staticmethod
    def verify_via_smaller_instances(
        quantum_sampler, classical_simulator,
        small_n: int, n_samples: int = 10000
    ) -> Dict[str, float]:
        """Verify quantum device on tractable instances."""
        quantum_samples = quantum_sampler.sample(small_n, n_samples)
        ideal_probs = classical_simulator.compute_distribution(small_n)
        xeb = cross_entropy_benchmark(quantum_samples, ideal_probs)
        return {'xeb_fidelity': xeb}
```
