## 1. The Dequantization Revolution

In 2018, Ewin Tang's breakthrough showed that some claimed quantum speedups could be matched classically:

<Definition term="Dequantization">
Dequantization is the process of finding classical algorithms that match the performance of quantum algorithms by leveraging the same structural assumptions (like sampling access to data).
</Definition>

<Warning>
Dequantization doesn't mean quantum computers are useless. It reveals that certain speedups came from data structure assumptions, not quantum mechanics itself.
</Warning>

## 2. Sample and Query Access

The key insight: many quantum linear algebra algorithms assume special data access:

<Definition term="Sample and Query Access">
A vector $v \in \mathbb{R}^n$ has sample-and-query access if:
1. **Query**: Access any entry $v_i$ in $O(1)$
2. **Sample**: Sample index $i$ with probability $|v_i|^2 / \|v\|^2$ in $O(1)$
3. **Norm**: Query $\|v\|$ in $O(1)$

This is what quantum state preparation provides "for free."
</Definition>

```python
import numpy as np
from typing import Callable

class SampleQueryAccess:
    """
    Data structure providing sample and query access to a vector.

    This is the classical analog of quantum state access.
    """

    def __init__(self, v: np.ndarray):
        """
        Build sample-query data structure.

        Args:
            v: Vector to provide access to
        """
        self.v = v.copy()
        self.n = len(v)
        self.norm = np.linalg.norm(v)

        # Build sampling tree (binary indexed tree for prefix sums)
        self.sq_cumsum = np.cumsum(v ** 2)
        self.total_sq = self.sq_cumsum[-1]

    def query(self, i: int) -> float:
        """Query entry v[i] in O(1)."""
        return self.v[i]

    def sample(self) -> int:
        """Sample index i with probability |v_i|^2/||v||^2."""
        u = np.random.random() * self.total_sq
        return np.searchsorted(self.sq_cumsum, u)

    def get_norm(self) -> float:
        """Return ||v||."""
        return self.norm

class MatrixSampleQuery:
    """
    Sample-query access to matrix rows and entries.
    """

    def __init__(self, A: np.ndarray):
        """
        Build sample-query structure for matrix.

        Args:
            A: Matrix to provide access to
        """
        self.A = A.copy()
        self.m, self.n = A.shape

        # Row norms
        self.row_norms = np.linalg.norm(A, axis=1)
        self.frob_norm = np.linalg.norm(A, 'fro')

        # Row sampling weights
        self.row_weights = self.row_norms ** 2
        self.row_cumsum = np.cumsum(self.row_weights)

        # Column sampling for each row
        self.row_sq = [np.cumsum(A[i]**2) for i in range(self.m)]

    def query(self, i: int, j: int) -> float:
        """Query entry A[i,j]."""
        return self.A[i, j]

    def sample_row(self) -> int:
        """Sample row i with probability ||A_i||^2/||A||_F^2."""
        u = np.random.random() * self.row_cumsum[-1]
        return np.searchsorted(self.row_cumsum, u)

    def sample_column(self, row: int) -> int:
        """Sample column j from row with probability |A[row,j]|^2/||A_row||^2."""
        u = np.random.random() * self.row_sq[row][-1]
        return np.searchsorted(self.row_sq[row], u)

    def get_row_norm(self, i: int) -> float:
        """Return ||A_i||."""
        return self.row_norms[i]

    def get_frobenius_norm(self) -> float:
        """Return ||A||_F."""
        return self.frob_norm

# Demonstrate sampling
np.random.seed(42)
v = np.array([1, 2, 3, 4, 5], dtype=float)
sq_v = SampleQueryAccess(v)

print("Sample-Query Access Demo")
print("=" * 40)
print(f"Vector: {v}")
print(f"Norm: {sq_v.get_norm():.4f}")
print(f"Query v[2]: {sq_v.query(2)}")

# Sample distribution

print(f"Empirical: {empirical}")
```

## 3. Tang's Dequantization of Recommendation Systems

The quantum recommendation algorithm claimed exponential speedup. Tang showed classical can match it:

```python
import numpy as np

def quantum_inspired_recommendation(A: np.ndarray, user_idx: int,
                                      rank: int, n_samples: int) -> dict:
    """
    Quantum-inspired recommendation using sampling.

    This is a simplified version of Tang's algorithm.

    Args:
        A: User-item rating matrix
        user_idx: User to generate recommendations for
        rank: Low-rank approximation parameter
        n_samples: Number of samples for estimation

    Returns:
        Recommended items and analysis
    """
    m, n = A.shape
    A_sq = MatrixSampleQuery(A)

    # Step 1: Sample rows to build low-rank approximation
    sampled_rows = []
    row_samples = [A_sq.sample_row() for _ in range(n_samples)]

    # Get approximate row space
    for row_idx in set(row_samples):
        count = row_samples.count(row_idx)
        weight = A_sq.get_frobenius_norm() / (np.sqrt(n_samples) * A_sq.get_row_norm(row_idx))
        sampled_rows.append((row_idx, weight, A[row_idx]))

    # Step 2: Build approximate SVD using sampled rows
    if len(sampled_rows) >= rank:
        R = np.array([r[2] for r in sampled_rows[:rank]])
        # Simple QR for orthogonal basis
        Q, _ = np.linalg.qr(R.T)
        V_approx = Q[:, :rank]  # Approximate right singular vectors
    else:
        V_approx = np.eye(n)[:, :rank]

    # Step 3: Project user vector onto approximate subspace
    user_vec = A[user_idx]
    user_projected = V_approx @ (V_approx.T @ user_vec)

    # Step 4: Generate recommendations (items with high projected values)
    recommendations = np.argsort(user_projected)[::-1]

    return {
        'recommendations': recommendations[:10],
        'projected_scores': user_projected[recommendations[:10]],
        'n_samples_used': n_samples,
        'effective_rank': len(sampled_rows)
    }

def compare_quantum_classical_recommendation(A: np.ndarray, user_idx: int,
                                               rank: int) -> dict:
    """
    Compare quantum-inspired vs full SVD recommendation.
    """
    # Full SVD (classical baseline)
    U, S, Vh = np.linalg.svd(A, full_matrices=False)

    # Low-rank approximation
    U_k = U[:, :rank]
    S_k = S[:rank]
    Vh_k = Vh[:rank, :]

    A_k = U_k @ np.diag(S_k) @ Vh_k
    true_recommendations = np.argsort(A_k[user_idx])[::-1][:10]

    # Quantum-inspired
    qi_result = quantum_inspired_recommendation(A, user_idx, rank, n_samples=100)
    qi_recommendations = qi_result['recommendations']

    # Overlap metric
    overlap = len(set(true_recommendations) & set(qi_recommendations))

    return {
        'svd_recommendations': true_recommendations,
        'qi_recommendations': qi_recommendations,
        'overlap': overlap,
        'precision': overlap / 10
    }

# Example
np.random.seed(42)
m, n = 100, 50  # 100 users, 50 items
rank_true = 5

# Low-rank rating matrix
U = np.random.randn(m, rank_true)
V = np.random.randn(rank_true, n)
A = U @ V + 0.1 * np.random.randn(m, n)

print("\nQuantum-Inspired Recommendation")
print("=" * 40)

result = compare_quantum_classical_recommendation(A, user_idx=0, rank=5)

print(f"SVD recommendations: {result['svd_recommendations']}")
print(f"QI recommendations:  {result['qi_recommendations']}")
print(f"Overlap: {result['overlap']}/10")
print(f"Precision: {result['precision']:.0%}")
```

## 4. What Does Dequantization Tell Us?

<Definition term="Lessons from Dequantization">
1. **Sampling assumptions matter**: If quantum gets sampling "free," classical can build it
2. **Query vs time complexity**: Quantum query speedups may not translate to time speedups
3. **Input model is key**: How data is provided determines possible speedups
4. **Some advantages remain**: Not all quantum algorithms are dequantizable
</Definition>

```python
import numpy as np

def analyze_dequantization_conditions(problem: dict) -> dict:
    """
    Analyze whether a quantum algorithm is susceptible to dequantization.

    Args:
        problem: Problem specification

    Returns:
        Analysis of dequantization vulnerability
    """
    # Key factors
    factors = {
        'sampling_assumed': problem.get('quantum_sampling', False),
        'low_rank_structure': problem.get('low_rank', False),
        'output_is_samples': problem.get('sample_output', False),
        'data_is_classical': problem.get('classical_data', True),
        'qram_required': problem.get('needs_qram', False)
    }

    # Vulnerability score
    vulnerability = 0
    reasons = []

    if factors['sampling_assumed']:
        vulnerability += 2
        reasons.append("Sampling access can be provided classically with preprocessing")

    if factors['low_rank_structure']:
        vulnerability += 2
        reasons.append("Low-rank structure enables efficient classical sampling")

    if factors['output_is_samples']:
        vulnerability += 1
        reasons.append("Sample output can often be matched by classical sampling")

    if factors['data_is_classical']:
        vulnerability += 1
        reasons.append("Classical data requires O(n) loading, may dominate")

    if factors['qram_required']:
        vulnerability += 1
        reasons.append("QRAM assumption enables classical SQ structure")

    # Robust advantages
    robust_factors = {
        'superposition_essential': not factors['output_is_samples'],
        'exponential_state_space': problem.get('exponential_states', False),
        'interference_critical': problem.get('needs_interference', False),
        'no_classical_analog': problem.get('quantum_native', False)
    }

    robustness = sum(robust_factors.values())

    return {
        'vulnerability_score': vulnerability,
        'robustness_score': robustness,
        'likely_dequantizable': vulnerability > robustness,
        'vulnerability_reasons': reasons,
        'robust_factors': robust_factors
    }
```

## 6. Query Complexity vs Time Complexity

<Warning>
Query complexity measures oracle calls, not total operations. A quantum algorithm with fewer queries may still be slower due to:
- State preparation overhead
- Gate complexity per query
- Classical pre/post-processing
</Warning>

```python
import numpy as np

def complexity_decomposition(algorithm: str, n: int, params: dict) -> dict:
    """
    Break down total complexity into components.

    Args:
        algorithm: Algorithm name
        n: Problem size
        params: Algorithm parameters

    Returns:
        Detailed complexity breakdown
    """
    if algorithm == 'HHL':
        kappa = params.get('kappa', 10)
        sparsity = params.get('sparsity', 10)
        precision = params.get('precision', 0.01)

        # Query complexity (number of matrix accesses)
        query_complexity = np.log2(n) * sparsity * kappa / precision

        # Gate complexity per query (Hamiltonian simulation)
        gates_per_query = sparsity * np.log2(n)

        # Total gate complexity
        total_gates = query_complexity * gates_per_query

        # State preparation (often dominates!)
        state_prep_gates = n  # General case
        state_prep_qram = np.log2(n)  # With QRAM

        # Output extraction
        output_full = n * (1 / precision**2)  # Tomography
        output_sample = 1 / precision**2  # Single measurement

        return {
            'query_complexity': query_complexity,
            'gates_per_query': gates_per_query,
            'total_gates': total_gates,
            'state_prep': {
                'general': state_prep_gates,
                'qram': state_prep_qram
            },
            'output': {
                'full_vector': output_full,
                'single_sample': output_sample
            },
            'total_realistic': state_prep_gates + total_gates + output_sample,
            'total_optimistic': state_prep_qram + total_gates + output_sample
        }

    elif algorithm == 'Classical_CG':
        kappa = params.get('kappa', 10)
        sparsity = params.get('sparsity', 10)

        # Conjugate gradient iterations
        iterations = np.sqrt(kappa) * np.log(1/params.get('precision', 0.01))

        # Work per iteration (sparse matrix-vector product)
        work_per_iter = n * sparsity

        return {
            'iterations': iterations,
            'work_per_iteration': work_per_iter,
            'total_operations': iterations * work_per_iter
        }

    return {}
```

## 7. The Data Loading Problem

The critical bottleneck for classical data:

```python
import numpy as np

def analyze_data_loading_overhead(data_size: int, data_type: str) -> dict:
    """
    Analyze the data loading overhead for quantum algorithms.

    Args:
        data_size: Size of classical data
        data_type: Type of data structure

    Returns:
        Loading analysis
    """
    methods = {
        'state_preparation': {
            'general': data_size,
            'description': 'Arbitrary amplitude encoding requires O(n) gates',
            'example': 'Loading a classical vector into |psi>'
        },
        'qram': {
            'general': np.log2(data_size),
            'description': 'QRAM provides O(log n) access but hardware is speculative',
            'example': 'Quantum random access to stored data'
        },
        'data_structure': {
            'general': data_size,  # One-time preprocessing
            'query': np.log2(data_size),
            'description': 'Build classical data structure for efficient quantum access',
            'example': 'KP-trees for sample-query access'
        },
        'structured_data': {
            'general': np.log2(data_size) ** 2,
            'description': 'Exploit structure (sparsity, low-rank) for efficient loading',
            'example': 'Sparse matrices, tensor products'
        }
    }

    # Impact on speedup
    algorithm_speedup = np.log2(data_size) ** 2  # Typical quantum algorithm improvement

    analysis = {}
    for method, info in methods.items():
        loading_cost = info['general']
        effective_speedup = algorithm_speedup / loading_cost * data_size  # Relative to classical

        analysis[method] = {
            'cost': loading_cost,
            'effective_speedup': effective_speedup,
            'viable': effective_speedup > 1,
            'description': info['description']
        }

    return analysis
```

## 8. When Does Quantum Win?

<Definition term="Conditions for Genuine Quantum Advantage">
Quantum linear algebra provides genuine advantage when:

1. **Data is quantum**: Output of quantum sensors or simulations
2. **Output is quantum**: Feeds into further quantum processing
3. **Exponential state space**: Problem inherently requires superposition
4. **Structured input**: Efficient state preparation possible
5. **Sampling suffices**: Don't need full classical output
</Definition>


| Hybrid algorithms | Combine classical and quantum optimally | NISQ focus |
| Error mitigation | Make noisy quantum useful | Near-term priority |

## 10. Summary

Understanding dequantization reveals the true nature of quantum advantage:

1. **Sampling assumptions**: Classical can match quantum when data has efficient sampling access
2. **Input/output overhead**: Loading and extracting classical data can negate quantum speedups
3. **Query vs time**: Query complexity hides significant practical costs
4. **Robust advantages**: Exist for quantum simulation, certain optimization, cryptography
5. **Hybrid future**: Best applications likely combine quantum and classical

The key insight: quantum advantage is real but requires careful end-to-end analysis, not just asymptotic query complexity.

<Tip>
When evaluating quantum algorithm claims, always ask:
1. What input model is assumed?
2. What output is actually needed?
3. What are the condition number and other problem-specific parameters?
4. Is there structure that could enable classical speedup?
</Tip>
