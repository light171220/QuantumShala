## 1. Linear Systems in Machine Learning

Many machine learning algorithms reduce to solving linear systems:

<Definition term="ML as Linear Algebra">
Core ML problems involving linear systems:
- **Linear Regression**: $(X^TX)\beta = X^Ty$
- **Ridge Regression**: $(X^TX + \lambda I)\beta = X^Ty$
- **Least Squares Classification**: Similar normal equations
- **Kernel Methods**: $K\alpha = y$ where $K_{ij} = k(x_i, x_j)$
</Definition>

<Tip>
HHL and related algorithms can potentially speed up these computations, but require careful analysis of input/output overheads.
</Tip>

## 2. Quantum Linear Regression

```python
import numpy as np
from scipy.linalg import solve

def classical_linear_regression(X: np.ndarray, y: np.ndarray,
                                  regularization: float = 0.0) -> dict:
    """
    Solve linear regression via normal equations.

    Args:
        X: Design matrix (n_samples x n_features)
        y: Target vector
        regularization: Ridge regularization parameter

    Returns:
        Regression coefficients and analysis
    """
    n_samples, n_features = X.shape

    # Normal equations: (X^T X + λI) β = X^T y
    XTX = X.T @ X
    XTy = X.T @ y

    if regularization > 0:
        XTX += regularization * np.eye(n_features)

    # Solve
    beta = solve(XTX, XTy)

    # Predictions and metrics
    y_pred = X @ beta
    mse = np.mean((y - y_pred) ** 2)
    r2 = 1 - np.sum((y - y_pred)**2) / np.sum((y - np.mean(y))**2)

    # Condition number (affects quantum complexity)
    cond = np.linalg.cond(XTX)

    return {
        'coefficients': beta,
        'predictions': y_pred,
        'mse': mse,
        'r2': r2,
        'condition_number': cond,
        'matrix_size': XTX.shape
    }

def quantum_regression_complexity(n_features: int, n_samples: int,
                                    sparsity: int, condition: float,
                                    precision: float = 0.01) -> dict:
    """
    Analyze quantum complexity for linear regression.

    Args:
        n_features: Number of features (matrix dimension)
        n_samples: Number of samples
        sparsity: Sparsity of X^T X
        condition: Condition number
        precision: Target precision

    Returns:
        Complexity comparison
    """
    # Classical complexity
    # Normal equations: O(n_features² * n_samples + n_features³)
    classical_form = n_features ** 2 * n_samples
    classical_solve = n_features ** 3
    classical_total = classical_form + classical_solve

    # Quantum complexity (HHL-based)
    # O(log(n_features) * sparsity² * condition² / precision)
    quantum_hhl = np.log2(n_features) * sparsity**2 * condition**2 / precision

    # State preparation overhead
    # Need to load X into quantum state
    state_prep = n_samples * n_features  # Can dominate!

    # Output extraction
    # Getting all coefficients: O(n_features)
    output_extraction = n_features

    # Total quantum (optimistic: assumes QRAM)
    quantum_with_qram = quantum_hhl + output_extraction

    # Total quantum (realistic: includes state prep)
    quantum_realistic = state_prep + quantum_hhl + output_extraction

    return {
        'classical': classical_total,
        'quantum_optimistic': quantum_with_qram,
        'quantum_realistic': quantum_realistic,
        'speedup_optimistic': classical_total / quantum_with_qram,
        'speedup_realistic': classical_total / quantum_realistic,
        'state_prep_fraction': state_prep / quantum_realistic
    }

# Example: Linear regression
np.random.seed(42)
n_samples, n_features = 100, 10

# Generate data
X = np.random.randn(n_samples, n_features)
true_beta = np.random.randn(n_features)
y = X @ true_beta + 0.1 * np.random.randn(n_samples)

print("Quantum Linear Regression")
print("=" * 50)

result = classical_linear_regression(X, y, regularization=0.01)
print(f"Classical solution:")
print(f"  R² score: {result['r2']:.4f}")
print(f"  MSE: {result['mse']:.6f}")
print(f"  Condition number: {result['condition_number']:.2f}")

# Complexity analysis
complexity = quantum_regression_complexity(
    n_features=n_features,
    n_samples=n_samples,
    sparsity=n_features,  # Dense
    condition=result['condition_number']
)

print(f"\nComplexity comparison:")
print(f"  Classical: O({complexity['classical']:.0f})")
print(f"  Quantum (optimistic): O({complexity['quantum_optimistic']:.0f})")
print(f"  Quantum (realistic): O({complexity['quantum_realistic']:.0f})")
print(f"  State prep overhead: {complexity['state_prep_fraction']:.1%}")
```

## 3. Quantum Support Vector Machines

Kernel methods naturally fit the quantum framework:

<Formula>
SVM dual problem: Find $\alpha$ solving
$$\max_\alpha \sum_i \alpha_i - \frac{1}{2} \alpha^T Q \alpha$$
where $Q_{ij} = y_i y_j K(x_i, x_j)$.

This involves solving $K\alpha = y$ type systems.
</Formula>

```python
import numpy as np

def rbf_kernel(X: np.ndarray, gamma: float = 1.0) -> np.ndarray:
    """Compute RBF kernel matrix."""
    sq_dist = np.sum(X**2, axis=1).reshape(-1, 1) + \
              np.sum(X**2, axis=1).reshape(1, -1) - \
              2 * X @ X.T
    return np.exp(-gamma * sq_dist)

def classical_kernel_svm(X: np.ndarray, y: np.ndarray,
                          gamma: float = 1.0, C: float = 1.0) -> dict:
    """
    Simplified SVM using kernel matrix inversion.

    For large-scale problems, this demonstrates the
    linear system structure.

    Args:
        X: Training data
        y: Labels (+1/-1)
        gamma: RBF kernel parameter
        C: Regularization

    Returns:
        SVM model parameters
    """
    n = len(y)

    # Kernel matrix
    K = rbf_kernel(X, gamma)

    # Regularized system: (K + λI) α = y
    # (simplified from full SVM dual)
    lambda_reg = 1 / (2 * C * n)
    K_reg = K + lambda_reg * np.eye(n)

    # Solve
    alpha = np.linalg.solve(K_reg, y)

    # Condition number
    cond = np.linalg.cond(K_reg)

    # Decision function
    def predict(X_test):
        K_test = rbf_kernel(X_test, gamma)
        return np.sign(K_test @ alpha)

    return {
        'alpha': alpha,
        'kernel_matrix': K,
        'condition_number': cond,
        'predict': predict
    }

def quantum_svm_analysis(n_samples: int, n_features: int,
                          kernel_condition: float) -> dict:
    """
    Analyze quantum speedup for kernel SVM.

    Args:
        n_samples: Training set size
        n_features: Feature dimension
        kernel_condition: Condition number of kernel matrix

    Returns:
        Complexity analysis
    """
    # Classical: Form K (O(n²d)) + Solve (O(n³))
    classical_kernel = n_samples ** 2 * n_features
    classical_solve = n_samples ** 3
    classical_total = classical_kernel + classical_solve

    # Quantum (with QRAM for kernel access)
    # HHL on n×n system
    log_n = np.log2(n_samples)
    sparsity = n_samples  # Dense kernel
    precision = 0.01

    quantum_hhl = log_n * sparsity**2 * kernel_condition**2 / precision

    # Quantum kernel computation (potential speedup for certain kernels)
    quantum_kernel = log_n * n_features  # Optimistic

    quantum_total = quantum_kernel + quantum_hhl

    return {
        'n_samples': n_samples,
        'classical_total': classical_total,
        'quantum_total': quantum_total,
        'speedup': classical_total / quantum_total,
        'condition_dependency': f'O(κ²) where κ={kernel_condition:.1f}'
    }

# Example
np.random.seed(42)
n = 50
X = np.random.randn(n, 5)
y = np.sign(X[:, 0] + 0.5 * X[:, 1])  # Simple linear boundary

print("\nQuantum Kernel SVM")
print("=" * 50)

svm_result = classical_kernel_svm(X, y, gamma=0.5)
print(f"Kernel matrix condition number: {svm_result['condition_number']:.2f}")

analysis = quantum_svm_analysis(n, 5, svm_result['condition_number'])
print(f"\nComplexity analysis for n={n} samples:")
print(f"  Classical: O({analysis['classical_total']:.0f})")
print(f"  Quantum: O({analysis['quantum_total']:.0f})")
print(f"  Speedup: {analysis['speedup']:.1f}x")
print(f"  Note: {analysis['condition_dependency']}")
```

## 4. Solving Differential Equations

Discretized PDEs become sparse linear systems:

<Definition term="PDE to Linear System">
Discretizing a PDE on an $n$-point grid yields:
$$A\mathbf{u} = \mathbf{f}$$
where $A$ encodes the differential operator and boundary conditions.
For regular grids, $A$ is sparse with $O(n)$ nonzeros.
</Definition>

```python
import numpy as np
from scipy.sparse import diags
from scipy.sparse.linalg import spsolve

def create_poisson_1d(n: int, boundary: tuple = (0, 0)) -> dict:
    """
    Create 1D Poisson equation discretization.

    Solves: -u''(x) = f(x) on [0,1]
    with Dirichlet boundaries u(0) = a, u(1) = b

    Args:
        n: Number of interior grid points
        boundary: (u(0), u(1)) boundary values

    Returns:
        System matrix and grid information
    """
    h = 1 / (n + 1)  # Grid spacing

    # Tridiagonal matrix: [-1, 2, -1] / h²
    diagonals = [
        -np.ones(n-1) / h**2,
        2 * np.ones(n) / h**2,
        -np.ones(n-1) / h**2
    ]
    A = diags(diagonals, [-1, 0, 1], format='csr')

    # Grid points
    x = np.linspace(h, 1-h, n)

    # Boundary contribution to RHS
    bc_contribution = np.zeros(n)
    bc_contribution[0] = boundary[0] / h**2
    bc_contribution[-1] = boundary[1] / h**2

    return {
        'A': A,
        'x': x,
        'h': h,
        'bc_contribution': bc_contribution,
        'sparsity': 3,  # Nonzeros per row
        'condition_estimate': 4 * n**2 / np.pi**2  # For Poisson
    }

def solve_poisson_1d(f_func, n: int, boundary: tuple = (0, 0)) -> dict:
    """
    Solve 1D Poisson equation.

    Args:
        f_func: Right-hand side function f(x)
        n: Number of grid points
        boundary: Boundary conditions

    Returns:
        Solution and analysis
    """
    system = create_poisson_1d(n, boundary)

    # Build RHS
    f = np.array([f_func(xi) for xi in system['x']])
    rhs = f + system['bc_contribution']

    # Solve
    u = spsolve(system['A'], rhs)

    # Add boundary values for complete solution
    u_full = np.concatenate([[boundary[0]], u, [boundary[1]]])
    x_full = np.concatenate([[0], system['x'], [1]])

    return {
        'u': u_full,
        'x': x_full,
        'grid_points': n,
        'h': system['h'],
        'condition_number': system['condition_estimate']
    }

def quantum_pde_complexity(n_points: int, sparsity: int,
                            condition: float, precision: float) -> dict:
    """
    Analyze quantum vs classical complexity for PDE solving.

    Args:
        n_points: Number of grid points
        sparsity: Nonzeros per row
        condition: Condition number
        precision: Target precision

    Returns:
        Complexity comparison
    """
    # Classical: Sparse direct solver O(n) for 1D banded
    # or iterative O(n * iterations) where iterations ~ sqrt(κ)
    classical_direct = n_points  # For 1D banded
    classical_iterative = n_points * np.sqrt(condition)

    # Quantum HHL: O(log(n) * s² * κ² / ε)
    quantum_hhl = np.log2(n_points) * sparsity**2 * condition**2 / precision

    return {
        'classical_direct': classical_direct,
        'classical_iterative': classical_iterative,
        'quantum_hhl': quantum_hhl,
        'speedup_over_direct': classical_direct / quantum_hhl,
        'speedup_over_iterative': classical_iterative / quantum_hhl,
        'note': 'Exponential speedup in n, but κ² dependence is limiting'
    }

# Example: Solve Poisson equation
print("Quantum PDE Solving")
print("=" * 50)

# Define source term: f(x) = sin(πx)
f = lambda x: np.sin(np.pi * x)

# Exact solution: u(x) = sin(πx) / π²
u_exact = lambda x: np.sin(np.pi * x) / np.pi**2

for n in [10, 100, 1000]:
    result = solve_poisson_1d(f, n, boundary=(0, 0))

    # Compute error
    u_exact_vals = u_exact(result['x'])
    max_error = np.max(np.abs(result['u'] - u_exact_vals))

    complexity = quantum_pde_complexity(
        n, sparsity=3, condition=result['condition_number'], precision=0.01
    )

    print(f"\nn = {n} grid points:")
    print(f"  Max error: {max_error:.2e}")
    print(f"  Condition number: {result['condition_number']:.1f}")
    print(f"  Classical: O({complexity['classical_direct']:.0f})")
    print(f"  Quantum HHL: O({complexity['quantum_hhl']:.0f})")
```

## 5. Two-Dimensional PDEs

Higher dimensions increase the problem size exponentially:

```python
import numpy as np
from scipy.sparse import diags, kron, eye
from scipy.sparse.linalg import spsolve

def create_poisson_2d(nx: int, ny: int) -> dict:
    """
    Create 2D Poisson equation discretization.

    Solves: -∇²u = f on unit square [0,1]²
    with zero Dirichlet boundary conditions.

    Args:
        nx, ny: Grid points in each direction

    Returns:
        System matrix and grid
    """
    hx = 1 / (nx + 1)
    hy = 1 / (ny + 1)

    # 1D Laplacian matrices
    Dx = diags([-1, 2, -1], [-1, 0, 1], shape=(nx, nx)) / hx**2
    Dy = diags([-1, 2, -1], [-1, 0, 1], shape=(ny, ny)) / hy**2

    # 2D Laplacian via Kronecker product
    # A = Dx ⊗ I + I ⊗ Dy
    Ix = eye(nx)
    Iy = eye(ny)

    A = kron(Dx, Iy) + kron(Ix, Dy)

    # Grid
    x = np.linspace(hx, 1-hx, nx)
    y = np.linspace(hy, 1-hy, ny)
    X, Y = np.meshgrid(x, y)

    # Total unknowns
    n_total = nx * ny

    return {
        'A': A.tocsr(),
        'X': X,
        'Y': Y,
        'x': x,
        'y': y,
        'nx': nx,
        'ny': ny,
        'n_total': n_total,
        'hx': hx,
        'hy': hy,
        'sparsity': 5  # 5-point stencil
    }

def solve_poisson_2d(f_func, nx: int, ny: int) -> dict:
    """
    Solve 2D Poisson equation.

    Args:
        f_func: Source term f(x, y)
        nx, ny: Grid points

    Returns:
        Solution on grid
    """
    system = create_poisson_2d(nx, ny)

    # Build RHS
    f_values = f_func(system['X'], system['Y'])
    f_flat = f_values.flatten()

    # Solve
    u_flat = spsolve(system['A'], f_flat)
    u = u_flat.reshape(ny, nx)

    return {
        'u': u,
        'X': system['X'],
        'Y': system['Y'],
        'n_total': system['n_total']
    }

def quantum_2d_pde_analysis(nx: int, ny: int) -> dict:
    """
    Analyze quantum advantage for 2D PDEs.

    Args:
        nx, ny: Grid dimensions

    Returns:
        Analysis results
    """
    n_total = nx * ny
    sparsity = 5

    # Condition number for Poisson: O(n²) where n = grid size
    condition = (nx * ny) ** 2 * np.pi ** (-2)

    # Classical: Sparse direct solver O(n^1.5) for 2D
    # or multigrid O(n)
    classical_direct = n_total ** 1.5
    classical_multigrid = n_total * np.log(n_total)

    # Quantum HHL
    precision = 0.01
    quantum_hhl = np.log2(n_total) * sparsity**2 * condition**2 / precision

    # QLSA improvements (e.g., QSVT-based)
    # Better κ dependence: O(κ) instead of O(κ²)
    quantum_qsvt = np.log2(n_total) * sparsity * condition * np.log(1/precision)

    return {
        'grid_size': (nx, ny),
        'total_unknowns': n_total,
        'condition_number': condition,
        'classical_direct': classical_direct,
        'classical_multigrid': classical_multigrid,
        'quantum_hhl': quantum_hhl,
        'quantum_qsvt': quantum_qsvt,
        'hhl_speedup': classical_multigrid / quantum_hhl,
        'qsvt_speedup': classical_multigrid / quantum_qsvt
    }

# Example
print("\n2D Poisson Equation")
print("=" * 50)

# Source: f(x,y) = 2π²sin(πx)sin(πy)
f_2d = lambda X, Y: 2 * np.pi**2 * np.sin(np.pi * X) * np.sin(np.pi * Y)

nx, ny = 20, 20
result = solve_poisson_2d(f_2d, nx, ny)

print(f"Grid: {nx}x{ny} = {result['n_total']} unknowns")
print(f"Max solution value: {np.max(result['u']):.4f}")
print(f"Expected max (at center): {1.0:.4f}")  # sin(πx)sin(πy) peaks at 1

# Quantum analysis
analysis = quantum_2d_pde_analysis(nx, ny)
print(f"\nComplexity analysis:")
print(f"  Condition number: {analysis['condition_number']:.2e}")
print(f"  Classical multigrid: O({analysis['classical_multigrid']:.0f})")
print(f"  Quantum HHL: O({analysis['quantum_hhl']:.2e})")
print(f"  Quantum QSVT: O({analysis['quantum_qsvt']:.2e})")
print(f"  QSVT speedup: {analysis['qsvt_speedup']:.2e}x")
```

## 6. Finite Element Methods

FEM produces sparse systems with structure suitable for quantum solvers:

```python
import numpy as np
from scipy.sparse import lil_matrix
from scipy.sparse.linalg import spsolve

def fem_1d_poisson(n_elements: int, source_func, boundary: tuple) -> dict:
    """
    1D Finite Element Method for Poisson equation.

    Uses linear elements on uniform mesh.

    Args:
        n_elements: Number of elements
        source_func: Source term f(x)
        boundary: (u(0), u(1))

    Returns:
        FEM solution
    """
    # Mesh
    n_nodes = n_elements + 1
    h = 1 / n_elements
    x = np.linspace(0, 1, n_nodes)

    # Global stiffness matrix (assembled from element matrices)
    K = lil_matrix((n_nodes, n_nodes))

    # Element stiffness matrix for linear elements
    k_elem = np.array([[1, -1], [-1, 1]]) / h

    # Assembly
    for e in range(n_elements):
        nodes = [e, e + 1]
        for i, ni in enumerate(nodes):
            for j, nj in enumerate(nodes):
                K[ni, nj] += k_elem[i, j]

    # Load vector
    f = np.zeros(n_nodes)
    for e in range(n_elements):
        x_mid = (x[e] + x[e+1]) / 2
        f_val = source_func(x_mid)
        # Linear element load
        f[e] += f_val * h / 2
        f[e+1] += f_val * h / 2

    # Apply boundary conditions (modify K and f)
    K_csr = K.tocsr()

    # Zero out boundary rows and set diagonal to 1
    K_csr[0, :] = 0
    K_csr[0, 0] = 1
    f[0] = boundary[0]

    K_csr[-1, :] = 0
    K_csr[-1, -1] = 1
    f[-1] = boundary[1]

    # Solve
    u = spsolve(K_csr, f)

    return {
        'u': u,
        'x': x,
        'K': K_csr,
        'n_elements': n_elements,
        'n_dof': n_nodes - 2  # Interior DOFs
    }

def quantum_fem_complexity(n_elements: int, dim: int = 1) -> dict:
    """
    Analyze quantum complexity for FEM problems.

    Args:
        n_elements: Number of elements
        dim: Spatial dimension

    Returns:
        Complexity analysis
    """
    n_dof = n_elements ** dim  # Degrees of freedom

    # Sparsity (bandwidth for 1D, more for higher dim)
    sparsity = 2 * dim + 1  # Stencil size

    # Condition number estimate (mesh-dependent)
    condition = n_elements ** 2  # h^{-2} scaling

    # Classical: Direct solver complexity
    if dim == 1:
        classical = n_dof  # Banded
    elif dim == 2:
        classical = n_dof ** 1.5  # 2D sparse direct
    else:
        classical = n_dof ** 2  # 3D approximate

    # Quantum HHL
    precision = 0.01
    quantum = np.log2(n_dof) * sparsity**2 * condition**2 / precision

    return {
        'dimension': dim,
        'degrees_of_freedom': n_dof,
        'sparsity': sparsity,
        'condition_number': condition,
        'classical': classical,
        'quantum': quantum,
        'speedup': classical / quantum
    }

# Example
print("\nFinite Element Method")
print("=" * 50)

# Solve -u'' = 1 with u(0) = u(1) = 0
# Exact: u(x) = x(1-x)/2

for n_elem in [10, 100, 1000]:
    result = fem_1d_poisson(n_elem, lambda x: 1.0, (0, 0))

    # Error at midpoint
    u_exact_mid = 0.125  # x(1-x)/2 at x=0.5
    u_numerical_mid = result['u'][n_elem // 2]
    error = abs(u_exact_mid - u_numerical_mid)

    complexity = quantum_fem_complexity(n_elem, dim=1)

    print(f"\n{n_elem} elements:")
    print(f"  Error at midpoint: {error:.2e}")
    print(f"  Classical: O({complexity['classical']:.0f})")
    print(f"  Quantum: O({complexity['quantum']:.0f})")
```

## 7. Time-Dependent Problems

Time evolution can be cast as sequences of linear systems:

```python
import numpy as np
from scipy.sparse import diags, eye
from scipy.sparse.linalg import spsolve

def heat_equation_implicit(n_x: int, n_t: int, T_final: float,
                            initial_condition, alpha: float = 1.0) -> dict:
    """
    Solve 1D heat equation using implicit Euler.

    u_t = α u_xx

    Implicit scheme: (I - αΔt/h² D) u^{n+1} = u^n

    Args:
        n_x: Spatial grid points
        n_t: Time steps
        T_final: Final time
        initial_condition: Function u(x, 0)
        alpha: Diffusion coefficient

    Returns:
        Solution history
    """
    h = 1 / (n_x + 1)
    dt = T_final / n_t
    x = np.linspace(h, 1-h, n_x)

    # Diffusion matrix
    r = alpha * dt / h**2
    D = diags([-r, 1 + 2*r, -r], [-1, 0, 1], shape=(n_x, n_x), format='csr')

    # Initial condition
    u = np.array([initial_condition(xi) for xi in x])
    history = [u.copy()]

    # Time stepping
    for n in range(n_t):
        u = spsolve(D, u)
        history.append(u.copy())

    return {
        'u_final': u,
        'history': np.array(history),
        'x': x,
        't': np.linspace(0, T_final, n_t + 1),
        'n_solves': n_t,
        'matrix_condition': 1 + 2*r  # Dominant eigenvalue ratio
    }

def quantum_time_stepping_analysis(n_x: int, n_t: int,
                                     condition_per_step: float) -> dict:
    """
    Analyze quantum vs classical for time-dependent PDEs.

    Args:
        n_x: Spatial DOF
        n_t: Number of time steps
        condition_per_step: Condition number of each linear system

    Returns:
        Analysis
    """
    # Classical: Solve n_t systems, each O(n_x) for banded
    classical_per_step = n_x
    classical_total = n_t * classical_per_step

    # Quantum: Each step uses HHL
    precision = 0.01
    quantum_per_step = np.log2(n_x) * 9 * condition_per_step**2 / precision
    quantum_total = n_t * quantum_per_step

    # Alternative: Encode time evolution unitarily
    # May offer better parallelism
    quantum_parallel = np.log2(n_x * n_t) * 9 * condition_per_step**2 / precision

    return {
        'n_spatial': n_x,
        'n_temporal': n_t,
        'classical_total': classical_total,
        'quantum_sequential': quantum_total,
        'quantum_parallel': quantum_parallel,
        'sequential_speedup': classical_total / quantum_total,
        'parallel_speedup': classical_total / quantum_parallel
    }
```

## 8. Practical Considerations

<Warning>
End-to-end quantum advantage requires careful accounting of:
1. **State preparation**: Loading classical data into quantum states
2. **Query access**: How the matrix is provided
3. **Output extraction**: What information is needed
4. **Condition number**: κ² dependence can dominate
</Warning>

```python
import numpy as np

def analyze_end_to_end_advantage(problem_params: dict) -> dict:
    """
    Comprehensive analysis of quantum advantage for linear system applications.

    Args:
        problem_params: Dictionary with problem specifications

    Returns:
        End-to-end analysis
    """
    n = problem_params['dimension']
    sparsity = problem_params.get('sparsity', n)
    condition = problem_params.get('condition_number', 10)
    precision = problem_params.get('precision', 0.01)

    # Classical baseline
    if sparsity < n / 10:
        classical_solve = n * sparsity  # Sparse
    else:
        classical_solve = n ** 3  # Dense

    # Quantum components
    # 1. State preparation
    state_prep = problem_params.get('state_prep_cost', n)  # Default: linear

    # 2. Matrix access (QRAM or sparse oracle)
    if problem_params.get('qram_available', False):
        matrix_access = np.log2(n)
    else:
        matrix_access = sparsity * np.log2(n)

    # 3. HHL solve
    hhl_solve = np.log2(n) * sparsity**2 * condition**2 / precision

    # 4. Output extraction
    output_type = problem_params.get('output_type', 'full')
    if output_type == 'full':
        output_cost = n  # Tomography
    elif output_type == 'expectation':
        output_cost = 1 / precision**2  # Sampling
    else:
        output_cost = np.log2(n)  # Single amplitude

    # Total costs
    quantum_total = state_prep + matrix_access + hhl_solve + output_cost
    classical_total = classical_solve

    # Speedup analysis
    speedup = classical_total / quantum_total
    bottleneck = max(
        ('state_prep', state_prep),
        ('matrix_access', matrix_access),
        ('hhl_solve', hhl_solve),
        ('output', output_cost),
        key=lambda x: x[1]
    )

    return {
        'classical_total': classical_total,
        'quantum_total': quantum_total,
        'speedup': speedup,
        'advantageous': speedup > 1,
        'bottleneck': bottleneck[0],
        'bottleneck_fraction': bottleneck[1] / quantum_total,
        'components': {
            'state_prep': state_prep,
            'matrix_access': matrix_access,
            'hhl_solve': hhl_solve,
            'output': output_cost
        }
    }
```

## 9. Current Research Directions

Several areas are advancing quantum linear system solvers:

| Direction | Goal | Status |
|-----------|------|--------|
| Better κ dependence | O(κ) instead of O(κ²) | QSVT achieves this |
| Efficient state prep | Avoid O(n) loading | Data structures, QRAM |
| Output extraction | Efficient amplitude estimation | Improved sampling |
| Preconditioners | Reduce effective κ | Active research |
| Near-term algorithms | VQE for linear systems | NISQ exploration |

## 10. Summary

Quantum algorithms for linear systems offer potential speedups for:

1. **Machine Learning**: Linear regression, SVM, kernel methods
2. **PDEs**: Poisson, heat, wave equations
3. **FEM**: Structural mechanics, fluid dynamics
4. **Time Evolution**: Implicit schemes, Hamiltonian simulation

Key considerations:
- Exponential speedup in dimension requires QRAM or structured input
- Condition number dependence O(κ²) or O(κ) is often limiting
- Output type matters: sampling vs full solution
- State preparation overhead can dominate

The most promising applications have:
- Large, sparse systems
- Moderate condition numbers
- Sampling or expectation value outputs
- Quantum or efficiently preparable inputs
