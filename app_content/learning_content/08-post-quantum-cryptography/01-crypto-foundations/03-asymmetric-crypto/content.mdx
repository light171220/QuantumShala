## 1. The Public-Key Revolution

In 1976, Whitfield Diffie and Martin Hellman published a paper that revolutionized cryptography. They proposed a seemingly impossible idea: what if encryption and decryption used *different* keys?

<Definition term="Asymmetric Cryptography">
A cryptographic system using a mathematically related key pair: a public key (freely distributed) for encryption, and a private key (kept secret) for decryption. Anyone can encrypt, but only the private key holder can decrypt.
</Definition>

<Formula>
$$C = E_{K_{pub}}(P) \quad \text{and} \quad P = D_{K_{priv}}(C)$$
</Formula>

```
    ┌─────────────────────────────────────────────────────┐
    │                    Alice                            │
    │   Private Key: K_priv (SECRET!)                    │
    │   Public Key:  K_pub  (Share with everyone)        │
    └─────────────────────────────────────────────────────┘
                              │
                              │ Publishes K_pub
                              ▼
    ┌─────────────────────────────────────────────────────┐
    │                     Bob                             │
    │   1. Gets Alice's public key K_pub                 │
    │   2. Encrypts: C = E_{K_pub}(message)              │
    │   3. Sends C to Alice                              │
    └─────────────────────────────────────────────────────┘
                              │
                              │ Sends ciphertext C
                              ▼
    ┌─────────────────────────────────────────────────────┐
    │                    Alice                            │
    │   Decrypts: message = D_{K_priv}(C)                │
    │   Only Alice can decrypt (only she has K_priv)     │
    └─────────────────────────────────────────────────────┘
```

<Tip>
Think of it like a mailbox: anyone can drop mail in (public key = mail slot), but only the owner with the key can retrieve it (private key = mailbox key).
</Tip>

## 2. Mathematical Foundations: Trapdoor Functions

Asymmetric cryptography relies on **trapdoor one-way functions**:

<Definition term="Trapdoor One-Way Function">
A function that is easy to compute in one direction but computationally infeasible to reverse—unless you have special "trapdoor" information (the private key).
</Definition>

### Key Properties

| Property | Description |
|----------|-------------|
| **Easy forward** | Computing $f(x)$ is efficient |
| **Hard inverse** | Computing $f^{-1}(y)$ without trapdoor is infeasible |
| **Trapdoor** | With secret info, computing $f^{-1}(y)$ becomes easy |

### The Three Main Trapdoors

| Problem | Based On | Used By |
|---------|----------|---------|
| Integer Factorization | Factoring $N = p \times q$ | RSA |
| Discrete Logarithm | Finding $x$ in $g^x = h \pmod{p}$ | Diffie-Hellman, DSA |
| Elliptic Curve DLP | Finding $k$ in $kG = P$ | ECDH, ECDSA |

<Warning>
**All three problems are efficiently solvable by quantum computers using Shor's algorithm!** This is why we need post-quantum cryptography.
</Warning>

## 3. RSA: The First Practical Public-Key System

RSA (Rivest-Shamir-Adleman, 1977) was the first practical public-key encryption system and remains widely used today.

### RSA Key Generation

<Formula>
$$\text{1. Choose primes } p, q$$
$$\text{2. Compute } N = p \times q$$
$$\text{3. Compute } \phi(N) = (p-1)(q-1)$$
$$\text{4. Choose } e \text{ coprime to } \phi(N)$$
$$\text{5. Compute } d \equiv e^{-1} \pmod{\phi(N)}$$
</Formula>

- **Public key**: $(N, e)$
- **Private key**: $(N, d)$ or equivalently $(p, q, d)$

### RSA Encryption/Decryption

<Formula>
$$\text{Encrypt: } C = M^e \pmod{N}$$
$$\text{Decrypt: } M = C^d \pmod{N}$$
</Formula>

```python
import random
from math import gcd

def generate_prime(bits):
    """Generate a prime number (simplified - use proper library in practice)"""
    from sympy import isprime, randprime
    return randprime(2**(bits-1), 2**bits)

def mod_inverse(e, phi):
    """Extended Euclidean algorithm for modular inverse"""
    def extended_gcd(a, b):
        if a == 0:
            return b, 0, 1
        gcd_val, x1, y1 = extended_gcd(b % a, a)
        x = y1 - (b // a) * x1
        y = x1
        return gcd_val, x, y
    
    _, x, _ = extended_gcd(e % phi, phi)
    return (x % phi + phi) % phi

def generate_rsa_keypair(bits=512):
    """Generate RSA key pair (use larger bits in practice!)"""
    # Step 1: Generate two large primes
    p = generate_prime(bits // 2)
    q = generate_prime(bits // 2)
    
    # Step 2: Compute N = p * q
    N = p * q
    
    # Step 3: Compute φ(N) = (p-1)(q-1)
    phi = (p - 1) * (q - 1)
    
    # Step 4: Choose e (commonly 65537)
    e = 65537
    while gcd(e, phi) != 1:
        e += 2
    
    # Step 5: Compute d = e^(-1) mod φ(N)
    d = mod_inverse(e, phi)
    
    return {
        'public': {'N': N, 'e': e},
        'private': {'N': N, 'd': d, 'p': p, 'q': q}
    }

def rsa_encrypt(message: int, public_key: dict) -> int:
    """Encrypt: C = M^e mod N"""
    return pow(message, public_key['e'], public_key['N'])

def rsa_decrypt(ciphertext: int, private_key: dict) -> int:
    """Decrypt: M = C^d mod N"""
    return pow(ciphertext, private_key['d'], private_key['N'])

# Demo
keys = generate_rsa_keypair(512)
message = 12345

ciphertext = rsa_encrypt(message, keys['public'])
decrypted = rsa_decrypt(ciphertext, keys['private'])

print(f"Original message: {message}")
print(f"Ciphertext: {ciphertext}")
print(f"Decrypted: {decrypted}")
print(f"Success: {message == decrypted}")
```

### Why RSA Works

The security of RSA rests on the **factoring problem**:

<Example>
**The Factoring Problem**

Given $N = p \times q$ where $p$ and $q$ are large primes, find $p$ and $q$.

```python
# Easy to multiply (forward direction)
p = 61
q = 53
N = p * q  # N = 3233, instant computation

# Hard to factor (reverse direction) - for large numbers
# Given N = 3233, find p and q
# For small numbers it's easy, but for 2048-bit N, it's infeasible

# RSA-2048 challenge number (not yet factored):
# N = 25195908475657893494027183240048398571...
# Factoring this would take billions of years classically
```

If an attacker could factor $N$, they could compute $\phi(N)$ and then $d$, breaking RSA.
</Example>

### RSA Key Sizes

| Key Size | Security (Classical) | Status |
|----------|---------------------|--------|
| 1024 bits | ~80 bits | **Deprecated** |
| 2048 bits | ~112 bits | Minimum acceptable |
| 3072 bits | ~128 bits | Recommended |
| 4096 bits | ~140 bits | High security |

## 4. Diffie-Hellman Key Exchange

Before RSA, Diffie and Hellman invented a way for two parties to establish a shared secret over an insecure channel.

<Definition term="Diffie-Hellman Key Exchange">
A protocol allowing two parties to establish a shared secret key over an insecure channel without any prior shared secrets.
</Definition>

### The Protocol

<Formula>
$$\text{Public parameters: prime } p, \text{ generator } g$$
$$\text{Alice: } a \text{ (private)}, \quad A = g^a \pmod{p} \text{ (public)}$$
$$\text{Bob: } b \text{ (private)}, \quad B = g^b \pmod{p} \text{ (public)}$$
$$\text{Shared secret: } K = B^a = A^b = g^{ab} \pmod{p}$$
</Formula>

```python
import random

def diffie_hellman_demo():
    """Demonstrate Diffie-Hellman key exchange"""
    # Public parameters (in practice, use standardized groups)
    p = 23  # Small prime for demo
    g = 5   # Generator
    
    # Alice generates private key and public value
    a = random.randint(2, p-2)  # Alice's private key
    A = pow(g, a, p)            # Alice's public value
    
    # Bob generates private key and public value
    b = random.randint(2, p-2)  # Bob's private key
    B = pow(g, b, p)            # Bob's public value
    
    # Key exchange (public values are shared)
    # Alice computes shared secret
    K_alice = pow(B, a, p)      # B^a mod p = g^(ab) mod p
    
    # Bob computes shared secret
    K_bob = pow(A, b, p)        # A^b mod p = g^(ab) mod p
    
    print(f"Public parameters: p={p}, g={g}")
    print(f"Alice: private={a}, public={A}")
    print(f"Bob: private={b}, public={B}")
    print(f"Shared secret (Alice): {K_alice}")
    print(f"Shared secret (Bob): {K_bob}")
    print(f"Keys match: {K_alice == K_bob}")
    
    return K_alice == K_bob

diffie_hellman_demo()
```

### Security: Discrete Logarithm Problem

<Definition term="Discrete Logarithm Problem (DLP)">
Given $g$, $p$, and $A = g^a \pmod{p}$, find $a$. This is believed to be computationally hard for properly chosen parameters.
</Definition>

An eavesdropper sees $p$, $g$, $A$, and $B$ but cannot compute $g^{ab}$ without knowing $a$ or $b$.

## 5. Elliptic Curve Cryptography (ECC)

ECC provides the same security as RSA with much smaller keys.

<Definition term="Elliptic Curve">
A curve defined by an equation of the form $y^2 = x^3 + ax + b$ over a finite field. Points on the curve form a group under a special addition operation.
</Definition>

<Formula>
$$y^2 = x^3 + ax + b \pmod{p}$$
</Formula>

### Why ECC is More Efficient

| Security Level | RSA Key Size | ECC Key Size |
|---------------|--------------|--------------|
| 80 bits | 1024 bits | 160 bits |
| 112 bits | 2048 bits | 224 bits |
| 128 bits | 3072 bits | 256 bits |
| 256 bits | 15360 bits | 512 bits |

<Tip>
ECC achieves the same security with ~1/10th the key size! This means faster operations and smaller certificates—critical for mobile devices and IoT.
</Tip>

### Elliptic Curve Diffie-Hellman (ECDH)

```python
from cryptography.hazmat.primitives.asymmetric import ec
from cryptography.hazmat.primitives import serialization

def ecdh_key_exchange():
    """Demonstrate ECDH key exchange using the cryptography library"""
    
    # Alice generates her key pair
    alice_private = ec.generate_private_key(ec.SECP256R1())
    alice_public = alice_private.public_key()
    
    # Bob generates his key pair
    bob_private = ec.generate_private_key(ec.SECP256R1())
    bob_public = bob_private.public_key()
    
    # Alice computes shared secret using Bob's public key
    alice_shared = alice_private.exchange(ec.ECDH(), bob_public)
    
    # Bob computes shared secret using Alice's public key
    bob_shared = bob_private.exchange(ec.ECDH(), alice_public)
    
    print(f"Alice's shared secret: {alice_shared.hex()[:32]}...")
    print(f"Bob's shared secret: {bob_shared.hex()[:32]}...")
    print(f"Secrets match: {alice_shared == bob_shared}")
    
    return alice_shared == bob_shared

ecdh_key_exchange()
```

### The Elliptic Curve Discrete Logarithm Problem (ECDLP)

<Definition term="ECDLP">
Given points $G$ (generator) and $P = kG$ on an elliptic curve, find the scalar $k$. This is believed to be harder than the standard DLP.
</Definition>

<Formula>
$$P = kG = \underbrace{G + G + \cdots + G}_{k \text{ times}}$$
</Formula>

## 6. RSA vs ECC Comparison

```python
from cryptography.hazmat.primitives.asymmetric import rsa, ec
from cryptography.hazmat.primitives import serialization
import time

def compare_key_generation():
    """Compare RSA and ECC key generation times"""
    
    # RSA-2048
    start = time.time()
    rsa_key = rsa.generate_private_key(
        public_exponent=65537,
        key_size=2048
    )
    rsa_time = time.time() - start
    
    rsa_public_bytes = len(rsa_key.public_key().public_bytes(
        encoding=serialization.Encoding.DER,
        format=serialization.PublicFormat.SubjectPublicKeyInfo
    ))
    
    # ECC P-256 (equivalent security)
    start = time.time()
    ec_key = ec.generate_private_key(ec.SECP256R1())
    ec_time = time.time() - start
    
    ec_public_bytes = len(ec_key.public_key().public_bytes(
        encoding=serialization.Encoding.DER,
        format=serialization.PublicFormat.SubjectPublicKeyInfo
    ))
    
    print("Key Generation Comparison (≈112-bit security)")
    print(f"RSA-2048: {rsa_time*1000:.2f}ms, public key: {rsa_public_bytes} bytes")
    print(f"ECC P-256: {ec_time*1000:.2f}ms, public key: {ec_public_bytes} bytes")
    print(f"ECC is ~{rsa_time/ec_time:.1f}x faster, ~{rsa_public_bytes/ec_public_bytes:.1f}x smaller")

compare_key_generation()
```

### Comparison Summary

| Feature | RSA | ECC |
|---------|-----|-----|
| Key size (128-bit security) | 3072 bits | 256 bits |
| Key generation | Slower | Faster |
| Encryption speed | Moderate | Faster |
| Signature size | Larger | Smaller |
| Standardization | Older, widespread | Newer, growing |
| Patent status | Expired | Some concerns (resolved) |

## 7. The Quantum Threat to Asymmetric Cryptography

<Warning>
**Shor's algorithm breaks ALL currently deployed public-key cryptography!**
</Warning>

| Problem | Classical Complexity | Quantum Complexity |
|---------|---------------------|-------------------|
| Factoring (RSA) | Sub-exponential | **Polynomial** |
| DLP (Diffie-Hellman) | Sub-exponential | **Polynomial** |

## 8. Using Asymmetric Crypto in Practice

### RSA Encryption with OAEP Padding

```python
from cryptography.hazmat.primitives.asymmetric import rsa, padding
from cryptography.hazmat.primitives import hashes

def rsa_encrypt_decrypt_demo():
    """Proper RSA encryption using OAEP padding"""
    
    # Generate key pair
    private_key = rsa.generate_private_key(
        public_exponent=65537,
        key_size=2048
    )
    public_key = private_key.public_key()
    
    # Message to encrypt
    message = b"This is a secret message!"
    
    # Encrypt with OAEP padding (always use padding!)
    ciphertext = public_key.encrypt(
        message,
        padding.OAEP(
            mgf=padding.MGF1(algorithm=hashes.SHA256()),
            algorithm=hashes.SHA256(),
            label=None
        )
    )
    
    # Decrypt
    plaintext = private_key.decrypt(
        ciphertext,
        padding.OAEP(
            mgf=padding.MGF1(algorithm=hashes.SHA256()),
            algorithm=hashes.SHA256(),
            label=None
        )
    )
    
    print(f"Original: {message}")
    print(f"Ciphertext length: {len(ciphertext)} bytes")
    print(f"Decrypted: {plaintext}")
    
    return message == plaintext

rsa_encrypt_decrypt_demo()
```

<Warning>
**Never use "textbook RSA"!** Always use proper padding schemes like OAEP for encryption or PSS for signatures. Raw RSA is vulnerable to many attacks.
</Warning>

### Hybrid Encryption (Best Practice)

```python
from cryptography.hazmat.primitives.asymmetric import rsa, padding
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
from cryptography.hazmat.primitives import hashes
import os

def hybrid_encrypt(public_key, plaintext: bytes) -> dict:
    """
    Hybrid encryption: RSA + AES-GCM
    Best practice for encrypting data of any size
    """
    # Generate random symmetric key
    symmetric_key = os.urandom(32)  # 256-bit AES key
    
    # Encrypt symmetric key with RSA
    encrypted_key = public_key.encrypt(
        symmetric_key,
        padding.OAEP(
            mgf=padding.MGF1(algorithm=hashes.SHA256()),
            algorithm=hashes.SHA256(),
            label=None
        )
    )
    
    # Encrypt data with symmetric key
    aesgcm = AESGCM(symmetric_key)
    nonce = os.urandom(12)
    ciphertext = aesgcm.encrypt(nonce, plaintext, None)
    
    return {
        'encrypted_key': encrypted_key,
        'nonce': nonce,
        'ciphertext': ciphertext
    }

def hybrid_decrypt(private_key, encrypted_data: dict) -> bytes:
    """Decrypt hybrid-encrypted data"""
    # Decrypt symmetric key with RSA
    symmetric_key = private_key.decrypt(
        encrypted_data['encrypted_key'],
        padding.OAEP(
            mgf=padding.MGF1(algorithm=hashes.SHA256()),
            algorithm=hashes.SHA256(),
            label=None
        )
    )
    
    # Decrypt data with symmetric key
    aesgcm = AESGCM(symmetric_key)
    plaintext = aesgcm.decrypt(
        encrypted_data['nonce'],
        encrypted_data['ciphertext'],
        None
    )
    
    return plaintext

# Demo
private_key = rsa.generate_private_key(public_exponent=65537, key_size=2048)
public_key = private_key.public_key()

message = b"This can be any size! " * 100
encrypted = hybrid_encrypt(public_key, message)
decrypted = hybrid_decrypt(private_key, encrypted)

print(f"Original size: {len(message)} bytes")
print(f"Encrypted key size: {len(encrypted['encrypted_key'])} bytes")
print(f"Decrypted matches: {message == decrypted}")
```

## Summary

In this lesson, you learned about asymmetric cryptography:

1. **Public-key cryptography** uses key pairs: public (encrypt) and private (decrypt)
2. **Trapdoor functions** are easy one way, hard to reverse without the private key
3. **RSA** security relies on the difficulty of factoring large numbers
4. **Diffie-Hellman** enables key exchange over insecure channels using the DLP
5. **ECC** provides equivalent security with much smaller keys than RSA
6. **Shor's algorithm** breaks RSA, DH, and ECC—all current public-key crypto!
7. **Hybrid encryption** combines asymmetric (key exchange) with symmetric (data encryption)
8. **Post-quantum cryptography** is needed to replace vulnerable algorithms

The quantum threat to asymmetric cryptography is the primary driver for post-quantum cryptography. In the next lessons, we'll explore digital signatures and then dive into quantum-resistant alternatives.
