## 1. What is Cryptography?

In an age where billions of digital transactions occur every second, cryptography stands as the invisible guardian of our digital lives. Every time you send a message, make a payment, or log into an account, cryptography is working behind the scenes.

<Definition term="Cryptography">
Cryptography is the science and art of securing communication and data through mathematical techniques that transform information into an unreadable format for unauthorized parties, while allowing authorized parties to recover the original information.
</Definition>

The word comes from the Greek *kryptós* (hidden) and *graphein* (to write)—literally "hidden writing."

### Related Fields

| Field | Focus | Goal |
|-------|-------|------|
| **Cryptography** | Designing secure systems | Protect information |
| **Cryptanalysis** | Breaking secure systems | Find weaknesses |
| **Cryptology** | Both combined | Complete study of codes |
| **Steganography** | Hiding message existence | Concealment |

<Tip>
Cryptography hides the *meaning* of a message, while steganography hides the *existence* of a message. Modern systems often combine both for defense in depth.
</Tip>

## 2. The Four Pillars of Cryptographic Security

Cryptography aims to achieve four fundamental security goals, often remembered as **CIA + Non-repudiation**:

### Confidentiality

<Definition term="Confidentiality">
Ensuring that information is accessible only to authorized parties. Even if an attacker intercepts the data, they cannot understand it without the proper key.
</Definition>

```python
# Simple demonstration of confidentiality concept
original_message = "ATTACK AT DAWN"
encrypted_message = "XGGXFP XG QXBK"  # Caesar cipher (not secure!)

# Without the key, the meaning is hidden
print(f"Original: {original_message}")
print(f"Encrypted: {encrypted_message}")
print("An eavesdropper sees only the encrypted version")
```

### Integrity

<Definition term="Integrity">
Guaranteeing that information has not been altered in transit or storage. Any modification, whether accidental or malicious, should be detectable.
</Definition>

### Authentication

<Definition term="Authentication">
Verifying the identity of parties involved in communication. You need assurance that you're communicating with who you think you are.
</Definition>

### Non-repudiation

<Definition term="Non-repudiation">
Preventing a party from denying their actions. If Alice signs a contract digitally, she cannot later claim she didn't sign it.
</Definition>

<Example>
**Real-world scenario: Online Banking**

When you transfer money online:
- **Confidentiality**: Your account details are encrypted
- **Integrity**: The transfer amount cannot be changed in transit
- **Authentication**: The bank verifies it's really you

## 3. Basic Cryptographic Components

Every cryptographic system consists of these fundamental components:

### Plaintext and Ciphertext

<Formula>
$$\text{Plaintext} \xrightarrow{\text{Encryption}} \text{Ciphertext} \xrightarrow{\text{Decryption}} \text{Plaintext}$$
</Formula>

- **Plaintext (P)**: The original, readable message
- **Ciphertext (C)**: The encrypted, unreadable message

### Keys

<Definition term="Cryptographic Key">
A piece of information (typically a string of bits) that determines the output of a cryptographic algorithm. The same algorithm with different keys produces different ciphertexts.
</Definition>

### Encryption and Decryption Functions

<Formula>
$$C = E_K(P) \quad \text{and} \quad P = D_K(C)$$
</Formula>

Where:
- $E$ is the encryption function
- $D$ is the decryption function
- $K$ is the key
- $E_K$ means "encryption using key K"

## 4. Kerckhoffs's Principle

One of the most important principles in cryptography, formulated by Auguste Kerckhoffs in 1883:

<Definition term="Kerckhoffs's Principle">
A cryptographic system should be secure even if everything about the system, except the key, is public knowledge. Security must depend solely on the secrecy of the key, not on the secrecy of the algorithm.
</Definition>

<Warning>
**Security through obscurity is not security!** Never rely on keeping your algorithm secret. History has repeatedly shown that secret algorithms get reverse-engineered, leaked, or independently discovered.
</Warning>

### Why This Matters

| Approach | Problem |
|----------|---------|
| Secret algorithm | If leaked, entire system compromised |
| Public algorithm, secret key | If key leaked, only that key's data compromised |

<Example>
**AES (Advanced Encryption Standard)**

AES is completely public—its specification is freely available to anyone. Yet it secures:
- Banking transactions worth trillions of dollars
- Military communications
- Your WhatsApp messages

The security comes from the key, not from hiding how AES works.

```python
# AES is public knowledge - this is fine!
from cryptography.hazmat.primitives.ciphers import Cipher, algorithms, modes
import os

# Security comes from the SECRET KEY
secret_key = os.urandom(32)  # 256-bit key - KEEP THIS SECRET

# The algorithm is public, the key is not
cipher = Cipher(algorithms.AES(secret_key), modes.GCM(os.urandom(12)))
```
</Example>

## 5. Historical Evolution of Cryptography

### Classical Era (Ancient - 1900s)

**Caesar Cipher** (c. 50 BCE)
Simple substitution by shifting letters:

```python
def caesar_encrypt(plaintext: str, shift: int) -> str:
    """Classical Caesar cipher - NOT SECURE!"""
    result = ""
    for char in plaintext.upper():
        if char.isalpha():
            shifted = (ord(char) - ord('A') + shift) % 26
            result += chr(shifted + ord('A'))
        else:
            result += char
    return result

def caesar_decrypt(ciphertext: str, shift: int) -> str:
    return caesar_encrypt(ciphertext, -shift)

# Example
message = "ATTACK AT DAWN"
encrypted = caesar_encrypt(message, 3)
print(f"Original: {message}")
print(f"Encrypted: {encrypted}")  # DWWDFN DW GDZQ
print(f"Decrypted: {caesar_decrypt(encrypted, 3)}")
```

<Warning>
Classical ciphers like Caesar are trivially broken by modern standards. Caesar cipher has only 25 possible keys—a computer can try all of them instantly!
</Warning>

### Mechanical Era (1900s - 1970s)

**Enigma Machine** (WWII)
- Used by Nazi Germany
- Broken by Alan Turing and team at Bletchley Park
- Demonstrated that even complex mechanical systems could be broken

### Modern Era (1970s - Present)

**Key Developments**:

| Year | Development | Significance |
|------|-------------|--------------|
| 1976 | Diffie-Hellman | First public key exchange |
| 1977 | RSA | First practical public-key encryption |
| 1977 | DES | First standardized symmetric cipher |
| 2001 | AES | Current symmetric standard |
| 2005 | SHA-3 competition | Modern hash standard |

### Post-Quantum Era (2016 - Future)

<Definition term="Post-Quantum Cryptography">
Cryptographic algorithms designed to be secure against attacks by both classical and quantum computers. These algorithms rely on mathematical problems that remain hard even for quantum computers.
</Definition>

## 6. The Quantum Threat

This is why you're taking this course! Quantum computers threaten the foundation of modern cryptography.

### What's at Risk

<Formula>
$$\text{Shor's Algorithm: Factors } N \text{ in } O((\log N)^3) \text{ time}$$
</Formula>

| System | Classical Security | Quantum Security |
|--------|-------------------|------------------|
| RSA-2048 | ~112 bits | **BROKEN** |
| ECC P-256 | ~128 bits | **BROKEN** |
| AES-256 | 256 bits | 128 bits (reduced but safe) |
| SHA-256 | 256 bits | 128 bits (reduced but safe) |

<Example>
**The Scale of the Threat**

```python
# Classical computer breaking RSA-2048
classical_operations = 2**112  # Operations needed
classical_ops_per_second = 10**12  # 1 trillion/sec
classical_years = classical_operations / (classical_ops_per_second * 60 * 60 * 24 * 365)
print(f"Classical attack: {classical_years:.2e} years")

# Quantum computer with Shor's algorithm
# RSA-2048 could be broken in hours/days with sufficient qubits
print("Quantum attack: Hours to days (with ~4000 logical qubits)")
```

Output:
```
Classical attack: 1.64e+20 years
Quantum attack: Hours to days (with ~4000 logical qubits)
```
</Example>

<Warning>
**"Harvest Now, Decrypt Later"**: Adversaries are already collecting encrypted data today, waiting for quantum computers to decrypt it in the future. Sensitive data with long-term value (medical records, state secrets, financial data) is at risk NOW.
</Warning>

## 7. Symmetric vs Asymmetric Cryptography Preview

Modern cryptography uses two main paradigms:

### Symmetric Cryptography
- Same key for encryption and decryption
- Fast and efficient
- Challenge: How to share the key securely?

<Formula>
$$C = E_K(P) \quad \text{and} \quad P = D_K(C) \quad \text{(same } K \text{)}$$
</Formula>

### Asymmetric Cryptography
- Different keys: public key encrypts, private key decrypts
- Solves key distribution problem
- Much slower than symmetric

<Formula>
$$C = E_{K_{pub}}(P) \quad \text{and} \quad P = D_{K_{priv}}(C)$$
</Formula>

```python
# Conceptual comparison
class SymmetricCrypto:
    """Same key for both parties - like a shared house key"""
    def __init__(self, shared_key):
        self.key = shared_key
    
    def encrypt(self, message):
        return f"ENC({message}, {self.key})"
    
    def decrypt(self, ciphertext):
        return f"DEC({ciphertext}, {self.key})"

class AsymmetricCrypto:
    """Different keys - like a mailbox (anyone can put mail in, only owner can retrieve)"""
    def __init__(self):
        self.public_key = "PUBLIC_KEY"
        self.private_key = "PRIVATE_KEY"  # Keep secret!
    
    def encrypt(self, message):
        return f"ENC({message}, {self.public_key})"
    
    def decrypt(self, ciphertext):
        return f"DEC({ciphertext}, {self.private_key})"
```

<Tip>
In practice, we use **hybrid encryption**: asymmetric crypto to exchange a symmetric key, then symmetric crypto for the actual data. This combines the best of both worlds!
</Tip>
