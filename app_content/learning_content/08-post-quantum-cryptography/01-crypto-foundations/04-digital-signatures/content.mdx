## 1. What Are Digital Signatures?

Digital signatures are the electronic equivalent of handwritten signatures—but cryptographically stronger.

<Definition term="Digital Signature">
A cryptographic mechanism that provides authentication (proving who created a message), integrity (proving the message wasn't modified), and non-repudiation (the signer cannot deny signing).
</Definition>

### Properties of Digital Signatures

| Property | Description |
|----------|-------------|
| **Authentication** | Verifies the signer's identity |
| **Integrity** | Detects any modification to the signed data |
| **Non-repudiation** | Signer cannot deny having signed |
| **Verifiable** | Anyone with the public key can verify |

<Tip>
Think of it like a wax seal on a letter: it proves who sent it (authentication), shows if the letter was opened (integrity), and the sender can't deny using their seal (non-repudiation).
</Tip>

## 2. How Digital Signatures Work

Digital signatures use asymmetric cryptography in reverse:

<Formula>
$$\text{Sign: } \sigma = S_{K_{priv}}(M)$$
$$\text{Verify: } V_{K_{pub}}(M, \sigma) \rightarrow \{\text{valid}, \text{invalid}\}$$
</Formula>

```
Signing (Private Key):
┌─────────────┐    ┌──────────────┐    ┌─────────────┐
│   Message   │───▶│  Sign with   │───▶│  Signature  │
│     M       │    │ Private Key  │    │     σ       │
└─────────────┘    └──────────────┘    └─────────────┘

Verification (Public Key):
┌─────────────┐
│   Message   │──┐
│     M       │  │    ┌──────────────┐    ┌─────────────┐
└─────────────┘  ├───▶│ Verify with  │───▶│Valid/Invalid│
┌─────────────┐  │    │ Public Key   │    └─────────────┘
│  Signature  │──┘    └──────────────┘
│     σ       │
└─────────────┘
```

<Warning>
**Never sign raw messages!** Always hash the message first, then sign the hash. This is faster (hash is fixed size) and provides additional security properties.
</Warning>

## 3. Hash-Then-Sign Paradigm

In practice, we hash the message before signing:

<Formula>
$$\sigma = S_{K_{priv}}(H(M))$$
$$V_{K_{pub}}(H(M), \sigma) \rightarrow \{\text{valid}, \text{invalid}\}$$
</Formula>

```python
import hashlib

def hash_then_sign_concept(message: bytes, private_key):
    """
    Hash-then-sign paradigm (conceptual)
    """
    # Step 1: Hash the message
    message_hash = hashlib.sha256(message).digest()
    
    # Step 2: Sign the hash (not the original message)
    signature = sign(message_hash, private_key)
    
    return signature

def verify_concept(message: bytes, signature: bytes, public_key) -> bool:
    """
    Verification (conceptual)
    """
    # Step 1: Hash the message
    message_hash = hashlib.sha256(message).digest()
    
    # Step 2: Verify signature against hash
    return verify(message_hash, signature, public_key)
```

### Why Hash First?

| Reason | Explanation |
|--------|-------------|
| **Efficiency** | Signing fixed-size hash is fast regardless of message size |
| **Security** | Some attacks require signing chosen messages; hashing prevents this |
| **Compatibility** | Any size message can be signed |

## 4. RSA Signatures

RSA can be used for both encryption and signatures.

### RSA Signature (Simplified)

<Formula>
$$\text{Sign: } \sigma = H(M)^d \pmod{N}$$
$$\text{Verify: } \sigma^e \pmod{N} \stackrel{?}{=} H(M)$$
</Formula>

```python
from cryptography.hazmat.primitives.asymmetric import rsa, padding
from cryptography.hazmat.primitives import hashes

def rsa_signature_demo():
    """RSA signature creation and verification"""
    
    # Generate key pair
    private_key = rsa.generate_private_key(
        public_exponent=65537,
        key_size=2048
    )
    public_key = private_key.public_key()
    
    # Message to sign
    message = b"This message is authentic!"
    
    # Sign with PSS padding (recommended for signatures)
    signature = private_key.sign(
        message,
        padding.PSS(
            mgf=padding.MGF1(hashes.SHA256()),
            salt_length=padding.PSS.MAX_LENGTH
        ),
        hashes.SHA256()
    )
    
    print(f"Message: {message}")
    print(f"Signature length: {len(signature)} bytes")
    print(f"Signature (hex): {signature[:32].hex()}...")
    
    # Verify signature
    try:
        public_key.verify(
            signature,
            message,
            padding.PSS(
                mgf=padding.MGF1(hashes.SHA256()),
                salt_length=padding.PSS.MAX_LENGTH
            ),
            hashes.SHA256()
        )
        print("✓ Signature is VALID")
    except Exception as e:
        print(f"✗ Signature is INVALID: {e}")
    
    # Try to verify with modified message
    try:
        public_key.verify(
            signature,
            b"This message was MODIFIED!",
            padding.PSS(
                mgf=padding.MGF1(hashes.SHA256()),
                salt_length=padding.PSS.MAX_LENGTH
            ),
            hashes.SHA256()
        )
        print("✓ Modified message signature is VALID")
    except Exception:
        print("✗ Modified message signature is INVALID (as expected!)")

rsa_signature_demo()
```

### RSA-PSS vs PKCS#1 v1.5

| Scheme | Security | Properties |
|--------|----------|------------|
| **RSA-PSS** | Better | Randomized, proven secure |
| **PKCS#1 v1.5** | Legacy | Deterministic, some vulnerabilities |

<Tip>
Always use **RSA-PSS** for new applications. PKCS#1 v1.5 is still common in legacy systems but has known weaknesses.
</Tip>

## 5. ECDSA: Elliptic Curve Digital Signature Algorithm

ECDSA provides the same security as RSA with much smaller signatures.

<Definition term="ECDSA">
A digital signature algorithm based on elliptic curve cryptography. It produces compact signatures and is widely used in TLS, Bitcoin, and modern cryptographic systems.
</Definition>

### ECDSA Signature Size Comparison

| Algorithm | Security | Signature Size |
|-----------|----------|---------------|
| RSA-2048 | ~112 bits | 256 bytes |
| RSA-3072 | ~128 bits | 384 bytes |
| ECDSA P-256 | ~128 bits | **64 bytes** |
| ECDSA P-384 | ~192 bits | **96 bytes** |

```python
from cryptography.hazmat.primitives.asymmetric import ec
from cryptography.hazmat.primitives import hashes

def ecdsa_signature_demo():
    """ECDSA signature creation and verification"""
    
    # Generate key pair (P-256 curve)
    private_key = ec.generate_private_key(ec.SECP256R1())
    public_key = private_key.public_key()
    
    # Message to sign
    message = b"ECDSA produces compact signatures!"
    
    # Sign
    signature = private_key.sign(
        message,
        ec.ECDSA(hashes.SHA256())
    )
    
    print(f"Message: {message}")
    print(f"Signature length: {len(signature)} bytes")
    print(f"Signature (hex): {signature.hex()}")
    
    # Verify
    try:
        public_key.verify(
            signature,
            message,
            ec.ECDSA(hashes.SHA256())
        )
        print("✓ Signature is VALID")
    except Exception:
        print("✗ Signature is INVALID")
    
    return signature

sig = ecdsa_signature_demo()
```

### ECDSA Mathematics (Simplified)

<Formula>
$$\text{Sign}(m, d):$$
$$k \leftarrow \text{random}$$
$$(x, y) = kG$$
$$r = x \mod n$$
$$s = k^{-1}(H(m) + rd) \mod n$$
$$\text{Signature} = (r, s)$$
</Formula>

<Warning>
**ECDSA requires unique random k for every signature!** Reusing k, or using a predictable k, completely breaks security. This has led to real-world attacks on Bitcoin and PlayStation 3.
</Warning>

## 6. Deterministic Signatures: Ed25519

To avoid the randomness pitfall in ECDSA, modern systems use deterministic schemes.

<Definition term="Ed25519">
A deterministic signature scheme based on the Edwards curve Curve25519. It derives the random value k deterministically from the message and private key, eliminating the risk of randomness failure.
</Definition>

```python
from cryptography.hazmat.primitives.asymmetric import ed25519

def ed25519_demo():
    """Ed25519 signatures - fast and secure"""
    
    # Generate key pair
    private_key = ed25519.Ed25519PrivateKey.generate()
    public_key = private_key.public_key()
    
    # Sign (deterministic - same message always gives same signature)
    message = b"Ed25519 is fast and safe!"
    signature = private_key.sign(message)
    
    print(f"Message: {message}")
    print(f"Signature length: {len(signature)} bytes")
    print(f"Public key length: 32 bytes")
    
    # Verify
    try:
        public_key.verify(signature, message)
        print("✓ Signature is VALID")
    except Exception:
        print("✗ Signature is INVALID")
    
    # Demonstrate determinism
    signature2 = private_key.sign(message)
    print(f"Deterministic: {signature == signature2}")

ed25519_demo()
```

### Signature Scheme Comparison

| Scheme | Key Size | Sig Size | Speed | Security |
|--------|----------|----------|-------|----------|
| RSA-2048 | 256 B | 256 B | Slow | ~112 bits |
| RSA-3072 | 384 B | 384 B | Slower | ~128 bits |
| ECDSA P-256 | 64 B | 64 B | Fast | ~128 bits |
| Ed25519 | 32 B | 64 B | **Fastest** | ~128 bits |

## 7. Real-World Applications

### TLS/SSL Certificates

```python
from cryptography import x509
from cryptography.x509.oid import NameOID
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.primitives.asymmetric import ec
import datetime

def create_self_signed_cert():
    """Create a self-signed certificate (simplified demo)"""
    
    # Generate key pair
    private_key = ec.generate_private_key(ec.SECP256R1())
    public_key = private_key.public_key()
    
    # Build certificate
    subject = issuer = x509.Name([
        x509.NameAttribute(NameOID.COUNTRY_NAME, "US"),
        x509.NameAttribute(NameOID.STATE_OR_PROVINCE_NAME, "California"),
        x509.NameAttribute(NameOID.LOCALITY_NAME, "San Francisco"),
        x509.NameAttribute(NameOID.ORGANIZATION_NAME, "QuantShala"),
        x509.NameAttribute(NameOID.COMMON_NAME, "quantshala.example.com"),
    ])
    
    cert = (
        x509.CertificateBuilder()
        .subject_name(subject)
        .issuer_name(issuer)
        .public_key(public_key)
        .serial_number(x509.random_serial_number())
        .not_valid_before(datetime.datetime.utcnow())
        .not_valid_after(datetime.datetime.utcnow() + datetime.timedelta(days=365))
        .sign(private_key, hashes.SHA256())
    )
    
    print(f"Certificate created!")
    print(f"Subject: {cert.subject}")
    print(f"Signature Algorithm: {cert.signature_algorithm_oid}")
    
    return cert

create_self_signed_cert()
```

### Code Signing

Digital signatures verify software hasn't been tampered with:

```python
def sign_software_package(package_data: bytes, private_key) -> dict:
    """
    Sign a software package for distribution
    """
    from cryptography.hazmat.primitives.asymmetric import padding
    from cryptography.hazmat.primitives import hashes
    import hashlib
    import json
    
    # Compute hash of package
    package_hash = hashlib.sha256(package_data).hexdigest()
    
    # Sign the hash
    signature = private_key.sign(
        package_data,
        padding.PSS(
            mgf=padding.MGF1(hashes.SHA256()),
            salt_length=padding.PSS.MAX_LENGTH
        ),
        hashes.SHA256()
    )
    
    # Create manifest
    manifest = {
        'package_hash': package_hash,
        'package_size': len(package_data),
        'signature': signature.hex(),
        'algorithm': 'RSA-PSS-SHA256'
    }
    
    return manifest
```

## 8. The Quantum Threat to Digital Signatures

<Warning>
**All widely-deployed signature schemes are broken by quantum computers!**
</Warning>

### Shor's Algorithm Impact on Signatures

| Signature Scheme | Based On | Quantum Status |
|-----------------|----------|----------------|
| RSA | Factoring | **BROKEN** |
| DSA | DLP | **BROKEN** |
| ECDSA | ECDLP | **BROKEN** |
| Ed25519 | ECDLP | **BROKEN** |

<Example>
**Why Signature Vulnerability is Critical**

Unlike encryption (where old messages are at risk), signature vulnerabilities are:

1. **Immediate**: Once quantum computers exist, forgery is instant
2. **Complete**: Any message can be signed with any key
3. **Retroactive**: Old signed software/documents can't be trusted
4. **Cascading**: Certificate hierarchies collapse
