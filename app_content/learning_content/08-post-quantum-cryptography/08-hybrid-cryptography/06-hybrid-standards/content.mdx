def nist_guidance():

    print("""

    │ "NIST recommends the use of hybrid key establishment           │
    │  schemes during the transition period to post-quantum          │
    │  cryptography. This provides protection in case either         │
    │  the classical or post-quantum component is compromised."      │
    │                                                                 │
    │                    - NIST PQC Migration Guidance               │
    └─────────────────────────────────────────────────────────────────┘
    
    
    SP 800-227 (Forthcoming): PQC Transition Guidelines
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    • Key derivation requirements
    • Testing and validation requirements
    • Migration timeline recommendations
    
    
    NIST RECOMMENDED COMBINATIONS:
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    ┌─────────────────────────────────────────────────────────────────┐
    │ Use Case            │ Classical     │ Post-Quantum             │
    ├─────────────────────┼───────────────┼──────────────────────────┤
    │ Key Encapsulation   │ X25519        │ ML-KEM-768               │
    │ Key Encapsulation   │ P-256 ECDH    │ ML-KEM-768               │
    │ Key Encapsulation   │ X448          │ ML-KEM-1024              │
    │ Digital Signatures  │ Ed25519       │ ML-DSA-65                │
    │ Digital Signatures  │ ECDSA P-256   │ ML-DSA-65                │
    │ High Security       │ P-384 ECDH    │ ML-KEM-1024              │
    └─────────────────────────────────────────────────────────────────┘
    
    
    KEY NIST REQUIREMENTS:
    ━━━━━━━━━━━━━━━━━━━━━━━
    
    1. PROPER KEY COMBINATION
       • Use approved KDF (HKDF per SP 800-56C)
       • Include both components in derivation
       • Add domain separation/context
    
    2. BOTH COMPONENTS SECURE
       • Each component must meet security requirements
       • Hybrid security ≥ max(classical, PQC)
    
    3. INDEPENDENT FAILURES
       • Components should fail independently
       • No shared secrets or keys
    """)

nist_guidance()

## Summary

Hybrid cryptography standards landscape:

1. **NIST**: Recommends hybrid during transition, SP 800-227 coming
2. **IETF**: TLS hybrid drafts progressing, code points assigned
3. **Government**: NSA, NCSC, BSI all support hybrid
4. **Industry**: Financial, healthcare, telecom developing guidance
5. **Future**: Build agility, track standards, plan for updates

Standards are maturing rapidly—stay engaged and plan for evolution.
