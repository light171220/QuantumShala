    ┌─────────────────────────────────────────────────────────────────┐
    │ • Symmetric crypto (AES) - quantum resistant with 2x key size  │
    │ • Asymmetric crypto (RSA, ECC) - BROKEN by Shor's algorithm    │
    │ • Hash functions - quantum resistant with 2x output size       │
    │ • Digital signatures - need post-quantum replacements          │
    └─────────────────────────────────────────────────────────────────┘
    
    MODULE 2: THE QUANTUM THREAT
    ┌─────────────────────────────────────────────────────────────────┐
    │ • Shor's algorithm: Breaks RSA, ECC in polynomial time         │
    │ • Grover's algorithm: Provides quadratic speedup for search    │
    │ • Harvest Now, Decrypt Later: Data at risk TODAY               │
    │ • Timeline: CRQC expected 2030-2050, but uncertainty high      │
    └─────────────────────────────────────────────────────────────────┘
    
    MODULE 3: LATTICE CRYPTOGRAPHY
    ┌─────────────────────────────────────────────────────────────────┐
    │ • Lattices: Discrete point grids in n-dimensional space        │
    │ • Hard problems: SVP, CVP, LWE, SIS                            │
    │ • ML-KEM (Kyber): Primary key encapsulation standard           │
    │ • ML-DSA (Dilithium): Primary signature standard               │
    └─────────────────────────────────────────────────────────────────┘
    
    MODULE 4: HASH-BASED SIGNATURES
    ┌─────────────────────────────────────────────────────────────────┐
    │ • One-time signatures: Lamport, WOTS+                          │
    │ • Merkle trees: Enable many signatures from one key            │
    │ • XMSS: Stateful hash-based signatures (RFC 8391)              │
    │ • SLH-DSA (SPHINCS+): Stateless, conservative backup           │
    └─────────────────────────────────────────────────────────────────┘
    
    MODULE 5: CODE-BASED CRYPTOGRAPHY
    ┌─────────────────────────────────────────────────────────────────┐
    │ • Error-correcting codes: Linear, Goppa, QC-MDPC               │
    │ • McEliece: 45+ years of security, huge keys                   │
    │ • Classic McEliece: Maximum confidence option                  │
    │ • BIKE, HQC: Compact alternatives under evaluation             │
    └─────────────────────────────────────────────────────────────────┘
    
    MODULE 6: ISOGENY CRYPTOGRAPHY
    ┌─────────────────────────────────────────────────────────────────┐
    │ • Elliptic curves and isogenies                                │
    │ • SIDH/SIKE: Broken in 2022 by polynomial attack               │
    │ • CSIDH: Survives, but slow and needs large parameters         │
    │ • Lesson: Auxiliary data can enable unexpected attacks         │
    └─────────────────────────────────────────────────────────────────┘
    
    MODULE 7: NIST STANDARDS
    ┌─────────────────────────────────────────────────────────────────┐
    │ • FIPS 203: ML-KEM (key encapsulation)                         │
    │ • FIPS 204: ML-DSA (digital signatures)                        │
    │ • FIPS 205: SLH-DSA (hash-based signatures)                    │
    │ • Coming: FALCON, additional signatures                        │
    └─────────────────────────────────────────────────────────────────┘
    
    MODULE 8: HYBRID CRYPTOGRAPHY
    ┌─────────────────────────────────────────────────────────────────┐
