```python
def additional_signature_call():
    """NIST's 2023 call for additional signatures."""
    print("\nNIST ADDITIONAL SIGNATURE CALL (2023)")
    print("=" * 70)
    
    print("""
    WHY ADDITIONAL SIGNATURES?
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    ML-DSA signatures are 2-5 KB. Some applications need:
    • Shorter signatures (< 1 KB)
    • Faster verification
    • Different security assumptions (diversity)
    
    
    NOTABLE CANDIDATES:
    ┌─────────────────────────────────────────────────────────────────┐
    │ Scheme        │ Type         │ Sig Size  │ Key Feature         │
    ├───────────────┼──────────────┼───────────┼─────────────────────┤
    │ SQIsign       │ Isogeny      │ ~204 B    │ Extremely compact   │
    │ MAYO          │ Multivariate │ ~321 B    │ Short, fast verify  │
    │ UOV           │ Multivariate │ ~96 B     │ Very short sigs     │
    │ LESS          │ Code-based   │ ~6 KB     │ Code diversity      │
    │ CROSS         │ Code-based   │ ~10 KB    │ Novel approach      │
    │ HAWK          │ Lattice      │ ~555 B    │ Compact lattice     │
    │ SPHINCS-α     │ Hash-based   │ ~6 KB     │ Improved SPHINCS+   │
    └─────────────────────────────────────────────────────────────────┘
    
    
    SQISIGN: THE COMPACT CHAMPION
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    SQIsign (Short Quaternion and Isogeny Signature):
    • Based on isogenies (like SIDH, but different construction)
    • Signature: ~200 bytes (smallest PQC signature!)
    • Public key: ~64 bytes
    • NOT affected by SIDH attack
    
    Tradeoffs:
    • Slow signing (~1 second)
    • Complex implementation
    • Newer cryptanalysis
    
    
    MAYO AND UOV: MULTIVARIATE REVIVAL
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    After Rainbow's failure, multivariate is back:
    
    MAYO:
    • Oil-and-Vinegar variant
    • ~321 byte signatures
    • Fast verification
    • Large public keys (~1-2 KB)
    
    UOV (Unbalanced Oil and Vinegar):
    • Classic scheme, carefully parameterized
    • ~96 byte signatures (very short!)
    • Very large public keys (~66 KB)
    """)

additional_signature_call()
```

    ┌─────────────────────────────────────────────────────────────────┐
    │ • QC-MDPC codes (quasi-cyclic)                                 │
    │ • Compact: ~1.5 KB public keys                                 │
    │ • Concern: Decapsulation Failure Rate (DFR)                    │
    │ • Must ensure DFR < 2^-128                                     │
    │ • Status: Under continued analysis                             │
    └─────────────────────────────────────────────────────────────────┘
    
    
    HQC (Hamming Quasi-Cyclic):
    ┌─────────────────────────────────────────────────────────────────┐
    │ • Different code structure than BIKE                           │
    │ • ~2.2 KB public keys                                          │
    │ • Cleaner security analysis                                    │
    │ • Status: Under continued analysis                             │
    └─────────────────────────────────────────────────────────────────┘
    
    
    WHY KEEP EVALUATING?
    ━━━━━━━━━━━━━━━━━━━━━
    
    Diversity is crucial:
    • If lattice problems are broken, need alternatives
    • Code-based has different mathematical foundation
    • Classic McEliece is the most conservative option
    """)

round4_kems()

## Summary

The PQC landscape continues to evolve beyond the first NIST standards:

1. **Additional Signatures**: SQIsign, MAYO, HAWK offer shorter signatures
2. **Round 4 KEMs**: BIKE, HQC provide code-based alternatives
3. **Classic McEliece**: Maximum confidence with large keys
4. **Novel Constructions**: Ongoing research in all PQC families
5. **Timeline**: Additional standards expected 2025-2027

For production, use NIST standards. For future-proofing, design for cryptographic agility.
