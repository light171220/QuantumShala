## 2. Polynomial Rings: The Key to Efficiency

<Definition term="Polynomial Ring R_q">
The ring $R_q = \mathbb{Z}_q[x] / (x^n + 1)$ consists of polynomials with coefficients in $\mathbb{Z}_q$ modulo the polynomial $x^n + 1$.

$$R_q = \{a_0 + a_1 x + \cdots + a_{n-1} x^{n-1} : a_i \in \mathbb{Z}_q\}$$

with addition and multiplication modulo both $q$ and $x^n + 1$.
</Definition>

```python
import numpy as np
from typing import List, Tuple

class PolynomialRing:
    """
    Polynomial ring R_q = Z_q[x] / (x^n + 1).
    
    Elements are polynomials of degree < n with coefficients in Z_q.
    Multiplication is modulo x^n + 1 (negacyclic convolution).
    """
    
    def __init__(self, n: int, q: int):
        """
        Initialize polynomial ring parameters.
        
        Args:
            n: Degree of modulus polynomial (must be power of 2)
            q: Coefficient modulus
        """
        if n & (n - 1) != 0:
            raise ValueError("n must be a power of 2")
        
        self.n = n
        self.q = q
    
    def add(self, a: np.ndarray, b: np.ndarray) -> np.ndarray:
        """Add two polynomials coefficient-wise mod q."""
        return (a + b) % self.q
    
    def subtract(self, a: np.ndarray, b: np.ndarray) -> np.ndarray:
        """Subtract polynomials coefficient-wise mod q."""
        return (a - b) % self.q
    
    def multiply_naive(self, a: np.ndarray, b: np.ndarray) -> np.ndarray:
        """
        Multiply polynomials using schoolbook algorithm.
        
        This is O(n²) - we'll optimize later with NTT.
        """
        n = self.n
        q = self.q
        
        # Full convolution (degree up to 2n-2)
        result = np.zeros(2 * n - 1, dtype=np.int64)
        
        for i in range(n):
            for j in range(n):
                result[i + j] = (result[i + j] + a[i] * b[j]) % q
        
        # Reduce modulo x^n + 1
        # x^n ≡ -1 (mod x^n + 1), so x^(n+k) ≡ -x^k
        reduced = np.zeros(n, dtype=np.int64)
        for i in range(2 * n - 1):
            if i < n:
                reduced[i] = (reduced[i] + result[i]) % q
            else:
                # x^i = x^(i mod n) * (-1)^(i // n)
                reduced[i - n] = (reduced[i - n] - result[i]) % q
        
        return reduced
    
    def sample_uniform(self) -> np.ndarray:
        """Sample a uniformly random polynomial."""
        return np.random.randint(0, self.q, size=self.n)
    
    def sample_small(self, bound: int = 1) -> np.ndarray:
        """Sample a polynomial with small coefficients in [-bound, bound]."""
        coeffs = np.random.randint(-bound, bound + 1, size=self.n)
        return coeffs % self.q
    
    def sample_gaussian(self, sigma: float) -> np.ndarray:
        """Sample polynomial with Gaussian coefficients."""
        coeffs = np.round(np.random.normal(0, sigma, size=self.n)).astype(int)
        return coeffs % self.q


def demonstrate_polynomial_ring():
```

### Why $x^n + 1$?

<Formula>
$$\text{The polynomial } x^n + 1 \text{ (for } n = 2^k \text{) is:}$$
$$\text{1. Irreducible over } \mathbb{Q} \text{ (enables security reductions)}$$
$$\text{2. Splits nicely mod } q \text{ for certain primes (enables fast NTT)}$$
$$\text{3. Creates negacyclic structure (multiplication wraps with negation)}$$
</Formula>

## 3. The Ring-LWE Problem

<Definition term="Ring-LWE">
Given samples $(a_i, b_i = a_i \cdot s + e_i) \in R_q \times R_q$ where:
- $a_i$ are uniformly random in $R_q$
- $s \in R_q$ is the secret polynomial
- $e_i$ are small error polynomials from distribution $\chi$

The Ring-LWE problem asks to recover $s$.
</Definition>

```python
class RingLWE:
    """
    Ring-LWE problem instance.
    
    Ring-LWE: Given (a, b = a·s + e) in R_q × R_q, find s.
    """
    
    def __init__(self, n: int, q: int, sigma: float):
        """
        Initialize Ring-LWE parameters.
        
        Args:
            n: Ring dimension (polynomial degree)
            q: Modulus
            sigma: Error standard deviation
        """
        self.n = n
        self.q = q
        self.sigma = sigma
        self.ring = PolynomialRing(n, q)
    
    def keygen(self) -> Tuple[Tuple[np.ndarray, np.ndarray], np.ndarray]:
        """
        Generate Ring-LWE public/secret key pair.
        
        Returns:
            public_key: (a, b) where b = a·s + e
            secret_key: s
        """
        # Secret polynomial with small coefficients
        s = self.ring.sample_small(bound=1)
        
        # Uniform random a
        a = self.ring.sample_uniform()
        
        # Small error
        e = self.ring.sample_gaussian(self.sigma)
        
        # b = a·s + e
        b = self.ring.add(
            self.ring.multiply_naive(a, s),
            e
        )
        
        return (a, b), s
    
    def encrypt_polynomial(self, pk: Tuple[np.ndarray, np.ndarray],
                          message: np.ndarray) -> Tuple[np.ndarray, np.ndarray]:
        """
        Encrypt a message polynomial.
        
        Encoding: message coefficients in {0, 1} scaled to {0, q/2}.
        
        Returns:
            ciphertext: (u, v)
        """
        a, b = pk
        
        # Ephemeral secrets
        r = self.ring.sample_small(bound=1)
        e1 = self.ring.sample_gaussian(self.sigma)
        e2 = self.ring.sample_gaussian(self.sigma)
        
        # u = a·r + e1
        u = self.ring.add(
            self.ring.multiply_naive(a, r),
            e1
        )
        
        # v = b·r + e2 + encode(message)
        br = self.ring.multiply_naive(b, r)
        encoded_msg = (message * (self.q // 2)) % self.q
        v = self.ring.add(self.ring.add(br, e2), encoded_msg)
        
        return (u, v)
    
    def decrypt_polynomial(self, sk: np.ndarray,
                          ct: Tuple[np.ndarray, np.ndarray]) -> np.ndarray:
        """
        Decrypt ciphertext to recover message polynomial.
        """
        s = sk
        u, v = ct
        
        # Compute v - u·s
        us = self.ring.multiply_naive(u, s)
        diff = self.ring.subtract(v, us)
        
        # Decode: round to 0 or q/2
        message = np.zeros(self.n, dtype=int)
        for i in range(self.n):
            coeff = diff[i]
            # Center the coefficient
            if coeff > self.q // 2:
                coeff -= self.q
            # Decode
            if abs(coeff) < self.q // 4:
                message[i] = 0
            else:
                message[i] = 1
        
        return message


def demonstrate_ring_lwe():
    """

    """
    print("\nRING-LWE ENCRYPTION")

    # Parameters (small for demonstration)
```

## 4. The Number Theoretic Transform (NTT)

The NTT enables O(n log n) polynomial multiplication instead of O(n²).

<Definition term="Number Theoretic Transform (NTT)">
For a prime $q \equiv 1 \pmod{2n}$ with primitive $2n$-th root of unity $\omega$, the NTT of polynomial $a(x)$ is:

$$\hat{a}_i = \sum_{j=0}^{n-1} a_j \cdot \omega^{ij} \pmod{q}$$

Multiplication in NTT domain is pointwise: $\widehat{a \cdot b} = \hat{a} \odot \hat{b}$.
</Definition>

```python
class NTTPolynomialRing:
    """
    Polynomial ring with NTT-accelerated multiplication.
    """
    
    def __init__(self, n: int, q: int):
        """
        Initialize with NTT-friendly parameters.
        
        Requirements: q ≡ 1 (mod 2n) for 2n-th roots of unity.
        """
        self.n = n
        self.q = q
        
        # Find primitive 2n-th root of unity
        self.omega = self._find_primitive_root(2 * n)
        self.omega_inv = pow(self.omega, -1, q)
        
        # Precompute twiddle factors
        self.psi = pow(self.omega, (q - 1) // (2 * n), q)  # 2n-th root for negacyclic
        self.psi_inv = pow(self.psi, -1, q)
        
        self.n_inv = pow(n, -1, q)
    
    def _find_primitive_root(self, order: int) -> int:
        """Find a primitive root of given order mod q."""
        # For q prime, g^((q-1)/order) is a primitive order-th root
        # where g is a primitive root mod q
        
        # Find primitive root mod q
        for g in range(2, self.q):
            if pow(g, (self.q - 1) // 2, self.q) != 1:
                # g is likely a primitive root
                root = pow(g, (self.q - 1) // order, self.q)
                if pow(root, order, self.q) == 1 and pow(root, order // 2, self.q) != 1:
                    return root
        raise ValueError(f"Could not find primitive {order}-th root mod {self.q}")
    
    def ntt(self, a: np.ndarray) -> np.ndarray:
        """
        Compute NTT of polynomial a.
        
        Uses Cooley-Tukey butterfly algorithm.
        """
        a = np.array(a, dtype=np.int64)
        n = self.n
        q = self.q
        
        # Bit-reversal permutation
        result = self._bit_reverse(a)
        
        # Iterative NTT
        length = 2
        while length <= n:
            wlen = pow(self.omega, n // length, q)
            
            for i in range(0, n, length):
                w = 1
                for j in range(length // 2):
                    u = result[i + j]
                    v = (result[i + j + length // 2] * w) % q
                    result[i + j] = (u + v) % q
                    result[i + j + length // 2] = (u - v) % q
                    w = (w * wlen) % q
            
            length *= 2
        
        return result
    
    def intt(self, a: np.ndarray) -> np.ndarray:
        """
        Compute inverse NTT.
        """
        a = np.array(a, dtype=np.int64)
        n = self.n
        q = self.q
        
        result = self._bit_reverse(a)
        
        length = 2
        while length <= n:
            wlen = pow(self.omega_inv, n // length, q)
            
            for i in range(0, n, length):
                w = 1
                for j in range(length // 2):
                    u = result[i + j]
                    v = (result[i + j + length // 2] * w) % q
                    result[i + j] = (u + v) % q
                    result[i + j + length // 2] = (u - v) % q
                    w = (w * wlen) % q
            
            length *= 2
        
        # Scale by 1/n
        result = (result * self.n_inv) % q
        return result
    
    def _bit_reverse(self, a: np.ndarray) -> np.ndarray:
        """Bit-reversal permutation."""
        n = len(a)
        bits = int(np.log2(n))
        result = np.zeros_like(a)
        for i in range(n):
            rev = int('{:0{w}b}'.format(i, w=bits)[::-1], 2)
            result[rev] = a[i]
        return result
    
    def multiply_ntt(self, a: np.ndarray, b: np.ndarray) -> np.ndarray:
        """
        Multiply polynomials using NTT.
        
        O(n log n) instead of O(n²)!
        """
        # Transform to NTT domain
        a_ntt = self.ntt(a)
        b_ntt = self.ntt(b)
        
        # Pointwise multiplication
        c_ntt = (a_ntt * b_ntt) % self.q
        
        # Transform back
        c = self.intt(c_ntt)
        
        return c
```

## 5. Module-LWE: The Best of Both Worlds

<Definition term="Module-LWE">
Module-LWE generalizes both standard LWE and Ring-LWE by working over vectors of ring elements:

$$\mathbf{b} = \mathbf{A} \cdot \mathbf{s} + \mathbf{e} \in R_q^k$$

where $\mathbf{A} \in R_q^{k \times k}$, $\mathbf{s}, \mathbf{e} \in R_q^k$.
</Definition>

```python
class ModuleLWE:
    """
    Module-LWE: vectors of ring elements.
    
    This is what Kyber uses! It provides:
    - Better security than Ring-LWE (no algebraic structure attacks)
    - Better efficiency than standard LWE (ring structure)
    - Flexible security levels by varying k
    """
    
    def __init__(self, n: int, k: int, q: int, sigma: float):
        """
        Initialize Module-LWE parameters.
        
        Args:
            n: Ring dimension
            k: Module rank (vector dimension)
            q: Modulus
            sigma: Error parameter
        """
        self.n = n
        self.k = k
        self.q = q
        self.sigma = sigma
        self.ring = PolynomialRing(n, q)
    
    def sample_matrix(self) -> List[List[np.ndarray]]:
        """Sample uniform random k×k matrix of polynomials."""
        return [[self.ring.sample_uniform() for _ in range(self.k)] 
                for _ in range(self.k)]
    
    def sample_vector(self, small: bool = False) -> List[np.ndarray]:
        """Sample vector of k polynomials."""
        if small:
            return [self.ring.sample_small(bound=1) for _ in range(self.k)]
        return [self.ring.sample_uniform() for _ in range(self.k)]
    
    def sample_error_vector(self) -> List[np.ndarray]:
        """Sample vector of k error polynomials."""
        return [self.ring.sample_gaussian(self.sigma) for _ in range(self.k)]
    
    def matrix_vector_multiply(self, A: List[List[np.ndarray]], 
                               s: List[np.ndarray]) -> List[np.ndarray]:
        """
        Compute A · s where A is k×k matrix and s is k-vector.
        """
        result = []
        for i in range(self.k):
            # Sum of A[i][j] * s[j] for all j
            acc = np.zeros(self.n, dtype=np.int64)
            for j in range(self.k):
                prod = self.ring.multiply_naive(A[i][j], s[j])
                acc = self.ring.add(acc, prod)
            result.append(acc)
        return result
    
    def vector_add(self, a: List[np.ndarray], 
                   b: List[np.ndarray]) -> List[np.ndarray]:
        """Add two vectors of polynomials."""
        return [self.ring.add(a[i], b[i]) for i in range(self.k)]
    
    def keygen(self):
        """
        Generate Module-LWE key pair.
        
        Public key: (A, t = As + e)
        Secret key: s
        """
        A = self.sample_matrix()
        s = self.sample_vector(small=True)
        e = self.sample_error_vector()
        
        As = self.matrix_vector_multiply(A, s)
        t = self.vector_add(As, e)
        
        return (A, t), s
```

## Summary

In this lesson, you learned about structured LWE variants:

1. **Standard LWE** is secure but inefficient: O(n²) keys
2. **Polynomial rings** $R_q = \mathbb{Z}_q[x]/(x^n+1)$ enable compact representation
3. **Ring-LWE** works in $R_q$: O(n) keys, O(n log n) operations via NTT
4. **NTT** transforms polynomial multiplication to pointwise operations
5. **Module-LWE** balances security (vectors of ring elements) and efficiency
6. **Security hierarchy**: Standard > Module > Ring (for same parameters)
7. **Kyber** uses Module-LWE with specific optimizations
8. **Parameter choice**: k=2,3,4 gives 128,192,256-bit security

Understanding these structures is essential for Kyber and modern lattice cryptography.
