## 1. The Appeal of Hash-Based Signatures

<Definition term="Hash-Based Signatures">
Digital signature schemes whose security relies solely on the properties of cryptographic hash functions—specifically collision resistance, preimage resistance, and second-preimage resistance.
</Definition>

## 2. Required Hash Function Properties

<Definition term="Preimage Resistance">
Given $h$, it is computationally infeasible to find any $m$ such that $H(m) = h$.

**Security level**: For $n$-bit hash, requires $O(2^n)$ operations.
</Definition>

<Definition term="Second-Preimage Resistance">
Given $m_1$, it is computationally infeasible to find $m_2 \neq m_1$ such that $H(m_1) = H(m_2)$.

**Security level**: For $n$-bit hash, requires $O(2^n)$ operations.
</Definition>

<Definition term="Collision Resistance">
It is computationally infeasible to find any pair $(m_1, m_2)$ with $m_1 \neq m_2$ such that $H(m_1) = H(m_2)$.

**Security level**: For $n$-bit hash, requires $O(2^{n/2})$ operations (birthday bound).
</Definition>

```python
def demonstrate_hash_properties():
    """
    Demonstrate and analyze hash function security properties.
    """
    print("\nHASH FUNCTION SECURITY PROPERTIES")
    print("=" * 65)
    
    # Using SHA-256 as example
    def sha256(data: bytes) -> bytes:
        return hashlib.sha256(data).digest()
    
    # Property 1: Deterministic
    m = b"Hello, Post-Quantum World!"
    h1 = sha256(m)
    h2 = sha256(m)
    print(f"1. DETERMINISTIC")
    print(f"   Same input always produces same output: {h1 == h2}")
    
    # Property 2: Fixed output size
    messages = [b"a", b"a" * 100, b"a" * 10000]
    print(f"\n2. FIXED OUTPUT SIZE")
    for msg in messages:
        h = sha256(msg)
        print(f"   Input: {len(msg)} bytes → Output: {len(h)} bytes ({len(h)*8} bits)")
    
    # Property 3: Avalanche effect (small change → big difference)
    m1 = b"Message version 1"
    m2 = b"Message version 2"
    h1 = sha256(m1)
    h2 = sha256(m2)
    
    # Count differing bits
    diff_bits = sum(bin(a ^ b).count('1') for a, b in zip(h1, h2))
    
    print(f"\n3. AVALANCHE EFFECT")
    print(f"   Input 1: {m1}")
    print(f"   Input 2: {m2}")
    print(f"   Bits changed in hash: {diff_bits}/256 ({100*diff_bits/256:.1f}%)")
    print(f"   Expected for random: ~128 bits (50%)")
    
    # Security levels
    print(f"\n4. SECURITY LEVELS FOR SHA-256")
    print(f"   Preimage resistance:        2^256 operations")
    print(f"   Second-preimage resistance: 2^256 operations")
    print(f"   Collision resistance:       2^128 operations (birthday)")
    
    # Quantum security
    print(f"\n5. QUANTUM SECURITY (Grover's Algorithm)")
    print(f"   Classical preimage:  2^256 → Quantum: 2^128")
    print(f"   Classical collision: 2^128 → Quantum: 2^85 (BHT algorithm)")
    print(f"   Recommendation: Use SHA-384 or SHA-512 for post-quantum")

demonstrate_hash_properties()
```

## 3. The Basic Idea: One-Time Signatures

The foundation of all hash-based signatures is the one-time signature (OTS).

<Formula>
$$\text{Core Insight: Hash functions can create asymmetric key pairs}$$
$$\text{Secret Key: } sk = \text{random value}$$
$$\text{Public Key: } pk = H(sk)$$
$$\text{Signature reveals } sk \text{, so can only sign once!}$$
</Formula>

```python
class SimpleOneTimeSignature:
    """
    Simplified one-time signature to illustrate the concept.
    
    WARNING: This is for education only - not secure for real use!
    """
    
    def __init__(self, hash_bytes: int = 32):
        self.hash_bytes = hash_bytes
        self.hash_bits = hash_bytes * 8
    
    def _hash(self, data: bytes) -> bytes:
        """Hash function wrapper."""
        return hashlib.sha256(data).digest()[:self.hash_bytes]
    
    def keygen(self) -> Tuple[List[Tuple[bytes, bytes]], List[Tuple[bytes, bytes]]]:
        """
        Generate one-time key pair.
        
        For each bit position, generate two random values.
        Secret key: the random values
        Public key: hashes of the random values
        
        Returns:
            sk: List of (sk_0, sk_1) for each bit
            pk: List of (pk_0, pk_1) = (H(sk_0), H(sk_1)) for each bit
        """
        sk = []
        pk = []
        
        for _ in range(self.hash_bits):
            # Two secret values per bit position
            sk_0 = secrets.token_bytes(self.hash_bytes)
            sk_1 = secrets.token_bytes(self.hash_bytes)
            
            # Public key is hash of secrets
            pk_0 = self._hash(sk_0)
            pk_1 = self._hash(sk_1)
            
            sk.append((sk_0, sk_1))
            pk.append((pk_0, pk_1))
        
        return sk, pk
    
    def sign(self, sk: List[Tuple[bytes, bytes]], message: bytes) -> List[bytes]:
        """
        Sign a message (ONE TIME ONLY!).
        
        Hash the message, then for each bit:
        - If bit is 0: reveal sk_0
        - If bit is 1: reveal sk_1
        """
        # Hash message to fixed length
        msg_hash = self._hash(message)
        
        signature = []
        for i, byte in enumerate(msg_hash):
            for j in range(8):
                bit_index = i * 8 + j
                if bit_index >= len(sk):
                    break
                
                bit = (byte >> (7 - j)) & 1
                
                # Reveal the secret corresponding to this bit
                if bit == 0:
                    signature.append(sk[bit_index][0])
                else:
                    signature.append(sk[bit_index][1])
        
        return signature
    
    def verify(self, pk: List[Tuple[bytes, bytes]], message: bytes, 
               signature: List[bytes]) -> bool:
        """
        Verify a signature.
        
        For each bit of message hash:
        - If bit is 0: check H(sig[i]) == pk_0
        - If bit is 1: check H(sig[i]) == pk_1
        """
        msg_hash = self._hash(message)
        
        for i, byte in enumerate(msg_hash):
            for j in range(8):
                bit_index = i * 8 + j
                if bit_index >= len(signature):
                    break
                
                bit = (byte >> (7 - j)) & 1
                
                # Check hash of revealed secret matches public key
                revealed = signature[bit_index]
                expected_pk = pk[bit_index][bit]
                
                if self._hash(revealed) != expected_pk:
                    return False
        
        return True


def demonstrate_one_time_signature():
    """
    Demonstrate one-time signature concept.
    """
    print("\nONE-TIME SIGNATURE DEMONSTRATION")
    print("=" * 65)
    
    # Use small hash for demonstration
    ots = SimpleOneTimeSignature(hash_bytes=4)  # 32-bit for demo
    
    # Key generation
    print("1. KEY GENERATION")
    sk, pk = ots.keygen()
    print(f"   Secret key: {len(sk)} pairs of {len(sk[0][0])}-byte values")
    print(f"   Public key: {len(pk)} pairs of {len(pk[0][0])}-byte hashes")
    print(f"   SK size: {len(sk) * 2 * len(sk[0][0])} bytes")
    print(f"   PK size: {len(pk) * 2 * len(pk[0][0])} bytes")
    
    # Sign a message
    print("\n2. SIGNING")
    message = b"Transfer $1000 to Alice"
    signature = ots.sign(sk, message)
    print(f"   Message: {message}")
    print(f"   Signature: {len(signature)} revealed secrets")
    print(f"   Signature size: {len(signature) * len(signature[0])} bytes")
    
    # Verify
    print("\n3. VERIFICATION")
    valid = ots.verify(pk, message, signature)
    print(f"   Signature valid: {valid}")
    
    # Test with wrong message
    wrong_message = b"Transfer $9999 to Eve"
    valid_wrong = ots.verify(pk, wrong_message, signature)
    print(f"   Wrong message verification: {valid_wrong}")
    
    # Demonstrate the "one-time" problem
    print("\n4. THE ONE-TIME PROBLEM")
    print("   After signing, half the secret key is revealed!")
    print("   If we sign a second message, attacker learns more secrets")
    print("   Eventually: attacker can forge signatures on new messages")

demonstrate_one_time_signature()
```

## 4. Why One-Time Signatures Are Secure

```python
def analyze_ots_security():
    """
    Analyze the security of one-time signatures.
    """
    print("\nONE-TIME SIGNATURE SECURITY ANALYSIS")
    print("=" * 65)
    
    analysis = """
    SECURITY ARGUMENT:
    
    To forge a signature on message M', attacker needs secret values
    that were NOT revealed in the legitimate signature on M.
    
    Case Analysis:
    ┌─────────────────────────────────────────────────────────────────┐
    │ Bit position i │ M[i] │ M'[i] │ Revealed │ Needed │ Attack?    │
    ├─────────────────────────────────────────────────────────────────┤
    │       j        │   0  │   0   │   sk_0   │  sk_0  │ Have it    │
    │       j        │   0  │   1   │   sk_0   │  sk_1  │ Need to    │
    │                │      │       │          │        │ find sk_1  │
    │                │      │       │          │        │ from pk_1  │
    │       j        │   1  │   0   │   sk_1   │  sk_0  │ Need to    │
    │                │      │       │          │        │ find sk_0  │
    │                │      │       │          │        │ from pk_0  │
    │       j        │   1  │   1   │   sk_1   │  sk_1  │ Have it    │
    └─────────────────────────────────────────────────────────────────┘
    
    For M' ≠ M: At least one bit differs.
    For that bit: Attacker needs to find preimage of hash.
    Preimage finding: Requires 2^n operations for n-bit hash.
    
    THEREFORE: OTS is secure as long as hash function is preimage-resistant!
    """
    print(analysis)
    
    # Quantitative analysis
    print("\nSECURITY LEVELS:")
    
    hash_sizes = [128, 192, 256, 384, 512]
    print(f"\n{'Hash bits':<12} {'Classical':<15} {'Quantum (Grover)':<18}")
    print("-" * 50)
    
    for bits in hash_sizes:
        classical = f"2^{bits}"
        quantum = f"2^{bits//2}"
        print(f"{bits:<12} {classical:<15} {quantum:<18}")
    
    print("\nFor 128-bit post-quantum security: Use 256-bit hash output")

analyze_ots_security()
```

## 5. The Key Size Problem

One-time signatures have a fundamental efficiency problem:

```python
def analyze_ots_sizes():
    """
    Analyze the size overhead of one-time signatures.
    """
    print("\nONE-TIME SIGNATURE SIZE ANALYSIS")
    print("=" * 65)
    
    # For n-bit security with Lamport OTS
    security_bits = [128, 192, 256]
    
    print("LAMPORT ONE-TIME SIGNATURE SIZES")
    print("(Simplest OTS: one pair per message bit)")
    print()
    
    print(f"{'Security':<12} {'Hash':<10} {'SK':<12} {'PK':<12} {'Sig':<12}")
    print("-" * 60)
    
    for sec in security_bits:
        hash_bits = 2 * sec  # For quantum security
        hash_bytes = hash_bits // 8
        msg_bits = hash_bits  # Sign hash of message
        
        # Lamport: 2 secrets per bit, each hash_bytes long
        sk_bytes = 2 * msg_bits * hash_bytes
        pk_bytes = 2 * msg_bits * hash_bytes
        sig_bytes = msg_bits * hash_bytes
        
        print(f"{sec}-bit      {hash_bits}-bit    "
              f"{sk_bytes//1024:.1f} KB      {pk_bytes//1024:.1f} KB      "
              f"{sig_bytes//1024:.1f} KB")
    
    print("\n❌ PROBLEM: Keys and signatures are HUGE!")
    print("   Compare to RSA-2048: SK=1KB, PK=0.25KB, Sig=0.25KB")
    print("   Compare to ECDSA-256: SK=32B, PK=64B, Sig=64B")
    
    print("\n✓ SOLUTION: Winternitz OTS (next lesson)")
    print("   Trade computation for size")
    print("   Reduce signature from n·hash_bytes to (n/w)·hash_bytes")

analyze_ots_sizes()
```

## 6. From One-Time to Many-Time Signatures

The challenge: How to sign multiple messages with one key pair?

```python
def explain_many_time_challenge():
    """
    Explain the challenge of creating many-time signatures from OTS.
    """
    print("\nFROM ONE-TIME TO MANY-TIME SIGNATURES")
    print("=" * 65)
    
    explanation = """
    THE CHALLENGE:
    
    One-Time Signature (OTS):
    ┌────────────────────────────────────────────────────────────────┐
    │  • Generate fresh key pair for EVERY signature                 │
    │  • After signing once, key is "used up"                        │
    │  • Reusing key allows forgery                                  │
    └────────────────────────────────────────────────────────────────┘
    
    We want: Sign millions of messages with ONE public key
    
    SOLUTION 1: Merkle Trees (Merkle, 1979)
    ┌────────────────────────────────────────────────────────────────┐
    │  • Pre-generate 2^h OTS key pairs                              │
    │  • Build binary tree of hashes over all public keys            │
    │  • Root hash = single public key                               │
    │  • Signature includes OTS signature + authentication path      │
    │  • STATE: Must track which OTS keys are used                   │
    └────────────────────────────────────────────────────────────────┘
    
    SOLUTION 2: SPHINCS+ (Bernstein et al., 2015)
    ┌────────────────────────────────────────────────────────────────┐
    │  • Stateless: No need to track used keys                       │
    │  • Use randomized message-dependent key selection              │
    │  • Multiple layers of trees (hypertrees)                       │
    │  • NIST standardized as SLH-DSA (FIPS 205)                     │
    └────────────────────────────────────────────────────────────────┘
    
    TRADEOFF SUMMARY:
    
    Scheme        │ State │ Sign Time │ Verify │ Signature Size
    ──────────────┼───────┼───────────┼────────┼───────────────
    XMSS          │ Yes   │ Fast      │ Fast   │ ~2.5 KB
    LMS           │ Yes   │ Fast      │ Fast   │ ~1-2 KB
    SPHINCS+      │ No    │ Slow      │ Fast   │ ~8-50 KB
    Dilithium     │ No    │ Fast      │ Fast   │ ~2.5-4.5 KB
    """
    print(explanation)

explain_many_time_challenge()
```

## Summary

In this lesson, you learned the fundamentals of hash-based signatures:

1. **Minimal Assumptions**: Security relies only on hash function properties
2. **Hash Properties**: Preimage, second-preimage, and collision resistance
3. **One-Time Signatures**: Reveal secret values corresponding to message bits
4. **Security Proof**: Forgery requires breaking preimage resistance
5. **Size Problem**: Basic OTS has very large keys and signatures
6. **Many-Time Solutions**: Merkle trees (stateful) or SPHINCS+ (stateless)
7. **Quantum Security**: Simply double hash output for post-quantum
8. **NIST Standards**: SLH-DSA (SPHINCS+), XMSS, LMS

Hash-based signatures offer the most conservative security foundation, making them ideal for long-term security applications.
