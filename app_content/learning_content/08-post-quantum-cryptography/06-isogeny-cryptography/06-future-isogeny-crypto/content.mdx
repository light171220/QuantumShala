## 2. SQIsign: Isogeny Signatures

```python
def sqisign_overview():
    """Overview of SQIsign signature scheme."""
    print("\nSQISIGN: SUPERSINGULAR QUATERNION ISOGENY SIGNATURES")
    print("=" * 70)
    
    overview = """
    WHAT IS SQIsign?
    ━━━━━━━━━━━━━━━━
    
    SQIsign (2020) is a signature scheme based on isogenies.
    
    Key insight: Use quaternion algebra structure of
                 supersingular curve endomorphisms
    
    NOT vulnerable to the SIDH attack because:
    • Different construction (not Diffie-Hellman style)
    • Uses endomorphism rings, not auxiliary points
    
    
    SQIsign PARAMETERS:
    ┌─────────────────────────────────────────────────────────────────────┐
    │ Security      │ Public Key  │ Signature   │ Combined              │
    ├─────────────────────────────────────────────────────────────────────┤
    │ NIST Level 1  │ 64 bytes    │ 177 bytes   │ 241 bytes             │
    │ NIST Level 3  │ 96 bytes    │ 263 bytes   │ 359 bytes             │
    │ NIST Level 5  │ 128 bytes   │ 335 bytes   │ 463 bytes             │
    └─────────────────────────────────────────────────────────────────────┘
    
    Compare to ML-DSA (Dilithium) Level 1:
    • Public Key: 1,312 bytes (vs SQIsign 64 bytes - 20× smaller!)
    • Signature: 2,420 bytes (vs SQIsign 177 bytes - 14× smaller!)
    
    SQIsign has the SMALLEST signatures of any PQC scheme!
    
    
    THE CATCH: SPEED
    ━━━━━━━━━━━━━━━━━
    
    ┌─────────────────────────────────────────────────────────────────────┐
    │ Operation     │ SQIsign (ms) │ ML-DSA (ms)  │ Ratio               │
    ├─────────────────────────────────────────────────────────────────────┤
    │ Key Gen       │ ~1000-5000   │ ~0.1         │ 10,000-50,000×      │
    │ Sign          │ ~200-1000    │ ~0.3         │ 700-3000×           │
    │ Verify        │ ~50-100      │ ~0.3         │ 150-300×            │
    └─────────────────────────────────────────────────────────────────────┘
    
    SQIsign is MUCH slower but MUCH smaller.
    
    
    HOW SQIsign WORKS (High Level):
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    1. SECRET KEY: An ideal in quaternion algebra (encodes isogeny)
    2. PUBLIC KEY: The codomain curve of the secret isogeny
    3. SIGNING: Fiat-Shamir with commitment using isogeny
    4. VERIFY: Check isogeny relations hold
    
    The math is deep: quaternion algebras, ideal classes, KLPT algorithm
    """
    print(overview)


def sqisign_use_cases():
    """Use cases for SQIsign."""
    print("\nSQISIGN USE CASES")
    print("-" * 50)
    
    use_cases = """
    WHERE SQIsign MAKES SENSE:
    
    ✓ Certificate chains (small certs, verification once)
    ✓ Firmware signing (sign once, verify occasionally)
    ✓ Long-term document signing
    ✓ Bandwidth-constrained environments
    ✓ Storage-constrained embedded systems
    
    WHERE SQIsign DOESN'T WORK:
    
    ✗ Real-time authentication
    ✗ High-volume signing (servers)
    ✗ Interactive protocols needing fast signing
    ✗ Blockchain (many signatures per block)
    """
    print(use_cases)

sqisign_overview()
sqisign_use_cases()
```
