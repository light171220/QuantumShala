## 2. How CSIDH Works

```python
def csidh_protocol():
    """Explain CSIDH protocol."""
    print("\nCSIDH PROTOCOL")
    print("=" * 70)
    
    protocol = """
    SETUP:
    ━━━━━━
    
    • Prime p ≡ 3 (mod 8), p = 4·ℓ₁·ℓ₂·...·ℓₙ - 1
    • Starting curve E₀: y² = x³ + x (supersingular over F_p)
    • Small primes ℓ₁, ℓ₂, ..., ℓₙ for isogeny degrees
    
    
    CLASS GROUP ACTION:
    ━━━━━━━━━━━━━━━━━━━
    
    For each small prime ℓᵢ, there's an ℓᵢ-isogeny:
    
    • E has two ℓᵢ-isogenies: one "forward" and one "backward"
    • Can compose these: [ℓᵢ]^{eᵢ} means apply eᵢ times
    • Action is COMMUTATIVE: order doesn't matter!
    
    
    SECRET KEY:
    ━━━━━━━━━━━
    
    Alice's secret: exponent vector (e₁, e₂, ..., eₙ)
    where eᵢ ∈ {-m, ..., -1, 0, 1, ..., m}
    
    Alice computes: E_A = [ℓ₁]^{e₁} ∘ [ℓ₂]^{e₂} ∘ ... ∘ [ℓₙ]^{eₙ} (E₀)
    
    
    KEY EXCHANGE:
    ┌─────────────────────────────────────────────────────────────────────┐
    │                                                                     │
    │  1. Alice: secret (a₁,...,aₙ), publishes E_A = action(E₀)         │
    │  2. Bob:   secret (b₁,...,bₙ), publishes E_B = action(E₀)         │
    │                                                                     │
    │  3. Alice computes: E_{AB} = action_A(E_B)                         │
    │  4. Bob computes:   E_{BA} = action_B(E_A)                         │
    │                                                                     │
    │  COMMUTATIVITY: E_{AB} = E_{BA} !                                  │
    │                                                                     │
    └─────────────────────────────────────────────────────────────────────┘
    
    No auxiliary points needed because actions COMMUTE!
    
    
    WHY IT'S COMMUTATIVE:
    ━━━━━━━━━━━━━━━━━━━━━
    
    The class group Cl(O) of the endomorphism ring acts on curves.
    
    Group actions are always "commutative" in the sense that:
    [a] ∘ [b] (E) = [b] ∘ [a] (E) = [a+b] (E)
    
    This is fundamentally different from SIDH's non-commutative setup.
    """
    print(protocol)


class CSIDHSimulator:
    """Simplified CSIDH simulator."""
    
    def __init__(self, primes: list, max_exp: int = 5):
        """
        Initialize CSIDH simulator.
        
        Args:
            primes: List of small primes for isogenies
            max_exp: Maximum absolute value of exponents
        """
        self.primes = primes
        self.n = len(primes)
        self.max_exp = max_exp
        self.j_0 = 1728  # Starting j-invariant (symbolic)
    
    def _class_group_action(self, j: int, exponents: list) -> int:
        """
        Simulate class group action.
        
        In reality, this involves computing isogenies.
        Here we just simulate with a hash-like function.
        """
        result = j
        for i, e in enumerate(exponents):
            # Simplified: add prime * exponent to j
            result = (result + self.primes[i] * e * 101) % 10000
        return result
    
    def keygen(self, secret_exponents: list) -> int:
        """Generate public key from secret exponents."""
        if len(secret_exponents) != self.n:
            raise ValueError(f"Need {self.n} exponents")
        
        return self._class_group_action(self.j_0, secret_exponents)
    
    def shared_secret(self, my_exponents: list, their_public: int) -> int:
        """Compute shared secret."""
        return self._class_group_action(their_public, my_exponents)


def demonstrate_csidh():
    """Demonstrate CSIDH key exchange."""
    print("\nCSIDH KEY EXCHANGE DEMONSTRATION")
    print("-" * 50)
    
    # Use small primes for demonstration
    primes = [3, 5, 7, 11, 13]
    csidh = CSIDHSimulator(primes)
    
    print(f"Primes for isogenies: {primes}")
    print(f"Starting j-invariant: {csidh.j_0}")
    
    # Alice and Bob generate secrets
    alice_secret = [2, -1, 3, 0, -2]  # Exponents
    bob_secret = [-1, 2, 1, -3, 1]
    
    print(f"\nAlice's secret exponents: {alice_secret}")
    print(f"Bob's secret exponents: {bob_secret}")
    
    # Public keys
    alice_public = csidh.keygen(alice_secret)
    bob_public = csidh.keygen(bob_secret)
    
    print(f"\nAlice's public key (j_A): {alice_public}")
    print(f"Bob's public key (j_B): {bob_public}")
    
    # Shared secrets
    alice_shared = csidh.shared_secret(alice_secret, bob_public)
    bob_shared = csidh.shared_secret(bob_secret, alice_public)
    
    print(f"\nAlice's computed shared: {alice_shared}")
    print(f"Bob's computed shared: {bob_shared}")
    print(f"\nShared secrets match: {alice_shared == bob_shared}")
    print("\n(Real CSIDH uses actual isogenies, this is a simulation)")

csidh_protocol()
demonstrate_csidh()
```

```python
def csidh_security():

    print("\nCSIDH SECURITY ANALYSIS")

    security = """

    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    The Castryck-Decru attack requires:
    ✗ Auxiliary torsion point images φ(P), φ(Q)
    ✗ Torsion of a specific structure (2^a and 3^b coprime)
    
    CSIDH provides:
    ✓ ONLY the codomain curve j-invariant
    ✓ NO auxiliary points
    
    The attack simply doesn't have the data it needs!
    
    
    CSIDH HARD PROBLEM:
    ━━━━━━━━━━━━━━━━━━━
    
    GROUP ACTION INVERSE PROBLEM (GAIP):
    
    Given: E₀, E_A (both supersingular over F_p)
    Find:  The class group element [a] such that E_A = [a](E₀)
    
    Best known attacks:
    • Classical: Subexponential (L_p[1/2])
    • Quantum: Subexponential (Kuperberg's algorithm)
    
    NOT polynomial time like the SIDH attack!
    
    
    QUANTUM SECURITY CONCERN:
    ┌─────────────────────────────────────────────────────────────────────┐
    │                                                                     │
    │ Kuperberg's algorithm solves hidden shift problem                  │
    │                                                                     │
    │ Expected complexity: 2^{O(√log p)}                                 │
    │                                                                     │
    │ This is SUBEXPONENTIAL, not polynomial                             │
    │ But worse than Grover's quadratic speedup                          │
    │                                                                     │
    │ CSIDH parameters must be LARGER to resist quantum                  │
    │                                                                     │
    └─────────────────────────────────────────────────────────────────────┘
    
    
    PARAMETER SIZES:
```

## Summary

In this lesson, you learned about CSIDH:

1. **Different Design**: Uses ordinary curves, no auxiliary points
2. **Class Group Action**: Commutative operation enables clean protocol
3. **SIDH Attack Immune**: No auxiliary data means attack doesn't apply
4. **Security**: Subexponential quantum (Kuperberg), not polynomial
5. **Performance**: Very slow but extremely small keys
6. **Use Cases**: Bandwidth-constrained, non-real-time applications

CSIDH represents the surviving branch of isogeny-based cryptography.
