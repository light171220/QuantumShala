```python
    Alice: φ_A: E₀ → E_A            Bob: φ_B: E₀ → E_B
```

## 2. SIDH Protocol Design

```python
def sidh_protocol():
    """Detailed SIDH protocol."""
    print("\nSIDH PROTOCOL DESIGN")
    print("=" * 70)
    
    protocol = """
    SETUP:
    ━━━━━━
    
    Parameters:
    • Prime p = 2^a · 3^b · f - 1 (where f is small cofactor)
    • Supersingular curve E₀ over F_p²
    • Bases: (P_A, Q_A) for E₀[2^a], (P_B, Q_B) for E₀[3^b]
    
    
    KEY GENERATION:
    ━━━━━━━━━━━━━━━
    
    Alice (works in 2-torsion):
    1. Choose random α, β ∈ Z_{2^a}
    2. Kernel K_A = <P_A + α·Q_A> (order 2^a subgroup)
    3. Compute φ_A: E₀ → E_A = E₀/K_A
    4. Compute auxiliary points φ_A(P_B), φ_A(Q_B)
    5. Publish (E_A, φ_A(P_B), φ_A(Q_B))
    
    Bob (works in 3-torsion):
    1. Choose random γ, δ ∈ Z_{3^b}  
    2. Kernel K_B = <P_B + γ·Q_B> (order 3^b subgroup)
    3. Compute φ_B: E₀ → E_B = E₀/K_B
    4. Compute auxiliary points φ_B(P_A), φ_B(Q_A)
    5. Publish (E_B, φ_B(P_A), φ_B(Q_A))
    
    
    SHARED SECRET:
    ━━━━━━━━━━━━━━
    
    Alice computes:
    1. Reconstruct kernel in E_B: K'_A = <φ_B(P_A) + α·φ_B(Q_A)>
    2. Compute φ'_A: E_B → E_{BA}
    3. Shared = j(E_{BA})
    
    Bob computes:
    1. Reconstruct kernel in E_A: K'_B = <φ_A(P_B) + γ·φ_A(Q_B)>  
    2. Compute φ'_B: E_A → E_{AB}
    3. Shared = j(E_{AB})
    
    
    WHY IT WORKS:
    ┌─────────────────────────────────────────────────────────────────────┐
    │                                                                     │
    │      E₀ ----φ_A----> E_A                                           │
    │       │               │                                            │
    │      φ_B            φ'_B                                           │
    │       │               │                                            │
    │       ↓               ↓                                            │
    │      E_B ----φ'_A---> E_{AB} = E_{BA}                              │
    │                                                                     │
    │   The square commutes: j(E_{AB}) = j(E_{BA})                       │
    └─────────────────────────────────────────────────────────────────────┘
    """
    print(protocol)


class SIDHSimulator:
    """
    Simplified SIDH simulator for educational purposes.
    
    Note: Real SIDH uses much more sophisticated arithmetic.
    """
    
    def __init__(self, a: int, b: int):
        """
        Initialize SIDH with 2^a and 3^b isogeny degrees.
        """
        self.a = a  # Alice's isogeny degree: 2^a
        self.b = b  # Bob's isogeny degree: 3^b
        
        # Simplified: represent curves by j-invariants
        # Real SIDH uses Montgomery curves over F_p²
        self.j_0 = 0  # Starting curve (placeholder)
    
    def alice_keygen(self, secret: int) -> Dict:
        """
        Alice's key generation.
        
        Returns public key containing:
        - j_A: j-invariant of E_A
        - aux: auxiliary torsion points (simulated)
        """
        # In reality: compute 2^a-isogeny from kernel
        # Simplified: just hash the secret
        j_A = (self.j_0 + secret * 17) % 1000
        
        # Auxiliary points would be φ_A(P_B), φ_A(Q_B)
        aux = (secret * 3 % 100, secret * 7 % 100)
        
        return {
            'j': j_A,
            'aux': aux,
            'secret': secret  # For demonstration
        }
    
    def bob_keygen(self, secret: int) -> Dict:
        """Bob's key generation."""
        j_B = (self.j_0 + secret * 23) % 1000
        aux = (secret * 11 % 100, secret * 13 % 100)
        
        return {
            'j': j_B,
            'aux': aux,
            'secret': secret
        }
    
    def alice_shared_secret(self, alice_secret: int, bob_public: Dict) -> int:
        """Alice computes shared secret."""
        # In reality: reconstruct kernel in E_B, compute isogeny
        # Simplified: combine secrets
        return (bob_public['j'] + alice_secret * 17) % 1000
    
    def bob_shared_secret(self, bob_secret: int, alice_public: Dict) -> int:
        """Bob computes shared secret."""
        return (alice_public['j'] + bob_secret * 23) % 1000


def demonstrate_sidh():
    """Demonstrate SIDH key exchange."""
    print("\nSIDH KEY EXCHANGE SIMULATION")
    print("-" * 50)
    
    sidh = SIDHSimulator(a=216, b=137)  # SIKE-p434 parameters
    
    # Key generation
    alice_secret = 42
    bob_secret = 73
    
    alice_pk = sidh.alice_keygen(alice_secret)
    bob_pk = sidh.bob_keygen(bob_secret)
    
    print(f"Alice's secret: {alice_secret}")
    print(f"Alice's public j-invariant: {alice_pk['j']}")
    print(f"Alice's auxiliary points: {alice_pk['aux']}")
    
    print(f"\nBob's secret: {bob_secret}")
    print(f"Bob's public j-invariant: {bob_pk['j']}")
    print(f"Bob's auxiliary points: {bob_pk['aux']}")
    
    # Shared secret computation
    alice_shared = sidh.alice_shared_secret(alice_secret, bob_pk)
    bob_shared = sidh.bob_shared_secret(bob_secret, alice_pk)
    
    print(f"\nAlice's computed shared secret: {alice_shared}")
    print(f"Bob's computed shared secret: {bob_shared}")
    print(f"\nNote: This is a simplified simulation. Real SIDH uses")
    print(f"sophisticated arithmetic over F_p² with p ≈ 2^434.")

sidh_protocol()
demonstrate_sidh()
```

## 3. SIKE: The KEM Version

```python
def sike_overview():
    """Overview of SIKE (Supersingular Isogeny Key Encapsulation)."""
    print("\nSIKE: SUPERSINGULAR ISOGENY KEY ENCAPSULATION")
    print("=" * 70)
    
    overview = """
    FROM SIDH TO SIKE:
    ━━━━━━━━━━━━━━━━━━
    
    SIDH is a key exchange (like Diffie-Hellman)
    SIKE wraps SIDH into a KEM using Fujisaki-Okamoto transform
    
    Why KEM?
    • IND-CCA2 security (active attacker resistance)
    • Better for TLS integration
    • Standard NIST KEM interface
    
    
    SIKE PARAMETER SETS (submitted to NIST):
    ┌─────────────────────────────────────────────────────────────────────┐
    │ Name      │ Prime p          │ 2^a    │ 3^b    │ Security         │
    ├─────────────────────────────────────────────────────────────────────┤
    │ SIKEp434  │ 2²¹⁶·3¹³⁷-1      │ 2²¹⁶   │ 3¹³⁷   │ NIST Level 1     │
    │ SIKEp503  │ 2²⁵⁰·3¹⁵⁹-1      │ 2²⁵⁰   │ 3¹⁵⁹   │ NIST Level 2     │
    │ SIKEp610  │ 2³⁰⁵·3¹⁹²-1      │ 2³⁰⁵   │ 3¹⁹²   │ NIST Level 3     │
    │ SIKEp751  │ 2³⁷²·3²³⁹-1      │ 2³⁷²   │ 3²³⁹   │ NIST Level 5     │
    └─────────────────────────────────────────────────────────────────────┘
    
    
    SIZE COMPARISON (at NIST Level 1):
    ┌─────────────────────────────────────────────────────────────────────┐
    │ Scheme          │ Public Key  │ Ciphertext │ Total               │
    ├─────────────────────────────────────────────────────────────────────┤

    │ Kyber-512       │ 800 bytes   │ 768 bytes  │ 1,568 bytes         │

    └─────────────────────────────────────────────────────────────────────┘
    
    SIKE had the SMALLEST combined key+ciphertext size!

    SIKE PERFORMANCE:
    ━━━━━━━━━━━━━━━━━

    • KeyGen: ~10-50 ms (vs Kyber ~0.1 ms)

    The small size was traded for slow speed.
    
    This speed penalty made the size advantage less compelling,
    but SIKE was still considered for size-constrained applications.
    """
    print(overview)
```

## 4. Why SIDH Was Considered Secure

```python
def security_assumptions():
    """Explain the security assumptions behind SIDH."""
    print("\nSIDH SECURITY ASSUMPTIONS (Pre-2022)")
    print("=" * 70)
    
    assumptions = """
    THE HARD PROBLEM (supposedly):
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    SUPERSINGULAR ISOGENY PROBLEM:
    
    Given: E₀, E_A (both supersingular curves)
    Find:  The isogeny φ: E₀ → E_A
    
    This was believed to be hard because:
    
    1. EXPONENTIALLY MANY PATHS
       From E₀, there are (2+1)^a ≈ 3^a paths of length a
       Brute force: exponential in a
    
    2. NO KNOWN QUANTUM SPEEDUP
       Unlike discrete log, no quantum algorithm known
       Best quantum: only quadratic speedup via Grover
    
    3. GRAPH STRUCTURE
       Supersingular isogeny graph is Ramanujan
       Random walks mix quickly
       No shortcuts known
    
    
    THE FATAL OVERSIGHT:
    ┌─────────────────────────────────────────────────────────────────────┐
    │                                                                     │
    │  SIDH publishes AUXILIARY TORSION POINTS: φ(P), φ(Q)               │
    │                                                                     │
    │  This is MUCH MORE information than just the codomain curve!       │
    │                                                                     │
    │  The security of SIDH ≠ security of generic isogeny problem        │
    │                                                                     │
    └─────────────────────────────────────────────────────────────────────┘
    
    
    WHAT WAS KNOWN:
    ━━━━━━━━━━━━━━━
    
    • 2016: Galbraith et al. showed some attacks using auxiliary points
    • 2017: GPST attack against static keys (mitigated by re-randomization)
    • Various papers showed the auxiliary data gives information
    
    BUT: No polynomial-time attack was found until 2022!
    
    
    THE CONFIDENCE:
    ━━━━━━━━━━━━━━━
    
    Despite warnings, SIDH/SIKE were considered promising:
    
    • Smallest key sizes among PQC candidates
    • Different mathematical foundation (diversity!)
    • 10+ years of analysis without breaks
    • Endorsed by major companies (Microsoft, etc.)
    
    The cryptographic community was SHOCKED by the 2022 attack.
    """
    print(assumptions)

security_assumptions()
```

## Summary

In this lesson, you learned about SIDH and SIKE:

1. **SIDH Design**: Key exchange using isogenies between supersingular curves
2. **Auxiliary Points**: φ(P), φ(Q) published to enable protocol
3. **Parameter Choices**: p = 2^a · 3^b - 1 for efficient computation
4. **SIKE**: KEM wrapper with Fujisaki-Okamoto transform
5. **Size Advantage**: Smallest keys among PQC candidates
6. **Speed Tradeoff**: Much slower than lattice schemes
7. **Security Assumption**: Generic isogeny problem (but SIDH wasn't generic!)

The auxiliary data would prove to be SIDH's fatal flaw.
