## 1. Introduction to Shor's Algorithm

In 1994, mathematician Peter Shor developed a quantum algorithm that would eventually shake the foundations of modern cryptography.

<Definition term="Shor's Algorithm">
A quantum algorithm that efficiently solves integer factorization and the discrete logarithm problem in polynomial time, breaking the mathematical assumptions underlying RSA, Diffie-Hellman, and elliptic curve cryptography.
</Definition>

<Formula>
$$\text{Classical factoring: } O\left(e^{1.9(\ln N)^{1/3}(\ln \ln N)^{2/3}}\right) \text{ (sub-exponential)}$$
$$\text{Shor's algorithm: } O((\log N)^3) \text{ (polynomial)}$$
</Formula>

```python
import math

def compare_factoring_complexity(bits: int):
    """
    Compare classical vs quantum factoring complexity.
    """
    N = 2 ** bits
    log_N = bits * math.log(2)
    
    # Classical (General Number Field Sieve - approximate)
    classical_exp = 1.9 * (log_N ** (1/3)) * (math.log(log_N) ** (2/3))
    classical_ops = math.exp(classical_exp)
    
    # Quantum (Shor's algorithm)
    quantum_ops = (bits ** 3)  # O((log N)^3)
    
    print(f"Factoring a {bits}-bit number:")
    print(f"  Classical operations: ~2^{classical_exp/math.log(2):.0f}")
    print(f"  Quantum operations: ~{quantum_ops:,}")
    print(f"  Speedup factor: ~2^{classical_exp/math.log(2) - math.log2(quantum_ops):.0f}")
    
    return classical_ops, quantum_ops

# RSA-2048
compare_factoring_complexity(2048)
print()
# RSA-4096
compare_factoring_complexity(4096)
```

<Warning>
**This is not a small improvement—it's a fundamental change in complexity class!** Going from sub-exponential to polynomial time means that increasing key size provides diminishing returns against quantum attacks.
</Warning>

## 2. How Shor's Algorithm Works

Shor's algorithm converts the factoring problem into a period-finding problem, which quantum computers can solve efficiently.

### The Key Insight

<Formula>
$$\text{To factor } N: \text{ Find the period } r \text{ of } f(x) = a^x \mod N$$
$$\text{Once } r \text{ is known: } \gcd(a^{r/2} \pm 1, N) \text{ often yields factors}$$
</Formula>

### Algorithm Overview

```
Shor's Algorithm for Factoring N:

1. Classical preprocessing:
   ├── Choose random a < N
   ├── Check gcd(a, N) - if > 1, we found a factor!
   └── Otherwise, proceed to quantum part

2. Quantum period finding:
   ├── Create superposition: |0⟩|0⟩ → Σ|x⟩|0⟩
   ├── Compute: Σ|x⟩|0⟩ → Σ|x⟩|aˣ mod N⟩
   ├── Apply Quantum Fourier Transform to first register
   └── Measure to get information about period r

3. Classical post-processing:
   ├── Use continued fractions to extract period r
   ├── If r is odd or a^(r/2) ≡ -1 (mod N), restart
   └── Compute gcd(a^(r/2) ± 1, N) to find factors
```

### Quantum Fourier Transform (QFT)

<Definition term="Quantum Fourier Transform">
The quantum analog of the discrete Fourier transform. It can be implemented efficiently on a quantum computer with O(n²) gates, compared to O(n·2ⁿ) for classical FFT on the same problem size.
</Definition>

<Formula>
$$QFT|x\rangle = \frac{1}{\sqrt{N}} \sum_{k=0}^{N-1} e^{2\pi i xk/N}|k\rangle$$
</Formula>

```python
def shors_algorithm_conceptual(N: int):
    """
    Conceptual walkthrough of Shor's algorithm.
    (Not actual quantum computation)
    """
    import random
    from math import gcd
    
    print(f"Factoring N = {N}")
    
    # Step 1: Choose random a
    a = random.randint(2, N - 1)
    print(f"1. Chose random a = {a}")
    
    # Check if we got lucky
    g = gcd(a, N)
    if g > 1:
        print(f"   Lucky! gcd({a}, {N}) = {g} is a factor!")
        return g, N // g
    
    # Step 2: Find period (classically for demo, quantum in reality)
    # f(x) = a^x mod N
    print(f"2. Finding period of f(x) = {a}^x mod {N}")
    
    # Compute sequence until we find repetition
    seen = {}
    x = 0
    while True:
        val = pow(a, x, N)
        if val in seen:
            r = x - seen[val]
            print(f"   Found period r = {r}")
            break
        seen[val] = x
        x += 1
        if x > N:  # Safety limit
            print("   Period not found in reasonable time")
            return None
    
    # Step 3: Extract factors
    if r % 2 == 1:
        print(f"3. Period {r} is odd, restart needed")
        return None
    
    candidate1 = pow(a, r // 2, N)
    if candidate1 == N - 1:  # a^(r/2) ≡ -1 (mod N)
        print(f"3. a^(r/2) ≡ -1 (mod N), restart needed")
        return None
    
    factor1 = gcd(candidate1 - 1, N)
    factor2 = gcd(candidate1 + 1, N)
    
    print(f"3. Computing gcd(a^(r/2) ± 1, N):")
    print(f"   gcd({candidate1} - 1, {N}) = {factor1}")
    print(f"   gcd({candidate1} + 1, {N}) = {factor2}")
    
    if factor1 > 1 and factor1 < N:
        return factor1, N // factor1
    if factor2 > 1 and factor2 < N:
        return factor2, N // factor2
    
    return None

# Demo with small numbers
result = shors_algorithm_conceptual(15)
if result:
    print(f"\nResult: {result[0]} × {result[1]} = {result[0] * result[1]}")
```

## 3. Breaking RSA

RSA security relies entirely on the difficulty of factoring the modulus N = p × q.

### The Attack

<Example>
**Shor's Attack on RSA**

```python
def shors_attack_on_rsa():
    """
    Conceptual demonstration of Shor's attack on RSA.
    """
    # RSA Setup (small example)
    p, q = 61, 53
    N = p * q  # 3233
    phi_N = (p - 1) * (q - 1)  # 3120
    e = 17  # Public exponent
    
    # Compute private key d (normally secret!)
    d = pow(e, -1, phi_N)  # 2753
    
    print("RSA Public Key: (N={}, e={})".format(N, e))
    print("RSA Private Key: d={} (SECRET!)".format(d))
    
    # Encrypt a message
    message = 42
    ciphertext = pow(message, e, N)
    print(f"\nOriginal message: {message}")
    print(f"Ciphertext: {ciphertext}")
    
    # === Shor's Attack ===
    print("\n--- Shor's Algorithm Attack ---")
    
    # Step 1: Factor N using Shor's algorithm
    # In reality, this is done on a quantum computer
    p_found, q_found = 61, 53  # "Found" by Shor's
    print(f"Factored N: {p_found} × {q_found}")
    
    # Step 2: Compute φ(N)
    phi_N_recovered = (p_found - 1) * (q_found - 1)
    print(f"Computed φ(N) = {phi_N_recovered}")
    
    # Step 3: Compute private key d
    d_recovered = pow(e, -1, phi_N_recovered)
    print(f"Recovered private key d = {d_recovered}")
    
    # Step 4: Decrypt!
    decrypted = pow(ciphertext, d_recovered, N)
    print(f"\nDecrypted message: {decrypted}")
    print(f"Attack successful: {decrypted == message}")

shors_attack_on_rsa()
```
</Example>

### Impact on RSA Key Sizes

| RSA Key Size | Classical Attack | Quantum Attack (Shor's) |
|--------------|------------------|-------------------------|
| RSA-1024 | ~2⁸⁰ ops (years) | Hours |
| RSA-2048 | ~2¹¹² ops (infeasible) | Hours-Days |
| RSA-4096 | ~2¹⁴⁰ ops (infeasible) | Days-Weeks |
| RSA-8192 | ~2¹⁵⁶ ops (infeasible) | Weeks |

<Warning>
**Increasing RSA key size does not help!** The quantum algorithm is polynomial in key size, so doubling the key size only increases attack time by a factor of ~8 (2³), not by an exponential amount.
</Warning>

## 4. Breaking Diffie-Hellman

Diffie-Hellman key exchange relies on the discrete logarithm problem (DLP).

<Formula>
$$\text{Given } g, p, \text{ and } A = g^a \mod p, \text{ find } a$$
</Formula>

### Shor's Algorithm for Discrete Log

The same quantum techniques that solve factoring also solve the discrete logarithm problem:

```python
def shors_attack_on_dh():
    """
    Conceptual Shor's attack on Diffie-Hellman.
    """
    # DH Parameters (small example)
    p = 23  # Prime
    g = 5   # Generator
    
    # Alice's secret and public value
    a = 6  # Alice's secret (normally unknown!)
    A = pow(g, a, p)  # A = 8
    
    # Bob's secret and public value
    b = 15  # Bob's secret
    B = pow(g, b, p)  # B = 19
    
    print("Public parameters: p={}, g={}".format(p, g))
    print(f"Alice's public value: A = {A}")
    print(f"Bob's public value: B = {B}")
    
    # Shared secret (what Alice and Bob compute)
    shared_alice = pow(B, a, p)
    shared_bob = pow(A, b, p)
    print(f"\nShared secret: {shared_alice}")
    
    # === Shor's Attack ===
    print("\n--- Shor's Algorithm Attack ---")
    
    # Attacker sees: p, g, A, B
    # Shor's algorithm solves: find a such that g^a ≡ A (mod p)
    
    # Using Shor's algorithm (simulated)
    a_recovered = 6  # "Found" by quantum computer
    print(f"Recovered Alice's secret: a = {a_recovered}")
    
    # Now compute shared secret
    shared_attacker = pow(B, a_recovered, p)
    print(f"Computed shared secret: {shared_attacker}")
    print(f"Attack successful: {shared_attacker == shared_alice}")

shors_attack_on_dh()
```

## 7. What Shor's Algorithm Does NOT Break

Important: Shor's algorithm specifically targets mathematical structures used in public-key cryptography.

### Safe from Shor's Algorithm

| System | Why Safe |
|--------|----------|
| **AES-256** | Symmetric; no factoring/DLP involved |
| **SHA-3** | Hash function; different structure |
| **ChaCha20** | Symmetric stream cipher |
| **Lattice-based crypto** | Different hard problem (LWE) |
| **Hash-based signatures** | Relies only on hash security |
| **Code-based crypto** | Different hard problem |

