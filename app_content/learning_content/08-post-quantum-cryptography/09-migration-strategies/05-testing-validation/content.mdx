## 2. Functional Testing

```python
def functional_testing():
    """Functional testing for PQC migration."""
    print("\nFUNCTIONAL TESTING")
    print("=" * 70)
    
    print("""
    CRYPTO OPERATION TESTS:
    ━━━━━━━━━━━━━━━━━━━━━━━━
    
    Key Encapsulation (ML-KEM):
    □ Key generation produces valid keys
    □ Encapsulation produces valid ciphertext
    □ Decapsulation recovers correct shared secret
    □ Invalid ciphertext triggers implicit rejection
    □ Different key pairs produce different secrets
    
    Digital Signatures (ML-DSA):
    □ Key generation produces valid key pair
    □ Signing produces valid signature
    □ Verification accepts valid signatures
    □ Verification rejects invalid signatures
    □ Verification rejects wrong message
    □ Verification rejects wrong key
    
    Hybrid Operations:
    □ Both classical and PQC components work
    □ Key combination produces consistent results
    □ Fallback to classical works if PQC fails
    
    
    KNOWN ANSWER TESTS (KAT):
    ━━━━━━━━━━━━━━━━━━━━━━━━━━
    
    NIST provides official test vectors:
    • Deterministic outputs for given inputs
    • Must pass ALL vectors exactly
    • Available for ML-KEM, ML-DSA, SLH-DSA
    """)
    
    print('''
    ```python
    def test_ml_kem_kat():
        """Test ML-KEM against NIST KAT vectors."""
        kat_vectors = load_nist_kat("ML-KEM-768")
        
        for vector in kat_vectors:
            # Test key generation
            pk, sk = ml_kem_keygen(seed=vector['seed'])
            assert pk == vector['pk'], "Public key mismatch"
            assert sk == vector['sk'], "Secret key mismatch"
            
            # Test encapsulation
            ct, ss = ml_kem_encaps(pk, coins=vector['coins'])
            assert ct == vector['ct'], "Ciphertext mismatch"
            assert ss == vector['ss'], "Shared secret mismatch"
            
            # Test decapsulation
            ss_dec = ml_kem_decaps(sk, ct)
            assert ss_dec == ss, "Decapsulated secret mismatch"
        
        print("All KAT vectors passed!")
    ```
    ''')

functional_testing()
```

def example_benchmark():
    """Example performance benchmark code."""
    print("\nEXAMPLE BENCHMARK:")
    print("-" * 50)
    
    print('''
    ```python
    import time
    import statistics
    
    def benchmark_tls_handshake(endpoint, iterations=1000):
        """Benchmark TLS handshake latency."""
        latencies = []
        
        for _ in range(iterations):
            start = time.perf_counter()
            conn = establish_tls_connection(endpoint)
            end = time.perf_counter()
            latencies.append((end - start) * 1000)  # ms
            conn.close()
        
        return {
            'mean': statistics.mean(latencies),
            'p50': statistics.median(latencies),
            'p95': statistics.quantiles(latencies, n=20)[18],
            'p99': statistics.quantiles(latencies, n=100)[98],
            'max': max(latencies)
        }
    
    # Compare classical vs hybrid
    classical = benchmark_tls_handshake("classical.example.com")
    hybrid = benchmark_tls_handshake("hybrid.example.com")
    
    print(f"Classical P50: {classical['p50']:.2f}ms")
    print(f"Hybrid P50: {hybrid['p50']:.2f}ms")
    print(f"Overhead: {(hybrid['p50']/classical['p50']-1)*100:.1f}%")
    ```
    ''')

example_benchmark()

## Summary

Comprehensive testing ensures successful PQC migration:

1. **Functional**: KAT vectors, correctness, hybrid operation
2. **Performance**: Latency, throughput, resource usage
3. **Security**: Algorithm verification, constant-time, attack resistance
4. **Acceptance**: Clear criteria and sign-off process
5. **Continuous**: Ongoing monitoring and regression testing

Never skip testing—the cost of failure in production is much higher.
