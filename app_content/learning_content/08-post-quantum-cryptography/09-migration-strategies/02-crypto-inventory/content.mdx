## Learning Objectives

After completing this lesson, you will be able to:
- Identify all cryptographic assets in an organization
- Use discovery tools and techniques
- Create and maintain a crypto inventory
- Track algorithm usage across systems

## 3. Inventory Data Model

```python
def inventory_data_model():
    """Structure for cryptographic inventory data."""
    print("\nINVENTORY DATA MODEL")
    print("=" * 70)
    
    print("""
    CRYPTO ASSET RECORD:
    ━━━━━━━━━━━━━━━━━━━━━
    """)
    
    print('''
    ```python
    class CryptoAsset:
        """Represents a cryptographic asset in inventory."""
        
        # Identification
        asset_id: str           # Unique identifier
        name: str               # Human-readable name
        description: str        # What it does
        
        # Classification
        asset_type: str         # certificate, key, algorithm, protocol
        algorithm: str          # RSA-2048, ECDSA-P256, AES-256, etc.
        key_size: int           # Key size in bits
        
        # Location
        system: str             # System name
        environment: str        # prod, staging, dev
        owner: str              # Team/person responsible
        
        # Risk Assessment
        quantum_vulnerable: bool  # Broken by quantum?
        data_sensitivity: str     # high, medium, low
        secrecy_duration: int     # Years data must stay secret
        
        # Metadata
        discovered_date: date
        last_updated: date
        expiration_date: date     # For certificates
        
        # Migration Status
        migration_status: str     # not_started, planned, in_progress, complete
        migration_priority: int   # 1 (highest) to 5 (lowest)
    ```
    ''')
    
    print("""
    
    EXAMPLE INVENTORY ENTRIES:

## 7. Common Pitfalls

```python
def common_pitfalls():
    """Common pitfalls in crypto inventory."""
    print("\nCOMMON INVENTORY PITFALLS")
    print("=" * 70)
    
    print("""
    PITFALL 1: INCOMPLETE DISCOVERY
    ┌─────────────────────────────────────────────────────────────────┐
    │ Problem: Only scanning external-facing systems                 │
    │ Reality: Internal systems often have more crypto               │
    │ Solution: Scan all networks, include internal traffic          │
    └─────────────────────────────────────────────────────────────────┘
    
    PITFALL 2: POINT-IN-TIME SNAPSHOT
    ┌─────────────────────────────────────────────────────────────────┐
    │ Problem: Running inventory once and declaring "done"           │
    │ Reality: New systems deployed constantly                       │
    │ Solution: Continuous discovery integrated with CI/CD           │
    └─────────────────────────────────────────────────────────────────┘
    
    PITFALL 3: IGNORING EMBEDDED CRYPTO
    ┌─────────────────────────────────────────────────────────────────┐
    │ Problem: Only tracking certificates and TLS                    │
    │ Reality: Crypto in firmware, HSMs, IoT, databases              │
    │ Solution: Interview teams, review architecture docs            │
    └─────────────────────────────────────────────────────────────────┘
    
    PITFALL 4: NO OWNERSHIP ASSIGNMENT
    ┌─────────────────────────────────────────────────────────────────┐
    │ Problem: Inventory without clear ownership                     │
    │ Reality: Nobody responsible = nobody migrates                  │
    │ Solution: Assign owner to every crypto asset                   │
    └─────────────────────────────────────────────────────────────────┘
    
    PITFALL 5: MISSING THIRD-PARTY CRYPTO
    ┌─────────────────────────────────────────────────────────────────┐
    │ Problem: Only inventorying internal systems                    │
    │ Reality: SaaS, APIs, partners use crypto too                   │
    │ Solution: Vendor assessments, contract reviews                 │
    └─────────────────────────────────────────────────────────────────┘
    """)

common_pitfalls()
```

## Summary

Cryptographic inventory is the foundation of PQC migration:

1. **Scope**: Algorithms, certificates, keys, protocols, libraries
2. **Discovery**: Automated scanning + manual review
3. **Data Model**: Structured records with risk assessment
4. **Maintenance**: Continuous updates and integration
5. **Reporting**: Executive and technical views
6. **Pitfalls**: Incomplete discovery, point-in-time, no ownership

You can't migrate what you don't know about.
