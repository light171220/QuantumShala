# Union-Find Decoder

## Introduction

The Union-Find decoder offers near-linear time complexity, making it ideal for real-time quantum error correction. It works by growing clusters around defects and merging them when they meet, achieving good accuracy with excellent speed.

## The Union-Find Data Structure

```python
class UnionFindDataStructure:
    """
    Explain the Union-Find data structure.
    """

    def __init__(self, n: int):
        self.parent = list(range(n))  # Each element is its own root
        self.rank = [0] * n           # For union by rank

    def find(self, x: int) -> int:
        """Find root with path compression."""
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # Path compression
        return self.parent[x]

    def union(self, x: int, y: int):
        """Union by rank."""
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return

        # Attach smaller tree under larger
        if self.rank[rx] < self.rank[ry]:
            self.parent[rx] = ry
        elif self.rank[rx] > self.rank[ry]:
            self.parent[ry] = rx
        else:
            self.parent[ry] = rx
            self.rank[rx] += 1
```

## Variants and Improvements

```
Union-Find Decoder Variants
в•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җ

1. WEIGHTED UNION-FIND
    - Use edge weights (not just distances)
    - Better handles non-uniform error rates
    - Slightly higher threshold

2. PEELING DECODER
    - Variant of Union-Find
    - "Peels" away corrections layer by layer
    - Very fast for sparse errors

3. ALMOST-LINEAR UNION-FIND
    - Optimized growth strategy
    - Achieves O(dВІ Оұ(dВІ)) truly
    - Best for large d

4. PARALLEL UNION-FIND
    - Divide lattice into regions
    - Decode regions in parallel
    - Merge at boundaries
    - Near-linear speedup with processors


FUSION BLOSSOM (hybrid):
    - Uses Union-Find for initial clustering
    - Then MWPM on reduced graph
    - Best of both worlds: speed + accuracy
```

## Summary

The Union-Find decoder offers an excellent speed-accuracy trade-off:

1. **Core Idea**: Grow clusters around defects, merge when meeting
2. **Data Structure**: Union-Find with path compression
3. **Complexity**: O(dВІ Оұ(dВІ)) вүҲ nearly linear
4. **Threshold**: ~0.9% (slightly below MWPM)
5. **Real-Time**: Excellent for practical implementation
6. **Parallelizable**: Scales well with hardware

Union-Find is often the decoder of choice for real-time QEC systems.
