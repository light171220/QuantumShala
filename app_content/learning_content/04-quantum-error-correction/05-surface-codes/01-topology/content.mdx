# Topological Foundations

## Introduction

Surface codes derive their power from topology—the mathematical study of properties preserved under continuous deformations. By encoding quantum information in topological features of a 2D lattice, surface codes achieve remarkable error resilience. This lesson introduces the topological concepts essential for understanding surface codes.

## The 2D Lattice Structure

```
The Surface Code Lattice
════════════════════════

Surface codes are built on a 2D square lattice where:
    - Qubits sit on EDGES of the lattice
    - X-stabilizers are associated with VERTICES
    - Z-stabilizers are associated with FACES (plaquettes)

Basic lattice (3×3 example):

    ●───○───●───○───●
    │   │   │   │   │
    ○───●───○───●───○
    │   │   │   │   │
    ●───○───●───○───●
    │   │   │   │   │
    ○───●───○───●───○
    │   │   │   │   │
    ●───○───●───○───●

Legend:
    ● = Vertex (X-stabilizer center)
    ○ = Face center (Z-stabilizer center)
    ─ and │ = Edges (qubit locations)

Each edge hosts ONE physical qubit.
```

## Primal and Dual Lattices

```python
Key insight:
┌─────────────────────────────────────────────────────────────────┐
│  X-errors on primal edges ↔ Detected at primal vertices        │
│  Z-errors on primal edges ↔ Detected at dual vertices (faces)  │
│                                                                 │
│  The dual lattice for Z is the primal lattice for X!           │
└─────────────────────────────────────────────────────────────────┘
        """)

    def count_elements(self):
        """Count lattice elements."""
        L = self.L
        print(f"\nLattice Element Count for {L}×{L} Surface Code")

        # For planar code with smooth/rough boundaries
        vertices = L * L
        h_edges = L * (L - 1)
        v_edges = (L - 1) * L
        total_edges = h_edges + v_edges
        faces = (L - 1) * (L - 1)

        print(f"Vertices (X-stabilizer sites): {vertices}")
        print(f"Edges (data qubits): {total_edges}")

        print(f"Z-stabilizers: {faces}")
        print(f"Logical qubits: k = E - V - F + 2 = 1 (for planar)")


lattice = LatticeStructure(5)
lattice.show_dual_lattice_concept()
lattice.count_elements()
```

## Boundary Conditions

```
Boundary Types in Surface Codes
═══════════════════════════════

SMOOTH BOUNDARY (Primal boundary):
    - Edges end at the boundary
    - X-stabilizers are truncated (3-body instead of 4-body)
    - Z logical operator can terminate here

ROUGH BOUNDARY (Dual boundary):
    - Vertices sit on the boundary
    - Z-stabilizers are truncated
    - X logical operator can terminate here

Planar Surface Code Layout:

         Smooth (top)
    ┏━━━━━━━━━━━━━━━━━━━┓
```

## Homology: The Mathematics of Loops

```python
class HomologyBasics:
    """
    Homology explains why surface codes work topologically.
    """

    def explain_homology(self):
        """Explain homology in the context of surface codes."""
        print("""
Homology: Loops and Boundaries
══════════════════════════════

In topology, we classify paths by whether they can be contracted:

TRIVIAL LOOP (Boundary):
    A loop that can be continuously shrunk to a point.
    In surface codes: A closed error chain that's detectable.

    ┌───────────┐
    │  ┌───┐    │
    │  │ ● │    │   This loop encloses something
    │  └───┘    │   It CAN be detected!
    └───────────┘

NON-TRIVIAL LOOP (Cycle):
    A loop that cannot be contracted - it wraps around something.
    In surface codes: A logical operator!

    On a torus:
    ╭─────────────────╮
    │ ←←←←←←←←←←←←←← │   This loop wraps around
    │                 │   It CANNOT be detected!
    ╰─────────────────╯

HOMOLOGY CLASSES:
    Two loops are "homologous" if their difference is a boundary.
    Same homology class → Same effect on logical state.

For surface codes:
┌─────────────────────────────────────────────────────────────────┐
│  Detectable errors = Boundaries (trivial loops)                │
│  Logical operators = Non-trivial cycles                        │
│  Equivalent errors = Differ by a stabilizer (boundary)         │
└─────────────────────────────────────────────────────────────────┘

This is why we call it "topological" protection:
    - The logical information is encoded in the TOPOLOGY
    - Small local errors can't change the topology
    - Only errors spanning the full system can cause logical errors
        """)

    def show_chain_complex(self):
        """Explain the chain complex structure."""
        print("""
The Chain Complex
═════════════════

Surface codes have a natural chain complex:

    C₂ (faces) ──∂₂──> C₁ (edges) ──∂₁──> C₀ (vertices)

Where:
    ∂₂ maps each face to its boundary (4 edges)
    ∂₁ maps each edge to its boundary (2 vertices)

Key property: ∂₁ ∘ ∂₂ = 0 (boundary of boundary is zero)

This means:
    Z-stabilizers (face boundaries) → Z-error syndromes (vertices)
    The syndrome of a stabilizer is zero!

Error chains and detection:
    Error chain E ⊂ C₁ (set of edges)
    Syndrome = ∂₁(E) (boundary of chain)
    Syndrome = 0 iff E is a cycle (closed loop)

Undetectable errors are cycles:
    - Trivial cycle (boundary of region) = Stabilizer
    - Non-trivial cycle = Logical operator
        """)


homology = HomologyBasics()
homology.explain_homology()
homology.show_chain_complex()
```

## Topological Protection

```python

    """
    Explain why topology provides error protection.
    """

    def show_protection_mechanism(self):
        """Demonstrate topological protection."""
        print("""
Why Topological Codes Are Robust
════════════════════════════════

LOCAL ERRORS CREATE SYNDROMES:

A single X error creates syndrome at its endpoints:

    Before:              After X error:
    ○   ○   ○           ○   ○   ○
      ┼   ┼               ┼   ┼
    ●───●───●           ●─X─●───●
      ┼   ┼               ┼   ┼
    ○   ○   ○           ○   ○   ○

    Syndrome: (0,0,0)   Syndrome: (1,1,0) at endpoints

SHORT CHAINS CAN BE CORRECTED:

Decoder finds minimum weight chain matching syndromes:

    Syndrome defects:    Correction chain:
         ★                    ★
                         ═════════
         ★                    ★

LOGICAL ERROR REQUIRES SPANNING CHAIN:

To cause a logical error, must connect opposite boundaries:

    ★═══════════════════════════════★
    │                               │
    │        Logical X_L            │
    │   (spans rough to rough)      │
    │                               │
    ★═══════════════════════════════★

This requires ~L errors on an L×L lattice!

PROTECTION SCALING:
┌─────────────────────────────────────────────────────────────────┐
│  Code distance d = L (linear dimension)                        │
│  Corrects up to ⌊(d-1)/2⌋ = ⌊(L-1)/2⌋ errors                  │
│                                                                 │
│  Logical error rate: p_L ∝ (p/p_th)^(L/2)                      │
│  Exponential suppression with increasing L!                    │
└─────────────────────────────────────────────────────────────────┘
        """)

    def show_error_types(self):
        """Illustrate different error scenarios."""
        print("""
Error Scenarios on Surface Code
═══════════════════════════════

1. CORRECTABLE ERROR (short chain):

   ○   ○   ○   ○   ○
     ┼   ┼   ┼   ┼
   ●───●─X─●───●───●    Single error
     ┼   ┼   ┼   ┼       → Syndrome at 2 points
```

## Genus and Logical Qubits

```
Topology Determines Encoding Capacity
═════════════════════════════════════

The number of logical qubits depends on the surface topology:

PLANAR CODE (disk topology):
    k = 1 logical qubit
    One pair of boundaries (smooth/rough)

TORIC CODE (torus topology):
    k = 2 logical qubits
    Two independent non-contractible loops

HIGHER GENUS SURFACES:
    Genus g surface: k = 2g logical qubits
    Each "handle" adds two logical qubits

    Sphere (g=0):  k = 0
    Torus (g=1):   k = 2
    Double torus (g=2): k = 4

Formula (Euler characteristic):
┌─────────────────────────────────────────────────────────────────┐
│  k = 2 - χ  where χ = V - E + F (Euler characteristic)         │
│                                                                 │
│  Planar with boundaries: χ = 1, so k = 1                        │
│  Torus (periodic boundaries): χ = 0, so k = 2                   │
└─────────────────────────────────────────────────────────────────┘

The topology of the surface directly determines how much
quantum information can be topologically protected!
```

## Summary

Topological concepts provide the foundation for surface codes:

1. **Lattice structure**: Qubits on edges, stabilizers at vertices and faces
2. **Dual lattices**: X and Z errors have dual detection structures
3. **Boundaries**: Determine where logical operators can terminate
4. **Homology**: Distinguishes detectable errors from logical operators
5. **Topological protection**: Distance scales with lattice size

This topological perspective explains why surface codes achieve such high thresholds and are leading candidates for practical quantum error correction.
