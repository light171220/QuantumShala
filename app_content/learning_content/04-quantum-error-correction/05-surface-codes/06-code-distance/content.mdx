# Code Distance and Error Thresholds

## Introduction

The code distance determines how many errors a surface code can correct and ultimately governs the logical error rate. Understanding distance and thresholds is essential for designing practical quantum error correction systems.

## Code Distance Defined

```
Code Distance of Surface Codes
══════════════════════════════

The CODE DISTANCE d is the minimum weight of any logical operator:

    d = min(weight(X_L), weight(Z_L), weight(Y_L))

For a d×d rotated planar surface code:

    - X_L spans horizontally: weight = d
    - Z_L spans vertically: weight = d
    - Y_L = X_L × Z_L: weight ≈ 2d - 1 (overlapping paths)

Therefore: CODE DISTANCE = d

Visual intuition:

    ●───●───●───●───●
    │   │   │   │   │
    ●───●───●───●───●
    │   │   │   │   │     For d=5:
    ●───●───●───●───●     - Z_L needs 5 Z ops (one column)
    │   │   │   │   │     - X_L needs 5 X ops (one row)
    ●───●───●───●───●     - Distance = 5
    │   │   │   │   │
    ●───●───●───●───●

The distance equals the minimum number of errors needed to
cause a logical error without being detected!
```

## Error Correction Capability

```python
import numpy as np
from typing import Tuple

class ErrorCorrectionCapability:
    """
    Analyze error correction capability from code distance.
    """

    def __init__(self, distance: int):
        self.d = distance

    def correction_capability(self) -> Tuple[int, int]:
        """
        Calculate correction and detection capability.

        Returns:
            (t, s) where t = correctable errors, s = detectable errors
        """
        t = (self.d - 1) // 2  # Correctable
        s = self.d - 1          # Detectable
        return t, s

    def explain_capability(self):
        """Explain error correction capability."""
        t, s = self.correction_capability()

        print(f"\nError Correction Capability for d={self.d}")
        print("=" * 50)

        print(f"""
Distance d = {self.d}

CORRECTION CAPABILITY:
━━━━━━━━━━━━━━━━━━━━━
    Can correct up to t = ⌊(d-1)/2⌋ = {t} errors

    Why? With t errors, the closest logical operator is at
    least t+1 steps away. Decoder finds correct matching.

    Example: d={self.d}, t={t}
    - {t} random errors → decoder finds correct recovery
    - {t+1} errors → might decode to wrong logical class

DETECTION CAPABILITY:
━━━━━━━━━━━━━━━━━━━━━
    Can detect up to s = d-1 = {s} errors

    Why? Any error pattern with <d errors has non-trivial
    syndrome (not a logical operator).

    Only weight-d patterns can have no syndrome
    (they ARE logical operators)

PRACTICAL INTERPRETATION:
━━━━━━━━━━━━━━━━━━━━━━━
    Physical error rate: p
    Logical error rate: p_L ∝ (p/p_th)^{{(d+1)/2}}

    If p < p_th (threshold), increasing d exponentially
    suppresses logical errors!
        """)

    def print_capability_table(self, max_d: int = 15):
        """Print table of capabilities for various distances."""
        print("\nCode Distance vs Capability")
        print("=" * 50)
        print(f"{'d':^5} {'Qubits':^10} {'Correct':^10} {'Detect':^10}")
        print("-" * 50)

        for d in range(3, max_d + 1, 2):
            n = d * d
            t = (d - 1) // 2
            s = d - 1
            print(f"{d:^5} {n:^10} {t:^10} {s:^10}")


cap = ErrorCorrectionCapability(7)
cap.explain_capability()
cap.print_capability_table()
```

## The Threshold Theorem

```
The Surface Code Threshold
══════════════════════════

THRESHOLD THEOREM:
    There exists a threshold error rate p_th such that:

    If p < p_th:
        Logical error rate p_L → 0 as d → ∞

    If p > p_th:
        Logical error rate p_L → 1/2 as d → ∞
        (No error correction benefit)

For surface codes with optimal decoding:

    p_th ≈ 10.9% (code capacity)
    p_th ≈ 2.9% (phenomenological noise)
    p_th ≈ 0.8-1.1% (circuit-level noise)

Visual representation:

    Logical Error Rate p_L
         │
    0.5 ─┤               ╱ p > p_th (fails)
         │             ╱
         │           ╱
         │         ╱
    0 ───┼────────●─────── p < p_th (succeeds)
         │        ↑
         └────────┴────────→ Code Distance d
                 p_th

Why the threshold exists:
┌─────────────────────────────────────────────────────────────────┐
│  With p < p_th, most errors are sparse and correctable        │
│  As d increases, rare dense errors become exponentially rare  │
│  Error suppression wins against error accumulation            │
└─────────────────────────────────────────────────────────────────┘
```

## Logical Error Rate Scaling

```python
class LogicalErrorRate:
    """
    Model logical error rate as function of physical error rate and distance.
    """

    def __init__(self, threshold: float = 0.01):
        """
        Initialize with threshold error rate.

        Args:
            threshold: Physical error threshold (default 1%)
        """
        self.p_th = threshold

    def estimate_logical_error_rate(self, p: float, d: int) -> float:
        """
        Estimate logical error rate.

        Approximation: p_L ≈ A × (p/p_th)^((d+1)/2)

        where A is a constant of order 1.
        """
        if p >= self.p_th:
            return 0.5  # Above threshold

        # Simplified model
        A = 0.1  # Empirical prefactor
        exponent = (d + 1) / 2
        p_L = A * (p / self.p_th) ** exponent

        return min(p_L, 0.5)

    def print_scaling_analysis(self, physical_rate: float):
        """Analyze scaling for given physical error rate."""
        print(f"\nLogical Error Rate Scaling")
        print(f"Physical error rate: p = {physical_rate*100:.2f}%")
        print(f"Threshold: p_th = {self.p_th*100:.2f}%")
        print("=" * 60)

        if physical_rate >= self.p_th:
            print("WARNING: Physical rate exceeds threshold!")
            print("Error correction provides no benefit.")
            return

        print(f"\n{'Distance':^10} {'Data Qubits':^12} {'p_L':^15} {'Improvement':^15}")
        print("-" * 60)

        prev_pL = 1.0
        for d in [3, 5, 7, 9, 11, 13, 15, 21, 31]:
            n = d * d
            p_L = self.estimate_logical_error_rate(physical_rate, d)
            improvement = prev_pL / p_L if p_L > 0 else float('inf')

            print(f"{d:^10} {n:^12} {p_L:^15.2e} {improvement:^15.1f}x")
            prev_pL = p_L

    def required_distance(self, p: float, target_pL: float) -> int:
        """Calculate required distance for target logical error rate."""
        if p >= self.p_th:
            return float('inf')

        # Solve: target_pL = A × (p/p_th)^((d+1)/2)
        # (d+1)/2 = log(target_pL/A) / log(p/p_th)

        A = 0.1
        ratio = p / self.p_th
        if ratio >= 1:
            return float('inf')

        exponent = np.log(target_pL / A) / np.log(ratio)
        d = 2 * exponent - 1

        # Round up to nearest odd number
        d = int(np.ceil(d))
        if d % 2 == 0:
            d += 1

        return max(d, 3)


# Analysis
ler = LogicalErrorRate(threshold=0.01)
ler.print_scaling_analysis(0.001)  # 0.1% physical error rate
```

## Distance Selection

```python
class DistanceSelection:
    """
    Help select appropriate code distance for applications.
    """

    def __init__(self):
        self.threshold = 0.01  # 1% default

    def recommend_distance(self, physical_rate: float,
                           target_logical_rate: float,
                           num_operations: int) -> dict:
        """
        Recommend code distance for application.

        Args:
            physical_rate: Expected physical error rate per gate
            target_logical_rate: Target logical error rate for full computation
            num_operations: Number of logical operations in computation
        """
        # Error per operation should be target / num_ops
        per_op_target = target_logical_rate / num_operations

        # Calculate required distance
        ler = LogicalErrorRate(self.threshold)
        d = ler.required_distance(physical_rate, per_op_target)

        if d == float('inf'):
            return {
                'feasible': False,
                'reason': 'Physical error rate exceeds threshold'
            }

        n = d * d
        total_qubits = 2 * n  # Approximate (data + ancilla)

        return {
            'feasible': True,
            'distance': d,
            'data_qubits': n,
            'total_qubits': total_qubits,
            'logical_error_per_op': per_op_target,
            'physical_rate': physical_rate,
            'target_rate': target_logical_rate
        }

    def example_applications(self):
        """Show distance requirements for example applications."""
        print("\nCode Distance Requirements for Applications")
        print("=" * 70)

        applications = [
            ("Simple demo (100 ops)", 100, 0.1),
            ("Medium circuit (10^4 ops)", 10000, 0.01),
            ("RSA breaking (10^9 ops)", 1e9, 0.001),
            ("Quantum chemistry (10^12 ops)", 1e12, 0.001)
        ]

        for name, num_ops, target in applications:
            print(f"\n{name}")
            print(f"  Operations: {num_ops:.0e}")
            print(f"  Target success: {(1-target)*100:.1f}%")

            for p in [0.01, 0.001, 0.0001]:
                rec = self.recommend_distance(p, target, int(num_ops))
                if rec['feasible']:
                    print(f"  p={p*100:.2f}%: d={rec['distance']}, "
                          f"{rec['total_qubits']} qubits")
                else:
                    print(f"  p={p*100:.2f}%: NOT FEASIBLE")


selection = DistanceSelection()
selection.example_applications()
```

## Asymptotic Behavior

```
Scaling Behavior of Surface Codes
═════════════════════════════════

Below threshold (p < p_th):

    Logical error rate: p_L ∝ exp(-α × d)

    where α depends on p/p_th.
    EXPONENTIAL suppression with distance!

Resource scaling:

    Physical qubits: n = d²
    Syndrome qubits: ~d²
    Total qubits: ~2d²

    For 10× logical error reduction:
        Need d → d + 2 (roughly)
        Qubits: d² → (d+2)² ≈ d² + 4d

    Very efficient scaling!

Comparison to other codes:

    Code Type        Qubits    Distance    Threshold
    ─────────────────────────────────────────────────
    Surface code     O(d²)     d           ~1%
    Color code       O(d²)     d           ~0.1%
    Concatenated     O(d^log d) d          ~10^-4
    LDPC codes       O(d)      d           varies

Surface code advantage:
┌─────────────────────────────────────────────────────────────────┐
│  1. High threshold (~1%) - tolerates realistic hardware        │
│  2. Local stabilizers - only nearest-neighbor gates needed     │
│  3. Simple structure - well-understood decoders                │
│  4. Practical - demonstrated on current hardware               │
└─────────────────────────────────────────────────────────────────┘
```

## Hardware Requirements

```python
class HardwareRequirements:
    """
    Calculate hardware requirements for surface code QEC.
    """

    def __init__(self, distance: int):
        self.d = distance

    def calculate_requirements(self) -> dict:
        """Calculate all hardware requirements."""
        d = self.d

        # Qubit counts
        data_qubits = d * d
        x_ancillas = (d * d - 1) // 2
        z_ancillas = (d * d - 1) // 2
        total_qubits = data_qubits + x_ancillas + z_ancillas

        # Gate counts per round
        cnots_per_round = 4 * (x_ancillas + z_ancillas)
        single_qubit_per_round = 2 * x_ancillas  # Hadamards

        # Connectivity (maximum degree)
        max_connectivity = 4  # Each data qubit touches ≤4 ancillas

        return {
            'distance': d,
            'data_qubits': data_qubits,
            'x_ancillas': x_ancillas,
            'z_ancillas': z_ancillas,
            'total_qubits': total_qubits,
            'cnots_per_round': cnots_per_round,
            'connectivity': max_connectivity,
            'syndrome_rounds': d  # For fault tolerance
        }

    def print_requirements(self):
        """Print hardware requirements."""
        req = self.calculate_requirements()

        print(f"\nHardware Requirements for d={self.d} Surface Code")
        print("=" * 55)

        print(f"\nQUBIT COUNTS:")
        print(f"  Data qubits:      {req['data_qubits']:>5}")
        print(f"  X-ancilla qubits: {req['x_ancillas']:>5}")
        print(f"  Z-ancilla qubits: {req['z_ancillas']:>5}")
        print(f"  ─────────────────────────")
        print(f"  Total qubits:     {req['total_qubits']:>5}")

        print(f"\nGATE REQUIREMENTS (per syndrome round):")
        print(f"  CNOT gates:       {req['cnots_per_round']:>5}")
        print(f"  Single-qubit:     {req['total_qubits']:>5} (reset/measure)")

        print(f"\nCONNECTIVITY:")
        print(f"  Max connections:  {req['connectivity']:>5} per qubit")
        print(f"  Layout:           2D nearest-neighbor")

        print(f"\nFAULT TOLERANCE:")
        print(f"  Syndrome rounds:  {req['syndrome_rounds']:>5} per QEC cycle")


# Example requirements
for d in [3, 5, 7, 11, 17]:
    hr = HardwareRequirements(d)
    hr.print_requirements()
    print()
```

## Practical Considerations

```
Choosing Code Distance in Practice
══════════════════════════════════

STEP 1: Determine physical error rate
    - Characterize your hardware
    - Include gate errors, measurement errors, idle errors
    - Effective rate: p_eff (often 0.1% - 1%)

STEP 2: Determine target logical error rate
    - Based on algorithm requirements
    - Number of logical operations
    - Acceptable failure probability

STEP 3: Calculate required distance
    - Use scaling formulas
    - Add safety margin (d → d + 2)

STEP 4: Verify feasibility
    - Check qubit count vs available qubits
    - Check connectivity requirements
    - Consider cycle time constraints

Example Decision Process:

    Application: Quantum simulation with 10^6 T-gates
    Physical rate: p = 0.1%
    Target success: 99%

    Required per-gate error: 10^-8
    Required distance: d ≈ 15-19
    Qubits needed: ~500-700 per logical qubit

    For 100 logical qubits: ~50,000-70,000 physical qubits

Trade-offs:
┌─────────────────────────────────────────────────────────────────┐
│  Larger d → Lower logical errors, more qubits, longer cycles   │
│  Smaller d → Fewer qubits, higher logical errors               │
│  Optimize for your specific hardware and application!          │
└─────────────────────────────────────────────────────────────────┘
```

## Summary

Code distance and thresholds are central to surface code design:

1. **Distance d**: Minimum weight of logical operators, equals lattice dimension
2. **Correction capability**: Corrects up to ⌊(d-1)/2⌋ errors
3. **Threshold**: p_th ≈ 1% for circuit-level noise
4. **Scaling**: Logical error rate ∝ (p/p_th)^((d+1)/2)
5. **Trade-offs**: Larger d means better protection but more qubits

The surface code's high threshold and favorable scaling make it the leading candidate for near-term fault-tolerant quantum computing.
