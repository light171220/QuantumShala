# Logical Qubits and Operators

## Introduction

A surface code encodes one logical qubit in many physical qubits. The logical information is stored in the global topology of the code state, protected from local errors. This lesson explores how logical qubits are defined and manipulated in surface codes.

## The Code Space

```
Surface Code Code Space
в•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җ

The code space is the +1 eigenspace of ALL stabilizers:

    |ПҲ_Lвҹ© вҲҲ Code Space  вҹә  S_i |ПҲ_Lвҹ© = +|ПҲ_Lвҹ©  for all stabilizers S_i

For a distance-d planar surface code:
    - n = dВІ data qubits
    - (dВІ-1)/2 X-stabilizers + (dВІ-1)/2 Z-stabilizers
    - Total: dВІ - 1 independent stabilizers
    - Code space dimension: 2^(dВІ - (dВІ-1)) = 2^1 = 2

This 2D code space is our LOGICAL QUBIT!

Logical basis states:

    |0_Lвҹ© = ОЈ (all states with even X-parity along Z_L)
    |1_Lвҹ© = ОЈ (all states with odd X-parity along Z_L)

    or equivalently:

    |+_Lвҹ© = ОЈ (all states with even Z-parity along X_L)
    |-_Lвҹ© = ОЈ (all states with odd Z-parity along X_L)
```

## Logical Operators

```python
import numpy as np
from typing import List, Tuple, Set

class LogicalOperators:
    """
    Define and analyze logical operators for surface code.
    """

    def __init__(self, distance: int):
        self.d = distance

    def get_logical_z(self, column: int = 0) -> List[Tuple[int, int]]:
        """
        Get Z_L operator positions.

        Z_L is a chain of Z operators connecting rough boundaries
        (top to bottom in standard orientation).

        Any column works - they're all equivalent (differ by stabilizers).
        """
        # Chain runs vertically through column
        return [(row, column) for row in range(self.d)]

    def get_logical_x(self, row: int = 0) -> List[Tuple[int, int]]:
        """
        Get X_L operator positions.

        X_L is a chain of X operators connecting smooth boundaries
        (left to right in standard orientation).

        Any row works - they're all equivalent (differ by stabilizers).
        """
        # Chain runs horizontally through row
        return [(row, col) for col in range(self.d)]

    def visualize_operators(self):
        """Visualize logical operators on the lattice."""
        d = self.d
        print(f"\nLogical Operators for d={d} Surface Code")
        print("=" * 60)

        # Z_L visualization
        print("\nZ_L (connects ROUGH boundaries, top вҶ” bottom):")
        print("    Rough boundary (top)")
        for row in range(d):
            line = "    "
            for col in range(d):
                if col == 0:  # Z_L on first column
                    line += "Zв”Җ"
                else:
                    line += "в—Ҹв”Җ"
            print(line[:-1])  # Remove trailing dash
        print("    Rough boundary (bottom)")

        # X_L visualization
        print("\nX_L (connects SMOOTH boundaries, left вҶ” right):")
        print("  S")
        print("  m")
        for row in range(d):
            if row == 0:  # X_L on first row
                line = "  o    Xв”ҖXв”ҖXв”Җ" + "Xв”Җ" * (d-3) + "X"
            else:
                line = "  o    в—Ҹв”Җв—Ҹв”Җв—Ҹв”Җ" + "в—Ҹв”Җ" * (d-3) + "в—Ҹ"
            print(line)
        print("  t")
        print("  h")

    def verify_commutation(self):
        """Verify logical operators commute with stabilizers."""
        print("\nLogical Operator Verification")
        print("=" * 50)

        print("""
Properties of logical operators:

1. COMMUTE WITH ALL STABILIZERS:
   [X_L, S_i] = 0 for all stabilizers S_i
   [Z_L, S_i] = 0 for all stabilizers S_i

   Why: X_L overlaps with Z-stabilizers on 0 or 2 qubits
        Z_L overlaps with X-stabilizers on 0 or 2 qubits
        Even overlap вҶ’ commutation

2. ANTICOMMUTE WITH EACH OTHER:
   {X_L, Z_L} = 0  (X_L Z_L = -Z_L X_L)

   Why: X_L and Z_L share exactly 1 qubit
        (corner where horizontal meets vertical)

3. NOT IN STABILIZER GROUP:
   X_L вҲү вҹЁSвҹ© (cannot write X_L as product of stabilizers)
   Z_L вҲү вҹЁSвҹ©

   Why: They span the code (non-trivial topology)
        """)


logical = LogicalOperators(5)
logical.visualize_operators()
logical.verify_commutation()
```

```python
class LogicalStatePreparation:
    """
    Prepare logical states in the surface code.
    """

    def __init__(self, distance: int):
        self.d = distance

    def explain_preparation(self):
        """Explain logical state preparation methods."""
        print("""
Logical State Preparation
в•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җ

METHOD 1: Direct initialization + stabilizer measurement

    Step 1: Initialize all data qubits to |0вҹ©
            This is an eigenstate of all Z stabilizers
            but NOT of X stabilizers (random Вұ1)

    Step 2: Measure all X stabilizers
            Projects into +1 eigenspace of X stabilizers
            Random logical state: |0_Lвҹ© or |1_Lвҹ©

    Step 3: If Z_L measurement = -1, apply X_L
            Corrects to deterministic |0_Lвҹ©

    |0вҹ©^вҠ—n в”Җв”Җ[measure X-stabs]в”Җв”ҖвҶ’ |Вұ_Lвҹ© в”Җв”Җ[conditional X_L]в”Җв”ҖвҶ’ |0_Lвҹ©

METHOD 2: Product state initialization

    |0_Lвҹ©: Initialize all qubits to |0вҹ©, measure stabilizers
           Natural Z_L = +1 state

    |+_Lвҹ©: Initialize all qubits to |+вҹ©, measure stabilizers
           Natural X_L = +1 state

    Other states require logical gates or measurement feedback.

METHOD 3: Fault-tolerant preparation (advanced)

    Prepare multiple copies, verify with transversal CNOTs
    Post-select on consistent results
    Slower but higher fidelity
        """)

    def prepare_zero_state_circuit(self) -> List[str]:
        """Generate circuit to prepare |0_Lвҹ©."""
        circuit = []

        # Initialize all to |0вҹ©
        circuit.append(f"# Initialize all {self.d**2} data qubits to |0вҹ©")
        for i in range(self.d ** 2):
            circuit.append(f"RESET q[{i}]")

        # Measure X stabilizers
        circuit.append("# Measure all X stabilizers")
        circuit.append("measure_x_stabilizers()")

        # Conditional correction
        circuit.append("# If Z_L eigenvalue is -1, apply X_L")
        circuit.append("if measure_ZL() == -1:")
        circuit.append("    apply_XL()")

        circuit.append("# State is now |0_Lвҹ©")
        return circuit

    def prepare_plus_state_circuit(self) -> List[str]:
        """Generate circuit to prepare |+_Lвҹ©."""
        circuit = []

        # Initialize all to |+вҹ©
        circuit.append(f"# Initialize all {self.d**2} data qubits to |+вҹ©")
        for i in range(self.d ** 2):
            circuit.append(f"RESET q[{i}]; H q[{i}]")

        # Measure Z stabilizers
        circuit.append("# Measure all Z stabilizers")
        circuit.append("measure_z_stabilizers()")

        # Conditional correction
        circuit.append("# If X_L eigenvalue is -1, apply Z_L")
        circuit.append("if measure_XL() == -1:")
        circuit.append("    apply_ZL()")

        circuit.append("# State is now |+_Lвҹ©")
        return circuit
```

```python
class LogicalMeasurement:
    """
    Measure logical operators in the surface code.
    """

    def __init__(self, distance: int):
        self.d = distance

    def explain_measurement(self):
        """Explain logical measurement methods."""
        print("""
Logical Measurement

METHOD 1: Destructive measurement

    Measure Z_L:
    1. Measure all data qubits in Z basis
    2. Compute product of outcomes along Z_L path
    3. Result: Вұ1 eigenvalue of Z_L
    4. State collapses to |0_Lвҹ© or |1_Lвҹ©

    1. Measure all data qubits in X basis
    2. Compute product of outcomes along X_L path
    3. Result: Вұ1 eigenvalue of X_L
    4. State collapses to |+_Lвҹ© or |-_Lвҹ©

    Destructive: Code is destroyed after measurement!

METHOD 2: Non-destructive measurement (with ancilla code)

    Use lattice surgery or other techniques to measure
    logical operator without destroying the code.
    (Covered in advanced topics)

METHOD 3: Indirect via syndrome history

    Track syndromes over time
    Errors that flip Z_L can be inferred from syndrome changes
    Gives information about logical state evolution
        """)

        """Generate Z_L measurement circuit."""
        circuit = []

        circuit.append("# Destructive Z_L measurement")
        circuit.append("# Measure all data qubits in Z basis")
```

## Summary

Logical qubits in surface codes are encoded topologically:

1. **Code space**: 2D subspace where all stabilizers have eigenvalue +1
2. **Logical X**: Chain of X operators connecting smooth boundaries
3. **Logical Z**: Chain of Z operators connecting rough boundaries
4. **Equivalence**: Multiple physical operators implement same logical operation
5. **Preparation**: Initialize physical qubits, measure stabilizers, apply corrections
6. **Measurement**: Destructive measurement of all qubits or lattice surgery

The topological nature of logical operators provides natural protectionвҖ”only spanning chains can flip the logical state.
