# QEC on Trapped Ion Systems

## Introduction

Trapped ion quantum computers offer the highest gate fidelities and longest coherence times among current platforms. Their all-to-all connectivity enables flexible code implementations, making them excellent for exploring various QEC approaches.

## Trapped Ion Fundamentals

```
Trapped Ion Qubit Basics
════════════════════════

PHYSICAL SETUP:
    - Individual ions (e.g., Yb⁺, Ca⁺, Ba⁺) in electromagnetic trap
    - Radio-frequency Paul trap confines ions in 1D chain
    - Laser/microwave pulses manipulate qubits

    ●──●──●──●──●──●   Ion chain in trap
    0  1  2  3  4  5   (can interact any pair)

QUBIT ENCODING:
    - Hyperfine states (nuclear spin)
    - Optical transitions (electronic states)
    - Very long coherence: seconds to minutes

GATE OPERATIONS:
    Single-qubit: Individual laser addressing
    Two-qubit: Mølmer-Sørensen or light-shift gates
               Mediated by collective ion motion (phonons)

TYPICAL PERFORMANCE:
    Single-qubit: 99.99%+ fidelity, ~10 μs
    Two-qubit: 99.5-99.9% fidelity, ~50-200 μs
    Measurement: 99.9%+ fidelity, ~100 μs
    T2: seconds to minutes
```

## Scaling with QCCD Architecture

```
Quantum Charge Coupled Device (QCCD)
════════════════════════════════════

For more than ~50-100 ions, use modular approach:

ARCHITECTURE:
    Multiple trap zones connected by junctions
    Ions shuttle between zones for operations

    ┌─────────────────────────────────────────────────┐
    │                                                 │
    │  ●●●●●●●●    ──────────    ●●●●●●●●            │
    │   Zone A         │         Zone B              │
    │                  │                              │
    │                  ▼                              │
    │            ●●●●●●●●                            │
    │             Zone C                              │
    │                                                 │
    └─────────────────────────────────────────────────┘

OPERATIONS:
    1. Within-zone gates: All-to-all, high fidelity
    2. Between-zone: Shuttle ions to shared zone
    3. Junctions: Move ions between zones

SHUTTLING:
    Speed: ~100 μm/μs
    Fidelity: 99.9%+ for careful shuttling
    Adds time overhead

QEC IMPLICATIONS:
    - Logical qubit in one zone (if fits)
    - Multi-zone operations for larger codes
    - Shuttling adds error locations
    - Schedule to minimize shuttling
```

## Ion-Specific Error Handling

```python
class IonErrors:
    """
    Handle ion-specific error sources.
    """

    def __init__(self):
        pass

    def error_sources(self):
        """Describe ion-specific errors."""
        print("""
Trapped Ion Error Sources
═════════════════════════

1. GATE ERRORS (dominant)
    Two-qubit gates: ~0.1-0.5%
    Source: Laser intensity/phase fluctuations
    Mitigation: Better lasers, composite pulses

2. MOTIONAL HEATING
    Ion motion (phonons) picks up noise
    Affects two-qubit gates
    Mitigation: Cryogenic traps, better surfaces

3. CROSSTALK
    Off-resonant excitation of nearby ions
    Individual addressing imperfect
    Mitigation: Better beam focusing, encoding choice

4. ION LOSS
    Background gas collisions
    Very rare (~1/hour) but catastrophic
    Mitigation: Better vacuum, reloading protocols

5. SHUTTLING ERRORS
    Motion during shuttle heats ions
    Adds time for decoherence
    Mitigation: Careful trajectory design


COMPARED TO SUPERCONDUCTING:
    + Much longer coherence (seconds vs microseconds)
    + No leakage to |2⟩ (clean two-level)
    + Higher gate fidelity achieved
    - Slower gates (μs vs ns)
    - Ion loss is unique challenge
        """)

    def fewer_qec_cycles(self):
        """Discuss reduced QEC need."""
        print("""
Long Coherence = Fewer QEC Cycles
═════════════════════════════════

With T2 ~ 10 seconds and gate time ~ 100 μs:

    Gates before decoherence: 10s / 100μs = 100,000

    Can run deep circuits without continuous QEC!

COMPARISON:
    Superconducting: T2 ~ 100 μs, gate ~ 50 ns
                     → 2000 gates, need frequent QEC

    Trapped Ion: T2 ~ 10 s, gate ~ 100 μs
                 → 100,000 gates, QEC less urgent

IMPLICATIONS:
    - May not need real-time decoding
    - Batch QEC at algorithm boundaries
    - Lower QEC overhead for same computation
        """)


errors = IonErrors()
errors.error_sources()
errors.fewer_qec_cycles()
```

## Summary

Trapped ion QEC has unique advantages:

1. **Connectivity**: All-to-all enables any code structure
2. **Fidelity**: Highest gate fidelities achieved
3. **Coherence**: Long T2 reduces QEC frequency
4. **Codes**: Steane, color codes more natural than surface
5. **QCCD**: Modular scaling through shuttling
6. **Trade-off**: Slower gates, but fewer QEC cycles needed

Trapped ions complement superconducting approaches with different strengths.
