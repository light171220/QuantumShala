# Hardware Platforms for QEC

## Introduction

Quantum error correction must ultimately run on physical hardware. Different quantum computing platforms—superconducting circuits, trapped ions, photons, and others—have distinct characteristics that affect QEC implementation. This lesson surveys the major platforms and their QEC suitability.

## Hardware Requirements for QEC

```
What Hardware Needs for QEC
═══════════════════════════

ERROR RATES:
    Physical error rate < threshold (~0.1-1%)
    Current best: ~0.1-0.5%
    Target: <0.1% for efficient QEC

QUBIT COUNT:
    [[n,k,d]] code: n physical per logical
    Surface code d=11: ~242 physical qubits
    1000 logical qubits: ~250,000 physical

CONNECTIVITY:
    Surface code: 2D nearest-neighbor
    Steane code: Long-range connections helpful
    LDPC codes: Sparse, potentially long-range

GATE SET:
    Minimum: Single-qubit + two-qubit entangling
    Typical: {X, Y, Z, H, CNOT} or equivalent

MEASUREMENT:
    Fast, non-destructive readout
    High fidelity (>99%)
    Mid-circuit measurement capability

RESET:
    Ability to reset qubits to |0⟩
    Fast reset for ancilla recycling

SPEED:
    Gate time << coherence time
    Syndrome extraction faster than error accumulation
```

## Platform-Specific QEC

```
QEC Implementation by Platform
══════════════════════════════

SUPERCONDUCTING:

    Preferred code: Surface code
    Why: 2D connectivity matches hardware

    Layout:
    ┌─●─┬─●─┬─●─┐
    │ X │ Z │ X │    ● = data qubit
    ├─●─┼─●─┼─●─┤    X = X-check ancilla
    │ Z │ X │ Z │    Z = Z-check ancilla
    ├─●─┼─●─┼─●─┤
    │ X │ Z │ X │
    └─●─┴─●─┴─●─┘

    Google Sycamore: demonstrated d=5 surface code
    IBM: moving toward surface code arrays


TRAPPED IONS:

    Preferred codes: [[7,1,3]] Steane, small codes
    Why: All-to-all enables any code structure

    Approach:
    - Use flexibility for transversal gates
    - Code switching easier (connected qubits)
    - Fewer QEC cycles needed (long coherence)

    IonQ, Quantinuum: demonstrated small codes


NEUTRAL ATOMS:

    Preferred codes: Surface code, color codes
    Why: Can arrange atoms in any pattern

    Unique capability:
    - Move atoms to reconfigure
    - Grow/shrink code dynamically
    - Potentially topological codes

    Atom Computing, QuEra: exploring QEC


PHOTONIC:

    Preferred codes: Fusion-based, GKP
    Why: Different error model (loss)

    Approach:
    - Generate entangled photon states
    - Use linear optics for gates
    - Measurement-based approach
    - Loss is dominant error

    Xanadu, PsiQuantum: developing photonic QEC
```

## Error Budget Analysis

```python
class ErrorBudget:
    """
    Analyze error contributions in hardware QEC.
    """

    def __init__(self, platform: str):
        self.platform = platform

    def budget_breakdown(self):
        """Break down error sources."""
        print(f"""
Error Budget for {self.platform}
{'═' * 40}

TYPICAL ERROR SOURCES:

1. GATE ERRORS
    Single-qubit: 0.01-0.1%
    Two-qubit: 0.1-1%
    These are often the largest contribution

2. MEASUREMENT ERRORS
    Readout infidelity: 0.5-2%
    Affects syndrome extraction
    Mitigated by repeated measurements

3. IDLE ERRORS (DECOHERENCE)
    T1 decay: depends on time
    T2 dephasing: depends on time
    Surface code: ~4d gate times per cycle

4. LEAKAGE
    Escape from computational subspace
    Transmon: leakage to |2⟩ state
    Must detect and reset

5. CROSSTALK
    Unintended qubit-qubit coupling
    Addressed via calibration, scheduling


SUPERCONDUCTING EXAMPLE (d=5):

    Error Source          Contribution
    ─────────────────────────────────────
    Two-qubit gates       0.3%
    Single-qubit gates    0.03%
    Measurement           0.5%
    Idle (in cycle)       0.05%
    Crosstalk             0.1%
    ─────────────────────────────────────
    TOTAL per cycle       ~1%

    This is near threshold—improvements needed!
        """)


budget = ErrorBudget("Superconducting")
budget.budget_breakdown()
```

## Summary

Different hardware platforms offer distinct advantages for QEC:

1. **Superconducting**: Fast gates, 2D connectivity, surface code natural fit
2. **Trapped Ions**: Highest fidelities, all-to-all, flexible code choice
3. **Neutral Atoms**: Large scale, reconfigurable, emerging QEC
4. **Photonic**: Different paradigm, loss-based codes
5. **Requirements**: Sub-1% errors, fast syndrome extraction, scalability

The surface code on superconducting qubits is currently the leading approach.
