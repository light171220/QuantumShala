# System Integration and Scaling

## Introduction

Building a fault-tolerant quantum computer requires integrating many components: quantum hardware, classical control, error correction codes, decoders, and application software. This lesson explores how these pieces fit together and what's needed to scale to practical systems.

    - Readout lines

SCALING PROJECTION:

## Resource Estimation

```
Resource Estimation for Fault-Tolerant Computation
══════════════════════════════════════════════════

To run a useful quantum algorithm, we need to estimate:
1. Logical qubits required
2. T gates (non-Clifford operations)
3. Circuit depth
4. Physical qubit overhead

EXAMPLE: Factoring 2048-bit RSA

    Logical requirements:
    - Logical qubits: ~4,000-6,000
    - T gates: ~10^10 - 10^12
    - Circuit depth: ~10^9 cycles

    Physical overhead (surface code, p_phys = 10^-3):
    - Code distance needed: d ≈ 25-30
    - Physical qubits per logical: ~2d² ≈ 1,500
    - Total physical qubits: ~6-9 million
    - Magic state factories: significant overhead

    ┌──────────────────────────────────────────────────────┐
    │              Resource Breakdown                      │
    │                                                      │
    │  Data qubits:          ~20% of physical qubits      │
    │  Syndrome ancillas:    ~20% of physical qubits      │
    │  Magic state factories: ~60% of physical qubits     │
    │                                                      │
    │  Total: ~10 million physical qubits                 │
    │  Runtime: ~hours to days                            │
    └──────────────────────────────────────────────────────┘

RESOURCE ESTIMATION FORMULA:

    Physical qubits = (logical_qubits × overhead_per_logical)
                    + (factory_qubits)

    where:
        overhead_per_logical = 2d² (surface code)
        factory_qubits ≈ 10-100 × overhead_per_logical × num_factories

    Runtime = (T_gates × time_per_T) + (Clifford_depth × cycle_time)
```

## Scaling Strategies

```python
class ScalingStrategies:
    """
    Strategies for scaling QEC systems.
    """

    def __init__(self):
        pass

    def modular_approach(self):
        """Modular scaling architecture."""
        print("""
Modular Quantum Computing
═════════════════════════

CONCEPT:
    Build medium-scale modules (~1000 qubits)
    Connect modules via quantum links
    Each module is self-contained

    ┌────────────┐     ┌────────────┐
    │  Module A  │════════  Module B  │
    │  ~1000 Qs │     │  ~1000 Qs │
    │  Local QEC │     │  Local QEC │
    └────────────┘     └────────────┘
          ║                   ║
          ║                   ║
    ┌────────────┐     ┌────────────┐
    │  Module C  │════════  Module D  │
    │  ~1000 Qs │     │  ~1000 Qs │
    └────────────┘     └────────────┘

INTER-MODULE CONNECTIONS:
    1. Optical links (photonic interconnects)
       - Convert qubit state to photon
       - Transmit via fiber
       - Convert back at destination
       - Fidelity: ~90-95% (needs QEC)

    2. Microwave links (superconducting)
       - Direct coupling between chips
       - Limited distance (~cm)
       - Higher fidelity (~99%)

    3. Ion shuttling (trapped ions)
       - Physically move ions between zones
       - QCCD architecture
       - Fidelity: ~99.9%

ADVANTAGES:
    ✓ Each module is proven technology
    ✓ Parallel manufacturing
    ✓ Failure isolation
    ✓ Incremental scaling

CHALLENGES:
    ✗ Inter-module operations slower/noisier
    ✗ Need distributed QEC protocols
    ✗ Routing overhead for logical operations
        """)

    def code_distance_scaling(self):
        """Scaling code distance for lower error."""
        print("""
Code Distance Scaling
═════════════════════

To reduce logical error rate:
    p_L ∝ (p_phys / p_th)^((d+1)/2)

SCALING REQUIREMENTS:

    Target p_L     p_phys=10^-3    p_phys=10^-4
    ──────────────────────────────────────────
    10^-6          d ≈ 7           d ≈ 5
    10^-9          d ≈ 13          d ≈ 9
    10^-12         d ≈ 19          d ≈ 13
    10^-15         d ≈ 25          d ≈ 17

PHYSICAL QUBIT REQUIREMENTS:

    Distance    Data Qubits    Total (with ancilla)
    ────────────────────────────────────────────────
    d = 5       25             ~50
    d = 11      121            ~250
    d = 21      441            ~900
    d = 31      961            ~2000

ONE LOGICAL QUBIT AT d=25:
    - Data qubits: 625
    - Ancilla qubits: ~600
    - Total: ~1,200-1,500 physical qubits
    - Per logical qubit overhead: >1000×

IMPLICATIONS:
    1000 logical qubits at d=25:
    → 1-2 million physical qubits minimum
    → Plus magic state factories: 2-3× more
    → Total: 3-6 million physical qubits
        """)

    def factory_scaling(self):
        """Magic state factory scaling."""
        print("""
Magic State Factory Scaling
═══════════════════════════

T GATES REQUIRE MAGIC STATES:
    Each T gate consumes one |T⟩ state
    Factories produce |T⟩ via distillation

FACTORY PRODUCTION RATE:
    Level-1 factory:
    - Input: 15 noisy |T⟩ states
    - Output: 1 cleaner |T⟩ state
    - Success rate: ~85%
    - Qubits: ~200-500

    Level-2 factory (cascaded):
    - Takes level-1 outputs
    - Further distills to p_T < 10^-10
    - Qubits: ~1000-2000

DEMAND vs SUPPLY:

    Algorithm needs: 10^9 T gates in 10^6 cycles
    → 1000 T gates per cycle average

    One factory produces: ~1 T state per 10-100 cycles
    → Need: ~10,000-100,000 factories!

    Factory qubits: 10,000 × 1000 = 10 million
    Data qubits: 10,000 logical × 1000 = 10 million
    Total: ~20 million physical qubits

OPTIMIZATION:
    - T gate parallelization
    - Factory output buffering
    - Just-in-time production
    - Reuse distillation ancillas
        """)


scaling = ScalingStrategies()
scaling.modular_approach()
scaling.code_distance_scaling()
scaling.factory_scaling()
```

## End-to-End System Design

```
Designing a Complete System: 100 Logical Qubit Machine
══════════════════════════════════════════════════════

TARGET SPECS:
    - 100 logical qubits
    - Logical error rate: 10^-10 per operation
    - 1000 T gates per algorithm (small demo)

STEP 1: CODE SELECTION
    Surface code with d = 17
    - Physical error rate: 10^-3
    - Logical error: (10^-3/10^-2)^9 ≈ 10^-9 per cycle
    - With 1000 cycles: 10^-6 total (need higher d or better phys)

    Revised: d = 21 for 10^-10 target

STEP 2: PHYSICAL QUBIT COUNT
    Per logical qubit (d=21):
    - Data: 21² = 441 qubits
    - Ancilla: ~400 qubits
    - Total: ~850 qubits

    100 logical qubits: 85,000 qubits

STEP 3: MAGIC STATE FACTORIES
    1000 T gates, 100 cycles each = 10 T/cycle
    Factory production: 1 T per 50 cycles
    Factories needed: ~500

    Qubits per factory: ~500
    Factory total: 250,000 qubits

STEP 4: TOTAL SYSTEM
    ┌─────────────────────────────────────────┐
    │  Component           │  Qubits          │
    ├─────────────────────────────────────────┤
    │  Logical qubits (100)│  85,000          │
    │  Magic factories     │  250,000         │
    │  Routing/ancilla     │  ~65,000         │
    ├─────────────────────────────────────────┤
    │  TOTAL               │  ~400,000        │
    └─────────────────────────────────────────┘

STEP 5: CLASSICAL REQUIREMENTS
    - FPGAs: ~100 (one per logical qubit)
    - Decoding: Union-Find at d=21
    - Cycle time: ~1 μs
    - Data rate: ~400 Gbps (syndrome data)

STEP 6: PHYSICAL INFRASTRUCTURE
    - Dilution refrigerator: custom large-scale
    - Control electronics: ~100,000 channels
    - Power: ~100 kW classical control
    - Space: ~100 m² for equipment
```

## Summary

System integration for fault-tolerant quantum computing requires:

1. **Architecture**: Layered design from algorithms to physical hardware
2. **Wiring**: Millions of connections need multiplexing and cryo-electronics
3. **Calibration**: Automated systems for scaling to millions of qubits
4. **Resources**: 10^6-10^7 physical qubits for useful computation
5. **Modularity**: Smaller modules connected for practical scaling
6. **Factories**: Magic state distillation dominates qubit count

Current systems (~1000 qubits) are prototypes; useful fault-tolerant machines require 100-1000× more qubits.
