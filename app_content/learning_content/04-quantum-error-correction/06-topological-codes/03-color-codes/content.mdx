# Color Codes

## Introduction

Color codes are topological quantum codes defined on three-colorable lattices. Their key advantage is supporting transversal implementation of all Clifford gates, making them valuable for fault-tolerant quantum computation despite having lower error thresholds than surface codes.

## Three-Colorable Lattices

```
Color Code Lattice Structure
════════════════════════════

Color codes require a 3-COLORABLE lattice:
    Every face can be colored with 3 colors (R, G, B)
    Adjacent faces have different colors

Common lattices:

1. HEXAGONAL (6.6.6) - Three hexagons meet at each vertex
   ─────────────────────────────────────────────────────

        ╱ ╲   ╱ ╲   ╱ ╲
       │ R │ G │ R │ G │
        ╲ ╱ ╲ ╱ ╲ ╱ ╲ ╱
         │ B │ R │ B │
        ╱ ╲ ╱ ╲ ╱ ╲ ╱ ╲
       │ G │ B │ G │ B │
        ╲ ╱ ╲ ╱ ╲ ╱ ╲ ╱

2. SQUARE-OCTAGON (4.8.8) - Squares and octagons
   ─────────────────────────────────────────────

      ┌───┐       ┌───┐
      │ R │       │ R │
    ╱─┴───┴─╲   ╱─┴───┴─╲
   │    B    │ │    G    │
    ╲─┬───┬─╱   ╲─┬───┬─╱
      │ R │       │ R │
      └───┘       └───┘

Key property:
┌─────────────────────────────────────────────────────────────────┐
│  QUBITS sit on VERTICES (not edges like surface codes)         │
│  STABILIZERS are associated with FACES                         │
│  Each face has both X-type and Z-type stabilizer               │
└─────────────────────────────────────────────────────────────────┘
```

## The 7-Qubit Color Code

```python
import numpy as np
from typing import List, Tuple, Set, Dict

class SevenQubitColorCode:
    """
    The smallest color code: [[7, 1, 3]] on triangular lattice.
    This is the same as the Steane code!
    """

    def __init__(self):
        self.n = 7
        self.k = 1
        self.d = 3

        # Qubit positions (vertices of triangular patch)
        self.qubits = list(range(7))

        # Face colors and qubit membership
        # Red faces
        self.red_faces = [
            [0, 1, 2],      # Top red
            [4, 5, 6],      # Bottom red
        ]

        # Green faces
        self.green_faces = [
            [0, 2, 3],      # Left green
            [1, 4, 5],      # Right green
        ]

        # Blue faces
        self.blue_faces = [
            [1, 2, 4],      # Center-left blue
            [2, 3, 4],      # Center-right blue
        ]

    def visualize(self):
        """Show the 7-qubit color code structure."""
        print("""
7-Qubit Color Code (Steane Code)
════════════════════════════════

Qubit layout:

            0
           ╱ ╲
          ╱ R ╲
         1─────2
        ╱ ╲ B ╱ ╲
       ╱ G ╲╱ G ╲
      5─────4─────3
       ╲ R ╱
        ╲ ╱
         6

Vertices: Qubits 0-6
Faces: 4 triangles colored R, G, B
       (2 red, 2 green, 2 blue partially visible)

Stabilizers:
    Each COLOR defines stabilizers:
    X-stabilizers: X on all qubits of faces with that color
    Z-stabilizers: Z on all qubits of faces with that color

For Red faces [0,1,2] and [4,5,6]:
    X stabilizer: X₀X₁X₂ and X₄X₅X₆
    Z stabilizer: Z₀Z₁Z₂ and Z₄Z₅Z₆
        """)

    def get_stabilizers(self) -> Dict[str, List]:
        """Return X and Z stabilizers grouped by color."""
        all_faces = {
            'red': self.red_faces,
            'green': self.green_faces,
            'blue': self.blue_faces
        }

        x_stabs = []
        z_stabs = []

        for color, faces in all_faces.items():
            for face in faces:
                x_stabs.append({'color': color, 'qubits': face, 'type': 'X'})
                z_stabs.append({'color': color, 'qubits': face, 'type': 'Z'})

        return {'X': x_stabs, 'Z': z_stabs}

    def print_stabilizers(self):
        """Print all stabilizers."""
        stabs = self.get_stabilizers()

        print("\n7-Qubit Color Code Stabilizers")
        print("=" * 50)

        print("\nX-type stabilizers:")
        for s in stabs['X']:
            qubits = ''.join([f"X{q}" for q in s['qubits']])
            print(f"  {s['color']:5}: {qubits}")

        print("\nZ-type stabilizers:")
        for s in stabs['Z']:
            qubits = ''.join([f"Z{q}" for q in s['qubits']])
            print(f"  {s['color']:5}: {qubits}")


code = SevenQubitColorCode()
code.visualize()
code.print_stabilizers()
```

## Transversal Clifford Gates

```
Why Color Codes Enable Transversal Cliffords
═════════════════════════════════════════════

TRANSVERSAL GATE: Apply same single-qubit gate to each physical qubit.

For color codes, all Clifford gates are transversal:

1. TRANSVERSAL X:
   X_L = X⊗X⊗X⊗X⊗X⊗X⊗X (all 7 qubits)

2. TRANSVERSAL Z:
   Z_L = Z⊗Z⊗Z⊗Z⊗Z⊗Z⊗Z (all 7 qubits)

3. TRANSVERSAL H (Hadamard):
   H_L = H⊗H⊗H⊗H⊗H⊗H⊗H

   Why it works: H swaps X↔Z stabilizers
   X stabilizer X₀X₁X₂ → Z₀Z₁Z₂ (also a stabilizer!)
   The code is self-dual under H.

4. TRANSVERSAL S (Phase gate):
   S_L = S⊗S⊗S⊗S⊗S⊗S⊗S

   Why it works: S: X→Y, Z→Z
   Stabilizer structure preserved.

5. TRANSVERSAL CNOT:
   Between two copies: CNOT each qubit i to corresponding qubit i

Compare to surface code:
┌─────────────────────────────────────────────────────────────────┐
│  SURFACE CODE: Only CNOT is transversal                        │
│  COLOR CODE: All Clifford gates are transversal!               │
│                                                                 │
│  This is the main advantage of color codes                     │
└─────────────────────────────────────────────────────────────────┘
```

## General Color Codes

```python
class ColorCode:
    """
    General color code on 3-colorable lattice.
    """

    def __init__(self, lattice_type: str = 'hexagonal'):
        self.lattice_type = lattice_type

    def explain_structure(self):
        """Explain general color code structure."""
        print("""
General Color Code Structure
════════════════════════════

Building a color code:

1. START with a 3-colorable 2D lattice
   - Every face can be colored R, G, or B
   - No two adjacent faces share a color

2. PLACE qubits on VERTICES
   - Each vertex is where 3+ faces meet

3. DEFINE stabilizers from faces:
   For each face F:
   - X-stabilizer: ∏_{v ∈ F} X_v
   - Z-stabilizer: ∏_{v ∈ F} Z_v

4. LOGICAL operators span the code:
   - X_L: String of X operators
   - Z_L: String of Z operators
   - Both must connect boundaries of matching colors

Face-color to stabilizer mapping:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    RED faces → Red X-stabilizers, Red Z-stabilizers
    GREEN faces → Green X-stabilizers, Green Z-stabilizers
    BLUE faces → Blue X-stabilizers, Blue Z-stabilizers

Each qubit is in exactly 3 X-stabilizers and 3 Z-stabilizers
(one of each color).
        """)

    def lattice_comparison(self):
        """Compare different color code lattices."""
        print("""
Color Code Lattice Comparison
═════════════════════════════

HEXAGONAL (6.6.6):
    - 6-sided faces only
    - Weight-6 stabilizers
    - Simple geometry
    - Common choice for 2D implementations

    Qubits per distance d: ~(3/4)d²
    Distance: Related to linear dimension

SQUARE-OCTAGON (4.8.8):
    - 4-sided and 8-sided faces
    - Weight-4 and weight-8 stabilizers
    - Matches square grid hardware
    - Used in some experiments

TRIANGULAR (3.12.12):
    - 3-sided and 12-sided faces
    - Weight-3 and weight-12 stabilizers
    - Higher stabilizer weight

Threshold comparison:
┌─────────────────┬───────────┬───────────────┐
│ Lattice         │ Threshold │ Note          │
├─────────────────┼───────────┼───────────────┤
│ 6.6.6           │ ~0.11%    │ Circuit level │
│ 4.8.8           │ ~0.08%    │ Circuit level │
│ Surface code    │ ~1.0%     │ Circuit level │
└─────────────────┴───────────┴───────────────┘

Color codes have lower thresholds but transversal Cliffords!
        """)


color = ColorCode()
color.explain_structure()
color.lattice_comparison()
```

## String Operators and Boundaries

```
Logical Operators in Color Codes
════════════════════════════════

Logical operators are STRING operators connecting boundaries:

            RED boundary
    ╱ ╲   ╱ ╲   ╱ ╲   ╱ ╲
   │ R │ G │ R │ G │ R │
    ╲ ╱ ╲ ╱ ╲ ╱ ╲ ╱ ╲ ╱
G    │ B │ R │ B │ R │    G
r   ╱ ╲ ╱ ╲ ╱ ╲ ╱ ╲ ╱ ╲   r
e  │ G │ B │ G │ B │ G │  e
e   ╲ ╱ ╲ ╱ ╲ ╱ ╲ ╱ ╲ ╱   e
n    │ R │═══════│ R │    n
    ╱ ╲ ╱ ╲ Z_L ╱ ╲ ╱ ╲
         └───────┘
          BLUE boundary

Z_L string from BLUE boundary to BLUE boundary
(Must not cross RED or GREEN boundaries)

String operators:
    - Z string terminates at same-color boundaries
    - X string terminates at complementary-color boundaries
    - Logical operators cross between paired boundaries

Color code boundaries:
┌─────────────────────────────────────────────────────────────────┐
│  3 boundary types (R, G, B) vs 2 for surface code              │
│  More complex boundary structure                                │
│  But enables richer code deformations                          │
└─────────────────────────────────────────────────────────────────┘
```

## Error Correction

```python
class ColorCodeErrorCorrection:
    """
    Error correction in color codes.
    """

    def __init__(self):
        pass

    def explain_syndromes(self):
        """Explain color code syndromes."""
        print("""
Color Code Syndromes
════════════════════

Each error creates syndromes at adjacent faces:

Single X error at vertex v:
    Triggers Z-stabilizer on all faces containing v
    For hexagonal: 3 faces (one of each color!)

       ╱ R ╲
      ●─────●
     ╱ G ╲X╱ B ╲    X error at center vertex
    ●─────●─────●   Syndrome at R, G, and B faces
     ╲   ╱ ╲   ╱
      ╲ ╱   ╲ ╱

    Z-syndrome: (R=1, G=1, B=1) for this error

Single Z error:
    Triggers X-stabilizers similarly

Y error = XZ:
    Triggers both X and Z syndromes at same faces

Key property:
    EVERY error creates syndrome in ALL THREE colors!
    This constrains where errors can be.

Decoding:
    Match syndromes of each color separately
    Combine to get consistent correction
        """)

    def explain_decoding(self):
        """Explain color code decoding."""
        print("""
Color Code Decoding
═══════════════════

COLOR CODE DECODING is more complex than surface code:

Option 1: PROJECT to surface code
    Ignore one color, treat as surface code
    Decode with MWPM
    Repeat for another color, verify consistency

Option 2: UNION-FIND based
    Generalize union-find to handle 3-colorable structure
    Faster but more complex to implement

Option 3: RESTRICTION DECODER
    Use the 3-color structure directly
    Most accurate but computationally intensive

Performance:
    Lower threshold than surface code (~0.1% vs ~1%)
    But errors detected at all 3 color syndromes
    May be advantageous in some noise models

Trade-off:
┌─────────────────────────────────────────────────────────────────┐
│  SURFACE CODE: Higher threshold, simpler decoding              │
│  COLOR CODE: Lower threshold, but transversal Cliffords        │
│                                                                 │
│  Choice depends on application requirements                    │
└─────────────────────────────────────────────────────────────────┘
        """)


ecc = ColorCodeErrorCorrection()
ecc.explain_syndromes()
ecc.explain_decoding()
```

## Comparison with Surface Codes

```python
class ColorVsSurface:
    """
    Compare color codes to surface codes.
    """

    def comparison_table(self):
        """Print comparison table."""
        print("""
Color Code vs Surface Code Comparison
═════════════════════════════════════

┌────────────────────┬─────────────────┬─────────────────┐
│ Property           │ Surface Code    │ Color Code      │
├────────────────────┼─────────────────┼─────────────────┤
│ Qubits on          │ Edges           │ Vertices        │
│ Stabilizers on     │ Vertices/Faces  │ Faces (both)    │
│ Stabilizer weight  │ 4 (bulk)        │ 6+ (hexagonal)  │
│ Circuit threshold  │ ~1.0%           │ ~0.1%           │
│ Transversal gates  │ CNOT only       │ All Clifford    │
│ Logical qubits     │ 1 (planar)      │ 1 (patch)       │
│ Decoding           │ MWPM (simple)   │ More complex    │
│ Boundary types     │ 2 (rough/smooth)│ 3 (R/G/B)       │
│ Self-dual          │ No              │ Yes (under H)   │
└────────────────────┴─────────────────┴─────────────────┘

When to use color codes:
    - Need transversal Clifford gates
    - Noise rates are very low
    - Gate count is the bottleneck

When to use surface codes:
    - Higher noise environment
    - Simple decoder needed
    - CNOT-based computation sufficient
        """)

comparison.comparison_table()
```

## Summary

Color codes offer unique advantages in topological quantum error correction:

1. **Structure**: Qubits on vertices of 3-colorable lattices
2. **Transversal Cliffords**: All Clifford gates implementable transversally
3. **Three colors**: Stabilizers and boundaries organized by color
4. **Trade-off**: Lower threshold than surface codes
5. **Applications**: Best when low error rates and Clifford gates dominate

Color codes complement surface codes in the toolkit for fault-tolerant quantum computing.
