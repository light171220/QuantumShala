# The Toric Code

## Introduction

The toric code, introduced by Alexei Kitaev in 1997, was the first topological quantum error-correcting code. By placing qubits on the edges of a lattice on a torus, the code encodes two logical qubits in topological properties that are naturally protected from local errors.

## Toric Code Construction

```
Building the Toric Code
═══════════════════════

The toric code is defined on a 2D square lattice with PERIODIC
boundary conditions (i.e., the lattice wraps around to form a torus).

For an L×L torus:

    ●═══●═══●═══●═══●═══● ─┐
    ║   ║   ║   ║   ║   ║  │
    ●═══●═══●═══●═══●═══● ─│── Periodic
    ║   ║   ║   ║   ║   ║  │   vertically
    ●═══●═══●═══●═══●═══● ─│
    ║   ║   ║   ║   ║   ║  │
    ●═══●═══●═══●═══●═══● ─┘
    │                   │
    └───────────────────┘
     Periodic horizontally

Elements:
    ● = Vertex (L² total)
    ═ or ║ = Edge with qubit (2L² total)
    Each face is a plaquette (L² total)

Key difference from planar code:
┌─────────────────────────────────────────────────────────────────┐
│  PLANAR:  Open boundaries → 1 logical qubit                    │
│  TORIC:   Periodic boundaries → 2 logical qubits               │
│                                                                 │
│  The two "holes" of the torus give two independent cycles      │
└─────────────────────────────────────────────────────────────────┘
```

## Stabilizers on the Torus

```python
import numpy as np
from typing import List, Tuple, Set, Dict

class ToricCode:
    """
    Implement the toric code on an L×L torus.
    """

    def __init__(self, L: int):
        """
        Create L×L toric code.

        Qubit indexing:
        - Horizontal edges: (row, col, 'h') for edge right of (row, col)
        - Vertical edges: (row, col, 'v') for edge below (row, col)
        """
        self.L = L
        self.n_qubits = 2 * L * L  # 2L² edges

    def vertex_stabilizer(self, row: int, col: int) -> List[Tuple[int, int, str]]:
        """
        Get edges adjacent to vertex (row, col).
        X-stabilizer: product of X on these edges.
        """
        L = self.L
        edges = [
            (row, col, 'h'),                    # Edge to the right
            (row, (col - 1) % L, 'h'),          # Edge to the left
            (row, col, 'v'),                    # Edge below
            ((row - 1) % L, col, 'v'),          # Edge above
        ]
        return edges

    def plaquette_stabilizer(self, row: int, col: int) -> List[Tuple[int, int, str]]:
        """
        Get edges around plaquette (row, col).
        Z-stabilizer: product of Z on these edges.
        """
        L = self.L
        edges = [
            (row, col, 'h'),                    # Top edge
            ((row + 1) % L, col, 'h'),          # Bottom edge
            (row, col, 'v'),                    # Left edge
            (row, (col + 1) % L, 'v'),          # Right edge
        ]
        return edges

    def print_stabilizers(self):
        """Display stabilizer structure."""
        L = self.L
        print(f"\nToric Code Stabilizers (L={L})")
        print("=" * 50)

        print(f"\nVertex (X) stabilizers: {L*L}")
        print("Each acts on 4 edges meeting at a vertex")
        print("  Example at (0,0):", self.vertex_stabilizer(0, 0))

        print(f"\nPlaquette (Z) stabilizers: {L*L}")
        print("Each acts on 4 edges around a face")
        print("  Example at (0,0):", self.plaquette_stabilizer(0, 0))

        print(f"\nTotal stabilizers: {2 * L * L}")
        print(f"But 2 constraints: product of all X-stab = I")
        print(f"                   product of all Z-stab = I")
        print(f"Independent stabilizers: {2 * L * L - 2}")

        n = self.n_qubits
        k = n - (2 * L * L - 2)
        print(f"\nLogical qubits: k = {n} - {2*L*L-2} = {k}")

    def visualize_torus(self):
        """Visualize the toric code structure."""
        print(f"""
Toric Code on {self.L}×{self.L} Torus
═══════════════════════════

Conceptual view (unfolded with periodic identification):

    Column:   0     1     2    ...   L-1    0
           ┌─────────────────────────────────┐
    Row 0: │  ●═h═●═h═●═h═  ...  ●═h═●═h═●  │
           │  v   v   v         v   v   v   │
    Row 1: │  ●═h═●═h═●═h═  ...  ●═h═●═h═●  │
           │  v   v   v         v   v   v   │
      ...  │  :   :   :         :   :   :   │
           │  v   v   v         v   v   v   │
   Row L-1:│  ●═h═●═h═●═h═  ...  ●═h═●═h═●  │
           │  v   v   v         v   v   v   │
    Row 0: │  ●═h═●═h═●═h═  ...  ●═h═●═h═●  │  ← Wraps to top
           └─────────────────────────────────┘
                                     └── Wraps to left

    h = horizontal edge (qubit)
    v = vertical edge (qubit)
    ● = vertex (X-stabilizer site)
    Plaquettes between vertices (Z-stabilizer sites)
        """)


toric = ToricCode(4)
toric.print_stabilizers()
toric.visualize_torus()
```

## Two Logical Qubits

```
The Two Logical Qubits of the Toric Code
════════════════════════════════════════

The torus has two independent non-contractible cycles:

    HORIZONTAL CYCLE          VERTICAL CYCLE
    (around the "waist")      (through the "hole")

    ╭─────────────╮           ╭─────────────╮
    │ →→→→→→→→→→→ │           │  ↓       ↓  │
    │             │           │  ↓       ↓  │
    │             │           │  ↓       ↓  │
    ╰─────────────╯           ╰──↓───────↓──╯

These give TWO independent logical qubits:

    Logical qubit 1:           Logical qubit 2:
    Z₁ = Z on horizontal loop  Z₂ = Z on vertical loop
    X₁ = X on vertical path    X₂ = X on horizontal path

Operator definitions (for L×L code):

    Z₁ᴸ: Z on all horizontal edges in row 0
        = Z_{(0,0,h)} Z_{(0,1,h)} ... Z_{(0,L-1,h)}

    X₁ᴸ: X on all vertical edges in column 0
        = X_{(0,0,v)} X_{(1,0,v)} ... X_{(L-1,0,v)}

    Z₂ᴸ: Z on all vertical edges in column 0
        = Z_{(0,0,v)} Z_{(1,0,v)} ... Z_{(L-1,0,v)}

    X₂ᴸ: X on all horizontal edges in row 0
        = X_{(0,0,h)} X_{(0,1,h)} ... X_{(0,L-1,h)}

Note: X₁ᴸ and X₂ᴸ overlap at one edge → they commute
      Z₁ᴸ and Z₂ᴸ overlap at one edge → they commute
      But X₁ᴸ anticommutes with Z₁ᴸ (share 1 qubit)
      And X₂ᴸ anticommutes with Z₂ᴸ (share 1 qubit)
```

## Logical Operators in Detail

```python
class ToricCodeLogical:
    """
    Define and analyze logical operators for toric code.
    """

    def __init__(self, L: int):
        self.L = L

    def logical_z1(self) -> List[Tuple[int, int, str]]:
        """Z₁: Horizontal loop of Z operators."""
        return [(0, col, 'h') for col in range(self.L)]

    def logical_x1(self) -> List[Tuple[int, int, str]]:
        """X₁: Vertical path of X operators."""
        return [(row, 0, 'v') for row in range(self.L)]

    def logical_z2(self) -> List[Tuple[int, int, str]]:
        """Z₂: Vertical loop of Z operators."""
        return [(row, 0, 'v') for row in range(self.L)]

    def logical_x2(self) -> List[Tuple[int, int, str]]:
        """X₂: Horizontal path of X operators."""
        return [(0, col, 'h') for col in range(self.L)]

    def verify_logical_operators(self):
        """Verify logical operator properties."""
        print(f"\nLogical Operator Verification (L={self.L})")
        print("=" * 55)

        z1 = set(self.logical_z1())
        x1 = set(self.logical_x1())
        z2 = set(self.logical_z2())
        x2 = set(self.logical_x2())

        print("\nOperator weights:")
        print(f"  |Z₁ᴸ| = {len(z1)}, |X₁ᴸ| = {len(x1)}")
        print(f"  |Z₂ᴸ| = {len(z2)}, |X₂ᴸ| = {len(x2)}")

        print("\nOverlaps (determines commutation):")
        print(f"  Z₁ᴸ ∩ X₁ᴸ: {len(z1 & x1)} edges → ", end="")
        print("ANTICOMMUTE" if len(z1 & x1) % 2 == 1 else "COMMUTE")

        print(f"  Z₂ᴸ ∩ X₂ᴸ: {len(z2 & x2)} edges → ", end="")
        print("ANTICOMMUTE" if len(z2 & x2) % 2 == 1 else "COMMUTE")

        print(f"  Z₁ᴸ ∩ X₂ᴸ: {len(z1 & x2)} edges → ", end="")
        print("ANTICOMMUTE" if len(z1 & x2) % 2 == 1 else "COMMUTE")

        print(f"  Z₂ᴸ ∩ X₁ᴸ: {len(z2 & x1)} edges → ", end="")
        print("ANTICOMMUTE" if len(z2 & x1) % 2 == 1 else "COMMUTE")

        print("\nExpected for two independent logical qubits:")
        print("  X₁, Z₁ anticommute; X₂, Z₂ anticommute")
        print("  All other pairs commute")

    def show_equivalent_operators(self):
        """Show equivalent logical operators."""
        print(f"""
Equivalent Logical Operators
════════════════════════════

Different rows/columns give equivalent logical operators:

Z₁ᴸ via row 0:  ─Z─Z─Z─Z─Z─
                           } Differ by plaquette
Z₁ᴸ via row 1:  ─Z─Z─Z─Z─Z─  stabilizers

Any horizontal Z-loop is equivalent to Z₁ᴸ!

Similarly:
    - Any vertical Z-loop ∼ Z₂ᴸ
    - Any vertical X-path ∼ X₁ᴸ
    - Any horizontal X-path ∼ X₂ᴸ

This is the topological nature:
    The HOMOLOGY CLASS matters, not the specific path.
        """)


logical = ToricCodeLogical(5)
logical.verify_logical_operators()
logical.show_equivalent_operators()
```

## Code Parameters

```python
class ToricCodeParameters:
    """
    Compute and analyze toric code parameters.
    """

    def compute_parameters(self, L: int) -> Dict:
        """Compute [[n, k, d]] for L×L toric code."""
        n = 2 * L * L  # Edges on L×L torus
        k = 2          # Always 2 logical qubits (genus 1)
        d = L          # Distance = minimum of L (loop size)

        return {
            'L': L,
            'n': n,
            'k': k,
            'd': d,
            'rate': k / n,
            'stabilizers': 2 * L * L - 2
        }

    def print_parameter_table(self, max_L: int = 10):
        """Print parameter table."""
        print("\nToric Code Parameters [[n, k, d]]")
        print("=" * 60)
        print(f"{'L':^5} {'n':^10} {'k':^5} {'d':^5} {'Rate':^12} {'Stabilizers':^12}")
        print("-" * 60)

        for L in range(2, max_L + 1):
            p = self.compute_parameters(L)
            print(f"{L:^5} {p['n']:^10} {p['k']:^5} {p['d']:^5} "
                  f"{p['rate']:^12.4f} {p['stabilizers']:^12}")

    def compare_to_planar(self, d: int):
        """Compare toric to planar surface code at same distance."""
        print(f"\nToric vs Planar Comparison (distance d={d})")
        print("=" * 55)

        # Toric code
        n_toric = 2 * d * d
        k_toric = 2
        rate_toric = k_toric / n_toric

        # Planar code
        n_planar = d * d
        k_planar = 1
        rate_planar = k_planar / n_planar

        print(f"\n{'Property':<20} {'Toric':^15} {'Planar':^15}")
        print("-" * 55)
        print(f"{'Data qubits n':<20} {n_toric:^15} {n_planar:^15}")
        print(f"{'Logical qubits k':<20} {k_toric:^15} {k_planar:^15}")
        print(f"{'Distance d':<20} {d:^15} {d:^15}")
        print(f"{'Encoding rate k/n':<20} {rate_toric:^15.4f} {rate_planar:^15.4f}")
        print(f"{'Boundaries':<20} {'Periodic':^15} {'Open':^15}")


params = ToricCodeParameters()
params.print_parameter_table()
params.compare_to_planar(5)
```

## Error Correction on the Torus

```
Error Correction in the Toric Code
══════════════════════════════════

Error chains on a torus have special properties:

1. OPEN CHAINS (Detectable errors)
   ───────────────────────────────
   Create syndrome pairs, just like planar code

       ●═══●═══●═══●
       ║   ║ X ║   ║
       ●═══●═══●═══●
       ║   ★   ★   ║    Two syndromes
       ●═══●═══●═══●

2. TRIVIAL CLOSED LOOPS (Stabilizers)
   ──────────────────────────────────
   Contractible loops = stabilizers = no effect

       ●═══●═══●═══●
       ║   X═X ║   ║
       ●═══X═X═●═══●    This is a stabilizer!
       ║   ║   ║   ║    No syndrome, no logical effect
       ●═══●═══●═══●

3. NON-TRIVIAL LOOPS (Logical operators)
   ─────────────────────────────────────
   Loops wrapping around torus = logical operators!

       ●═X═●═X═●═X═●═X═● ─┐
       ║   ║   ║   ║   ║  │
       ●═══●═══●═══●═══● ─│ Wraps around
       ║   ║   ║   ║   ║  │ horizontally
       ●═══●═══●═══●═══● ─┘

   No syndrome, but flips logical qubit!

Decoding challenge:
┌─────────────────────────────────────────────────────────────────┐
│  Given syndrome, find minimum-weight correction                │
│  Must avoid accidentally completing a non-trivial loop!        │
│  Same MWPM decoder works (with periodic boundary handling)     │
└─────────────────────────────────────────────────────────────────┘
```

## Implementation

```python
class ToricCodeSimulator:
    """
    Simulate the toric code with errors and correction.
    """

    def __init__(self, L: int):
        self.L = L
        self.n_qubits = 2 * L * L

        # Qubit states (0 = no error, 1 = error)
        self.x_errors = np.zeros((L, L, 2), dtype=int)  # [row, col, h/v]
        self.z_errors = np.zeros((L, L, 2), dtype=int)

    def edge_index(self, row: int, col: int, direction: str) -> Tuple[int, int, int]:
        """Convert edge to array index."""
        d = 0 if direction == 'h' else 1
        return (row % self.L, col % self.L, d)

    def apply_x_error(self, row: int, col: int, direction: str):
        """Apply X error to an edge."""
        idx = self.edge_index(row, col, direction)
        self.x_errors[idx] ^= 1

    def apply_z_error(self, row: int, col: int, direction: str):
        """Apply Z error to an edge."""
        idx = self.edge_index(row, col, direction)
        self.z_errors[idx] ^= 1

    def get_x_syndrome(self) -> np.ndarray:
        """Compute X syndrome (from Z errors) at each vertex."""
        L = self.L
        syndrome = np.zeros((L, L), dtype=int)

        for row in range(L):
            for col in range(L):
                # Sum Z errors on adjacent edges
                s = 0
                s ^= self.z_errors[row, col, 0]              # Right
                s ^= self.z_errors[row, (col-1) % L, 0]      # Left
                s ^= self.z_errors[row, col, 1]              # Down
                s ^= self.z_errors[(row-1) % L, col, 1]      # Up
                syndrome[row, col] = s

        return syndrome

    def get_z_syndrome(self) -> np.ndarray:
        """Compute Z syndrome (from X errors) at each plaquette."""
        L = self.L
        syndrome = np.zeros((L, L), dtype=int)

        for row in range(L):
            for col in range(L):
                # Sum X errors around plaquette
                s = 0
                s ^= self.x_errors[row, col, 0]              # Top
                s ^= self.x_errors[(row+1) % L, col, 0]      # Bottom
                s ^= self.x_errors[row, col, 1]              # Left
                s ^= self.x_errors[row, (col+1) % L, 1]      # Right
                syndrome[row, col] = s

        return syndrome

    def print_state(self):
        """Print current error state and syndromes."""
        print(f"\nToric Code State (L={self.L})")
        print("=" * 40)

        x_syn = self.get_x_syndrome()
        z_syn = self.get_z_syndrome()

        print(f"X errors: {np.sum(self.x_errors)}")
        print(f"Z errors: {np.sum(self.z_errors)}")
        print(f"X syndrome defects: {np.sum(x_syn)}")
        print(f"Z syndrome defects: {np.sum(z_syn)}")


# Example
sim = ToricCodeSimulator(4)
sim.apply_x_error(1, 2, 'h')
sim.apply_x_error(1, 3, 'h')
sim.print_state()
```

## Summary

The toric code is a foundational topological quantum code:

1. **Structure**: Qubits on edges of a torus, stabilizers at vertices and plaquettes
2. **Two logical qubits**: Comes from the genus-1 topology (two non-contractible cycles)
3. **Logical operators**: Non-trivial loops wrapping around the torus
4. **Distance**: L for an L×L torus (size of smallest loop)
5. **Comparison**: More qubits but encodes 2 logical qubits vs 1 for planar

The toric code demonstrates how topology naturally provides error protection and has inspired a wide family of topological codes.
