# Introduction to Topological Codes

## Introduction

Topological quantum codes encode information in global properties of a quantum system that are immune to local perturbations. By exploiting topology—the study of properties preserved under continuous deformations—these codes achieve remarkable error protection with simple local operations.

## What Makes Codes Topological?

```
The Topological Protection Principle
════════════════════════════════════

A code is TOPOLOGICAL if:

1. LOGICAL INFORMATION IS GLOBAL
   ────────────────────────────
   Encoded in non-local properties of the system
   No local measurement can reveal the logical state

   Example: Parity of a loop that wraps around a torus
            Cannot be determined by looking at any small region

2. LOCAL ERRORS ARE DETECTED
   ─────────────────────────
   Any local perturbation creates a detectable signature
   Small errors cannot silently corrupt the logical state

   Example: Cutting a loop creates endpoints (syndromes)
            The system "knows" something changed

3. LOGICAL ERRORS REQUIRE GLOBAL CHANGES
   ─────────────────────────────────────
   Corrupting logical information requires non-local action
   Error must span the system to cause logical failure

   Example: Changing loop parity requires adding/removing
            a complete loop—impossible with local changes

Visual intuition:

    LOCAL ERROR              LOGICAL ERROR
    (detectable)             (undetectable)

    ●═══●═══●═══●            ●═══●═══●═══●
    ║   X   ║   ║     vs     X   X   X   X
    ●═══●═══●═══●            ●═══●═══●═══●
    ↑                        └───────────┘
    Creates                  Complete loop
    syndromes                No syndromes!
```

   ───────────

   Disk (bounded):    Annulus:         Planar surface code:
   ╭─────╮           ╭─────────╮       ╭─────────────────╮

    def genus_to_logical_qubits(self, genus: int, boundaries: int = 0) -> int:
        """Calculate logical qubits from topology."""
        # For closed surface: k = 2g
        # Each boundary adds potential for more logical structure
        return 2 * genus + max(0, boundaries - 1)

## Topological Invariants

```python
class TopologicalInvariants:
    """
    Compute topological invariants relevant to quantum codes.
    """

    def euler_characteristic(self, vertices: int, edges: int, faces: int) -> int:
        """Compute Euler characteristic χ = V - E + F."""
        return vertices - edges + faces

    def genus_from_euler(self, chi: int, boundaries: int = 0) -> float:
        """Compute genus from Euler characteristic."""
        # χ = 2 - 2g - b for surface with b boundaries
        return (2 - chi - boundaries) / 2

    def analyze_lattice(self, name: str, V: int, E: int, F: int, b: int = 0):
        """Analyze topological properties of a code lattice."""
        chi = self.euler_characteristic(V, E, F)
        g = self.genus_from_euler(chi, b)

        print(f"\nTopological Analysis: {name}")
        print("=" * 50)
        print(f"Vertices: {V}, Edges: {E}, Faces: {F}")
        print(f"Boundaries: {b}")
        print(f"Euler characteristic: χ = {chi}")
        print(f"Genus: g = {g}")

        # Logical qubits
        if b == 0:  # Closed surface
            k = int(2 * g)
        else:  # Open surface
            k = 1  # For planar code with opposite boundary types

        print(f"Logical qubits: k = {k}")
        print(f"Data qubits (edges): n = {E}")

        return {'chi': chi, 'genus': g, 'k': k, 'n': E}

    def compare_topologies(self):
        """Compare different code topologies."""
        print("\nComparing Code Topologies")
        print("=" * 60)

        # 5×5 toric code
        self.analyze_lattice("5×5 Toric Code", V=25, E=50, F=25, b=0)

        # 5×5 planar code
        self.analyze_lattice("5×5 Planar Code", V=25, E=40, F=16, b=4)

        # Hexagonal color code (7 qubits)
        self.analyze_lattice("7-qubit Color Code", V=7, E=12, F=6, b=1)


invariants = TopologicalInvariants()
invariants.compare_topologies()
```

## Anyons and Topological Order

```
Anyons: Particles of Topological Codes
══════════════════════════════════════

WHAT ARE ANYONS?
    Particle-like excitations in 2D topological systems
    Neither bosons nor fermions—they're "any-ons"!

    When two anyons are exchanged (braided):
    - Bosons: ψ → ψ (no change)
    - Fermions: ψ → -ψ (sign flip)
    - Anyons: ψ → e^(iθ)ψ (arbitrary phase)
    - Non-abelian anyons: ψ → U ψ (unitary transformation!)

IN SURFACE CODES:
    Syndromes ARE anyons!

    e-anyon: X-syndrome defect (electric charge)
    m-anyon: Z-syndrome defect (magnetic flux)

    Properties:
    ┌─────────────────────────────────────────────────────────┐
    │  e × e = 1  (two e-anyons can annihilate)              │
    │  m × m = 1  (two m-anyons can annihilate)              │
    │  e × m = ε  (bound state is a fermion!)                │
    │  Moving e around m → -1 phase (mutual statistics)      │
    └─────────────────────────────────────────────────────────┘

BRAIDING = ERROR CORRECTION
    Creating, moving, and annihilating anyons corresponds to:
    - Creating: Error occurs
    - Moving: Error chain extends
    - Annihilating: Error correction

       e ●                    ● e
         ╲                  ╱
          ╲   Error       ╱
           ╲  Chain     ╱
            ╲         ╱
             ●═══════●
             Annihilate!
             (Correction)
```

## Code Parameters from Topology

```python
class CodeFromTopology:
    """
    Derive code parameters from topological properties.
    """

    def surface_code_params(self, d: int, topology: str = 'planar') -> dict:
        """
        Compute surface code parameters from topology and distance.

        Args:
            d: Code distance
            topology: 'planar' or 'toric'
        """
        if topology == 'planar':
            n = d * d  # Data qubits
            k = 1      # Logical qubits
        else:  # toric
            n = 2 * d * d  # Data qubits (edges of d×d torus)
            k = 2          # Logical qubits (genus 1 surface)

        return {
            'name': f"[[{n}, {k}, {d}]] {topology} surface code",
            'n': n,
            'k': k,
            'd': d,
            'rate': k / n,
            'topology': topology
        }

    def color_code_params(self, structure: str) -> dict:
        """
        Compute color code parameters.

        Args:
            structure: '4.8.8', '6.6.6', or 'steane'
        """
        if structure == 'steane':
            return {'name': "[[7, 1, 3]] Steane/color code", 'n': 7, 'k': 1, 'd': 3}
        elif structure == '4.8.8':
            # 4.8.8 lattice color code
            return {'name': "4.8.8 color code", 'note': "d² qubits for distance d"}
        else:
            return {'name': f"{structure} color code"}

    def print_comparison(self):
        """Compare topological codes."""
        print("\nTopological Code Comparison")
        print("=" * 70)

        codes = [
            self.surface_code_params(3, 'planar'),
            self.surface_code_params(5, 'planar'),
            self.surface_code_params(3, 'toric'),
            self.surface_code_params(5, 'toric'),
            self.color_code_params('steane')
        ]

        print(f"{'Code':^35} {'n':^8} {'k':^5} {'d':^5} {'Rate':^10}")
        print("-" * 70)

        for code in codes:
            if 'rate' in code:
                print(f"{code['name']:^35} {code['n']:^8} {code['k']:^5} "
                      f"{code['d']:^5} {code['rate']:^10.4f}")


params = CodeFromTopology()
params.print_comparison()
```

## Why Topological Codes?

```
Advantages of Topological Codes
═══════════════════════════════

1. HIGH ERROR THRESHOLDS
   ─────────────────────
   Surface codes: ~1% threshold (circuit level)
   Compare: Concatenated codes ~10^-4

   Why? Local structure of stabilizers matches
   local structure of errors

2. LOCAL STABILIZERS
   ──────────────────
   All stabilizers involve only nearby qubits
   Perfect for 2D hardware layouts

   ●═══●═══●       No long-range
   ║   ║   ║       connections
   ●═══●═══●       needed!
   ║   ║   ║
   ●═══●═══●

3. NATURAL ERROR MODEL FIT
   ────────────────────────
   Errors in hardware are local
   Code structure naturally filters local errors

4. SCALABLE DECODING
   ──────────────────
   Efficient decoders exist (MWPM, Union-Find)
   Decoding time scales polynomially

5. ROBUST TO IMPERFECTIONS
   ────────────────────────
   Code still works with missing qubits, varied connectivity
   Adapts to hardware defects

Challenges:
┌─────────────────────────────────────────────────────────────────┐
│  • Low encoding rate (1/d² for surface codes)                  │
│  • Non-transversal non-Clifford gates                          │
│  • Need for magic state distillation                           │
│  • Large qubit overhead for useful computation                 │
└─────────────────────────────────────────────────────────────────┘
```

## Summary

Topological codes represent a powerful paradigm for quantum error correction:

1. **Global encoding**: Logical information in topological properties immune to local errors
2. **Local stabilizers**: All measurements are local, matching hardware constraints
3. **High thresholds**: Surface codes tolerate ~1% errors per gate
4. **Anyon physics**: Syndrome defects are anyonic excitations
5. **Code families**: Surface codes, color codes, and more exotic constructions

This topological perspective provides both practical advantages and deep theoretical insights into quantum error correction.
