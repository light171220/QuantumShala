# Topological Quantum Computation

## Introduction

While topological codes provide excellent error protection, performing computation requires manipulating logical qubits. This lesson explores how to implement fault-tolerant logical gates using the topological structure of surface codes, including lattice surgery, code deformation, and magic state injection.

## The Gate Challenge

```
Logical Gates in Topological Codes
══════════════════════════════════

CHALLENGE: How do we compute with encoded qubits?

We need a UNIVERSAL gate set:
    - Clifford gates: H, S, CNOT
    - Non-Clifford: T gate (or equivalent)

For SURFACE CODES:
    ✓ Transversal CNOT (between aligned codes)
    ✗ Transversal H (not directly)
    ✗ Transversal S (not directly)
    ✗ Transversal T (impossible by Eastin-Knill theorem)

SOLUTIONS:
    1. LATTICE SURGERY: Merge/split codes for CNOT, measurements
    2. CODE DEFORMATION: Twist boundaries for Hadamard
    3. MAGIC STATES: Inject T gates via special states

For COLOR CODES:
    ✓ Transversal H, S, CNOT (all Clifford)
    ✗ Transversal T (still impossible)

The Eastin-Knill theorem:
┌─────────────────────────────────────────────────────────────────┐
│  No quantum error-correcting code can have a universal set     │
│  of TRANSVERSAL gates. At least one gate must be non-local.   │
│                                                                 │
│  This is why we need magic states for the T gate!              │
└─────────────────────────────────────────────────────────────────┘
```

class LatticeSurgery:
    """
    Implement lattice surgery operations for surface codes.
    """

    def __init__(self):
        pass

    def explain_surgery(self):
        """Explain lattice surgery concept."""
        print("""
Lattice Surgery: Merging and Splitting Codes
═════════════════════════════════════════════

CORE IDEA: Merge two surface code patches to perform joint
operations, then split them back.

MERGE (ZZ measurement):
──────────────────────
    Start with two separate patches:

    ┌─────────┐     ┌─────────┐
    │  Code A │     │  Code B │
    │         │  →  │         │
    │  |ψ_A⟩  │     │  |ψ_B⟩  │
    └─────────┘     └─────────┘

    Merge boundaries:

    ┌─────────────────────────┐
    │  Code A  ══════  Code B │
    │  |ψ_A⟩   merge   |ψ_B⟩  │
    └─────────────────────────┘

    Effect: Measures Z_A ⊗ Z_B (joint Z parity)

SPLIT (reverse the merge):
──────────────────────────
    After splitting, codes are separate again
    State has been projected by the measurement

Surgical operations:
    XX measurement: Use X-type boundary merge
    ZZ measurement: Use Z-type boundary merge
    Joint operations: Build from these primitives
        """)

    def cnot_via_surgery(self):
        """Show CNOT via lattice surgery."""
        print("""
CNOT via Lattice Surgery
════════════════════════

CNOT = (I⊗H) · CZ · (I⊗H)

Or equivalently using measurements:

    CNOT(A→B) can be done with:
    1. Create ancilla in |+⟩
    2. Measure Z_A ⊗ Z_anc (ZZ surgery)
    3. Measure X_anc ⊗ X_B (XX surgery)
    4. Apply corrections based on outcomes

Visual:
    Step 1: Three patches

    ┌────┐     ┌────┐     ┌────┐
    │ A  │     │anc │     │ B  │
    │    │     │ |+⟩│     │    │
    └────┘     └────┘     └────┘

    Step 2: ZZ merge (A with ancilla)

    ┌─────────────┐     ┌────┐
    │ A ═══ anc   │     │ B  │
    │  ZZ merge   │     │    │
    └─────────────┘     └────┘

    Step 3: XX merge (ancilla with B)

    ┌────┐     ┌─────────────┐
    │ A  │     │ anc ═══ B   │
    │    │     │  XX merge   │
    └────┘     └─────────────┘

    Result: CNOT applied (up to known corrections)

Time cost: ~3d rounds for distance-d code
        """)

    def multi_qubit_parity(self):
        """Explain multi-qubit parity measurements."""
        print("""
Multi-Qubit Parity Measurements
═══════════════════════════════

Lattice surgery naturally measures PARITY operators:

    ZZZ...Z or XXX...X on multiple logical qubits

Layout for ZZZ on three qubits:

    ┌────┐
    │ A  ├──┐
    └────┘  │
            ├──── Merge region
    ┌────┐  │     (ancilla)
    │ B  ├──┤
    └────┘  │
            │
    ┌────┐  │
    │ C  ├──┘
    └────┘

Merge all three with central ancilla
→ Measures Z_A ⊗ Z_B ⊗ Z_C

This is useful for:
    - Syndrome extraction (checking parities)

    - Preparing entangled states

## Code Deformation

```
Code Deformation for Hadamard
═════════════════════════════

The Hadamard gate can be implemented by ROTATING the code:

IDEA: Swap the roles of X and Z boundaries

    Original:               After 90° rotation:

    ═════ Rough ═════       ║ S ║
    ║               ║       ║ m ║
    ║   Surface     ║  →    ║ o ║  Rough→Smooth
    ║   Code        ║       ║ o ║
    ║               ║       ║ t ║
    ═════ Rough ═════       ║ h ║

What this does:
    - X_L (left↔right) becomes Z_L' (top↔bottom)
    - Z_L (top↔bottom) becomes X_L' (left↔right)
    - This IS the Hadamard transformation!

Implementation:
    1. Gradually rotate stabilizers
    2. Adjust boundary conditions
    3. Move qubits smoothly
    4. Takes O(d) code cycles

Alternative: DEFECT BRAIDING
    Create and braid defects (holes in the code)
    Also implements topological gates
    More complex but sometimes more flexible

┌─────────────────────────────────────────────────────────────────┐
│  Code deformation is SLOW but FAULT-TOLERANT                   │
│  Each step maintains code structure                            │
│  No single point of failure                                    │
└─────────────────────────────────────────────────────────────────┘
```

## Magic States

```python
class MagicStates:
    """
    Magic state injection for T gates.
    """

    def __init__(self):
        # Magic state for T gate
        self.T_state = "|T⟩ = (|0⟩ + e^(iπ/4)|1⟩)/√2"

    def explain_magic_states(self):
        """Explain magic state concept."""
        print(f"""
Magic States for Non-Clifford Gates
════════════════════════════════════

THE PROBLEM:
    T gate is non-Clifford
    Cannot be implemented transversally or via surgery alone

THE SOLUTION:
    Prepare special "magic" states
    Use them to implement T via gate teleportation

Magic state for T gate:
    |T⟩ = T|+⟩ = {self.T_state}

    Also written as: |A⟩ = |T⟩ (same state)

Gate teleportation circuit:

    |ψ⟩ ───●───────────────── T|ψ⟩ (after corrections)
           │
    |T⟩ ───X─── H ─── M ───→ If M=1: apply S†Z

    The T gate is "teleported" from the magic state!

Cost:
    One magic state consumed per T gate
    Magic states are expensive to prepare
        """)

    def explain_distillation(self):
        """Explain magic state distillation."""
        print("""
Magic State Distillation
════════════════════════

PROBLEM: Preparing |T⟩ directly gives noisy states!
SOLUTION: DISTILLATION - combine many noisy states to get few clean ones

15-to-1 Protocol:
    Input: 15 noisy |T⟩ states (error rate p_in)
    Output: 1 cleaner |T⟩ state (error rate p_out ≈ 35 p_in³)

    Massive error reduction! But costly.

Distillation hierarchy:
    Level 0: Physical |T⟩ prep, error ~1%
    Level 1: 15→1 distillation, error ~0.003%
    Level 2: 15→1 again, error ~10⁻⁹

Resource cost:
    Each level: 15× more magic states
    Level 2: 15² = 225 physical preps per output
    Huge overhead!

State of the art:
    Better protocols exist (e.g., 20-to-4)
    Active area of research
    Key bottleneck for fault-tolerant QC

┌─────────────────────────────────────────────────────────────────┐
│  T gates are the EXPENSIVE resource in fault-tolerant QC       │
│  Most of the physical qubits go to magic state factories       │
│  Reducing T count is crucial for practical algorithms          │
└─────────────────────────────────────────────────────────────────┘
        """)


magic = MagicStates()
magic.explain_magic_states()
magic.explain_distillation()
```

## Computation Time Analysis

```
Time Costs for Operations
═════════════════════════

For a distance-d surface code:

SYNDROME ROUND: ~1 μs (superconducting qubits)
    - Reset, gates, measurement

OPERATIONS PER CODE CYCLE:

    Operation          Code Cycles    Wall Time (d=10)
    ─────────────────────────────────────────────────
    Pauli gate         1              ~1 μs
    CNOT (surgery)     ~3d            ~30 μs
    Hadamard (deform)  ~d             ~10 μs
    T gate (inject)    ~d             ~10 μs
    ─────────────────────────────────────────────────

For 10⁶ T gates:
    Injection time: 10⁶ × 10 μs = 10 seconds
    Plus Clifford overhead
    Total: ~minutes to hour

Parallelization:
    T gates are sequential bottleneck
    Clifford gates can parallelize
    Layout optimization crucial
```

## Summary

Topological quantum computation combines error correction with logical operations:

1. **Lattice surgery**: Merge/split codes for joint measurements and CNOT
2. **Code deformation**: Rotate or twist codes for Hadamard and S gates
3. **Magic states**: Enable non-Clifford T gates via state injection
4. **Distillation**: Purify noisy magic states through error correction
5. **Resource costs**: T gates dominate the overhead

This framework provides a complete path to fault-tolerant universal quantum computation.
