        """Print threshold comparison."""
        print("\nError Threshold Comparison")

        print("""
NOISE MODEL MATTERS:
────────────────────

   - Only data qubit errors

   Color (hex):   ~10%

   - Syndrome measurement errors
   - But no gate-level errors
   - More realistic

   Color:   ~1-2%

   - Full gate error model
   - Measurement errors
   - Idle errors

        """Describe logical error rate scaling."""

## Gate Implementation Comparison

```
Logical Gate Implementations
════════════════════════════

SURFACE CODE:
─────────────
    Pauli:    Transversal (apply to all)
    CNOT:     Transversal (matching orientation)
              OR Lattice surgery (~3d cycles)
    Hadamard: Code deformation (~d cycles)
    S gate:   Code deformation (~d cycles)
    T gate:   Magic state injection (~d cycles)

COLOR CODE:
───────────
    Pauli:    Transversal
    CNOT:     Transversal
    Hadamard: Transversal!  ← Big advantage
    S gate:   Transversal!  ← Big advantage
    T gate:   Magic state injection

TORIC CODE:
───────────
    Similar to surface code
    Two logical qubits per patch
    Good for CNOT-heavy circuits

COMPARISON FOR ALGORITHM TYPES:
───────────────────────────────

Algorithm Type              Best Code Choice
──────────────────────────────────────────────────
Clifford-heavy (QEC)        Color code
T-gate heavy (Shor)         Surface code (threshold)
Low error budget            Surface code
Very clean hardware         Color code
Simple implementation       Surface code
Two-qubit operations        Toric code
```

## Summary

Topological codes offer various trade-offs:

1. **Surface codes**: Highest threshold, simplest structure, best for near-term
2. **Color codes**: Transversal Cliffords, lower threshold, needs clean hardware
3. **Toric codes**: Two logical qubits, periodic boundary, theoretical importance
4. **Emerging codes**: Floquet, QLDPC, pushing boundaries of what's possible

Choose based on your specific hardware, algorithm, and error budget.
