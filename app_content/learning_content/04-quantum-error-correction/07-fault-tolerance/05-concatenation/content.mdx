# Concatenated Codes

## Introduction

Concatenation is a powerful technique that achieves arbitrarily low logical error rates by recursively encoding quantum information. By encoding a qubit in a code, then encoding each physical qubit of that code in another code, we can suppress errors exponentially with only polynomial overhead.

## The Concatenation Concept

```
What is Concatenation?
══════════════════════

IDEA: Encode qubits recursively, layer by layer

Level 0: Physical qubit
         |ψ⟩

Level 1: Encode in code C₁ (e.g., Steane [[7,1,3]])
         |ψ⟩ → |ψ_L⟩ using 7 physical qubits

Level 2: Each of those 7 qubits → encode again
         49 qubits total (7 × 7)

Level 3: Each of those 49 → encode again
         343 qubits total (7 × 7 × 7)

Level k: n^k physical qubits for one logical qubit


VISUALIZATION (Steane code, 2 levels):

Level 2:   ┌─────────────────────────────────┐
           │         Logical Qubit           │
           └─────────────────────────────────┘
                          │
           ┌──────────────┼──────────────┐
           ▼              ▼              ▼
Level 1: [===]          [===]          [===]    (7 blocks)
          │               │               │
        ┌─┴─┐           ┌─┴─┐           ┌─┴─┐
Level 0: ● ● ● ● ● ● ● ● ● ● ● ● ● ● ● ● ● ● ● ● ●  (49 qubits)

Each [===] is a level-1 encoded qubit
Each ● is a physical qubit
```

## Error Suppression Through Concatenation

```python
class ConcatenationAnalysis:
    """
    Analyze error suppression in concatenated codes.
    """

    def __init__(self, base_code_distance: int, threshold: float):
        self.d = base_code_distance
        self.p_th = threshold  # Threshold error rate
        self.t = (self.d - 1) // 2  # Errors correctable

    def explain_error_suppression(self):
        """Explain how concatenation suppresses errors."""
        print("""
Error Suppression in Concatenated Codes
═══════════════════════════════════════

KEY INSIGHT: Each level squares the (relative) error rate!

For distance-3 code (corrects 1 error):
    Level 0: physical error rate = p
    Level 1: logical error rate ≈ c·p²
             (need 2+ errors to fail, probability ~ p²)
    Level 2: logical error rate ≈ c·(c·p²)² = c³·p⁴
    Level 3: logical error rate ≈ c⁷·p⁸
    ...
    Level k: logical error rate ≈ (p/p_th)^(2^k) · p_th

WHERE:
    p_th = 1/c is the threshold
    c is a constant depending on code and implementation

CONDITION FOR IMPROVEMENT:
    p < p_th  →  Each level reduces error rate
    p > p_th  →  Each level INCREASES error rate (bad!)

ERROR RATE FORMULA:
    p_L^(k) ≈ p_th · (p/p_th)^(2^k)

Example (p = 0.001, p_th = 0.01):
    p/p_th = 0.1
    Level 1: p_L ≈ 0.01 × 0.1² = 10⁻⁴
    Level 2: p_L ≈ 0.01 × 0.1⁴ = 10⁻⁶
    Level 3: p_L ≈ 0.01 × 0.1⁸ = 10⁻¹⁰
    Level 4: p_L ≈ 0.01 × 0.1¹⁶ = 10⁻¹⁸

DOUBLY EXPONENTIAL suppression!
        """)

    def calculate_error_rate(self, p_physical: float, levels: int) -> float:
        """Calculate logical error rate after k levels."""
        if p_physical >= self.p_th:
            return float('inf')  # Above threshold

        ratio = p_physical / self.p_th
        exponent = 2 ** levels
        p_logical = self.p_th * (ratio ** exponent)
        return p_logical

    def demonstrate(self, p_physical: float = 0.001):
        """Show error suppression for various levels."""
        print(f"\nError Suppression Demo (p = {p_physical}, p_th = {self.p_th})")
        print("=" * 55)
        print(f"{'Level':<10} {'Error Rate':<20} {'Improvement':<15}")
        print("-" * 55)

        prev_rate = p_physical
        for level in range(6):
            rate = self.calculate_error_rate(p_physical, level)
            improvement = p_physical / rate if rate > 0 else float('inf')
            print(f"{level:<10} {rate:<20.2e} {improvement:<15.2e}×")
            prev_rate = rate


analysis = ConcatenationAnalysis(base_code_distance=3, threshold=0.01)
analysis.explain_error_suppression()
analysis.demonstrate(p_physical=0.001)
```

## Resource Requirements

```python
class ConcatenationResources:
    """
    Calculate resource requirements for concatenated codes.
    """

    def __init__(self, base_qubits: int, base_gates: int):
        self.n = base_qubits  # Qubits per level-1 block
        self.g = base_gates   # Gates per syndrome extraction

    def explain_resources(self):
        """Explain resource scaling."""
        print(f"""
Resource Scaling for Concatenated Codes
═══════════════════════════════════════

For base code with n qubits, k concatenation levels:

QUBIT COUNT:
    Physical qubits = n^k for one logical qubit

    Example (Steane, n=7):
        Level 1: 7 qubits
        Level 2: 49 qubits
        Level 3: 343 qubits
        Level 4: 2,401 qubits
        Level 5: 16,807 qubits

TIME OVERHEAD:
    Each level adds latency for syndrome extraction
    Total rounds ∝ k (polylogarithmic in 1/p_L)

GATES PER QEC ROUND:
    Level 1: g gates
    Level 2: g × n gates (syndrome for each block)
    Level k: g × n^(k-1) gates total

OVERALL OVERHEAD:
    For target error rate ε:
        Levels needed: k = O(log log(1/ε))
        Qubits needed: O(poly(log(1/ε)))

    This is POLYLOGARITHMIC overhead!
    (Much better than polynomial)
        """)

    def calculate_resources(self, levels: int):
        """Calculate resources for given concatenation level."""
        qubits = self.n ** levels
        gates_per_round = self.g * (self.n ** (levels - 1)) if levels > 0 else 0

        return {
            'qubits': qubits,
            'gates_per_round': gates_per_round,
            'levels': levels
        }

    def tabulate(self, max_levels: int = 6):
        """Show resource table."""
        print("\nResource Requirements Table")
        print("=" * 50)
        print(f"{'Level':<8} {'Qubits':<15} {'Gates/Round':<15}")
        print("-" * 50)

        for k in range(max_levels + 1):
            res = self.calculate_resources(k)
            print(f"{k:<8} {res['qubits']:<15,} {res['gates_per_round']:<15,}")


resources = ConcatenationResources(base_qubits=7, base_gates=50)
resources.explain_resources()
resources.tabulate()
```

## Concatenation Hierarchy

```
Concatenated Code Structure
═══════════════════════════

HIERARCHICAL ORGANIZATION:

Level 2 (1 block):
    ┌───────────────────────────────────────────┐
    │              Logical Qubit                │
    │                                           │
    │  Syndrome: S₂ = {S₂₁, S₂₂, ..., S₂₆}     │
    │  Error correction at level 2              │
    └───────────────────────────────────────────┘
                        │
                        ▼
Level 1 (7 blocks):
    ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐
    │ B₁  │ │ B₂  │ │ B₃  │ │ B₄  │ │ B₅  │ │ B₆  │ │ B₇  │
    │     │ │     │ │     │ │     │ │     │ │     │ │     │
    │ S₁  │ │ S₁  │ │ S₁  │ │ S₁  │ │ S₁  │ │ S₁  │ │ S₁  │
    └─────┘ └─────┘ └─────┘ └─────┘ └─────┘ └─────┘ └─────┘
       │       │       │       │       │       │       │
       ▼       ▼       ▼       ▼       ▼       ▼       ▼
Level 0 (49 qubits):
    ●●●●●●● ●●●●●●● ●●●●●●● ●●●●●●● ●●●●●●● ●●●●●●● ●●●●●●●


ERROR CORRECTION ORDER:

Option 1: Bottom-up
    1. Correct each level-1 block
    2. Use corrected blocks for level-2 syndrome
    3. Correct level-2 logical errors

Option 2: Top-down (Knill style)
    1. Measure syndromes at all levels
    2. Decode starting from top level
    3. Propagate corrections downward

FAULT TOLERANCE:
    Each level must use fault-tolerant gadgets!
    Error at level k propagates to level k-1
    But still protected by level k error correction
```

## Practical Concatenation Schemes

```python
class ConcatenationSchemes:
    """
    Explore practical concatenation implementations.
    """

    def __init__(self):
        self.schemes = {}

    def steane_concatenation(self):
        """Explain Steane code concatenation."""
        print("""
Steane Code Concatenation [[7,1,3]]
═══════════════════════════════════

WHY STEANE:
    - CSS code (X and Z errors handled separately)
    - Transversal Clifford gates (H, S, CNOT)
    - Only T gate needs magic states

CONCATENATION STRUCTURE:
    Level k: 7^k physical qubits

    k=1: 7 qubits, corrects 1 error
    k=2: 49 qubits, corrects errors on any 1 level-1 block
    k=3: 343 qubits

SYNDROME EXTRACTION:
    At each level, extract syndromes for all blocks
    Use Shor-style or Steane-style measurement
    Decode and correct before moving up

THRESHOLD:
    Steane concatenation: p_th ≈ 10⁻⁵ to 10⁻⁴
    (Depends on gate set and error model)

ADVANTAGES:
    + Simple structure
    + Transversal Cliffords
    + Well-understood

DISADVANTAGES:
    - Lower threshold than topological codes
    - Deep circuits for syndrome extraction
        """)

    def bacon_shor_concatenation(self):
        """Explain Bacon-Shor code concatenation."""
        print("""
Bacon-Shor Code Concatenation
═════════════════════════════

BACON-SHOR [[n²,1,n]]:
    Subsystem code on n×n grid
    Distance n with n² physical qubits

    Example: [[9,1,3]] Bacon-Shor

    q00 ─ q01 ─ q02
     │     │     │
    q10 ─ q11 ─ q12
     │     │     │
    q20 ─ q21 ─ q22

    X stabilizers: XX on rows
    Z stabilizers: ZZ on columns

CONCATENATION:
    Each qubit of outer Bacon-Shor → encoded in inner code
    Can use same or different inner code

ADVANTAGE:
    2-qubit measurements only!
    Simpler syndrome extraction

HYBRID CONCATENATION:
    Outer: Bacon-Shor (simple measurements)
    Inner: Steane (transversal gates)
    Best of both worlds
        """)

    def knill_scheme(self):
        """Explain Knill's C4/C6 scheme."""
        print("""
Knill's Postselection Scheme (C4/C6)
════════════════════════════════════

APPROACH: Different philosophy
    Don't correct errors during computation
    Instead: postselect on no errors detected

CODES USED:
    C4: [[4,2,2]] code (detects 1 error)
    C6: [[6,2,2]] code

CONCATENATION:
    Level 1: C4 with 4 qubits
    Level 2: Each C4 qubit → another C4
    ...

ERROR HANDLING:
    If error detected at any level → ABORT
    Only accept computation if all levels pass

THRESHOLD:
    Very high! p_th ≈ 3-5%

WHY SO HIGH:
    Postselection is powerful
    Don't need to identify AND correct, just detect

DISADVANTAGE:
    Low success probability
    Need many attempts to succeed
    Suitable for: small circuits, magic state prep
        """)
```

## Comparison: Concatenation vs Topological

```
Concatenation vs Topological Codes
══════════════════════════════════

                    CONCATENATION        TOPOLOGICAL (Surface)
────────────────────────────────────────────────────────────────
Threshold           ~10⁻⁴ - 10⁻³         ~1%
Qubit overhead      O(polylog(1/ε))      O(log²(1/ε))
Circuit depth       O(log(1/ε))          O(1) per round
Connectivity        Low (hierarchical)   2D grid
Transversal gates   Many (code-dep.)     Limited
Magic states        Distillation         Distillation
Decoder             Simple (local)       Global (MWPM, Union-Find)

WHEN TO USE CONCATENATION:
    - Low physical error rates (p << 10⁻³)
    - Limited connectivity hardware
    - Need many transversal gates
    - Modest target error rates

WHEN TO USE TOPOLOGICAL:
    - Higher physical error rates (~10⁻³ to 10⁻²)
    - 2D grid connectivity available
    - Very low target error rates needed
    - Real-time decoding required

HYBRID APPROACHES:
    Concatenate surface codes!
    Inner: surface code (high threshold)
    Outer: simple code (boost suppression)

    Or use surface code with concatenated magic states
```

## Optimizing Concatenation

```python
class ConcatenationOptimization:
    """
    Optimize concatenation strategies.
    """

    def __init__(self):
        pass

    def choose_levels(self, p_physical: float, p_target: float,
                      p_threshold: float = 0.01):
        """Determine optimal concatenation levels."""
        if p_physical >= p_threshold:
            return None  # Cannot achieve target

        ratio = p_physical / p_threshold

        # Solve: p_th * ratio^(2^k) <= p_target
        # 2^k >= log(p_target/p_th) / log(ratio)
        import math

        if p_target >= p_threshold:
            return 0

        log_ratio = math.log(ratio)
        log_target = math.log(p_target / p_threshold)

        power_needed = log_target / log_ratio
        k = math.ceil(math.log2(power_needed)) if power_needed > 1 else 1

        return max(1, k)

    def optimize_for_circuit(self, circuit_size: int, p_physical: float,
                             p_threshold: float = 0.01, n_base: int = 7):
        """Optimize concatenation for circuit requirements."""
        print(f"""
Optimization for Circuit Size = {circuit_size:,}
═══════════════════════════════════════════════

Physical error rate: {p_physical}
Threshold: {p_threshold}

To have < 1 logical error in entire circuit:
    Need p_logical < 1/{circuit_size}
    Target: p_logical < {1/circuit_size:.2e}
        """)

        target = 1 / circuit_size
        levels = self.choose_levels(p_physical, target, p_threshold)

        if levels is None:
            print("Cannot achieve target: physical rate above threshold!")
            return

        qubits = n_base ** levels
        print(f"""
Required concatenation levels: {levels}
Qubits per logical qubit: {qubits:,}

For {circuit_size:,} gate circuit on 100 logical qubits:
    Total physical qubits: {100 * qubits:,}
        """)


optimizer = ConcatenationOptimization()
optimizer.optimize_for_circuit(circuit_size=10**9, p_physical=0.001)
optimizer.optimize_for_circuit(circuit_size=10**6, p_physical=0.0001)
```

## Summary

Concatenated codes provide a systematic approach to fault tolerance:

1. **Recursive encoding**: Each level protects the next
2. **Doubly exponential suppression**: Error rate ~ (p/p_th)^(2^k)
3. **Polylogarithmic overhead**: Qubits scale as O(polylog(1/ε))
4. **Threshold requirement**: Must be below threshold for improvement
5. **Trade-offs**: Lower threshold but simpler structure vs topological

Concatenation was the original proof of fault tolerance and remains relevant for specific applications.
