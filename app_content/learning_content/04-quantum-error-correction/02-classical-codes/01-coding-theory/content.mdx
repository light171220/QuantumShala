## Fundamental Concepts

```python
import numpy as np
from typing import List, Tuple
from itertools import combinations

class CodingTheoryBasics:
    """
    Introduction to classical coding theory.

    ┌─────────────────────────────────────────────────────────────────┐
    │                    KEY DEFINITIONS                              │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  CODEWORD: A valid encoded message                             │
    │  ─────────────────────────────────────                         │
    │  Example: In a 3-bit repetition code                           │
    │    000 and 111 are codewords                                   │
    │    001, 010, etc. are NOT codewords                            │
    │                                                                 │
    │  CODE: Set of all valid codewords                              │
    │  ─────────────────────────────────                             │
    │  C = {c₁, c₂, ..., cₘ}                                         │
    │                                                                 │
    │  HAMMING DISTANCE: Number of positions where two               │
    │  ─────────────────  strings differ                             │
    │  d(000, 111) = 3 (all three positions differ)                  │
    │  d(010, 011) = 1 (only last position differs)                  │
    │                                                                 │
    │  MINIMUM DISTANCE: Smallest Hamming distance between           │
    │  ─────────────────  any two distinct codewords                 │
    │  d = min{d(cᵢ, cⱼ) : cᵢ ≠ cⱼ}                                  │
    │                                                                 │
    └─────────────────────────────────────────────────────────────────┘
    """

    @staticmethod
    def hamming_distance(s1: str, s2: str) -> int:
        """
        Calculate Hamming distance between two binary strings.

        The Hamming distance is the number of positions where the
        corresponding symbols differ.
        """
        if len(s1) != len(s2):
            raise ValueError("Strings must have equal length")

        return sum(c1 != c2 for c1, c2 in zip(s1, s2))

    @staticmethod
    def hamming_weight(s: str) -> int:
        """
        Calculate Hamming weight (number of 1s) in binary string.
        """
        return sum(c == '1' for c in s)

    @staticmethod
    def minimum_distance(codewords: List[str]) -> int:
        """
        Find minimum Hamming distance of a code.
        """
        if len(codewords) < 2:
            return float('inf')

        min_dist = float('inf')
        for i, c1 in enumerate(codewords):
            for c2 in codewords[i+1:]:
                dist = CodingTheoryBasics.hamming_distance(c1, c2)
                min_dist = min(min_dist, dist)

        return min_dist

# Demonstrate basic concepts
basics = CodingTheoryBasics()

# Example: Repetition code

print("=" * 60)

# Codewords of 3-bit repetition code

print(f"\n3-bit repetition code: {rep3_code}")
print(f"Minimum distance: {basics.minimum_distance(rep3_code)}")

# Hamming distance examples
print("\nHamming distance examples:")
print(f"  d(000, 111) = {basics.hamming_distance('000', '111')}")
print(f"  d(000, 001) = {basics.hamming_distance('000', '001')}")
print(f"  d(010, 110) = {basics.hamming_distance('010', '110')}")

# Hamming weight
print("\nHamming weight examples:")
print(f"  w(000) = {basics.hamming_weight('000')}")
print(f"  w(101) = {basics.hamming_weight('101')}")
print(f"  w(111) = {basics.hamming_weight('111')}")
```

## Code Parameters: [n, k, d]

```
┌─────────────────────────────────────────────────────────────────────┐
│                    CODE PARAMETERS [n, k, d]                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  n = BLOCK LENGTH                                                   │
│      Number of bits in each codeword                               │
│      Example: 3-bit repetition code has n=3                        │
│                                                                     │
│  k = DIMENSION                                                      │
│      Number of information bits encoded                            │
│      Example: 3-bit repetition encodes 1 bit, so k=1              │
│                                                                     │
│  d = MINIMUM DISTANCE                                               │
│      Minimum Hamming distance between codewords                    │
│      Example: 3-bit repetition has d=3                             │
│                                                                     │
│  NOTATION: An [n, k, d] code                                       │
│  3-bit repetition code = [3, 1, 3] code                           │
│                                                                     │
│  CODE RATE: R = k/n                                                │
│  Fraction of bits that carry information                           │
│  3-bit repetition: R = 1/3 = 33.3%                                │
│                                                                     │
│  ENCODING OVERHEAD: (n-k)/k                                        │
│  How many redundant bits per information bit                       │
│  3-bit repetition: (3-1)/1 = 2 redundant bits per info bit        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

```python
import numpy as np
from dataclasses import dataclass
from typing import List, Set

@dataclass
class CodeParameters:
    """
    Parameters of an error-correcting code.
    """
    n: int  # Block length
    k: int  # Dimension (information bits)
    d: int  # Minimum distance

    @property
    def rate(self) -> float:
        """Code rate R = k/n."""
        return self.k / self.n

    @property
    def redundancy(self) -> int:
        """Number of redundant (check) bits."""
        return self.n - self.k

    @property
    def error_detection(self) -> int:
        """Number of errors that can be detected."""
        return self.d - 1

    @property
    def error_correction(self) -> int:
        """Number of errors that can be corrected."""
        return (self.d - 1) // 2

    def describe(self) -> str:
        """Return description of code capabilities."""
        return (
            f"[{self.n}, {self.k}, {self.d}] Code:\n"
            f"  Rate: {self.rate:.2%}\n"
            f"  Redundancy: {self.redundancy} bits\n"
            f"  Can detect up to {self.error_detection} errors\n"
            f"  Can correct up to {self.error_correction} errors"
        )

class CodeAnalyzer:
    """
    Analyze code parameters and capabilities.
    """

    @staticmethod
    def from_codewords(codewords: List[str]) -> CodeParameters:
        """
        Determine code parameters from set of codewords.
        """
        if not codewords:
            raise ValueError("Need at least one codeword")

        n = len(codewords[0])

        # k = log2(number of codewords) for linear codes
        k = int(np.log2(len(codewords)))

        # Calculate minimum distance
        d = float('inf')
        for i, c1 in enumerate(codewords):
            for c2 in codewords[i+1:]:
                dist = sum(a != b for a, b in zip(c1, c2))
                d = min(d, dist)

        return CodeParameters(n, k, int(d))

# Example codes
print("\nCode Parameter Examples")
print("=" * 60)

# 3-bit repetition code [3, 1, 3]
rep3 = CodeParameters(n=3, k=1, d=3)
print(rep3.describe())

# 5-bit repetition code [5, 1, 5]
print()
rep5 = CodeParameters(n=5, k=1, d=5)
print(rep5.describe())

# Hamming [7, 4, 3] code
print()
hamming74 = CodeParameters(n=7, k=4, d=3)
print(hamming74.describe())

# Compare rates
print("\nCode Rate Comparison:")
codes = [("Rep-3", rep3), ("Rep-5", rep5), ("Hamming", hamming74)]
for name, code in codes:
    print(f"  {name}: {code.rate:.1%} efficiency, corrects {code.error_correction} error(s)")
```

## Error Detection vs Error Correction

```
┌─────────────────────────────────────────────────────────────────────┐
│              ERROR DETECTION VS CORRECTION                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  DETECTION: Know an error occurred (but not which one)             │
│  CORRECTION: Know exactly which error occurred and fix it          │
│                                                                     │
│  EXAMPLE with [3, 1, 3] code (d=3):                                │
│                                                                     │
│  Codewords:  000 ●────────────────────────● 111                    │
│                  ←───── distance 3 ──────→                         │
│                                                                     │
│  1 error:   001 010 100   or   110 101 011                         │
│             ↓   ↓   ↓          ↓   ↓   ↓                           │
│          closer to 000      closer to 111                          │
│          → decode to 0      → decode to 1                          │
│                                                                     │
│  2 errors: 011 101 110   or   100 010 001                         │
│            ↓   ↓   ↓          ↓   ↓   ↓                           │
│         closer to 111!     closer to 000!                          │
│         → WRONG decode!   → WRONG decode!                          │
│                                                                     │
│  RULE:                                                              │
│  • Can DETECT up to (d-1) errors                                   │
│  • Can CORRECT up to ⌊(d-1)/2⌋ errors                             │
│                                                                     │
│  For d=3: detect 2, correct 1                                      │
│  For d=5: detect 4, correct 2                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

```python
import numpy as np
from typing import Tuple, Optional

class ErrorDetectionCorrection:
    """
    Demonstrate error detection and correction.
    """

    def __init__(self, codewords: list):
        """
        Initialize with list of valid codewords.
        """
        self.codewords = codewords
        self.n = len(codewords[0])

    def nearest_codeword(self, received: str) -> Tuple[str, int]:
        """
        Find nearest codeword by Hamming distance.

        Returns (nearest_codeword, distance).
        """
        min_dist = float('inf')
        nearest = None

        for cw in self.codewords:
            dist = sum(a != b for a, b in zip(received, cw))
            if dist < min_dist:
                min_dist = dist
                nearest = cw

        return nearest, min_dist

    def decode(self, received: str, max_correct: int = 1) -> Tuple[Optional[str], str]:
        """
        Decode received string.

        Returns (decoded_codeword, status).
        Status is one of: 'valid', 'corrected', 'detected', 'failed'
        """
        # Check if already valid
        if received in self.codewords:
            return received, 'valid'

        nearest, dist = self.nearest_codeword(received)

        if dist <= max_correct:
            return nearest, 'corrected'
        elif dist <= 2 * max_correct:
            return None, 'detected'  # Error detected but not corrected
        else:
            return nearest, 'failed'  # May have decoded incorrectly

    def simulate_channel(self, codeword: str, error_prob: float) -> str:
        """
        Simulate binary symmetric channel.
        """
        received = ""
        for bit in codeword:
            if np.random.random() < error_prob:
                received += '0' if bit == '1' else '1'
            else:
                received += bit
        return received

# Demonstrate with repetition code
rep3 = ErrorDetectionCorrection(['000', '111'])

print("\nError Detection and Correction Demo")
print("=" * 60)

print("\n3-bit Repetition Code [3, 1, 3]")
print("Can correct 1 error, detect 2 errors")

test_cases = [
    ('000', "No error"),
    ('001', "1 error"),
    ('010', "1 error"),
    ('011', "2 errors"),
    ('111', "No error"),
    ('110', "1 error"),
    ('100', "2 errors (ambiguous)")
]

print("\nReceived | Nearest | Distance | Status")
print("-" * 50)
for received, description in test_cases:
    decoded, status = rep3.decode(received, max_correct=1)
    nearest, dist = rep3.nearest_codeword(received)
    print(f"  {received}   |   {nearest}   |    {dist}     | {status} ({description})")
```

## Sphere Packing and Code Bounds

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SPHERE PACKING BOUND                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  CONCEPT: Codewords are centers of non-overlapping spheres         │
│                                                                     │
│     ●─────●          Sphere of radius t around each codeword       │
│    /       \         t = ⌊(d-1)/2⌋ = correction capability         │
│   ○    c₁   ○                                                       │
│    \       /        Spheres must not overlap for unique decoding   │
│     ●─────●                                                         │
│                                                                     │
│  HAMMING BOUND (Sphere Packing):                                    │
│                                                                     │
│        M × Σᵢ₌₀ᵗ (n choose i) ≤ 2ⁿ                                 │
│                                                                     │
│  where:                                                             │
│    M = number of codewords = 2ᵏ                                    │
│    n = block length                                                │
│    t = ⌊(d-1)/2⌋ = error correction capability                    │
│                                                                     │
│  PERFECT CODE: Achieves equality (spheres exactly fill space)     │
│    Examples: Hamming codes, Golay codes                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

```python
import numpy as np
from math import comb
from typing import Tuple

class CodeBounds:
    """
    Calculate theoretical bounds on code parameters.
    """

    @staticmethod
    def hamming_bound(n: int, k: int, t: int) -> bool:
        """
        Check if [n, k, d] code satisfies Hamming (sphere-packing) bound.

        Args:
            n: Block length
            k: Dimension
            t: Error correction capability t = floor((d-1)/2)

        Returns:
            True if bound is satisfied
        """
        M = 2 ** k  # Number of codewords

        # Volume of sphere of radius t
        sphere_volume = sum(comb(n, i) for i in range(t + 1))

        # Total volume of space
        total_volume = 2 ** n

        return M * sphere_volume <= total_volume

    @staticmethod
    def singleton_bound(n: int, k: int, d: int) -> bool:
        """
        Check Singleton bound: d ≤ n - k + 1

        Codes achieving equality are called MDS (Maximum Distance Separable).
        """
        return d <= n - k + 1

    @staticmethod
    def plotkin_bound(n: int, d: int) -> int:
        """
        Plotkin bound on number of codewords for given n and d.
        """
        if 2 * d > n:
            return 2 * d // (2 * d - n)
        else:
            return 2 ** n  # Bound not useful

    @staticmethod
    def max_distance(n: int, k: int) -> int:
        """
        Maximum achievable distance for [n, k] code (Singleton bound).
        """
        return n - k + 1

    @staticmethod
    def min_redundancy(n: int, d: int) -> int:
        """
        Minimum redundancy needed for distance d (Singleton bound).
        """
        return d - 1

    @staticmethod
    def is_perfect(n: int, k: int, d: int) -> bool:
        """
        Check if code achieves Hamming bound with equality (perfect code).
        """
        t = (d - 1) // 2
        M = 2 ** k
        sphere_volume = sum(comb(n, i) for i in range(t + 1))
        total_volume = 2 ** n

        return M * sphere_volume == total_volume

# Analyze various codes
bounds = CodeBounds()

print("\nCode Bounds Analysis")
print("=" * 60)

codes = [
    ("Repetition [3,1,3]", 3, 1, 3),
    ("Repetition [5,1,5]", 5, 1, 5),
    ("Hamming [7,4,3]", 7, 4, 3),
    ("Extended Hamming [8,4,4]", 8, 4, 4),
    ("Golay [23,12,7]", 23, 12, 7),
]

print(f"{'Code':<25} {'Hamming':<10} {'Singleton':<12} {'Perfect':<10}")
print("-" * 60)

for name, n, k, d in codes:
    t = (d - 1) // 2
    hamm = "✓" if bounds.hamming_bound(n, k, t) else "✗"
    sing = "✓" if bounds.singleton_bound(n, k, d) else "✗"
    perf = "✓" if bounds.is_perfect(n, k, d) else "✗"

    print(f"{name:<25} {hamm:<10} {sing:<12} {perf:<10}")

print("\n✓ Perfect codes achieve sphere-packing equality")
print("  Only known binary perfect codes: Hamming codes, Golay code, trivial codes")
```

## The Redundancy-Protection Trade-off

```python
import numpy as np

class RedundancyAnalysis:
    """
    Analyze trade-off between redundancy and error protection.

    ┌─────────────────────────────────────────────────────────────────┐
    │              REDUNDANCY VS PROTECTION TRADE-OFF                 │
    ├─────────────────────────────────────────────────────────────────┤
    │                                                                 │
    │  MORE REDUNDANCY = MORE PROTECTION but LOWER RATE              │
    │                                                                 │
    │  Rate                                                           │
    │   ↑                                                             │
    │ 1 ┼─────────────────── No code (no protection)                 │
    │   │   ●                                                         │
    │   │     ●                                                       │
    │   │       ● Hamming codes                                      │
    │ ½ ┼─────────●─────────────                                     │
    │   │           ●                                                 │
    │   │             ●                                               │
    │   │                 ● Reed-Solomon                             │
    │ 0 ┼──────────────────────────→ Protection (d)                  │
    │   0    1    2    3    4    5                                   │
    │                                                                 │
    │  OPTIMAL CODES: Maximize distance for given rate               │
    │  or maximize rate for given distance                           │
    │                                                                 │
    └─────────────────────────────────────────────────────────────────┘
    """

    @staticmethod
    def compare_codes():
        """Compare various code families."""

        code_families = {
            "Repetition": [(3, 1, 3), (5, 1, 5), (7, 1, 7)],
            "Hamming": [(7, 4, 3), (15, 11, 3), (31, 26, 3)],
            "Extended Hamming": [(8, 4, 4), (16, 11, 4), (32, 26, 4)],
        }

        print("\nCode Family Comparison")
        print("=" * 70)

        for family, codes in code_families.items():
            print(f"\n{family} codes:")
            print(f"  {'[n,k,d]':<12} {'Rate':>8} {'Redundancy':>12} {'Correct':>10}")
            print("  " + "-" * 45)

            for n, k, d in codes:
                rate = k / n
                redundancy = n - k
                correct = (d - 1) // 2

                print(f"  [{n},{k},{d}]{'':5} {rate:>8.1%} {redundancy:>12} {correct:>10}")

    @staticmethod
    def efficiency_for_protection(target_errors: int):
        """
        Find most efficient code for given error protection.
        """
        # Need d >= 2t + 1 to correct t errors
        required_d = 2 * target_errors + 1

        candidates = [
            ("Repetition", 2*target_errors + 1, 1, required_d),
            ("Hamming", 2**(target_errors+2) - 1, 2**(target_errors+2) - target_errors - 2, 3),
        ]

        print(f"\nCodes that correct {target_errors} error(s):")
        print(f"  {'Code':<20} {'[n,k,d]':<15} {'Rate':>10}")
        print("  " + "-" * 45)

        for name, n, k, d in candidates:
            if d >= required_d:
                print(f"  {name:<20} [{n},{k},{d}]{'':5} {k/n:>10.1%}")

# Run analysis
analysis = RedundancyAnalysis()
analysis.compare_codes()
analysis.efficiency_for_protection(1)
```

## Summary

```
┌─────────────────────────────────────────────────────────────────────┐
│                 CODING THEORY FUNDAMENTALS SUMMARY                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  KEY CONCEPTS:                                                      │
│  • Hamming distance: positions where strings differ                │
│  • Minimum distance d: closest pair of codewords                   │
│  • Code parameters [n, k, d]: length, dimension, distance          │
│                                                                     │
│  ERROR CAPABILITIES:                                                │
│  • Detect up to (d-1) errors                                       │
│  • Correct up to ⌊(d-1)/2⌋ errors                                 │
│                                                                     │
│  FUNDAMENTAL TRADE-OFFS:                                            │
│  • More redundancy → better protection but lower rate              │
│  • Bounds (Hamming, Singleton) limit achievable codes              │
│  • Perfect codes achieve theoretical optimum                       │
│                                                                     │
│  RELEVANCE TO QUANTUM:                                              │
│  • Quantum codes need to correct both bit and phase errors         │
│  • Classical concepts (distance, syndrome) carry over              │
│  • But quantum has additional constraints (no-cloning)             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

These classical concepts form the foundation for understanding quantum error correction, where we must protect quantum information against both bit-flip and phase-flip errors.
