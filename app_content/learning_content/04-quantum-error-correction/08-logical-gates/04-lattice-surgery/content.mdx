# Lattice Surgery Operations

## Introduction

Lattice surgery is a powerful technique for implementing logical gates on surface codes without requiring physical connections between distant qubits. By merging and splitting code patches, we can perform entangling operations using only local 2D operations.

## The Concept of Lattice Surgery

```
Lattice Surgery: Basic Idea
═══════════════════════════

PROBLEM with transversal gates on surface code:
    - CNOT requires connecting all pairs of qubits
    - In 2D layout, this means long-range connections
    - Or moving code patches (slow and error-prone)

SOLUTION: Lattice Surgery
    - Instead of moving qubits, modify the code itself
    - MERGE two patches → joint measurement
    - SPLIT back → projective operation performed

VISUALIZATION:

    Before:                        During Merge:
    ┌─────────┐   ┌─────────┐     ┌─────────────────────┐
    │         │   │         │     │                     │
    │ Patch A │   │ Patch B │ ──► │   Merged Patch      │
    │         │   │         │     │                     │
    └─────────┘   └─────────┘     └─────────────────────┘

    After Split:
    ┌─────────┐   ┌─────────┐
    │         │   │         │
    │ Patch A'│   │ Patch B'│
    │         │   │         │
    └─────────┘   └─────────┘

    The state has changed! Joint measurement was performed.


KEY INSIGHT:
    Merging measures a JOINT operator (like Z_A ⊗ Z_B)
    This projects onto an eigenstate
    Controlled operations emerge from this projection
```

## Merge Operations

```python
class LatticemergeOperations:
    """
    Implement lattice surgery merge operations.
    """

    def __init__(self, distance: int):
        self.d = distance

    def rough_merge(self):
        """Merge along rough boundaries (ZZ measurement)."""
        print("""
Rough Merge: Measuring Z_A ⊗ Z_B
════════════════════════════════

When merging two patches along their ROUGH boundaries:

    Before:
    ┌─────────┐    ┌─────────┐
    │    A    ░    ░    B    │
    │         ░────░         │
    │         ░    ░         │
    └─────────┘    └─────────┘
       rough       rough
       boundary    boundary

    During merge (d rounds):
    ┌─────────────────────────┐
    │    A    ░░░░░░    B    │
    │         ░░░░░░         │  New stabilizers connect A and B
    │         ░░░░░░         │
    └─────────────────────────┘

    The new stabilizers in the merge region have eigenvalue = Z_A ⊗ Z_B

WHAT HAPPENS:
    - New Z-type stabilizers span the merge region
    - Their eigenvalues give Z_A ⊗ Z_B measurement outcome
    - Takes d rounds to measure reliably

MEASUREMENT OUTCOME:
    m = eigenvalue of Z_A ⊗ Z_B
    m = 0: states have same Z eigenvalue
    m = 1: states have opposite Z eigenvalues
        """)

    def smooth_merge(self):
        """Merge along smooth boundaries (XX measurement)."""
        print("""
Smooth Merge: Measuring X_A ⊗ X_B
═════════════════════════════════

When merging along SMOOTH boundaries:

    Before:
    ┌─────────┐
    │    A    │─── smooth boundary
    └─────────┘
         │
    smooth merge region
         │
    ┌─────────┐
    │    B    │─── smooth boundary
    └─────────┘

    During merge:
    ┌─────────┐
    │    A    │
    ├─────────┤
    │ merged  │  New X stabilizers connect A and B
    ├─────────┤
    │    B    │
    └─────────┘

WHAT HAPPENS:
    - New X-type stabilizers span the merge region
    - Their eigenvalues give X_A ⊗ X_B measurement outcome
    - Takes d rounds to measure reliably

Used for different types of entangling operations.
        """)


merge = LatticemergeOperations(5)
merge.rough_merge()
merge.smooth_merge()
```

## Split Operations

```
Split Operations: Projecting onto Product States
════════════════════════════════════════════════

After merge and measurement, we split patches back.

ROUGH SPLIT:
    ┌─────────────────────────┐
    │    A         B         │
    │         ░░░░░░         │
    │         ░░░░░░         │
    └─────────────────────────┘
                │
                ▼ Split
    ┌─────────┐    ┌─────────┐
    │    A    │    │    B    │
    │         │    │         │
    └─────────┘    └─────────┘

WHAT HAPPENS DURING SPLIT:
    - Stop measuring the joint stabilizers
    - Measure local stabilizers instead
    - Each patch returns to independent state

PRODUCT STATE PROJECTION:
    Splitting projects the merged state onto product states
    The outcome depends on what measurement was performed

TIMING:
    - Split takes d rounds (to establish new boundaries)
    - Total merge-split cycle: ~2d rounds
```

## Logical CNOT via Lattice Surgery

```python
class LatticeSurgeryCNOT:
    """
    Implement CNOT using lattice surgery.
    """

    def __init__(self, distance: int):
        self.d = distance

    def cnot_protocol(self):
        """Describe CNOT via lattice surgery."""
        print("""
CNOT via Lattice Surgery
════════════════════════

Use an ancilla patch to mediate the CNOT.

CIRCUIT EQUIVALENCE:

    Control ──●──     Control ─────●─────
              │                    │
    Target  ──X──  =  Ancilla |+⟩──X────●────M_X──►Correct
                                        │
                      Target  ─────────X──────────


LATTICE SURGERY STEPS:

    Step 1: Prepare ancilla in |+_L⟩

    ┌─────────┐   ┌─────────┐   ┌─────────┐
    │ Control │   │ Ancilla │   │ Target  │
    │    C    │   │   |+⟩   │   │    T    │
    └─────────┘   └─────────┘   └─────────┘

    Step 2: Rough merge Control with Ancilla (measure Z_C ⊗ Z_anc)

    ┌─────────────────────┐   ┌─────────┐
    │  Control + Ancilla  │   │ Target  │
    │       Z_C Z_anc     │   │         │
    └─────────────────────┘   └─────────┘

    Step 3: Split, then Smooth merge Ancilla with Target (X_anc ⊗ X_T)

    ┌─────────┐   ┌─────────────────────┐
    │ Control │   │  Ancilla + Target   │
    │         │   │     X_anc X_T       │
    └─────────┘   └─────────────────────┘

    Step 4: Split, measure Ancilla in X basis

    ┌─────────┐   ┌─────────┐   ┌─────────┐
    │ Control │   │ Ancilla │   │ Target' │
    │         │   │   M_X   │   │         │
    └─────────┘   └─────────┘   └─────────┘

    Step 5: Apply corrections based on measurements
        """)

    def timing_analysis(self):
        """Analyze timing of lattice surgery CNOT."""
        print(f"""
Timing Analysis for CNOT (d = {self.d})
{'═' * 40}

Operation              Syndrome Rounds
─────────────────────────────────────
Prepare ancilla        d
First merge (ZZ)       d
First split            d
Second merge (XX)      d
Second split           d
Measure ancilla        d
─────────────────────────────────────
TOTAL                  ~6d rounds

For d = 5, cycle time = 1μs:
    CNOT time ≈ 30 μs

Compare to transversal CNOT:
    Would need long-range connections
    OR patch movement (also slow)

Lattice surgery trades time for locality!
        """)


cnot = LatticeSurgeryCNOT(5)
cnot.cnot_protocol()
cnot.timing_analysis()
```

## Multi-Qubit Operations

```
Multi-Qubit Lattice Surgery
═══════════════════════════

More complex operations through multiple merges.

EXAMPLE: Measure Z₁Z₂Z₃ (3-qubit parity)

    ┌───┐   ┌───┐   ┌───┐   ┌───┐
    │ 1 │   │ 2 │   │ 3 │   │anc│
    └───┘   └───┘   └───┘   └───┘
      │       │       │       │
      └───────┼───────┼───────┘
              │       │
         ZZ merge with ancilla
              │
      ┌───────┴───────┴───────┐
      │   Merged chain        │
      └───────────────────────┘

    Measure ancilla → get Z₁Z₂Z₃ parity


EXAMPLE: Logical Toffoli (CCZ)

    More complex sequence:
    1. Prepare magic state for CCZ
    2. Multiple lattice surgery operations
    3. Measure and correct

    OR use magic state injection with distilled |CCZ⟩


LAYOUT CONSIDERATIONS:

    Patches must be adjacent for surgery
    Planning patch placement is crucial
    May need "routing" patches for distant operations

    ┌───┐       ┌───┐
    │ 1 │       │ 2 │
    └───┘       └───┘
      │           │
    ┌─┴───────────┴─┐
    │    Router     │  ← Ancilla patches for routing
    └─┬───────────┬─┘
      │           │
    ┌───┐       ┌───┐
```

## Space-Time Costs

```python
class SpaceTimeCosts:
    """
    Analyze space-time costs of lattice surgery.
    """

    def __init__(self, distance: int):
        self.d = distance
        self.qubits_per_patch = 2 * distance * distance

    def single_qubit_gate(self, gate: str):
        """Cost of single-qubit logical gates."""
        costs = {
            'X': ('Pauli frame', 0, 0),
            'Z': ('Pauli frame', 0, 0),
            'H': ('Transversal', 1, 0),
            'S': ('Lattice surgery', self.d, 0),
            'T': ('Magic state', 2*self.d, 1)  # 1 magic state consumed
        }

        method, time, magic = costs.get(gate, ('Unknown', 0, 0))
        print(f"""
{gate} Gate Cost (d = {self.d})
{'═' * 35}
Method: {method}
Time: {time} syndrome rounds
Magic states: {magic}
        """)

    def two_qubit_gate(self, gate: str):
        """Cost of two-qubit logical gates."""
        if gate == 'CNOT':
            print(f"""
CNOT Gate Cost (d = {self.d})
{'═' * 35}
Method: Lattice surgery
Time: ~6d = {6*self.d} syndrome rounds
Ancilla patches: 1
Space: ~{self.qubits_per_patch} extra qubits
            """)

    def resource_estimate(self, circuit_gates: dict):
        """Estimate resources for a circuit."""
        total_time = 0
        total_magic = 0

        gate_times = {
            'X': 0, 'Z': 0, 'H': 1,
            'S': self.d, 'T': 2*self.d,
            'CNOT': 6*self.d
        }

        for gate, count in circuit_gates.items():
            total_time += gate_times.get(gate, 0) * count
            if gate == 'T':
                total_magic += count

        print(f"""
Circuit Resource Estimate
═════════════════════════
Gates: {circuit_gates}

Total time: ~{total_time} syndrome rounds
Magic states needed: {total_magic}
        """)


costs = SpaceTimeCosts(5)
costs.single_qubit_gate('T')
costs.two_qubit_gate('CNOT')
costs.resource_estimate({'H': 10, 'CNOT': 20, 'T': 50})
```

## Layout and Routing

```
Surface Code Layout for Lattice Surgery
═══════════════════════════════════════

TYPICAL LAYOUT:

    ┌──────┬──────┬──────┬──────┐
    │      │      │      │      │
    │  D1  │  D2  │  D3  │  D4  │   Data patches
    │      │      │      │      │
    ├──────┼──────┼──────┼──────┤
    │      │      │      │      │
    │  R   │  R   │  R   │  R   │   Routing patches
    │      │      │      │      │
    ├──────┼──────┼──────┼──────┤
    │      │      │      │      │
    │  D5  │  D6  │  D7  │  D8  │   Data patches
    │      │      │      │      │
    └──────┴──────┴──────┴──────┘

    D = Data qubit patches
    R = Routing/ancilla patches for surgery

ROUTING EXAMPLE: CNOT from D1 to D8

    D1 ─┬─ merge with R ─┬─ D8
        │                │
        └───── R ────────┘

    Use routing patches to connect non-adjacent data


DESIGN CONSIDERATIONS:

1. Patch Spacing
    - Patches need space for boundaries
    - Gap of 1-2 qubits between patches

2. Surgery Directions
    - Can merge in X or Y direction
    - Different directions for XX vs ZZ measurements

3. Parallelism
    - Independent surgeries can proceed in parallel
    - Schedule to minimize total time

4. Magic State Factories
    - Dedicate region for distillation
    - Inject into computation region as needed
```

## Summary

Lattice surgery is the primary method for logical gates on surface codes:

1. **Merge**: Combine patches to measure joint operators
2. **Split**: Separate patches to complete operations
3. **CNOT**: Implemented via two merges with ancilla
4. **Timing**: ~6d rounds per CNOT (d = code distance)
5. **Layout**: 2D arrangement with routing patches
6. **Trade-off**: Slower than transversal, but only needs local operations

Lattice surgery makes surface code computation practical in 2D architectures.
