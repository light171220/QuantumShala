# Quantum Monte Carlo Fundamentals

Quantum computers offer a fundamentally different approach to Monte Carlo estimation, potentially providing quadratic speedup over classical methods.

<Definition term="Quantum Monte Carlo">
A class of algorithms that leverage quantum mechanics to estimate expected values and integrals. By encoding probabilities in quantum amplitudes and using amplitude estimation, these methods achieve O(1/ε) complexity versus the classical O(1/ε²) for achieving accuracy ε.
</Definition>

## The Quantum Advantage

The key insight is that quantum superposition allows simultaneous evaluation of many samples, and amplitude estimation can extract the mean more efficiently than classical sampling.

<Formula expression="\\text{Classical: } O(1/\\epsilon^2) \\text{ samples} \\quad \\Rightarrow \\quad \\text{Quantum: } O(1/\\epsilon) \\text{ queries}" description="Quadratic speedup from quantum amplitude estimation" />

## Quantum State Preparation

The first step in quantum Monte Carlo is encoding the probability distribution into quantum amplitudes.

```python
def probability_loading_concept():
    """Illustrate the concept of loading probabilities into amplitudes."""

    # Classical probability distribution (discretized)
    n_qubits = 3
    n_states = 2**n_qubits  # 8 states

    # Example: Discretized normal distribution
    x = np.linspace(-3, 3, n_states)
    probs = np.exp(-x**2/2) / np.sqrt(2*np.pi)
    probs = probs / np.sum(probs)  # Normalize

    # Quantum amplitudes = sqrt(probabilities)
    amplitudes = np.sqrt(probs)

    # Verify: |amplitude|^2 = probability
    recovered_probs = np.abs(amplitudes)**2

    print("Probability Loading into Quantum State:")
    print(f"{'State':>6} {'Prob':>10} {'Amplitude':>12} {'|Amp|²':>10}")
    print("-" * 42)
    for i in range(n_states):
        state_label = format(i, f'0{n_qubits}b')
        print(f"|{state_label}⟩ {probs[i]:>10.4f} {amplitudes[i]:>12.4f} {recovered_probs[i]:>10.4f}")

    print(f"\nSum of probabilities: {np.sum(probs):.4f}")
    print(f"Sum of |amplitudes|²: {np.sum(recovered_probs):.4f}")

    return probs, amplitudes

probs, amplitudes = probability_loading_concept()
```

### Quantum State Representation

```python
def quantum_state_representation():
    """Show how a quantum state encodes a probability distribution."""

    # For n qubits, we have 2^n basis states
    # Each state |i⟩ has amplitude α_i where |α_i|² = P(i)

    # Example: 2-qubit system encoding 4 outcomes
    # |ψ⟩ = α₀|00⟩ + α₁|01⟩ + α₂|10⟩ + α₃|11⟩

    # Financial interpretation: 4 possible stock price outcomes
    prices = [90, 95, 105, 110]
    probabilities = [0.1, 0.3, 0.4, 0.2]

    print("Quantum State for Price Distribution:")
    print("="*50)
    print("|ψ⟩ = ", end="")

    state_terms = []
    for i, (price, prob) in enumerate(zip(prices, probabilities)):
        amp = np.sqrt(prob)
        state = format(i, '02b')
        sign = "+" if i > 0 else ""
        state_terms.append(f"{sign} {amp:.3f}|{state}⟩")

    print(" ".join(state_terms))

    print("\nInterpretation:")
    for i, (price, prob) in enumerate(zip(prices, probabilities)):
        state = format(i, '02b')
        print(f"  |{state}⟩ → Price ${price}, Probability {prob*100:.0f}%")

    # Expected price calculation
    expected_price = sum(p * pr for p, pr in zip(prices, probabilities))
    print(f"\nExpected price: ${expected_price:.2f}")

    return prices, probabilities

prices, probs = quantum_state_representation()
```

### Simple State Preparation Example

```python
def simple_state_preparation():
    """
    Demonstrate simple state preparation for a 2-qubit system.

    Target: Prepare state with specific probabilities using rotations.
    """

    # Target probabilities: [0.25, 0.25, 0.25, 0.25] (uniform)
    # This is achieved by applying H gates to both qubits

    # Target: [0.1, 0.2, 0.3, 0.4]
    target_probs = np.array([0.1, 0.2, 0.3, 0.4])
    target_amps = np.sqrt(target_probs)

    # For this specific distribution, we can use:
    # 1. First, split between |0x⟩ and |1x⟩ subspaces
    #    P(|0x⟩) = 0.1 + 0.2 = 0.3, P(|1x⟩) = 0.3 + 0.4 = 0.7

    p_0x = 0.3  # P(first qubit = 0)
    p_1x = 0.7  # P(first qubit = 1)

    # Rotation angle for first qubit: Ry(θ₁)|0⟩ = cos(θ₁/2)|0⟩ + sin(θ₁/2)|1⟩
    # We want cos²(θ₁/2) = 0.3, so θ₁ = 2*arccos(√0.3)
    theta1 = 2 * np.arccos(np.sqrt(p_0x))

    # Within |0x⟩ subspace: P(|00⟩) / P(|0x⟩) = 0.1/0.3, P(|01⟩) / P(|0x⟩) = 0.2/0.3
    # Within |1x⟩ subspace: P(|10⟩) / P(|1x⟩) = 0.3/0.7, P(|11⟩) / P(|1x⟩) = 0.4/0.7

    theta2_if_0 = 2 * np.arccos(np.sqrt(0.1/0.3))
    theta2_if_1 = 2 * np.arccos(np.sqrt(0.3/0.7))

    print("State Preparation via Controlled Rotations:")
    print("=" * 50)
    print(f"Target probabilities: {target_probs}")
    print(f"\nStep 1: Ry(θ₁) on qubit 1, θ₁ = {np.degrees(theta1):.2f}°")
    print(f"  Creates: √{p_0x:.2f}|0⟩ + √{p_1x:.2f}|1⟩")
    print(f"\nStep 2: Controlled-Ry on qubit 2")
    print(f"  If qubit 1 = |0⟩: Ry({np.degrees(theta2_if_0):.2f}°)")
    print(f"  If qubit 1 = |1⟩: Ry({np.degrees(theta2_if_1):.2f}°)")

    # Verify
    amp_00 = np.cos(theta1/2) * np.cos(theta2_if_0/2)
    amp_01 = np.cos(theta1/2) * np.sin(theta2_if_0/2)
    amp_10 = np.sin(theta1/2) * np.cos(theta2_if_1/2)
    amp_11 = np.sin(theta1/2) * np.sin(theta2_if_1/2)

    result_probs = np.array([amp_00, amp_01, amp_10, amp_11])**2

    print(f"\nVerification - resulting probabilities: {result_probs}")
    print(f"Target probabilities: {target_probs}")
    print(f"Match: {np.allclose(result_probs, target_probs)}")

    return theta1, theta2_if_0, theta2_if_1

simple_state_preparation()
```

<Tip>
For log-concave distributions (like normal distributions), efficient state preparation algorithms exist with polynomial complexity. For arbitrary distributions, the complexity can be exponential in the number of qubits.
</Tip>

## Quantum Sampling Process

Once the distribution is loaded, we can "sample" by measuring the quantum state.

```python
def quantum_sampling_simulation():
    """
    Simulate quantum sampling from a prepared state.
    """

    # Prepared state amplitudes (for 3 qubits = 8 states)
    n_qubits = 3
    n_states = 2**n_qubits

    # Lognormal-like distribution for stock prices
    mu, sigma = 0, 0.2
    x = np.linspace(-1.5, 1.5, n_states)
    probs = np.exp(-(x - mu)**2 / (2*sigma**2))
    probs = probs / np.sum(probs)

    amplitudes = np.sqrt(probs)

    # Simulate quantum measurements (sampling)
    n_measurements = 1000
    np.random.seed(42)
    measurements = np.random.choice(n_states, size=n_measurements, p=probs)

    # Count outcomes
    unique, counts = np.unique(measurements, return_counts=True)
    empirical_probs = np.zeros(n_states)
    empirical_probs[unique] = counts / n_measurements

    print("Quantum Sampling Simulation:")
    print(f"{'State':>8} {'True Prob':>12} {'Measured':>12} {'|Amplitude|²':>12}")
    print("-" * 50)

    for i in range(n_states):
        state = format(i, f'0{n_qubits}b')
        print(f"|{state}⟩ {probs[i]:>12.4f} {empirical_probs[i]:>12.4f} {amplitudes[i]**2:>12.4f}")

    return probs, empirical_probs

probs, emp_probs = quantum_sampling_simulation()
```

## From Sampling to Expectation Values

The key quantum advantage comes from computing expectation values, not just sampling.

```python
def expectation_value_quantum():
    """
    Illustrate how quantum computes expectations differently than classical.
    """

    print("Classical vs Quantum Expectation Value Computation:")
    print("=" * 55)

    print("\nClassical Monte Carlo:")
    print("  1. Sample X₁, X₂, ..., Xₙ from distribution")
    print("  2. Compute Ê[f(X)] = (1/n) Σ f(Xᵢ)")
    print("  3. Error: O(σ/√n)")
    print("  4. For accuracy ε: need n = O(σ²/ε²) samples")

    print("\nQuantum Monte Carlo:")
    print("  1. Prepare |ψ⟩ = Σ √p(x) |x⟩")
    print("  2. Apply function oracle: Σ √p(x) |x⟩|f(x)⟩")
    print("  3. Use amplitude estimation to find E[f(X)]")
    print("  4. Error: O(1/n)")
    print("  5. For accuracy ε: need n = O(1/ε) queries")

    print("\nSpeedup Factor: O(1/ε²) / O(1/ε) = O(1/ε)")

    # Example calculation
    target_accuracy = 0.001
    classical_samples = int(1 / target_accuracy**2)
    quantum_queries = int(1 / target_accuracy)
    speedup = classical_samples / quantum_queries

    print(f"\nExample for ε = {target_accuracy}:")
    print(f"  Classical samples needed: {classical_samples:,}")
    print(f"  Quantum queries needed: {quantum_queries:,}")
    print(f"  Speedup: {speedup:,}x")

expectation_value_quantum()
```

<Warning>
The quantum speedup assumes efficient state preparation. If loading the distribution requires O(2^n) gates, the advantage may be negated. Practical quantum Monte Carlo requires distributions that can be loaded efficiently.
</Warning>

## Quantum Oracle for Payoff Functions

In quantum Monte Carlo for finance, we need to encode the payoff function as a quantum oracle.

```python
def payoff_oracle_concept():
    """
    Illustrate the concept of a payoff function oracle.
    """

    # For option pricing: payoff(S) = max(S - K, 0)
    # Oracle: |S⟩|0⟩ → |S⟩|payoff(S)⟩

    # Discretized example
    n_qubits = 3
    n_states = 2**n_qubits

    # Price levels
    S_min, S_max = 80, 120
    prices = np.linspace(S_min, S_max, n_states)
    K = 100  # Strike

    # Payoffs
    payoffs = np.maximum(prices - K, 0)

    # Normalize for quantum encoding
    max_payoff = S_max - K
    normalized_payoffs = payoffs / max_payoff

    print("Payoff Oracle for European Call:")
    print(f"Strike K = ${K}")
    print("=" * 55)
    print(f"{'State':>8} {'Price':>10} {'Payoff':>10} {'Normalized':>12}")
    print("-" * 45)

    for i in range(n_states):
        state = format(i, f'0{n_qubits}b')
        print(f"|{state}⟩ ${prices[i]:>9.2f} ${payoffs[i]:>9.2f} {normalized_payoffs[i]:>12.4f}")

    print("\nQuantum Oracle Action:")
    print("  |price⟩|0⟩ → |price⟩|payoff(price)⟩")
    print("  Implemented via controlled rotations on ancilla qubits")

    return prices, payoffs, normalized_payoffs

prices, payoffs, norm_payoffs = payoff_oracle_concept()
```

## Grover Operator for Amplitude Estimation

The Grover operator is central to quantum amplitude estimation.

```python
def grover_operator_explanation():
    """
    Explain the role of Grover operator in QAE.
    """

    print("Grover Operator in Quantum Amplitude Estimation:")
    print("=" * 55)

    print("\nSetup:")
    print("  • State: |ψ⟩ = √a |good⟩ + √(1-a) |bad⟩")
    print("  • Goal: Estimate amplitude 'a' (the probability of 'good' states)")
    print("  • 'good' states = in-the-money option outcomes")

    print("\nGrover Operator Q:")
    print("  Q = A S₀ A† Sχ")
    print("  where:")
    print("    • A = state preparation operator")
    print("    • S₀ = reflection about |0⟩")
    print("    • Sχ = reflection about 'bad' states (oracle)")

    print("\nKey Property:")
    print("  Q rotates the state in 2D subspace spanned by |good⟩, |bad⟩")
    print("  Each application rotates by angle 2θ where sin(θ) = √a")

    print("\nAmplitude Estimation Process:")
    print("  1. Apply Q^k for k = 0, 1, 2, 4, 8, ... (powers of 2)")
    print("  2. Measure phase using Quantum Phase Estimation")
    print("  3. Phase reveals θ, hence a = sin²(θ)")

    # Illustrate rotation
    theta = np.pi / 6  # Example: a = sin²(π/6) = 0.25
    a = np.sin(theta)**2

    print(f"\nExample: θ = π/6 = 30°")
    print(f"  a = sin²(θ) = {a:.4f}")
    print(f"  After k Grover iterations: state angle = (2k+1)θ")

    return a
```

## Simulation of Quantum Monte Carlo

```python
def simulate_quantum_mc():
    """
    Simulate the key steps of quantum Monte Carlo numerically.
    """

    # Problem: Estimate E[max(S - K, 0)] for call option

    # Parameters
    S0 = 100
    K = 100
    r = 0.05
    sigma = 0.2
    T = 1.0

    # Discretization (limited by qubits)
    n_qubits = 5
    n_prices = 2**n_qubits

    # Price range (±3 standard deviations of log-price)
    log_S0 = np.log(S0)
    drift = (r - 0.5*sigma**2) * T
    vol = sigma * np.sqrt(T)

    log_prices = np.linspace(log_S0 + drift - 3*vol, log_S0 + drift + 3*vol, n_prices)
    prices = np.exp(log_prices)

    # Risk-neutral probabilities (discretized lognormal)
    from scipy.stats import norm
    z_values = (log_prices - log_S0 - drift) / vol
    pdf = norm.pdf(z_values)
    probs = pdf / np.sum(pdf)

    # Payoffs
    payoffs = np.maximum(prices - K, 0)

    # True expected payoff (discretized)
    expected_payoff = np.sum(probs * payoffs)
    discounted_price = np.exp(-r*T) * expected_payoff

    print("Simulated Quantum Monte Carlo for Option Pricing:")
    print(f"Parameters: S0=${S0}, K=${K}, r={r}, σ={sigma}, T={T}")
    print(f"Using {n_qubits} qubits = {n_prices} price levels")
    print("-" * 45)

    # "Amplitude" that QAE would estimate
    # For simplicity, we're estimating P(in the money) weighted by payoff
    max_payoff = np.max(payoffs)
    normalized_payoffs = payoffs / max_payoff if max_payoff > 0 else payoffs

    # The "amplitude squared" we want to estimate
    target_amplitude_sq = np.sum(probs * normalized_payoffs)

    print(f"\nDiscretized price range: ${prices[0]:.2f} to ${prices[-1]:.2f}")
    print(f"In-the-money probability: {np.sum(probs[prices > K])*100:.2f}%")
    print(f"Expected payoff (discretized): ${expected_payoff:.4f}")
    print(f"Discounted option price: ${discounted_price:.4f}")

    # Classical MC for comparison
    from scipy.stats import norm as norm_dist
    d1 = (np.log(S0/K) + (r + 0.5*sigma**2)*T) / (sigma*np.sqrt(T))
    d2 = d1 - sigma*np.sqrt(T)
    bs_price = S0*norm_dist.cdf(d1) - K*np.exp(-r*T)*norm_dist.cdf(d2)

    print(f"\nBlack-Scholes price: ${bs_price:.4f}")
    print(f"Discretization error: ${abs(discounted_price - bs_price):.4f}")

    return discounted_price, bs_price

qmc_price, bs_price = simulate_quantum_mc()
```

## Summary

Quantum Monte Carlo fundamentals:

| Concept | Description |
|---------|-------------|
| State Preparation | Load probability distribution into quantum amplitudes |
| Quantum Parallelism | Superposition evaluates function on all inputs simultaneously |
| Amplitude Estimation | Extract expected value with O(1/ε) complexity |
| Grover Operator | Amplifies amplitude of "good" states |

Key insights:

- Quantum advantage comes from amplitude estimation, not just sampling
- State preparation is a critical bottleneck
- Quadratic speedup: O(1/ε²) → O(1/ε)
- For 0.1% accuracy: Classical needs ~1M samples, Quantum needs ~1K queries
