# Quantum Amplitude Estimation

Quantum Amplitude Estimation (QAE) is the core algorithm that enables quadratic speedup in quantum Monte Carlo. It efficiently extracts the probability amplitude of a marked state, which corresponds to an expected value in financial applications.

<Definition term="Quantum Amplitude Estimation">
An algorithm that estimates the amplitude a of a quantum state |ψ⟩ = √a|good⟩ + √(1-a)|bad⟩ with O(1/ε) queries to achieve accuracy ε, compared to O(1/ε²) classical samples.
</Definition>

## Mathematical Foundation

The setup for amplitude estimation:

<Formula expression="A|0\\rangle = \\sqrt{a}|\\Psi_{good}\\rangle + \\sqrt{1-a}|\\Psi_{bad}\\rangle" description="State preparation operator A creates superposition of good and bad states" />

```python
import numpy as np
import matplotlib.pyplot as plt
from scipy.optimize import minimize_scalar

def amplitude_estimation_setup():
    """Illustrate the mathematical setup for QAE."""

    # Example: a = 0.3 (30% probability of 'good' outcome)
    a = 0.3

    # State representation in 2D subspace
    # |ψ⟩ = √a |good⟩ + √(1-a) |bad⟩
    # Can be written as:
    # |ψ⟩ = sin(θ) |good⟩ + cos(θ) |bad⟩ where sin²(θ) = a

    theta = np.arcsin(np.sqrt(a))

    print("Amplitude Estimation Setup:")
    print("=" * 50)
    print(f"Target amplitude: a = {a}")
    print(f"θ = arcsin(√a) = {np.degrees(theta):.2f}°")
    print(f"\nState representation:")
    print(f"|ψ⟩ = sin({np.degrees(theta):.2f}°)|good⟩ + cos({np.degrees(theta):.2f}°)|bad⟩")
    print(f"    = {np.sqrt(a):.4f}|good⟩ + {np.sqrt(1-a):.4f}|bad⟩")
    print(f"\nP(good) = sin²(θ) = {np.sin(theta)**2:.4f} = a")

    return a, theta

a, theta = amplitude_estimation_setup()
```

## The Grover Operator

The Grover operator Q rotates the state in the 2D subspace spanned by |good⟩ and |bad⟩.

```python
def grover_rotation_analysis():
    """Analyze the rotation effect of the Grover operator."""

    # Initial angle
    theta_initial = np.pi / 6  # a = sin²(π/6) = 0.25

    # After k applications of Q, the state angle becomes (2k+1)θ
    k_values = np.arange(0, 10)
    angles = (2*k_values + 1) * theta_initial

    # Probability of 'good' state after k iterations
    probs = np.sin(angles)**2

    print("Grover Operator Rotation Analysis:")
    print(f"Initial θ = {np.degrees(theta_initial):.1f}° (a = {np.sin(theta_initial)**2:.4f})")
    print("-" * 45)
    print(f"{'k':>4} {'Angle':>12} {'P(good)':>12}")
    print("-" * 45)

    for k, angle, prob in zip(k_values, angles, probs):
        print(f"{k:>4} {np.degrees(angle):>11.1f}° {prob:>12.4f}")

    # Find optimal k (closest to π/2)
    optimal_k = int(np.round((np.pi/2 - theta_initial) / (2*theta_initial)))
    print(f"\nOptimal k = {optimal_k} (angle closest to 90°)")

    return k_values, probs

k_vals, probs = grover_rotation_analysis()
```

<Warning>
The Grover operator can "overshoot" - if applied too many times, the probability of the good state decreases. The optimal number of iterations is approximately (π/4θ) - 1/2.
</Warning>

## Canonical QAE with Phase Estimation

The canonical amplitude estimation algorithm uses quantum phase estimation (QPE) to extract the rotation angle θ.

```python
def canonical_qae_simulation():
    """
    Simulate canonical QAE using phase estimation.
    """

    # True amplitude
    a_true = 0.3
    theta_true = np.arcsin(np.sqrt(a_true))

    # Eigenvalue of Grover operator: e^(±2iθ)
    # Phase estimation measures this phase

    print("Canonical QAE Algorithm:")
    print("=" * 50)
    print(f"True amplitude: a = {a_true}")
    print(f"True θ = {np.degrees(theta_true):.4f}°")

    # Number of precision qubits determines accuracy
    precision_qubits = [4, 6, 8, 10]

    print(f"\n{'Precision Qubits':>18} {'θ estimate':>12} {'a estimate':>12} {'Error':>10}")
    print("-" * 55)

    for n_precision in precision_qubits:
        # Phase estimation gives θ with precision 2π/2^n
        phase_precision = 2 * np.pi / (2**n_precision)

        # Simulate noisy estimation
        theta_measured = theta_true + np.random.uniform(-phase_precision/2, phase_precision/2)

        # Convert back to amplitude
        a_estimated = np.sin(theta_measured)**2

        error = abs(a_estimated - a_true)

        print(f"{n_precision:>18} {np.degrees(theta_measured):>11.4f}° {a_estimated:>12.6f} {error:>10.6f}")

    print("\nNote: Theoretical error scales as O(1/2^n) = O(1/N) where N = number of Grover iterations")

    return a_true

canonical_qae_simulation()
```

## Iterative Quantum Amplitude Estimation (IQAE)

IQAE provides the same speedup without requiring quantum phase estimation, using only Grover iterations and classical post-processing.

```python
def iterative_qae_simulation():
    """
    Simulate Iterative Quantum Amplitude Estimation.
    """

    # True amplitude
    a_true = 0.3
    theta_true = np.arcsin(np.sqrt(a_true))

    def simulate_measurement(theta, k, n_shots=100):
        """Simulate measuring after k Grover iterations."""
        angle = (2*k + 1) * theta
        prob_good = np.sin(angle)**2

        # Binomial sampling
        n_good = np.random.binomial(n_shots, prob_good)
        return n_good / n_shots

    print("Iterative QAE Simulation:")
    print("=" * 50)

    # IQAE works by narrowing down the interval containing θ
    theta_min, theta_max = 0, np.pi/2
    n_shots_per_round = 100

    rounds = []
    estimates = []

    for round_num in range(8):
        # Choose k based on current interval
        theta_mid = (theta_min + theta_max) / 2
        k = max(1, int(np.pi / (4 * (theta_max - theta_min))))

        # Measure
        prob_measured = simulate_measurement(theta_true, k, n_shots_per_round)

        # Update interval based on measurement
        # This is simplified; real IQAE uses likelihood ratio tests
        theta_est = np.arcsin(np.sqrt(prob_measured)) / (2*k + 1)

        # Refine interval
        interval_width = (theta_max - theta_min) / 2
        theta_min = max(0, theta_est - interval_width)
        theta_max = min(np.pi/2, theta_est + interval_width)

        a_est = np.sin(theta_mid)**2
        rounds.append(round_num)
        estimates.append(a_est)

        print(f"Round {round_num}: k={k}, θ ∈ [{np.degrees(theta_min):.2f}°, {np.degrees(theta_max):.2f}°], a ≈ {a_est:.4f}")

    print(f"\nFinal estimate: a ≈ {estimates[-1]:.4f}")
    print(f"True value: a = {a_true}")
    print(f"Error: {abs(estimates[-1] - a_true):.4f}")

    return estimates

iterative_qae_simulation()
```

<Tip>
Iterative QAE is more practical for near-term quantum computers because it doesn't require the complex QPE circuit. It trades some sample complexity for simpler circuits.
</Tip>

## Maximum Likelihood Amplitude Estimation (MLAE)

MLAE uses classical maximum likelihood estimation to extract the amplitude from measurement statistics.

```python
def ml_amplitude_estimation():
    """
    Simulate Maximum Likelihood Amplitude Estimation.
    """
    from scipy.optimize import minimize_scalar

    a_true = 0.3
    theta_true = np.arcsin(np.sqrt(a_true))

    # Run experiments with different numbers of Grover iterations
    k_values = [0, 1, 2, 4, 8, 16]
    n_shots = 100

    measurements = {}
    for k in k_values:
        angle = (2*k + 1) * theta_true
        prob_good = np.sin(angle)**2
        n_good = np.random.binomial(n_shots, prob_good)
        measurements[k] = {'n_good': n_good, 'n_total': n_shots}

    def neg_log_likelihood(theta):
        """Negative log-likelihood for amplitude estimation."""
        nll = 0
        for k, data in measurements.items():
            angle = (2*k + 1) * theta
            prob = np.sin(angle)**2
            # Avoid log(0)
            prob = np.clip(prob, 1e-10, 1-1e-10)

            n_good = data['n_good']
            n_bad = data['n_total'] - n_good

            nll -= n_good * np.log(prob) + n_bad * np.log(1 - prob)
        return nll

    # Find MLE
    result = minimize_scalar(neg_log_likelihood, bounds=(0, np.pi/2), method='bounded')
    theta_mle = result.x
    a_mle = np.sin(theta_mle)**2

    print("Maximum Likelihood Amplitude Estimation:")
    print("=" * 50)
    print("Measurement data:")
    for k, data in measurements.items():
        print(f"  k={k}: {data['n_good']}/{data['n_total']} good outcomes")

    print(f"\nMLE result:")
    print(f"  θ_MLE = {np.degrees(theta_mle):.4f}°")
    print(f"  a_MLE = {a_mle:.6f}")
    print(f"  True a = {a_true}")
    print(f"  Error = {abs(a_mle - a_true):.6f}")

    # Fisher information analysis
    total_queries = sum(k * data['n_total'] for k, data in measurements.items())
    print(f"\nTotal Grover queries used: {total_queries}")

    return a_mle, a_true

ml_amplitude_estimation()
```

## Circuit Depth vs Accuracy Tradeoff

```python
def depth_accuracy_tradeoff():
    """Analyze the tradeoff between circuit depth and accuracy."""

    # For canonical QAE
    target_accuracies = np.array([0.1, 0.05, 0.01, 0.005, 0.001])

    # Grover iterations needed: O(1/ε)
    grover_iterations = 1 / target_accuracies

    # Assuming each Grover iteration has depth d_G
    d_G = 100  # gates per Grover iteration (example)
    circuit_depths = grover_iterations * d_G

    print("Circuit Depth vs Accuracy Tradeoff:")
    print("=" * 55)
    print(f"{'Accuracy ε':>12} {'Grover Iters':>15} {'Circuit Depth':>15}")
    print("-" * 45)

    for eps, iters, depth in zip(target_accuracies, grover_iterations, circuit_depths):
        print(f"{eps:>12.3f} {int(iters):>15,} {int(depth):>15,}")

    # NISQ considerations
    print("\nNISQ Hardware Constraints:")
    print("  Typical gate fidelity: 99.5%")
    print("  Coherence time limits: ~1000-10000 gates")

    # Calculate achievable accuracy given hardware limits
    max_depth = 5000  # Example hardware limit
    achievable_iters = max_depth / d_G
    achievable_accuracy = 1 / achievable_iters

    print(f"\nWith max depth {max_depth}:")
    print(f"  Achievable Grover iterations: {achievable_iters:.0f}")
    print(f"  Achievable accuracy: ε ≈ {achievable_accuracy:.4f}")

    return target_accuracies, circuit_depths

depth_accuracy_tradeoff()
```

<Warning>
On current NISQ devices, the limited circuit depth severely restricts the achievable accuracy. Error correction is needed to run the deep circuits required for practical quantum advantage.
</Warning>

## Summary

Quantum Amplitude Estimation is the key enabler of quantum Monte Carlo speedup:

| Aspect | Canonical QAE | Iterative QAE | ML-QAE |
|--------|---------------|---------------|---------|
| Speedup | O(1/ε) | O(1/ε) | O(1/ε) |
| Circuit Type | Deep + QPE | Medium depth | Multiple shallow |
| Near-term Friendly | No | Somewhat | Yes |
| Classical Processing | Low | Medium | High |

Key takeaways:

- QAE achieves quadratic speedup by extracting amplitude information coherently
- The Grover operator rotates the state by 2θ per application
- Multiple variants exist with different circuit depth/measurement tradeoffs
- Practical implementation faces challenges from state preparation and noise
- Near-term devices may use iterative or variational approaches
