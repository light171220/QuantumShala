## 1. Introduction to Stochastic Processes

Financial markets are inherently uncertain. Stochastic processes provide the mathematical framework to model this randomness.

<Definition term="Stochastic Process">
A stochastic process is a collection of random variables indexed by time, representing the evolution of a system subject to random influences.
</Definition>

In finance, stochastic processes model:
- Stock prices over time
- Interest rate movements
- Exchange rate fluctuations
- Credit risk evolution

```python
import numpy as np
import matplotlib.pyplot as plt

# Simple random walk - the foundation of stochastic processes
np.random.seed(42)
n_steps = 1000
steps = np.random.choice([-1, 1], size=n_steps)
random_walk = np.cumsum(steps)

plt.figure(figsize=(12, 5))
plt.plot(random_walk)
plt.title('Simple Random Walk')
plt.xlabel('Time Step')
plt.ylabel('Position')
plt.axhline(y=0, color='r', linestyle='--', alpha=0.5)
plt.grid(True, alpha=0.3)
plt.show()
```

## 2. Brownian Motion (Wiener Process)

Brownian motion is the continuous-time limit of a random walk and is fundamental to financial modeling.

<Definition term="Brownian Motion">
Standard Brownian motion $W_t$ is a continuous-time stochastic process with:
1. $W_0 = 0$
2. Independent increments: $W_t - W_s$ is independent of $W_s$ for $t > s$
3. Gaussian increments: $W_t - W_s \sim N(0, t-s)$
4. Continuous paths (almost surely)
</Definition>

### Key Properties

<Formula>
$$\mathbb{E}[W_t] = 0$$
$$\text{Var}(W_t) = t$$
$$W_t - W_s \sim N(0, t-s)$$
</Formula>

```python
import numpy as np
import matplotlib.pyplot as plt

def simulate_brownian_motion(T, n_steps, n_paths=1):
    """
    Simulate Brownian motion paths.

    Args:
        T: Total time
        n_steps: Number of time steps
        n_paths: Number of paths to simulate

    Returns:
        times: Array of time points
        paths: Array of shape (n_paths, n_steps+1)
    """
    dt = T / n_steps
    times = np.linspace(0, T, n_steps + 1)

    # Generate increments: dW ~ N(0, dt)
    dW = np.random.normal(0, np.sqrt(dt), size=(n_paths, n_steps))

    # Cumulative sum to get paths (start at 0)
    W = np.zeros((n_paths, n_steps + 1))
    W[:, 1:] = np.cumsum(dW, axis=1)

    return times, W

# Simulate multiple Brownian motion paths
T = 1.0
n_steps = 1000
n_paths = 5

times, paths = simulate_brownian_motion(T, n_steps, n_paths)

plt.figure(figsize=(12, 6))
for i in range(n_paths):
    plt.plot(times, paths[i], alpha=0.7, label=f'Path {i+1}')
plt.title('Brownian Motion Paths')
plt.xlabel('Time')
plt.ylabel('W(t)')
plt.legend()
plt.grid(True, alpha=0.3)
plt.show()
```

<Tip>
Brownian motion has "rough" paths - it's continuous but nowhere differentiable. This roughness captures the unpredictable nature of financial markets.
</Tip>

## 3. Geometric Brownian Motion

Stock prices can't go negative, so we use Geometric Brownian Motion (GBM) which models the logarithm of prices.

<Definition term="Geometric Brownian Motion">
A stock price $S_t$ follows GBM if it satisfies:
$$dS_t = \mu S_t \, dt + \sigma S_t \, dW_t$$
where $\mu$ is the drift (expected return) and $\sigma$ is the volatility.
</Definition>

### The Solution

The explicit solution to the GBM stochastic differential equation is:

<Formula>
$$S_t = S_0 \exp\left[\left(\mu - \frac{\sigma^2}{2}\right)t + \sigma W_t\right]$$
</Formula>

```python
import numpy as np
import matplotlib.pyplot as plt

def simulate_gbm(S0, mu, sigma, T, n_steps, n_paths=1):
    """
    Simulate Geometric Brownian Motion paths.

    Args:
        S0: Initial stock price
        mu: Drift (expected return)
        sigma: Volatility
        T: Total time
        n_steps: Number of time steps
        n_paths: Number of paths

    Returns:
        times: Array of time points
        paths: Array of stock price paths
    """
    dt = T / n_steps
    times = np.linspace(0, T, n_steps + 1)

    # Generate Brownian motion increments
    dW = np.random.normal(0, np.sqrt(dt), size=(n_paths, n_steps))

    # Cumulative Brownian motion
    W = np.zeros((n_paths, n_steps + 1))
    W[:, 1:] = np.cumsum(dW, axis=1)

    # GBM solution
    t = times.reshape(1, -1)
    paths = S0 * np.exp((mu - 0.5 * sigma**2) * t + sigma * W)

    return times, paths

# Simulate stock price paths
S0 = 100      # Initial price
mu = 0.10     # 10% annual drift
sigma = 0.20  # 20% annual volatility
T = 1.0       # 1 year
n_steps = 252 # Daily steps
n_paths = 10

np.random.seed(42)
times, stock_paths = simulate_gbm(S0, mu, sigma, T, n_steps, n_paths)

plt.figure(figsize=(12, 6))
for i in range(n_paths):
    plt.plot(times, stock_paths[i], alpha=0.7)
plt.title(f'GBM Stock Price Simulation (μ={mu:.0%}, σ={sigma:.0%})')
plt.xlabel('Time (Years)')
plt.ylabel('Stock Price ($)')
plt.axhline(y=S0, color='r', linestyle='--', alpha=0.5, label='Initial Price')
plt.grid(True, alpha=0.3)
plt.show()

# Statistics at T=1
final_prices = stock_paths[:, -1]
print(f"Final Price Statistics:")
print(f"  Mean: ${np.mean(final_prices):.2f}")
print(f"  Std: ${np.std(final_prices):.2f}")
print(f"  Min: ${np.min(final_prices):.2f}")
print(f"  Max: ${np.max(final_prices):.2f}")
```

### Log-Normal Distribution

Under GBM, the stock price $S_T$ is log-normally distributed:

<Formula>
$$\ln(S_T) \sim N\left(\ln(S_0) + \left(\mu - \frac{\sigma^2}{2}\right)T, \sigma^2 T\right)$$
</Formula>

<Warning>
Notice the term $-\frac{\sigma^2}{2}$ in the drift. This is the Itô correction - it accounts for the fact that the exponential of a random variable has a different mean than the exponential of the mean.
</Warning>

## 4. Monte Carlo Simulation

Monte Carlo methods use random sampling to compute numerical results - essential for pricing complex derivatives.

<Definition term="Monte Carlo Simulation">
Monte Carlo simulation estimates expected values by averaging over many random samples:
$$\mathbb{E}[f(X)] \approx \frac{1}{N}\sum_{i=1}^{N} f(X_i)$$
</Definition>

```python
def monte_carlo_european_call(S0, K, T, r, sigma, n_paths=100000):
    """
    Price a European call option using Monte Carlo simulation.

    Args:
        S0: Initial stock price
        K: Strike price
        T: Time to maturity
        r: Risk-free rate
        sigma: Volatility
        n_paths: Number of simulation paths

    Returns:
        price: Option price
        std_error: Standard error of estimate
    """
    # Simulate terminal stock prices under risk-neutral measure
    Z = np.random.standard_normal(n_paths)
    ST = S0 * np.exp((r - 0.5 * sigma**2) * T + sigma * np.sqrt(T) * Z)

    # Calculate payoffs
    payoffs = np.maximum(ST - K, 0)

    # Discount expected payoff
    price = np.exp(-r * T) * np.mean(payoffs)
    std_error = np.exp(-r * T) * np.std(payoffs) / np.sqrt(n_paths)

    return price, std_error

# Price a call option
S0 = 100
K = 100
T = 1.0
r = 0.05
sigma = 0.20

np.random.seed(42)
price, std_error = monte_carlo_european_call(S0, K, T, r, sigma)

print(f"European Call Option Price")
print(f"  Monte Carlo: ${price:.4f} ± ${1.96*std_error:.4f}")
```

### Convergence

Monte Carlo error decreases as $O(1/\sqrt{N})$:

```python
# Demonstrate Monte Carlo convergence
n_paths_list = [100, 1000, 10000, 100000, 1000000]
results = []

for n in n_paths_list:
    price, std_err = monte_carlo_european_call(S0, K, T, r, sigma, n)
    results.append((n, price, std_err))
    print(f"N={n:>7}: Price=${price:.4f}, StdErr=${std_err:.4f}")
```

<Tip>
To halve the Monte Carlo error, you need 4× more samples. This $O(1/\sqrt{N})$ convergence is why quantum amplitude estimation is exciting - it offers $O(1/N)$ convergence!
</Tip>

## 5. Other Important Processes

### Mean-Reverting Process (Ornstein-Uhlenbeck)

Used for interest rates and volatility:

<Formula>
$$dX_t = \theta(\mu - X_t) dt + \sigma dW_t$$
</Formula>

```python
def simulate_ou_process(x0, theta, mu, sigma, T, n_steps, n_paths=1):
    """
    Simulate Ornstein-Uhlenbeck (mean-reverting) process.

    Args:
        x0: Initial value
        theta: Mean reversion speed
        mu: Long-term mean
        sigma: Volatility
    """
    dt = T / n_steps
    times = np.linspace(0, T, n_steps + 1)

    X = np.zeros((n_paths, n_steps + 1))
    X[:, 0] = x0

    for t in range(n_steps):
        dW = np.random.normal(0, np.sqrt(dt), n_paths)
        X[:, t+1] = X[:, t] + theta * (mu - X[:, t]) * dt + sigma * dW

    return times, X

# Simulate interest rate (mean-reverting to 5%)
times, rates = simulate_ou_process(0.03, 0.5, 0.05, 0.01, 10, 1000, 5)

plt.figure(figsize=(12, 5))
for i in range(5):
    plt.plot(times, rates[i] * 100, alpha=0.7)
plt.axhline(y=5, color='r', linestyle='--', label='Long-term mean (5%)')
plt.title('Mean-Reverting Interest Rate (Vasicek Model)')
plt.xlabel('Time (Years)')
plt.ylabel('Interest Rate (%)')
plt.legend()
plt.grid(True, alpha=0.3)
plt.show()
```

### Jump Diffusion

Asset prices can experience sudden jumps:

<Formula>
$$dS_t = \mu S_t \, dt + \sigma S_t \, dW_t + J_t S_t \, dN_t$$
</Formula>

where $N_t$ is a Poisson process and $J_t$ is the jump size.

```python
def simulate_jump_diffusion(S0, mu, sigma, lam, jump_mean, jump_std, T, n_steps):
    """
    Simulate Merton jump-diffusion process.

    Args:
        lam: Jump intensity (expected jumps per year)
        jump_mean: Mean of log-jump size
        jump_std: Std of log-jump size
    """
    dt = T / n_steps
    times = np.linspace(0, T, n_steps + 1)

    S = np.zeros(n_steps + 1)
    S[0] = S0

    for t in range(n_steps):
        # Brownian component
        dW = np.random.normal(0, np.sqrt(dt))

        # Jump component (Poisson number of jumps)
        n_jumps = np.random.poisson(lam * dt)
        if n_jumps > 0:
            jumps = np.random.normal(jump_mean, jump_std, n_jumps)
            jump_factor = np.exp(np.sum(jumps))
        else:
            jump_factor = 1.0

        # Update price
        S[t+1] = S[t] * np.exp((mu - 0.5*sigma**2)*dt + sigma*dW) * jump_factor

    return times, S
```

## 6. Quantum Connections

Stochastic processes connect directly to quantum computing advantages:

### Quantum Amplitude Estimation

<Definition term="Quantum Amplitude Estimation">
QAE can estimate expectations with $O(1/\epsilon)$ queries instead of $O(1/\epsilon^2)$ for classical Monte Carlo.
</Definition>

| Method | Samples for 1% accuracy |
|--------|------------------------|
| Classical MC | 10,000 |
| Quantum AE | 100 |

### State Preparation

Quantum states can encode probability distributions:

<Formula>
$$|\psi\rangle = \sum_i \sqrt{p_i} |i\rangle$$
</Formula>

This allows quantum algorithms to process all possible outcomes simultaneously.

```python
# Classical Monte Carlo convergence
def mc_convergence_analysis(true_value, n_simulations, estimator_func):
    """
    Analyze Monte Carlo convergence.

    Shows why quantum speedup matters.
    """
    errors = []
    sample_sizes = np.logspace(2, 6, 50).astype(int)

    for n in sample_sizes:
        estimates = [estimator_func(n) for _ in range(n_simulations)]
        rmse = np.sqrt(np.mean((np.array(estimates) - true_value)**2))
        errors.append(rmse)

    return sample_sizes, errors

# The O(1/sqrt(N)) convergence explains why we need quantum speedup
# for computationally intensive financial calculations
```
