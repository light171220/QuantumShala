## 1. The Risk-Return Tradeoff

One of the most fundamental principles in finance is that higher expected returns come with higher risk. Investors demand compensation (higher returns) for bearing uncertainty.

<Definition term="Risk-Return Tradeoff">
The risk-return tradeoff states that potential return rises with an increase in risk. Low levels of uncertainty (risk) are associated with low potential returns; high levels of uncertainty are associated with high potential returns.
</Definition>

```python
import numpy as np
import matplotlib.pyplot as plt

# Historical annualized returns and volatilities (approximate)
assets = {
    'T-Bills': (0.02, 0.01),
    'Bonds': (0.05, 0.06),
    'Large Cap Stocks': (0.10, 0.15),
    'Small Cap Stocks': (0.12, 0.20),
    'Emerging Markets': (0.14, 0.25)
}

# Plot risk-return relationship
plt.figure(figsize=(10, 6))
for name, (ret, vol) in assets.items():
    plt.scatter(vol, ret, s=100)
    plt.annotate(name, (vol, ret), textcoords="offset points",
                 xytext=(10, 5), fontsize=10)

plt.xlabel('Risk (Volatility)', fontsize=12)
plt.ylabel('Expected Return', fontsize=12)
plt.title('Risk-Return Tradeoff', fontsize=14)
plt.grid(True, alpha=0.3)
plt.show()
```

<Tip>
The key insight is that risk itself is not bad - it's the price of potentially higher returns. Smart investing is about getting the best return per unit of risk taken.
</Tip>

## 2. Measuring Risk: Volatility

The most common measure of investment risk is **volatility** - the standard deviation of returns.

### Variance

<Definition term="Variance">
The variance $\sigma^2$ of returns measures the average squared deviation from the mean:
$$\sigma^2 = \frac{1}{n-1} \sum_{t=1}^{n} (r_t - \bar{r})^2$$
</Definition>

### Standard Deviation (Volatility)

<Definition term="Volatility">
Volatility $\sigma$ is the standard deviation of returns:
$$\sigma = \sqrt{\sigma^2} = \sqrt{\frac{1}{n-1} \sum_{t=1}^{n} (r_t - \bar{r})^2}$$
</Definition>

```python
import numpy as np

def calculate_volatility(returns, annualize=True, periods_per_year=252):
    """
    Calculate the volatility (standard deviation) of returns.

    Args:
        returns: Array of periodic returns
        annualize: Whether to annualize the volatility
        periods_per_year: Number of periods per year (252 for daily)

    Returns:
        Volatility (annualized if specified)
    """
    vol = np.std(returns, ddof=1)  # ddof=1 for sample std

    if annualize:
        vol *= np.sqrt(periods_per_year)

    return vol

# Example: Daily returns for a stock
np.random.seed(42)
daily_returns = np.random.normal(0.0004, 0.015, 252)  # ~1.5% daily vol

daily_vol = calculate_volatility(daily_returns, annualize=False)
annual_vol = calculate_volatility(daily_returns, annualize=True)

print(f"Daily Volatility: {daily_vol:.2%}")
print(f"Annualized Volatility: {annual_vol:.2%}")
```

<Warning>
Volatility scales with the square root of time: $\sigma_{annual} = \sigma_{daily} \times \sqrt{252}$. This assumes returns are independent, which is approximately true for liquid markets.
</Warning>

### Why Volatility Measures Risk

Volatility captures uncertainty - high volatility means:
- Returns vary widely from the expected value
- Future outcomes are less predictable
- Potential for both large gains AND large losses

## 3. The Sharpe Ratio

The Sharpe ratio is the most widely used risk-adjusted return metric.

<Definition term="Sharpe Ratio">
The Sharpe ratio measures excess return per unit of risk:
$$\text{Sharpe Ratio} = \frac{R_p - R_f}{\sigma_p}$$
where $R_p$ is portfolio return, $R_f$ is risk-free rate, and $\sigma_p$ is portfolio volatility.
</Definition>

```python
import numpy as np

def sharpe_ratio(returns, risk_free_rate=0.02, periods_per_year=252):
    """
    Calculate the annualized Sharpe ratio.

    Args:
        returns: Array of periodic returns
        risk_free_rate: Annual risk-free rate
        periods_per_year: Number of periods per year

    Returns:
        Annualized Sharpe ratio
    """
    # Annualized return
    mean_return = np.mean(returns)
    annual_return = mean_return * periods_per_year

    # Annualized volatility
    annual_vol = np.std(returns, ddof=1) * np.sqrt(periods_per_year)

    # Sharpe ratio
    sharpe = (annual_return - risk_free_rate) / annual_vol

    return sharpe

# Example
np.random.seed(42)
returns = np.random.normal(0.0005, 0.012, 252)  # Daily returns

sr = sharpe_ratio(returns, risk_free_rate=0.02)
print(f"Sharpe Ratio: {sr:.2f}")
```

### Interpreting Sharpe Ratios

| Sharpe Ratio | Interpretation |
|--------------|----------------|
| < 0 | Return below risk-free rate |
| 0 - 1 | Suboptimal but positive excess return |
| 1 - 2 | Good risk-adjusted performance |
| 2 - 3 | Very good performance |
| > 3 | Excellent (verify calculations!) |

<Tip>
A Sharpe ratio above 1 is generally considered good. Hedge funds typically target Sharpe ratios of 1.5-2.0. Ratios above 3 are rare and should be verified carefully.
</Tip>

## 4. Other Risk-Adjusted Metrics

### Sortino Ratio

The Sortino ratio only penalizes downside volatility, recognizing that upside volatility is desirable.

<Definition term="Sortino Ratio">
$$\text{Sortino Ratio} = \frac{R_p - R_f}{\sigma_d}$$
where $\sigma_d$ is the downside deviation (standard deviation of negative returns only).
</Definition>

```python
def sortino_ratio(returns, risk_free_rate=0.02, periods_per_year=252):
    """Calculate the Sortino ratio using downside deviation."""
    # Annualized return
    annual_return = np.mean(returns) * periods_per_year

    # Daily risk-free rate
    daily_rf = risk_free_rate / periods_per_year

    # Downside returns (below risk-free rate)
    downside_returns = returns[returns < daily_rf] - daily_rf

    if len(downside_returns) == 0:
        return np.inf

    # Downside deviation (annualized)
    downside_dev = np.std(downside_returns, ddof=1) * np.sqrt(periods_per_year)

    return (annual_return - risk_free_rate) / downside_dev

# Example
np.random.seed(42)
returns = np.random.normal(0.0005, 0.012, 252)
sortino = sortino_ratio(returns)
print(f"Sortino Ratio: {sortino:.2f}")
```

### Maximum Drawdown

<Definition term="Maximum Drawdown">
Maximum drawdown (MDD) is the largest peak-to-trough decline in portfolio value:
$$\text{MDD} = \min_t \left(\frac{V_t - \max_{s \leq t} V_s}{\max_{s \leq t} V_s}\right)$$
</Definition>

```python
def max_drawdown(returns):
    """
    Calculate the maximum drawdown from a return series.

    Returns:
        Maximum drawdown as a positive percentage
    """
    # Calculate cumulative returns (wealth index)
    wealth = np.cumprod(1 + returns)

    # Running maximum
    running_max = np.maximum.accumulate(wealth)

    # Drawdown at each point
    drawdowns = (wealth - running_max) / running_max

    # Maximum drawdown
    mdd = np.min(drawdowns)

    return -mdd  # Return as positive number

# Example
np.random.seed(42)
returns = np.random.normal(0.0003, 0.015, 500)
mdd = max_drawdown(returns)
print(f"Maximum Drawdown: {mdd:.2%}")
```

### Calmar Ratio

<Definition term="Calmar Ratio">
The Calmar ratio measures return relative to maximum drawdown:
$$\text{Calmar Ratio} = \frac{\text{Annual Return}}{\text{Maximum Drawdown}}$$
</Definition>

## 5. Systematic vs Unsystematic Risk

Total risk can be decomposed into two components:

<Definition term="Systematic Risk">
Systematic risk (market risk) affects all securities and cannot be eliminated through diversification. Examples: interest rate changes, recessions, pandemics.
</Definition>

<Definition term="Unsystematic Risk">
Unsystematic risk (specific risk) is unique to individual securities and CAN be eliminated through diversification. Examples: management changes, product recalls, lawsuits.
</Definition>

<Formula>
$$\text{Total Risk}^2 = \text{Systematic Risk}^2 + \text{Unsystematic Risk}^2$$
</Formula>

```python
import numpy as np
import matplotlib.pyplot as plt

def diversification_effect(n_assets, individual_vol=0.30, avg_correlation=0.3):
    """
    Show how portfolio volatility decreases with number of assets.

    Assumes equal weighting and uniform correlation.
    """
    weights = np.ones(n_assets) / n_assets

    # Build correlation matrix
    corr_matrix = np.full((n_assets, n_assets), avg_correlation)
    np.fill_diagonal(corr_matrix, 1.0)

    # Covariance matrix
    cov_matrix = corr_matrix * (individual_vol ** 2)

    # Portfolio variance
    port_var = weights @ cov_matrix @ weights

    return np.sqrt(port_var)

# Show diversification effect
n_range = range(1, 101)
volatilities = [diversification_effect(n) for n in n_range]

# Systematic risk floor (from average correlation)
systematic_risk = 0.30 * np.sqrt(0.3)  # vol * sqrt(correlation)

plt.figure(figsize=(10, 6))
plt.plot(n_range, volatilities, 'b-', linewidth=2, label='Portfolio Volatility')
plt.axhline(y=systematic_risk, color='r', linestyle='--',
            label=f'Systematic Risk Floor ({systematic_risk:.1%})')
plt.fill_between(n_range, systematic_risk, volatilities, alpha=0.3,
                  label='Diversifiable Risk')
plt.xlabel('Number of Assets')
plt.ylabel('Portfolio Volatility')
plt.title('Diversification and Risk Reduction')
plt.legend()
plt.grid(True, alpha=0.3)
plt.show()
```

<Tip>
Most of the diversification benefit is achieved with 20-30 uncorrelated assets. Beyond that, you're only reducing unsystematic risk marginally while systematic risk remains.
</Tip>

## 6. Beta and the Capital Asset Pricing Model

Beta measures an asset's sensitivity to market movements.

<Definition term="Beta">
Beta ($\beta$) measures the systematic risk of an asset relative to the market:
$$\beta_i = \frac{\text{Cov}(r_i, r_m)}{\text{Var}(r_m)} = \frac{\sigma_{im}}{\sigma_m^2}$$
</Definition>

```python
import numpy as np

def calculate_beta(asset_returns, market_returns):
    """
    Calculate the beta of an asset relative to the market.
    """
    covariance = np.cov(asset_returns, market_returns)[0, 1]
    market_variance = np.var(market_returns, ddof=1)

    return covariance / market_variance

# Example: Calculate beta for a stock
np.random.seed(42)

# Market returns
market_returns = np.random.normal(0.0004, 0.01, 252)

# Stock returns (correlated with market, higher volatility)
stock_returns = 1.5 * market_returns + np.random.normal(0, 0.008, 252)

beta = calculate_beta(stock_returns, market_returns)
print(f"Stock Beta: {beta:.2f}")
```

### Interpreting Beta

| Beta | Interpretation |
|------|----------------|
| β = 0 | No correlation with market |
| β = 1 | Moves with the market |
| β > 1 | More volatile than market (aggressive) |
| 0 < β < 1 | Less volatile than market (defensive) |
| β < 0 | Moves opposite to market (rare) |

### Capital Asset Pricing Model (CAPM)

<Formula>
$$\mathbb{E}[R_i] = R_f + \beta_i (\mathbb{E}[R_m] - R_f)$$
</Formula>

The CAPM states that expected return equals the risk-free rate plus a risk premium proportional to beta.

## 7. The Efficient Frontier

The efficient frontier represents optimal portfolios offering the highest expected return for each level of risk.

<Definition term="Efficient Frontier">
The efficient frontier is the set of portfolios that maximize expected return for each level of risk (volatility). No portfolio on the efficient frontier can be improved without increasing risk.
</Definition>

```python
import numpy as np

def generate_efficient_frontier(expected_returns, cov_matrix, n_points=100):
    """
    Generate points on the efficient frontier using random portfolios.
    This is a simplified approach - true optimization uses quadratic programming.
    """
    n_assets = len(expected_returns)
    results = []

    # Generate many random portfolios
    for _ in range(10000):
        weights = np.random.random(n_assets)
        weights /= weights.sum()

        port_return = np.dot(weights, expected_returns)
        port_vol = np.sqrt(weights @ cov_matrix @ weights)

        results.append((port_vol, port_return, weights))

    return results
```

Portfolios BELOW the efficient frontier are inefficient - you can get higher return for the same risk, or the same return for less risk. Never hold an inefficient portfolio!

## 8. Risk Metrics in Quantum Finance

Quantum computing can accelerate risk calculations in several ways:

| Risk Calculation | Classical Approach | Quantum Advantage |
|------------------|-------------------|-------------------|
| Portfolio VaR | Monte Carlo simulation | Quantum amplitude estimation |
| Correlation estimation | Sample statistics | Quantum principal component analysis |
| Optimal portfolio | Quadratic programming | QAOA/VQE |
| Scenario analysis | Brute force | Quantum sampling |

<Tip>
Quantum speedups in risk calculation could enable real-time risk management that currently takes hours on classical computers.
</Tip>
