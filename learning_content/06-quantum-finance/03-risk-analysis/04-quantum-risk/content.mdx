## 1. Quantum Advantage for Risk Calculations

Many risk calculations rely on Monte Carlo simulation, where quantum computing offers potential speedups.

<Definition term="Quantum Amplitude Estimation (QAE)">
A quantum algorithm that estimates the probability amplitude of a quantum state, achieving quadratic speedup over classical Monte Carlo: O(1/ε) vs O(1/ε²) for accuracy ε.
</Definition>

### Classical vs Quantum Complexity

| Calculation | Classical MC | Quantum QAE | Speedup |
|-------------|--------------|-------------|---------|
| Probability estimation | O(1/ε²) | O(1/ε) | Quadratic |
| Expected value | O(1/ε²) | O(1/ε) | Quadratic |
| VaR/CVaR | O(1/ε²) | O(1/ε) | Quadratic |

<Formula>
Classical Monte Carlo error scales as:
$$\epsilon_{classical} = \frac{\sigma}{\sqrt{N}}$$
Quantum amplitude estimation achieves:
$$\epsilon_{quantum} = \frac{\pi}{M}$$
where $M$ is the number of Grover iterations.
</Formula>

## 2. Quantum Amplitude Estimation

The core algorithm for quantum risk calculations.

```python
import numpy as np
from qiskit import QuantumCircuit, QuantumRegister, ClassicalRegister
from qiskit.circuit.library import QFT

def create_state_preparation_circuit(probabilities):
    """
    Create circuit that prepares a state encoding probabilities.

    |ψ⟩ = √p₀|0⟩ + √p₁|1⟩ + ...

    Args:
        probabilities: Array of probabilities (must sum to 1)

    Returns:
        QuantumCircuit
    """
    n_qubits = int(np.ceil(np.log2(len(probabilities))))
    n_states = 2**n_qubits

    # Pad probabilities if needed
    probs = np.zeros(n_states)
    probs[:len(probabilities)] = probabilities

    # Normalize
    probs = probs / probs.sum()

    # Calculate amplitudes
    amplitudes = np.sqrt(probs)

    qc = QuantumCircuit(n_qubits)

    # Use state preparation (simplified - real impl uses decomposition)
    qc.initialize(amplitudes, range(n_qubits))

    return qc

def create_grover_operator(state_prep, oracle):
    """
    Create Grover diffusion operator for amplitude estimation.

    Q = A S₀ A† Sχ

    Args:
        state_prep: State preparation circuit A
        oracle: Oracle circuit Sχ marking good states

    Returns:
        Grover operator circuit
    """
    n_qubits = state_prep.num_qubits
    qc = QuantumCircuit(n_qubits)

    # Oracle Sχ
    qc.compose(oracle, inplace=True)

    # A†
    qc.compose(state_prep.inverse(), inplace=True)

    # S₀ (reflection about |0⟩)
    qc.x(range(n_qubits))
    qc.h(n_qubits - 1)
    qc.mcx(list(range(n_qubits - 1)), n_qubits - 1)
    qc.h(n_qubits - 1)
    qc.x(range(n_qubits))

    # A
    qc.compose(state_prep, inplace=True)

    return qc
```

## 3. Quantum VaR Estimation

Apply quantum algorithms to Value at Risk calculation.

```python
class QuantumVaREstimator:
    """
    Quantum-inspired VaR estimation using amplitude estimation concepts.
    """

    def __init__(self, n_assets, n_scenarios=1000):
        self.n_assets = n_assets
        self.n_scenarios = n_scenarios

    def encode_returns(self, returns_matrix):
        """
        Encode return scenarios as quantum amplitudes.

        In a full quantum implementation, this would prepare
        a superposition of all scenarios.
        """
        self.scenarios = returns_matrix

    def classical_var(self, weights, confidence_level=0.95):
        """Classical VaR for comparison."""
        portfolio_returns = self.scenarios @ weights
        percentile = (1 - confidence_level) * 100
        return -np.percentile(portfolio_returns, percentile)

    def quantum_var_estimate(self, weights, confidence_level=0.95, n_oracle_calls=100):
        """
        Simulated quantum VaR using amplitude estimation principle.

        In reality, this would:
        1. Prepare superposition of all scenarios
        2. Use oracle to mark scenarios with loss > threshold
        3. Apply QAE to estimate probability
        4. Binary search on threshold to find VaR

        Here we simulate the quadratic speedup.
        """
        # Classical needs O(1/ε²) samples
        # Quantum needs O(1/ε) oracle calls

        # Simulate quantum estimation with reduced samples
        effective_samples = n_oracle_calls ** 2  # Quadratic advantage

        portfolio_returns = self.scenarios @ weights

        # Subsample to simulate limited quantum queries
        n_use = min(effective_samples, len(portfolio_returns))
        indices = np.random.choice(len(portfolio_returns), n_use, replace=True)
        sampled_returns = portfolio_returns[indices]

        percentile = (1 - confidence_level) * 100
        return -np.percentile(sampled_returns, percentile)

    def estimate_speedup(self, target_accuracy):
        """
        Estimate quantum speedup for VaR calculation.
        """
        classical_samples = 1 / target_accuracy**2
        quantum_queries = 1 / target_accuracy
        return classical_samples / quantum_queries

# Example
np.random.seed(42)
n_assets = 5
n_scenarios = 10000

# Generate return scenarios
mean_returns = np.array([0.10, 0.08, 0.12, 0.06, 0.09]) / 252
cov = np.eye(n_assets) * 0.04 / 252 + np.ones((n_assets, n_assets)) * 0.01 / 252
scenarios = np.random.multivariate_normal(mean_returns, cov, n_scenarios)

estimator = QuantumVaREstimator(n_assets, n_scenarios)
estimator.encode_returns(scenarios)

weights = np.array([0.25, 0.20, 0.25, 0.15, 0.15])

classical_var = estimator.classical_var(weights)
quantum_var = estimator.quantum_var_estimate(weights, n_oracle_calls=100)

print("VaR Estimation:")
print(f"  Classical VaR (10000 samples): {classical_var:.4f}")
print(f"  Quantum VaR (100 queries): {quantum_var:.4f}")
print(f"  Difference: {abs(quantum_var - classical_var):.4f}")

speedup = estimator.estimate_speedup(0.01)
print(f"\nTheoretical speedup at 1% accuracy: {speedup:.0f}x")
```

## 4. Quantum Monte Carlo for Risk

Quantum speedup for Monte Carlo simulation of risk metrics.

```python
class QuantumMonteCarlo:
    """
    Quantum Monte Carlo for risk calculations.

    Demonstrates the principles of quantum speedup for
    expectation value estimation.
    """

    def __init__(self, n_qubits_state=4, n_qubits_precision=6):
        """
        Args:
            n_qubits_state: Qubits for encoding the probability distribution
            n_qubits_precision: Qubits for QAE precision
        """
        self.n_state = n_qubits_state
        self.n_precision = n_qubits_precision

    def classical_expectation(self, samples, payoff_fn, n_samples=10000):
        """
        Classical Monte Carlo expectation.

        Error = σ/√n
        """
        payoffs = np.array([payoff_fn(s) for s in samples[:n_samples]])
        return np.mean(payoffs), np.std(payoffs) / np.sqrt(n_samples)

    def quantum_expectation_simulation(self, samples, payoff_fn, n_queries=100):
        """
        Simulated quantum expectation estimation.

        Error = O(1/n_queries) instead of O(1/√n_samples)
        """
        # In quantum, we'd prepare |ψ⟩ = Σ√pᵢ|i⟩
        # Then use QAE to estimate E[payoff]

        # Simulate with effective sample count
        effective_samples = n_queries ** 2

        n_use = min(effective_samples, len(samples))
        indices = np.random.choice(len(samples), n_use, replace=True)
        used_samples = samples[indices]

        payoffs = np.array([payoff_fn(s) for s in used_samples])
        estimate = np.mean(payoffs)
        error = np.std(payoffs) / n_queries  # Quantum error scaling

        return estimate, error

    def calculate_var_cvar(self, returns, confidence=0.95):
        """
        Calculate VaR and CVaR using quantum-inspired methods.
        """
        var = -np.percentile(returns, (1 - confidence) * 100)
        cvar = -np.mean(returns[returns < -var])

        return var, cvar

# Example: Option pricing with quantum MC
def option_payoff(spot_return, strike=1.0, option_type='call'):
    """European option payoff."""
    final_spot = 100 * (1 + spot_return)
    if option_type == 'call':
        return max(final_spot - strike * 100, 0)
    else:
        return max(strike * 100 - final_spot, 0)

np.random.seed(42)

# Quantum MC (simulated)

    returns, option_payoff, n_queries=100
)

print("Option Price Estimation:")
```

## 5. Practical Quantum Risk Circuits

Building blocks for quantum risk calculations.

```python
from qiskit import QuantumCircuit
from qiskit.circuit.library import GroverOperator

def create_loss_oracle(n_qubits, threshold_state):
    """
    Create oracle marking states with loss > threshold.

    Args:
        n_qubits: Number of qubits encoding loss levels
        threshold_state: Binary state representing threshold

    Returns:
        Oracle circuit
    """
    qc = QuantumCircuit(n_qubits)

    # Mark states >= threshold by flipping sign
    # This is simplified - real implementation needs
    # comparator circuit

    # For demonstration: mark specific state
    for i, bit in enumerate(threshold_state[::-1]):
        if bit == '0':
            qc.x(i)

    # Multi-controlled Z
    if n_qubits > 1:
        qc.h(n_qubits - 1)
        qc.mcx(list(range(n_qubits - 1)), n_qubits - 1)
        qc.h(n_qubits - 1)
    else:
        qc.z(0)

    # Undo X gates
    for i, bit in enumerate(threshold_state[::-1]):
        if bit == '0':
            qc.x(i)

    return qc

def create_qae_circuit(state_prep, grover_op, n_counting_qubits):
    """
    Create quantum amplitude estimation circuit.

    Uses quantum phase estimation on Grover operator.
    """
    n_state = state_prep.num_qubits
    total_qubits = n_counting_qubits + n_state

    qc = QuantumCircuit(total_qubits, n_counting_qubits)

    # Counting qubits in superposition
    qc.h(range(n_counting_qubits))

    # State preparation
    qc.compose(state_prep, qubits=range(n_counting_qubits, total_qubits), inplace=True)

    # Controlled Grover operators
    for i in range(n_counting_qubits):
        power = 2**i
        for _ in range(power):
            # Controlled Grover
            controlled_grover = grover_op.control(1)
            qc.compose(controlled_grover,
                      qubits=[i] + list(range(n_counting_qubits, total_qubits)),
                      inplace=True)

    # Inverse QFT on counting register
    qc.compose(QFT(n_counting_qubits, inverse=True),
              qubits=range(n_counting_qubits), inplace=True)

    # Measure counting register
    qc.measure(range(n_counting_qubits), range(n_counting_qubits))

    return qc
```

## 6. Current Limitations and Future Outlook

<Warning>
Current NISQ devices cannot realize the full quadratic speedup for practical risk calculations. The following limitations exist:
- Limited qubit count (~100-1000 noisy qubits)
- Short coherence times
- High gate error rates
- Limited connectivity
</Warning>

### Near-term vs Future Potential

| Aspect | Current (NISQ) | Future (Fault-Tolerant) |
|--------|---------------|------------------------|
| Qubits | ~100-1000 | Millions |
| Error rate | 10⁻³ - 10⁻² | 10⁻⁹ - 10⁻⁶ |
| Circuit depth | ~100 gates | Millions |
| VaR calculation | Research only | Practical advantage |
| Speedup realized | None (noise) | Quadratic |
