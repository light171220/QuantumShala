# Mathematical Foundations of ML-DSA

ML-DSA's security rests on well-studied mathematical problems from lattice cryptography. This lesson explores the Module-LWE and Module-SIS problems, polynomial ring arithmetic, and the ingenious Fiat-Shamir with Aborts paradigm that makes secure signing possible.

## The Polynomial Ring

ML-DSA operates over a polynomial ring with special properties:

```
┌─────────────────────────────────────────────────────────────────┐
│                  THE POLYNOMIAL RING                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Definition:                                                   │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   R_q = Z_q[X] / (X^n + 1)                             │  │
│   │                                                         │  │
│   │   Where:                                                │  │
│   │   • n = 256 (polynomial degree)                        │  │
│   │   • q = 8,380,417 (prime modulus)                      │  │
│   │   • Z_q = integers modulo q                            │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   A Polynomial in R_q:                                          │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   a(X) = a₀ + a₁X + a₂X² + ... + a₂₅₅X²⁵⁵            │  │
│   │                                                         │  │
│   │   Each coefficient aᵢ ∈ {0, 1, ..., q-1}               │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Special Property: X^256 ≡ -1 (mod X^256 + 1)                 │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   When multiplying, high-degree terms wrap around:     │  │
│   │                                                         │  │
│   │   X^256 → -1                                           │  │
│   │   X^257 → -X                                           │  │
│   │   X^258 → -X²                                          │  │
│   │   etc.                                                  │  │
│   │                                                         │  │
│   │   This creates a negacyclic convolution structure      │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Why q = 8,380,417?                                           │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   • q ≡ 1 (mod 512) enabling efficient NTT             │  │
│   │   • q = 2²³ - 2¹³ + 1 (special form for fast mod)     │  │
│   │   • Prime, allowing modular inverse                    │  │
│   │   • Large enough for security, small enough for speed │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Module-LWE Problem

ML-DSA's key generation security relies on Module-LWE:

```
┌─────────────────────────────────────────────────────────────────┐
│                  MODULE-LWE PROBLEM                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Setup:                                                        │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   A ∈ R_q^{k×l}    (public matrix)                     │  │
│   │   s₁ ∈ R_q^l       (secret vector, small coefficients) │  │
│   │   s₂ ∈ R_q^k       (secret vector, small coefficients) │  │
│   │                                                         │  │
│   │   Compute: t = A·s₁ + s₂                               │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   The Problem:                                                  │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Given: (A, t)                                        │  │
│   │   Find:  (s₁, s₂) such that t = A·s₁ + s₂             │  │
│   │                                                         │  │
│   │   This is computationally hard when:                   │  │
│   │   • s₁, s₂ have small coefficients (bounded by η)     │  │
│   │   • The lattice dimension is large enough             │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Decisional Version (MLWE):                                   │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Distinguish between:                                  │  │
│   │   • (A, t = A·s₁ + s₂) with small s₁, s₂              │  │
│   │   • (A, u) where u is uniformly random                 │  │
│   │                                                         │  │
│   │   These should be computationally indistinguishable    │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   In ML-DSA:                                                    │
│   • Public key contains (A, t) - looks random                  │
│   • Private key contains (s₁, s₂) - enables signing           │
│   • Finding s from (A, t) would break the scheme              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Module-SIS Problem

Signature security relies on Module-SIS:

```
┌─────────────────────────────────────────────────────────────────┐
│                  MODULE-SIS PROBLEM                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Short Integer Solution (SIS):                                 │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Given: A ∈ R_q^{k×l}                                 │  │
│   │   Find:  non-zero z ∈ R_q^l such that:                 │  │
│   │          • A·z = 0 (mod q)                             │  │
│   │          • ||z||∞ ≤ β (coefficients bounded)           │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Why It's Hard:                                                │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   • Many solutions exist (kernel of A is large)        │  │
│   │   • But finding SHORT solutions is hard                │  │
│   │   • Related to finding short vectors in lattices       │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Inhomogeneous SIS (ISIS) - Used in Signing:                  │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Given: A ∈ R_q^{k×l}, target u ∈ R_q^k              │  │
│   │   Find:  short z such that A·z = u (mod q)            │  │
│   │                                                         │  │
│   │   The signer can do this (knows trapdoor s₁)          │  │
│   │   Forger cannot (would need to solve ISIS)            │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Connection to Signatures:                                     │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   A valid signature (z, h) must satisfy:               │  │
│   │                                                         │  │
│   │   A·z = c·t + w  (approximately)                       │  │
│   │                                                         │  │
│   │   • z must be SHORT (bounded coefficients)            │  │
│   │   • Without secret key, finding such z is ISIS-hard   │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Coefficient Distributions

ML-DSA uses specific distributions for sampling coefficients:

```
┌─────────────────────────────────────────────────────────────────┐
│              COEFFICIENT DISTRIBUTIONS                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   1. Uniform Distribution (for matrix A):                       │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Each coefficient uniformly in {0, 1, ..., q-1}       │  │
│   │   Generated via XOF (SHAKE-128) from seed ρ            │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   2. Bounded Uniform (for secrets s₁, s₂):                     │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Coefficients uniformly in {-η, ..., -1, 0, 1, ..., η}│  │
│   │                                                         │  │
│   │   η values by variant:                                 │  │
│   │   • ML-DSA-44: η = 2                                   │  │
│   │   • ML-DSA-65: η = 4                                   │  │
│   │   • ML-DSA-87: η = 2                                   │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   3. Bounded Uniform (for y in signing):                       │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Coefficients uniformly in {-γ₁+1, ..., γ₁}           │  │
│   │                                                         │  │
│   │   γ₁ values by variant:                                │  │
│   │   • ML-DSA-44: γ₁ = 2¹⁷ = 131,072                     │  │
│   │   • ML-DSA-65: γ₁ = 2¹⁹ = 524,288                     │  │
│   │   • ML-DSA-87: γ₁ = 2¹⁹ = 524,288                     │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   4. Challenge Polynomial (for c):                              │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   c has exactly τ coefficients that are ±1            │  │
│   │   Remaining 256-τ coefficients are 0                   │  │
│   │                                                         │  │
│   │   τ values by variant:                                 │  │
│   │   • ML-DSA-44: τ = 39                                  │  │
│   │   • ML-DSA-65: τ = 49                                  │  │
│   │   • ML-DSA-87: τ = 60                                  │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## The Fiat-Shamir Transform

Fiat-Shamir converts an interactive identification protocol to a non-interactive signature:

```
┌─────────────────────────────────────────────────────────────────┐
│              FIAT-SHAMIR TRANSFORM                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Interactive Protocol (Identification):                        │
│                                                                 │
│   ┌──────────────┐                      ┌──────────────────┐   │
│   │   Prover     │                      │     Verifier     │   │
│   │   (knows s)  │                      │   (knows t=As+e) │   │
│   └──────┬───────┘                      └────────┬─────────┘   │
│          │                                       │              │
│          │  1. Commitment: w = Ay                │              │
│          │ ─────────────────────────────────────▶│              │
│          │                                       │              │
│          │  2. Challenge: c ← random             │              │
│          │ ◀─────────────────────────────────────│              │
│          │                                       │              │
│          │  3. Response: z = y + cs              │              │
│          │ ─────────────────────────────────────▶│              │
│          │                                       │              │
│          │              4. Verify: Az ≈ w + ct   │              │
│          │                                       │              │
│                                                                 │
│   Non-Interactive Signature (Fiat-Shamir):                     │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Replace random challenge with hash:                   │  │
│   │                                                         │  │
│   │   c = H(w, message)                                    │  │
│   │                                                         │  │
│   │   Now anyone can verify without interaction:           │  │
│   │   1. Compute w' from (z, c) using public key          │  │
│   │   2. Check c == H(w', message)                        │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Security Intuition:                                           │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   • Prover commits to w before seeing c               │  │
│   │   • Can't choose w based on c (would need to break H) │  │
│   │   • Response z reveals no info about s (if done right)│  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Why Aborts Are Necessary

The basic Fiat-Shamir response leaks information:

```
┌─────────────────────────────────────────────────────────────────┐
│              THE LEAKAGE PROBLEM                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Without Aborts:                                               │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   z = y + c·s                                          │  │
│   │                                                         │  │
│   │   Problem: z is shifted by c·s                         │  │
│   │                                                         │  │
│   │   If s = [1, 0, -1, ...] and c = [...],               │  │
│   │   then z is biased in the direction of c·s            │  │
│   │                                                         │  │
│   │   Collecting many signatures → statistical attack     │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Visualization:                                                │
│                                                                 │
│   Distribution of z WITHOUT aborts:                            │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │         y (uniform)        c·s (fixed shift)           │  │
│   │      ╭───────────╮      ───────▶                       │  │
│   │   ───┼───────────┼───────────────────────▶ z           │  │
│   │      │     ●     │        ●                            │  │
│   │      ╰───────────╯   (shifted)                         │  │
│   │                                                         │  │
│   │   The center of z's distribution reveals c·s!         │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Attack Strategy:                                              │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   1. Collect many signatures (c₁,z₁), (c₂,z₂), ...    │  │
│   │   2. Estimate E[z|c] = E[y] + c·s = c·s (since E[y]=0)│  │
│   │   3. Solve linear system to recover s                  │  │
│   │                                                         │  │
│   │   With enough signatures, s is fully recovered!       │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Rejection Sampling: The Solution

```
┌─────────────────────────────────────────────────────────────────┐
│                  REJECTION SAMPLING                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Key Idea:                                                     │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Only output z if it "looks like" it could have come │  │
│   │   from ANY secret s, not just the real one.           │  │
│   │                                                         │  │
│   │   Reject and restart if z reveals too much about s.   │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   ML-DSA's Approach:                                            │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   1. Sample y with large range: |y| < γ₁               │  │
│   │   2. Compute z = y + c·s                               │  │
│   │   3. If ||z||∞ ≥ γ₁ - β, ABORT and restart            │  │
│   │   4. Otherwise, output z                               │  │
│   │                                                         │  │
│   │   Where β bounds ||c·s||∞                              │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Visualization:                                                │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │      ← ─ ─ ─ ─ ─ ─ ─ 2γ₁ ─ ─ ─ ─ ─ ─ ─ →              │  │
│   │      ╭───────────────────────────────────╮              │  │
│   │      │                                   │              │  │
│   │   ───┼───────────────────────────────────┼───          │  │
│   │      │    ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓   │              │  │
│   │      │        Acceptable zone           │              │  │
│   │      ╰───────────────────────────────────╯              │  │
│   │              ← ─ 2(γ₁-β) ─ →                           │  │
│   │                                                         │  │
│   │   Accept only if z falls in acceptable zone            │  │
│   │   This zone is same regardless of s!                  │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Expected Iterations:                                          │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Probability of success ≈ (γ₁ - β)^n / γ₁^n          │  │
│   │                                                         │  │
│   │   ML-DSA-44: ~4.25 iterations on average               │  │
│   │   ML-DSA-65: ~5.1 iterations on average                │  │
│   │   ML-DSA-87: ~3.85 iterations on average               │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Mathematical Security Proof Sketch

```
┌─────────────────────────────────────────────────────────────────┐
│                  SECURITY PROOF OUTLINE                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Theorem: ML-DSA is EUF-CMA secure if MLWE and MSIS are hard  │
│                                                                 │
│   EUF-CMA = Existential Unforgeability under Chosen Message    │
│             Attack (strongest standard signature security)      │
│                                                                 │
│   Proof Structure:                                              │
│                                                                 │
│   1. Key Indistinguishability (from MLWE):                     │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Public key (A, t) where t = A·s₁ + s₂               │  │
│   │   ≈ indistinguishable from ≈                           │  │
│   │   Random (A, u)                                        │  │
│   │                                                         │  │
│   │   → Attacker can't distinguish real key from random   │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   2. Unforgeability (from MSIS):                               │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   A forged signature (z, c, h) must satisfy:           │  │
│   │   • Az = c·t + w (approximately)                       │  │
│   │   • ||z||∞ bounded                                     │  │
│   │                                                         │  │
│   │   Creating such z without knowing s₁ is MSIS-hard     │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   3. Zero-Knowledge (from rejection sampling):                 │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Signatures reveal nothing about s beyond public key  │  │
│   │                                                         │  │
│   │   The distribution of valid (z, c) pairs is:          │  │
│   │   • Independent of the specific secret s              │  │
│   │   • Simulatable without knowing s                      │  │
│   │                                                         │  │
│   │   → Multiple signatures don't help attacker           │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## NTT in ML-DSA

Like ML-KEM, ML-DSA uses NTT for efficient polynomial operations:

```
┌─────────────────────────────────────────────────────────────────┐
│                  NTT IN ML-DSA                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Why NTT?                                                      │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Polynomial multiplication:                            │  │
│   │   • Naive: O(n²) = O(65,536) operations               │  │
│   │   • NTT:   O(n log n) = O(2,048) operations           │  │
│   │                                                         │  │
│   │   ML-DSA requires many polynomial multiplications     │  │
│   │   NTT makes it practical                               │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   ML-DSA's Modulus q = 8,380,417:                              │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   q = 2²³ - 2¹³ + 1                                    │  │
```

## Summary

ML-DSA's mathematical foundations:

**Polynomial Ring:**
- R_q = Z_q[X]/(X^256 + 1) with q = 8,380,417
- Enables efficient NTT-based arithmetic
- Negacyclic structure from X^256 ≡ -1

**Hard Problems:**
- Module-LWE: Hiding the secret in t = As₁ + s₂
- Module-SIS: Creating short solutions without the trapdoor
- Both reduce to hard lattice problems (SVP/CVP)

**Fiat-Shamir with Aborts:**
- Converts identification to signature
- Rejection sampling prevents information leakage
- Ensures signatures are independent of secret key
- ~4-5 attempts average per signature

**Security:**
- EUF-CMA secure under MLWE/MSIS hardness
- Quantum-resistant (no known efficient quantum attacks)
- Well-studied with conservative parameters
