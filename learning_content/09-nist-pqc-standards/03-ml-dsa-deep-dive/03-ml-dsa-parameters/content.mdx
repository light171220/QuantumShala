# ML-DSA Parameter Sets

ML-DSA offers three standardized parameter sets, each targeting different security levels. Understanding these parameters is essential for selecting the right variant and correctly implementing the algorithm.

## Parameter Definitions

```
┌─────────────────────────────────────────────────────────────────┐
│              PARAMETER DEFINITIONS                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Core Parameters:                                              │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │  q = 8,380,417                                         │  │
│   │      Modulus for polynomial coefficients               │  │
│   │      q = 2²³ - 2¹³ + 1 (enables fast arithmetic)      │  │
│   │                                                         │  │
│   │  n = 256                                               │  │
│   │      Polynomial degree (ring R_q = Z_q[X]/(X^n+1))    │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Matrix Dimensions:                                            │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │  k = rows of matrix A (4, 6, or 8)                    │  │
│   │  l = columns of matrix A (4, 5, or 7)                 │  │
│   │                                                         │  │
│   │  Higher k, l → more security but larger sizes         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Secret Key Parameters:                                        │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │  η = coefficient bound for secrets s₁, s₂             │  │
│   │      Coefficients uniform in [-η, η]                   │  │
│   │                                                         │  │
│   │  β = τ·η = maximum norm of c·s                        │  │
│   │      Used for rejection bound                          │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Signing Parameters:                                           │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │  γ₁ = bound for masking polynomial y                  │  │
│   │       y coefficients in [-γ₁+1, γ₁]                   │  │
│   │                                                         │  │
│   │  γ₂ = rounding parameter for signature                │  │
│   │       Controls precision of hint computation          │  │
│   │                                                         │  │
│   │  τ = number of ±1 coefficients in challenge c         │  │
│   │                                                         │  │
│   │  ω = maximum number of 1s in hint h                   │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Key and Signature Sizes

```
┌─────────────────────────────────────────────────────────────────┐
│                  SIZE CALCULATIONS                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Public Key (pk = ρ ∥ t₁):                                    │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   pk_size = 32 + k · 320                               │  │
│   │                                                         │  │
│   │   • ρ: 32 bytes (seed for matrix A)                   │  │
│   │   • t₁: k polynomials, 10 bits/coefficient            │  │
│   │         k · 256 · 10 / 8 = k · 320 bytes             │  │
│   │                                                         │  │
│   │   ML-DSA-44: 32 + 4·320 = 1,312 bytes                 │  │
│   │   ML-DSA-65: 32 + 6·320 = 1,952 bytes                 │  │
│   │   ML-DSA-87: 32 + 8·320 = 2,592 bytes                 │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Secret Key (sk = ρ ∥ K ∥ tr ∥ s₁ ∥ s₂ ∥ t₀):               │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   sk_size = 32 + 32 + 64 + l·32η + k·32η + k·416      │  │
│   │                                                         │  │
│   │   • ρ: 32 bytes                                        │  │
│   │   • K: 32 bytes (signing randomness seed)             │  │
│   │   • tr: 64 bytes (hash of public key)                 │  │
│   │   • s₁: l polynomials, packed                         │  │
│   │   • s₂: k polynomials, packed                         │  │
│   │   • t₀: k polynomials, 13 bits/coefficient            │  │
│   │                                                         │  │
│   │   ML-DSA-44: 2,560 bytes                              │  │
│   │   ML-DSA-65: 4,032 bytes                              │  │
│   │   ML-DSA-87: 4,896 bytes                              │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Signature (σ = c̃ ∥ z ∥ h):                                  │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   σ_size = 32 + l·32·(1 + log₂(γ₁-1)) + ω + k         │  │
│   │                                                         │  │
│   │   • c̃: 32 bytes (hashed challenge)                   │  │
│   │   • z: l polynomials, packed                          │  │
│   │   • h: encoded hint (variable, max ω+k bytes)         │  │
│   │                                                         │  │
│   │   ML-DSA-44: 2,420 bytes                              │  │
│   │   ML-DSA-65: 3,293 bytes                              │  │
│   │   ML-DSA-87: 4,595 bytes                              │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Summary

**ML-DSA-44:**
- Public key: 1,312 bytes, Signature: 2,420 bytes
- NIST Category 2 (~128-bit security)
- Smallest variant, minimal security margin
- Use only when size constraints are critical

**ML-DSA-65 (Recommended):**
- Public key: 1,952 bytes, Signature: 3,293 bytes
- NIST Category 3 (~192-bit security)
- Best balance of security and efficiency
- Default choice for most applications

**ML-DSA-87:**
- Public key: 2,592 bytes, Signature: 4,595 bytes
- NIST Category 5 (~256-bit security)
- Highest security, largest sizes
- For classified data and critical infrastructure
