# ML-DSA Security Analysis

Understanding ML-DSA's security is essential for confident deployment. This lesson examines the security foundations, attack resistance, side-channel considerations, and long-term security outlook.

## Security Reduction

```
┌─────────────────────────────────────────────────────────────────┐
│              SECURITY REDUCTION                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Theorem: ML-DSA is EUF-CMA secure if:                        │
│   1. Module-LWE is hard (for key hiding)                       │
│   2. Module-SIS is hard (for forgery resistance)               │
│   3. Hash functions behave as random oracles                   │
│                                                                 │
│   Reduction Structure:                                          │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   If adversary A forges ML-DSA signatures,             │  │
│   │                                                         │  │
│   │   Then we can build:                                   │  │
│   │                                                         │  │
│   │   • Algorithm B that solves MLWE                       │  │
│   │     (by simulating signing oracle)                     │  │
│   │                                                         │  │
│   │   OR                                                    │  │
│   │                                                         │  │
│   │   • Algorithm C that solves MSIS                       │  │
│   │     (by extracting short vector from forgery)         │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Key Hiding (from MLWE):                                      │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   pk = (A, t = As₁ + s₂)                               │  │
│   │                                                         │  │
│   │   t looks random (indistinguishable from uniform)      │  │
│   │   by MLWE assumption                                   │  │
│   │                                                         │  │
│   │   → Adversary can't extract s₁, s₂ from public key    │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Forgery Resistance (from MSIS):                              │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   A valid forgery (c*, z*, h*) satisfies:              │  │
│   │                                                         │  │
│   │   A·z* - c*·t = w' where HighBits(w') matches         │  │
│   │                                                         │  │
│   │   This gives: A·z* = c*·t + w' ≈ short                │  │
│   │                                                         │  │
│   │   → Finding such short z* is MSIS-hard                │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Attack Analysis

### Forgery Attacks

```
┌─────────────────────────────────────────────────────────────────┐
│              FORGERY ATTACK ANALYSIS                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Direct Forgery Attempt:                                       │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Goal: Find (z, h) for target message M without sk   │  │
│   │                                                         │  │
│   │   Approach 1: Solve for z directly                    │  │
│   │   • Need z such that A·z - c·t has correct high bits  │  │
│   │   • This is Module-SIS: find short preimage          │  │
│   │   • Complexity: ~2^{163} for ML-DSA-65 (quantum)     │  │
│   │                                                         │  │
│   │   Approach 2: Guess y, compute signature              │  │
│   │   • Need y such that z = y + c·s₁ is short           │  │
│   │   • Without knowing s₁, y is independent of c        │  │
│   │   • Probability of success: negligible               │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Key Recovery Attack:                                          │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Goal: Recover s₁, s₂ from public key or signatures  │  │
│   │                                                         │  │
│   │   From public key:                                     │  │
│   │   • t = A·s₁ + s₂ is Module-LWE sample               │  │
│   │   • Best attack: lattice reduction (BKZ/sieving)     │  │
│   │   • Complexity: ~2^{163} for ML-DSA-65 (quantum)     │  │
│   │                                                         │  │
│   │   From signatures:                                     │  │
│   │   • z = y + c·s₁ reveals nothing about s₁            │  │
│   │   • Rejection sampling ensures z independent of s₁   │  │
│   │   • Infinite signatures don't help                    │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Side-Channel Considerations

```
┌─────────────────────────────────────────────────────────────────┐
│              SIDE-CHANNEL ATTACKS                               │

│                                                                 │

│   │                                                         │  │
│   │   Attack vectors:                                       │  │

│   3. Fault Attacks                                              │

│   │                                                         │  │
```

## Summary

ML-DSA security analysis:

**Mathematical Security:**
- EUF-CMA secure under MLWE and MSIS assumptions
- Quantum-resistant: Shor's algorithm doesn't apply
- Security reduces to well-studied lattice problems
- Rejection sampling provides zero-knowledge property

**Attack Resistance:**
- Direct forgery requires solving MSIS (~2^163 for ML-DSA-65)
- Key recovery requires solving MLWE (same complexity)
- Signatures leak no information about secret key
- Classical and quantum attacks both exponential

**Implementation Considerations:**
- Constant-time operations essential
- Rejection sampling timing can be side-channel
- Fault attacks must be mitigated
- Memory protection for secrets

**Recommendations:**
- ML-DSA-65 for most applications
- ML-DSA-87 for highest security needs
- Use validated implementations
- Plan for algorithm agility
