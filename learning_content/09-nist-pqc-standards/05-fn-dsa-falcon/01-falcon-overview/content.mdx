# FN-DSA (FALCON) Overview

FN-DSA (Fast Fourier Lattice-based Compact Signatures over NTRU) is an upcoming NIST post-quantum signature standard. Based on FALCON, it offers the smallest signatures among NIST's lattice-based signature candidates while maintaining strong security guarantees.

## NTRU Lattice Foundation

```
┌─────────────────────────────────────────────────────────────────┐
│              NTRU LATTICES                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   NTRU vs Module-LWE (used in ML-DSA):                         │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Module-LWE (ML-DSA):                                 │  │
│   │   • Public key: A (random matrix), t = As + e         │  │
│   │   • Security: Hard to find s given (A, t)             │  │
│   │   • Error term e needed for security                  │  │
│   │                                                         │  │
│   │   NTRU (FN-DSA):                                       │  │
│   │   • Public key: h = g/f in polynomial ring            │  │
│   │   • Security: Hard to find (f, g) given h             │  │
│   │   • f, g are short polynomials                        │  │
│   │   • No explicit error term (structure is different)   │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   NTRU Ring Structure:                                         │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Ring: R = Z[X]/(X^n + 1)  (cyclotomic)              │  │
│   │   Quotient ring: R_q = R/qR = Z_q[X]/(X^n + 1)        │  │
│   │                                                         │  │
│   │   n = 512 or 1024 (power of 2)                        │  │
│   │   q = 12289 (prime, q ≡ 1 mod 2n)                     │  │
│   │                                                         │  │
│   │   Key relationship:                                    │  │
│   │   h · f ≡ g (mod q)                                   │  │
│   │                                                         │  │
│   │   where f, g are "short" (small coefficients)         │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   NTRU Lattice:                                                 │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Given public key h, the NTRU lattice is:            │  │
│   │                                                         │  │
│   │   L_h = { (u, v) ∈ R² : u + hv ≡ 0 (mod q) }         │  │
│   │                                                         │  │
│   │   The secret key (f, g) forms a short basis for L_h   │  │
│   │   Finding short vectors in L_h is hard (SIS problem)  │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## How FN-DSA Works (High Level)

```
┌─────────────────────────────────────────────────────────────────┐
│              FN-DSA SIGNATURE SCHEME                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Key Generation:                                               │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   1. Generate short polynomials f, g, F, G such that:  │  │
│   │      fG - gF = q                                       │  │
│   │                                                         │  │
│   │   2. Compute public key:                               │  │
│   │      h = g·f⁻¹ mod q                                  │  │
│   │                                                         │  │
│   │   3. Secret key: (f, g, F, G) forms NTRU trapdoor     │  │
│   │      This is a "good basis" for the NTRU lattice      │  │
│   │                                                         │  │
│   │   Secret Key: (f, g, F, G) ~ 1,281 - 2,305 bytes      │  │
│   │   Public Key: h ~ 897 - 1,793 bytes                   │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Signing:                                                      │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   1. Hash message to polynomial c = H(message)         │  │
│   │                                                         │  │
│   │   2. Using trapdoor (f, g, F, G), sample short        │  │
│   │      signature (s₁, s₂) such that:                    │  │
│   │      s₁ + h·s₂ ≡ c (mod q)                           │  │
│   │                                                         │  │
│   │   3. Sampling uses Fast Fourier Transform (FFT)       │  │
│   │      and Gaussian distribution                        │  │
│   │                                                         │  │
│   │   4. Signature: s₂ (compressed) ~ 666-1,280 bytes     │  │
│   │      (s₁ can be recovered from s₂ and c)             │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Verification:                                                 │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   1. Recompute c = H(message)                          │  │
│   │                                                         │  │
│   │   2. Recover s₁ = c - h·s₂ mod q                      │  │
│   │                                                         │  │
│   │   3. Check: ||(s₁, s₂)|| < bound                      │  │
│   │      (Signature must be "short")                       │  │
│   │                                                         │  │
│   │   Very fast: Just polynomial multiplication and norm  │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Summary

FN-DSA (FALCON) key characteristics:

**Design Philosophy:**
- Optimize for compact signatures
- Use NTRU lattices for efficiency
- Fast Fourier Gaussian sampling

**Key Features:**
- Smallest PQC signatures (666-1,280 bytes)
- Very fast verification
- Different lattice structure (NTRU vs Module-LWE)

**Comparison with ML-DSA:**
- 3-4× smaller signatures
- Slightly smaller public keys
- Slower key generation
- More complex implementation

**Comparison with SLH-DSA:**
- 10-25× smaller signatures
- Larger public keys
- Different security assumptions (lattice vs hash)
- Faster operations

**Best Use Cases:**
- Bandwidth-constrained environments
- Certificate chains (cumulative size matters)
- High-volume verification
- Algorithm diversity

**Implementation Challenges:**
- Floating-point arithmetic required
- Complex Gaussian sampling
- Side-channel protection harder
- Key generation is slow
