# FN-DSA Parameter Sets

FN-DSA defines two parameter sets: FN-DSA-512 and FN-DSA-1024. This lesson examines their characteristics, security levels, and appropriate use cases.

## Parameter Selection Guide

```python
from dataclasses import dataclass
from typing import List

@dataclass
class FNDSAProfile:
    name: str
    n: int
    pk_size: int
    sk_size: int
    sig_avg: int
    sig_max: int
    keygen_ms: float
    sign_us: float
    verify_us: float
    security_bits: int
    nist_category: int

FNDSA_PARAMS = {
    'FN-DSA-512': FNDSAProfile(
        'FN-DSA-512', n=512,
        pk_size=897, sk_size=1281,
        sig_avg=666, sig_max=809,
        keygen_ms=10, sign_us=400, verify_us=50,
        security_bits=108, nist_category=1
    ),
    'FN-DSA-1024': FNDSAProfile(
        'FN-DSA-1024', n=1024,
        pk_size=1793, sk_size=2305,
        sig_avg=1280, sig_max=1462,
        keygen_ms=25, sign_us=900, verify_us=100,
        security_bits=254, nist_category=5
    ),
}

def recommend_fndsa(
    min_security_category: int = 1,
    max_signature_bytes: int = None,
    keygen_latency_critical: bool = False,
    verify_heavy: bool = False
) -> List[str]:
    """
    Recommend FN-DSA parameter set based on requirements.
    """
    candidates = []

    for name, params in FNDSA_PARAMS.items():
        # Check security level
        if params.nist_category < min_security_category:
            continue

        # Check signature size
        if max_signature_bytes and params.sig_max > max_signature_bytes:
            continue

        # Check if keygen latency is acceptable
        if keygen_latency_critical and params.keygen_ms > 15:
            continue

        candidates.append((name, params))

    # Sort by preference
    def score(item):
        name, params = item
        s = 0

        # Prefer matching security level (don't overprovision)
        if params.nist_category == min_security_category:
            s += 100

        # Prefer smaller signatures
        s -= params.sig_avg / 100

        # If verify-heavy, prefer faster verification
        if verify_heavy:
            s -= params.verify_us

        return -s

    candidates.sort(key=score)
    return [name for name, _ in candidates]


# Example usage
print("FN-DSA Parameter Selection Guide")
print("=" * 50)

scenarios = [
    ("Basic PQC (Cat 1)", {'min_security_category': 1}),
    ("Maximum security", {'min_security_category': 5}),
    ("Size-constrained (<1KB)", {'max_signature_bytes': 1000}),
    ("High-volume verification", {'verify_heavy': True}),
]

for scenario_name, kwargs in scenarios:
    recommendations = recommend_fndsa(**kwargs)
    print(f"\n{scenario_name}:")
    if recommendations:
        for rec in recommendations:
            params = FNDSA_PARAMS[rec]
            print(f"  {rec}: sig={params.sig_avg}B, verify={params.verify_us}μs")
    else:
        print("  No suitable FN-DSA variant (consider ML-DSA)")
```

## Summary

FN-DSA parameter sets:

**FN-DSA-512 (Category 1):**
- Smallest PQC signatures (~666 bytes)
- Fast verification (~50 μs)
- Slow key generation (~10 ms)
- Best for bandwidth-constrained, verify-heavy use cases

**FN-DSA-1024 (Category 5):**
- Maximum security level
- Signatures ~1,280 bytes
- Slowest key generation (~25 ms)
- Best for long-term, high-security applications

**Key Trade-offs:**
- No Category 3 option (gap in security levels)
- Complex implementation (floating-point required)
- Slow key generation compared to ML-DSA
- Fastest verification among PQC signatures

**Selection Guidance:**
- Choose FN-DSA when signature size is critical
- Use ML-DSA for simpler implementation
- Use SLH-DSA for maximum confidence in assumptions
