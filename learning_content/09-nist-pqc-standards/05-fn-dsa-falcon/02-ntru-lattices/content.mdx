# NTRU Lattices and Trapdoors

NTRU lattices form the mathematical foundation of FN-DSA. Understanding how these lattices work and how trapdoors enable efficient signing is essential for understanding FALCON-based signatures.

## Polynomial Rings in FN-DSA

```
┌─────────────────────────────────────────────────────────────────┐
│              POLYNOMIAL RING STRUCTURE                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Base Ring:                                                    │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   R = Z[X]/(X^n + 1)                                   │  │
│   │                                                         │  │
│   │   • Polynomials with integer coefficients              │  │
│   │   • Degree at most n-1                                 │  │
│   │   • X^n = -1 (cyclotomic reduction)                   │  │
│   │   • n = 512 or 1024 (power of 2)                      │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Quotient Ring:                                                │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   R_q = Z_q[X]/(X^n + 1)                               │  │
│   │                                                         │  │
│   │   • Coefficients reduced mod q                         │  │
│   │   • q = 12289 (prime, q ≡ 1 mod 2n)                   │  │
│   │   • Enables efficient NTT/FFT                         │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Polynomial Example (n=4, for illustration):                  │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   f(X) = 3 + 2X - X² + 5X³                             │  │
│   │   g(X) = 1 - X + 2X³                                   │  │
│   │                                                         │  │
│   │   Multiplication in R:                                 │  │
│   │   f(X) · g(X) mod (X⁴ + 1)                            │  │
│   │                                                         │  │
│   │   When X⁴ appears, replace with -1                    │  │
│   │   When X⁵ appears, replace with -X                    │  │
│   │   etc.                                                  │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Why This Ring?                                                │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │   • FFT-friendly: n is power of 2                      │  │
│   │   • NTT works: q ≡ 1 mod 2n                           │  │
│   │   • Security: Based on hard lattice problems          │  │
│   │   • Compact: Polynomial representation is efficient    │  │
│   └─────────────────────────────────────────────────────────┘  │

└─────────────────────────────────────────────────────────────────┘
```

## NTRU Key Structure

```
┌─────────────────────────────────────────────────────────────────┐
│              NTRU KEY RELATIONSHIP                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Key Generation Produces:                                      │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Short polynomials: f, g, F, G ∈ R                    │  │
│   │                                                         │  │
│   │   Satisfying the NTRU equation:                        │  │
│   │   ┌─────────────────────────────────────────────────┐  │  │
│   │   │                                                 │  │  │
│   │   │        f · G - g · F = q                       │  │  │
│   │   │                                                 │  │  │
│   │   └─────────────────────────────────────────────────┘  │  │
│   │                                                         │  │
│   │   Public key: h = g · f⁻¹ mod q                       │  │
│   │   (f must be invertible in R_q)                        │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   "Short" Means:                                                │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Coefficients have small absolute values              │  │
│   │   ||f||, ||g||, ||F||, ||G|| are small                │  │
│   │                                                         │  │
│   │   Typical coefficient distribution:                    │  │
│   │   • Gaussian with small standard deviation            │  │
│   │   • Most coefficients in range [-σ, σ] for small σ   │  │
│   │                                                         │  │
│   │   FN-DSA uses σ ≈ 1.17√(q/2n)                        │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Matrix Form:                                                  │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Secret key as 2×2 polynomial matrix:                 │  │
│   │                                                         │  │
│   │        ┌       ┐                                       │  │
│   │   B =  │ f  F  │                                       │  │
│   │        │ g  G  │                                       │  │
│   │        └       ┘                                       │  │
│   │                                                         │  │
│   │   det(B) = fG - gF = q                                │  │
│   │                                                         │  │
│   │   B forms a "good basis" for the NTRU lattice         │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## The NTRU Lattice

```
┌─────────────────────────────────────────────────────────────────┐
│              NTRU LATTICE STRUCTURE                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Given public key h ∈ R_q, define the NTRU lattice:           │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   L_h = { (u, v) ∈ R² : u + h·v ≡ 0 (mod q) }         │  │
│   │                                                         │  │
│   │   In integer coordinates (2n dimensions):              │  │
│   │   Embed each polynomial as vector of coefficients     │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Lattice Visualization (simplified):                          │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │           ·  ·  ·  ·  ·  ·  ·  ·  ·                   │  │
│   │         ·  ·  ·  ·  ·  ·  ·  ·  ·                     │  │
│   │       ·  ·  ·  (f,g)  ·  ·  ·  ·        Short vector  │  │
│   │     ·  ·  ·  ·  ●  ·  ·  ·  ·  ·        (secret key)  │  │
│   │       ·  ·  ·  ·  ·  ·  ·  ·  ·                       │  │
│   │         ·  ·  ·  ·  ·  ·  ·  ·                        │  │
│   │           ·  ·  ·  ·  ·  ·  ·                         │  │
│   │                                                         │  │
│   │   Most lattice points are "long"                       │  │
│   │   (f, g) is an exceptionally short lattice point      │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Security Assumption:                                         │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Given only h, it's hard to find (f, g)               │  │
│   │                                                         │  │
│   │   This is the NTRU problem:                            │  │
│   │   • Find short vectors in L_h                          │  │
│   │   • Related to Shortest Vector Problem (SVP)          │  │
│   │                                                         │  │
│   │   Best known attacks:                                   │  │
│   │   • Lattice reduction (BKZ algorithm)                 │  │
│   │   • Meet-in-the-middle (combinatorial)                │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Trapdoor Construction

```
┌─────────────────────────────────────────────────────────────────┐
│              NTRU TRAPDOOR                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   What is a Trapdoor?                                          │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   A trapdoor is secret information that makes a        │  │
│   │   normally hard problem easy.                          │  │
│   │                                                         │  │
│   │   Hard problem: Find short (s₁, s₂) with              │  │
│   │                 s₁ + h·s₂ ≡ c (mod q)                 │  │
│   │                                                         │  │
│   │   With trapdoor (f, g, F, G): Problem becomes easy!   │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   How the Trapdoor Works:                                      │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Goal: Find short (s₁, s₂) with s₁ + h·s₂ ≡ c       │  │
│   │                                                         │  │
│   │   Step 1: Compute target point                        │  │
│   │           t = (c, 0) in the lattice coset             │  │
│   │           (Any solution (s₁, s₂) satisfies            │  │
│   │            (s₁, s₂) - (c, 0) ∈ L_h)                   │  │
│   │                                                         │  │
│   │   Step 2: Find closest lattice point to t             │  │
│   │           Using the good basis B = [[f,F],[g,G]]      │  │
│   │                                                         │  │
│   │   Step 3: The difference is the signature             │  │
│   │           (s₁, s₂) = t - closest_lattice_point       │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Why Short Basis Matters:                                     │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Short basis B:                                       │  │
│   │   • Gram-Schmidt vectors are short                    │  │
│   │   • Can find closest point efficiently                │  │
│   │   • Result has small norm (short signature)           │  │
│   │                                                         │  │
│   │   Random/long basis:                                   │  │
│   │   • Gram-Schmidt vectors are long                     │  │
│   │   • Closest point algorithm fails                     │  │
│   │   • Cannot produce short signatures                   │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Gram-Schmidt Orthogonalization

```
┌─────────────────────────────────────────────────────────────────┐
│              GRAM-SCHMIDT AND BASIS QUALITY                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Gram-Schmidt Process:                                         │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Given basis B = [b₁, b₂, ...], compute orthogonal:  │  │
│   │                                                         │  │
│   │   b₁* = b₁                                             │  │
│   │   b₂* = b₂ - proj(b₂ onto b₁*)                        │  │
│   │   b₃* = b₃ - proj(b₃ onto b₁*) - proj(b₃ onto b₂*)   │  │
│   │   ...                                                   │  │
│   │                                                         │  │
│   │   where proj(a onto b) = (⟨a,b⟩/⟨b,b⟩) · b            │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Visual Example (2D):                                         │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │        b₂                      b₂*                     │  │
│   │        ↗                       ↑                        │  │
│   │       /                        |                        │  │
│   │      /            →           |                        │  │
│   │     /                          |                        │  │
│   │    ──────→ b₁              ────┼────→ b₁* = b₁        │  │
│   │                                                         │  │
│   │   Original basis          Orthogonalized              │  │
│   │   (may be skewed)         (perpendicular)              │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   Basis Quality Measure:                                        │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                                                         │  │
│   │   Good basis: ||b_i*|| are similar sizes              │  │
│   │   Bad basis: ||b_i*|| decay rapidly                   │  │
│   │                                                         │  │
│   │   max ||b_i*|| determines sampling quality            │  │
│   │                                                         │  │
│   │   FN-DSA needs: max ||b_i*|| ≤ σ for security        │  │
│   │   (σ is the Gaussian sampling parameter)              │  │
│   │                                                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## NTRU Implementation Example

```python
import numpy as np
from numpy.polynomial import polynomial as P

class NTRUPolynomialRing:
    """
    Simple NTRU polynomial ring R_q = Z_q[X]/(X^n + 1).
    Educational implementation - not cryptographically secure.
    """

    def __init__(self, n: int, q: int):
        self.n = n
        self.q = q

    def reduce(self, poly: np.ndarray) -> np.ndarray:
        """Reduce polynomial mod (X^n + 1) and mod q."""
        # Reduce coefficients mod q
        poly = np.array(poly) % self.q

        # Handle coefficients >= n by folding with negation
        while len(poly) > self.n:
            for i in range(len(poly) - self.n):
                poly[i] = (poly[i] - poly[self.n + i]) % self.q
            poly = poly[:self.n]

        # Pad if needed
        if len(poly) < self.n:
            poly = np.concatenate([poly, np.zeros(self.n - len(poly))])

        return poly.astype(int) % self.q

    def mul(self, a: np.ndarray, b: np.ndarray) -> np.ndarray:
        """Multiply polynomials in R_q."""
        # Standard polynomial multiplication
        product = np.convolve(a, b)
        # Reduce to ring
        return self.reduce(product)

    def add(self, a: np.ndarray, b: np.ndarray) -> np.ndarray:
        """Add polynomials in R_q."""
        # Ensure same length
        max_len = max(len(a), len(b))
        a_padded = np.concatenate([a, np.zeros(max_len - len(a))])
        b_padded = np.concatenate([b, np.zeros(max_len - len(b))])
        return ((a_padded + b_padded) % self.q).astype(int)

    def neg(self, a: np.ndarray) -> np.ndarray:
        """Negate polynomial in R_q."""
        return (-a % self.q).astype(int)

    def scalar_mul(self, a: np.ndarray, c: int) -> np.ndarray:
        """Multiply polynomial by scalar in R_q."""
        return ((a * c) % self.q).astype(int)
```

## Summary

NTRU lattices in FN-DSA:

**Polynomial Ring:**
- R_q = Z_q[X]/(X^n + 1)
- n = 512 or 1024, q = 12289
- Enables efficient FFT/NTT

**Key Structure:**
- Secret: Short polynomials (f, g, F, G)
- Relation: fG - gF = q
- Public: h = g·f⁻¹ mod q

**NTRU Lattice:**
- L_h = {(u, v) : u + h·v ≡ 0 (mod q)}
- (f, g) is a short vector in L_h
- Finding short vectors is hard without trapdoor

**Trapdoor Property:**
- Matrix B = [[f, F], [g, G]] is a "good basis"
- Enables finding closest lattice point
- This makes signing efficient

**Security:**
- Based on hardness of finding short vectors in L_h
- Related to SVP (Shortest Vector Problem)
- Resistant to known quantum algorithms
