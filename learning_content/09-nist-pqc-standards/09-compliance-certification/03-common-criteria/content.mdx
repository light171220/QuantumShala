# Common Criteria Evaluation for PQC

Common Criteria (CC) provides an international framework for security evaluation. Understanding how PQC integrates with CC evaluation is important for global market access.

## Evaluation Assurance Levels

```
EAL Levels and PQC Implications:
═══════════════════════════════════════════════════════════════════

EAL 1: Functionally Tested
├─ Basic testing, limited documentation
├─ Suitable for low-risk applications
└─ PQC: Algorithm correctness testing only

EAL 2: Structurally Tested
├─ Developer security testing
├─ Vulnerability analysis
└─ PQC: Basic side-channel considerations

EAL 3: Methodically Tested and Checked
├─ Development environment controls
├─ Configuration management
└─ PQC: Documented implementation security

EAL 4: Methodically Designed, Tested, and Reviewed
├─ Design documentation analysis
├─ Source code review
└─ PQC: Constant-time implementation evidence

EAL 5: Semi-formally Designed and Tested
├─ Formal security model
├─ Covert channel analysis
└─ PQC: Formal analysis of crypto operations

EAL 6: Semi-formally Verified Design and Tested
├─ Structured representation of implementation
├─ Modular design analysis
└─ PQC: Detailed side-channel resistance

EAL 7: Formally Verified Design and Tested
├─ Formal representation of functional specification
├─ Mathematical proof of security properties
└─ PQC: Formal verification of algorithm properties
```

```python
from dataclasses import dataclass, field
from typing import List, Dict, Optional
from enum import Enum

class EvaluationLevel(Enum):
    EAL1 = 1
    EAL2 = 2
    EAL3 = 3
    EAL4 = 4
    EAL5 = 5
    EAL6 = 6
    EAL7 = 7

@dataclass
class SecurityRequirement:
    """Common Criteria security requirement."""
    id: str  # e.g., FCS_COP.1
    name: str
    description: str
    pqc_considerations: List[str] = field(default_factory=list)

@dataclass
class ProtectionProfile:
    """Common Criteria Protection Profile."""
    name: str
    version: str
    issuer: str
    target_product: str
    eal: EvaluationLevel
    sfrs: List[SecurityRequirement] = field(default_factory=list)
    pqc_support: str = "Pending"  # Pending, Partial, Full

class CCCryptoRequirements:
    """
    Common Criteria cryptographic requirements for PQC.
    """

    def __init__(self):
        self.sfrs = self._define_crypto_sfrs()

    def _define_crypto_sfrs(self) -> Dict[str, SecurityRequirement]:
        """Define cryptographic SFRs relevant to PQC."""
        return {
            "FCS_COP.1": SecurityRequirement(
                id="FCS_COP.1",
                name="Cryptographic Operation",
                description="Perform cryptographic operations in accordance with specified algorithms and key sizes",
                pqc_considerations=[
                    "Must specify ML-KEM, ML-DSA, or SLH-DSA as algorithm",
                    "Key sizes: ML-KEM-768/1024, ML-DSA-65/87",
                    "Reference FIPS 203/204/205 for algorithm specification"
                ]
            ),
            "FCS_CKM.1": SecurityRequirement(
                id="FCS_CKM.1",
                name="Cryptographic Key Generation",
                description="Generate cryptographic keys in accordance with specified algorithm and key sizes",
                pqc_considerations=[
                    "PQC key generation requires high-quality entropy",
                    "Larger key sizes require more randomness",
                    "Polynomial sampling must be uniform"
                ]
            ),
            "FCS_CKM.2": SecurityRequirement(
                id="FCS_CKM.2",
                name="Cryptographic Key Establishment",
                description="Establish cryptographic keys using specified methods",
                pqc_considerations=[
                    "ML-KEM provides IND-CCA2 secure key encapsulation",
                    "Hybrid modes: specify both classical and PQC components",
                    "Key derivation function specification (HKDF, etc.)"
                ]
            ),
            "FCS_CKM.4": SecurityRequirement(
                id="FCS_CKM.4",
                name="Cryptographic Key Destruction",
                description="Destroy cryptographic keys following specified methods",
                pqc_considerations=[
                    "PQC keys are larger; ensure complete zeroization",
                    "Polynomial coefficients must all be cleared",
                    "Consider memory-safe languages or explicit wiping"
                ]
            ),
            "FPT_TST.1": SecurityRequirement(
                id="FPT_TST.1",
                name="TSF Testing",
                description="Run self-tests to demonstrate correct operation",
                pqc_considerations=[
                    "Include PQC algorithm self-tests",
                    "KAT tests for keygen, encaps/decaps, sign/verify",
                    "Health checks for underlying components"
                ]
            ),
        }

    def generate_pqc_guidance(self) -> str:
        """Generate PQC-specific CC guidance."""
        lines = [
            "COMMON CRITERIA PQC GUIDANCE",
            "=" * 50,
            "",
        ]

        for sfr_id, sfr in self.sfrs.items():
            lines.append(f"{sfr_id}: {sfr.name}")
            lines.append(f"  {sfr.description}")
            lines.append("  PQC Considerations:")
            for consideration in sfr.pqc_considerations:
                lines.append(f"    • {consideration}")
            lines.append("")

        return "\n".join(lines)

crypto_reqs = CCCryptoRequirements()
print(crypto_reqs.generate_pqc_guidance())
```

```python
from dataclasses import dataclass, field
from typing import List, Dict, Optional
from datetime import date

@dataclass
class EvaluationEvidence:
    """Evidence required for CC evaluation."""
    name: str
    description: str
    format: str
    pqc_specific: bool = False

@dataclass
class CCEvaluationPackage:
    """Package of evidence for CC evaluation."""
    target_pp: str
    target_eal: str
    evidence: List[EvaluationEvidence] = field(default_factory=list)

class CCEvaluationPrep:
    """
    Prepare for Common Criteria evaluation with PQC.
    """

    def __init__(self, product_name: str):
        self.product = product_name
        self.packages: List[CCEvaluationPackage] = []

    def define_standard_evidence(self) -> List[EvaluationEvidence]:
        """Define standard CC evidence with PQC additions."""
        return [
            # Standard CC evidence
            EvaluationEvidence(
                "Security Target",
                "Document describing security claims and TOE",
                "Document"
            ),
            EvaluationEvidence(
                "Functional Specification",
                "External interfaces and security functions",
                "Document"
            ),
            EvaluationEvidence(
                "Design Documentation",
                "Architecture and subsystem design",
                "Document"
            ),
            EvaluationEvidence(
                "Test Documentation",
                "Test coverage and results",
                "Document + Test Logs"
            ),
            EvaluationEvidence(
                "Guidance Documentation",
                "User and admin guides",
                "Document"
            ),

            # PQC-specific evidence
            EvaluationEvidence(
                "PQC Algorithm Implementation Evidence",
                "Proof of correct implementation per FIPS 203/204/205",
                "Document + CAVP Certificates",
                pqc_specific=True
            ),
            EvaluationEvidence(
                "Constant-Time Implementation Analysis",
                "Evidence that crypto operations are constant-time",
                "Analysis Report + Tool Output",
                pqc_specific=True
            ),
            EvaluationEvidence(
                "PQC Key Management Procedures",
                "Handling of larger PQC keys throughout lifecycle",
                "Document",
                pqc_specific=True
            ),
            EvaluationEvidence(
                "Hybrid Mode Documentation",
                "How classical and PQC algorithms are combined",
                "Document",
                pqc_specific=True
            ),
            EvaluationEvidence(
                "Side-Channel Countermeasures",
                "Documentation of side-channel protections",
                "Analysis Report",
                pqc_specific=True
            ),
        ]

    def generate_checklist(self) -> str:
        """Generate evaluation preparation checklist."""
        evidence = self.define_standard_evidence()

        lines = [
            f"CC EVALUATION CHECKLIST: {self.product}",
            "=" * 60,
            "",
            "STANDARD EVIDENCE:",
        ]

        for e in [x for x in evidence if not x.pqc_specific]:
            lines.append(f"  □ {e.name}")
            lines.append(f"      {e.description}")
            lines.append(f"      Format: {e.format}")
            lines.append("")

        lines.append("PQC-SPECIFIC EVIDENCE:")
        for e in [x for x in evidence if x.pqc_specific]:
            lines.append(f"  □ {e.name}")
            lines.append(f"      {e.description}")
            lines.append(f"      Format: {e.format}")
            lines.append("")

        return "\n".join(lines)

prep = CCEvaluationPrep("PQC Security Module")
print(prep.generate_checklist())
```

## Summary

Common Criteria evaluation for PQC products requires:

1. **Protection Profiles**: Identify applicable PPs; many are being updated for PQC
2. **SFRs**: Cryptographic requirements must reference NIST PQC standards
3. **Evidence**: Additional documentation for PQC-specific considerations
4. **Coordination**: Align with FIPS validation for shared evidence

International recognition through CC facilitates global market access for PQC products.
