# FIPS 140-3 Validation for PQC

FIPS 140-3 validation is mandatory for cryptographic modules used in federal systems. Understanding the validation process for PQC algorithms is essential for compliance.

## FIPS 140-3 Overview

```
FIPS 140-3 Security Levels:
═══════════════════════════════════════════════════════════════════

Level 1: Basic Security
┌─────────────────────────────────────────────────────────────────┐
│ • Production-grade components                                   │
│ • Software-only modules allowed                                 │
│ • No physical security requirements                             │
│ • Self-tests for cryptographic algorithms                       │
└─────────────────────────────────────────────────────────────────┘

Level 2: Tamper Evidence
┌─────────────────────────────────────────────────────────────────┐
│ • Tamper-evident coatings or seals                              │
│ • Role-based authentication                                     │
│ • OS requirements (CC EAL2 or equivalent)                       │
│ • Physical security mechanisms                                  │
└─────────────────────────────────────────────────────────────────┘

Level 3: Tamper Resistance
┌─────────────────────────────────────────────────────────────────┐
│ • Tamper-resistant enclosures                                   │
│ • Identity-based authentication                                 │
│ • Physical/logical isolation                                    │
│ • Environmental failure protection                              │
└─────────────────────────────────────────────────────────────────┘

Level 4: Maximum Security
┌─────────────────────────────────────────────────────────────────┐
│ • Complete physical protection envelope                         │
│ • Zeroization on tamper detection                               │
│ • Environmental failure testing                                 │
│ • Multi-factor authentication                                   │
└─────────────────────────────────────────────────────────────────┘
```

## CMVP Validation Process

```
Cryptographic Module Validation Program (CMVP) Process:
═══════════════════════════════════════════════════════════════════

Phase 1: Preparation (Vendor)
┌────────────────────┐
│ 1. Design module   │
│ 2. Implement algos │──▶ Typically 3-12 months
│ 3. Prepare docs    │
│ 4. Select lab      │
└────────────────────┘
         │
         ▼
Phase 2: Testing (Accredited Lab)
┌────────────────────┐
│ 1. Review docs     │
│ 2. Test algorithms │──▶ 3-6 months
│ 3. Test module     │    (CAVP + module testing)
│ 4. Write report    │
└────────────────────┘
         │
         ▼
Phase 3: Review (NIST/CSE)
┌────────────────────┐
│ 1. Review package  │
│ 2. Clarifications  │──▶ 6-24 months
│ 3. Final approval  │    (current backlog)
│ 4. Issue cert      │
└────────────────────┘
         │
         ▼
┌────────────────────┐
│   VALIDATED        │
│   Certificate      │
│   5-year validity  │
└────────────────────┘

Total Timeline: 12-36+ months (depending on complexity and backlog)
```

```python
from dataclasses import dataclass, field
from datetime import date, timedelta
from typing import List, Optional, Dict
from enum import Enum

class ValidationPhase(Enum):
    DESIGN = "Design & Development"
    LAB_SELECTION = "Lab Selection"
    CAVP_TESTING = "CAVP Algorithm Testing"
    MODULE_TESTING = "Module Testing"
    REPORT_WRITING = "Report Writing"
    CMVP_QUEUE = "CMVP Review Queue"
    CMVP_REVIEW = "CMVP Review"
    CLARIFICATIONS = "Addressing Clarifications"
    VALIDATION = "Validated"

@dataclass
class ValidationMilestone:
    """Milestone in the validation process."""
    phase: ValidationPhase
    description: str
    start_date: Optional[date]
    estimated_duration_days: int
    actual_completion: Optional[date] = None
    notes: str = ""

@dataclass
class CMVPTimeline:
    """Track CMVP validation timeline."""
    module_name: str
    vendor: str
    security_level: int
    milestones: List[ValidationMilestone] = field(default_factory=list)

    def add_milestone(self, milestone: ValidationMilestone):
        self.milestones.append(milestone)

    def estimate_completion(self) -> date:
        """Estimate validation completion date."""
        total_days = sum(m.estimated_duration_days for m in self.milestones)
        start = self.milestones[0].start_date or date.today()
        return start + timedelta(days=total_days)

    def get_current_phase(self) -> Optional[ValidationPhase]:
        """Get current validation phase."""
        for m in self.milestones:
            if m.actual_completion is None:
                return m.phase
        return ValidationPhase.VALIDATION

    def generate_gantt_text(self) -> str:
        """Generate text-based Gantt chart."""
        lines = [
            f"CMVP TIMELINE: {self.module_name}",
            "=" * 60,
        ]

        current_date = self.milestones[0].start_date or date.today()

        for m in self.milestones:
            weeks = m.estimated_duration_days // 7
            bar = "█" * min(weeks, 20)
            status = "✓" if m.actual_completion else "○"
            lines.append(
                f"{status} {m.phase.value[:25]:25} │{bar:20}│ {weeks}w"
            )

        lines.append("")
        lines.append(f"Estimated Completion: {self.estimate_completion()}")

        return "\n".join(lines)

# Example timeline for PQC module
timeline = CMVPTimeline(
    module_name="PQC Crypto Module v1.0",
    vendor="Example Corp",
    security_level=2
)

# Add typical milestones
timeline.add_milestone(ValidationMilestone(
    ValidationPhase.DESIGN,
    "Module design and implementation",
    date(2024, 1, 1),
    180  # 6 months
))

timeline.add_milestone(ValidationMilestone(
    ValidationPhase.LAB_SELECTION,
    "Select and engage accredited lab",
    None,
    30  # 1 month
))

timeline.add_milestone(ValidationMilestone(
    ValidationPhase.CAVP_TESTING,
    "CAVP algorithm validation for PQC",
    None,
    60  # 2 months
))

timeline.add_milestone(ValidationMilestone(
    ValidationPhase.MODULE_TESTING,
    "Full module testing by lab",
    None,
    90  # 3 months
))

timeline.add_milestone(ValidationMilestone(
    ValidationPhase.REPORT_WRITING,
    "Lab prepares validation report",
    None,
    45  # 1.5 months
))

timeline.add_milestone(ValidationMilestone(
    ValidationPhase.CMVP_QUEUE,
    "Wait in CMVP review queue",
    None,
    180  # 6 months (conservative)
))

timeline.add_milestone(ValidationMilestone(
    ValidationPhase.CMVP_REVIEW,
    "NIST/CSE review",
    None,
    90  # 3 months
))

timeline.add_milestone(ValidationMilestone(
    ValidationPhase.CLARIFICATIONS,
    "Address review comments",
    None,
    60  # 2 months
))

print(timeline.generate_gantt_text())
```

```python
from dataclasses import dataclass
from typing import List, Dict, Optional
from enum import Enum

class CAVPStatus(Enum):
    NOT_SUBMITTED = "Not Submitted"
    SUBMITTED = "Submitted"
    TESTING = "In Testing"
    PASSED = "Passed"
    FAILED = "Failed"

@dataclass
class CAVPTest:
    """CAVP algorithm test result."""
    algorithm: str
    parameter_set: str
    test_type: str
    status: CAVPStatus
    certificate_number: Optional[str] = None
    notes: str = ""

class CAVPTracker:
    """Track CAVP algorithm validation status."""

    def __init__(self):
        self.tests: List[CAVPTest] = []

    def add_test(self, test: CAVPTest):
        self.tests.append(test)

    def get_by_algorithm(self, algorithm: str) -> List[CAVPTest]:
        return [t for t in self.tests if algorithm in t.algorithm]

    def is_ready_for_cmvp(self) -> tuple:
        """Check if all algorithms have passed CAVP."""
        passed = [t for t in self.tests if t.status == CAVPStatus.PASSED]
        not_passed = [t for t in self.tests if t.status != CAVPStatus.PASSED]

        return len(not_passed) == 0, not_passed

    def generate_status_report(self) -> str:
        """Generate CAVP status report."""
        lines = [
            "CAVP ALGORITHM VALIDATION STATUS",
            "=" * 50,
        ]

        # Group by algorithm
        by_algo: Dict[str, List[CAVPTest]] = {}
        for t in self.tests:
            by_algo.setdefault(t.algorithm, []).append(t)

        for algo, tests in by_algo.items():
            lines.append(f"\n{algo}:")
            for t in tests:
                status_icon = {
                    CAVPStatus.PASSED: "✓",
                    CAVPStatus.FAILED: "✗",
                    CAVPStatus.TESTING: "○",
                    CAVPStatus.SUBMITTED: "◐",
                    CAVPStatus.NOT_SUBMITTED: "·",
                }[t.status]

                cert = f" (#{t.certificate_number})" if t.certificate_number else ""
                lines.append(
                    f"  {status_icon} {t.parameter_set:15} - "
                    f"{t.test_type:20} {t.status.value}{cert}"
                )

        ready, issues = self.is_ready_for_cmvp()
        lines.append("")
        if ready:
            lines.append("✓ Ready for CMVP submission")
        else:
            lines.append(f"✗ {len(issues)} tests not yet passed")

        return "\n".join(lines)

# Example CAVP tracking
tracker = CAVPTracker()

# ML-KEM tests
tracker.add_test(CAVPTest("ML-KEM", "ML-KEM-768", "KeyGen", CAVPStatus.PASSED, "A1234"))
tracker.add_test(CAVPTest("ML-KEM", "ML-KEM-768", "Encapsulation", CAVPStatus.PASSED, "A1235"))
tracker.add_test(CAVPTest("ML-KEM", "ML-KEM-768", "Decapsulation", CAVPStatus.PASSED, "A1236"))
tracker.add_test(CAVPTest("ML-KEM", "ML-KEM-1024", "KeyGen", CAVPStatus.TESTING))
tracker.add_test(CAVPTest("ML-KEM", "ML-KEM-1024", "Encapsulation", CAVPStatus.SUBMITTED))

# ML-DSA tests
tracker.add_test(CAVPTest("ML-DSA", "ML-DSA-65", "KeyGen", CAVPStatus.PASSED, "A1237"))
tracker.add_test(CAVPTest("ML-DSA", "ML-DSA-65", "Sign", CAVPStatus.PASSED, "A1238"))
tracker.add_test(CAVPTest("ML-DSA", "ML-DSA-65", "Verify", CAVPStatus.PASSED, "A1239"))

print(tracker.generate_status_report())
```

```python
from dataclasses import dataclass, field
from typing import List, Dict, Set

@dataclass
class AlgorithmEntry:
    """Algorithm entry for security policy."""
    name: str
    type: str  # KEM, Signature, Symmetric, Hash
    fips_reference: str
    parameter_sets: List[str]
    cavp_certificates: List[str]
    approved: bool = True

@dataclass
class ServiceDescription:
    """Cryptographic service description."""
    name: str
    description: str
    algorithms_used: List[str]
    roles_allowed: List[str]

@dataclass
class SecurityPolicy:
    """FIPS 140-3 Security Policy document structure."""
    module_name: str
    version: str
    security_level: int
    vendor: str

    algorithms: List[AlgorithmEntry] = field(default_factory=list)
    services: List[ServiceDescription] = field(default_factory=list)
    operating_environments: List[str] = field(default_factory=list)
    physical_security: List[str] = field(default_factory=list)

    def add_algorithm(self, algo: AlgorithmEntry):
        self.algorithms.append(algo)

    def add_service(self, service: ServiceDescription):
        self.services.append(service)

    def generate_summary(self) -> str:
        """Generate security policy summary."""
        lines = [
            f"SECURITY POLICY: {self.module_name}",
            f"Version: {self.version}",
            f"Security Level: {self.security_level}",
            f"Vendor: {self.vendor}",
            "=" * 60,
            "",
            "APPROVED ALGORITHMS:",
        ]

        for algo in self.algorithms:
            if algo.approved:
                lines.append(f"\n  {algo.name} ({algo.type})")
                lines.append(f"    Reference: {algo.fips_reference}")
                lines.append(f"    Parameters: {', '.join(algo.parameter_sets)}")
                lines.append(f"    CAVP: {', '.join(algo.cavp_certificates)}")

        lines.append("\nCRYPTOGRAPHIC SERVICES:")
        for svc in self.services:
            lines.append(f"\n  {svc.name}")
            lines.append(f"    Description: {svc.description}")
            lines.append(f"    Algorithms: {', '.join(svc.algorithms_used)}")
            lines.append(f"    Roles: {', '.join(svc.roles_allowed)}")

        lines.append("\nOPERATING ENVIRONMENTS:")
        for env in self.operating_environments:
            lines.append(f"  • {env}")

        return "\n".join(lines)

# Example security policy

    version="1.0.0",

policy.add_algorithm(AlgorithmEntry(

))

policy.add_algorithm(AlgorithmEntry(
    name="ML-DSA",
    type="Signature",
    fips_reference="FIPS 204",
    parameter_sets=["ML-DSA-65", "ML-DSA-87"],

))

policy.add_algorithm(AlgorithmEntry(
    name="AES",
    type="Symmetric",
    fips_reference="FIPS 197",
    parameter_sets=["AES-256"],
    cavp_certificates=["A1001"]
))

policy.add_service(ServiceDescription(
    name="Key Encapsulation",
    description="Establish shared secrets using ML-KEM",
    algorithms_used=["ML-KEM-768", "ML-KEM-1024"],
```

## PQC-Specific Considerations

```
PQC-Specific FIPS Validation Considerations:
═══════════════════════════════════════════════════════════════════

Larger Key/Signature Sizes:
┌─────────────────────────────────────────────────────────────────┐
│ Challenge: PQC keys/signatures are 10-100x larger              │
│                                                                  │
│ Impact on validation:                                           │
│ • Memory allocation testing more complex                        │
│ • Buffer management critical for security                       │
│ • Performance testing at different sizes                        │
│ • Storage and transmission considerations                       │
└─────────────────────────────────────────────────────────────────┘

New Algorithm Structures:
┌─────────────────────────────────────────────────────────────────┐
│ Challenge: Labs learning new algorithm families                 │
│                                                                  │
│ Impact on validation:                                           │
│ • Extended testing period for novel constructs                  │
│ • NTT implementation verification                               │
│ • Polynomial arithmetic correctness                             │
│ • Sampling procedure validation                                 │
└─────────────────────────────────────────────────────────────────┘

Hybrid Modes:
┌─────────────────────────────────────────────────────────────────┐
│ Challenge: Combining classical + PQC algorithms                 │
│                                                                  │
│ Impact on validation:                                           │
│ • Both algorithms must be CAVP validated                        │
│ • Combination method must be documented                         │
│ • Key derivation (for KEM) properly specified                   │

Side-Channel Resistance:
┌─────────────────────────────────────────────────────────────────┐

```

## Summary

FIPS 140-3 validation for PQC modules requires:

1. **CAVP testing**: Algorithm validation for ML-KEM, ML-DSA, SLH-DSA per new FIPS standards
2. **Module testing**: Full cryptographic module validation at target security level
3. **Documentation**: Security policy, finite state model, algorithm specifications
4. **Timeline planning**: 12-36+ months depending on complexity and CMVP backlog

Early engagement with accredited labs is critical given the learning curve for PQC algorithms.
