## 2. Detailed Category Definitions

```python
def detailed_definitions():
    """Precise definitions of each security category."""
    print("\nDETAILED CATEGORY DEFINITIONS")
    print("=" * 70)

    print("""
    CATEGORY 1: AES-128 EQUIVALENT
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    Definition: Any attack must require computational resources
    comparable to or greater than key recovery on AES-128.

    ┌─────────────────────────────────────────────────────────────────┐
    │ Classical Security:                                            │
    │   • Brute-force AES-128: 2^128 operations                      │
    │   • Attack complexity: ≥ 2^128 classical operations            │
    │                                                                 │
    │ Quantum Security (Grover's Algorithm):                         │
    │   • Grover reduces search to √N = 2^64 quantum operations      │
    │   • But quantum operations are expensive                       │
    │   • Effective security remains very high                       │
    │                                                                 │
    │ Practical Meaning:                                             │
    │   • Secure against all known classical attacks                 │
    │   • Secure against quantum computers for foreseeable future    │
    │   • Suitable for most commercial applications                  │
    └─────────────────────────────────────────────────────────────────┘


    CATEGORY 2: SHA-256 COLLISION EQUIVALENT
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    Definition: Finding collisions in the scheme must be at least
    as hard as finding collisions in SHA-256.

    ┌─────────────────────────────────────────────────────────────────┐
    │ Classical Security:                                            │
    │   • Birthday attack on SHA-256: 2^128 operations               │
    │   • Attack complexity: ≥ 2^128 classical operations            │
    │                                                                 │
    │ Quantum Security:                                              │
    │   • Quantum collision finding: ~2^85 quantum operations        │
    │   • Using BHT algorithm                                        │
    │                                                                 │
    │ Practical Meaning:                                             │
    │   • Primarily for signature schemes                            │
    │   • Similar to Category 1 in practice                          │
    └─────────────────────────────────────────────────────────────────┘


    CATEGORY 3: AES-192 EQUIVALENT
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    Definition: Attacks must require resources comparable to
    key recovery on AES-192.

    ┌─────────────────────────────────────────────────────────────────┐
    │ Classical Security:                                            │
    │   • Brute-force AES-192: 2^192 operations                      │
    │   • Significant margin over Category 1                         │
    │                                                                 │
    │ Quantum Security:                                              │
    │   • Grover on AES-192: 2^96 quantum operations                 │
    │                                                                 │
    │ Practical Meaning:                                             │
    │   • Recommended for most government applications               │
    │   • Good balance of security and performance                   │
    │   • Suitable for long-term protection (10-30 years)            │
    └─────────────────────────────────────────────────────────────────┘


    CATEGORY 4: SHA-384 COLLISION EQUIVALENT
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    Definition: Collision resistance equivalent to SHA-384.

    ┌─────────────────────────────────────────────────────────────────┐
    │ Classical Security:                                            │
    │   • Birthday attack on SHA-384: 2^192 operations               │
    │                                                                 │
    │ Quantum Security:                                              │
    │   • Quantum collision: ~2^128 quantum operations               │
    │                                                                 │
    │ Practical Meaning:                                             │
    │   • High assurance signature applications                      │
    │   • Between Category 3 and 5                                   │
    └─────────────────────────────────────────────────────────────────┘


    CATEGORY 5: AES-256 EQUIVALENT
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    Definition: Attacks must require resources comparable to
    key recovery on AES-256. THE HIGHEST SECURITY LEVEL.

    ┌─────────────────────────────────────────────────────────────────┐
    │ Classical Security:                                            │
    │   • Brute-force AES-256: 2^256 operations                      │
    │   • Beyond any conceivable classical attack                    │
    │                                                                 │
    │ Quantum Security:                                              │
    │   • Grover on AES-256: 2^128 quantum operations                │
    │   • Still extremely secure                                     │
    │                                                                 │
    │ Practical Meaning:                                             │
    │   • Maximum available security                                 │
    │   • Required for TOP SECRET classified data                    │
    │   • Suitable for 50+ year protection                           │
    │   • Largest keys and signatures                                │
    └─────────────────────────────────────────────────────────────────┘
    """)
```

## Summary

NIST's five security categories provide a framework for selecting appropriate post-quantum cryptographic parameters:

1. **Category 1**: AES-128 equivalent - suitable for most consumer applications
2. **Category 2**: SHA-256 collision equivalent - for signature schemes
3. **Category 3**: AES-192 equivalent - recommended for enterprise/government
4. **Category 4**: SHA-384 collision equivalent - high-assurance signatures
5. **Category 5**: AES-256 equivalent - maximum security for classified data

## Key Takeaways

- Security categories enable appropriate security/performance trade-offs
- Category 1 provides sufficient quantum resistance for most applications
- Category 3 is the recommended default for enterprise deployments
- Category 5 should be reserved for highest-sensitivity applications
- Consider data lifetime, not just current sensitivity, when choosing
