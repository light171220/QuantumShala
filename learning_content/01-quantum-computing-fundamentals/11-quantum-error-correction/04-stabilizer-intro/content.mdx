# Introduction to Stabilizer Codes

## Introduction

The stabilizer formalism provides a unified framework for describing quantum error correction codes. It uses the Pauli group to elegantly define code spaces, syndromes, and logical operations.

## The Stabilizer Concept

```
What is a Stabilizer?
═════════════════════

A stabilizer S is a Pauli operator such that:
    S|ψ⟩ = +|ψ⟩

The state |ψ⟩ is an eigenstate of S with eigenvalue +1.

EXAMPLE: |00⟩
    Z₁Z₂|00⟩ = Z|0⟩ ⊗ Z|0⟩ = |0⟩ ⊗ |0⟩ = |00⟩

    So Z₁Z₂ stabilizes |00⟩


STABILIZER GROUP:
    Multiple commuting Paulis can simultaneously stabilize a state

    For the bit flip code |0_L⟩ = |000⟩:
    - Z₁Z₂ stabilizes (checks if q1, q2 same)
    - Z₂Z₃ stabilizes (checks if q2, q3 same)

    The stabilizer group S = ⟨Z₁Z₂, Z₂Z₃⟩


CODE SPACE:
    The code space is the +1 eigenspace of ALL stabilizers:

    Code = {|ψ⟩ : S|ψ⟩ = |ψ⟩ for all S in stabilizer group}

    ┌─────────────────────────────────────────────────┐
    │  Stabilizers: Z₁Z₂, Z₂Z₃                       │
    │                                                 │
    │  Code space: span{|000⟩, |111⟩}                │
    │                                                 │
    │  Logical states:                               │
    │    |0_L⟩ = |000⟩                               │
    │    |1_L⟩ = |111⟩                               │
    └─────────────────────────────────────────────────┘
```

## Syndromes from Stabilizers

```python
def syndrome_from_stabilizers():
    """
    How stabilizers give syndrome information.
    """
    print("""
Syndrome Measurement via Stabilizers
════════════════════════════════════

KEY INSIGHT:
    Errors anticommute with some stabilizers!
    Measuring stabilizer eigenvalue reveals error info

EXAMPLE: Bit Flip Code

    Stabilizers: S₁ = Z₁Z₂, S₂ = Z₂Z₃
    Error: X₂ (bit flip on qubit 2)

    Check commutation:
    S₁X₂ = Z₁Z₂X₂ = Z₁X₂Z₂ = -Z₁Z₂X₂ = -X₂S₁

    X₂ ANTICOMMUTES with S₁!

    Similarly: X₂ anticommutes with S₂


SYNDROME TABLE:
    ┌────────────────────────────────────────────┐
    │  Error  │ S₁=Z₁Z₂ │ S₂=Z₂Z₃ │ Syndrome    │
    ├────────────────────────────────────────────┤
    │  I      │   +1    │   +1    │  (0, 0)     │
    │  X₁     │   -1    │   +1    │  (1, 0)     │
    │  X₂     │   -1    │   -1    │  (1, 1)     │
    │  X₃     │   +1    │   -1    │  (0, 1)     │
    └────────────────────────────────────────────┘

    Unique syndromes → can identify error location!


MEASUREMENT CIRCUIT:
    |0⟩ ───[H]───●───────●───[H]───[M]──→ syndrome bit
                 │       │
    |ψ⟩ ─────────Z───────┼──────────────
                         │
    |ψ⟩ ─────────────────Z──────────────

    Measures Z₁Z₂ without disturbing encoded state
    """)

syndrome_from_stabilizers()
```

## Stabilizer Code Parameters

```
[[n, k, d]] Stabilizer Codes
════════════════════════════

PARAMETERS:
    n = number of physical qubits
    k = number of logical qubits encoded
    d = code distance

RELATIONSHIP:
    n qubits → 2ⁿ dimensional Hilbert space
    n-k independent stabilizer generators
    2ᵏ dimensional code space (encodes k logical qubits)


IMPORTANT CODES:

    ┌──────────────────────────────────────────────────────┐
    │  Code          │  [[n,k,d]] │  Stabilizers          │
    ├──────────────────────────────────────────────────────┤
    │  Bit flip      │  [[3,1,3]] │  Z₁Z₂, Z₂Z₃           │
    │  Phase flip    │  [[3,1,3]] │  X₁X₂, X₂X₃           │
    │  Shor          │  [[9,1,3]] │  8 stabilizers        │
    │  Steane        │  [[7,1,3]] │  6 stabilizers        │
    │  5-qubit       │  [[5,1,3]] │  4 stabilizers        │
    │  Surface (d=3) │  [[9,1,3]] │  8 stabilizers        │
    └──────────────────────────────────────────────────────┘


5-QUBIT PERFECT CODE:
    Smallest code correcting any single error

    Stabilizers:
    S₁ = X₁Z₂Z₃X₄I₅
    S₂ = I₁X₂Z₃Z₄X₅
    S₃ = X₁I₂X₃Z₄Z₅
    S₄ = Z₁X₂I₃X₄Z₅

    Achieves quantum Hamming bound!
```

## Logical Operators

```
Logical Operations on Stabilizer Codes
══════════════════════════════════════

LOGICAL OPERATORS:
    Pauli operators that:
    1. Commute with all stabilizers
    2. Are NOT in the stabilizer group
    3. Act non-trivially on the code space

EXAMPLE: Bit Flip Code [[3,1,3]]

    Stabilizers: Z₁Z₂, Z₂Z₃

    Logical X: X_L = X₁X₂X₃
    - Commutes with Z₁Z₂: [X₁X₂X₃, Z₁Z₂] = 0 ✓
    - Not a stabilizer ✓
    - X_L|000⟩ = |111⟩ (flips logical bit)

    Logical Z: Z_L = Z₁ (or Z₂, or Z₃)
    - Commutes with Z₁Z₂: [Z₁, Z₁Z₂] = 0 ✓
    - Not a stabilizer ✓
    - Z_L|0_L⟩ = |0_L⟩, Z_L|1_L⟩ = -|1_L⟩


CODE DISTANCE:
    d = minimum weight of logical operator

    Bit flip code: weight(X_L) = 3, so d = 3
    This means single errors don't cause logical errors


SUMMARY:
    ┌─────────────────────────────────────────────────┐
    │  Operator Type     │  Properties              │
    ├─────────────────────────────────────────────────┤
    │  Stabilizers       │  In code, eigenvalue +1  │
    │  Logical operators │  Commute, not in group   │
    │  Errors            │  Anticommute with some S │
    │  Correctable errors│  Weight < d             │
    └─────────────────────────────────────────────────┘
```

## Summary

The stabilizer formalism provides:

1. **Code space definition**: +1 eigenspace of stabilizer group
2. **Syndrome extraction**: Errors anticommute with stabilizers
3. **Logical operations**: Operators commuting with but not in stabilizer group
4. **Code distance**: Minimum weight of logical operators

This framework underlies modern QEC including surface codes and color codes.
