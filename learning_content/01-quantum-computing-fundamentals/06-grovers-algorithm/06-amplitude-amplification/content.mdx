## 1. From Grover's to Amplitude Amplification

Grover's algorithm finds a marked item in a database of N items. But what if we generalize this? What if instead of searching, we want to boost the success probability of **any** quantum algorithm?

<Definition term="Amplitude Amplification">
Given any quantum algorithm A that produces a "good" state with probability p, amplitude amplification can boost this probability to near certainty using O(1/√p) applications of A and its inverse.
</Definition>

This is remarkably powerful: we can take an algorithm that succeeds only 1% of the time and boost it to near-certain success with just ~8 iterations!

```python
import numpy as np

def optimal_iterations(success_prob):
    """
    Calculate optimal number of amplitude amplification iterations.
    
    For initial success probability p, we need approximately:
    k = floor(π / (4 * arcsin(√p)))
    """
    if success_prob <= 0 or success_prob >= 1:
        raise ValueError("Success probability must be in (0, 1)")
    
    theta = np.arcsin(np.sqrt(success_prob))
    k = int(np.floor(np.pi / (4 * theta)))
    return k

print("Amplitude Amplification: Iterations Needed")
print("=" * 50)
print(f"{'Initial p':>12} {'Iterations':>12} {'Final p (approx)':>18}")
print("-" * 50)

for p in [0.5, 0.25, 0.1, 0.04, 0.01, 0.001]:
    k = optimal_iterations(p)
    # Final probability after k iterations
    theta = np.arcsin(np.sqrt(p))
    final_p = np.sin((2*k + 1) * theta)**2
    print(f"{p:>12.3f} {k:>12} {final_p:>18.4f}")
```

## 2. The Mathematical Framework

Let's define the framework precisely. Suppose we have:

- **Algorithm A**: Prepares state |ψ⟩ = sin(θ)|good⟩ + cos(θ)|bad⟩
- **Initial success probability**: p = sin²(θ)

The amplitude amplification operator is:

<Formula>
$$Q = -AS_0A^{-1}S_\chi$$
</Formula>

where:
- S_χ marks "good" states (phase flip)
- S_0 reflects about |0⟩ (equivalent to diffusion)
- A is the original algorithm

```python
import numpy as np

def amplitude_amplification_demo(p_initial, show_evolution=True):
    """
    Demonstrate amplitude amplification for a given initial probability.
    """
    # Initial angle
    theta = np.arcsin(np.sqrt(p_initial))
    
    # Optimal iterations
    k_opt = int(np.floor(np.pi / (4 * theta)))
    
    if show_evolution:
        print(f"\nAmplitude Amplification for p = {p_initial}")
        print(f"Initial angle θ = {np.degrees(theta):.2f}°")
        print(f"Optimal iterations: {k_opt}")
        print()
        print(f"{'Iteration':>10} {'Angle':>15} {'P(good)':>12}")
        print("-" * 40)
    
    probabilities = []
    for k in range(k_opt + 3):  # Show a few extra
        current_angle = (2*k + 1) * theta
        prob = np.sin(current_angle)**2
        probabilities.append(prob)
        
        if show_evolution:
            marker = " ← optimal" if k == k_opt else ""
            print(f"{k:>10} {np.degrees(current_angle):>12.2f}° {prob:>12.4f}{marker}")
    
    return probabilities

# Demo with 10% initial success rate
probs = amplitude_amplification_demo(0.10)
```

## 3. Geometric Interpretation

Just like Grover's algorithm, amplitude amplification has a beautiful geometric interpretation:

1. **State space**: Lives in 2D plane spanned by |good⟩ and |bad⟩
2. **Initial state**: Angle θ from |bad⟩ axis (where sin²θ = p)
3. **Each iteration**: Rotates by 2θ toward |good⟩
4. **Target**: Reach angle close to 90° (pure |good⟩ state)

```python
import numpy as np

def geometric_analysis(p):
    """
    Analyze the geometry of amplitude amplification.
    """
    theta = np.arcsin(np.sqrt(p))
    rotation_per_iteration = 2 * theta
    
    # How many rotations to reach ~90 degrees?
    target_angle = np.pi / 2  # 90 degrees
    initial_angle = theta
    
    k_needed = (target_angle - initial_angle) / rotation_per_iteration
    
    print(f"Geometric Analysis for p = {p}")
    print(f"  Initial angle from |bad⟩: {np.degrees(theta):.2f}°")
    print(f"  Rotation per iteration: {np.degrees(rotation_per_iteration):.2f}°")
    print(f"  Iterations to reach 90°: ~{k_needed:.1f}")
    print(f"  Optimal k (floor): {int(np.floor(k_needed))}")

for p in [0.25, 0.10, 0.01]:
    geometric_analysis(p)
    print()
```

## 4. Connection to Grover's Algorithm

Grover's algorithm is a special case of amplitude amplification:

| Aspect | Grover's | Amplitude Amplification |
|--------|----------|------------------------|
| Initial state | Uniform superposition | Any state from algorithm A |
| Initial p | 1/N (one marked item) | Any p ∈ (0, 1) |
| Oracle | Marks database items | Marks "good" algorithm outputs |
| Iterations | O(√N) | O(1/√p) |

```python
import numpy as np

def compare_grover_aa():
    """Compare Grover's iterations with amplitude amplification formula."""
    print("Grover's Algorithm as Amplitude Amplification")
    print("=" * 55)
    print(f"{'N':>8} {'p=1/N':>10} {'Grover O(√N)':>14} {'AA O(1/√p)':>14}")
    print("-" * 55)
    
    for N in [4, 16, 64, 256, 1024]:
        p = 1 / N
        grover_k = int(np.round(np.pi/4 * np.sqrt(N)))
        
        theta = np.arcsin(np.sqrt(p))
        aa_k = int(np.floor(np.pi / (4 * theta)))
        
        print(f"{N:>8} {p:>10.4f} {grover_k:>14} {aa_k:>14}")

compare_grover_aa()
```

## 5. Applications Beyond Search

### 5.1 Quantum Counting

Use amplitude amplification's sensitivity to p to estimate the number of solutions:

```python
import numpy as np

def quantum_counting_idea():
    """
    Quantum counting uses phase estimation on the Grover operator
    to estimate the number of solutions k.
    
    If k solutions exist in N items:
    - sin²(θ) = k/N
    - Phase estimation gives θ
    - We can recover k = N·sin²(θ)
    """
    print("Quantum Counting")
    print("-" * 40)
    print("Given: N items, unknown k solutions")
    print("Goal: Estimate k")
    print()
    print("Method:")
    print("1. Grover operator G has eigenvalues e^(±2iθ)")
    print("2. Phase estimation gives θ")
    print("3. Compute k = N·sin²(θ)")
    print()
    
    # Example
    N = 1000
    k_true = 42
    theta = np.arcsin(np.sqrt(k_true / N))
    k_estimated = N * np.sin(theta)**2
    
    print(f"Example: N={N}, true k={k_true}")
    print(f"θ = {np.degrees(theta):.2f}°")
    print(f"Estimated k = {k_estimated:.1f}")

quantum_counting_idea()
```

## 6. Practical Considerations

<Warning>
**Over-amplification**: Unlike classical boosting, amplitude amplification can "overshoot" if you do too many iterations. The probability oscillates rather than monotonically increasing!
</Warning>

```python
import numpy as np

def show_overshooting():
    """Demonstrate over-amplification problem."""
    p = 0.1
    theta = np.arcsin(np.sqrt(p))
    k_opt = int(np.floor(np.pi / (4 * theta)))
    
    print("Over-amplification Demonstration (p=0.1)")
    print("-" * 45)
    
    for k in range(k_opt + 5):
        prob = np.sin((2*k + 1) * theta)**2
        bar = "█" * int(prob * 30)
        marker = " ← optimal" if k == k_opt else ""
        print(f"k={k:>2}: {prob:.3f} {bar}{marker}")

show_overshooting()
```

## Summary

In this lesson, you learned:

1. **Amplitude amplification** generalizes Grover's to boost any quantum algorithm's success probability
2. **Optimal iterations**: k ≈ π/(4·arcsin(√p)) for initial probability p
3. **Geometric view**: Each iteration rotates state by 2θ toward |good⟩
4. **Applications**: Quantum counting, minimum finding, boosting probabilistic algorithms
5. **Caution**: Over-amplification causes probability to decrease!

<Tip>
Amplitude amplification is one of the most versatile tools in quantum computing. Whenever you have a quantum algorithm with low success probability, consider whether amplitude amplification can help!
</Tip>

## Key Formulas

<Formula>
$$\text{Iterations: } k = \left\lfloor \frac{\pi}{4\theta} \right\rfloor \text{ where } \sin^2\theta = p$$
</Formula>

<Formula>
$$\text{Final probability: } P_{\text{success}} = \sin^2((2k+1)\theta)$$
</Formula>
