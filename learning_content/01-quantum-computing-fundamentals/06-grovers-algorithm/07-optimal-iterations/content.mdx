## The Optimal Count

Each Grover iteration rotates by angle 2θ where sin(θ) = 1/√N.

<Formula>
$$k_{optimal} = \left\lfloor \frac{\pi}{4\theta} \right\rfloor \approx \frac{\pi}{4}\sqrt{N}$$
</Formula>

```python
import numpy as np

def optimal_iterations(N):
    theta = np.arcsin(1/np.sqrt(N))
    return int(np.round(np.pi / (4 * theta)))

for N in [4, 16, 64, 256, 1024]:
    k = optimal_iterations(N)
    print(f"N={N:>4}: k={k:>3} (theoretical: {np.pi/4*np.sqrt(N):.1f})")
```

## Over-rotation

Running more iterations than optimal decreases probability! The state rotates past the solution.

## Summary

- Optimal: k ≈ (π/4)√N iterations
- Over-rotation decreases success probability
- Must stop at the right time
