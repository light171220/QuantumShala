# Success Probability Analysis

## Introduction

Shor's algorithm is probabilistic—it doesn't always succeed on the first run. Understanding the success probability helps design efficient implementations and determine how many repetitions are needed for reliable factorization.

## Measurement Outcome Distribution

```
Measurement Probabilities After QFT†
════════════════════════════════════

After inverse QFT on the control register, we measure
outcome m with probability concentrated at:

    m ≈ k · (2^2n / r)  for k = 0, 1, 2, ..., r-1

PROBABILITY DISTRIBUTION:

    Pr(m) peaks when m/2^2n ≈ k/r

    │
    │  █                     █
    │  █                     █
P(m)│  █         █           █         █
    │  █         █           █         █
    │  █    █    █    █      █    █    █
    └──┴────┴────┴────┴──────┴────┴────┴───▶ m
       0   64  128  192    0   64  128  192
              (r=4, 2^2n=256)

WIDTH OF PEAKS:
    Each peak has width ≈ 1
    Height ≈ r/2^2n (normalized)
    Total probability in "good" outcomes ≈ 4/π² ≈ 40%
```

## Continued Fractions Method

```python
def continued_fractions_demo():
    """
    Demonstrate continued fractions for period extraction.
    """
    print("""
Continued Fractions Algorithm
═════════════════════════════

Given measurement m and register size 2^2n,
find r from m/2^2n ≈ k/r

CONTINUED FRACTION EXPANSION:
    Any real number x can be written as:
    x = a₀ + 1/(a₁ + 1/(a₂ + 1/(a₃ + ...)))

    Notation: x = [a₀; a₁, a₂, a₃, ...]

CONVERGENTS:
    Truncating gives rational approximations p/q
    These are the "best" approximations with small q

EXAMPLE: m = 64, 2^2n = 256
    m/2^2n = 64/256 = 0.25 = 1/4

    Continued fraction: 0.25 = [0; 4]
    Convergent: 0/1, then 1/4

    Period candidate: r = 4 ✓

EXAMPLE: m = 85, 2^2n = 256
    m/2^2n = 85/256 ≈ 0.332

    Continued fraction: [0; 3, 85/256 - 1/3 remainder...]
                      = [0; 3, 0, 1, 5, ...]

    Convergents: 0/1, 1/3, 1/3, 2/6, ...

    Period candidate: r = 3

    Check: is a³ mod N = 1? If yes, found period!

ALGORITHM:
    1. Compute m/2^2n
    2. Find continued fraction expansion
    3. Check each convergent denominator q
    4. If a^q mod N = 1 and q is even, try factoring
    """)

continued_fractions_demo()
```

## Success Probability Breakdown

```python
def success_probability_analysis():
    """
    Analyze sources of failure and overall success probability.
    """
    print("""
Success Probability Components
══════════════════════════════

STEP 1: Choosing a Good Base 'a'
    Need: gcd(a, N) = 1
    Probability: 1 - O(1/√N) ≈ 1 for large N

    If gcd(a, N) > 1, we found a factor immediately!
    (Very rare but lucky case)

STEP 2: Measuring a Useful m
    "Good" m values: those where m/2^2n ≈ k/r exactly
    Probability of good m: ≥ 4/π² ≈ 40.5%

    More precisely: Σ sin²(πkr/2^2n) / (π²k²/r)
    Summing over k coprime to r

STEP 3: Extracting Period via Continued Fractions
    If m is "good", continued fractions gives r
    with high probability

    Failure case: k and r share common factor
    Probability of gcd(k,r)=1: φ(r)/r

    For r with few small factors: ≈ 30-60%

STEP 4: Period Leads to Factor
    Need: r is even AND a^(r/2) ≢ -1 (mod N)

    Probability: ≥ 1 - 1/2^(k-1)
    where k = number of distinct odd prime factors of N

    For N = pq (two primes): ≥ 1/2

OVERALL SUCCESS (Single Run):
    P(success) ≥ 0.4 × 0.4 × 0.5 = 8%

    Conservative estimate for one iteration

PRACTICAL SUCCESS:
    With optimizations: ~20-40% per run
    After ~10 runs: >99% success
    """)

success_probability_analysis()
```

## Improving Success Rate

```
Strategies to Improve Success
═════════════════════════════

1. MULTIPLE MEASUREMENTS
    Run algorithm k times independently
    P(all fail) = (1 - p)^k

    For p = 0.2 (single success):
    k = 5:  P(success) = 1 - 0.8^5 ≈ 67%
    k = 10: P(success) = 1 - 0.8^10 ≈ 89%
    k = 20: P(success) = 1 - 0.8^20 ≈ 99%

2. TRYING MULTIPLE VALUES OF a
    If one base doesn't work, try another
    Different a may have different period structure

3. CHECKING MULTIPLES OF r
    If we get r' = r/gcd(k,r), try 2r', 3r', ...
    Check if a^(nr') mod N = 1

4. USING PARTIAL INFORMATION
    Combine results from multiple measurements
    LCM of candidate periods often gives true r

5. QUANTUM ERROR MITIGATION
    Better hardware → more accurate peaks
    Larger control register → narrower peaks


SUCCESS AFTER k RUNS:

    P(success by run k) = 1 - (1 - p)^k

    ┌──────────────────────────────────────┐
    │  Runs │  p=0.1  │  p=0.2  │  p=0.4  │
    ├──────────────────────────────────────┤
    │   1   │  10%    │  20%    │  40%    │
    │   5   │  41%    │  67%    │  92%    │
    │  10   │  65%    │  89%    │  99.4%  │

    │  50   │  99.5%  │  99.99% │  ~100%  │
    └──────────────────────────────────────┘
```

## Mathematical Details

```python
def mathematical_probability():
    """
    Detailed probability calculation.
    """
    print("""
Detailed Probability Calculation
════════════════════════════════

STATE BEFORE MEASUREMENT:
    |ψ⟩ = (1/√r) Σⱼ e^(2πijk/r) |j·(2^2n/r)⟩

    where j runs over r terms (one for each residue)

PROBABILITY OF MEASURING m:
    Pr(m) = |⟨m|ψ⟩|²

    For m near k·(2^2n/r):

    Pr(m) ≈ (1/r) |Σⱼ e^(2πij(k-mr/2^2n))|²

PEAK STRUCTURE:
    Maximum at m = k·(2^2n/r) (if integer)

    Peak height: Pr(m_peak) ≈ 1/r
    Peak width: Δm ≈ 1

    Total probability per peak: ≈ 1/r
    Number of peaks: r
    Total in peaks: ≈ r × (1/r) = 1 (normalized)

USEFUL PEAKS (k coprime to r):
    φ(r) peaks give useful information
    Fraction useful: φ(r)/r

    For prime r: φ(r)/r = (r-1)/r → 1
    For composite r: φ(r)/r ≥ 1/3 typically

CONTINUED FRACTION SUCCESS:
    For randomly chosen k coprime to r:
    CF convergent gives r with probability ≈ 1

    Overall per measurement: ~40%

PERIOD-TO-FACTOR SUCCESS:
    Given correct period r:
    P(r even) = 1/2 (if random)
    P(a^(r/2) ≢ -1 mod N | r even) ≥ 1/2

    Combined: ≥ 1/4

FINAL TALLY:
    P(good m) × P(CF works) × P(factor found)
    ≈ 0.4 × 0.9 × 0.5 ≈ 18%
    """)

mathematical_probability()
```

## Summary

Success probability in Shor's algorithm:

1. **Measurement distribution**: Peaks at m ≈ k·(2^2n/r) with ~40% in useful outcomes
2. **Continued fractions**: High success when gcd(k,r) = 1
3. **Period to factors**: ~50% success for two-prime composite N
4. **Single run success**: Approximately 10-40% depending on N
5. **Repetition strategy**: O(log(1/ε)) runs for (1-ε) success probability

The probabilistic nature is not a weakness—polynomial repetitions give arbitrarily high confidence.
