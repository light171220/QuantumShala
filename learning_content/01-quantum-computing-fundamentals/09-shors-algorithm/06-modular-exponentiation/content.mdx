# Modular Exponentiation

## Introduction

Modular exponentiation—computing a^x mod N—is the computational core of Shor's algorithm. Implementing this efficiently on a quantum computer requires careful circuit design using repeated squaring and reversible arithmetic.

## Classical Foundation

```python
def classical_mod_exp(a, x, N):
    """
    Compute a^x mod N using repeated squaring.

    Key insight: x = x₀ + 2x₁ + 4x₂ + ... (binary expansion)
    So: a^x = a^(x₀) · a^(2x₁) · a^(4x₂) · ...
            = (a^1)^x₀ · (a^2)^x₁ · (a^4)^x₂ · ...
    """
    print(f"""
Classical Modular Exponentiation
════════════════════════════════

Computing {a}^x mod {N}

REPEATED SQUARING METHOD:
    Instead of x multiplications, use O(log x) squarings

    Binary expansion of x:
    x = Σ xᵢ · 2^i where xᵢ ∈ {{0, 1}}

    Then:
    a^x = Π (a^(2^i))^xᵢ = Π a^(2^i) for each i where xᵢ = 1

EXAMPLE: Compute 7^13 mod 15

    13 in binary: 1101 = 8 + 4 + 1

    Precompute powers of 7 mod 15:
    7^1 mod 15 = 7
    7^2 mod 15 = 49 mod 15 = 4
    7^4 mod 15 = 4² mod 15 = 16 mod 15 = 1
    7^8 mod 15 = 1² mod 15 = 1

    Combine:
    7^13 = 7^8 · 7^4 · 7^1 mod 15
         = 1 · 1 · 7 mod 15
         = 7

COMPLEXITY:
    Multiplications: O(log x) instead of O(x)
    Each multiplication: O(n²) for n-bit numbers
    Total: O(n² log x) = O(n³) for x up to N
    """)

classical_mod_exp(7, 13, 15)
```

## Quantum Modular Multiplication

```
Quantum Modular Multiplication Circuit
══════════════════════════════════════

To compute |y⟩ → |ay mod N⟩, we need reversible circuits.

CHALLENGE:
    Classical multiplication is not reversible
    Must design unitary operation U_a where:
    U_a|y⟩ = |ay mod N⟩

SOLUTION: Controlled addition with shift

    Multiplication by a:
    a · y = a · (y₀ + 2y₁ + 4y₂ + ...)
          = a·y₀ + 2a·y₁ + 4a·y₂ + ...

    Build from controlled additions:

    |y⟩ ─────●─────●─────●─────
             │     │     │
    |0⟩ ───[+a]──[+2a]─[+4a]──── |ay mod N⟩

    Each [+ka] is controlled addition of ka mod N


MODULAR ADDITION CIRCUIT:

    To compute |x⟩|y⟩ → |x⟩|x+y mod N⟩:

    |x⟩ ─────●───────────●───────────
             │           │
    |y⟩ ───[+x]───────[−N if ≥N]──── |x+y mod N⟩
             │           │
    |0⟩ ────────[compare]──────────── ancilla

    Steps:
    1. Add x to y: y → x + y
    2. Compare with N
    3. If x + y ≥ N, subtract N
    4. Uncompute comparison ancilla
```

## Controlled Modular Exponentiation

```python
def controlled_mod_exp_structure():
    """
    Structure of controlled modular exponentiation.
    """
    print("""
Controlled Modular Exponentiation
═════════════════════════════════

For Shor's algorithm, we need CONTROLLED operations:
    |x⟩|1⟩ → |x⟩|a^x mod N⟩

DECOMPOSITION:
    x = x₀ + 2x₁ + 4x₂ + ... + 2^(2n-1)·x_{2n-1}

    a^x = a^(x₀) · a^(2·x₁) · a^(4·x₂) · ...

    Use controlled multiplications:

    |x₀⟩ ──────●───────────────────────────────
               │
    |x₁⟩ ──────┼──────●────────────────────────
               │      │
    |x₂⟩ ──────┼──────┼──────●─────────────────
               │      │      │
        ═══════════════════════════════════════
               │      │      │
    |1⟩ ─────[×a¹]──[×a²]──[×a⁴]───── |a^x mod N⟩


CIRCUIT FOR EACH CONTROLLED MULTIPLICATION:

    Control qubit xᵢ determines whether to multiply by a^(2^i)

    |xᵢ⟩ ──────●──────────────
               │
    |y⟩  ────[×a^(2^i) mod N]──── |y · a^(2^i·xᵢ) mod N⟩

    If xᵢ = 0: y unchanged
    If xᵢ = 1: y → y · a^(2^i) mod N


PRECOMPUTATION:
    Before running the circuit, classically compute:
    a^1 mod N, a^2 mod N, a^4 mod N, ..., a^(2^(2n-1)) mod N

    These constants are "baked into" the circuit
    """)

controlled_mod_exp_structure()
```

## Building Blocks: Reversible Arithmetic

```
Reversible Arithmetic Operations
════════════════════════════════

All operations must be REVERSIBLE (unitary).

1. REVERSIBLE ADDITION (Draper adder using QFT):

    |a⟩ ──[QFT]──[+φ(b)]──[QFT†]── |a+b⟩

    QFT converts to phase representation
    Addition becomes phase rotation
    QFT† converts back

2. REVERSIBLE SUBTRACTION:
    Same as addition but with negative phases

3. REVERSIBLE COMPARISON:

    |a⟩ ─────────●───────────
                 │
    |b⟩ ─────────┼───────────
                 │
    |0⟩ ───[a<b? → 1]──────── |a<b⟩

4. MODULAR REDUCTION:

    |x⟩ ───────●─────────────
               │
    |N⟩ ───────┼─────────────
               │
    |0⟩ ──[compare]──●───────
                     │
    |x⟩ ───────────[−N]────── |x mod N⟩


CONTROLLED MODULAR MULTIPLICATION:

    Overall structure for |y⟩ → |ay mod N⟩ controlled on |c⟩:

    |c⟩ ─────────────────●──────────────────
                         │
    |y⟩ ─────────────────┼──────────────────
                         │
    |0⟩ ───[controlled multiply]─[swap if c]── |ay mod N⟩

    Key: Use ancilla to hold result, then swap back
```

## Complexity Analysis

```python
def complexity_analysis():
    """
    Analyze complexity of modular exponentiation circuit.
    """
    print("""
Modular Exponentiation Complexity
═════════════════════════════════

For n-bit number N:

CONTROLLED MODULAR MULTIPLICATION:
    One multiplication: O(n²) gates
    Why? n controlled additions, each O(n) gates

FULL MODULAR EXPONENTIATION:
    Number of controlled multiplications: 2n
    (one for each bit of x in control register)

    Total gates: 2n × O(n²) = O(n³)

BREAKDOWN:
┌─────────────────────────────────────────────────────┐
│  Operation                    │  Gate Count         │
├─────────────────────────────────────────────────────┤
│  Single controlled addition   │  O(n)               │
│  Single modular multiplication│  O(n²)              │
│  Full modular exponentiation  │  O(n³)              │
└─────────────────────────────────────────────────────┘

QUBIT COUNT:
    Target register:    n qubits (holds y)
    Ancilla for mult:   n qubits (workspace)
    Ancilla for mod:    O(n) qubits (comparison)
    Total workspace:    O(n) qubits

OPTIMIZATIONS:
    1. Windowed exponentiation: reduce multiplications
    2. Parallel additions: reduce depth
    3. Approximate QFT: reduce controlled rotations
    4. Recycling ancillas: reduce space

COMPARISON TO CLASSICAL:
    Classical: O(n³) bit operations
    Quantum: O(n³) quantum gates

    Same scaling, but quantum runs in superposition!
    """)

complexity_analysis()
```

## Practical Implementation Considerations

```
Implementation Strategies
═════════════════════════

STRATEGY 1: SCHOOLBOOK MULTIPLICATION
    Direct implementation of grade-school algorithm
    Simple but not optimal
    O(n²) gates per multiplication

    |a⟩ ──●──●──●──●──
          │  │  │  │
    |b⟩ ──┼──┼──┼──┼──
          │  │  │  │
    |0⟩ ─[+]─[+]─[+]─[+]── |ab⟩


STRATEGY 2: KARATSUBA-BASED
    Divide-and-conquer approach
    O(n^1.58) gates per multiplication
    More complex circuit

STRATEGY 3: QFT-BASED (Draper)
    Convert to phase representation
    Addition becomes phase shift
    Clean and parallelizable

    Advantages:
    ✓ No carry propagation
    ✓ Easily made approximate
    ✓ Good for modular arithmetic

STRATEGY 4: LOOKUP TABLE
    For small N, precompute all products
    Trade qubits for gates
    Only practical for small numbers


REAL-WORLD CONSIDERATIONS:

    Noise sensitivity:
    - Deep circuits accumulate errors
    - Prefer shallow, parallel structures

    Connectivity:
    - Hardware has limited qubit connections
    - May need SWAP operations

    Error correction:
    - Must be fault-tolerant for large N
    - Adds significant overhead
```

## Summary

Modular exponentiation in Shor's algorithm:

1. **Classical foundation**: Repeated squaring reduces O(x) to O(log x) multiplications
2. **Quantum implementation**: Controlled multiplications by powers of a
3. **Reversibility**: All operations must be unitary, requiring ancilla qubits
4. **Complexity**: O(n³) gates for n-bit numbers
5. **Building blocks**: QFT-based adders, modular reduction, controlled operations

This is the most resource-intensive part of Shor's algorithm and a key focus of optimization research.
