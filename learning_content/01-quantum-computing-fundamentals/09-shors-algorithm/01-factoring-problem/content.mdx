## 1. The Factoring Problem

<Definition term="Integer Factorization Problem">
Given a composite integer N, find its prime factors p₁, p₂, ..., pₖ such that:

$$N = p_1^{a_1} \times p_2^{a_2} \times \cdots \times p_k^{a_k}$$
</Definition>

For RSA cryptography, we focus on **semiprimes**: N = p × q where p and q are large primes.

```python
import random

def is_prime(n, k=10):
    """Miller-Rabin primality test."""
    if n < 2:
        return False
    if n == 2 or n == 3:
        return True
    if n % 2 == 0:
        return False
    
    # Write n-1 as 2^r * d
    r, d = 0, n - 1
    while d % 2 == 0:
        r += 1
        d //= 2
    
    # Witness loop
    for _ in range(k):
        a = random.randrange(2, n - 1)
        x = pow(a, d, n)
        
        if x == 1 or x == n - 1:
            continue
        
        for _ in range(r - 1):
            x = pow(x, 2, n)
            if x == n - 1:
                break
        else:
            return False
    return True

def generate_rsa_modulus(bits):
    """Generate an RSA modulus N = p * q."""
    def random_prime(bits):
        while True:
            p = random.getrandbits(bits) | (1 << (bits - 1)) | 1
            if is_prime(p):
                return p
    
    p = random_prime(bits // 2)
    q = random_prime(bits // 2)
    return p * q, p, q

# Example: Generate a small RSA modulus
N, p, q = generate_rsa_modulus(32)
print(f"RSA Modulus Example (32-bit)")
print(f"=" * 40)
print(f"N = {N}")
print(f"p = {p}")
print(f"q = {q}")
print(f"Verification: p × q = {p * q}")
print(f"Match: {N == p * q}")
```

## 2. Why Factoring is Hard

The asymmetry between multiplication and factoring is the foundation of RSA:

```python
import time

def time_operation(func, *args):
    """Time an operation."""
    start = time.perf_counter()
    result = func(*args)
    elapsed = time.perf_counter() - start
    return result, elapsed

def trial_division(n):
    """Naive factoring by trial division."""
    factors = []
    d = 2
    while d * d <= n:
        while n % d == 0:
            factors.append(d)
            n //= d
        d += 1
    if n > 1:
        factors.append(n)
    return factors

# Demonstrate the asymmetry
print("Multiplication vs. Factoring Asymmetry")
print("=" * 50)

for bits in [20, 24, 28, 32]:
    p = random.getrandbits(bits // 2) | (1 << (bits // 2 - 1)) | 1
    q = random.getrandbits(bits // 2) | (1 << (bits // 2 - 1)) | 1
    
    # Time multiplication
    _, mult_time = time_operation(lambda: p * q)
    N = p * q
    
    # Time factoring
    _, factor_time = time_operation(trial_division, N)
    
    ratio = factor_time / mult_time if mult_time > 0 else float('inf')
    print(f"\n{bits}-bit N:")
    print(f"  Multiply: {mult_time*1e6:.2f} μs")
    print(f"  Factor:   {factor_time*1e6:.2f} μs")
    print(f"  Ratio:    {ratio:.0f}x slower")
```

## 3. Computational Complexity

The best known classical algorithms for factoring:

| Algorithm | Time Complexity | Type |
|-----------|-----------------|------|
| Trial Division | O(√N) = O(2^(n/2)) | Exponential |
| Pollard's rho | O(N^(1/4)) = O(2^(n/4)) | Subexponential |
| Quadratic Sieve | O(e^(√(n ln n))) | Subexponential |
| General Number Field Sieve | O(e^(1.9 n^(1/3) (ln n)^(2/3))) | Subexponential |
| **Shor's Algorithm** | **O(n³)** | **Polynomial!** |

```python
import numpy as np

def complexity_comparison(n_bits):
    """Compare factoring algorithm complexities."""
    n = n_bits
    
    # Approximate operation counts (simplified)
    trial_div = 2 ** (n / 2)
    pollard = 2 ** (n / 4)
    quadratic_sieve = np.exp(np.sqrt(n * np.log(n)))
    gnfs = np.exp(1.9 * (n ** (1/3)) * (np.log(n) ** (2/3)))
    shor = n ** 3
    
    return {
        'Trial Division': trial_div,
        'Pollard rho': pollard,
        'Quadratic Sieve': quadratic_sieve,
        'GNFS': gnfs,
        "Shor's (quantum)": shor
    }

print("Factoring Complexity Comparison")
print("=" * 60)
print(f"{'Algorithm':<20} {'512-bit':>15} {'1024-bit':>15} {'2048-bit':>15}")
print("-" * 60)

for alg in ['Trial Division', 'Pollard rho', 'GNFS', "Shor's (quantum)"]:
    vals = []
    for bits in [512, 1024, 2048]:
        c = complexity_comparison(bits)
        val = c[alg]
        if val > 1e50:
            vals.append(f"10^{np.log10(val):.0f}")
        else:
            vals.append(f"{val:.2e}")
    print(f"{alg:<20} {vals[0]:>15} {vals[1]:>15} {vals[2]:>15}")

print("\n⚠️ Shor's algorithm is POLYNOMIAL - exponentially faster!")
```

## 4. The RSA Connection

RSA encryption relies on the hardness of factoring:

```python
def rsa_key_generation_demo():
    """
    Demonstrate RSA key generation.
    Security depends on inability to factor N.
    """
    # For demo, use small primes
    p, q = 61, 53
    N = p * q  # Public modulus
    phi_N = (p - 1) * (q - 1)  # Euler's totient
    
    # Choose public exponent
    e = 17  # Common choice
    
    # Compute private exponent
    # d such that e*d ≡ 1 (mod φ(N))
    d = pow(e, -1, phi_N)
    
    print("RSA Key Generation Demo")
    print("=" * 50)
    print(f"\nSecret primes:")
    print(f"  p = {p}")
    print(f"  q = {q}")
    print(f"\nPublic key:")
    print(f"  N = {N} (modulus)")
    print(f"  e = {e} (exponent)")
    print(f"\nPrivate key:")
    print(f"  d = {d}")
    print(f"\nVerification: e × d mod φ(N) = {(e * d) % phi_N}")
    
    # Encrypt and decrypt a message
    message = 42
    ciphertext = pow(message, e, N)
    decrypted = pow(ciphertext, d, N)
    
    print(f"\nEncryption example:")
    print(f"  Message: {message}")
    print(f"  Encrypted: {ciphertext}")
    print(f"  Decrypted: {decrypted}")
    
    print(f"\n⚠️ If an attacker can factor N = {N}...")
    print(f"   They find p = {p}, q = {q}")
    print(f"   Compute φ(N) = {phi_N}")
    print(f"   Derive d = {d}")
    print(f"   → Complete key recovery!")

rsa_key_generation_demo()
```

## 5. Historical Context

<Tip>
**1994**: Peter Shor discovered his algorithm, showing that a quantum computer could factor integers in polynomial time.

This was revolutionary because:
1. First evidence that quantum computers could solve classically hard problems
2. Threatened the foundation of internet security (RSA, Diffie-Hellman)
3. Launched the field of post-quantum cryptography
4. Motivated decades of research in quantum computing
</Tip>


## Summary

In this lesson, you learned:

- **Factoring Problem**: Given N = p × q, find the prime factors p and q
- **Asymmetry**: Multiplication is easy (polynomial), factoring is hard (believed exponential)
- **RSA Security**: Entirely based on the difficulty of factoring
- **Shor's Breakthrough**: Polynomial-time quantum algorithm (O(n³) vs classical subexponential)
- **Impact**: Threatens all RSA-based cryptography when large quantum computers exist

## Key Insight

<Formula>
$$\text{Classical: } O(e^{c \cdot n^{1/3}}) \quad \text{vs} \quad \text{Quantum: } O(n^3)$$
</Formula>

This is an **exponential speedup** - the holy grail of quantum algorithms!
