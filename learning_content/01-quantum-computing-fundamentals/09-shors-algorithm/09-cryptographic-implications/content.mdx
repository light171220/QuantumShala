# Cryptographic Implications

## Introduction

Shor's algorithm poses an existential threat to widely-used public-key cryptography. This lesson explores which systems are vulnerable, how we're preparing, and the timeline for transitioning to quantum-resistant alternatives.

## Post-Quantum Cryptography

```
Post-Quantum Cryptographic Approaches
═════════════════════════════════════

NIST PQC STANDARDIZATION (2024):

Selected algorithms for standardization:

1. ML-KEM (CRYSTALS-Kyber) - Key Encapsulation
    Basis: Learning with errors (lattices)
    Key size: ~1.5 KB public key
    Performance: Fast, practical
    Status: FIPS 203 standard

2. ML-DSA (CRYSTALS-Dilithium) - Signatures
    Basis: Lattice problems
    Signature: ~2.5 KB
    Performance: Fast signing/verification
    Status: FIPS 204 standard

3. SLH-DSA (SPHINCS+) - Signatures
    Basis: Hash functions only
    Signature: ~8-50 KB
    Performance: Slower, very conservative
    Status: FIPS 205 standard


OTHER APPROACHES:

┌──────────────────────────────────────────────────────────┐
```

## Summary

Shor's algorithm's cryptographic implications:

1. **Vulnerable systems**: RSA, ECC, DH—all current public-key crypto
2. **Symmetric crypto**: Weakened but fixable with larger keys
3. **Post-quantum solutions**: Lattice-based (Kyber, Dilithium), hash-based (SPHINCS+)
4. **HNDL threat**: Long-lived secrets need protection now
5. **Migration**: Hybrid approach during transition, full PQC by 2035+

The quantum threat is real but manageable with proactive preparation.
