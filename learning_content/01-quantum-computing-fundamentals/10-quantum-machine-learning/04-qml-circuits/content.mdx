# Variational Circuit Design

## Introduction

The heart of variational quantum machine learning is the parameterized quantum circuit, also called an ansatz. Designing effective ansätze requires balancing expressibility (ability to represent useful functions) with trainability (ability to optimize parameters).

## Ansatz Architecture

```
Variational Circuit Structure
═════════════════════════════

GENERAL FORM:
    |ψ(θ)⟩ = U(θ)|0⟩^⊗n

    Where U(θ) = ∏_l U_l(θ_l) is a product of layers

TYPICAL LAYER:
    ┌─────────────────────────────────────────────────────────┐
    │                                                         │
    │  |q₀⟩ ─[Ry(θ₁)]─●─────────────[Ry(θ₅)]─               │
    │                 │                                       │
    │  |q₁⟩ ─[Ry(θ₂)]─X─●───────────[Ry(θ₆)]─               │
    │                   │                                     │
    │  |q₂⟩ ─[Ry(θ₃)]───X─●─────────[Ry(θ₇)]─               │
    │                     │                                   │
    │  |q₃⟩ ─[Ry(θ₄)]─────X─────────[Ry(θ₈)]─               │
    │                                                         │
    │  └── Rotations ──┘└─ Entangling ─┘└── Rotations ──┘   │
    │                                                         │
    └─────────────────────────────────────────────────────────┘

COMPONENTS:
    1. ROTATION GATES: Parameterized single-qubit gates (Rx, Ry, Rz)
    2. ENTANGLING GATES: Create qubit correlations (CNOT, CZ)
    3. LAYERS: Repeated structure for depth
```

## Common Ansatz Designs

```python
def common_ansatze():
    """
    Survey common variational circuit architectures.
    """
    print("""
Common Ansatz Architectures
═══════════════════════════

1. HARDWARE EFFICIENT ANSATZ (HEA)
    Designed to match hardware connectivity
    Alternating rotation and entangling layers

    Layer:
    |0⟩ ─[Ry]─[Rz]─●───────[Ry]─[Rz]─
                   │
    |0⟩ ─[Ry]─[Rz]─X─●─────[Ry]─[Rz]─
                     │
    |0⟩ ─[Ry]─[Rz]───X─●───[Ry]─[Rz]─
                       │
    |0⟩ ─[Ry]─[Rz]─────X───[Ry]─[Rz]─

    Pros: Low depth, matches hardware
    Cons: Prone to barren plateaus


2. STRONGLY ENTANGLING LAYERS
    All pairs interact within layer

    |0⟩ ─[Ry]─●─────●─────────[Ry]─
              │     │
    |0⟩ ─[Ry]─X──●──┼──●──────[Ry]─
                 │  │  │
    |0⟩ ─[Ry]────X──┼──┼──●───[Ry]─
                    │  │  │
    |0⟩ ─[Ry]───────X──X──X───[Ry]─

    Pros: High expressibility
    Cons: Deep circuits, noise


3. TREE TENSOR NETWORK (TTN)
    Hierarchical entanglement structure

        ●───●       ●───●
          │           │
          ●─────────●
                │
               OUTPUT

    Pros: Structured, interpretable
    Cons: Limited expressibility


4. QAOA-STYLE (for optimization)
    Problem-specific structure

    |+⟩ ─[e^{-iγC}]─[e^{-iβB}]─ ... ─[measure]

    C = cost Hamiltonian
    B = mixer Hamiltonian

    Pros: Problem-matched
    Cons: Problem-specific design needed
    """)

common_ansatze()
```

## Expressibility and Entangling Capability

```
Circuit Metrics
═══════════════

EXPRESSIBILITY:
    How uniformly can the circuit explore state space?

    High expressibility = can reach many different states
    Measured by how close output distribution is to Haar random

    ┌─────────────────────────────────────────────────────┐
    │  Expressibility Spectrum                           │
    │                                                     │
    │  Low                           High                │
    │   │                             │                  │
    │   ▼                             ▼                  │
    │  Product        Random-ish      Haar-random       │
    │  states         coverage        (maximum)         │
    │                                                     │
    │  Simple         Hardware        Deep random       │
    │  rotations      efficient       circuits          │
    └─────────────────────────────────────────────────────┘


ENTANGLING CAPABILITY:
    How much entanglement can the circuit produce?

    Meyer-Wallach measure: Q(|ψ⟩) ∈ [0, 1]
    Q = 0: Product state (no entanglement)
    Q = 1: Maximally entangled

    ┌─────────────────────────────────────────────────────┐
    │  Circuit              │  Avg Entangling Capability │
    ├─────────────────────────────────────────────────────┤
    │  No CNOT              │  0                         │
    │  Linear CNOT chain    │  ~0.3                      │
    │  All-to-all CNOT      │  ~0.7                      │
    │  Deep random          │  ~0.9                      │
    └─────────────────────────────────────────────────────┘


THE TRADEOFF:
    More expressibility → harder to train (barren plateaus)
    Less expressibility → may not fit the target function

    Find the "Goldilocks zone" for your problem!
```

## Barren Plateaus

```python
def barren_plateaus_explanation():
    """
    Explain the barren plateau phenomenon.
    """
    print("""
Barren Plateaus
═══════════════

PROBLEM:
    In random deep circuits, gradients vanish exponentially
    ∂⟨O⟩/∂θ → 0  as #qubits or depth increases

CONSEQUENCE:
    Optimization landscape becomes flat
    Random initialization gives no gradient signal
    Training fails to find good minima

    ┌─────────────────────────────────────────────────────┐
    │                                                     │
    │  Loss                                              │
    │   │     ╱╲                                         │
    │   │    ╱  ╲     Good landscape                     │
    │   │   ╱    ╲    (has gradient)                     │
    │   │  ╱      ╲                                      │
    │   └──────────────────────────                      │
    │                                                     │
    │  Loss                                              │
    │   │  ────────────────────                          │
    │   │                      Barren plateau            │
    │   │                      (flat, no gradient)       │
    │   │                                                │
    │   └──────────────────────────                      │
    │                                                     │
    └─────────────────────────────────────────────────────┘


CAUSES:
    1. Deep random circuits → too expressible
    2. Global cost functions → information diluted
    3. High entanglement → local structure washed out
    4. Hardware noise → gradients suppressed


MITIGATION STRATEGIES:

    1. SHALLOW CIRCUITS
       Limit depth, reduce expressibility
       Keep gradients meaningful

    2. LOCAL COST FUNCTIONS
       Measure few qubits at a time
       Local gradients don't vanish as fast

    3. STRUCTURED INITIALIZATION
       Start near identity (small rotations)
       Problem-informed initial parameters

    4. LAYERWISE TRAINING
       Train one layer at a time
       Avoid deep random circuits

    5. PROBLEM-TAILORED ANSATZ
       Use problem structure
       Don't use generic random circuits
    """)

barren_plateaus_explanation()
```

## Practical Design Guidelines

```
Ansatz Design Guidelines
════════════════════════

1. MATCH HARDWARE TOPOLOGY
    Use native gate set
    Follow connectivity graph
    Minimize SWAP overhead

    Example (linear chain):
    |0⟩─●───────[R]─
        │
    |0⟩─X─●─────[R]─
          │
    |0⟩───X─●───[R]─
            │
    |0⟩─────X───[R]─


2. START SIMPLE, ADD COMPLEXITY
    Begin with few layers
    Increase depth if underfitting
    Monitor for barren plateaus


3. PARAMETER COUNT
    Rule of thumb: ~10-100 parameters per layer
    Total params ∝ problem complexity

    ┌────────────────────────────────────────┐
    │  Problem Size  │  Suggested Params    │
    ├────────────────────────────────────────┤
    │  Small (n≤5)   │  10-50               │
    │  Medium (n≤10) │  50-200              │
    │  Large (n≤20)  │  100-500             │
    └────────────────────────────────────────┘


4. INITIALIZATION MATTERS
    Near-identity: small random angles
    Structured: based on problem
    Avoid uniform random on [0, 2π]


5. CONSIDER SYMMETRIES
    Preserve problem symmetries in ansatz
    Reduces parameter space
    Often improves training
```

## Summary

Variational circuit design requires:

1. **Architecture choice**: HEA, strongly entangling, or custom
2. **Expressibility balance**: Enough to fit, not so much to cause barren plateaus
3. **Hardware awareness**: Match native gates and connectivity
4. **Training considerations**: Initialization, depth, and cost function design

Good ansatz design is crucial for practical QML success.
