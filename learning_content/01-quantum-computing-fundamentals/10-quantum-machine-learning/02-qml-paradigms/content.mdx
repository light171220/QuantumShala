# QML Paradigms

## Introduction

Quantum machine learning encompasses several distinct approaches, each with different requirements, capabilities, and near-term feasibility. Understanding these paradigms helps identify which problems might benefit from quantum approaches.

## The QML Landscape

```
Quantum Machine Learning Paradigms
══════════════════════════════════

┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   Classical Data        Quantum Data                           │
│   ┌─────────────┐       ┌─────────────┐                        │
│   │             │       │             │                        │
│   │  CC → CQ    │       │  QC → QQ    │                        │
│   │             │       │             │                        │
│   └─────────────┘       └─────────────┘                        │
│                                                                 │
│   CC: Classical data, Classical algorithm (traditional ML)     │
│   CQ: Classical data, Quantum algorithm (most QML today)       │
│   QC: Quantum data, Classical algorithm (tomography)           │
│   QQ: Quantum data, Quantum algorithm (quantum sensing)        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

NEAR-TERM FOCUS:
    CQ paradigm - Using quantum computers to process classical data
    This is where most current QML research operates
```

## Variational Quantum Algorithms (VQAs)

```python
def variational_paradigm():
    """
    Explain the variational quantum algorithm paradigm.
    """
    print("""
Variational Quantum Algorithms
══════════════════════════════

CORE IDEA:
    Hybrid classical-quantum optimization
    Quantum circuit with tunable parameters
    Classical optimizer adjusts parameters

STRUCTURE:
    ┌─────────────────────────────────────────────────────────┐
    │                                                         │
    │   Classical        Quantum            Classical         │
    │   ┌───────┐       ┌───────┐          ┌───────┐         │
    │   │Prepare│──────▶│Execute│─────────▶│Measure│         │
    │   │ θ     │       │U(θ)   │          │       │         │
    │   └───────┘       └───────┘          └───┬───┘         │
    │       ▲                                  │              │
    │       │                                  ▼              │
    │       │           ┌───────┐         ┌───────┐          │
    │       └───────────│Update │◀────────│ Loss  │          │
    │                   │  θ    │         │       │          │
    │                   └───────┘         └───────┘          │
    │                                                         │
    └─────────────────────────────────────────────────────────┘

KEY EXAMPLES:

1. VQE (Variational Quantum Eigensolver)
    - Find ground state energy of molecules
    - Parameterized quantum circuit
    - Minimize ⟨ψ(θ)|H|ψ(θ)⟩

2. QAOA (Quantum Approximate Optimization)
    - Solve combinatorial problems
    - Alternating mixer and problem layers
    - Parameter count grows with depth

3. VQC (Variational Quantum Classifiers)
    - Classification using quantum circuits
    - Encode data, apply variational layer
    - Measure to get class prediction


ADVANTAGES:
    ✓ Works on noisy near-term devices
    ✓ Shallow circuits feasible
    ✓ Flexible architecture

CHALLENGES:
    ✗ Barren plateaus (vanishing gradients)
    ✗ Local minima
    ✗ Unclear quantum advantage
    """)

variational_paradigm()
```

## Quantum Kernel Methods

```
Quantum Kernel Methods
══════════════════════

CLASSICAL KERNEL IDEA:
    Map data to high-dimensional feature space
    Compute inner products (kernel) in that space
    Linear methods become powerful in feature space

    K(x, x') = ⟨φ(x)|φ(x')⟩

QUANTUM KERNEL:
    Use quantum computer to compute kernel
    Feature map = quantum circuit
    Inner product = measurement overlap

    ┌─────────────────────────────────────────────────────────┐
    │                                                         │
    │  x ──▶ [U(x)] ──▶ |φ(x)⟩                               │
    │                      │                                  │
    │                      ▼                                  │
    │  K(x,x') = |⟨φ(x)|φ(x')⟩|²                             │
    │                      ▲                                  │
    │                      │                                  │
    │  x' ──▶ [U(x')] ──▶ |φ(x')⟩                            │
    │                                                         │
    └─────────────────────────────────────────────────────────┘

COMPUTING QUANTUM KERNEL:

    Method 1: Swap test
    K(x,x') = |⟨0|U†(x')U(x)|0⟩|²

    Method 2: Direct overlap
    Prepare |φ(x)⟩, apply U†(x'), measure |0⟩

POTENTIAL ADVANTAGE:
    Quantum feature maps access exponentially large space
    Some kernels classically intractable to compute
    May capture patterns classical kernels miss

EXAMPLE KERNEL CIRCUIT:
    |0⟩ ─[H]─[Rz(x₁)]─●─[Rz(x₂)]─ ... ─[measure]
    |0⟩ ─[H]─[Rz(x₂)]─X─[Rz(x₁)]─ ...
```

## Quantum Neural Networks

```python
def quantum_neural_networks():
    """
    Describe quantum neural network approaches.
    """
    print("""
Quantum Neural Networks (QNNs)
══════════════════════════════

CONCEPT:
    Quantum circuits as neural network layers
    Parameters in gates ≈ neural network weights
    Training via gradient descent

ARCHITECTURE:

    Input     Encoding     Variational     Measurement
    ─────     ────────     ───────────     ───────────

    x ──────▶ [U_enc(x)] ─▶ [U_var(θ)] ─▶ [M] ──▶ ŷ

    U_enc(x): Encode classical data into quantum state
    U_var(θ): Trainable quantum layers
    M: Measurement to extract prediction


LAYER STRUCTURE (typical):

    |0⟩ ─[Ry(θ₁)]─●─────────[Ry(θ₅)]─
                  │
    |0⟩ ─[Ry(θ₂)]─X─●───────[Ry(θ₆)]─
                    │
    |0⟩ ─[Ry(θ₃)]───X─●─────[Ry(θ₇)]─
                      │
    |0⟩ ─[Ry(θ₄)]─────X─────[Ry(θ₈)]─

    Layer = rotations + entangling gates


TRAINING:
    1. Forward pass: Run circuit, measure
    2. Compute loss: Compare prediction to label
    3. Gradient: Parameter shift rule or finite difference
    4. Update: Classical optimizer adjusts θ

PARAMETER SHIFT RULE:
    ∂/∂θ ⟨O⟩ = (⟨O⟩_{θ+π/2} - ⟨O⟩_{θ-π/2}) / 2

    Exact gradients from circuit evaluations!


CHALLENGES:
    - Barren plateaus: gradients vanish exponentially
    - Expressibility vs trainability tradeoff
    - Limited qubit count restricts capacity
    """)

quantum_neural_networks()
```

## Comparison of Paradigms

```
Paradigm Comparison
═══════════════════

┌──────────────────────────────────────────────────────────────────┐
│  Paradigm    │  Qubits  │  Depth  │  Advantage    │  Maturity   │
├──────────────────────────────────────────────────────────────────┤
│  VQA         │  10-100  │  Low    │  Unclear      │  High       │
│  Quantum     │  10-100  │  Medium │  Theoretical  │  Medium     │
│   Kernels    │          │         │               │             │
│  QNN         │  10-50   │  Medium │  Unclear      │  Medium     │
│  Quantum     │  1000+   │  High   │  Proven       │  Low        │
│   Sampling   │          │         │  (specific)   │             │
│  Fault-tol.  │  10^6+   │  Any    │  Exponential  │  Future     │
│   QML        │          │         │  (some)       │             │
└──────────────────────────────────────────────────────────────────┘

WHEN TO USE WHAT:

    VQAs: Near-term, optimization problems, chemistry

    Quantum Kernels: When classical kernels insufficient,
                     data has quantum-like structure

    QNNs: Pattern recognition, when you'd use neural nets

    Quantum Sampling: Generative models, specific distributions


KEY CONSIDERATIONS:

    1. Problem structure: Does it have quantum speedup potential?
    2. Data size: Quantum loading can dominate runtime
    3. Noise tolerance: Variational more robust than deep
    4. Classical baseline: Always compare to classical ML
```

## Summary

QML paradigms offer different tradeoffs:

1. **Variational (VQA)**: Hybrid, near-term feasible, widely studied
2. **Quantum Kernels**: Potentially powerful feature spaces
3. **QNNs**: Quantum analogs of neural networks
4. **Classical data**: Focus of current research, quantum loading is a bottleneck

The key question remains: for which problems do quantum approaches provide genuine advantage over classical ML?
