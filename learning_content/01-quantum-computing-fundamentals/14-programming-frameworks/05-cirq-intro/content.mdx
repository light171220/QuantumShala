# Introduction to Cirq

## Introduction

Cirq is Google's open-source quantum computing framework, designed for creating, editing, and running quantum circuits, particularly for NISQ algorithms.

## Cirq Design Philosophy

```
Cirq Core Concepts
══════════════════

1. EXPLICIT QUBIT CONTROL
   You choose qubit layout explicitly
   LineQubit, GridQubit, NamedQubit

2. MOMENT-BASED CIRCUITS
   Operations grouped into moments
   Parallelism explicit

3. NISQ FOCUS
   Optimized for near-term hardware
   Fine-grained control
   Noise simulation

4. GOOGLE HARDWARE NATIVE
   Direct access to Google quantum processors
   Sycamore architecture support
```

## Basic Circuit Creation

```python
import cirq

# Create qubits
q0 = cirq.LineQubit(0)
q1 = cirq.LineQubit(1)
# Or: q0, q1 = cirq.LineQubit.range(2)

# Create circuit
circuit = cirq.Circuit([
    cirq.H(q0),
    cirq.CNOT(q0, q1),
    cirq.measure(q0, q1, key='result')
])

print("Bell State Circuit:")
print(circuit)

# Output:
# 0: ───H───@───M('result')───
#           │   │
# 1: ───────X───M─────────────
```

## Qubit Types

```python
import cirq

# LineQubit: 1D arrangement
line_qubits = cirq.LineQubit.range(5)
# 0, 1, 2, 3, 4

# GridQubit: 2D arrangement (like Sycamore)
grid_qubits = cirq.GridQubit.rect(3, 3)
# (0,0), (0,1), (0,2),
# (1,0), (1,1), (1,2),
# (2,0), (2,1), (2,2)

# NamedQubit: Custom names
alice = cirq.NamedQubit('alice')
bob = cirq.NamedQubit('bob')

print("Qubit Types:")
print(f"  LineQubit: {line_qubits[:3]}")
print(f"  GridQubit: {grid_qubits[:3]}")
print(f"  NamedQubit: {alice}, {bob}")
```

## Gates and Operations

```python
import cirq
import numpy as np

q = cirq.LineQubit(0)
q0, q1 = cirq.LineQubit.range(2)

# Single-qubit gates
circuit = cirq.Circuit([
    cirq.X(q),           # Pauli X
    cirq.Y(q),           # Pauli Y
    cirq.Z(q),           # Pauli Z
    cirq.H(q),           # Hadamard
    cirq.S(q),           # S gate
    cirq.T(q),           # T gate
    cirq.rx(np.pi/4)(q), # Rotation X
    cirq.ry(np.pi/4)(q), # Rotation Y
    cirq.rz(np.pi/4)(q), # Rotation Z
])

# Two-qubit gates
two_qubit = cirq.Circuit([
    cirq.CNOT(q0, q1),
    cirq.CZ(q0, q1),
    cirq.SWAP(q0, q1),
    cirq.ISWAP(q0, q1),
])

print("Available gates: X, Y, Z, H, S, T, rx, ry, rz, CNOT, CZ, SWAP...")
```

## Simulation

```python
import cirq

# Create circuit
q0, q1 = cirq.LineQubit.range(2)
circuit = cirq.Circuit([
    cirq.H(q0),
    cirq.CNOT(q0, q1),
    cirq.measure(q0, q1, key='m')
])

# Simulate
simulator = cirq.Simulator()
result = simulator.run(circuit, repetitions=1000)

# Get counts
print("Measurement Results:")
print(result.histogram(key='m'))

# Statevector simulation (no measurements)
circuit_no_meas = cirq.Circuit([
    cirq.H(q0),
    cirq.CNOT(q0, q1)
])
final_state = simulator.simulate(circuit_no_meas)
print(f"\nStatevector: {final_state.final_state_vector}")
```

## Moments and Scheduling

```python
import cirq

q0, q1, q2 = cirq.LineQubit.range(3)

# Explicit moment construction
circuit = cirq.Circuit([
    cirq.Moment([cirq.H(q0), cirq.H(q1), cirq.H(q2)]),  # Parallel
    cirq.Moment([cirq.CNOT(q0, q1)]),
    cirq.Moment([cirq.CNOT(q1, q2)]),
])

print("Moment-based circuit:")
print(circuit)

# Automatic scheduling
ops = [cirq.H(q0), cirq.H(q1), cirq.CNOT(q0, q1), cirq.H(q2)]
auto_circuit = cirq.Circuit(ops)
print("\nAuto-scheduled:")
print(auto_circuit)
```

## Summary

Cirq provides fine-grained quantum circuit control:

1. **Qubits**: LineQubit, GridQubit, NamedQubit
2. **Moments**: Explicit parallel operation grouping
3. **Simulation**: Built-in simulator with statevector access
4. **NISQ focus**: Designed for near-term algorithms
5. **Google native**: Direct Sycamore hardware access
