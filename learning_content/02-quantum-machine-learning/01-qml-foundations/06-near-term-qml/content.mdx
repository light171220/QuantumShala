## 1. The NISQ Era

We are currently in the Noisy Intermediate-Scale Quantum (NISQ) era of quantum computing.

<Definition term="NISQ">
Noisy Intermediate-Scale Quantum devices have 50-1000+ qubits but lack full error correction. Algorithms must be noise-resilient and use shallow circuits (few gates).
</Definition>

## 2. Circuit Depth Limitations

The maximum useful circuit depth is fundamentally limited by noise accumulation.

```python
import numpy as np

def calculate_max_depth(gate_fidelity_1q, gate_fidelity_2q, n_qubits, min_fidelity=0.5):
    """
    Estimate maximum useful circuit depth.

    Args:
        gate_fidelity_1q: Single-qubit gate fidelity
        gate_fidelity_2q: Two-qubit gate fidelity
        n_qubits: Number of qubits
        min_fidelity: Minimum acceptable circuit fidelity

    Returns:
        Approximate maximum gate count
    """
    # Assume roughly equal single and two-qubit gates
    # Effective fidelity per layer
    avg_fidelity = (gate_fidelity_1q + gate_fidelity_2q) / 2

    # Solve: fidelity^depth = min_fidelity
    max_depth = np.log(min_fidelity) / np.log(avg_fidelity)

    return int(max_depth)

# Current hardware estimates
print("Maximum Circuit Depth Estimates:")
print("=" * 60)
print()

hardware_configs = [
    ("Current superconducting", 0.999, 0.99),
    ("Near-term improved", 0.9995, 0.995),
    ("Future NISQ", 0.9999, 0.999),
]

print(f"{'Hardware':<25} {'1Q Fidelity':<12} {'2Q Fidelity':<12} {'Max Depth':<10}")
print("-" * 60)

for name, f1q, f2q in hardware_configs:
    max_d = calculate_max_depth(f1q, f2q, 10)
    print(f"{name:<25} {f1q:<12} {f2q:<12} ~{max_d:<10}")

print("\n⚠️ These are rough estimates - actual limits depend on circuit structure")
```

<Warning>
With current 99% two-qubit gate fidelity, practical circuits are limited to roughly 50-100 gates before noise overwhelms the signal. This fundamentally constrains the complexity of NISQ algorithms.
</Warning>

## 4. Error Mitigation Strategies

Since full error correction isn't available, we use error mitigation techniques.

<Definition term="Error Mitigation">
Classical post-processing techniques that reduce the effect of noise on quantum computation results without requiring additional qubits for encoding. Unlike error correction, mitigation cannot eliminate errors completely.
</Definition>

```python
import numpy as np

def zero_noise_extrapolation_demo():
    """
    Demonstrate Zero Noise Extrapolation (ZNE).

    The idea: Run circuit at different noise levels,
    then extrapolate to zero noise.
    """

    print("Zero Noise Extrapolation (ZNE):")
    print("=" * 50)

    # Simulated expectation values at different noise levels
    # True value would be 1.0
    true_value = 1.0

    # Noise scaling factors
    scale_factors = np.array([1.0, 1.5, 2.0, 2.5, 3.0])

    # Simulated noisy measurements (exponential decay with noise)
    noise_rate = 0.1
    noisy_values = true_value * np.exp(-noise_rate * scale_factors)

    print(f"True (noiseless) value: {true_value}")
    print()
    print(f"{'Scale Factor':<15} {'Measured Value':<15}")
    print("-" * 30)
    for s, v in zip(scale_factors, noisy_values):
        print(f"{s:<15} {v:<15.4f}")

    # Linear extrapolation to scale_factor = 0
    # Using simple linear regression
    coeffs = np.polyfit(scale_factors, noisy_values, 1)
    extrapolated = coeffs[1]  # y-intercept

    print()
    print(f"Linear extrapolation to zero noise: {extrapolated:.4f}")
    print(f"Error: {abs(extrapolated - true_value):.4f}")

    # Better: exponential fit
    log_values = np.log(noisy_values)
    exp_coeffs = np.polyfit(scale_factors, log_values, 1)
    exp_extrapolated = np.exp(exp_coeffs[1])

    print(f"Exponential extrapolation: {exp_extrapolated:.4f}")
    print(f"Error: {abs(exp_extrapolated - true_value):.4f}")

zero_noise_extrapolation_demo()
```
