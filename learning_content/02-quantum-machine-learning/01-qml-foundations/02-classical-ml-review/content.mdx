## 2. Supervised Learning Fundamentals

Supervised learning is the most common ML paradigm and the primary focus of near-term QML.

<Definition term="Supervised Learning">
Learning from labeled examples where each input X has a corresponding target y. The goal is to learn a function that generalizes to unseen data.
</Definition>

```python
import numpy as np

# Simple linear regression example
np.random.seed(42)

# Generate synthetic data
n_samples = 100
X = np.random.randn(n_samples, 1)
y = 2 * X + 1 + 0.1 * np.random.randn(n_samples, 1)

# The goal: find w and b such that y ≈ wX + b
# This is what ML algorithms learn!

print("Supervised Learning Setup:")
print(f"  Input X shape: {X.shape}")
print(f"  Target y shape: {y.shape}")
print(f"  Goal: Learn f(X) = wX + b")
print(f"  True parameters: w=2, b=1")

# Simple closed-form solution (least squares)
X_bias = np.column_stack([X, np.ones(n_samples)])
w_learned = np.linalg.lstsq(X_bias, y, rcond=None)[0]

print(f"\nLearned parameters:")
print(f"  w = {w_learned[0,0]:.4f}")
print(f"  b = {w_learned[1,0]:.4f}")
```

## 3. Loss Functions and Optimization

The heart of ML is optimization: finding parameters that minimize a loss function.

<Definition term="Loss Function">
A function that measures how wrong the model's predictions are. The goal of training is to minimize this loss.
</Definition>

```python
import numpy as np

def mse_loss(y_true, y_pred):
    """Mean Squared Error - for regression"""
    return np.mean((y_true - y_pred) ** 2)

def cross_entropy_loss(y_true, y_pred):
    """Binary Cross Entropy - for classification"""
    epsilon = 1e-15  # Avoid log(0)
    y_pred = np.clip(y_pred, epsilon, 1 - epsilon)
    return -np.mean(y_true * np.log(y_pred) + (1 - y_true) * np.log(1 - y_pred))

# Example predictions
y_true = np.array([1, 0, 1, 1, 0])
y_pred_good = np.array([0.9, 0.1, 0.8, 0.95, 0.2])
y_pred_bad = np.array([0.5, 0.5, 0.5, 0.5, 0.5])

print("Loss Function Comparison:")
print("-" * 40)
print(f"Good predictions CE loss: {cross_entropy_loss(y_true, y_pred_good):.4f}")
print(f"Bad predictions CE loss:  {cross_entropy_loss(y_true, y_pred_bad):.4f}")
print("\nLower loss = better model")
```

<Formula>
$$\text{MSE} = \frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2$$
$$\text{Cross-Entropy} = -\frac{1}{n}\sum_{i=1}^{n}[y_i\log(\hat{y}_i) + (1-y_i)\log(1-\hat{y}_i)]$$
</Formula>

## 4. Gradient Descent

Gradient descent is the workhorse of ML optimization.

<Definition term="Gradient Descent">
An iterative optimization algorithm that updates parameters in the direction opposite to the gradient of the loss function, moving toward a minimum.
</Definition>

<Formula>
$$\theta_{t+1} = \theta_t - \eta \nabla_\theta \mathcal{L}(\theta_t)$$
</Formula>

```python
import numpy as np

def gradient_descent_1d(f, df, x0, learning_rate=0.1, n_steps=50):
    """
    Simple gradient descent on a 1D function.

    Args:
        f: Function to minimize
        df: Gradient of f
        x0: Initial point
        learning_rate: Step size (η)
        n_steps: Number of iterations
    """
    x = x0
    history = [x]

    for i in range(n_steps):
        gradient = df(x)
        x = x - learning_rate * gradient
        history.append(x)

    return x, history

# Minimize f(x) = x^2 (minimum at x=0)
f = lambda x: x ** 2
df = lambda x: 2 * x

x_min, history = gradient_descent_1d(f, df, x0=5.0, learning_rate=0.1)

print("Gradient Descent on f(x) = x²:")
print("-" * 40)
print(f"Starting point: x = 5.0")
print(f"Final point: x = {x_min:.6f}")
print(f"Steps to converge: {len(history) - 1}")
print(f"\nFirst 10 iterations: {[f'{h:.4f}' for h in history[:10]]}")
```

<Tip>
The learning rate η is crucial: too large causes divergence, too small causes slow convergence. This same challenge exists in quantum machine learning with variational circuits.
</Tip>

## 5. Neural Networks and Backpropagation

Neural networks are powerful function approximators trained via backpropagation.

<Definition term="Neural Network">
A computational model inspired by biological neurons, consisting of layers of interconnected nodes that transform inputs through weighted connections and nonlinear activations.
</Definition>

```python
import numpy as np

class SimpleNeuralNetwork:
    """A minimal 2-layer neural network for demonstration."""

    def __init__(self, input_dim, hidden_dim, output_dim):
        # Initialize weights randomly
        self.W1 = np.random.randn(input_dim, hidden_dim) * 0.1
        self.b1 = np.zeros(hidden_dim)
        self.W2 = np.random.randn(hidden_dim, output_dim) * 0.1
        self.b2 = np.zeros(output_dim)

    def sigmoid(self, x):
        return 1 / (1 + np.exp(-np.clip(x, -500, 500)))

    def forward(self, X):
        """Forward pass through the network."""
        self.z1 = X @ self.W1 + self.b1
        self.a1 = self.sigmoid(self.z1)
        self.z2 = self.a1 @ self.W2 + self.b2
        self.a2 = self.sigmoid(self.z2)
        return self.a2

# Create a small network
nn = SimpleNeuralNetwork(input_dim=4, hidden_dim=8, output_dim=2)

# Forward pass
X_sample = np.random.randn(1, 4)
output = nn.forward(X_sample)

print("Simple Neural Network:")
print("-" * 40)
print(f"Input shape: {X_sample.shape}")
print(f"Hidden layer: 8 neurons")
print(f"Output shape: {output.shape}")
print(f"Output values: {output[0]}")
print(f"\nTotal parameters: {nn.W1.size + nn.b1.size + nn.W2.size + nn.b2.size}")
```

<Warning>
Backpropagation relies on the chain rule to compute gradients. In quantum circuits, we use the parameter-shift rule instead, which is fundamentally different but serves the same purpose.
</Warning>

## 6. Kernel Methods

Kernel methods are particularly relevant to QML because quantum computers can compute kernels in exponentially large feature spaces.

<Definition term="Kernel">
A function K(x, x') that computes the inner product of data points in a high-dimensional feature space without explicitly computing the feature vectors: K(x, x') = ⟨φ(x), φ(x')⟩.
</Definition>

```python
import numpy as np

def linear_kernel(x1, x2):
    """K(x, x') = x · x'"""
    return np.dot(x1, x2)

def polynomial_kernel(x1, x2, degree=2):
    """K(x, x') = (x · x' + 1)^d"""
    return (np.dot(x1, x2) + 1) ** degree

def rbf_kernel(x1, x2, gamma=1.0):
    """K(x, x') = exp(-γ||x - x'||²)"""
    return np.exp(-gamma * np.sum((x1 - x2) ** 2))

# Example points
x1 = np.array([1.0, 2.0])
x2 = np.array([3.0, 4.0])

print("Kernel Computations:")
print("-" * 40)
print(f"Points: x1 = {x1}, x2 = {x2}")
print(f"\nLinear kernel: {linear_kernel(x1, x2):.4f}")
print(f"Polynomial kernel (d=2): {polynomial_kernel(x1, x2, degree=2):.4f}")
print(f"RBF kernel (γ=1): {rbf_kernel(x1, x2, gamma=1.0):.4f}")

print("\n→ Quantum computers can compute kernels in")
print("  exponentially large Hilbert spaces!")
```

<Tip>
Quantum kernels are a promising near-term QML approach. The quantum computer computes K(x, x') = |⟨φ(x)|φ(x')⟩|² where φ(x) is a quantum feature map, potentially accessing feature spaces classical computers cannot efficiently explore.
</Tip>

## 7. Computational Bottlenecks in Classical ML

Understanding where classical ML struggles helps identify opportunities for quantum advantage.

```python
import numpy as np

# Computational complexity of common ML operations
bottlenecks = {
    "Matrix Inversion": {
        "complexity": "O(n³)",
        "appears_in": "Linear regression, Gaussian processes",
        "quantum_speedup": "HHL algorithm offers O(log n) but with caveats"
    },
    "Eigendecomposition": {
        "complexity": "O(n³)",
        "appears_in": "PCA, spectral clustering",
        "quantum_speedup": "Quantum PCA for specific cases"
    },
    "Gradient Computation": {
        "complexity": "O(parameters × data)",
        "appears_in": "Neural network training",
        "quantum_speedup": "Uncertain, barren plateaus are a challenge"
    },
    "Kernel Matrix": {
        "complexity": "O(n² × d)",
        "appears_in": "SVM, kernel methods",

        "complexity": "Depends on distribution",
        "appears_in": "MCMC, Boltzmann machines",
        "quantum_speedup": "Quantum sampling might be faster"

print("Computational Bottlenecks in Classical ML:")
print("=" * 65)

    print(f"\n{operation}:")
    print(f"  Complexity: {details['complexity']}")
    print(f"  Appears in: {details['appears_in']}")

```

## 8. From Classical to Quantum

Key concepts map from classical ML to quantum ML:

```python
# Mapping classical concepts to quantum
mapping = {
    "Parameters θ": {
        "classical": "Weights and biases in neural networks",
        "quantum": "Rotation angles in parameterized circuits"
    },
    "Forward pass": {
        "classical": "Input → hidden layers → output",
        "quantum": "State prep → unitary gates → measurement"
    },
    "Loss function": {
        "classical": "MSE, cross-entropy",
        "quantum": "Expectation value ⟨ψ|H|ψ⟩"
    },
    "Gradient": {
        "classical": "Backpropagation, autodiff",
        "quantum": "Parameter-shift rule"
    },
    "Feature space": {
        "classical": "Finite dimensional",
        "quantum": "Hilbert space (exponentially large)"
    },
    "Training data": {
        "classical": "Stored in memory",
        "quantum": "Encoded into quantum states"
    }
}

print("Classical → Quantum Concept Mapping:")
print("=" * 65)
for concept, versions in mapping.items():
    print(f"\n{concept}:")
    print(f"  Classical: {versions['classical']}")
    print(f"  Quantum:   {versions['quantum']}")
```

<Warning>
Not all classical ML concepts translate directly to quantum. Some quantum concepts (superposition, entanglement, measurement collapse) have no classical analog and enable fundamentally new approaches.
</Warning>
