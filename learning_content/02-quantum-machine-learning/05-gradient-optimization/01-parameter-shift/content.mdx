# Parameter Shift Rule

The parameter-shift rule is a technique for computing exact gradients of parameterized quantum circuits that works directly on quantum hardware.

## Learning Objectives

After completing this lesson, you will be able to:

- Derive the parameter-shift rule
- Apply parameter-shift to common gates
- Implement gradient computation
- Understand hardware compatibility
- Compare with classical differentiation

## 1. Introduction to Quantum Gradients

Computing gradients is essential for training variational quantum circuits.

<Definition term="Parameter-Shift Rule">
A technique to compute the exact derivative of a quantum expectation value by evaluating the circuit at shifted parameter values: ∂f/∂θ = r[f(θ+s) - f(θ-s)], where r and s are gate-dependent constants.
</Definition>

## 2. Derivation of the Parameter-Shift Rule

Mathematical derivation of the rule for Pauli rotations.

```python
import numpy as np

def parameter_shift_derivation():
    """
    Derive the parameter-shift rule mathematically.
    """
    print("Parameter-Shift Rule Derivation:")
    print("=" * 50)

    print("\n1. Setup:")
    print("-" * 40)
    print("  Consider: f(θ) = ⟨ψ|U†(θ) O U(θ)|ψ⟩")
    print("  Where: U(θ) = exp(-i θ P / 2)")
    print("  And: P is a Pauli operator (P² = I)")

    print("\n\n2. Expansion:")
    print("-" * 40)
    print("  U(θ) = cos(θ/2) I - i sin(θ/2) P")
    print("  ")
    print("  Therefore f(θ) has form:")
    print("  f(θ) = A cos²(θ/2) + B sin²(θ/2) + C sin(θ)")

    print("\n\n3. Derivative:")
    print("-" * 40)
    print("  df/dθ = (-A + B) sin(θ/2) cos(θ/2) + C cos(θ)")
    print("        = (B - A)/2 sin(θ) + C cos(θ)")

    print("\n\n4. Shift Trick:")
    print("-" * 40)
    print("  f(θ + π/2) - f(θ - π/2) = ")
    print("    [A cos²((θ+π/2)/2) + ...] - [A cos²((θ-π/2)/2) + ...]")
    print("  ")
    print("  After simplification:")
    print("  f(θ + π/2) - f(θ - π/2) = 2 × df/dθ")

    print("\n\n5. Final Formula:")
    print("-" * 40)
    print("  ┌─────────────────────────────────────────┐")
    print("  │  ∂f/∂θ = (1/2)[f(θ+π/2) - f(θ-π/2)]    │")
    print("  └─────────────────────────────────────────┘")

    # Numerical verification
    print("\n\n6. Numerical Verification:")
    print("-" * 40)

    def f(theta):
        """Example function with Pauli structure."""
        A, B, C = 1.0, 0.5, 0.3
        return A * np.cos(theta/2)**2 + B * np.sin(theta/2)**2 + C * np.sin(theta)

    def analytical_derivative(theta):
        """Analytical derivative."""
        A, B, C = 1.0, 0.5, 0.3
        return (B - A)/2 * np.sin(theta) + C * np.cos(theta)

    theta = 0.7
    shift = np.pi / 2

    param_shift_grad = 0.5 * (f(theta + shift) - f(theta - shift))
    analytical_grad = analytical_derivative(theta)

    print(f"  θ = {theta:.2f}")
    print(f"  Parameter-shift: {param_shift_grad:.6f}")
    print(f"  Analytical:      {analytical_grad:.6f}")
    print(f"  Match: {np.isclose(param_shift_grad, analytical_grad)}")

parameter_shift_derivation()
```

<Formula>
For a Pauli rotation gate $R_P(\theta) = e^{-i\theta P/2}$ where $P \in \{X, Y, Z\}$:
$$\frac{\partial f}{\partial \theta} = \frac{1}{2}\left[f\left(\theta + \frac{\pi}{2}\right) - f\left(\theta - \frac{\pi}{2}\right)\right]$$
</Formula>

## 3. Implementation

Practical implementation of parameter-shift gradients.

```python
import numpy as np

class ParameterShiftGradient:
    """
    Compute gradients using the parameter-shift rule.
    """

    def __init__(self, circuit_fn, shift=np.pi/2):
        """
        Args:
            circuit_fn: Function that takes params and returns expectation value
            shift: Shift amount (default π/2 for Pauli gates)
        """
        self.circuit = circuit_fn
        self.shift = shift
        self.coefficient = 0.5  # For standard Pauli rotations

    def gradient(self, params):
        """
        Compute gradient with respect to all parameters.

        Args:
            params: Parameter array

        Returns:
            Gradient array
        """
        params = np.array(params)
        n_params = len(params)
        grad = np.zeros(n_params)

        for i in range(n_params):
            # Shift parameter up
            params_plus = params.copy()
            params_plus[i] += self.shift

            # Shift parameter down
            params_minus = params.copy()
            params_minus[i] -= self.shift

            # Compute gradient component
            f_plus = self.circuit(params_plus)
            f_minus = self.circuit(params_minus)
            grad[i] = self.coefficient * (f_plus - f_minus)

        return grad

    def hessian(self, params):
        """
        Compute Hessian (second derivatives) using double shifts.

        Args:
            params: Parameter array

        Returns:
            Hessian matrix
        """
        params = np.array(params)
        n_params = len(params)
        hess = np.zeros((n_params, n_params))

        for i in range(n_params):
            for j in range(n_params):
                if i == j:
                    # Diagonal: second derivative
                    p_pp = params.copy()
                    p_pp[i] += 2 * self.shift

                    p_mm = params.copy()
                    p_mm[i] -= 2 * self.shift

                    f_pp = self.circuit(p_pp)
                    f_mm = self.circuit(p_mm)
                    f_0 = self.circuit(params)

                    hess[i, i] = 0.25 * (f_pp + f_mm - 2 * f_0)
                else:
                    # Off-diagonal: mixed partial
                    p_ij = [params.copy() for _ in range(4)]
                    p_ij[0][i] += self.shift; p_ij[0][j] += self.shift
                    p_ij[1][i] += self.shift; p_ij[1][j] -= self.shift
                    p_ij[2][i] -= self.shift; p_ij[2][j] += self.shift
                    p_ij[3][i] -= self.shift; p_ij[3][j] -= self.shift

                    f_vals = [self.circuit(p) for p in p_ij]
                    hess[i, j] = 0.25 * (f_vals[0] - f_vals[1] - f_vals[2] + f_vals[3])

        return hess

# Demonstration
print("Parameter-Shift Implementation:")
print("=" * 50)

# Simulated quantum circuit
def quantum_circuit(params):
    """Simulated expectation value with Pauli structure."""
    theta, phi = params
    # <Z> after RY(θ) RZ(φ)
    return np.cos(theta) * np.cos(phi)

grad_calculator = ParameterShiftGradient(quantum_circuit)

# Test at a point
params = np.array([0.5, 0.8])
gradient = grad_calculator.gradient(params)

print(f"\nParameters: θ={params[0]:.2f}, φ={params[1]:.2f}")
print(f"Gradient: [{gradient[0]:.4f}, {gradient[1]:.4f}]")

# Analytical check
analytical = [
    -np.sin(params[0]) * np.cos(params[1]),
    -np.cos(params[0]) * np.sin(params[1])
]
print(f"Analytical: [{analytical[0]:.4f}, {analytical[1]:.4f}]")
print(f"Match: {np.allclose(gradient, analytical)}")
```

## 4. Generalized Parameter-Shift Rules

Extensions for gates with different eigenvalue spectra.

```python
import numpy as np

def generalized_parameter_shift():
    """
    Parameter-shift rules for various gate types.
    """
    print("Generalized Parameter-Shift Rules:")
    print("=" * 50)

    print("\n1. Two-Term Rule (Pauli Rotations):")
    print("-" * 40)
    print("  Gates: RX, RY, RZ")
    print("  Eigenvalues: ±1/2")
    print("  Shift: π/2")
    print("  Coefficient: 1/2")
    print("  Formula: ∂f/∂θ = (1/2)[f(θ+π/2) - f(θ-π/2)]")

    print("\n\n2. Four-Term Rule (Controlled Rotations):")
    print("-" * 40)
    print("  Gates: CRX, CRY, CRZ")
    print("  Multiple eigenvalue gaps")
    print("  Requires 4 circuit evaluations")

    def four_term_gradient(circuit_fn, params, param_idx):
        """Four-term parameter shift for controlled gates."""
        params = np.array(params)
        shifts = [np.pi/2, 3*np.pi/2]
        coeffs = [
            (np.sqrt(2) + 1) / (4 * np.sqrt(2)),
            (np.sqrt(2) - 1) / (4 * np.sqrt(2))
        ]

        grad = 0
        for s, c in zip(shifts, coeffs):
            p_plus = params.copy()
            p_plus[param_idx] += s
            p_minus = params.copy()
            p_minus[param_idx] -= s

            grad += c * (circuit_fn(p_plus) - circuit_fn(p_minus))

        return grad

    print("\n\n3. General Multi-Term Rule:")
    print("-" * 40)
    print("  For generator G with eigenvalues λ₁, λ₂, ..., λₖ:")
    print("  Number of terms = k(k-1)/2 distinct gaps")
    print("  Each gap requires a separate shift")

    # Example for two-qubit gates
    print("\n\n4. Common Gate Requirements:")
    print("-" * 40)

    gates = [
        ("RX, RY, RZ", 2, "Standard Pauli"),
        ("CRX, CRY, CRZ", 4, "Controlled Pauli"),
        ("CNOT, CZ", 0, "No parameters"),
        ("RXX, RYY, RZZ", 2, "Ising-type"),
        ("Arbitrary U", "Varies", "Decompose first"),
    ]

    print(f"  {'Gate':<15} {'Evaluations':<12} {'Notes':<20}")
    print("  " + "-" * 47)
    for gate, evals, notes in gates:
        print(f"  {gate:<15} {str(evals):<12} {notes:<20}")

generalized_parameter_shift()
```

<Warning>
Not all gates support the simple two-term parameter-shift rule. Check the gate's eigenvalue spectrum to determine the correct formula.
</Warning>

## 5. Cost and Optimization

Understanding the computational cost of parameter-shift gradients.

```python
import numpy as np

def gradient_cost_analysis():
    """
    Analyze the cost of computing gradients.
    """
    print("Gradient Computation Cost Analysis:")
    print("=" * 50)

    print("\n1. Cost Per Gradient:")
    print("-" * 40)
    print("  Parameter-shift rule: 2 × n_params evaluations")
    print("  Plus 1 evaluation for forward pass")
    print("  ")
    print("  Total: 2n + 1 circuit evaluations per step")

    print("\n\n2. Shot Noise Considerations:")
    print("-" * 40)

    def gradient_with_shots(circuit_fn, params, n_shots=1000):
        """Simulate shot noise in gradient estimation."""
        n_params = len(params)
        shift = np.pi / 2

        grad_samples = []

        for _ in range(100):  # Multiple gradient estimates
            grad = np.zeros(n_params)
            for i in range(n_params):
                p_plus = params.copy()
                p_plus[i] += shift
                p_minus = params.copy()
                p_minus[i] -= shift

                # Simulate shot noise
                f_plus = circuit_fn(p_plus) + np.random.normal(0, 1/np.sqrt(n_shots))
                f_minus = circuit_fn(p_minus) + np.random.normal(0, 1/np.sqrt(n_shots))

                grad[i] = 0.5 * (f_plus - f_minus)

            grad_samples.append(grad)

        return np.mean(grad_samples, axis=0), np.std(grad_samples, axis=0)

    def test_circuit(params):
        return np.cos(params[0]) * np.sin(params[1])

    params = np.array([0.5, 0.8])

    print("\n  Gradient variance vs shots:")
    print(f"  {'Shots':<10} {'Grad std':<15} {'Relative error':<15}")
    print("  " + "-" * 40)

    for n_shots in [100, 1000, 10000]:
        _, grad_std = gradient_with_shots(test_circuit, params, n_shots)
        mean_std = np.mean(grad_std)
        rel_error = mean_std / (np.abs(np.sin(params[0]) * np.sin(params[1])) + 0.1)
        print(f"  {n_shots:<10} {mean_std:<15.4f} {rel_error:<15.2%}")

    print("\n\n3. Optimization Strategies:")
    print("-" * 40)

    strategies = [
        ("Batch gradients", "Accumulate over multiple shots"),
        ("Stochastic selection", "Update subset of params per step"),
        ("Rotosolve", "Analytical 1D optimization, no shots needed"),
        ("SPSA", "2 evaluations total, approximate gradient"),
    ]

    for strategy, description in strategies:
        print(f"  • {strategy}: {description}")

gradient_cost_analysis()
```

## 6. Comparison with Other Methods

How parameter-shift compares to alternative gradient methods.

```python
import numpy as np

def method_comparison():
    """
    Compare parameter-shift with other gradient methods.
    """
    print("Gradient Method Comparison:")
    print("=" * 50)

    def test_circuit(params):
        """Test circuit with known gradient."""
        return np.cos(params[0]) * np.sin(params[1]) * np.exp(-params[2]**2)

    def analytical_gradient(params):
        """Analytical gradient for comparison."""
        return np.array([
            -np.sin(params[0]) * np.sin(params[1]) * np.exp(-params[2]**2),
            np.cos(params[0]) * np.cos(params[1]) * np.exp(-params[2]**2),
            -2 * params[2] * np.cos(params[0]) * np.sin(params[1]) * np.exp(-params[2]**2)
        ])

    def parameter_shift(circuit, params, shift=np.pi/2):
        """Parameter-shift gradient."""
        grad = np.zeros(len(params))
        for i in range(len(params)):
            p_plus = params.copy(); p_plus[i] += shift
            p_minus = params.copy(); p_minus[i] -= shift
            grad[i] = 0.5 * (circuit(p_plus) - circuit(p_minus))
        return grad

    def finite_difference(circuit, params, epsilon=0.01):
        """Finite difference gradient."""
        grad = np.zeros(len(params))
        for i in range(len(params)):
            p_plus = params.copy(); p_plus[i] += epsilon
            p_minus = params.copy(); p_minus[i] -= epsilon
            grad[i] = (circuit(p_plus) - circuit(p_minus)) / (2 * epsilon)
        return grad

    def forward_difference(circuit, params, epsilon=0.01):
        """Forward difference gradient."""
        f0 = circuit(params)
        grad = np.zeros(len(params))
        for i in range(len(params)):
            p_plus = params.copy(); p_plus[i] += epsilon
            grad[i] = (circuit(p_plus) - f0) / epsilon
        return grad

    # Test
    params = np.array([0.5, 0.8, 0.3])
    true_grad = analytical_gradient(params)

    methods = {
        'Parameter-shift': parameter_shift(test_circuit, params),
        'Finite diff (ε=0.01)': finite_difference(test_circuit, params, 0.01),
        'Finite diff (ε=0.001)': finite_difference(test_circuit, params, 0.001),
        'Forward diff': forward_difference(test_circuit, params, 0.01),
    }

    print("\n1. Accuracy Comparison:")
    print("-" * 40)
    print(f"  True gradient: {true_grad.round(4)}")
    print()

    for name, grad in methods.items():
        error = np.linalg.norm(grad - true_grad)
        print(f"  {name}:")
        print(f"    Gradient: {grad.round(4)}")
        print(f"    Error: {error:.6f}")

    print("\n\n2. Method Properties:")
    print("-" * 40)

    properties = [
        ("Method", "Exact?", "Evals", "Hardware"),
        ("Parameter-shift", "Yes", "2n", "Yes"),
        ("Finite diff", "No", "2n", "Yes"),
        ("Forward diff", "No", "n+1", "Yes"),
        ("Backprop", "Yes", "1", "No (sim only)"),
    ]

    for row in properties:
        print(f"  {row[0]:<18} {row[1]:<8} {row[2]:<8} {row[3]:<12}")

method_comparison()
```

## Summary

The parameter-shift rule is fundamental to quantum gradient computation:

1. **Core Formula**: ∂f/∂θ = (1/2)[f(θ+π/2) - f(θ-π/2)]
2. **Exactness**: Gives exact gradients, not approximations
3. **Hardware Compatible**: Works on real quantum devices
4. **Cost**: 2n circuit evaluations for n parameters
5. **Generalization**: Extended rules for non-Pauli gates
6. **Comparison**: More accurate than finite differences

Parameter-shift is the workhorse of variational quantum algorithm optimization.
