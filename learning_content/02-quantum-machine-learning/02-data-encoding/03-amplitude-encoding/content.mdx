## 1. The Amplitude Encoding Concept

Amplitude encoding stores classical data in the probability amplitudes of a quantum state.

<Definition term="Amplitude Encoding">
A mapping of a normalized classical vector x ∈ ℝᴺ to a quantum state |ψ(x)⟩ = Σᵢ xᵢ|i⟩, where each vector component becomes an amplitude of the corresponding basis state.
</Definition>

<Formula>
$$|ψ(x)\rangle = \sum_{i=0}^{N-1} x_i |i\rangle, \quad \text{where } \sum_{i=0}^{N-1} |x_i|^2 = 1$$
</Formula>

```python
import numpy as np

def amplitude_encode(data):
    """
    Encode a classical vector as quantum amplitudes.

    Args:
        data: Array of real numbers

    Returns:
        Normalized quantum state, number of qubits used
    """
    data = np.array(data, dtype=float)
    n = len(data)

    # Calculate qubits needed (pad to power of 2)
    n_qubits = int(np.ceil(np.log2(n))) if n > 1 else 1
    dim = 2 ** n_qubits

    # Pad data with zeros if needed
    padded = np.zeros(dim)
    padded[:n] = data

    # Normalize to unit vector
    norm = np.linalg.norm(padded)
    if norm < 1e-10:
        raise ValueError("Cannot encode zero vector")

    state = padded / norm

    return state, n_qubits

# Example
data = [3, 4, 0, 0]  # 3-4-5 triangle, padded
state, n_qubits = amplitude_encode(data)

print("Amplitude Encoding Example:")
print("=" * 50)
print(f"Original data: {data}")
print(f"Normalized state: {np.round(state, 4)}")
print(f"Qubits used: {n_qubits}")
print(f"Dimension: {len(state)}")
print(f"Verification |ψ|² = {np.sum(state**2):.6f}")
```

## 2. Exponential Compression

The key advantage: N values encoded in log₂(N) qubits.

```python
import numpy as np

def compression_analysis():
    """Analyze the compression achieved by amplitude encoding."""

    print("Amplitude Encoding Compression:")
    print("=" * 60)
    print(f"{'Data Size N':<15} {'Qubits Needed':<15} {'Compression Ratio':<20}")
    print("-" * 60)

    for n in [4, 16, 64, 256, 1024, 4096, 1_000_000]:
        qubits = int(np.ceil(np.log2(n)))
        ratio = n / qubits
        print(f"{n:<15,} {qubits:<15} {ratio:<20,.1f}x")

compression_analysis()

print("\n⚠️ CRITICAL CAVEAT:")
print("While storage is exponentially compressed,")
print("preparing the state may require O(N) operations!")
```

<Warning>
The exponential compression in qubits is real, but general amplitude encoding requires O(N) gates for N-dimensional vectors. This state preparation cost can eliminate the quantum advantage for many applications.
</Warning>

## 4. Simple Amplitude Encoding Implementation

A basic implementation for small vectors.

```python
import numpy as np

def prepare_amplitude_state_simple(amplitudes):
    """
    Prepare amplitude-encoded state using rotation gates.
    This is a simplified version for demonstration.

    For a 2-qubit system (4 amplitudes):
    |ψ⟩ = a₀|00⟩ + a₁|01⟩ + a₂|10⟩ + a₃|11⟩
    """
    amplitudes = np.array(amplitudes, dtype=float)
    n = len(amplitudes)

    if n not in [2, 4]:
        raise ValueError("This simple implementation only handles 2 or 4 amplitudes")

    # Normalize
    amplitudes = amplitudes / np.linalg.norm(amplitudes)

    if n == 2:
        # Single qubit: |ψ⟩ = a₀|0⟩ + a₁|1⟩
        # Use Ry(θ) where cos(θ/2) = a₀, sin(θ/2) = a₁
        theta = 2 * np.arctan2(amplitudes[1], amplitudes[0])
        return {'qubits': 1, 'rotations': [('Ry', 0, theta)]}

    elif n == 4:
        # Two qubits: more complex decomposition
        # First split into |0⟩ and |1⟩ subspaces
        norm_0 = np.sqrt(amplitudes[0]**2 + amplitudes[1]**2)
        norm_1 = np.sqrt(amplitudes[2]**2 + amplitudes[3]**2)

        # First rotation on qubit 0
        theta_0 = 2 * np.arctan2(norm_1, norm_0) if norm_0 > 1e-10 else np.pi

        # Conditional rotations on qubit 1
        if norm_0 > 1e-10:
            theta_1_cond0 = 2 * np.arctan2(amplitudes[1], amplitudes[0])
        else:
            theta_1_cond0 = 0

        if norm_1 > 1e-10:
            theta_1_cond1 = 2 * np.arctan2(amplitudes[3], amplitudes[2])
        else:
            theta_1_cond1 = 0

        return {
            'qubits': 2,
            'rotations': [
                ('Ry', 0, theta_0),
                ('CRy', (0, 1), theta_1_cond0),  # Controlled rotation
                ('X', 0, None),  # Flip control
                ('CRy', (0, 1), theta_1_cond1),
                ('X', 0, None)  # Flip back
            ]
        }

# Example
data = [1, 2, 3, 4]
result = prepare_amplitude_state_simple(data)
print("Amplitude Encoding Circuit (4 values):")
print("=" * 50)
print(f"Qubits needed: {result['qubits']}")
print("Gates:")
for gate in result['rotations']:
    if gate[2] is not None:
        print(f"  {gate[0]}({gate[2]:.4f}) on qubit(s) {gate[1]}")
    else:
        print(f"  {gate[0]} on qubit(s) {gate[1]}")
```

## 5. Verification and Fidelity

Verifying that amplitude encoding is correct.

```python
import numpy as np

def verify_amplitude_encoding(target_amplitudes, prepared_state):
    """
    Verify the quality of amplitude encoding.

    Returns fidelity between target and prepared states.
    """
    target = np.array(target_amplitudes, dtype=complex)
    target = target / np.linalg.norm(target)

    prepared = np.array(prepared_state, dtype=complex)
    prepared = prepared / np.linalg.norm(prepared)

    # Fidelity = |⟨target|prepared⟩|²
    fidelity = np.abs(np.vdot(target, prepared)) ** 2

    return fidelity

def build_state_from_circuit(n_qubits, gates):
    """Simulate circuit to build final state."""
    dim = 2 ** n_qubits
    state = np.zeros(dim, dtype=complex)
    state[0] = 1.0  # Start with |0...0⟩

    def Ry(theta):
        c, s = np.cos(theta/2), np.sin(theta/2)
        return np.array([[c, -s], [s, c]])

    X = np.array([[0, 1], [1, 0]])
    I = np.eye(2)

    for gate in gates:
        gate_type, qubit, param = gate

        if gate_type == 'Ry':
            op = Ry(param) if qubit == 0 else I
            for q in range(1, n_qubits):
                op = np.kron(op, Ry(param) if q == qubit else I)
            state = op @ state

        elif gate_type == 'X':
            op = X if qubit == 0 else I
            for q in range(1, n_qubits):
                op = np.kron(op, X if q == qubit else I)
            state = op @ state

    return state

# Test with simple example
target = [1, 1]  # Should give |+⟩
circuit = prepare_amplitude_state_simple(target)

print("Verification Example:")
print("=" * 50)
print(f"Target (normalized): {np.array(target)/np.linalg.norm(target)}")
print(f"Circuit: Ry({circuit['rotations'][0][2]:.4f})")
```

## 6. When to Use Amplitude Encoding

Guidelines for practical application.

```python
def amplitude_encoding_guidelines():
    """When is amplitude encoding appropriate?"""

    print("Amplitude Encoding Decision Guide:")
    print("=" * 60)

    scenarios = {
        "✓ GOOD Use Cases": [
            "Data is naturally normalized (probability distributions)",
            "Downstream algorithm queries amplitudes (quantum sampling)",
            "Data has special structure allowing efficient prep",
            "Running on fault-tolerant quantum computer with QRAM",
            "Inner products between vectors are the main operation"
        ],
        "✗ AVOID When": [
            "Data size N is very large and prep cost matters",
            "Need to access individual elements frequently",
            "Data changes frequently (re-encoding expensive)",
            "Working on NISQ device with limited gates",
            "Quantum advantage comes from other parts of algorithm"
        ],
        "Consider Alternatives": [
            "Angle encoding: if features << data points",
            "Basis encoding: if data is discrete",
            "Block encoding: for matrix representations",
            "Hybrid: encode only essential features"
        ]
    }

    for category, items in scenarios.items():
        print(f"\n{category}:")
        for item in items:
            print(f"  • {item}")

amplitude_encoding_guidelines()
```

## Summary

In this lesson, you learned about amplitude encoding:

1. **Concept**: Store N values in log₂(N) qubits as amplitudes
2. **Compression**: Exponential reduction in qubit count
3. **Cost**: General state preparation requires O(N) gates
4. **Methods**: General unitary, sparse, structured, QRAM
5. **Implementation**: Rotation-based preparation for small cases
6. **Verification**: Fidelity between target and prepared states
7. **Guidelines**: When amplitude encoding is practical

<Tip>
Amplitude encoding offers tantalizing exponential compression, but the state preparation cost is often the bottleneck. Consider it when you have structured data or when downstream algorithms specifically benefit from amplitude access.
</Tip>
