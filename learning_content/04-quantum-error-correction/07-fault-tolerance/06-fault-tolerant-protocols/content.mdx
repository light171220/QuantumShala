## Fault-Tolerant State Preparation

```python
class FaultTolerantStatePrep:
    """
    Fault-tolerant preparation of logical states.
    """

    def __init__(self, code_distance: int):
        self.d = code_distance

    def prepare_zero_logical(self):
        """Prepare |0_L⟩ fault-tolerantly."""
        print("""
Fault-Tolerant |0_L⟩ Preparation
════════════════════════════════

METHOD 1: Direct Encoding with Verification

    Step 1: Prepare physical |000...0⟩
    Step 2: Apply encoding circuit
    Step 3: Measure all stabilizers
    Step 4: If any syndrome ≠ 0: DISCARD and restart
    Step 5: Repeat syndrome measurement d times for verification

    Why verify? Encoding circuit might introduce errors
    Syndrome = 0 for d rounds → high confidence in |0_L⟩

METHOD 2: Steane Preparation (CSS codes)

    For CSS code with X stabilizers Sₓ and Z stabilizers Sᵤ:

    Step 1: Prepare |+⟩^⊗n (eigenstate of X^⊗n)
    Step 2: Measure Z stabilizers → project to code space
    Step 3: Measure X stabilizers → verify no Z errors
    Step 4: Result is |0_L⟩ or |1_L⟩ (determined by Z_L measurement)

    |+⟩^⊗n ──[Measure Sᵤ]──[Measure Sₓ]──► |0_L⟩ or |1_L⟩

METHOD 3: State Injection with Distillation

    For non-stabilizer states (|T⟩, |A⟩):

    Step 1: Prepare noisy encoded state
    Step 2: Distill using magic state distillation
    Step 3: Verify fidelity through repeated tests
        """)

    def prepare_plus_logical(self):
        """Prepare |+_L⟩ fault-tolerantly."""
        print("""
Fault-Tolerant |+_L⟩ Preparation
════════════════════════════════

OPTION 1: Apply logical H to |0_L⟩
    If H is transversal: Apply H^⊗n to |0_L⟩
    Result: |+_L⟩

OPTION 2: Direct Preparation (CSS codes)

    Step 1: Prepare |0⟩^⊗n
    Step 2: Measure X stabilizers → project to X eigenspace
    Step 3: Measure Z stabilizers to verify
    Step 4: Result is |+_L⟩ or |−_L⟩

OPTION 3: Lattice Surgery (surface codes)

    Grow a new patch from boundary
    New patch initialized in |+_L⟩
    Verify with d rounds of syndrome measurement
        """)


state_prep = FaultTolerantStatePrep(5)
state_prep.prepare_zero_logical()
state_prep.prepare_plus_logical()
```

class QECCycle:
    """
    Implement the quantum error correction cycle.
    """

    def __init__(self, code_distance: int, num_qubits: int):
        self.d = code_distance
        self.n = num_qubits

    def explain_cycle(self):
        """Explain the QEC cycle."""
        print(f"""
The Quantum Error Correction Cycle
══════════════════════════════════

One QEC cycle for distance-{self.d} code:

    ┌─────────────────────────────────────────────────────────┐
    │                     QEC CYCLE                           │
    │                                                         │
    │  ┌──────────┐   ┌──────────┐   ┌──────────┐            │
    │  │ Syndrome │   │ Syndrome │   │ Syndrome │  ... (d    │
    │  │ Round 1  │ → │ Round 2  │ → │ Round 3  │      rounds)│
    │  └──────────┘   └──────────┘   └──────────┘            │
    │       │              │              │                   │
    │       ▼              ▼              ▼                   │
    │  ┌────────────────────────────────────────────────────┐│
    │  │              DECODER                               ││
    │  │  Collect d rounds → Build 3D syndrome → Match      ││
    │  └────────────────────────────────────────────────────┘│
    │                         │                               │
    │                         ▼                               │
    │  ┌────────────────────────────────────────────────────┐│
    │  │         UPDATE PAULI FRAME                         ││
    │  │  (or apply physical corrections)                   ││
    │  └────────────────────────────────────────────────────┘│
    └─────────────────────────────────────────────────────────┘

SYNDROME ROUND DETAILS:
    1. Prepare ancilla qubits
    2. Apply syndrome extraction circuit (Shor-style or cat-style)
    3. Measure ancillas
    4. Record outcomes
    5. Reset ancillas for next round

TIMING CONSTRAINTS:
    - Must complete d rounds before errors accumulate
    - Decoder must keep up with incoming syndromes
    - Gate operations fit between QEC cycles
        """)

    def show_timeline(self):
        """Show QEC timeline with gates."""
        print("""
Timeline: Gates Interleaved with QEC
════════════════════════════════════

Time →
│
├─[QEC]─[QEC]─[QEC]─┤ Gate ├─[QEC]─[QEC]─[QEC]─┤ Gate ├─...
│                   │  G₁  │                   │  G₂  │
└───────────────────┴──────┴───────────────────┴──────┴───

Each [QEC] = one syndrome round
d rounds between gates for distance-d code

GATE TIMING:
    Transversal gates: 1 physical gate layer
    Lattice surgery: O(d) syndrome rounds
    T gate (via injection): 1 gate + magic state prep

PIPELINING:
    While running gate, decoder processes previous syndrome
    Overlapping computation and correction
        """)


qec = QECCycle(5, 17)  # d=5 surface code with 17 data qubits
qec.explain_cycle()
qec.show_timeline()

## Pauli Frame Tracking

```python
class PauliFrameTracking:
    """
    Track and manage Pauli frame during computation.
    """

    def __init__(self, num_logical_qubits: int):
        self.n = num_logical_qubits
        # Frame[i] = (x_bit, z_bit) for qubit i
        self.frame = [(0, 0) for _ in range(num_logical_qubits)]

    def explain_pauli_frame(self):
        """Explain Pauli frame tracking."""
        print("""
Pauli Frame Tracking
════════════════════

CONCEPT:
    Instead of physically applying corrections, track them classically
    The "Pauli frame" records which Paulis would have been applied

    Physical state: |ψ⟩
    Pauli frame: P = X^a Z^b
    Logical state: P|ψ⟩ = X^a Z^b |ψ⟩

WHY USE PAULI FRAME:
    1. Faster: No physical gates needed for corrections
    2. No new errors: Corrections don't introduce noise
    3. Efficient: Just update classical bits

HOW IT WORKS:
    - Decoder outputs: "apply X_L on qubit 3"
    - Don't apply X_L physically
    - Instead: update frame[3] to include X

FRAME REPRESENTATION:
    For each logical qubit, store (x, z) where:
    x = 1 means X correction pending
    z = 1 means Z correction pending

    Current correction for qubit i: X^frame[i].x × Z^frame[i].z
        """)

    def apply_correction(self, qubit: int, correction: str):
        """Update Pauli frame with correction."""
        x, z = self.frame[qubit]

        if correction == 'X':
            x = (x + 1) % 2
        elif correction == 'Z':
            z = (z + 1) % 2
        elif correction == 'Y':
            x = (x + 1) % 2
            z = (z + 1) % 2

        self.frame[qubit] = (x, z)
        return self.frame[qubit]

    def propagate_through_clifford(self, gate: str, qubits: list):
        """Update frame when Clifford gate is applied."""
        print(f"""
Frame Propagation Through {gate}
{'═' * 40}

When we apply gate G, the frame transforms:
    P|ψ⟩ → G P|ψ⟩ = (G P G†) G|ψ⟩

So frame P → G P G† (conjugate by gate)

Transformation rules:
    H: X ↔ Z
    S: X → Y = iXZ (so x-frame picks up z)
    CNOT: X₁ → X₁X₂, Z₂ → Z₁Z₂

Example (H on qubit 0 with frame X₀):
    Before: frame = [(1,0), (0,0), ...]
    After H: X₀ → Z₀
    After: frame = [(0,1), (0,0), ...]
        """)

    def final_correction(self, measurement_outcomes: dict):
        """Apply frame corrections to measurement outcomes."""
        print("""
Final Correction at Measurement
═══════════════════════════════

At end of computation, when measuring in Z basis:

    X frame: flips measurement outcome
    Z frame: no effect (Z eigenstate)

For qubit i measured with outcome m:
    If frame[i].x = 1: actual outcome = 1 - m
    If frame[i].x = 0: actual outcome = m

Z frame is accounted for (eigenvalue tracking) but doesn't flip result.
        """)


frame = PauliFrameTracking(5)
frame.explain_pauli_frame()
frame.propagate_through_clifford('H', [0])
frame.final_correction({})
```

## Complete Protocol Example

```
Complete Fault-Tolerant Protocol: Quantum Addition
═══════════════════════════════════════════════════

Task: Add two 4-bit quantum numbers |a⟩ + |b⟩ → |a⟩|a+b⟩

STEP 1: INITIALIZE
────────────────────
    Prepare 8 logical qubits for inputs a, b
    Prepare 4 logical ancilla qubits for carry/output

    For each logical qubit:
        - Encode in distance-5 surface code (17 data qubits each)
        - Verify with 5 syndrome rounds
        - Total: 12 × 17 = 204 data qubits + ancillas

STEP 2: RUN QEC CONTINUOUSLY
─────────────────────────────
    Start QEC cycles on all logical qubits
    Background process:
        syndrome → decoder → frame update

    Cycle time: ~1μs (for superconducting)
    Runs throughout computation

STEP 3: EXECUTE CIRCUIT
────────────────────────
    Addition circuit uses: CNOT, Toffoli gates

    For each gate:
        a) Wait for current QEC cycle to complete
        b) Apply gate:
           - CNOT: lattice surgery (5 syndrome rounds)
           - Toffoli: magic state injection
        c) Update Pauli frame based on gate
        d) Resume QEC

    Total gates: O(n) CNOTs + O(n) Toffolis

STEP 4: MEASURE OUTPUT
───────────────────────
    For output register:
        - Run final d syndrome rounds
        - Measure all data qubits in Z basis
        - Compute syndrome from measurements
        - Apply final error correction
        - Compute logical outcome from parities
        - Apply Pauli frame correction


RESOURCE SUMMARY:
    Physical qubits: ~2000-3000
    Gate time: ~100μs per logical CNOT
    Total time: ~1-10ms for 4-bit addition
    Success probability: > 99% (with p_physical ~ 0.001)
```

class RealTimeDecoding:
    """
    Manage real-time syndrome decoding.
    """

    def __init__(self, code_distance: int):
        self.d = code_distance
        self.syndrome_buffer = []

    def explain_real_time_requirements(self):
        """Explain real-time decoding requirements."""
        print("""
Real-Time Decoding Requirements
═══════════════════════════════

TIMING CONSTRAINT:
    Decoder must finish before next decision point

    Syndrome round time: T_syndrome
    Decoder latency: T_decode
    Gate time: T_gate

    Requirement: T_decode < d × T_syndrome (before next gate)

STRATEGIES:

1. PARALLEL DECODING
    Multiple decoders running in parallel
    Each handles different spatial region
    Combine results at boundaries

    ┌───────┐ ┌───────┐ ┌───────┐
    │Decoder│ │Decoder│ │Decoder│
    │   1   │ │   2   │ │   3   │
    └───┬───┘ └───┬───┘ └───┬───┘
        │         │         │
        └────────►└◄────────┘
               Merge

2. STREAMING DECODER
    Process syndromes as they arrive
    Don't wait for all d rounds
    Update correction as new info comes

    Round 1 → partial decode → update
    Round 2 → refine decode → update
    ...
    Round d → final decode → commit

3. PIPELINING
    While current cycle decodes, next cycle measures

    Time:  |----Measure Cycle i-----|
                       |----Decode Cycle i-1----|----Apply----|
        """)

    def decoder_pipeline(self):
        """Show decoder pipeline."""
        print("""
Decoder Pipeline Architecture
═════════════════════════════

Stage 1: SYNDROME COLLECTION
    - Receive measurement outcomes from hardware
    - Buffer d rounds of syndrome data
    - Detect changes between rounds

Stage 2: GRAPH CONSTRUCTION
    - Build 3D detection graph
    - Mark defect locations (syndrome changes)
    - Weight edges by error probability

Stage 3: MATCHING/DECODING
    - Run MWPM or Union-Find on graph
    - Find most likely error configuration
    - Output correction (Pauli string)

Stage 4: FRAME UPDATE
    - Convert correction to logical Pauli
    - Update Pauli frame bits
    - Ready for next cycle


LATENCY BUDGET (example: d=5, 1μs/round):
    Collection: 5μs (5 rounds)
    Graph build: 1μs
    Matching: 2-5μs
    Frame update: 0.1μs
    ─────────────────
    Total: ~8-11μs

Must be < time until next gate!
        """)


decoder = RealTimeDecoding(5)
decoder.explain_real_time_requirements()
decoder.decoder_pipeline()

## End-to-End Analysis

```python
class EndToEndAnalysis:
    """
    Analyze complete fault-tolerant computation.
    """

    def __init__(self):
        pass

    def error_budget(self, circuit_size: int, target_success: float):
        """Analyze error budget for computation."""
        import math

        p_fail_target = 1 - target_success
        p_logical_per_gate = p_fail_target / circuit_size

        print(f"""
Error Budget Analysis
═════════════════════

Circuit parameters:
    Total logical gates: {circuit_size:,}
    Target success probability: {target_success:.1%}

Error budget:
    Allowed failure probability: {p_fail_target:.2e}
    Per-gate logical error rate: {p_logical_per_gate:.2e}

For surface code:
    Logical error rate: p_L ≈ 0.1 × (p/p_th)^((d+1)/2)

    To achieve p_L = {p_logical_per_gate:.2e}:
        """)

        # Calculate required distance for various physical rates
        for p_phys in [0.01, 0.001, 0.0001]:
            p_th = 0.01  # ~1% threshold
            ratio = p_phys / p_th

            if ratio >= 1:
                print(f"    p = {p_phys}: Above threshold!")
                continue

            # Solve 0.1 * ratio^((d+1)/2) = p_logical
            target = p_logical_per_gate / 0.1
            log_target = math.log(target)
            log_ratio = math.log(ratio)

            d_needed = 2 * (log_target / log_ratio) - 1
            d_needed = int(math.ceil(d_needed))
            if d_needed % 2 == 0:
                d_needed += 1

            print(f"    p = {p_phys}: Need d ≥ {d_needed}")

    def resource_estimate(self, num_logical_qubits: int, distance: int):
        """Estimate total physical resources."""
        # Surface code: ~2d² data qubits per logical
        data_qubits = num_logical_qubits * 2 * distance * distance
        ancilla_qubits = data_qubits  # Roughly equal ancillas
        total = data_qubits + ancilla_qubits

        print(f"""
Resource Estimate
═════════════════

Configuration:
    Logical qubits: {num_logical_qubits}
    Code distance: {distance}
    Code: Surface code

Physical resources:
    Data qubits per logical: ~{2*distance*distance}
    Ancilla qubits per logical: ~{2*distance*distance}
    Total per logical: ~{4*distance*distance}

    TOTAL PHYSICAL QUBITS: ~{total:,}

Additional requirements:
    Classical processing: {num_logical_qubits} parallel decoders
    Control electronics: ~{total // 10} channels (with multiplexing)
    Cryogenic cooling: ~10-50 mW heat load
        """)


analysis = EndToEndAnalysis()
analysis.error_budget(circuit_size=10**6, target_success=0.99)
analysis.resource_estimate(num_logical_qubits=100, distance=15)
```

## Summary

Fault-tolerant quantum computing protocols integrate all components:

1. **State Preparation**: Verified encoding with syndrome checking
2. **QEC Cycles**: Continuous syndrome extraction and correction
3. **Pauli Frame**: Classical tracking of corrections
4. **Gate Execution**: Interleaved with QEC rounds
5. **Real-Time Decoding**: Low-latency syndrome processing
6. **Measurement**: Final error correction and outcome extraction

These protocols enable reliable quantum computation despite physical errors.
