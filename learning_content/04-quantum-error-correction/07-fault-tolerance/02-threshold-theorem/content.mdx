# The Threshold Theorem

## Introduction

The threshold theorem is one of the most important results in quantum computing theory. It guarantees that if physical error rates are below a certain threshold, arbitrarily accurate quantum computation is possible with a polynomial overhead. This lesson explores the theorem, its proof, and practical implications.

## Statement of the Theorem

```
The Quantum Threshold Theorem
═════════════════════════════

INFORMAL STATEMENT:
    If the physical error rate p is below a threshold value p_th,
    then arbitrarily long quantum computations can be performed
    with arbitrarily low logical error rate.

FORMAL STATEMENT:
    There exists a threshold p_th > 0 such that for any ε > 0
    and any quantum circuit C of size L, there exists a fault-
    tolerant circuit C' that:

    1. Simulates C with error probability < ε
    2. Has size |C'| = O(L · polylog(L/ε))
    3. Works whenever the physical error rate p < p_th

KEY IMPLICATIONS:
┌─────────────────────────────────────────────────────────────────┐
│  • Reliable quantum computing IS possible (not obvious!)       │
│  • Overhead is only polylogarithmic in accuracy                │
│  • The threshold p_th is a universal constant for each scheme  │
│  • Different codes/methods have different thresholds           │
└─────────────────────────────────────────────────────────────────┘

Historical note:
    Proved independently by multiple groups ~1996-1997
    Shor, Knill, Laflamme, Zurek, Preskill, Aharonov, Ben-Or, ...
```

## Proof Sketch

class ThresholdProof:
    """
    Outline the proof of the threshold theorem.
    """

    def __init__(self):
        pass

    def proof_outline(self):
        """Provide proof outline."""
        print("""
Proof Outline: Threshold Theorem
════════════════════════════════

The proof uses CONCATENATED CODES:

STEP 1: Choose a base code
────────────────────────────
    Use a distance-3 code (e.g., Steane [[7,1,3]])
    This corrects any 1 error per code block

STEP 2: Show one level of encoding reduces errors
──────────────────────────────────────────────────
    Physical error rate: p
    After encoding + fault-tolerant QEC: p_L = c·p²

    c = constant depending on circuit complexity
    p_L < p whenever p < 1/c (this defines threshold!)

STEP 3: Concatenate codes
─────────────────────────
    Level 0: Physical qubits (error rate p)
    Level 1: Encode in [[7,1,3]] (error rate c·p²)
    Level 2: Encode level-1 in [[7,1,3]] (error rate c·(c·p²)²)
    ...
    Level k: Error rate (c·p)^(2^k) / c

    If c·p < 1, this decreases DOUBLY EXPONENTIALLY!

STEP 4: Choose concatenation level
────────────────────────────────────
    To achieve logical error ε with circuit of length L:
    Need (c·p)^(2^k) < ε/L
    Solving: k = O(log log(L/ε))
    Total qubits: 7^k = (L/ε)^(O(log 7 / log 2)) = polylog

STEP 5: Analyze overhead
────────────────────────
    Each level adds factor ~7 in qubit count
    Each level adds factor ~O(1) in time
    Total: polynomial in log(1/ε)
        """)

## Noise Models and Thresholds

```
Different Noise Models Give Different Thresholds
═════════════════════════════════════════════════

CODE CAPACITY (idealized):
──────────────────────────
    Perfect syndrome measurement
    Only data qubit errors
    Highest theoretical threshold

    Surface code: p_th ≈ 10.9%
    Color code:   p_th ≈ 10%

PHENOMENOLOGICAL:
─────────────────
    Syndrome measurements have errors
    But gates are still perfect
    More realistic

    Surface code: p_th ≈ 2.9%

CIRCUIT-LEVEL (most realistic):
───────────────────────────────
    Each gate location can fail
    Includes 1Q, 2Q, prep, measure errors
    What matters for real hardware

    Surface code: p_th ≈ 0.8-1.1%
    Color code:   p_th ≈ 0.1%
    Concatenated: p_th ≈ 0.01%

DEPOLARIZING NOISE MODEL:
─────────────────────────
    Each location: apply random Pauli with prob p
    E(ρ) = (1-p)ρ + (p/3)(XρX + YρY + ZρZ)

BIASED NOISE:
─────────────
    Z errors more likely than X (or vice versa)
    Can achieve higher threshold with tailored codes
    p_th up to 50% for pure Z noise!

Threshold comparison:
┌─────────────────────────┬────────────┬───────────────┐
│ Code + Noise Model      │ Threshold  │ Notes         │
├─────────────────────────┼────────────┼───────────────┤
│ Surface + code capacity │ 10.9%      │ Theoretical   │
│ Surface + circuit       │ 0.8-1.1%   │ Practical     │
│ Color + circuit         │ 0.1%       │ Lower         │
│ Concatenated + circuit  │ 0.01%      │ Much lower    │
│ Surface + biased        │ ~40%       │ Z-biased only │
└─────────────────────────┴────────────┴───────────────┘
```

## Threshold Calculation

```python
import numpy as np

class ThresholdCalculation:
    """
    Calculate and analyze threshold values.
    """

    def __init__(self):
        pass

    def concatenation_threshold(self, c: float, levels: int = 5):
        """
        Calculate error rate through concatenation levels.

        Args:
            c: Circuit complexity constant
            levels: Number of concatenation levels
        """
        p_threshold = 1 / c

        print(f"\nConcatenation Analysis (c = {c})")
        print(f"Threshold: p_th = 1/c = {p_threshold:.4f}")
        print("=" * 60)

        for p in [0.1, 0.01, 0.001, 0.0001]:
            if p >= p_threshold:
                print(f"\np = {p:.4f} (ABOVE threshold - fails)")
                continue

            print(f"\np = {p:.4f} (below threshold)")
            error_rate = p

            for level in range(levels + 1):
                if level == 0:
                    print(f"  Level {level}: {error_rate:.2e} (physical)")
                else:
                    error_rate = c * error_rate ** 2
                    print(f"  Level {level}: {error_rate:.2e}")

                if error_rate < 1e-20:
                    print(f"  ... (negligible)")
                    break

    def resources_for_target(self, p: float, c: float, target_error: float):
        """Calculate resources needed for target error."""
        p_th = 1 / c

        if p >= p_th:
            return None, None

        # Find required levels
        # (c*p)^(2^k) / c < target
        # 2^k * log(c*p) < log(c * target)
        # k > log2(log(c*target) / log(c*p))

        log_ratio = np.log(c * target_error) / np.log(c * p)
        k = int(np.ceil(np.log2(log_ratio)))
        k = max(1, k)

        # Qubits per logical = 7^k for Steane code
        qubits = 7 ** k

        return k, qubits


calc = ThresholdCalculation()
calc.concatenation_threshold(c=100, levels=4)
```

## Below vs Above Threshold

```
Behavior Below and Above Threshold
══════════════════════════════════

BELOW THRESHOLD (p < p_th):
───────────────────────────
    Error rate DECREASES with code size
    Logical error: p_L ∝ (p/p_th)^((d+1)/2)

    Example: p = 0.1%, p_th = 1%

    d=3:  p_L ≈ 0.01  (10^-2)
    d=5:  p_L ≈ 0.001 (10^-3)
    d=7:  p_L ≈ 0.0001 (10^-4)
    ...
    As d→∞: p_L → 0

    Arbitrarily low error is achievable!

ABOVE THRESHOLD (p > p_th):
───────────────────────────
    Error rate INCREASES with code size
    Larger codes make things WORSE

    Example: p = 2%, p_th = 1%

    d=3:  p_L ≈ 0.04  (worse than p!)
    d=5:  p_L ≈ 0.08
    d=7:  p_L ≈ 0.16
    ...
    As d→∞: p_L → 0.5 (random guessing)

    Error correction provides no benefit!

AT THRESHOLD (p = p_th):
─────────────────────────
    Error rate stays constant
    p_L ≈ p regardless of code size
    Critical point / phase transition

Visual representation:

    p_L
     │
 0.5─┤                  ╱ p > p_th
     │                ╱
     │              ╱
     │            ╱
     │          ╱
     │        ╱
     │      ╱
     │    ● ← p = p_th
     │   ╲
   0─┼────╲───────────── p < p_th
     └────────────────────→ code distance d
```

   ────────────────────
   Know the overhead for any target accuracy

   Polynomial overhead means it's feasible

4. CODE SELECTION

## Summary

The threshold theorem is foundational to quantum computing:

1. **Statement**: Below threshold p_th, arbitrarily accurate QC is possible
2. **Proof method**: Concatenated codes with recursive error reduction
3. **Overhead**: Polylogarithmic in target accuracy
4. **Noise models**: Different models give different thresholds
5. **Practical value**: Sets engineering targets for hardware development

The theorem transforms quantum error correction from theoretical curiosity to practical engineering goal.
