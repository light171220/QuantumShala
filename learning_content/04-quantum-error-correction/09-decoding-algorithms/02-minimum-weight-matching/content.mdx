# Minimum Weight Perfect Matching

## Introduction

Minimum Weight Perfect Matching (MWPM) is the most widely used decoder for surface codes. It models errors as paths between syndrome defects and finds the correction that minimizes the total path length (error probability).

## The Matching Graph

```
Building the Matching Graph
в•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җ

STEP 1: Identify Defects
    Defect = syndrome check that flipped (changed from 0 to 1 or 1 to 0)

    Surface code syndrome:
    в”Ңв”Җв”Җв”Җв”¬в”Җв”Җв”Җв”¬в”Җв”Җв”Җв”¬в”Җв”Җв”Җв”җ
    в”Ӯ 0 в”Ӯ 0 в”Ӯ 1 в”Ӯ 0 в”Ӯ   вҶҗ Z-check syndrome
    в”ңв”Җв”Җв”Җв”јв”Җв”Җв”Җв”јв”Җв”Җв”Җв”јв”Җв”Җв”Җв”Ө
    в”Ӯ 0 в”Ӯ 1 в”Ӯ 0 в”Ӯ 0 в”Ӯ
    в”ңв”Җв”Җв”Җв”јв”Җв”Җв”Җв”јв”Җв”Җв”Җв”јв”Җв”Җв”Җв”Ө
    в”Ӯ 0 в”Ӯ 0 в”Ӯ 0 в”Ӯ 0 в”Ӯ
    в””в”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”ҳ

    Defects at: (0,2) and (1,1) вҶ’ mark as nodes

STEP 2: Build Complete Graph
    - Each defect is a node
    - Add edge between every pair of defects
    - Edge weight = distance (or -log probability)

    в—Ҹв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв—Ҹ
    (0,2)       (1,1)

    Weight = Manhattan distance = |0-1| + |2-1| = 2

STEP 3: Add Boundary Nodes
    - Errors can connect defect to boundary
    - Virtual boundary nodes for each defect
    - Distance = distance to nearest boundary

    в—Ҹв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв—Ҹ
    в”Ӯ           в”Ӯ
    в—Ӣ           в—Ӣ   (virtual boundary nodes)


COMPLETE GRAPH:
    For m defects: m(m-1)/2 edges + m boundary edges
```

## The MWPM Problem

```python
class MWPMProblem:
    """
    Explain the Minimum Weight Perfect Matching problem.
    """

    def __init__(self):
        pass

    def problem_definition(self):
        """Define the MWPM problem."""
        print("""
Minimum Weight Perfect Matching
в•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җ

GIVEN:
    Graph G = (V, E) with edge weights w(e)

FIND:
    Matching M вҠҶ E such that:
    1. Every vertex is matched exactly once (perfect matching)
    2. Total weight ОЈ w(e) for e вҲҲ M is minimized


FOR DECODING:
    Vertices = defects (+ boundary nodes)
    Edges = possible error chains between defects
    Weights = number of errors (or -log probability)

    MWPM finds lowest-weight error configuration!


EXAMPLE:
    4 defects: A, B, C, D

         3
    A в—Ҹв”Җв”Җв”Җв”Җв”Җв—Ҹ B
      в”Ӯ\\   /в”Ӯ
    2 в”Ӯ \\ / в”Ӯ 4
      в”Ӯ  X  в”Ӯ
    5 в”Ӯ / \\ в”Ӯ 1
      в”Ӯ/   \\в”Ӯ
    C в—Ҹв”Җв”Җв”Җв”Җв”Җв—Ҹ D
         2

    Possible matchings:
    {A-B, C-D}: weight = 3 + 2 = 5
    {A-C, B-D}: weight = 2 + 4 = 6
    {A-D, B-C}: weight = 5 + 1 = 6

    MWPM = {A-B, C-D} with weight 5
        """)

    def why_matching(self):
        """Explain why matching works for surface codes."""
        print("""
Why Matching Works for Surface Codes
в•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җ

KEY INSIGHT: Errors create PAIRS of defects

Single Z error:
    в—Ӣв”Җв”Җв”Җв—Ӣв”Җв”Җв”Җв—Ӣ
    в”Ӯ S в”Ӯ S в”Ӯ
    в—Ӣв”Җв”Җв”Җв—Ҹв”Җв”Җв”Җв—Ӣ   вҶҗ Z error on this edge
    в”Ӯ S в”Ӯ   в”Ӯ
    в—Ӣв”Җв”Җв”Җв—Ӣв”Җв”Җв”Җв—Ӣ

    Creates 2 adjacent defects (S checks fail)

Error chain:
    в—Ӣв”Җв”Җв”Җв—Ӣв”Җв”Җв”Җв—Ӣ
    в”Ӯ S в”Ӯ   в”Ӯ
    в—Ӣв”Җв”Җв”Җв—Ҹв”Җв”Җв”Җв—Ҹ   вҶҗ Z errors along path
    в”Ӯ S в”Ӯ S в”Ӯ
    в—Ӣв”Җв”Җв”Җв—Ӣв”Җв”Җв”Җв—Ӣ

    Defects at ENDS of chain only!

MATCHING INTERPRETATION:
    Each matching edge = error chain
    Matched pair = chain endpoints
    Weight = chain length

    MWPM finds shortest chains explaining all defects!

BOUNDARY:
    Chain can end at boundary (single defect)
    Boundary node represents this case
        """)
```

## The Blossom Algorithm

```python
class BlossomAlgorithm:
    """
    Explain Edmonds' Blossom algorithm for MWPM.
    """

    def __init__(self):
        pass

    def algorithm_overview(self):
        """High-level overview of Blossom algorithm."""
        print("""
Edmonds' Blossom Algorithm
в•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җ

COMPLEXITY: O(VВі) for V vertices

KEY IDEAS:

1. AUGMENTING PATHS
    Start with empty matching
    Find "augmenting path" that increases matching size
    Repeat until perfect matching

    Augmenting path:
    unmatched в”Җв”Җв”Җв”Җ matched в”Җв”Җв”Җв”Җ unmatched
    vertex        edge          vertex
          в””в”Җв”Җв”Җв”Җ unmatched в”Җв”Җв”Җв”Җв”ҳ
                  edge

2. BLOSSOMS (the tricky part!)
    Odd cycles create complications
    "Blossom" = odd cycle in graph
    Contract blossom to single vertex, solve, expand

    в—Ҹв”Җв”Җв”Җв”Җв”Җв—Ҹ
     \\   /
      \\ /
       в—Ҹ      вҶҗ Blossom (odd cycle)
       в”Ӯ
       в—Ҹ

    Contract to:
       в—Ӣ      вҶҗ Single vertex
       в”Ӯ
       в—Ҹ

3. DUAL VARIABLES
    Track "dual weights" for vertices and blossoms
    Ensure matching is optimal via complementary slackness


ALGORITHM STEPS:

    while not perfect_matching:
        find augmenting path (handling blossoms)
        augment matching along path

    return matching
        """)

    def implementation_notes(self):
        """Practical implementation considerations."""
        print("""
Implementation Notes
в•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җ

OPTIMIZED IMPLEMENTATIONS:

1. SPARSE GRAPHS
    Surface code graphs are sparse in syndrome patterns
    Don't need all O(VВІ) edges
    Use Dijkstra to find relevant edges

2. PYMATCHING LIBRARY
    Highly optimized Python implementation
    Handles 3D (space-time) decoding
    Threshold ~1% for surface codes

    from pymatching import Matching
    matching = Matching(check_matrix)
    correction = matching.decode(syndrome)

3. FUSION BLOSSOM
    Parallel Blossom algorithm
    Divides graph, solves in parallel, fuses solutions
    Near-linear time in practice


PERFORMANCE:

    Typical surface code (d=10):
    ~20 defects average
    Decode time: ~10-100 Ојs

    Scales as O(dВІ Г— Оұ(dВІ)) in practice
    Оұ = inverse Ackermann (essentially constant)
```

## Decoder Implementation

```python
class MWPMDecoder:
    """
    Sketch of MWPM decoder implementation.
    """

    def __init__(self, distance: int, error_rate: float):
        self.d = distance
        self.p = error_rate

    def build_graph(self, syndrome_history: list):
        """Build matching graph from syndrome history."""
        # Pseudocode
        print("""
Building Matching Graph
в•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җ

def build_graph(syndrome_history):
    defects = []

    # Find all defects (syndrome changes)
    for t, syndrome in enumerate(syndrome_history):
        if t == 0:
            prev = zeros_like(syndrome)
        else:
            prev = syndrome_history[t-1]

        for i, (s, p) in enumerate(zip(syndrome, prev)):
            if s != p:  # Syndrome changed
                defects.append((i, t))  # (space, time)

    # Build complete graph
    graph = Graph()
    for d1, d2 in all_pairs(defects):
        weight = distance(d1, d2)  # Manhattan in 3D
        graph.add_edge(d1, d2, weight)

    # Add boundary edges
    for d in defects:
        boundary_dist = distance_to_boundary(d)
        graph.add_boundary_edge(d, boundary_dist)

    return graph
        """)

    def decode(self, syndrome_history: list):
        """Run MWPM decoder."""
        print("""
Decoding with MWPM
в•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җ

def decode(syndrome_history):
    # Step 1: Build graph
    graph = build_graph(syndrome_history)

    # Step 2: Run MWPM
    matching = minimum_weight_perfect_matching(graph)

    # Step 3: Convert matching to correction
    correction = matching_to_pauli(matching)

    return correction


def matching_to_pauli(matching):
    # Each matched pair defines an error chain
    correction = identity()

    for (d1, d2) in matching:
        if d2 is boundary:
            chain = path_to_boundary(d1)
        else:
            chain = path_between(d1, d2)

        for edge in chain:
            correction *= error_on_edge(edge)

    return correction
        """)


decoder = MWPMDecoder(5, 0.001)
decoder.build_graph([])
decoder.decode([])
```

```
THRESHOLD:
    MWPM on surface code: p_th вүҲ 1.0-1.1%

    Comparison:
    - Optimal decoder: ~1.1%

    Blossom: O(VВі) worst case

LATENCY:
    Modern implementations: 1-10 Ојs for dвүӨ15
    Scales well with hardware acceleration

PROS:
    + Near-optimal accuracy
    + Well-understood theory

CONS:

    - Cubic worst-case complexity
```

## Summary

MWPM is the standard for surface code decoding:

1. **Graph Construction**: Defects become nodes, error chains become edges
2. **MWPM Problem**: Find lowest-weight perfect matching
3. **Blossom Algorithm**: O(VВі) algorithm handles odd cycles
4. **3D Extension**: Handles measurement errors with temporal edges
5. **Performance**: ~1% threshold, efficient in practice

MWPM provides near-optimal decoding at reasonable computational cost.
