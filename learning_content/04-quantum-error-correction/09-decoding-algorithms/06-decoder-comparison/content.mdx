# Decoder Comparison and Selection

## Introduction

Choosing the right decoder involves balancing accuracy, speed, scalability, and implementation complexity. This lesson provides a comprehensive comparison of decoding algorithms and guidelines for selecting the best decoder for specific applications.

    def threshold_breakdown(self):
        """Analyze thresholds for different noise models."""
        print("""
Threshold Analysis by Noise Model
═════════════════════════════════

CODE CAPACITY (perfect measurements):
    Optimal:     ~10.9%
    MWPM:        ~10.3%
    Union-Find:  ~9.9%

PHENOMENOLOGICAL (measurement errors = data errors):
    Optimal:     ~3.3%
    MWPM:        ~2.9%
    Union-Find:  ~2.7%

CIRCUIT-LEVEL (realistic gate errors):
    Optimal:     ~1.1%
    MWPM:        ~1.0%
    Union-Find:  ~0.9%
    BP:          ~0.8%

OBSERVATIONS:
    1. Gap between decoders shrinks at lower thresholds
    2. Measurement errors have biggest impact
    3. All decoders benefit from better hardware


WHAT THRESHOLD MEANS IN PRACTICE:

    Physical error rate: p
    Code distance: d
    Logical error rate: p_L ≈ (p/p_th)^((d+1)/2)

    Example (p = 0.1%, d = 11):
        MWPM (p_th = 1.0%):  p_L ≈ (0.1)^6 = 10^-6
        UF (p_th = 0.9%):    p_L ≈ (0.11)^6 = 2×10^-6

    Difference is factor of 2, not dramatic!
        """)


threshold = ThresholdComparison()
threshold.threshold_breakdown()

## Hybrid Decoding Strategies

```
Hybrid Decoder Approaches
═════════════════════════

1. UF + MWPM REFINEMENT

    Phase 1: Run Union-Find (fast, approximate)
    Phase 2: If uncertain, run MWPM on subgraph

    ┌─────────────┐
    │   Syndrome  │
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │ Union-Find  │────────┐
    └──────┬──────┘        │
           │               │
    Confident?         Uncertain
           │               │
           ▼               ▼
    ┌─────────────┐   ┌─────────────┐
    │   Return    │   │    MWPM     │
    │ UF result   │   │  refinement │
    └─────────────┘   └──────┬──────┘
                             │
                             ▼
                      ┌─────────────┐
                      │   Return    │
                      │ MWPM result │
                      └─────────────┘

    Best of both: Usually fast, accurate when needed


2. NN + MATCHING

    Use neural network to predict edge weights
    Run matching on weighted graph

    Benefits: NN learns correlations, matching is optimal


3. PARALLEL + MERGE

    Divide code into regions
    Decode regions in parallel
    Merge at boundaries

    Enables scalability on multi-core systems


4. SLIDING WINDOW

    For 3D decoding, don't wait for all rounds
    Decode in overlapping windows

    ├──────────┤
       ├──────────┤
          ├──────────┤

    Lower latency than full 3D decode
```

## Summary

Choosing a decoder requires balancing multiple factors:

1. **Accuracy**: MWPM ≈ 1%, UF ≈ 0.9%, NN ≈ 0.8-0.9%
2. **Speed**: UF and NN fastest, MWPM needs optimization
3. **Scalability**: UF and NN scale well, MWPM struggles
4. **Complexity**: UF simplest, MWPM moderate, NN most complex
5. **Adaptability**: NN best for device-specific noise
6. **Hybrids**: Combine strengths of multiple approaches

The "best" decoder depends on your specific requirements and constraints.
