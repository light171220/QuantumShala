# Belief Propagation Decoding

## Introduction

Belief Propagation (BP) is a probabilistic decoding algorithm that passes messages on a factor graph to compute marginal probabilities of errors. While it can struggle with loopy graphs like those in surface codes, it excels for LDPC codes and serves as a building block for more sophisticated decoders.

## Message Passing Algorithm

```python
class BeliefPropagation:
    """
    Implement Belief Propagation for quantum codes.
    """

    def __init__(self, parity_check: list, error_rate: float):
        self.H = parity_check
        self.p = error_rate

    def explain_message_passing(self):
        """Explain the message passing algorithm."""
        print("""
Belief Propagation Message Passing
в•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җ

Two types of messages passed iteratively:

1. VARIABLE вҶ’ FACTOR messages (Ој)
    "What I believe about my error, considering other factors"

    Ој_{vвҶ’f}(e) вҲқ prior(e) Г— вҲҸ_{f' вү  f} ОҪ_{f'вҶ’v}(e)

    v = variable, f = factor, e = error type (I, X, Y, Z)


2. FACTOR вҶ’ VARIABLE messages (ОҪ)
    "What I (factor) tell you about your error"

    ОҪ_{fвҶ’v}(e) вҲқ вҲ‘_{e': compatible} вҲҸ_{v' вү  v} Ој_{v'вҶ’f}(e')

    Compatible: errors consistent with syndrome


ALGORITHM:

    1. Initialize:
       Ој_{vвҶ’f}(e) = prior(e)  (e.g., P(error) = p)

    2. Iterate:
       for each factor f:
           compute ОҪ_{fвҶ’v} from Ој_{v'вҶ’f}
       for each variable v:
           compute Ој_{vвҶ’f} from ОҪ_{f'вҶ’v}

    3. Converge:
       Repeat step 2 until messages stop changing
       (or max iterations)

    4. Decide:
       belief(v, e) = prior(e) Г— вҲҸ_f ОҪ_{fвҶ’v}(e)
       decoded_error[v] = argmax_e belief(v, e)
        """)

    def log_domain(self):
        """Explain log-domain implementation."""
        print("""
Log-Domain Belief Propagation
в•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җ

For numerical stability, work in log domain:

    log Ој_{vвҶ’f}(e) = log prior(e) + вҲ‘_{f' вү  f} log ОҪ_{f'вҶ’v}(e)

    Log-likelihood ratios (LLR):
    L_v = log(P(e=I) / P(e=X))

    Update equations become additions instead of multiplications.


BINARY SYMMETRIC CHANNEL (simpler case):

    For X-errors only with probability p:

    L_v = log((1-p)/p)  (prior LLR)

    Variable-to-check:
    L_{vвҶ’c} = L_v + вҲ‘_{c' вү  c} L_{c'вҶ’v}

    Check-to-variable (using tanh):
    tanh(L_{cвҶ’v}/2) = вҲҸ_{v' вү  v} tanh(L_{v'вҶ’c}/2)

    This is the "sum-product" algorithm in log domain.
        """)


bp = BeliefPropagation([[1,1,1,0], [0,1,1,1]], 0.01)
bp.explain_message_passing()
bp.log_domain()
```

## Handling Loops

```
The Loop Problem
в•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җ

IDEAL: BP is exact on trees (no loops)
REALITY: Quantum code factor graphs have loops!

SURFACE CODE EXAMPLE:

    в—Ӣв”Җв”Җв”Җв—Ӣв”Җв”Җв”Җв—Ӣ
    в”Ӯ   в”Ӯ   в”Ӯ
    в– в”Җв”Җв”Җв– в”Җв”Җв”Җв–     Loops everywhere!
    в”Ӯ   в”Ӯ   в”Ӯ
    в—Ӣв”Җв”Җв”Җв—Ӣв”Җв”Җв”Җв—Ӣ

    Messages cycle around loops
    Can fail to converge or give wrong answer

CONSEQUENCES:
    1. BP may not converge
    2. Even if converges, may not be optimal
    3. Threshold lower than optimal


MITIGATION STRATEGIES:

1. LIMITED ITERATIONS
    Stop after fixed number of iterations
    Prevents runaway oscillation

2. DAMPING
    New message = Оұ Г— computed + (1-Оұ) Г— old
    Smooths updates, improves convergence

3. SCHEDULING
    Order of message updates affects convergence
    "Residual" scheduling: update largest changes first

4. NEURAL BP
    Learn damping factors from data
    Can adapt to specific code structure
```

## BP Variants

```
Belief Propagation Variants
в•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җв•җ

1. SUM-PRODUCT (Standard)
    Computes marginal probabilities
    Soft outputs for each error type

2. MIN-SUM
    Approximation of sum-product
    Uses min instead of sum in log domain
    Faster but slightly less accurate

3. NORMALIZED/OFFSET MIN-SUM
    Corrects bias in min-sum
    Better accuracy, still fast

4. ORDERED STATISTICS DECODING (OSD)
    Post-processing after BP
    Fixes remaining errors
    Approaches ML performance


OSD OVERVIEW:

    1. Run BP to get soft information
    2. Sort bits by reliability
    3. Fix most reliable bits
    4. Exhaustively search remaining bits
    5. Return best codeword found

    BP + OSD achieves near-optimal for many codes!


5. QUATERNARY BP
    For quantum codes: handle X, Y, Z errors
    Messages are 4-dimensional (I, X, Y, Z)
    More complex but handles degeneracy better
```

## Summary

Belief Propagation offers probabilistic decoding through message passing:

1. **Factor Graphs**: Represent code structure as bipartite graph
2. **Message Passing**: Exchange beliefs between variables and factors
3. **Loop Challenge**: BP struggles with loopy graphs like surface codes
4. **Threshold**: ~50-80% of optimal due to loops
5. **Best For**: LDPC codes with sparse structure
6. **Hybrids**: BP + OSD, BP + MWPM for improved performance

BP is essential for quantum LDPC codes and provides probabilistic insights for other decoders.
