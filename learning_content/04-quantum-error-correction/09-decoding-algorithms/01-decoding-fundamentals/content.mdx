# Decoding Fundamentals

## Introduction

The decoder is the classical brain of a quantum error correction system. It takes syndrome measurements as input and determines what corrections to apply. This lesson introduces the decoding problem and the key considerations for practical decoders.

## The Decoding Problem

```
What Does a Decoder Do?
═══════════════════════

INPUT:  Syndrome measurements s = (s₁, s₂, ..., sₘ)
OUTPUT: Correction operator C (a Pauli string)

GOAL: Find C such that C · E = L or I
      (correction times error equals logical identity or trivial)

THE CHALLENGE:
    - Many different errors can produce the same syndrome
    - Must choose the MOST LIKELY error given the syndrome
    - Must do this FAST (real-time constraint)


VISUALIZATION:

    ┌──────────────────────────────────────────────────────────┐
    │                     ERROR                                │
    │                       E                                  │
    │                       │                                  │
    │                       ▼                                  │
    │   ┌─────────────────────────────────────┐               │
    │   │       SYNDROME EXTRACTION           │               │
    │   │       Measure stabilizers           │               │
    │   └─────────────────────────────────────┘               │
    │                       │                                  │
    │                       ▼                                  │
    │                 Syndrome s                               │
    │                       │                                  │
    │                       ▼                                  │
    │   ┌─────────────────────────────────────┐               │
    │   │           DECODER                    │               │
    │   │   s → argmax P(E|s)                 │               │
    │   └─────────────────────────────────────┘               │
    │                       │                                  │
    │                       ▼                                  │
    │                Correction C                              │
    │                       │                                  │
    │                       ▼                                  │
    │   ┌─────────────────────────────────────┐               │
    │   │       APPLY CORRECTION              │               │
    │   │       State → C × State             │               │
    │   └─────────────────────────────────────┘               │
    │                                                          │
    └──────────────────────────────────────────────────────────┘
```

## Syndrome to Error Mapping

```python
class SyndromeErrorMapping:
    """
    Understand the relationship between syndromes and errors.
    """

    def __init__(self):
        pass

    def syndrome_degeneracy(self):
        """Explain syndrome degeneracy."""
        print("""
Syndrome Degeneracy: Multiple Errors, Same Syndrome
════════════════════════════════════════════════════

For a code with n physical qubits:
    - 4ⁿ possible Pauli errors (I, X, Y, Z on each qubit)
    - 2^(n-k) possible syndromes (for [[n,k,d]] code)

DEGENERACY:
    Multiple errors → same syndrome

EXAMPLE (Surface code patch):

    ○───○───○
    │ Z │ Z │
    ○───●───○   ● = Z error location
    │   │   │
    ○───○───○

    Error E₁: Z on center qubit → syndrome: two adjacent Z-checks fail

    ○───○───○
    │ Z │   │
    ●───○───○   Now Z on left qubit
    │   │   │
    ○───○───○

    Error E₂: Z on left qubit → different syndrome (left check fails)

    BUT what about E₁ · E₂?

    ○───○───○
    │   │ Z │
    ●───●───○   Z on both
    │   │   │
    ○───○───○

    E₁ · E₂ has yet another syndrome!


THE KEY: Equivalent errors
    Two errors E and E' are EQUIVALENT if:
    E · E' = Stabilizer (trivial) or Logical (different logical state)

    Decoder must identify equivalence class, not exact error.
        """)

    def coset_picture(self):
        """Explain coset structure."""
        print("""
Coset Structure of Errors
═════════════════════════

Pauli group P on n qubits partitions into COSETS:

    P = S ∪ (X_L · S) ∪ (Z_L · S) ∪ (Y_L · S) ∪ ...

Where S is the stabilizer group.

COSET REPRESENTATIVES:
    - Identity coset S: trivial errors
    - Logical cosets X_L·S, Z_L·S, Y_L·S: logical errors
    - Other cosets: correctable errors

SYNDROME → COSET:
    Each syndrome identifies a coset
    Decoder picks representative from that coset

    ┌────────────────────────────────────────────────────┐
    │  Syndrome s = 101101                              │
    │            │                                       │
    │            ▼                                       │
    │  Coset = {E₁, E₂, E₃, ...}                       │
    │  All errors giving syndrome s                     │
    │            │                                       │
    │            ▼                                       │
    │  Pick representative: lowest weight error         │
    │  Or: most probable error                          │
    └────────────────────────────────────────────────────┘

DECODER'S TASK:
    Don't need exact error E
    Just need representative from correct coset
    (Identity coset if error is correctable)
        """)


mapping = SyndromeErrorMapping()
mapping.syndrome_degeneracy()
mapping.coset_picture()
```

## Decoder Types

```
Classification of Decoders
══════════════════════════

BY APPROACH:

1. COMBINATORIAL DECODERS
    - Model errors as graph problems
    - Use optimization algorithms
    - Examples: MWPM, Union-Find

2. PROBABILISTIC DECODERS
    - Model errors using probability theory
    - Bayesian inference on error configurations
    - Examples: Belief Propagation, MLE

3. MACHINE LEARNING DECODERS
    - Learn syndrome → correction mapping
    - Neural networks, reinforcement learning
    - Examples: CNN decoders, RL decoders


BY OPTIMALITY:

1. OPTIMAL DECODERS (Maximum Likelihood)
    - Find most probable error class
    - Best possible accuracy
    - Often computationally expensive

2. APPROXIMATE DECODERS
    - Trade accuracy for speed
    - May miss some correctable errors
    - Practical for real-time use


BY TIME BEHAVIOR:

1. OFFLINE DECODERS
    - Process full syndrome history at once
    - Can use all available information
    - Not suitable for real-time

2. SLIDING WINDOW DECODERS
    - Process syndromes in chunks
    - Bounded memory and latency
    - Practical for real-time

3. STREAMING DECODERS
    - Process syndromes as they arrive
    - Lowest latency
    - May have reduced accuracy
```

## Performance Metrics

```python
class DecoderMetrics:
    """
    Key metrics for evaluating decoder performance.
    """

    def __init__(self):
        pass

    def accuracy_metrics(self):
        """Define accuracy-related metrics."""
        print("""
Accuracy Metrics
════════════════

1. LOGICAL ERROR RATE (p_L)
    Probability that decoder fails to correctly identify
    error equivalence class

    p_L = (# logical errors) / (# decoding attempts)

    Target: p_L < 10⁻¹⁵ for useful computation

2. THRESHOLD
    Physical error rate below which increasing code distance
    reduces logical error rate

    p_phys < p_threshold → decoder works

    Typical thresholds:
    - MWPM on surface code: ~1%
    - Union-Find: ~0.9%
    - Optimal decoder: ~1.1%

3. THRESHOLD GAP
    Difference between code capacity and achieved threshold

    gap = p_threshold(optimal) - p_threshold(decoder)

    Smaller gap = better decoder
        """)

    def speed_metrics(self):
        """Define speed-related metrics."""
        print("""
Speed Metrics
═════════════

1. LATENCY
    Time from syndrome input to correction output

    Requirement: latency < d × syndrome_round_time
    (Must decode before next gate application)

    Example: d=5, 1μs rounds → latency < 5μs

2. THROUGHPUT
    Syndromes processed per unit time

    Must keep up with syndrome measurement rate
    For parallel decoding: syndromes/second

3. SCALING
    How latency/throughput changes with code distance

    Ideal: O(1) or O(d)
    Acceptable: O(d²) or O(d³)
    Problematic: O(exp(d))

4. PARALLELIZABILITY
    How well decoder scales with parallel hardware

    Important for large-scale systems
    MWPM: moderately parallelizable
    Union-Find: highly parallelizable
        """)

    def resource_metrics(self):
        """Define resource-related metrics."""
        print("""
Resource Metrics
════════════════

1. MEMORY
    Storage required for decoder operation

    Per-qubit: O(1) is ideal
    Global: O(d²) for surface code state

2. COMPUTE
    Classical operations per syndrome

    FPGAs: limited operations
    ASICs: custom hardware possible
    GPUs: high parallelism

3. HARDWARE COMPATIBILITY
    Can run on available hardware?

    FPGAs: low latency, limited compute
    CPUs: flexible, higher latency
    Custom ASICs: best, but expensive to develop
        """)


metrics = DecoderMetrics()
metrics.accuracy_metrics()
metrics.speed_metrics()
metrics.resource_metrics()
```

## Real-Time Decoding Requirements

```
Real-Time Decoding Constraints
══════════════════════════════

For practical quantum computing, decoder must operate in real-time.

TIMING CONSTRAINT:

    Syndrome round time: T_syn ≈ 1μs (superconducting)
    Code distance: d
    Time until next decision: d × T_syn

    Decoder latency must satisfy:
        T_decode < d × T_syn

    For d=5: T_decode < 5μs
    For d=11: T_decode < 11μs


PIPELINE ARCHITECTURE:

    Time →
    ├─Syn1─┼─Syn2─┼─Syn3─┼─Syn4─┼─Syn5─┤Gate├─...
    │      │      │      │      │      │    │
    │      └──────┴──────┴──────┴──────┘    │
    │                    │                   │
    │             Decode d rounds            │
    │                    │                   │
    │                    ▼                   │
    │              Correction                │
    │                    │                   │
    │                    └───────────────────┤
    │                                        │
    └────────────────────────────────────────┘

    Decode WHILE next round measures
    Correction ready by gate time


BACKLOG PROBLEM:

    If decode time > measure time:
        Syndromes accumulate faster than decoded
        Eventually run out of memory!

    Must ensure: average decode time ≤ average measure time


STRATEGIES FOR REAL-TIME:

1. Fast Algorithms
    - Union-Find: O(d² α(d²)) nearly linear
    - Optimized MWPM implementations

2. Parallel Decoding
    - Decode different spatial regions in parallel
    - Combine results at boundaries

3. Pipelining
    - Start decoding before all rounds complete
    - Update correction as new syndromes arrive

4. Hardware Acceleration
    - FPGAs for low latency
    - Custom ASICs for highest performance
```

print("""
Example Decoder Usage
═════════════════════

# Initialize
decoder = UnionFindDecoder(
    code_distance=5,
    error_model={'X': 0.001, 'Z': 0.001, 'meas': 0.001}
)

# Single round decoding

corrections = decoder.decode_batch(syndrome_history)
""")

## Summary

Decoders are essential for translating syndrome measurements into corrections:

1. **The Problem**: Map syndromes to most likely error class
2. **Degeneracy**: Many errors give same syndrome — find equivalence class
3. **Decoder Types**: Combinatorial, probabilistic, ML-based
4. **Metrics**: Logical error rate, threshold, latency, throughput
5. **Real-Time**: Must decode faster than syndromes are measured
