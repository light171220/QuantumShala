# Code Switching and Deformation

## Introduction

Different quantum codes support different transversal gates. Code switching allows us to transfer encoded information between codes to access gates that aren't available in a single code. This lesson covers techniques for switching codes while maintaining fault tolerance.

## Motivation for Code Switching

```
Why Switch Codes?
═════════════════

THE PROBLEM (Eastin-Knill):
    No single code has transversal universal gates

CODE CAPABILITIES:
    [[7,1,3]] Steane:     Transversal H, S, CNOT (Clifford)
    [[15,1,3]] Reed-Muller: Transversal T, H, CNOT (no S!)
    Surface code:         Transversal CNOT only

THE IDEA:
    Use Steane code for Clifford gates
    Switch to Reed-Muller for T gate
    Switch back for more Cliffords

    ┌───────────────────────────────────────────────────────────┐
    │  Steane code          Reed-Muller         Steane code     │
    │  (Cliffords)    →→→    (T gate)    →→→    (Cliffords)    │
    │                                                           │
    │  H, S, CNOT           Apply T             H, S, CNOT     │
    └───────────────────────────────────────────────────────────┘

ADVANTAGES:
    - No magic state distillation overhead
    - Direct transversal T (cleaner)
    - Different resource trade-offs

CHALLENGES:
    - Switching must be fault-tolerant
    - Codes have different structures
    - Overhead of switching process
```

## Fault-Tolerant Code Switching

```python
class CodeSwitching:
    """
    Implement fault-tolerant code switching.
    """

    def __init__(self):
        pass

    def steane_to_reed_muller(self):
        """Switch from Steane to Reed-Muller code."""
        print("""
Steane [[7,1,3]] to Reed-Muller [[15,1,3]]
══════════════════════════════════════════

Both codes are CSS codes built from classical codes:
    Steane: [7,4,3] Hamming code
    Reed-Muller: [15,11,3] Reed-Muller code

RELATIONSHIP:
    The Steane code can be EMBEDDED in Reed-Muller!
    [7,4,3] ⊂ [15,11,3] (as subcodes)

SWITCHING PROTOCOL:

Step 1: Start with Steane-encoded |ψ_L⟩ (7 qubits)

    q₀ q₁ q₂ q₃ q₄ q₅ q₆   (Steane code)

Step 2: Add 8 ancilla qubits in |0⟩

    q₀ q₁ q₂ q₃ q₄ q₅ q₆  |0⟩⊗8

Step 3: Apply encoding circuit (Clifford)

    The encoding maps Steane structure to RM structure
    Uses CNOT gates to spread information

Step 4: Measure stabilizers of RM code

    Verify we're in RM code space
    Apply corrections if needed

Step 5: Now have |ψ_L⟩ in Reed-Muller code (15 qubits)

    Ready for transversal T!
        """)

    def reed_muller_to_steane(self):
        """Switch from Reed-Muller back to Steane."""
        print("""
Reed-Muller [[15,1,3]] to Steane [[7,1,3]]
══════════════════════════════════════════

SWITCHING BACK:

Step 1: Start with RM-encoded |ψ_L⟩ (15 qubits)

Step 2: Measure syndrome to detect/correct errors

Step 3: Apply decoding circuit (inverse of encoding)

    CNOTs to un-spread information
    Project 8 qubits back to |0⟩ or |1⟩

Step 4: Measure the 8 "extra" qubits

    Should get |0⟩ if no errors
    Outcome tells us if error occurred

Step 5: Result is Steane-encoded |ψ_L⟩ (7 qubits)

    q₀ q₁ q₂ q₃ q₄ q₅ q₆   (back in Steane code)


FAULT TOLERANCE:
    - Each step uses fault-tolerant operations
    - Syndrome measurement before/after switching
    - Single fault → correctable error
        """)


switching = CodeSwitching()
switching.steane_to_reed_muller()
switching.reed_muller_to_steane()
```

## Gauge Fixing in Subsystem Codes

```
Gauge Fixing: Another Approach to Code Switching
═════════════════════════════════════════════════

SUBSYSTEM CODES have extra degrees of freedom (gauge qubits).

Example: [[15,1,9]] Bacon-Shor subsystem code

    Gauge operators: Not fixed, can have any value
    Logical operator: The actual information

GAUGE FIXING:
    By measuring gauge operators, we can "fix" them
    This changes the effective code!


EXAMPLE: Color Code Gauge Fixing

    3D Color Code:
    - Has transversal CCZ (Toffoli)
    - Also called "Toric code with twist"

    By gauge fixing different ways:
    - Get code with transversal T
    - Get code with transversal H
    - Switch between them by changing gauge

ADVANTAGES:
    - Smoother transition (no full re-encoding)
    - Stay in same physical qubit layout
    - Change which gates are transversal


VISUALIZATION:

    ┌────────────────────────────────────────────┐
    │           Subsystem Code Space             │
    │                                            │
    │    ┌──────────┐         ┌──────────┐      │
    │    │  Gauge   │ ←─────→ │  Gauge   │      │
    │    │ Config A │  fixing │ Config B │      │
    │    │          │         │          │      │
    │    │ T trans. │         │ H trans. │      │
    │    └──────────┘         └──────────┘      │
    │                                            │
    └────────────────────────────────────────────┘

    Same logical qubit, different transversal gates!
```

## Code Deformation

```python
class CodeDeformation:
    """
    Implement code deformation techniques.
    """

    def __init__(self):
        pass

    def explain_deformation(self):
        """Explain code deformation concept."""
        print("""
Code Deformation
════════════════

IDEA: Gradually change the code structure
    - Start with code A
    - Smoothly morph into code B
    - Each step is a small change
    - Fault tolerance maintained throughout

SURFACE CODE DEFORMATION:

    Twist/untwist boundaries
    ┌─────────┐         ┌─────────┐
    │  Rough  │   →→    │  Twist  │   Deformation
    └─────────┘   ←←    └─────────┘

    Change code distance
    d=3 → d=5 → d=7     (grow code)
    d=7 → d=5 → d=3     (shrink code)


TOPOLOGICAL OPERATIONS:

    Moving defects in toric code
    ┌─────────────────────────────┐
    │   ●          →→        ●   │   Braid defects
    │        ●    →→   ●         │   for gates
    └─────────────────────────────┘

    The "code" effectively changes as defects move
        """)

    def defect_braiding(self):
        """Describe defect braiding as code deformation."""
        print("""
Defect Braiding in Topological Codes
════════════════════════════════════

For topological codes (toric, color codes):

DEFECTS = punctures in the code lattice
    - Rough defects (violations of Z stabilizers)
    - Smooth defects (violations of X stabilizers)

BRAIDING:
    Move defects around each other
    Creates logical operations!

    ●─────────────●         ●────────┐
    │             │    →    │        │    Braid
    │      ○      │    →    │   ○    │    creates
    │             │    →    │        │    gate
    ●─────────────●         ●────────┘

    ● = defect A,  ○ = defect B

    Different braiding patterns → different gates


COLOR CODE EXAMPLE:
    Moving defect through color code patch
    → Implements logical gate on enclosed qubits

NON-ABELIAN ANYONS:
    For non-abelian anyons, braiding alone is universal!
    But currently no experimental realization
        """)


deformation = CodeDeformation()
deformation.explain_deformation()
deformation.defect_braiding()
```

## Protocol Examples

```
Complete Code Switching Protocol
════════════════════════════════

GOAL: Apply T gate to Steane-encoded qubit

PROTOCOL:

Step 1: Error correction on Steane code
    - Measure all 6 Steane stabilizers
    - Decode and correct

Step 2: Prepare 8 verified ancilla qubits
    - Initialize |0⟩⊗8
    - Verify with Z measurements

Step 3: Apply encoding circuit
    - Series of CNOT gates
    - Maps (Steane ⊗ ancilla) → Reed-Muller
    - Circuit is transversal-like (fault-tolerant)

Step 4: Verify Reed-Muller encoding
    - Measure RM stabilizers
    - Should all be +1
    - If not, abort and retry

Step 5: Apply transversal T
    - T^⊗15 on all 15 qubits
    - This is the logical T gate!

Step 6: Error correction on RM code
    - Measure RM stabilizers
    - Decode and correct

Step 7: Apply decoding circuit
    - Inverse of encoding
    - Maps RM → (Steane ⊗ |?⟩⊗8)

Step 8: Measure 8 extra qubits
    - Verify they returned to |0⟩
    - Errors detected if not |0⟩

Step 9: Back in Steane code
    - Continue with Clifford gates


RESOURCE COST:
    ~100-200 physical gates per T
    Much less than magic state distillation for some regimes
```

## Summary

Code switching provides an alternative to magic states for non-Clifford gates:

1. **Concept**: Transfer between codes with different transversal gates
2. **Steane ↔ Reed-Muller**: Classic pair for T gate access
3. **Gauge Fixing**: Change effective code by fixing gauge degrees of freedom
4. **Code Deformation**: Gradually morph code structure
5. **Defect Braiding**: Topological operations via defect movement
6. **Trade-offs**: Less distillation overhead, but complex switching circuits

The choice between code switching and magic states depends on the specific computation and hardware constraints.
