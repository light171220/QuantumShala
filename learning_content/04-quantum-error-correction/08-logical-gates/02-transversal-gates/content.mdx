# Transversal Gate Implementation

## Introduction

Transversal gates are the gold standard for fault-tolerant logical gates. By applying physical gates to corresponding qubits in parallel, we automatically prevent error propagation. This lesson explores how to implement transversal gates and understand their limitations.

## What Makes a Gate Transversal?

```
Transversal Gate Definition
═══════════════════════════

A gate U is TRANSVERSAL for a code if:

    U_L = U₁ ⊗ U₂ ⊗ ... ⊗ Uₙ

Where each Uᵢ acts only on qubit i.

SIMPLEST CASE: U_L = U^⊗n (same gate on all qubits)

    ┌──────────────┐
    │ U ─ U ─ U ─ U│  =  U_L
    └──────────────┘

For two-qubit gates (between code blocks):

    Block A: q₀ ─ q₁ ─ q₂ ─ ... ─ qₙ₋₁
             │   │   │        │
             ▼   ▼   ▼        ▼
    Block B: q₀ ─ q₁ ─ q₂ ─ ... ─ qₙ₋₁

    Each pair (qᵢᴬ, qᵢᴮ) gets ONE physical gate.
    No cross-connections between different i,j pairs.


WHY TRANSVERSAL = FAULT-TOLERANT:

    Error on qubit i during U_L:
    → Affects at most qubit i in output
    → Can't spread to qubit j (no connection!)
    → At most 1 error per code block

    Single fault → single error per block ✓
```

```python
class TransversalPaulis:
    """
    Implement transversal Pauli gates.
    """

    def __init__(self, code_name: str, n_qubits: int):
        self.code = code_name
        self.n = n_qubits

    def logical_x(self):
        """Implement logical X transversally."""
        print(f"""
Logical X for {self.code} ({self.n} qubits)
{'═' * 45}

For CSS codes, X_L is transversal:

    X_L = X^⊗n  (X on every qubit)

    ┌─────────────────────────────┐
    │ X ─ X ─ X ─ ... ─ X        │
    │ q₀  q₁  q₂       qₙ₋₁      │
    └─────────────────────────────┘

WHY THIS WORKS:
    X^⊗n commutes with Z stabilizers (Z type)
    X^⊗n anticommutes with X stabilizers selectively
    Result: flips logical qubit, stays in code space

ALTERNATIVE (minimal weight X_L):
    For surface code: X_L = X along horizontal path
    Still fault-tolerant but fewer gates

    ●───X───X───X───X───●  (d gates for distance-d)
        """)

    def logical_z(self):
        """Implement logical Z transversally."""
        print(f"""
Logical Z for {self.code} ({self.n} qubits)
{'═' * 45}

Similarly, Z_L is transversal:

    Z_L = Z^⊗n  (Z on every qubit)

    ┌─────────────────────────────┐
    │ Z ─ Z ─ Z ─ ... ─ Z        │
    │ q₀  q₁  q₂       qₙ₋₁      │
    └─────────────────────────────┘

For surface code (minimal):
    Z_L = Z along vertical path

    ●
    │
    Z
    │
    Z      (d gates)
    │
    Z
    │
    ●
        """)

    def verify_pauli_transversality(self):
        """Verify that Paulis work transversally."""
        print("""
Verifying Transversality
════════════════════════

To verify X^⊗n implements X_L:

1. Check stabilizer preservation:
   For each stabilizer S:
   X^⊗n · S · X^⊗n = S  (X commutes with X, anticommutes with Z)

   Z-type stabilizers: X^⊗n · Zₐ...Zᵦ · X^⊗n = Zₐ...Zᵦ ✓
   X-type stabilizers: X^⊗n · Xₐ...Xᵦ · X^⊗n = Xₐ...Xᵦ ✓

2. Check logical action:
   X^⊗n anticommutes with Z_L
   X^⊗n commutes with X_L
   → Acts as X on logical qubit ✓

Same analysis for Z^⊗n → Z_L ✓
        """)


paulis = TransversalPaulis("Steane [[7,1,3]]", 7)
paulis.logical_x()
paulis.logical_z()
paulis.verify_pauli_transversality()
```

## Transversal Hadamard

```python
class TransversalHadamard:
    """
    Implement transversal Hadamard gate.
    """

    def __init__(self):
        pass

    def hadamard_on_css(self):
        """Hadamard on CSS codes."""
        print("""
Transversal Hadamard on CSS Codes
═════════════════════════════════

For SELF-DUAL CSS codes:

    H_L = H^⊗n

CSS Code Property:
    X stabilizers ↔ Z stabilizers under H^⊗n

If code is self-dual: Hₓ = Hᵤ (same parity check)
    → H^⊗n maps code space to itself!

STEANE CODE EXAMPLE:
    Stabilizers:   X₀X₁X₂X₃
                   X₁X₂X₄X₅
                   X₂X₃X₅X₆
                   Z₀Z₁Z₂Z₃
                   Z₁Z₂Z₄Z₅
                   Z₂Z₃Z₅Z₆

    Under H^⊗7:
    Xᵢ → Zᵢ for all i
    Zᵢ → Xᵢ for all i

    X stabilizers → Z stabilizers (which are also stabilizers!)
    → H^⊗7 preserves code space ✓

LOGICAL ACTION:
    X_L → Z_L (since X→Z for each qubit)
    Z_L → X_L
    → H_L: H action on logical qubit ✓
        """)

    def non_transversal_h(self):
        """When H is not transversal."""
        print("""
When Hadamard is NOT Transversal
════════════════════════════════

For non-self-dual codes:
    H^⊗n maps X stabilizers to Z stabilizers
    But Z stabilizers might NOT equal X stabilizers!

SURFACE CODE EXAMPLE:
    Rectangular surface code (not square)
    X stabilizers: 4-body terms at vertices
    Z stabilizers: 4-body terms at faces

    Under H^⊗n:
    X stabilizers → Z-type operators at vertices
    These are NOT the original face stabilizers!

    → H^⊗n does NOT preserve code space
    → H is NOT transversal for general surface codes

SOLUTION:
    Use lattice surgery or code rotation
    Or use square rotated surface code (has transversal H)
        """)


hadamard = TransversalHadamard()
hadamard.hadamard_on_css()
hadamard.non_transversal_h()
```

## Transversal CNOT

```python
class TransversalCNOT:
    """
    Implement transversal CNOT between code blocks.
    """

    def __init__(self, n_qubits: int):
        self.n = n_qubits

    def cnot_structure(self):
        """Show transversal CNOT structure."""
        print(f"""
Transversal CNOT Structure
══════════════════════════

For two code blocks A and B:

CNOT_L = CNOT₀ ⊗ CNOT₁ ⊗ ... ⊗ CNOT₍ₙ₋₁₎

    Block A (control):  ●───●───●───●───●───●───●
                        │   │   │   │   │   │   │
    Block B (target):   X───X───X───X───X───X───X

Each CNOTᵢ has:
    Control: qubit i of block A
    Target:  qubit i of block B


WHY THIS WORKS (CSS codes):

Error propagation:
    X error on control i → X error on target i (only!)
    Z error on target i → Z error on control i (only!)

    Single fault → single error per block ✓

Logical action:
    X_L^A → X_L^A ⊗ X_L^B  (control X spreads to target)
    Z_L^B → Z_L^A ⊗ Z_L^B  (target Z spreads to control)
    X_L^B → X_L^B          (target X unchanged)
    Z_L^A → Z_L^A          (control Z unchanged)

    → Correct CNOT_L action ✓
        """)

    def cnot_error_analysis(self):
        """Analyze error propagation in transversal CNOT."""
        print("""
Error Propagation in Transversal CNOT
═════════════════════════════════════

SCENARIO 1: X error on control qubit i before CNOT

    Block A: ─X────●─────  (X error before control)
                   │
    Block B: ──────X─────

    After CNOT:
    Block A: has X on qubit i (original error)
    Block B: has X on qubit i (propagated)

    Result: 1 error per block ✓ (correctable)

SCENARIO 2: Z error on target qubit j before CNOT

    Block A: ──────●─────
                   │
    Block B: ─Z────X─────

    Z commutes through CNOT target:
    Block A: gets Z on qubit j (back-propagated)
    Block B: has Z on qubit j (original)

    Result: 1 error per block ✓ (correctable)

SCENARIO 3: Error during CNOT gate itself

    Physical CNOT on pair i fails → both qubits affected
    But only pair i is affected, not other pairs

    Result: 1 error per block ✓ (correctable)
        """)


cnot = TransversalCNOT(7)
cnot.cnot_structure()
cnot.cnot_error_analysis()
```

## Transversal Phase Gate

```
Transversal Phase (S) Gate
══════════════════════════

For some CSS codes (like Steane):

    S_L = S^⊗n

Where S = diag(1, i) = √Z

STEANE CODE:
    S^⊗7 transforms stabilizers:
    X stabilizers → Y stabilizers (XᵢSᵢXᵢS†ᵢ = Yᵢ)
    Z stabilizers → Z stabilizers (unchanged)

    Key: The Y stabilizers are ALSO stabilizers!
    (Because code has special structure)

    Logical action:
    X_L → Y_L = iX_L Z_L
    Z_L → Z_L

    → Correct S_L action ✓


LIMITATION:
    S is NOT transversal for all CSS codes!

    For surface code:
    S^⊗n transforms X stabilizers to Y stabilizers
    Y stabilizers are NOT stabilizers of surface code
    → S is NOT transversal

CODES WITH TRANSVERSAL S:
    - Steane [[7,1,3]]
    - Color codes
    - Some special CSS codes

CODES WITHOUT TRANSVERSAL S:
    - Surface codes (need lattice surgery)
    - Shor [[9,1,3]]
```

## Limitations and the Eastin-Knill Theorem

```
The Eastin-Knill Theorem
════════════════════════

THEOREM:
    No stabilizer code can have a transversal UNIVERSAL gate set.

IMPLICATIONS:
    Every code is missing at least one gate from universality

    Steane [[7,1,3]]: Has H, S, CNOT — missing T
    Surface code:     Has CNOT — missing H, S, T
    [[15,1,3]] Reed-Muller: Has T — missing S!

WHY THIS HAPPENS:
    Transversal gates form a FINITE group (for finite codes)
    Universal gates form INFINITE (continuous) group
    Finite ⊄ Infinite

CONSEQUENCE:
    Must use non-transversal methods for some gates:
    - Magic state injection (for T)
    - Lattice surgery (for CNOT on surface code)
    - Code switching (between codes with different transversal sets)


PRACTICAL APPROACH:
    Choose code based on which gates you need most
    Use efficient non-transversal methods for missing gates

    Surface code: Great threshold, CNOT transversal
                 Use magic states for T, lattice surgery for others

    Color code: Transversal Clifford group
               Use magic states for T
```

## Summary

Transversal gates are the most elegant fault-tolerant implementations:

1. **Structure**: Same gate on corresponding qubits (U_L = U^⊗n)
2. **Fault Tolerance**: Automatic — errors can't spread between qubits
3. **Paulis**: Always transversal for stabilizer codes
4. **Hadamard**: Transversal for self-dual CSS codes
5. **CNOT**: Transversal between identical CSS code blocks
6. **Phase (S)**: Transversal for some CSS codes (Steane, color codes)
7. **Limitation**: No code has transversal universal set (Eastin-Knill)

The next lessons cover non-transversal methods needed for complete universality.
