# QEC on Superconducting Qubits

## Introduction

Superconducting qubits are the leading platform for near-term QEC demonstrations. This lesson covers the specific challenges and techniques for implementing surface codes on superconducting processors.

## Transmon Qubits

```
Transmon Qubit Basics
═════════════════════

PHYSICAL STRUCTURE:
    - Josephson junction + capacitor
    - Nonlinear LC oscillator
    - Qubit states: |0⟩ and |1⟩ (lowest two energy levels)

    ┌─────────────────────────┐
    │     Capacitor          │
    │    ═══════════         │
    │        │               │
    │    ╔═══╧═══╗           │
    │    ║   ×   ║  ← Josephson junction
    │    ╚═══╤═══╝           │
    │        │               │
    └─────────────────────────┘

ENERGY LEVELS:
    |0⟩ ─────────    Ground state
    |1⟩ ─────────    First excited (qubit state)
    |2⟩ ─────────    Second excited (leakage!)

    Anharmonicity: E₂ - E₁ ≠ E₁ - E₀
    Allows selective addressing of 0-1 transition

TYPICAL PARAMETERS:
    Frequency: 4-6 GHz
    Anharmonicity: 200-300 MHz
    T1: 50-200 μs
    T2: 50-150 μs
```

## Surface Code Chip Layout

```python
class SurfaceCodeChip:
    """
    Design surface code layout for superconducting chips.
    """

    def __init__(self, distance: int):
        self.d = distance

    def layout_design(self):
        """Explain chip layout."""
        print(f"""
Surface Code Chip Layout (d = {self.d})
{'═' * 40}

QUBIT ARRANGEMENT:
    - Data qubits at vertices
    - Ancilla qubits at face/vertex centers
    - Alternating X and Z check ancillas

    For d = {self.d}:
    Data qubits: {self.d * self.d}
    Ancilla qubits: {(self.d-1)**2 + self.d**2 - 1}
    Total: ~{2 * self.d * self.d} qubits

LAYOUT EXAMPLE (d=3):

    D●───A───D●───A───D●
     │   Z   │   Z   │
    A───D●───A───D●───A
     X       X       X
    D●───A───D●───A───D●
     │   Z   │   Z   │
    A───D●───A───D●───A
     X       X       X
    D●───A───D●───A───D●

    D● = Data qubit
    A  = Ancilla (X or Z labeled)

COUPLING:
    Each data qubit connects to 2-4 ancillas
    Each ancilla connects to 2-4 data qubits
    All connections are nearest-neighbor
        """)

    def frequency_allocation(self):
        """Explain frequency collision avoidance."""
        print("""
Frequency Allocation
════════════════════

CHALLENGE: Avoid frequency collisions

    If two nearby qubits have same frequency:
    → Unwanted ZZ coupling (always-on interaction)
    → Gate errors and crosstalk

SOLUTION: Frequency pattern

    Use 4 or more distinct frequencies
    Pattern them across the chip

    f₁───f₂───f₁───f₂
     │    │    │    │
    f₃───f₄───f₃───f₄
     │    │    │    │
    f₁───f₂───f₁───f₂

    No adjacent qubits share frequency

TYPICAL FREQUENCIES:
    f₁ = 4.5 GHz
    f₂ = 4.8 GHz
    f₃ = 5.1 GHz
    f₄ = 5.4 GHz

    Plus ancillas at different frequencies
        """)


chip = SurfaceCodeChip(5)
chip.layout_design()
chip.frequency_allocation()
```

## Syndrome Extraction Circuit

```
Surface Code Syndrome Extraction
════════════════════════════════

One syndrome round for all stabilizers:

TIMING (typical):
    1. Prepare ancillas: H on X-ancillas (20 ns)
    2. CNOT round 1: SW connections (60 ns)
    3. CNOT round 2: SE connections (60 ns)
    4. CNOT round 3: NW connections (60 ns)
    5. CNOT round 4: NE connections (60 ns)
    6. Complete ancillas: H on X-ancillas (20 ns)
    7. Measure all ancillas (500 ns)
    8. Reset ancillas (300 ns)
    ───────────────────────────────────
    Total: ~1000 ns = 1 μs per round

GATE ORDERING (critical for fault tolerance):

    X stabilizer (4 data qubits):

        D1──D2
         │╲╱│
         │╳│   A = X-check ancilla
         │╱╲│
        D3──D4

    Order: D1 → D2 → D3 → D4
    (or any order avoiding hook errors)

    Z stabilizer: Similar with CZ gates

PARALLELIZATION:
    All X checks done in parallel
    All Z checks done in parallel
    Total: 8 gate layers + measure/reset
```

## Handling Real-World Errors

```python
class RealWorldErrors:
    """
    Handle errors specific to superconducting QEC.
    """

    def __init__(self):
        pass

    def crosstalk(self):
        """Explain crosstalk mitigation."""
        print("""
Crosstalk Errors
════════════════

CAUSES:
    1. Stray capacitive coupling
    2. Microwave drive leakage
    3. Resonator-mediated coupling

EFFECTS:
    - Correlated errors between nearby qubits
    - Gate errors when parallel operations
    - Decoder must account for correlations

MITIGATION:
    1. SCHEDULING
       Avoid simultaneous gates on coupled qubits
       Increases circuit depth but reduces errors

    2. CALIBRATION
       Measure crosstalk matrix
       Compensate with modified pulses

    3. DECODER MODIFICATION
       Include correlated error model
       Weight edges for likely crosstalk patterns
        """)

    def leakage(self):
        """Explain leakage handling."""
        print("""
Leakage Errors
══════════════

PROBLEM:
    Population escapes to |2⟩, |3⟩, ... states
    Stabilizer codes don't correct leakage!

SOURCES:
    - Two-qubit gates (CZ, SWAP)
    - Repeated single-qubit gates
    - Thermal excitation

DETECTION:
    Leakage causes persistent syndrome patterns
    Modify decoder to detect anomalies

RESET:
    Leakage Reduction Units (LRUs)
    Apply reset pulse to bring back to |0⟩
    Add to QEC cycle periodically

LRU CIRCUIT:
    Measure in |0,1⟩ vs |2,3,...⟩ basis
    If leaked: reset to |0⟩
    If not: continue normally
        """)

    def measurement_errors(self):
        """Explain measurement error handling."""
        print("""
Measurement Errors
══════════════════

SOURCES:
    1. Readout resonator imperfections
    2. T1 decay during measurement
    3. State assignment errors

RATES: 0.5-2% typical

MITIGATION:

    1. REPEATED MEASUREMENT
       Measure d times, use 3D decoding
       Filters transient errors

    2. IMPROVED READOUT
       Purcell filters
       Parametric amplifiers
       Optimized resonator design

    3. SOFT DECODING
       Use analog measurement signal
       Provides confidence for decoder
        """)


errors = RealWorldErrors()
errors.crosstalk()
errors.leakage()
errors.measurement_errors()
```

## Summary

Superconducting QEC implementation involves:

1. **Layout**: 2D grid matching surface code connectivity
2. **Frequency Planning**: Avoid collisions between neighboring qubits
3. **Syndrome Circuits**: ~1 μs per round with careful gate ordering
4. **Crosstalk**: Mitigate through scheduling and calibration
5. **Leakage**: Detect and reset escaped population
6. **Measurement**: Handle errors with repeated measurements

This platform is currently leading in QEC demonstrations.
