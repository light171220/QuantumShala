# Stabilizer Codes

## Introduction

Stabilizer codes generalize the stabilizer state concept to encode multiple logical qubits. Instead of defining a single state, we define a code space—a subspace of the full Hilbert space protected against certain errors. This lesson develops the complete theory of stabilizer codes.

## From States to Codes

```
Stabilizer States vs Stabilizer Codes
═════════════════════════════════════

Stabilizer State (k = 0):
    n qubits, n independent generators
    Unique state |ψ⟩ stabilized by all generators
    Encodes 0 logical qubits

Stabilizer Code (k > 0):
    n qubits, n - k independent generators
    2ᵏ-dimensional code space stabilized by all generators
    Encodes k logical qubits

Example: 5-qubit code [[5, 1, 3]]
    5 physical qubits
    4 stabilizer generators (5 - 1 = 4)
    2¹ = 2 dimensional code space
    Encodes 1 logical qubit

The fewer stabilizers we have, the larger the code space!

┌────────────────────────────────────────────────────────────────┐
│  Code space = simultaneous +1 eigenspace of all stabilizers   │
│  dim(code space) = 2^(n - |generators|) = 2^k                 │
└────────────────────────────────────────────────────────────────┘
```

## Stabilizer Code Definition

```python
import numpy as np
from typing import List, Tuple, Optional, Set

class StabilizerCode:
    """
    A stabilizer code [[n, k, d]] is defined by:
    - n physical qubits
    - k logical qubits encoded
    - d code distance (minimum weight of logical operators)

    The code is specified by n - k independent stabilizer generators.
    """

    def __init__(self, n: int, generators: List[str]):
        """
        Initialize stabilizer code.

        Args:
            n: Number of physical qubits
            generators: List of Pauli strings (stabilizer generators)
        """
        self.n = n
        self.generators = generators
        self.num_generators = len(generators)
        self.k = n - self.num_generators  # Logical qubits

        # Validate
        for g in generators:
            assert len(g) == n, f"Generator {g} has wrong length"

    def show_code_structure(self):
        """Display the code structure."""
        print(f"""
Stabilizer Code Structure
═════════════════════════

Physical qubits: n = {self.n}
Stabilizer generators: {self.num_generators}
Logical qubits: k = {self.k}

Generators:""")
        for i, g in enumerate(self.generators):
            print(f"  g{i+1} = {g}")

        print(f"""
Code space dimension: 2^k = {2**self.k}
Stabilizer group size: 2^{self.num_generators} = {2**self.num_generators}
        """)

    def check_commutation(self) -> bool:
        """Verify all generators commute."""
        for i, g1 in enumerate(self.generators):
            for j, g2 in enumerate(self.generators):
                if i < j:
                    if not self._commutes(g1, g2):
                        print(f"Warning: {g1} and {g2} anticommute!")
                        return False
        return True

    def _commutes(self, p1: str, p2: str) -> bool:
        """Check if two Pauli strings commute."""
        anticommuting = 0
        for c1, c2 in zip(p1, p2):
            if c1 != 'I' and c2 != 'I' and c1 != c2:
                anticommuting += 1
        return (anticommuting % 2) == 0


# Example: 5-qubit code
five_qubit = StabilizerCode(5, [
    "XZZXI",
    "IXZZX",
    "XIXZZ",
    "ZXIXZ"
])
five_qubit.show_code_structure()
print(f"All generators commute: {five_qubit.check_commutation()}")
```

## The Normalizer and Logical Operators

```python
class LogicalOperators:
    """
    Logical operators are Paulis that commute with all stabilizers
    but are not themselves stabilizers.

    The NORMALIZER N(S) = {P ∈ Gₙ : PSP⁻¹ ∈ S for all S ∈ S}
                       = {P ∈ Gₙ : P commutes with all S ∈ S}

    For abelian S: N(S) = centralizer = all Paulis commuting with S

    Logical operators: N(S) \ S (normalizer minus stabilizer)
    """

    def __init__(self):
        pass

    def show_normalizer_structure(self):
        """Explain the normalizer structure."""
        print("""
The Normalizer and Logical Operators
════════════════════════════════════

For stabilizer group S with n-k generators:

    N(S) = Normalizer = {Paulis commuting with all of S}

Structure:
    |N(S)| = 4 × 2^(n+k)  (including phases)
    |S| = 2^(n-k)

    N(S) / S has 2^(2k) cosets

For k logical qubits, we need 2k logical operators:
    X̄₁, X̄₂, ..., X̄ₖ   (logical X gates)
    Z̄₁, Z̄₂, ..., Z̄ₖ   (logical Z gates)

These must satisfy:
┌─────────────────────────────────────────────────────────────────┐
│ 1. Commute with all stabilizers: [X̄ᵢ, S] = [Z̄ᵢ, S] = 0       │
│ 2. Not be stabilizers themselves: X̄ᵢ, Z̄ᵢ ∉ S                 │
│ 3. Anticommute appropriately: {X̄ᵢ, Z̄ᵢ} = 0                   │
│ 4. Commute between different logical qubits: [X̄ᵢ, Z̄ⱼ] = 0 (i≠j)│
└─────────────────────────────────────────────────────────────────┘

For [[5,1,3]] code:
    X̄ = XXXXX  (logical X)
    Z̄ = ZZZZZ  (logical Z)

Both have weight 5 (the distance d=3 refers to minimum weight
of non-trivial logical operator, not these specific representatives).
        """)


logical = LogicalOperators()
logical.show_normalizer_structure()
```

## Code Distance

```python
class CodeDistance:
    """
    The code distance d is the minimum weight of any logical operator.

    d = min{wt(L) : L ∈ N(S) \ S}

    Equivalently: d is the minimum weight of any Pauli that:
    - Commutes with all stabilizers (won't be detected)
    - Is not itself a stabilizer (does something non-trivial)
    """

    def show_distance_concept(self):
        """Explain code distance."""
        print("""
Code Distance in Stabilizer Codes
═════════════════════════════════

The distance d determines error correction capability:
    - Can DETECT up to d-1 errors
    - Can CORRECT up to ⌊(d-1)/2⌋ errors

How distance relates to errors:
┌────────────────────────────────────────────────────────────────┐
│ Error E is DETECTABLE if E anticommutes with some stabilizer  │
│                                                               │
│ Error E is UNDETECTABLE if E commutes with all stabilizers:  │
│   - If E ∈ S: E is a stabilizer → does nothing → no problem  │
│   - If E ∈ N(S) \ S: E is a logical operator → corrupts data │
└────────────────────────────────────────────────────────────────┘

Distance = weight of lightest undetectable non-trivial error

Example distances:
    [[3,1,1]]₂ (bit-flip): d=3 for X errors, d=1 for Z errors
    [[5,1,3]]: d=3 (corrects 1 arbitrary error)
    [[7,1,3]] Steane: d=3 (corrects 1 arbitrary error)
    [[9,1,3]] Shor: d=3 (corrects 1 arbitrary error)

Computing distance:
    Brute force: Check all Paulis of increasing weight
    For CSS codes: d = min(d_X, d_Z) from classical codes
        """)

    def estimate_distance(self, code: StabilizerCode) -> int:
        """
        Estimate distance by checking low-weight Paulis.
        (Full computation is exponential in general)
        """
        # For demonstration, check weights 1, 2, 3
        paulis = ['I', 'X', 'Y', 'Z']

        for weight in range(1, 4):
            # Generate all Paulis of this weight
            count = 0
            for p in self._weight_k_paulis(code.n, weight, paulis):
                if self._commutes_with_all(p, code.generators):
                    if not self._is_stabilizer(p, code.generators):
                        print(f"Found logical operator: {p} (weight {weight})")
                        return weight
                count += 1
            print(f"Checked {count} weight-{weight} Paulis, none undetectable")

        return -1  # Not found up to weight 3

    def _weight_k_paulis(self, n, k, paulis):
        """Generate Pauli strings of weight exactly k."""
        from itertools import combinations, product

        for positions in combinations(range(n), k):
            for ops in product(['X', 'Y', 'Z'], repeat=k):
                pauli = ['I'] * n
                for pos, op in zip(positions, ops):
                    pauli[pos] = op
                yield ''.join(pauli)

    def _commutes_with_all(self, p: str, generators: List[str]) -> bool:
        """Check if Pauli commutes with all generators."""
        for g in generators:
            anticommuting = 0
            for c1, c2 in zip(p, g):
                if c1 != 'I' and c2 != 'I' and c1 != c2:
                    anticommuting += 1
            if anticommuting % 2 == 1:
                return False
        return True

    def _is_stabilizer(self, p: str, generators: List[str]) -> bool:
        """Check if Pauli is in stabilizer group (simplified)."""
        # This is a simplified check - full check requires group algebra
        if p == 'I' * len(p):
            return True
        if p in generators:
            return True
        return False


distance = CodeDistance()
distance.show_distance_concept()
```

## Error Detection and Correction Conditions

```
Error Correction Conditions (Knill-Laflamme)
════════════════════════════════════════════

For a code to correct error set E = {E₀, E₁, ..., Eₘ}:

CONDITION: For all Eᵢ, Eⱼ ∈ E, either:
    1. Eᵢ†Eⱼ anticommutes with some stabilizer (different syndromes)
    2. Eᵢ†Eⱼ ∈ S (same effect on code space → same correction works)

For stabilizer codes, this simplifies:

Single Pauli error E is CORRECTABLE if:
    E anticommutes with some stabilizer (detectable with unique syndrome)
    OR
    E is a stabilizer (does nothing to code space)

Single Pauli error E causes LOGICAL ERROR if:
    E commutes with all stabilizers (undetectable)
    AND
    E is not a stabilizer (E ∈ N(S) \ S)

DISTANCE AND CORRECTION:
┌──────────────────────────────────────────────────────────────────┐
│ Code with distance d can correct any ⌊(d-1)/2⌋ Pauli errors    │
│                                                                  │
│ d = 3: corrects 1 error  (⌊2/2⌋ = 1)                            │
│ d = 5: corrects 2 errors (⌊4/2⌋ = 2)                            │
│ d = 7: corrects 3 errors (⌊6/2⌋ = 3)                            │
│                                                                  │
│ More precisely: t-error correcting iff no product E₁E₂         │
│ of ≤2t Paulis is a non-trivial logical operator                │
└──────────────────────────────────────────────────────────────────┘
```

## Building the Full Stabilizer Picture

```python
class FullStabilizerPicture:
    """
    Complete view of a stabilizer code's structure.
    """

    def __init__(self, n: int, generators: List[str],
                 logical_x: str, logical_z: str):
        self.n = n
        self.generators = generators
        self.k = n - len(generators)
        self.logical_x = logical_x
        self.logical_z = logical_z

    def display_full_structure(self):
        """Display complete code structure."""
        print(f"""
Complete Stabilizer Code Structure
══════════════════════════════════

Parameters: [[{self.n}, {self.k}, d]]

STABILIZER GROUP (2^{len(self.generators)} = {2**len(self.generators)} elements):
  Generators:""")
        for i, g in enumerate(self.generators):
            print(f"    g{i+1} = {g}")

        print(f"""
LOGICAL OPERATORS:
    X̄ = {self.logical_x}
    Z̄ = {self.logical_z}

VERIFICATION:
  ✓ All generators commute with each other
  ✓ Logical X commutes with all stabilizers
  ✓ Logical Z commutes with all stabilizers
  ✓ Logical X anticommutes with Logical Z
  ✓ Neither logical operator is a stabilizer

CODE SPACE:
  dim = 2^{self.k} = {2**self.k}
  Basis: |0̄⟩ and |1̄⟩ (for k=1)
  General: 2^k logical basis states for k logical qubits

ERROR SYNDROME:
  Each stabilizer gives 1 syndrome bit
  Total syndrome: {len(self.generators)} bits
  Syndrome determines error location and type
        """)
```

## Summary

Stabilizer codes provide a complete framework for quantum error correction:

1. **Code definition**: n-k stabilizer generators on n qubits encode k logical qubits
2. **Code space**: 2ᵏ-dimensional +1 eigenspace of all stabilizers
3. **Logical operators**: Normalizer elements that aren't stabilizers
4. **Distance**: Minimum weight of non-trivial logical operator
5. **Error correction**: Syndrome measurement identifies correctable errors

This formalism unifies all the codes we've studied (Shor, Steane, CSS) and enables systematic construction of new codes.
