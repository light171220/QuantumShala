## 1. The Computational Chemistry Landscape

Computational chemistry uses computers to solve chemical problems that are too complex for analytical solutions.

<Definition term="Computational Chemistry">
The use of computer simulations to solve chemical problems, predict molecular properties, and understand chemical phenomena. Methods range from classical force fields to high-level quantum mechanical calculations.
</Definition>

### The Accuracy-Cost Trade-off

<Formula>
$$\text{Accuracy} \propto \text{Computational Cost}$$
</Formula>

There is no free lunch in computational chemistry: more accurate methods require more computational resources.

## 2. Classical Methods: Molecular Mechanics

<Definition term="Molecular Mechanics (MM)">
A computational method that treats atoms as classical particles connected by springs (bonds), without explicitly considering electrons. Energies are calculated using empirical force fields.
</Definition>

### Force Field Energy

<Formula>
$$E_{total} = E_{bonds} + E_{angles} + E_{dihedrals} + E_{vdW} + E_{electrostatic}$$
</Formula>

```python
def force_field_energy_components():
    """Describe force field energy terms."""
    terms = {
        'E_bonds': {
            'formula': 'Σ k_b(r - r_0)²',
            'description': 'Bond stretching (harmonic)',
            'typical_k': '300-500 kcal/mol/Ų'
        },
        'E_angles': {
            'formula': 'Σ k_θ(θ - θ_0)²',
            'description': 'Angle bending (harmonic)',
            'typical_k': '50-100 kcal/mol/rad²'
        },
        'E_dihedrals': {
            'formula': 'Σ V_n[1 + cos(nφ - γ)]',
            'description': 'Torsional rotation',
            'typical_k': '1-5 kcal/mol'
        },
        'E_vdW': {
            'formula': 'Σ ε[(σ/r)¹² - 2(σ/r)⁶]',
            'description': 'Lennard-Jones potential',
            'typical_k': '0.1-0.5 kcal/mol'
        },
        'E_electrostatic': {
            'formula': 'Σ q_i q_j / (4πε₀r_ij)',
            'description': 'Coulomb interactions',
            'typical_k': 'Depends on charges'
        }
    }

    print("Force Field Energy Terms:")
    print("=" * 70)
    for term, info in terms.items():
        print(f"\n{term}:")
        print(f"  Formula: {info['formula']}")
        print(f"  Description: {info['description']}")
        print(f"  Typical strength: {info['typical_k']}")

force_field_energy_components()
```

<Warning>
Force fields cannot model bond breaking/forming, electron transfer, or excited states. For chemical reactions, you need quantum mechanical methods!
</Warning>

## 3. Quantum Mechanical Methods

### Ab Initio Methods

<Definition term="Ab Initio">
Latin for "from the beginning." These methods solve the Schrödinger equation using only fundamental constants, without empirical parameters (except for the basis set).
</Definition>

```python
def ab_initio_hierarchy():
    """Display the ab initio method hierarchy."""

    methods = {
        'Hartree-Fock (HF)': {
            'approximation': 'Mean-field (no correlation)',
            'wavefunction': 'Single Slater determinant',
            'error': '~10-50 kcal/mol for reactions'
        },
        'MP2 (Perturbation Theory)': {
            'approximation': 'Second-order perturbation',
            'wavefunction': 'HF + doubles excitations',
            'error': '~5-10 kcal/mol'
        },
        'CISD': {
            'approximation': 'CI with singles/doubles',
            'wavefunction': 'Linear combination of determinants',
            'error': '~5 kcal/mol'
        },
        'CCSD': {
            'approximation': 'Coupled cluster singles/doubles',
            'wavefunction': 'Exponential ansatz',
            'error': '~2-5 kcal/mol'
        },
        'CCSD(T)': {
            'approximation': 'CCSD + perturbative triples',
            'wavefunction': 'Exponential + triples correction',
            'error': '~1 kcal/mol ("gold standard")'
        },
        'Full CI': {
            'approximation': 'None (exact for basis)',
            'wavefunction': 'All possible determinants',
            'error': 'Basis set limit only'
        }
    }

    print("Ab Initio Method Hierarchy")
    print("=" * 75)
    for name, info in methods.items():
        print(f"\n{name}:")
        print(f"  Approximation: {info['approximation']}")
        print(f"  Wavefunction: {info['wavefunction']}")
        print(f"  Typical error: {info['error']}")

ab_initio_hierarchy()
```

### Density Functional Theory (DFT)

<Definition term="Density Functional Theory (DFT)">
A quantum mechanical method that uses the electron density ρ(r) rather than the wavefunction as the fundamental variable. It offers a good balance of accuracy and computational cost.
</Definition>

<Formula>
$$E[\rho] = T_s[\rho] + E_{ext}[\rho] + E_H[\rho] + E_{xc}[\rho]$$
</Formula>

```python

def method_selector(problem_type):
    """Suggest appropriate computational methods for different problems."""

    recommendations = {
        'geometry_optimization': {

            'accurate': 'DFT (ωB97X-D/def2-TZVP)',

            'quick': 'DFT (M06-2X/6-311+G(d,p))',

            'benchmark': 'MRCI or CASPT2'

            'quick': 'DFT (B3LYP)',

            'accurate': 'ωB97M-V',

            'quick': 'Molecular Mechanics (AMBER)',
            'accurate': 'QM/MM (DFT/MM)',
```

## 5. Why Quantum Computers for Chemistry?

The fundamental limitation of classical computers for chemistry is the exponential scaling of the wavefunction:

<Definition term="Exponential Wall">
The exponential growth of computational resources needed to exactly solve the Schrödinger equation as the number of electrons increases. This makes exact classical solutions intractable for molecules beyond ~20 electrons.
</Definition>

```python
import numpy as np
from math import comb

def classical_vs_quantum_scaling(n_electrons_list, n_orbitals=None):
    """Compare classical FCI vs quantum simulation resources."""

    print("Classical vs Quantum Resources for Chemistry")
    print("=" * 65)
    print(f"{'Electrons':<12} {'Orbitals':<10} {'FCI Determinants':<20} {'Qubits'}")
    print("-" * 65)

    for n_e in n_electrons_list:
        n_orb = n_orbitals if n_orbitals else n_e * 2
        n_spin_orb = 2 * n_orb

        # FCI: number of determinants
        n_alpha = n_e // 2
        n_beta = n_e - n_alpha
        n_det = comb(n_orb, n_alpha) * comb(n_orb, n_beta)

        # Quantum: qubits = spin orbitals
        n_qubits = n_spin_orb

        # Format large numbers
        if n_det > 1e12:
            det_str = f"{n_det:.2e}"
        elif n_det > 1e6:
            det_str = f"{n_det/1e6:.1f}M"
        else:
            det_str = str(n_det)

        print(f"{n_e:<12} {n_orb:<10} {det_str:<20} {n_qubits}")

    print("\nKey insight: Classical scales exponentially, quantum scales polynomially!")

classical_vs_quantum_scaling([4, 8, 12, 16, 20, 30, 50])
```

Output:
```
Classical vs Quantum Resources for Chemistry
=================================================================
Electrons    Orbitals   FCI Determinants     Qubits
-----------------------------------------------------------------
4            8          784                  16
8            16         1.7M                 32
12           24         1.0e+09              48
16           32         6.0e+12              64
20           40         4.0e+16              80
30           60         1.0e+26              120
50           100        1.0e+47              200

Key insight: Classical scales exponentially, quantum scales polynomially!
```

<Tip>
Quantum computers naturally represent quantum states! A 50-qubit quantum computer can, in principle, simulate wavefunctions that would require more storage than all classical computers on Earth combined.
</Tip>
