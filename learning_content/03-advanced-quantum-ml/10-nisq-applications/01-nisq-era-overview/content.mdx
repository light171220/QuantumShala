## 1. What is NISQ?

The term NISQ was coined by John Preskill in 2018 to describe the current generation of quantum computers.

<Definition term="NISQ">
Noisy Intermediate-Scale Quantum devices are quantum computers with 50-1000+ qubits that have significant noise and limited error correction, distinguishing them from both small research devices and future fault-tolerant quantum computers.
</Definition>

## 2. Key NISQ Limitations

Understanding limitations helps design algorithms that work within constraints.

```python
import numpy as np

def analyze_nisq_limitations():
    """
    Analyze key NISQ limitations and their impacts.
    """
    print("NISQ Limitations Analysis:")
    print("=" * 50)

    # Circuit depth limitation
    print("\n1. CIRCUIT DEPTH LIMITATION")
    print("-" * 40)

    def estimate_max_depth(t2_us, gate_time_ns, fidelity_per_gate):
        """Estimate maximum practical circuit depth."""
        # Coherence limit
        coherence_limit = (t2_us * 1000) / gate_time_ns

        # Fidelity limit (for 50% success probability)
        fidelity_limit = np.log(0.5) / np.log(fidelity_per_gate)

        return min(coherence_limit, fidelity_limit)

    scenarios = [
        {'name': 'Superconducting (good)', 't2': 100, 'gate': 50, 'fidelity': 0.995},
        {'name': 'Superconducting (avg)', 't2': 50, 'gate': 100, 'fidelity': 0.99},
        {'name': 'Trapped ion', 't2': 1000, 'gate': 100000, 'fidelity': 0.999},
    ]

    for s in scenarios:
        max_d = estimate_max_depth(s['t2'], s['gate'], s['fidelity'])
        print(f"  {s['name']}: ~{max_d:.0f} gates max")

    # Shot noise
    print("\n2. SHOT NOISE (SAMPLING)")
    print("-" * 40)

    def shot_noise_error(n_shots, probability=0.5):
        """Standard error of probability estimate."""
        return np.sqrt(probability * (1 - probability) / n_shots)

    for shots in [100, 1000, 10000, 100000]:
        error = shot_noise_error(shots)
        print(f"  {shots:6d} shots: ±{error:.4f} standard error")

    # Connectivity overhead
    print("\n3. CONNECTIVITY OVERHEAD")
    print("-" * 40)

    def swap_overhead(n_qubits, connectivity='linear'):
        """Estimate SWAP gates needed for all-to-all."""
        if connectivity == 'all-to-all':
            return 0
        elif connectivity == 'linear':
            return n_qubits * (n_qubits - 1) // 4  # Average
        elif connectivity == 'grid':
            return int(n_qubits * np.sqrt(n_qubits) / 4)

    for n in [4, 8, 16, 32]:
        linear_swaps = swap_overhead(n, 'linear')
        grid_swaps = swap_overhead(n, 'grid')
        print(f"  {n} qubits: ~{linear_swaps} SWAPs (linear), "
              f"~{grid_swaps} SWAPs (grid)")

analyze_nisq_limitations()
```

<Warning>
Circuit depth is the primary constraint in NISQ computing. Design algorithms with minimal depth, even if it means using more qubits or shots.
</Warning>

## 3. NISQ-Friendly Algorithm Design

Successful NISQ algorithms share common design principles.

<Formula>
NISQ algorithm success probability:
$$P_{success} \approx F_{gate}^{N_{gates}} \cdot F_{meas}^{N_{qubits}} \cdot e^{-t/T_2}$$
</Formula>

```python

def nisq_algorithm_principles():
    """
    Key principles for NISQ algorithm design.
    """
    print("NISQ Algorithm Design Principles:")
    print("=" * 50)

    principles = [
        {
            'name': 'Shallow Circuits',
            'description': 'Keep circuit depth logarithmic or constant in problem size',
            'example': 'Use hardware-efficient ansätze over problem-inspired ones',
            'trade_off': 'May require more parameters or iterations'
        },
        {
            'name': 'Variational Approach',
            'description': 'Use parameterized circuits optimized classically',
            'example': 'VQE, QAOA, variational classifiers',
            'trade_off': 'Susceptible to barren plateaus'
        },
        {
            'name': 'Error Mitigation',
            'description': 'Use techniques to reduce noise impact',
            'example': 'Zero-noise extrapolation, probabilistic error cancellation',
            'trade_off': 'Increases shot count'
        },
        {
            'name': 'Hybrid Classical-Quantum',
            'description': 'Offload computation to classical where possible',
            'example': 'Classical pre-processing, quantum kernel estimation',
            'trade_off': 'May not achieve full quantum speedup'
        },
        {
            'name': 'Native Gate Decomposition',
            'description': 'Compile to hardware-native gates',
            'example': 'Use CZ instead of CNOT if native',
            'trade_off': 'Less portable across platforms'
        }
    ]

    for i, p in enumerate(principles, 1):
        print(f"\n{i}. {p['name']}")
        print(f"   Description: {p['description']}")
        print(f"   Example: {p['example']}")
        print(f"   Trade-off: {p['trade_off']}")

def success_probability_calculator():
    """
    Calculate expected success probability for a circuit.
    """
    print("\n\nSuccess Probability Calculator:")
    print("=" * 50)

    def calc_success_prob(n_qubits, n_single_gates, n_two_gates,
                         single_fidelity, two_fidelity, meas_fidelity,
                         circuit_time_us, t2_us):
        """Calculate overall success probability."""
        # Gate fidelity contribution
        gate_success = (single_fidelity ** n_single_gates *
                       two_fidelity ** n_two_gates)

        # Measurement fidelity
        meas_success = meas_fidelity ** n_qubits

        # Coherence
        coherence_factor = np.exp(-circuit_time_us / t2_us)

        total = gate_success * meas_success * coherence_factor

        return {
            'gate': gate_success,
            'measurement': meas_success,
            'coherence': coherence_factor,
            'total': total
        }

    # Example circuits
    circuits = [
        {'name': 'Shallow VQC', 'qubits': 4, 'single': 20, 'two': 6, 'time': 1},
        {'name': 'Medium QAOA', 'qubits': 8, 'single': 50, 'two': 20, 'time': 5},
        {'name': 'Deep VQE', 'qubits': 10, 'single': 100, 'two': 50, 'time': 15},
    ]

    # Device parameters
    device = {
        'single_fidelity': 0.999,
        'two_fidelity': 0.99,
        'meas_fidelity': 0.97,
        't2_us': 100
    }

    print(f"\nDevice: {device}")
    print("\n" + "-" * 60)

    for c in circuits:
        prob = calc_success_prob(
            c['qubits'], c['single'], c['two'],
            device['single_fidelity'], device['two_fidelity'],
            device['meas_fidelity'], c['time'], device['t2_us']
        )
        print(f"\n{c['name']} ({c['qubits']}q, {c['two']} 2Q gates):")
        print(f"  Gate success: {prob['gate']:.3%}")
        print(f"  Measurement:  {prob['measurement']:.3%}")
        print(f"  Coherence:    {prob['coherence']:.3%}")
        print(f"  TOTAL:        {prob['total']:.3%}")

nisq_algorithm_principles()
success_probability_calculator()
```

## Summary

The NISQ era presents both opportunities and challenges:

1. **Characteristics**: 50-1000+ noisy qubits, no error correction
2. **Limitations**: Shallow circuits, shot noise, sparse connectivity
3. **Design Principles**: Variational, hybrid, error-mitigated
4. **Advantage**: Practical advantage not yet demonstrated
5. **Hardware**: Multiple platforms with different trade-offs
6. **Preparation**: Design for noise, validate thoroughly

Success in the NISQ era requires honest assessment of what's possible and careful algorithm design within constraints.
