# Transfer Learning Fundamentals

Transfer learning enables reusing knowledge from one task to improve learning on another. This lesson introduces transfer learning concepts and their applications in quantum machine learning.

## Learning Objectives

After completing this lesson, you will be able to:

- Understand transfer learning concepts
- Identify transfer learning opportunities in QML
- Recognize domain shift in quantum contexts
- Apply transfer learning to reduce training costs
- Evaluate when transfer learning is beneficial
- Compare quantum and classical transfer approaches

## 1. What is Transfer Learning?

Transfer learning reuses knowledge learned from a source task to improve performance on a target task.

<Definition term="Transfer Learning">
Transfer learning is a machine learning paradigm where knowledge gained from solving one problem (source domain/task) is applied to a different but related problem (target domain/task):

Knowledge: Source Domain → Target Domain

This reduces the need for labeled data and training time on the target task.
</Definition>

```python
import numpy as np

class TransferLearningBasics:
    """
    Demonstrate transfer learning concepts.
    """

    def __init__(self):
        self.source_knowledge = None

    def train_on_source(self, source_data, source_labels):
        """
        Train on source task to extract transferable knowledge.
        """
        # Learn simple linear relationship
        X = np.column_stack([source_data, np.ones(len(source_data))])
        self.source_knowledge = np.linalg.lstsq(X, source_labels, rcond=None)[0]
        print(f"Source knowledge: {self.source_knowledge}")
        return self.source_knowledge

    def transfer_to_target(self, target_data, target_labels, freeze_ratio=0.5):
        """
        Apply transfer learning to target task.

        Args:
            freeze_ratio: Fraction of knowledge to keep frozen
        """
        if self.source_knowledge is None:
            raise ValueError("Must train on source first")

        # Initialize from source
        transferred = self.source_knowledge.copy()

        # Fine-tune on target (partially freeze source knowledge)
        X = np.column_stack([target_data, np.ones(len(target_data))])
        target_solution = np.linalg.lstsq(X, target_labels, rcond=None)[0]

        # Blend source and target knowledge
        final = freeze_ratio * transferred + (1 - freeze_ratio) * target_solution

        return final

    def evaluate(self, data, labels, weights):
        """Evaluate model performance."""
        X = np.column_stack([data, np.ones(len(data))])
        predictions = X @ weights
        mse = np.mean((predictions - labels) ** 2)
        return mse

# Demonstrate transfer learning
print("Transfer Learning Demonstration:")
print("=" * 50)

np.random.seed(42)

# Source task: y = 2x + 1
source_x = np.linspace(0, 1, 50)
source_y = 2 * source_x + 1 + 0.1 * np.random.randn(50)

# Target task: y = 2.2x + 0.8 (similar but shifted)
target_x = np.linspace(0, 1, 10)  # Less data available
target_y = 2.2 * target_x + 0.8 + 0.1 * np.random.randn(10)

tl = TransferLearningBasics()

# Train on source
print("\n1. Training on source task (50 samples):")
source_weights = tl.train_on_source(source_x, source_y)

# Transfer to target
print("\n2. Transferring to target task (10 samples):")
transferred_weights = tl.transfer_to_target(target_x, target_y, freeze_ratio=0.3)
print(f"Transferred weights: {transferred_weights}")

# Compare: train from scratch on target
scratch_X = np.column_stack([target_x, np.ones(len(target_x))])
scratch_weights = np.linalg.lstsq(scratch_X, target_y, rcond=None)[0]
print(f"From-scratch weights: {scratch_weights}")

# Evaluate on test data
test_x = np.linspace(0, 1, 20)
test_y = 2.2 * test_x + 0.8

print("\n3. Test performance:")
print(f"  Transfer learning MSE: {tl.evaluate(test_x, test_y, transferred_weights):.4f}")
print(f"  From-scratch MSE: {tl.evaluate(test_x, test_y, scratch_weights):.4f}")
print(f"  Source-only MSE: {tl.evaluate(test_x, test_y, source_weights):.4f}")
```

## 2. Transfer Learning in QML

Apply transfer learning concepts to variational quantum circuits.

<Formula>
In quantum transfer learning, we partition circuit parameters into:

θ = (θ_frozen, θ_trainable)

where θ_frozen are pre-trained parameters kept fixed, and θ_trainable are adapted to the new task.
</Formula>

```python
class QuantumTransferLearning:
    """
    Transfer learning for variational quantum circuits.
    """

    def __init__(self, n_qubits, n_layers):
        self.n_qubits = n_qubits
        self.n_layers = n_layers
        self.n_params = n_qubits * n_layers

    def initialize_circuit_params(self):
        """Random initialization."""
        return np.random.randn(self.n_params) * 0.1

    def simulate_circuit(self, params, x):
        """
        Simulate quantum circuit output.

        Architecture: encoding -> variational layers
        """
        # Simple simulation: params modify the transformation
        result = x
        for i, p in enumerate(params):
            result = np.sin(p * result + 0.1 * i)
        return result

    def train_on_source(self, source_data, source_labels, epochs=100, lr=0.1):
        """Train on source task."""
        params = self.initialize_circuit_params()

        for epoch in range(epochs):
            total_loss = 0
            grad = np.zeros_like(params)

            for x, y in zip(source_data, source_labels):
                # Forward
                pred = self.simulate_circuit(params, x)
                loss = (pred - y) ** 2
                total_loss += loss

                # Gradient (numerical)
                for i in range(len(params)):
                    eps = 0.01
                    params[i] += eps
                    pred_plus = self.simulate_circuit(params, x)
                    params[i] -= 2 * eps
                    pred_minus = self.simulate_circuit(params, x)
                    params[i] += eps

                    grad[i] += 2 * (pred - y) * (pred_plus - pred_minus) / (2 * eps)

            # Update
            params -= lr * grad / len(source_data)

            if epoch % 20 == 0:
                print(f"Epoch {epoch}: Loss = {total_loss/len(source_data):.4f}")

        return params

    def transfer_and_finetune(self, pretrained_params, target_data, target_labels,
                               frozen_layers=1, epochs=50, lr=0.05):
        """
        Transfer pre-trained parameters and fine-tune.

        Args:
            pretrained_params: Parameters from source training
            frozen_layers: Number of layers to freeze
        """
        params = pretrained_params.copy()
        frozen_idx = frozen_layers * self.n_qubits

        print(f"Freezing first {frozen_layers} layer(s) ({frozen_idx} params)")

        for epoch in range(epochs):
            total_loss = 0
            grad = np.zeros_like(params)

            for x, y in zip(target_data, target_labels):
                pred = self.simulate_circuit(params, x)
                loss = (pred - y) ** 2
                total_loss += loss

                # Only compute gradient for non-frozen params
                for i in range(frozen_idx, len(params)):
                    eps = 0.01
                    params[i] += eps
                    pred_plus = self.simulate_circuit(params, x)
                    params[i] -= 2 * eps
                    pred_minus = self.simulate_circuit(params, x)
                    params[i] += eps

                    grad[i] += 2 * (pred - y) * (pred_plus - pred_minus) / (2 * eps)

            # Update only non-frozen params
            params[frozen_idx:] -= lr * grad[frozen_idx:] / len(target_data)

            if epoch % 10 == 0:
                print(f"Fine-tune epoch {epoch}: Loss = {total_loss/len(target_data):.4f}")

        return params

# Demonstrate quantum transfer learning
print("\nQuantum Transfer Learning:")
print("=" * 50)

np.random.seed(42)

qtl = QuantumTransferLearning(n_qubits=3, n_layers=2)

# Source task data
source_x = np.linspace(-1, 1, 30)
source_y = np.sin(2 * source_x)

# Target task data (related but different)
target_x = np.linspace(-1, 1, 8)
target_y = np.sin(2.5 * target_x) + 0.2

print("Training on source task:")
pretrained = qtl.train_on_source(source_x, source_y, epochs=60)

print("\nFine-tuning on target task:")
finetuned = qtl.transfer_and_finetune(
    pretrained, target_x, target_y,
    frozen_layers=1, epochs=30
)
```

<Tip>
Early layers in quantum circuits often learn general data encoding patterns that transfer well across tasks. Freeze these and only train later layers on new tasks.
</Tip>

## 3. Domain Shift in Quantum Contexts

Understand how domain shift affects quantum transfer learning.

```python
class DomainShiftAnalysis:
    """
    Analyze domain shift between source and target.
    """

    def __init__(self):
        pass

    def compute_domain_distance(self, source_data, target_data, method='mmd'):
        """
        Compute distance between source and target domains.

        Args:
            method: 'mmd' (Maximum Mean Discrepancy) or 'kl' (KL divergence)
        """
        if method == 'mmd':
            return self._mmd(source_data, target_data)
        elif method == 'kl':
            return self._kl_divergence(source_data, target_data)

    def _mmd(self, X, Y, gamma=1.0):
        """
        Maximum Mean Discrepancy between distributions.
        """
        def rbf_kernel(x, y, gamma):
            diff = x[:, np.newaxis] - y[np.newaxis, :]
            return np.exp(-gamma * diff ** 2)

        K_XX = rbf_kernel(X, X, gamma)
        K_YY = rbf_kernel(Y, Y, gamma)
        K_XY = rbf_kernel(X, Y, gamma)

        mmd = np.mean(K_XX) + np.mean(K_YY) - 2 * np.mean(K_XY)
        return mmd

    def _kl_divergence(self, X, Y, bins=20):
        """
        KL divergence between empirical distributions.
        """
        # Create histograms
        min_val = min(X.min(), Y.min())
        max_val = max(X.max(), Y.max())

        p, _ = np.histogram(X, bins=bins, range=(min_val, max_val), density=True)
        q, _ = np.histogram(Y, bins=bins, range=(min_val, max_val), density=True)

        # Add small constant to avoid log(0)
        p = p + 1e-10
        q = q + 1e-10

        # Normalize
        p = p / p.sum()
        q = q / q.sum()

        return np.sum(p * np.log(p / q))

    def recommend_transfer_strategy(self, domain_distance):
        """
        Recommend transfer strategy based on domain distance.
        """
        if domain_distance < 0.1:
            return "Direct transfer: Source and target are very similar"
        elif domain_distance < 0.5:
            return "Fine-tuning: Moderate adaptation needed"
        elif domain_distance < 1.0:
            return "Domain adaptation: Significant shift requires adaptation techniques"
        else:
            return "Limited transfer: Domains too different, train from scratch"

# Analyze domain shift
print("\nDomain Shift Analysis:")
print("=" * 50)

np.random.seed(42)
analyzer = DomainShiftAnalysis()

# Create different domain shift scenarios
scenarios = {
    "Same distribution": (
        np.random.randn(100),
        np.random.randn(100)
    ),
    "Shifted mean": (
        np.random.randn(100),
        np.random.randn(100) + 1.0
    ),
    "Different variance": (
        np.random.randn(100),
        np.random.randn(100) * 2.0
    ),
    "Different distribution": (
        np.random.randn(100),
        np.random.uniform(-2, 2, 100)
    )
}

for name, (source, target) in scenarios.items():
    mmd = analyzer.compute_domain_distance(source, target, 'mmd')
    kl = analyzer.compute_domain_distance(source, target, 'kl')
    recommendation = analyzer.recommend_transfer_strategy(mmd)

    print(f"\n{name}:")
    print(f"  MMD distance: {mmd:.4f}")
    print(f"  KL divergence: {kl:.4f}")
    print(f"  Recommendation: {recommendation}")
```

## 4. When to Use Transfer Learning

Evaluate conditions for effective transfer.

```python
class TransferBenefitAnalyzer:
    """
    Analyze when transfer learning is beneficial.
    """

    def __init__(self):
        self.metrics = {}

    def analyze_data_efficiency(self, source_size, target_size, similarity):
        """
        Estimate data efficiency gain from transfer.

        Args:
            source_size: Number of source samples
            target_size: Number of target samples
            similarity: Domain similarity (0 to 1)
        """
        # Effective data = target + similarity * source
        effective_data = target_size + similarity * source_size * 0.5
        efficiency_ratio = effective_data / target_size

        return {
            'target_only': target_size,
            'effective_data': effective_data,
            'efficiency_ratio': efficiency_ratio
        }

    def analyze_training_cost(self, source_epochs, target_epochs_scratch,
                              target_epochs_transfer):
        """
        Compare training costs.
        """
        cost_scratch = target_epochs_scratch
        cost_transfer = target_epochs_transfer  # Source already trained

        savings = (cost_scratch - cost_transfer) / cost_scratch * 100

        return {
            'scratch_cost': cost_scratch,
            'transfer_cost': cost_transfer,
            'savings_percent': savings
        }

    def should_transfer(self, source_size, target_size, domain_similarity,
                        source_performance, target_complexity):
        """
        Decision function for whether to use transfer learning.
        """
        score = 0
        reasons = []

        # Factor 1: Limited target data
        if target_size < 100:
            score += 2
            reasons.append("Limited target data favors transfer")

        # Factor 2: Domain similarity
        if domain_similarity > 0.7:
            score += 3
            reasons.append("High domain similarity")
        elif domain_similarity > 0.4:
            score += 1
            reasons.append("Moderate domain similarity")
        else:
            score -= 2
            reasons.append("Low domain similarity (risk)")

        # Factor 3: Good source performance
        if source_performance > 0.9:
            score += 2
            reasons.append("Strong source model available")

        # Factor 4: Task complexity
        if target_complexity == 'high':
            score += 1
            reasons.append("Complex task benefits from pretraining")

        decision = score >= 3

        return {
            'should_transfer': decision,
            'confidence_score': score,
            'reasons': reasons
        }

)
print(f"  Target only: {efficiency['target_only']} samples")
```

## 5. Transfer Learning Strategies

Compare different transfer approaches.

```python
class TransferStrategies:
    """
    Different transfer learning strategies for QML.
    """

    def __init__(self, n_params):
        self.n_params = n_params

    def feature_extraction(self, pretrained_params, target_data, target_labels):
        """
        Feature extraction: freeze all pretrained layers,
        train only final classifier.
        """
        # Pretrained params are completely frozen
        # Only train new classifier head
        features = self._extract_features(pretrained_params, target_data)

        # Train simple classifier on features
        classifier = np.linalg.lstsq(
            np.column_stack([features, np.ones(len(features))]),
            target_labels, rcond=None
        )[0]

        return {'frozen': pretrained_params, 'classifier': classifier}

    def fine_tuning(self, pretrained_params, target_data, target_labels,
                    frozen_ratio=0.5, epochs=50, lr=0.05):
        """
        Fine-tuning: freeze early layers, train later layers.
        """
        params = pretrained_params.copy()
        frozen_count = int(len(params) * frozen_ratio)

        for _ in range(epochs):
            for x, y in zip(target_data, target_labels):
                pred = self._forward(params, x)
                # Only update non-frozen params
                grad = self._gradient(params, x, y)
                grad[:frozen_count] = 0  # Zero gradient for frozen
                params -= lr * grad

        return params

    def full_finetuning(self, pretrained_params, target_data, target_labels,
                        epochs=50, lr=0.01):
        """
        Full fine-tuning: train all parameters with small LR.
        """
        params = pretrained_params.copy()

        for _ in range(epochs):
            for x, y in zip(target_data, target_labels):
                grad = self._gradient(params, x, y)
                params -= lr * grad

        return params

    def progressive_unfreezing(self, pretrained_params, target_data, target_labels,
                               epochs_per_layer=20, lr=0.05):
        """
        Progressive unfreezing: gradually unfreeze from last to first.
        """
        params = pretrained_params.copy()
        n_layers = 4  # Assume 4 logical layers
        params_per_layer = len(params) // n_layers

        for layer_idx in range(n_layers - 1, -1, -1):
            print(f"  Unfreezing from layer {layer_idx}")
            frozen_end = layer_idx * params_per_layer

            for _ in range(epochs_per_layer):
                for x, y in zip(target_data, target_labels):
                    grad = self._gradient(params, x, y)
                    grad[:frozen_end] = 0
                    params -= lr * grad

        return params

    def _forward(self, params, x):
        result = x
        for p in params:
            result = np.tanh(p * result)
        return result

    def _gradient(self, params, x, y, eps=0.01):
        grad = np.zeros_like(params)
        pred = self._forward(params, x)

        for i in range(len(params)):
            params[i] += eps
            pred_plus = self._forward(params, x)
            params[i] -= eps

            grad[i] = 2 * (pred - y) * (pred_plus - pred) / eps

        return grad

    def _extract_features(self, params, data):
        return np.array([self._forward(params, x) for x in data])

# Compare strategies
print("\nTransfer Learning Strategies Comparison:")
print("=" * 50)

np.random.seed(42)

strategies = TransferStrategies(n_params=8)
pretrained = np.random.randn(8) * 0.5

target_x = np.linspace(-1, 1, 15)
target_y = np.sin(target_x)

print("\n1. Feature Extraction (all frozen):")
result = strategies.feature_extraction(pretrained, target_x, target_y)
print(f"   Classifier trained on extracted features")

print("\n2. Partial Fine-tuning (50% frozen):")
finetuned = strategies.fine_tuning(pretrained, target_x, target_y, frozen_ratio=0.5)
print(f"   Early layers frozen, later layers trained")

print("\n3. Full Fine-tuning (small LR):")
full = strategies.full_finetuning(pretrained, target_x, target_y)
print(f"   All parameters updated with small learning rate")

print("\n4. Progressive Unfreezing:")
progressive = strategies.progressive_unfreezing(pretrained, target_x, target_y)
print(f"   Layers unfrozen gradually from last to first")
```

## 6. Quantum vs Classical Transfer

Compare transfer learning in quantum and classical settings.

```python
class QuantumClassicalComparison:
    """
    Compare transfer learning in quantum and classical ML.
    """

    def __init__(self):
        self.comparison = {}

    def analyze_differences(self):
        """
        Key differences between quantum and classical transfer.
        """
        differences = {
            'Parameter count': {
                'classical': 'Millions of parameters',
                'quantum': 'Typically O(n_qubits × n_layers) parameters',
                'implication': 'Quantum models may transfer with less overfitting risk'
            },
            'Feature extraction': {
                'classical': 'Deep hierarchical features',
                'quantum': 'Quantum feature maps and entanglement patterns',
                'implication': 'Different types of representations transfer'
            },
            'Training cost': {
                'classical': 'GPU-hours for large models',
                'quantum': 'QPU time is expensive and limited',
                'implication': 'Transfer learning especially valuable for quantum'
            },
            'Hardware variability': {
                'classical': 'Consistent across hardware',
                'quantum': 'Different noise profiles per device',
                'implication': 'May need device-specific adaptation'
            },
            'Expressibility': {
                'classical': 'Unlimited with depth',
                'quantum': 'Barren plateaus limit depth',
                'implication': 'Pretrained initialization may help avoid barren plateaus'

        """
        Common transfer learning scenarios in QML.

                'name': 'Classical to Quantum',
```
