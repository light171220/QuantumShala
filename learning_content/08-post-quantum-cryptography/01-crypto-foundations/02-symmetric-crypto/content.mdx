## 1. Symmetric Encryption Fundamentals

Symmetric encryption is the workhorse of modern cryptography. Every time you access a secure website, send an encrypted message, or store protected data, symmetric encryption is doing the heavy lifting.

<Definition term="Symmetric Encryption">
A cryptographic system where the same secret key is used for both encryption and decryption. Both the sender and receiver must possess and protect this shared key.
</Definition>

<Formula>
$$C = E_K(P) \quad \text{and} \quad P = D_K(C)$$
</Formula>

Where the **same key K** is used for both operations.

```python
from cryptography.hazmat.primitives.ciphers import Cipher, algorithms, modes
import os

# Symmetric encryption: same key for both operations
key = os.urandom(32)  # 256-bit key

# Encryption
def encrypt_symmetric(plaintext: bytes, key: bytes) -> tuple:
    iv = os.urandom(16)
    cipher = Cipher(algorithms.AES(key), modes.CBC(iv))
    encryptor = cipher.encryptor()
    # Padding required for CBC mode
    padded = plaintext + b'\x00' * (16 - len(plaintext) % 16)
    ciphertext = encryptor.update(padded) + encryptor.finalize()
    return iv, ciphertext

# Decryption uses the SAME key
def decrypt_symmetric(iv: bytes, ciphertext: bytes, key: bytes) -> bytes:
    cipher = Cipher(algorithms.AES(key), modes.CBC(iv))
    decryptor = cipher.decryptor()
    plaintext = decryptor.update(ciphertext) + decryptor.finalize()
    return plaintext.rstrip(b'\x00')

# Demo
message = b"Secret message!"
iv, encrypted = encrypt_symmetric(message, key)
decrypted = decrypt_symmetric(iv, encrypted, key)
print(f"Original: {message}")
print(f"Decrypted: {decrypted}")
```

## 2. The Key Distribution Problem

The fundamental challenge of symmetric cryptography:

<Warning>
**The Key Distribution Problem**: How do Alice and Bob share a secret key if they've never met and all their communications can be intercepted?
</Warning>

### Possible Solutions

| Solution | Pros | Cons |
|----------|------|------|
| **Physical exchange** | Very secure | Impractical at scale |
| **Trusted courier** | Works for high-value | Expensive, slow |
| **Key Derivation** | No transmission | Requires shared secret |
| **Asymmetric crypto** | Solves distribution | Slower, quantum-vulnerable |

<Tip>
In practice, we use **hybrid encryption**: Asymmetric cryptography (like RSA or key exchange) to establish a shared symmetric key, then use that symmetric key for fast bulk encryption.
</Tip>

```python
# Hybrid encryption concept
class HybridEncryption:
    def encrypt_message(self, recipient_public_key, message):
        # 1. Generate random symmetric key
        symmetric_key = os.urandom(32)
        
        # 2. Encrypt the symmetric key with recipient's public key (asymmetric)
        encrypted_key = rsa_encrypt(recipient_public_key, symmetric_key)
        
        # 3. Encrypt the actual message with symmetric key (fast!)
        encrypted_message = aes_encrypt(symmetric_key, message)
        
        return encrypted_key, encrypted_message
```

## 3. Block Ciphers vs Stream Ciphers

Symmetric ciphers come in two flavors:

### Block Ciphers

<Definition term="Block Cipher">
An encryption algorithm that operates on fixed-size blocks of data (typically 128 bits). The same key transforms each block independently.
</Definition>

**Examples**: AES, DES, 3DES, Blowfish

```
Plaintext:  [Block 1][Block 2][Block 3][Block 4]
                ↓        ↓        ↓        ↓
            E_K(B1)  E_K(B2)  E_K(B3)  E_K(B4)
                ↓        ↓        ↓        ↓
Ciphertext: [Cipher1][Cipher2][Cipher3][Cipher4]
```

### Stream Ciphers

<Definition term="Stream Cipher">
An encryption algorithm that encrypts data one bit or byte at a time by XORing with a pseudorandom keystream generated from the key.
</Definition>

**Examples**: ChaCha20, RC4 (deprecated), Salsa20

```
Key → [Keystream Generator] → Keystream: k₁k₂k₃k₄k₅...
                                           ⊕
Plaintext:                               p₁p₂p₃p₄p₅...
                                           ↓
Ciphertext:                              c₁c₂c₃c₄c₅...
```

<Formula>
$$c_i = p_i \oplus k_i$$
</Formula>

```python
# Stream cipher concept (simplified)
def stream_cipher_encrypt(plaintext: bytes, keystream: bytes) -> bytes:
    """XOR plaintext with keystream"""
    return bytes(p ^ k for p, k in zip(plaintext, keystream))

# Stream cipher decryption is identical (XOR is its own inverse)
def stream_cipher_decrypt(ciphertext: bytes, keystream: bytes) -> bytes:
    return bytes(c ^ k for c, k in zip(ciphertext, keystream))

# Demo
import os
plaintext = b"Hello!"
keystream = os.urandom(len(plaintext))

ciphertext = stream_cipher_encrypt(plaintext, keystream)
recovered = stream_cipher_decrypt(ciphertext, keystream)

print(f"Plaintext: {plaintext}")
print(f"Keystream: {keystream.hex()}")
print(f"Ciphertext: {ciphertext.hex()}")
print(f"Recovered: {recovered}")
```

### Comparison

| Feature | Block Cipher | Stream Cipher |
|---------|-------------|---------------|
| Operation | Fixed-size blocks | Bit/byte at a time |
| Speed | Fast with hardware | Very fast in software |
| Parallelization | Mode-dependent | Limited |
| Error propagation | Mode-dependent | Single bit |
| Use cases | General purpose | Real-time, streaming |

## 4. AES: The Gold Standard

The **Advanced Encryption Standard (AES)** is the most widely used symmetric cipher in the world.

<Definition term="AES (Advanced Encryption Standard)">
A symmetric block cipher adopted by NIST in 2001 after a public competition. It operates on 128-bit blocks with key sizes of 128, 192, or 256 bits.
</Definition>

### AES Parameters

| Variant | Key Size | Rounds | Security Level |
|---------|----------|--------|----------------|
| AES-128 | 128 bits | 10 | 128 bits |
| AES-192 | 192 bits | 12 | 192 bits |
| AES-256 | 256 bits | 14 | 256 bits |

### AES Structure

Each round consists of four operations:

```
    ┌─────────────────┐
    │   Plaintext     │ (128 bits)
    └────────┬────────┘
             ↓
    ┌─────────────────┐
    │  AddRoundKey    │ ← Round Key 0
    └────────┬────────┘
             ↓
    ┌─────────────────────────────────┐
    │         Round (×9/11/13)        │
    │  ┌───────────────────────────┐  │
    │  │      SubBytes            │  │ ← S-box substitution
    │  │      ShiftRows           │  │ ← Row permutation
    │  │      MixColumns          │  │ ← Column mixing
    │  │      AddRoundKey         │  │ ← XOR with round key
    │  └───────────────────────────┘  │
    └────────────────┬────────────────┘
                     ↓
    ┌─────────────────────────────────┐
    │         Final Round             │
    │      SubBytes                   │
    │      ShiftRows                  │
    │      AddRoundKey                │ ← Round Key N
    └────────────────┬────────────────┘
                     ↓
    ┌─────────────────┐
    │   Ciphertext    │ (128 bits)
    └─────────────────┘
```

<Example>
**AES Encryption in Python**

```python
from cryptography.hazmat.primitives.ciphers import Cipher, algorithms, modes
from cryptography.hazmat.backends import default_backend
import os

def aes_encrypt_block(plaintext: bytes, key: bytes) -> bytes:
    """
    Encrypt a single 128-bit block with AES (ECB mode for demonstration).
    WARNING: Never use ECB mode in practice!
    """
    cipher = Cipher(algorithms.AES(key), modes.ECB(), backend=default_backend())
    encryptor = cipher.encryptor()
    return encryptor.update(plaintext) + encryptor.finalize()

def aes_decrypt_block(ciphertext: bytes, key: bytes) -> bytes:
    """Decrypt a single AES block."""
    cipher = Cipher(algorithms.AES(key), modes.ECB(), backend=default_backend())
    decryptor = cipher.decryptor()
    return decryptor.update(ciphertext) + decryptor.finalize()

# Generate a random 256-bit key
key = os.urandom(32)  # 32 bytes = 256 bits

# Plaintext must be exactly 16 bytes (128 bits) for a single block
plaintext = b"AES is awesome!!"  # Exactly 16 bytes
assert len(plaintext) == 16

ciphertext = aes_encrypt_block(plaintext, key)
decrypted = aes_decrypt_block(ciphertext, key)

print(f"Key (hex): {key.hex()}")
print(f"Plaintext: {plaintext}")
print(f"Ciphertext (hex): {ciphertext.hex()}")
print(f"Decrypted: {decrypted}")
```
</Example>

## 5. Block Cipher Modes of Operation

A block cipher alone can only encrypt one block. **Modes of operation** extend block ciphers to handle messages of any length.

### ECB Mode (Electronic Codebook) - NEVER USE!

<Formula>
$$C_i = E_K(P_i)$$
</Formula>

Each block is encrypted independently.

<Warning>
**ECB mode is insecure!** Identical plaintext blocks produce identical ciphertext blocks, leaking patterns. The famous "ECB penguin" demonstrates this flaw.
</Warning>

```python
# Why ECB is dangerous - pattern preservation
from PIL import Image
import numpy as np

# Conceptual demonstration (simplified)
# If we encrypt an image with ECB:
# - Identical pixel blocks → identical encrypted blocks
# - Image patterns remain visible!

# This is why the "ECB penguin" is famous in cryptography education
```

### CBC Mode (Cipher Block Chaining)

<Formula>
$$C_i = E_K(P_i \oplus C_{i-1}), \quad C_0 = IV$$
</Formula>

Each block is XORed with the previous ciphertext block before encryption.

```python
from cryptography.hazmat.primitives.ciphers import Cipher, algorithms, modes
from cryptography.hazmat.primitives import padding
import os

def aes_cbc_encrypt(plaintext: bytes, key: bytes) -> tuple:
    """Encrypt with AES-CBC mode."""
    iv = os.urandom(16)
    
    # Pad plaintext to block size
    padder = padding.PKCS7(128).padder()
    padded_data = padder.update(plaintext) + padder.finalize()
    
    cipher = Cipher(algorithms.AES(key), modes.CBC(iv))
    encryptor = cipher.encryptor()
    ciphertext = encryptor.update(padded_data) + encryptor.finalize()
    
    return iv, ciphertext

def aes_cbc_decrypt(iv: bytes, ciphertext: bytes, key: bytes) -> bytes:
    """Decrypt with AES-CBC mode."""
    cipher = Cipher(algorithms.AES(key), modes.CBC(iv))
    decryptor = cipher.decryptor()
    padded_plaintext = decryptor.update(ciphertext) + decryptor.finalize()
    
    # Remove padding
    unpadder = padding.PKCS7(128).unpadder()
    plaintext = unpadder.update(padded_plaintext) + unpadder.finalize()
    
    return plaintext

# Example
key = os.urandom(32)
message = b"This is a longer message that spans multiple AES blocks!"

iv, ciphertext = aes_cbc_encrypt(message, key)
decrypted = aes_cbc_decrypt(iv, ciphertext, key)

print(f"Original: {message}")
print(f"IV: {iv.hex()}")
print(f"Ciphertext: {ciphertext.hex()}")
print(f"Decrypted: {decrypted}")
```

### CTR Mode (Counter)

<Formula>
$$C_i = P_i \oplus E_K(\text{Nonce} \| \text{Counter}_i)$$
</Formula>

Turns a block cipher into a stream cipher.

```python
from cryptography.hazmat.primitives.ciphers import Cipher, algorithms, modes
import os

def aes_ctr_encrypt(plaintext: bytes, key: bytes) -> tuple:
    """Encrypt with AES-CTR mode (no padding needed!)."""
    nonce = os.urandom(16)
    cipher = Cipher(algorithms.AES(key), modes.CTR(nonce))
    encryptor = cipher.encryptor()
    ciphertext = encryptor.update(plaintext) + encryptor.finalize()
    return nonce, ciphertext

def aes_ctr_decrypt(nonce: bytes, ciphertext: bytes, key: bytes) -> bytes:
    """Decrypt with AES-CTR mode."""
    cipher = Cipher(algorithms.AES(key), modes.CTR(nonce))
    decryptor = cipher.decryptor()
    return decryptor.update(ciphertext) + decryptor.finalize()

# CTR mode advantages: no padding, parallelizable, seekable
key = os.urandom(32)
message = b"CTR mode is very flexible!"  # Any length works

nonce, ciphertext = aes_ctr_encrypt(message, key)
decrypted = aes_ctr_decrypt(nonce, ciphertext, key)

print(f"Message length: {len(message)} bytes")
print(f"Ciphertext length: {len(ciphertext)} bytes")  # Same length!
print(f"Decrypted: {decrypted}")
```

### GCM Mode (Galois/Counter Mode) - RECOMMENDED

<Definition term="GCM (Galois/Counter Mode)">
An authenticated encryption mode that provides both confidentiality and integrity. It combines CTR mode encryption with GHASH authentication.
</Definition>

```python
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
import os

def aes_gcm_encrypt(plaintext: bytes, key: bytes, 
                    associated_data: bytes = None) -> tuple:
    """
    Encrypt with AES-GCM (authenticated encryption).
    Returns (nonce, ciphertext_with_tag).
    """
    aesgcm = AESGCM(key)
    nonce = os.urandom(12)  # 96-bit nonce recommended for GCM
    ciphertext = aesgcm.encrypt(nonce, plaintext, associated_data)
    return nonce, ciphertext

def aes_gcm_decrypt(nonce: bytes, ciphertext: bytes, key: bytes,
                    associated_data: bytes = None) -> bytes:
    """
    Decrypt with AES-GCM. Raises exception if authentication fails!
    """
    aesgcm = AESGCM(key)
    return aesgcm.decrypt(nonce, ciphertext, associated_data)

# Example with associated data (authenticated but not encrypted)
key = os.urandom(32)
plaintext = b"Secret payload"
associated_data = b"header:user123"  # Authenticated but visible

nonce, ciphertext = aes_gcm_encrypt(plaintext, key, associated_data)

# Successful decryption
decrypted = aes_gcm_decrypt(nonce, ciphertext, key, associated_data)
print(f"Decrypted: {decrypted}")

# Tampered ciphertext will fail authentication
try:
    tampered = bytearray(ciphertext)
    tampered[0] ^= 0xFF  # Flip bits
    aes_gcm_decrypt(nonce, bytes(tampered), key, associated_data)
except Exception as e:
    print(f"Tampering detected: {type(e).__name__}")
```

<Tip>
**Always use authenticated encryption (like GCM) in practice!** Encryption without authentication allows attackers to modify ciphertext in ways that cause predictable changes to the plaintext.
</Tip>

### Mode Comparison

| Mode | Confidentiality | Integrity | Parallelizable | IV Reuse Safe |
|------|-----------------|-----------|----------------|---------------|
| ECB | Weak | No | Yes | N/A |
| CBC | Yes | No | Decrypt only | No |
| CTR | Yes | No | Yes | **NO!** |
| GCM | Yes | **Yes** | Yes | **NO!** |

## 6. ChaCha20-Poly1305: The Modern Alternative

While AES dominates hardware, **ChaCha20** excels in software.

<Definition term="ChaCha20-Poly1305">
A modern authenticated encryption algorithm combining the ChaCha20 stream cipher with the Poly1305 message authentication code. It's the primary alternative to AES-GCM.
</Definition>

```python
from cryptography.hazmat.primitives.ciphers.aead import ChaCha20Poly1305
import os

def chacha_encrypt(plaintext: bytes, key: bytes, aad: bytes = None) -> tuple:
    """Encrypt with ChaCha20-Poly1305."""
    chacha = ChaCha20Poly1305(key)
    nonce = os.urandom(12)
    ciphertext = chacha.encrypt(nonce, plaintext, aad)
    return nonce, ciphertext

def chacha_decrypt(nonce: bytes, ciphertext: bytes, key: bytes, 
                   aad: bytes = None) -> bytes:
    """Decrypt with ChaCha20-Poly1305."""
    chacha = ChaCha20Poly1305(key)
    return chacha.decrypt(nonce, ciphertext, aad)

# ChaCha20 uses 256-bit keys
key = os.urandom(32)
message = b"ChaCha20 is fast in software!"

nonce, ct = chacha_encrypt(message, key)
pt = chacha_decrypt(nonce, ct, key)
print(f"Decrypted: {pt}")
```

### AES-GCM vs ChaCha20-Poly1305

| Feature | AES-GCM | ChaCha20-Poly1305 |
|---------|---------|-------------------|
| Hardware support | Excellent (AES-NI) | Limited |
| Software performance | Good | Excellent |
| Side-channel resistance | Requires care | Better by design |
| Standardization | Older, more widespread | Newer, TLS 1.3 |

## 7. Symmetric Crypto and Quantum Computers

Good news: Symmetric cryptography is relatively quantum-resistant!

### Grover's Algorithm Impact

<Formula>
$$\text{Grover's speedup: } O(2^n) \rightarrow O(2^{n/2})$$
</Formula>

| Algorithm | Classical Security | Post-Quantum Security |
|-----------|-------------------|----------------------|
| AES-128 | 128 bits | 64 bits |
| AES-192 | 192 bits | 96 bits |
| AES-256 | 256 bits | **128 bits** ✓ |
| ChaCha20 | 256 bits | **128 bits** ✓ |

<Example>
**Why AES-256 Survives Quantum Attacks**

```python
# Classical brute-force on AES-128
classical_ops_128 = 2**128
print(f"Classical AES-128 attack: {classical_ops_128:.2e} operations")

# Grover's attack on AES-128 (square root speedup)
quantum_ops_128 = 2**64
print(f"Quantum AES-128 attack: {quantum_ops_128:.2e} operations")
print("This is feasible with a large quantum computer!")

# Grover's attack on AES-256
quantum_ops_256 = 2**128
print(f"\nQuantum AES-256 attack: {quantum_ops_256:.2e} operations")
print("Still computationally infeasible!")

# Comparison
print(f"\nAES-256 post-quantum security: {quantum_ops_256 / quantum_ops_128:.0e}x harder than AES-128")
```

Output:
```
Classical AES-128 attack: 3.40e+38 operations
Quantum AES-128 attack: 1.84e+19 operations
This is feasible with a large quantum computer!

Quantum AES-256 attack: 3.40e+38 operations
Still computationally infeasible!

AES-256 post-quantum security: 1.84e+19x harder than AES-128
```
</Example>

<Tip>
**Recommendation**: Use AES-256 or ChaCha20 (256-bit key) for quantum-resistant symmetric encryption. The migration is simple—just use a longer key!
</Tip>

## 8. Best Practices for Symmetric Cryptography

### Do's ✓

1. **Use authenticated encryption**: AES-GCM or ChaCha20-Poly1305
2. **Use 256-bit keys** for quantum resistance
3. **Generate keys with cryptographic RNG**: `os.urandom()` or `secrets` module
4. **Never reuse nonces/IVs** with the same key
5. **Use established libraries**: Don't implement crypto yourself

### Don'ts ✗

1. **Never use ECB mode**
2. **Don't use the same key for different purposes**
3. **Don't use predictable IVs**
4. **Don't ignore authentication tags**
5. **Don't use deprecated algorithms** (DES, 3DES, RC4)

```python
import secrets
from cryptography.hazmat.primitives.ciphers.aead import AESGCM

# CORRECT: Secure key generation and usage
def secure_encrypt(plaintext: bytes) -> dict:
    """Example of secure symmetric encryption."""
    # Generate key with cryptographic RNG
    key = secrets.token_bytes(32)  # 256 bits
    
    # Use authenticated encryption
    aesgcm = AESGCM(key)
    
    # Generate unique nonce
    nonce = secrets.token_bytes(12)
    
    # Encrypt with authentication
    ciphertext = aesgcm.encrypt(nonce, plaintext, None)
    
    return {
        'key': key,  # Store securely!
        'nonce': nonce,
        'ciphertext': ciphertext
    }
```
