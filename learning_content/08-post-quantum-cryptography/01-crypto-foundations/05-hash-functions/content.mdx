## 1. What Are Hash Functions?

Hash functions are one of the most versatile tools in cryptography—used in almost every security protocol.

<Definition term="Cryptographic Hash Function">
A mathematical function that takes an input of arbitrary size and produces a fixed-size output (digest or hash) with specific security properties: determinism, efficiency, pre-image resistance, second pre-image resistance, and collision resistance.
</Definition>

<Formula>
$$H: \{0,1\}^* \rightarrow \{0,1\}^n$$
</Formula>

The hash function $H$ maps any input (of any length) to a fixed-size output of $n$ bits.

```python
import hashlib

def hash_demo():
    """Demonstrate basic hash function properties"""
    
    # Same input always produces same output (deterministic)
    message = b"Hello, World!"
    hash1 = hashlib.sha256(message).hexdigest()
    hash2 = hashlib.sha256(message).hexdigest()
    
    print(f"Message: {message}")
    print(f"Hash 1: {hash1}")
    print(f"Hash 2: {hash2}")
    print(f"Deterministic: {hash1 == hash2}")
    
    # Small change completely changes the hash (avalanche effect)
    message2 = b"Hello, World?"  # Just changed ! to ?
    hash3 = hashlib.sha256(message2).hexdigest()
    
    print(f"\nModified message: {message2}")
    print(f"Modified hash: {hash3}")
    print(f"Hashes match: {hash1 == hash3}")
    
    # Fixed output size regardless of input size
    small = hashlib.sha256(b"Hi").hexdigest()
    large = hashlib.sha256(b"A" * 1000000).hexdigest()
    
    print(f"\nHash of 2 bytes: {len(small)} hex chars = {len(small)*4} bits")
    print(f"Hash of 1M bytes: {len(large)} hex chars = {len(large)*4} bits")

hash_demo()
```

## 2. Essential Properties of Cryptographic Hash Functions

### Property 1: Pre-image Resistance (One-Way)

<Definition term="Pre-image Resistance">
Given a hash value $h$, it should be computationally infeasible to find any message $m$ such that $H(m) = h$.
</Definition>

```
    m  ────────▶  H(m) = h
    ?  ◀──────── h          ← Computationally infeasible!
```

### Property 2: Second Pre-image Resistance

<Definition term="Second Pre-image Resistance">
Given a message $m_1$, it should be computationally infeasible to find a different message $m_2$ such that $H(m_1) = H(m_2)$.
</Definition>

<Warning>
This is different from collision resistance! Here, the first message is fixed, and the attacker tries to find a second message with the same hash.
</Warning>

### Property 3: Collision Resistance

<Definition term="Collision Resistance">
It should be computationally infeasible to find any two different messages $m_1 \neq m_2$ such that $H(m_1) = H(m_2)$.
</Definition>

```python
def security_properties_demo():
    """Demonstrate why hash security properties matter"""
    
    print("Security Properties of Hash Functions")
    print("=" * 50)
    
    # Pre-image resistance
    print("\n1. PRE-IMAGE RESISTANCE")
    print("   Given: h = 'a591a6d40bf420404a011733cfb7b190d62c65bf0bcda32b57b277d9ad9f146e'")
    print("   Find:  m such that SHA256(m) = h")
    print("   This would require ~2^256 operations (infeasible)")
    
    # Second pre-image resistance
    print("\n2. SECOND PRE-IMAGE RESISTANCE")
    print("   Given: m1 = 'original document'")
    print("   Find:  m2 ≠ m1 such that SHA256(m1) = SHA256(m2)")
    print("   This would require ~2^256 operations (infeasible)")
    print("   Critical for: Document integrity, code signing")
    
    # Collision resistance
    print("\n3. COLLISION RESISTANCE")
    print("   Find: ANY m1 ≠ m2 such that SHA256(m1) = SHA256(m2)")
    print("   This requires ~2^128 operations (birthday attack)")
    print("   Critical for: Digital signatures, certificates")

security_properties_demo()
```

## 3. The Birthday Attack

<Definition term="Birthday Attack">
An attack exploiting the birthday paradox to find hash collisions. For an n-bit hash, collisions can be found in approximately $2^{n/2}$ operations rather than $2^n$.
</Definition>

<Formula>
$$\text{Security against collision: } \approx 2^{n/2} \text{ operations}$$
</Formula>

```python
import math

def birthday_paradox_demo():
    """Demonstrate the birthday paradox in cryptography"""
    
    # Classical birthday problem
    def collision_probability(n_people, n_days=365):
        """Probability of at least one shared birthday"""
        if n_people >= n_days:
            return 1.0
        prob_no_collision = 1.0
        for i in range(n_people):
            prob_no_collision *= (n_days - i) / n_days
        return 1 - prob_no_collision
    
    print("Birthday Paradox")
    print("-" * 40)
    for n in [10, 20, 23, 30, 50, 70]:
        prob = collision_probability(n)
        print(f"{n} people: {prob:.1%} chance of shared birthday")
    
    print("\n50% collision probability reached at ~23 people!")
    print(f"Square root of 365 ≈ {math.sqrt(365):.1f}")
    
    # Apply to hash functions
    print("\n" + "=" * 50)
    print("Application to Hash Functions")
    print("-" * 40)
    
    hash_functions = [
        ("MD5", 128),
        ("SHA-1", 160),
        ("SHA-256", 256),
        ("SHA-512", 512),
        ("SHA3-256", 256),
    ]
    
    for name, bits in hash_functions:
        collision_ops = 2 ** (bits // 2)
        print(f"{name} ({bits}-bit): ~2^{bits//2} = {collision_ops:.2e} ops for collision")

birthday_paradox_demo()
```

## 4. SHA-2 Family

SHA-2 (Secure Hash Algorithm 2) is the current workhorse of cryptographic hashing.

<Definition term="SHA-2">
A family of cryptographic hash functions designed by the NSA, standardized by NIST in 2001. Includes SHA-224, SHA-256, SHA-384, and SHA-512 variants.
</Definition>

### SHA-2 Variants

| Variant | Output Size | Block Size | Security Level |
|---------|-------------|------------|----------------|
| SHA-224 | 224 bits | 512 bits | 112 bits |
| SHA-256 | 256 bits | 512 bits | 128 bits |
| SHA-384 | 384 bits | 1024 bits | 192 bits |
| SHA-512 | 512 bits | 1024 bits | 256 bits |
| SHA-512/256 | 256 bits | 1024 bits | 128 bits |

```python
import hashlib

def sha2_demo():
    """Demonstrate SHA-2 family"""
    
    message = b"The quick brown fox jumps over the lazy dog"
    
    print("SHA-2 Family Demonstration")
    print(f"Message: {message.decode()}")
    print("=" * 70)
    
    variants = [
        ('SHA-224', hashlib.sha224),
        ('SHA-256', hashlib.sha256),
        ('SHA-384', hashlib.sha384),
        ('SHA-512', hashlib.sha512),
    ]
    
    for name, hash_func in variants:
        digest = hash_func(message).hexdigest()
        print(f"\n{name}:")
        print(f"  Length: {len(digest)} hex chars = {len(digest)*4} bits")
        print(f"  Digest: {digest[:64]}...")

sha2_demo()
```

## 5. SHA-3 Family

SHA-3 is the newest standard, based on a completely different design than SHA-2.

<Definition term="SHA-3">
A cryptographic hash function standardized by NIST in 2015, based on the Keccak algorithm. Uses a "sponge construction" fundamentally different from SHA-2's Merkle-Damgård construction.
</Definition>

### SHA-3 Variants

| Variant | Output Size | Security Level | Special Features |
|---------|-------------|----------------|------------------|
| SHA3-224 | 224 bits | 112 bits | Standard hash |
| SHA3-256 | 256 bits | 128 bits | Standard hash |
| SHA3-384 | 384 bits | 192 bits | Standard hash |
| SHA3-512 | 512 bits | 256 bits | Standard hash |
| SHAKE128 | Variable | 128 bits | Extendable output |
| SHAKE256 | Variable | 256 bits | Extendable output |

```python
import hashlib

def sha3_demo():
    """Demonstrate SHA-3 family"""
    
    message = b"SHA-3 uses the Keccak sponge construction"
    
    print("SHA-3 Family Demonstration")
    print(f"Message: {message.decode()}")
    print("=" * 70)
    
    # Standard SHA-3
    variants = [
        ('SHA3-224', hashlib.sha3_224),
        ('SHA3-256', hashlib.sha3_256),
        ('SHA3-384', hashlib.sha3_384),
        ('SHA3-512', hashlib.sha3_512),
    ]
    
    for name, hash_func in variants:
        digest = hash_func(message).hexdigest()
        print(f"\n{name}:")
        print(f"  Length: {len(digest)*4} bits")
        print(f"  Digest: {digest[:48]}...")
    
    # SHAKE - extendable output
    print("\n" + "-" * 50)
    print("SHAKE (Extendable Output Functions)")
    
    shake = hashlib.shake_256(message)
    for length in [32, 64, 128]:
        output = shake.hexdigest(length)
        print(f"\nSHAKE256 ({length} bytes): {output[:48]}...")

sha3_demo()
```

<Tip>
**SHAKE functions** are XOF (Extendable Output Functions)—they can produce output of any desired length. This is useful for key derivation and other applications needing variable-length output.
</Tip>

### SHA-2 vs SHA-3

| Feature | SHA-2 | SHA-3 |
|---------|-------|-------|
| Design | Merkle-Damgård | Sponge construction |
| Origin | NSA | Public competition (Keccak) |
| Standardized | 2001 | 2015 |
| Length extension | Vulnerable | Resistant |
| Hardware | Mature support | Growing support |
| Performance | Generally faster | Comparable |

## 6. Common Hash Function Applications

### Application 1: Data Integrity

```python
import hashlib
import os

def verify_file_integrity():
    """Use hashes to verify file integrity"""
    
    # Original file
    original_data = b"Important document content that must not be modified"
    original_hash = hashlib.sha256(original_data).hexdigest()
    
    print("Data Integrity Verification")
    print("=" * 50)
    print(f"Original hash: {original_hash}")
    
    # Verify unchanged data
    received_data = original_data  # Same data
    received_hash = hashlib.sha256(received_data).hexdigest()
    
    print(f"\nReceived (unchanged):")
    print(f"  Hash: {received_hash}")
    print(f"  Valid: {original_hash == received_hash}")
    
    # Detect modified data
    tampered_data = b"Important document content that must not be modified!"
    tampered_hash = hashlib.sha256(tampered_data).hexdigest()
    
    print(f"\nReceived (tampered - added '!'):")
    print(f"  Hash: {tampered_hash}")
    print(f"  Valid: {original_hash == tampered_hash}")

verify_file_integrity()
```

### Application 2: Password Storage

<Warning>
**Never store passwords in plain text or with simple hashes!** Use dedicated password hashing functions like bcrypt, scrypt, or Argon2.
</Warning>

```python
import hashlib
import os
import hmac

def password_hashing_demo():
    """Demonstrate secure password hashing"""
    
    print("Password Hashing (Educational Demo)")
    print("=" * 50)
    
    password = "MySecurePassword123!"
    
    # BAD: Simple hash (DON'T USE)
    bad_hash = hashlib.sha256(password.encode()).hexdigest()
    print(f"\n❌ BAD - Simple SHA256:")
    print(f"   Hash: {bad_hash[:32]}...")
    print("   Problem: No salt, vulnerable to rainbow tables")
    
    # BETTER: Salted hash
    salt = os.urandom(16)
    salted_hash = hashlib.sha256(salt + password.encode()).hexdigest()
    print(f"\n⚠️  BETTER - Salted SHA256:")
    print(f"   Salt: {salt.hex()}")
    print(f"   Hash: {salted_hash[:32]}...")
    print("   Problem: Still too fast, vulnerable to GPU attacks")
    
    # BEST: Use a proper password hashing function
    print(f"\n✓ BEST - Use dedicated password hashing:")
    print("   • bcrypt - time-tested, widely supported")
    print("   • scrypt - memory-hard, good for high security")
    print("   • Argon2 - winner of Password Hashing Competition")
    print("   These are intentionally SLOW to prevent brute-force attacks")
    
    # Example with PBKDF2 (built into hashlib)
    dk = hashlib.pbkdf2_hmac('sha256', password.encode(), salt, 100000)
    print(f"\n✓ PBKDF2-SHA256 (100k iterations):")
    print(f"   Derived key: {dk.hex()[:32]}...")

password_hashing_demo()
```

### Application 3: HMAC (Message Authentication Code)

<Definition term="HMAC">
Hash-based Message Authentication Code. Combines a hash function with a secret key to provide both integrity and authentication.
</Definition>

<Formula>
$$\text{HMAC}(K, m) = H((K' \oplus opad) \| H((K' \oplus ipad) \| m))$$
</Formula>

```python
import hmac
import hashlib

def hmac_demo():
    """Demonstrate HMAC for message authentication"""
    
    # Shared secret key
    key = b"super_secret_key_for_authentication"
    message = b"Transfer $1000 to Account 12345"
    
    # Create HMAC
    mac = hmac.new(key, message, hashlib.sha256).hexdigest()
    
    print("HMAC Demonstration")
    print("=" * 50)
    print(f"Key: {key.decode()}")
    print(f"Message: {message.decode()}")
    print(f"HMAC-SHA256: {mac}")
    
    # Verify HMAC
    def verify_message(key, message, received_mac):
        expected_mac = hmac.new(key, message, hashlib.sha256).hexdigest()
        return hmac.compare_digest(expected_mac, received_mac)
    
    print(f"\nVerification (correct message): {verify_message(key, message, mac)}")
    
    # Tampered message fails verification
    tampered = b"Transfer $9999 to Account 12345"
    print(f"Verification (tampered message): {verify_message(key, tampered, mac)}")
    
    # Wrong key fails verification
    wrong_key = b"wrong_key"
    print(f"Verification (wrong key): {verify_message(wrong_key, message, mac)}")

hmac_demo()
```

<Tip>
Always use `hmac.compare_digest()` for comparing MACs to prevent timing attacks!
</Tip>

## 7. Quantum Impact on Hash Functions

Good news: Hash functions are relatively quantum-resistant!

### Grover's Algorithm Impact

<Formula>
$$\text{Grover's speedup: } O(2^n) \rightarrow O(2^{n/2})$$
</Formula>

| Hash Function | Classical Security | Quantum Security |
|---------------|-------------------|------------------|
| SHA-256 | 128 bits (collision) | **64 bits** |
| SHA-384 | 192 bits (collision) | **96 bits** |
| SHA-512 | 256 bits (collision) | **128 bits** ✓ |
| SHA3-256 | 128 bits (collision) | **64 bits** |
| SHA3-512 | 256 bits (collision) | **128 bits** ✓ |

```python
def quantum_hash_analysis():
    """Analyze quantum impact on hash functions"""
    
    print("Quantum Impact on Hash Functions")
    print("=" * 60)
    
    hash_functions = [
        ("MD5", 128, "BROKEN even classically!"),
        ("SHA-1", 160, "BROKEN even classically!"),
        ("SHA-256", 256, "Reduced to 128-bit pre-image, 64-bit collision"),
        ("SHA-384", 384, "Reduced to 192-bit pre-image, 96-bit collision"),
        ("SHA-512", 512, "Reduced to 256-bit pre-image, 128-bit collision ✓"),
        ("SHA3-256", 256, "Reduced to 128-bit pre-image, 64-bit collision"),
        ("SHA3-512", 512, "Reduced to 256-bit pre-image, 128-bit collision ✓"),
    ]
    
    print(f"\n{'Hash':<12} {'Bits':<6} {'Quantum Status'}")
    print("-" * 60)
    
    for name, bits, status in hash_functions:
        print(f"{name:<12} {bits:<6} {status}")
    
    print("\n" + "=" * 60)
    print("Recommendations:")
    print("• For 128-bit post-quantum security: Use SHA-512 or SHA3-512")
    print("• SHA-256 still provides 128-bit pre-image resistance")
    print("• Collision resistance is halved (128→64 bits for SHA-256)")

quantum_hash_analysis()
```

<Tip>
**Recommendation**: Use SHA-512 or SHA3-512 for maximum quantum resistance. For most applications, SHA-256 remains secure due to its strong pre-image resistance.
</Tip>

## 8. Hash Function Selection Guide

```python
def hash_selection_guide():
    """Guide for selecting the right hash function"""
    
    use_cases = {
        "File integrity (checksums)": {
            "recommended": "SHA-256",
            "alternatives": ["SHA3-256", "BLAKE3"],
            "reason": "Good balance of speed and security"
        },
        "Digital signatures": {
            "recommended": "SHA-256 or SHA-384",
            "alternatives": ["SHA3-256", "SHA-512"],
            "reason": "Collision resistance is critical"
        },
        "Password hashing": {
            "recommended": "Argon2id",
            "alternatives": ["bcrypt", "scrypt"],
            "reason": "Memory-hard, intentionally slow"
        },
        "HMAC / MACs": {
            "recommended": "HMAC-SHA256",
            "alternatives": ["HMAC-SHA3-256"],
            "reason": "Well-analyzed, widely supported"
        },

            "recommended": "HKDF-SHA256",
            "alternatives": ["SHAKE256", "HKDF-SHA512"],

            "reason": "256-bit output provides 128-bit quantum security"
        }
    }
    
    print("Hash Function Selection Guide")
    print("=" * 60)
    
    for use_case, info in use_cases.items():
        print(f"\n{use_case}:")
        print(f"  ✓ Recommended: {info['recommended']}")
        print(f"  • Alternatives: {', '.join(info['alternatives'])}")
        print(f"  • Reason: {info['reason']}")

hash_selection_guide()
```

## Summary

In this lesson, you learned about cryptographic hash functions:

1. **Hash functions** produce fixed-size output from arbitrary input
2. **Three security properties**: pre-image resistance, second pre-image resistance, collision resistance
3. **Birthday attack** reduces collision resistance from $2^n$ to $2^{n/2}$
4. **SHA-2** is the current standard (SHA-256, SHA-512)
5. **SHA-3** offers different design and length extension resistance
6. **Applications**: integrity verification, password hashing (with proper functions), HMAC
7. **Quantum impact**: Grover's algorithm halves security; SHA-512 recommended for post-quantum
8. **Never use MD5 or SHA-1** for security purposes
