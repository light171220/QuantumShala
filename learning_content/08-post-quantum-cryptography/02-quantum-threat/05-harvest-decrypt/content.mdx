## 1. The HNDL Attack Model

<Definition term="Harvest Now, Decrypt Later (HNDL)">
An attack strategy where adversaries intercept and store encrypted communications today, with the intent to decrypt them in the future using quantum computers. The data may be valuable years or decades later.
</Definition>

## 2. Why HNDL Matters Now

### The Math of Retrospective Exposure

```python
def calculate_exposure_window(data_sensitivity_years: int,
                               crqc_estimate_years: int) -> dict:
    """
    Calculate how long data is exposed to HNDL attacks.
    """
    current_year = 2024
    
    # When data stops being sensitive
    sensitivity_end = current_year + data_sensitivity_years
    
    # When CRQC enables decryption
    crqc_year = current_year + crqc_estimate_years
    
    # If CRQC arrives while data is still sensitive, there's exposure
    if crqc_year < sensitivity_end:
        exposure_years = sensitivity_end - crqc_year
        at_risk = True
    else:
        exposure_years = 0
        at_risk = False
    
    return {
        'data_sensitive_until': sensitivity_end,
        'crqc_expected': crqc_year,
        'at_risk': at_risk,
        'exposure_years': exposure_years,
        'action': 'URGENT: Encrypt with PQC now' if at_risk else 'Lower priority'
    }

# Example analyses
scenarios = [
    ("State secrets (75 years)", 75, 15),
    ("Medical records (50 years)", 50, 15),
    ("Business plans (10 years)", 10, 15),
    ("Session tokens (1 day)", 0.003, 15)  # ~1 day
]

print("HNDL Exposure Analysis")
print("=" * 60)

for name, sensitivity, crqc in scenarios:
    result = calculate_exposure_window(sensitivity, crqc)
    print(f"\n{name}:")
    print(f"  Sensitive until: {result['data_sensitive_until']}")
    print(f"  At risk: {'YES' if result['at_risk'] else 'No'}")
    if result['at_risk']:
        print(f"  Exposure: {result['exposure_years']} years after CRQC")
    print(f"  Action: {result['action']}")
```

### Data That's Already Been Harvested

<Example>
**Historical Data Collection**

Consider data encrypted and transmitted in 2020:
- 4 years have already passed
- That data has potentially been stored by adversaries
- When CRQC arrives, that 2020 data can be decrypted
- If the data is still sensitive in 2035, it will be compromised

```python
def historical_exposure(year_transmitted: int,
                        sensitivity_years: int,
                        crqc_year: int = 2040) -> dict:
    """
    Analyze historical data's exposure to HNDL.
    """
    current_year = 2024
    
    # When does sensitivity expire?
    sensitivity_expires = year_transmitted + sensitivity_years
    
    # Is it still sensitive when CRQC arrives?
    compromised = sensitivity_expires > crqc_year
    
    # Years of exposure after CRQC
    if compromised:
        exposure = sensitivity_expires - crqc_year
    else:
        exposure = 0
    
    return {
        'transmitted': year_transmitted,
        'sensitive_until': sensitivity_expires,
        'crqc_year': crqc_year,
        'will_be_compromised': compromised,
        'exposure_years': exposure
    }

# Historical data analysis
print("\nHistorical Data at Risk")
print("=" * 50)

for year in [2010, 2015, 2020, 2024]:
    result = historical_exposure(year, 50)  # 50-year sensitivity
    status = "⚠️  AT RISK" if result['will_be_compromised'] else "✓ Safe"
    print(f"Data from {year}: {status}")
    if result['will_be_compromised']:
        print(f"  Exposed for {result['exposure_years']} years after CRQC")
```

## 5. Calculating Your Real Deadline

The HNDL threat changes how you think about migration timelines.

<Definition term="Effective Protection Deadline">
The date by which data must be encrypted with quantum-resistant algorithms to prevent HNDL exposure. This is CRQC arrival date minus the data's sensitivity lifetime.
</Definition>

<Formula>
$$\text{Effective Deadline} = \text{CRQC Date} - \text{Data Sensitivity Lifetime}$$
</Formula>

```python
def calculate_effective_deadline(data_types: list,
                                  crqc_estimate_year: int = 2040) -> list:
    """
    Calculate effective protection deadlines for different data types.
    """
    current_year = 2024
    results = []
    
    for data_type, sensitivity_years in data_types:
        # Effective deadline
        deadline_year = crqc_estimate_year - sensitivity_years
        
        # Time remaining
        years_remaining = deadline_year - current_year
        
        if years_remaining < 0:
            status = f"OVERDUE by {-years_remaining} years!"
            urgency = "CRITICAL"
        elif years_remaining < 3:
            status = f"{years_remaining} years remaining"
            urgency = "HIGH"
        elif years_remaining < 7:
            status = f"{years_remaining} years remaining"
            urgency = "MEDIUM"
        else:
            status = f"{years_remaining} years remaining"
            urgency = "LOW"
        
        results.append({
            'data_type': data_type,
            'sensitivity': f"{sensitivity_years} years",
            'deadline': deadline_year,
            'status': status,
            'urgency': urgency
        })
    
    return sorted(results, key=lambda x: x['deadline'])

# Example data types
data_types = [
    ("State secrets", 75),
    ("Medical records", 50),
    ("Trade secrets", 25),
    ("Business plans", 15),
    ("Session data", 1)
]

results = calculate_effective_deadline(data_types, crqc_estimate_year=2040)

print("Effective Protection Deadlines (CRQC estimate: 2040)")
```
