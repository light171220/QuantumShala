## 1. Introduction to Grover's Algorithm

While Shor's algorithm gets the most attention, Grover's algorithm (1996) also has significant cryptographic implications.

<Definition term="Grover's Algorithm">
A quantum algorithm for unstructured search that finds a marked item in an unsorted database of N items using O(√N) queries, compared to O(N) for classical algorithms—a quadratic speedup.
</Definition>

<Formula>
$$\text{Classical search: } O(N) \text{ queries}$$
$$\text{Grover's search: } O(\sqrt{N}) \text{ queries}$$
</Formula>

```python
import math

def compare_search_complexity(database_size: int):
    """
    Compare classical vs quantum search complexity.
    """
    classical_queries = database_size
    quantum_queries = math.sqrt(database_size)
    
    print(f"Searching database of {database_size:,} items:")
    print(f"  Classical: {classical_queries:,.0f} queries")
    print(f"  Quantum:   {quantum_queries:,.0f} queries")
    print(f"  Speedup:   {classical_queries/quantum_queries:,.0f}x")

# Example: searching all possible 128-bit keys
key_space_128 = 2**128
compare_search_complexity(key_space_128)
```

## 2. How Grover's Algorithm Works

Grover's algorithm uses quantum amplitude amplification to boost the probability of finding the correct answer.

### The Basic Idea

```
Classical Search (trying to find key that decrypts to known plaintext):
┌─────────────────────────────────────────────────────────┐
│ Try key 1... wrong                                     │
│ Try key 2... wrong                                     │
│ Try key 3... wrong                                     │
│ ...                                                     │
│ Try key N... maybe right!                              │
│ Expected: N/2 tries on average                         │
└─────────────────────────────────────────────────────────┘

Grover's Search:
┌─────────────────────────────────────────────────────────┐
│ 1. Create superposition of ALL keys                    │
│ 2. Mark the correct key (oracle query)                 │
│ 3. Amplify the marked state                            │
│ 4. Repeat √N times                                     │
│ 5. Measure: high probability of correct key            │
└─────────────────────────────────────────────────────────┘
```

### Algorithm Steps

<Formula>
$$|\psi_0\rangle = \frac{1}{\sqrt{N}} \sum_{x=0}^{N-1} |x\rangle \text{ (uniform superposition)}$$
$$\text{Apply } O(\sqrt{N}) \text{ Grover iterations}$$
$$\text{Each iteration: Oracle + Diffusion operator}$$
</Formula>

```python
def grover_conceptual_demo():
    """
    Conceptual demonstration of Grover's algorithm.
    """
    import random
    
    # Small example: 4-bit search space (16 items)
    N = 16
    secret_key = 7  # The key we're trying to find
    
    print(f"Search space size: N = {N}")
    print(f"Secret key: {secret_key}")
    
    # Classical search
    classical_queries = 0
    for guess in range(N):
        classical_queries += 1
        if guess == secret_key:
            break
    
    # Grover's search (simplified)
    import math
    grover_iterations = int(math.pi/4 * math.sqrt(N))
    
    print(f"\nClassical search: {classical_queries} queries (found at position {secret_key})")
    print(f"Grover's search: ~{grover_iterations} iterations")
    print(f"√N = {math.sqrt(N):.2f}")

grover_conceptual_demo()
```

### Optimal Number of Iterations

<Formula>
$$\text{Optimal iterations} \approx \frac{\pi}{4}\sqrt{N}$$
</Formula>

<Tip>
Grover's algorithm requires knowing approximately how many solutions exist. Too few or too many iterations reduce success probability. This is important for cryptographic applications.
</Tip>

## 3. Impact on Symmetric Cryptography

Grover's algorithm can be applied to brute-force key search:

<Definition term="Grover's Key Search">
Using Grover's algorithm to search for a symmetric encryption key by testing all possible keys in superposition. Effectively halves the bit security of symmetric ciphers.
</Definition>

<Formula>
$$\text{n-bit key classical security: } 2^n \text{ operations}$$
$$\text{n-bit key quantum security: } 2^{n/2} \text{ operations}$$
</Formula>

### Practical Impact

| Cipher | Key Size | Classical Security | Quantum Security |
|--------|----------|-------------------|------------------|
| AES-128 | 128 bits | 2¹²⁸ | 2⁶⁴ ⚠️ |
| AES-192 | 192 bits | 2¹⁹² | 2⁹⁶ |
| AES-256 | 256 bits | 2²⁵⁶ | 2¹²⁸ ✓ |
| ChaCha20 | 256 bits | 2²⁵⁶ | 2¹²⁸ ✓ |
| 3DES | 168 bits | 2¹⁶⁸ | 2⁸⁴ |

```python
def symmetric_quantum_security(key_bits: int) -> dict:
    """
    Calculate quantum security level for symmetric cipher.
    """
    classical_security = key_bits
    quantum_security = key_bits // 2
    
    # Security assessment
    if quantum_security >= 128:
        assessment = "Quantum-safe"
    elif quantum_security >= 80:
        assessment = "Marginal - consider upgrade"
    else:
        assessment = "Insufficient for long-term security"
    
    return {
        'key_bits': key_bits,
        'classical_security_bits': classical_security,
        'quantum_security_bits': quantum_security,
        'classical_operations': f"2^{classical_security}",
        'quantum_operations': f"2^{quantum_security}",
        'assessment': assessment
    }

# Analyze common ciphers
for key_size in [128, 192, 256]:
    result = symmetric_quantum_security(key_size)
    print(f"\n{key_size}-bit key:")
    print(f"  Classical: {result['classical_operations']}")
    print(f"  Quantum: {result['quantum_operations']}")
    print(f"  Assessment: {result['assessment']}")
```

<Warning>
**AES-128 provides only 64-bit security against quantum attacks!** This is generally considered insufficient for long-term security. Migrate to AES-256.
</Warning>

## 4. Impact on Hash Functions

Grover's algorithm also affects hash function security for preimage attacks.

### Preimage Attack

<Definition term="Preimage Attack">
Given a hash output h, find any input m such that Hash(m) = h. Classical complexity: O(2ⁿ) for n-bit hash. Quantum complexity: O(2^(n/2)).
</Definition>

### Collision Attack

<Definition term="Collision Attack">
Find any two different inputs m₁ ≠ m₂ such that Hash(m₁) = Hash(m₂). Classical: O(2^(n/2)) via birthday paradox. Quantum: O(2^(n/3)).
</Definition>

```python
def hash_function_security(output_bits: int) -> dict:
    """
    Calculate hash function security against various attacks.
    """
    return {
        'hash_output_bits': output_bits,
        'preimage': {
            'classical': output_bits,
            'quantum': output_bits // 2
        },
        'collision': {
            'classical': output_bits // 2,
            'quantum': output_bits // 3  # BHT algorithm
        },
        'second_preimage': {
            'classical': output_bits,
            'quantum': output_bits // 2
        }
    }

# Common hash functions
hash_functions = [
    ('SHA-256', 256),
    ('SHA-384', 384),
    ('SHA-512', 512),
    ('SHA3-256', 256),
]

print("Hash Function Quantum Security Analysis")
print("=" * 60)

for name, bits in hash_functions:
    sec = hash_function_security(bits)
    print(f"\n{name}:")
    print(f"  Preimage - Classical: 2^{sec['preimage']['classical']}, "
          f"Quantum: 2^{sec['preimage']['quantum']}")
    print(f"  Collision - Classical: 2^{sec['collision']['classical']}, "
          f"Quantum: 2^{sec['collision']['quantum']}")
```

### Hash Security Recommendations

| Hash Function | Classical Preimage | Quantum Preimage | Status |

| SHA-256 | 256 bits | 128 bits | ✓ Adequate |
| SHA-384 | 384 bits | 192 bits | ✓ Strong |
| SHA-512 | 512 bits | 256 bits | ✓ Very Strong |

## 5. The "Double Your Key" Rule

For quantum resistance, a simple rule emerges:

<Definition term="Double Key Size Rule">
To maintain the same security level against quantum attacks, double your symmetric key size. A 128-bit classical security target requires 256-bit keys in the quantum era.
</Definition>

```python
def quantum_resistant_key_size(target_security_bits: int) -> int:
    """
    Calculate required key size for target post-quantum security.
    
    Args:
        target_security_bits: Desired security level against quantum attacks
    
    Returns:
        Required key size in bits
    """
    return target_security_bits * 2

def analyze_security_targets():
    """
    Common security targets and required key sizes.
    """
    targets = [
        (64, "Short-term/Low-value"),
        (80, "Legacy minimum"),
        (128, "Standard security"),
        (192, "High security"),
        (256, "Maximum security")
    ]
    
    print("Post-Quantum Key Size Requirements")
    print("=" * 50)
    print(f"{'Target Security':<20} {'Required Key Size':<20} {'Use Case'}")
    print("-" * 50)
    
    for target, use_case in targets:
        key_size = quantum_resistant_key_size(target)
        print(f"{target}-bit{'':<12} {key_size}-bit{'':<12} {use_case}")

analyze_security_targets()
```

## 6. Grover's Algorithm Limitations

Important: Grover's speedup is quadratic, not exponential like Shor's.

### Why This Matters

```python
def quadratic_vs_exponential():
    """
    Compare quadratic vs exponential speedups.
    """
    print("Shor's Algorithm (Exponential Speedup):")
    print("  Classical: 2^112 operations for RSA-2048")
    print("  Quantum:   ~(2048)^3 operations")
    print("  Speedup:   From infeasible to hours")
    print("  Solution:  NEW algorithms needed (lattice-based)")
    
    print("\nGrover's Algorithm (Quadratic Speedup):")
    print("  Classical: 2^256 operations for AES-256")
    print("  Quantum:   2^128 operations")
    print("  Speedup:   2^128 times faster")
    print("  Solution:  Just use LONGER KEYS (same algorithm)")
    
    print("\nKey Insight:")
    print("  • Shor's breaks the underlying math → must change algorithms")
    print("  • Grover's speeds up search → just increase key size")

quadratic_vs_exponential()
```

### Practical Limitations of Grover's Algorithm

| Factor | Limitation |
|--------|------------|
| **Sequential** | Limited parallelization benefits |
| **Oracle calls** | Each "query" requires full encryption |
| **Error correction** | Massive overhead for each operation |
| **Circuit depth** | Very deep circuits required |

<Example>
**Grover Attack on AES-256 Resource Estimate**

```python
def grover_aes_resources():
    """
    Estimate resources for Grover attack on AES-256.
    """
    key_bits = 256
    grover_iterations = 2 ** (key_bits // 2)  # 2^128
    
    # Each iteration requires:
    # - ~6,000 logical qubits for AES
    # - ~2^12 T-gates per AES evaluation
    
    t_gates_per_aes = 2**12
    total_t_gates = grover_iterations * t_gates_per_aes
    
    # Physical resources (rough estimates)
    logical_qubits = 6000
    physical_per_logical = 1000  # Error correction overhead
    total_physical = logical_qubits * physical_per_logical
    
    print("Grover Attack on AES-256 Resources:")
    print(f"  Grover iterations: 2^128")
    print(f"  Logical qubits: ~{logical_qubits:,}")
    print(f"  Physical qubits: ~{total_physical:,}")
    print(f"  Total T-gates: 2^140")
    print(f"\nConclusion: Still astronomically expensive!")
    print("AES-256 remains secure even against quantum attacks.")

grover_aes_resources()
```
</Example>

<Tip>
Even with Grover's quadratic speedup, AES-256 requires 2¹²⁸ quantum operations. This is still computationally infeasible with any foreseeable technology!
</Tip>

## 7. Comparing Shor's and Grover's Impact

| Aspect | Shor's Algorithm | Grover's Algorithm |
|--------|------------------|-------------------|
| **Speedup** | Exponential | Quadratic |
| **Targets** | RSA, ECC, DH | Symmetric, hashes |
| **Effect** | Complete break | Security reduction |
| **Mitigation** | New algorithms | Larger keys |
| **Urgency** | Critical | Important but manageable |

```python
def compare_quantum_threats():
    """
    Compare the two main quantum threats to cryptography.
    """
    comparison = {
        'Shor\'s Algorithm': {
            'targets': ['RSA', 'ECDSA', 'ECDH', 'DSA', 'DH'],
            'speedup_type': 'Exponential',
            'effect': 'Complete break',
            'mitigation': 'Replace with PQC algorithms',
            'urgency': 'CRITICAL',
            'migration_complexity': 'High - new protocols needed'
        },
        'Grover\'s Algorithm': {
            'targets': ['AES', 'ChaCha20', 'SHA-256', 'Symmetric ciphers'],
            'speedup_type': 'Quadratic (√N)',
            'effect': 'Halves bit security',
            'mitigation': 'Double key/output sizes',
            'urgency': 'Important',
            'migration_complexity': 'Low - same algorithms, larger parameters'
        }
    }
    
    for algo, details in comparison.items():
        print(f"\n{algo}")
        print("=" * 40)
        for key, value in details.items():
            if isinstance(value, list):
                print(f"  {key}: {', '.join(value)}")
            else:
                print(f"  {key}: {value}")

compare_quantum_threats()
```

## Summary

In this lesson, you learned about Grover's algorithm:

1. **Grover's algorithm** provides quadratic speedup for unstructured search
2. **Symmetric security** is effectively halved (n-bit → n/2-bit quantum)
3. **Hash functions** need larger outputs for quantum resistance
4. **"Double the key"** rule: use 256-bit keys for 128-bit quantum security
5. **AES-256 is safe**: 2¹²⁸ quantum operations is still infeasible
6. **Unlike Shor's**, Grover's doesn't break algorithms—just weakens them
7. **Migration is simple**: same algorithms, larger parameters
8. **Start now**: use 256-bit symmetric keys and SHA-384/512

Grover's impact is manageable compared to Shor's. The main action item is ensuring you use sufficiently large key sizes (256 bits for symmetric encryption).
