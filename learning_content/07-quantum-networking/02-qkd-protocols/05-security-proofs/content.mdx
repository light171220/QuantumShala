# Security Proofs

QKD's unique claim is information-theoretic security—security guaranteed by the laws of physics rather than computational assumptions. This lesson explores the mathematical foundations of QKD security proofs, the assumptions they require, and how to calculate provably secure key rates.

## Information-Theoretic Security

Unlike classical cryptography, QKD security doesn't rely on computational hardness assumptions.

### Security Paradigms

```
┌─────────────────────────────────────────────────────────────────┐
│              Security Paradigms Comparison                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   COMPUTATIONAL SECURITY (Classical)                            │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │ Assumption: Certain problems are computationally hard    │  │
│   │                                                         │  │
```

### Information Theory Fundamentals

```python
import numpy as np
from typing import Tuple, Dict, List
from dataclasses import dataclass

class InformationTheory:
    """Information theory tools for security analysis."""

    @staticmethod
    def binary_entropy(p: float) -> float:
        """
        Binary entropy function H(p) = -p log p - (1-p) log(1-p).

        Measures uncertainty of a binary random variable.
        """
        if p <= 0 or p >= 1:
            return 0.0
        return -p * np.log2(p) - (1-p) * np.log2(1-p)

    @staticmethod
    def shannon_entropy(probs: List[float]) -> float:
        """
        Shannon entropy H(X) = -Σ p_i log p_i.

        Maximum uncertainty = log(n) for uniform distribution.
        """
        return -sum(p * np.log2(p) for p in probs if p > 0)

    @staticmethod
    def conditional_entropy(joint_probs: np.ndarray) -> float:
        """
        Conditional entropy H(X|Y) = H(X,Y) - H(Y).

        Remaining uncertainty about X given Y.
        """
        # H(X,Y)
        h_xy = -sum(p * np.log2(p) for p in joint_probs.flatten() if p > 0)

        # H(Y)
        p_y = joint_probs.sum(axis=0)
        h_y = -sum(p * np.log2(p) for p in p_y if p > 0)

        return h_xy - h_y

    @staticmethod
    def mutual_information(joint_probs: np.ndarray) -> float:
        """
        Mutual information I(X;Y) = H(X) + H(Y) - H(X,Y).

        Information shared between X and Y.
        """
        p_x = joint_probs.sum(axis=1)
        p_y = joint_probs.sum(axis=0)

        h_x = -sum(p * np.log2(p) for p in p_x if p > 0)
        h_y = -sum(p * np.log2(p) for p in p_y if p > 0)
        h_xy = -sum(p * np.log2(p) for p in joint_probs.flatten() if p > 0)

        return h_x + h_y - h_xy
```

## Security Proof Structure

Modern QKD security proofs follow a structured approach to bound Eve's information.

### Proof Framework

```
┌─────────────────────────────────────────────────────────────────┐
│                QKD Security Proof Structure                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   SECURITY DEFINITION                                           │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │ A QKD protocol is ε-secure if:                          │  │
│   │                                                         │  │
│   │ ρ_AKE ≈_ε ρ_U ⊗ ρ_E                                    │  │
│   │                                                         │  │
│   │ where:                                                  │  │
│   │ • ρ_AKE: Joint state of Alice, Key, Eve                │  │
│   │ • ρ_U: Uniform distribution over keys                   │  │
│   │ • ⊗: Tensor product (independence)                      │  │
│   │ • ≈_ε: Within trace distance ε                         │  │
│   │                                                         │  │
│   │ Meaning: Key is nearly uniform AND nearly independent  │  │
│   │          of everything Eve knows                        │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   PROOF STEPS                                                   │
│                                                                 │
│   Step 1: Model the protocol                                   │
│   ├── Quantum channel (lossy, noisy)                           │
│   ├── Measurements (imperfect detectors)                       │
│   └── Classical post-processing                                │
│                                                                 │
│   Step 2: Bound Eve's information                              │
│   ├── Most general attack consistent with observations         │
│   ├── Devetak-Winter formula for collective attacks            │
│   └── Renner's framework for general attacks                   │
│                                                                 │
│   Step 3: Calculate achievable key rate                        │
│   ├── R = I(A:B) - χ(A:E)  [direct reconciliation]            │
│   └── R = I(A:B) - χ(B:E)  [reverse reconciliation]           │
│                                                                 │
│   Step 4: Apply privacy amplification                          │
│   └── Universal hashing removes Eve's information              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Key Rate Formulas

```python
@dataclass
class SecurityProofResult:
    """Result of security analysis."""
    protocol: str
    key_rate: float
    eve_information: float
    security_parameter: float
    assumptions: List[str]

class KeyRateCalculator:
    """Calculate secure key rates for various protocols."""

    def __init__(self):
        self.h = InformationTheory.binary_entropy

    def bb84_asymptotic(self, qber: float, f_ec: float = 1.16) -> float:
        """
        BB84 asymptotic key rate under collective attacks.

        R = 1 - h(e) - f·h(e)

        where:
        - e: quantum bit error rate
        - f: error correction inefficiency (≥ 1)
        - h: binary entropy function
        """
        if qber >= 0.11:  # Threshold for BB84
            return 0.0

        # Alice-Bob mutual information after sifting
        I_AB = 1 - self.h(qber)

        # Error correction cost
        ec_cost = f_ec * self.h(qber)

        # Eve's information (Holevo bound for collective attacks)
        # For BB84: χ(A:E) = h(e) under collective attacks
        chi_AE = self.h(qber)

        # Key rate
        return max(0, I_AB - ec_cost - chi_AE + I_AB)  # Simplified: 1 - 2h(e) for perfect EC

    def bb84_devetak_winter(self, qber: float) -> Dict:
        """
        Full Devetak-Winter key rate for BB84.

        R = I(A:B) - χ(A:E)

        For BB84 with depolarizing channel:
        χ(A:E) = h((1 + √(1-2e)·√(2e))/2) for coherent attacks
        """
        if qber >= 0.11:
            return {'key_rate': 0, 'I_AB': 0, 'chi_AE': 1, 'threshold_exceeded': True}

        # Mutual information
        I_AB = 1 - self.h(qber)

        # Eve's Holevo information (collective attack bound)
        # Simplified: use intercept-resend bound
        chi_AE = self.h(qber)

        key_rate = max(0, I_AB - chi_AE)

        return {
            'key_rate': key_rate,
            'I_AB': I_AB,
            'chi_AE': chi_AE,
            'threshold_exceeded': False,
            'qber_margin': 0.11 - qber
        }

    def six_state_rate(self, qber: float) -> float:
        """
        Six-state protocol key rate.

        Higher threshold (~12.6%) due to additional basis.
        """
        if qber >= 0.126:
            return 0.0

        # Six-state has tighter bound on Eve
        I_AB = 1 - self.h(qber)

        # Eve's information is more constrained
        # Simplified bound
        chi_AE = self.h(0.5 * (1 + np.sqrt(1 - (qber/0.5)**2))) if qber < 0.5 else 1

        return max(0, I_AB - chi_AE)

    def e91_rate(self, bell_violation: float, detection_eff: float) -> float:
        """
        E91 key rate from Bell violation.

        Higher S → more key.
        """
        if bell_violation <= 2:
            return 0.0

        # Convert Bell violation to QBER bound
        # For maximum violation S = 2√2, QBER = 0
        # For S = 2, QBER = 0.11 (threshold)
        max_S = 2 * np.sqrt(2)
        normalized = (bell_violation - 2) / (max_S - 2)

        implied_qber = 0.11 * (1 - normalized)

        return self.bb84_asymptotic(implied_qber) * detection_eff

    def finite_key_rate(self, n: int, qber: float,
                        epsilon_sec: float = 1e-10,
                        epsilon_cor: float = 1e-15) -> Dict:
        """
        Finite-size key rate accounting for statistical fluctuations.

        Key rate is reduced by O(1/√n) corrections.
        """
        # Asymptotic rate
        r_asymp = self.bb84_asymptotic(qber)

        if r_asymp <= 0:
            return {'key_rate': 0, 'asymptotic_rate': 0, 'finite_penalty': 1.0}

        # Finite-size corrections
        # Simplified model - full calculation involves careful statistics

        # Parameter estimation uncertainty
        delta_pe = np.sqrt(np.log(1/epsilon_sec) / (2 * n))

        # Error correction leakage
        leak_ec = np.sqrt(n) * np.log(1/epsilon_cor) / n

        # Privacy amplification cost
        pa_cost = 2 * np.log(1/epsilon_sec) / n

        # Corrected rate
        effective_qber = min(0.5, qber + delta_pe)
        r_finite = self.bb84_asymptotic(effective_qber) - leak_ec - pa_cost

        return {
            'key_rate': max(0, r_finite),
            'asymptotic_rate': r_asymp,
            'finite_penalty': 1 - r_finite/r_asymp if r_asymp > 0 else 1,
            'n_signals': n,
            'effective_qber': effective_qber
        }
```

## Attack Models

Security proofs must consider increasingly powerful attack strategies.

### Attack Hierarchy

```
┌─────────────────────────────────────────────────────────────────┐
│                    Attack Model Hierarchy                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Most Restrictive                                              │
│        │                                                        │
│        ▼                                                        │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │ INDIVIDUAL ATTACKS                                      │  │
│   │ • Eve attacks each signal independently                 │  │
│   │ • Measures immediately after interaction                │  │
│   │ • Example: Intercept-resend                             │  │
│   │ • Easier to analyze, weaker Eve                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│        │                                                        │
│        ▼                                                        │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │ COLLECTIVE ATTACKS                                      │  │
│   │ • Eve attacks each signal identically                   │  │
│   │ • Can store quantum memory until sifting                │  │
│   │ • Optimizes measurement based on classical info         │  │
│   │ • Covers most practical scenarios                       │  │
│   └─────────────────────────────────────────────────────────┘  │
│        │                                                        │
│        ▼                                                        │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │ COHERENT (GENERAL) ATTACKS                              │  │
│   │ • Eve can do anything allowed by physics                │  │
│   │ • Joint operations on all signals                       │  │
│   │ • Unlimited quantum memory                              │  │
│   │ • Most difficult to prove security against              │  │
│   └─────────────────────────────────────────────────────────┘  │
│        │                                                        │
│        ▼                                                        │
│   Most Powerful                                                 │
│                                                                 │
```

```python
class EveAttacks:
    """Models for various eavesdropping attacks."""

    @staticmethod
    def intercept_resend(qubit_state: Dict, basis_guess: int) -> Tuple[Dict, int]:
        """
        Individual intercept-resend attack.

        Eve measures in guessed basis, resends measured state.
        Introduces 25% QBER if intercepting all qubits.
        """
        true_basis = qubit_state['basis']
        true_bit = qubit_state['bit']

        if basis_guess == true_basis:
            # Correct guess - no disturbance
            eve_bit = true_bit
            forwarded_state = qubit_state.copy()
        else:
            # Wrong guess - 50% error
            eve_bit = np.random.randint(0, 2)
            forwarded_state = {'basis': basis_guess, 'bit': eve_bit}

        return forwarded_state, eve_bit

    @staticmethod
    def photon_number_splitting(pulse_photon_number: int) -> Dict:
        """
        PNS attack on weak coherent pulses.

        If pulse has n > 1 photons, Eve keeps extras.
        """
        if pulse_photon_number <= 1:
            return {'attack_possible': False, 'photons_stolen': 0}

        # Eve takes extra photons
        photons_stolen = pulse_photon_number - 1

        return {
            'attack_possible': True,
            'photons_stolen': photons_stolen,
            'eve_has_copy': True
        }

    @staticmethod
    def unambiguous_state_discrimination(overlap: float) -> Dict:
        """
        USD attack on non-orthogonal states.

        Eve can sometimes conclusively identify state without disturbance.
        """
        # For B92 with |0⟩ and |+⟩, overlap = 1/√2
        # USD success probability
        p_success = 1 - overlap

        # When successful, Eve knows state exactly
        # When unsuccessful, must forward something

        return {
            'success_probability': p_success,
            'conclusive_info': True,
            'disturbance_on_fail': True
        }

class AttackAnalysis:
    """Analyze security under different attacks."""

    def __init__(self, protocol: str = 'BB84'):
        self.protocol = protocol
        self.calculator = KeyRateCalculator()

    def qber_from_intercept(self, intercept_fraction: float) -> float:
        """QBER introduced by partial intercept-resend."""
        # Eve guesses basis correctly 50% of time
        # When wrong, introduces 50% error
        return intercept_fraction * 0.5 * 0.5

    def eve_information_individual(self, intercept_fraction: float) -> float:
        """Eve's information from individual attack."""
        # Eve learns bit when she guesses basis correctly
        # That's 50% of intercepted signals
        return intercept_fraction * 0.5

    def eve_information_collective(self, qber: float) -> float:
        """Eve's Holevo information from collective attack."""
        h = InformationTheory.binary_entropy
        return h(qber)

    def information_disturbance_tradeoff(self) -> Dict:
        """
        Calculate information-disturbance tradeoff.

        More information → more disturbance
        """
        intercept_fractions = np.linspace(0, 1, 100)

        results = {
            'intercept_fraction': intercept_fractions,
            'qber': [],
            'eve_info': [],
            'remaining_key_rate': []
        }

        for f in intercept_fractions:
            qber = self.qber_from_intercept(f)
            results['qber'].append(qber)
            results['eve_info'].append(self.eve_information_individual(f))
            results['remaining_key_rate'].append(
                self.calculator.bb84_asymptotic(qber)
            )

        return results

    def pns_vulnerability(self, mean_photon: float, distance_km: float) -> Dict:
        """
        Analyze vulnerability to PNS attack.

        For weak coherent pulses with Poisson statistics.
        """
        # Poisson distribution
        p_0 = np.exp(-mean_photon)
        p_1 = mean_photon * np.exp(-mean_photon)
        p_multi = 1 - p_0 - p_1

        # Channel loss
        eta = 10 ** (-0.2 * distance_km / 10)

        # Eve can block single-photon, forward multi-photon
        # This creates "tagged" bits Eve knows completely

        tagged_fraction = p_multi / (p_1 * eta + p_multi)

        return {
            'mean_photon': mean_photon,
            'multi_photon_prob': p_multi,
            'tagged_fraction': tagged_fraction,
            'secure': tagged_fraction < 0.5,  # Simplified criterion
            'recommendation': 'Use decoy states' if tagged_fraction > 0.1 else 'OK'
        }
```

## Composable Security

Modern security proofs use composable frameworks that guarantee security when QKD is used as part of larger systems.

### Composable Security Framework

```
│                                                                 │
│   WHY COMPOSABILITY MATTERS:                                    │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │ • QKD key used in subsequent protocols (OTP, AES)       │  │
│   │ • Security must compose: QKD + Encryption = Secure      │  │

│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   UNIVERSAL COMPOSABILITY:                                      │

│   │   Real Protocol    ε-close to    Ideal Functionality   │  │
│   │        ┌───┐                          ┌───┐            │  │
│   │   A ───│QKD│─── K                K ←──│ $ │            │  │
│   │   B ───│   │─── K'               K'←──│   │            │  │

│   │        └───┘                          └───┘            │  │
│   │                                                         │  │
│   │   $ = uniform random key, ⊥ = no info                  │  │
```

```python
@dataclass
class ComposableSecurityParams:
    """Parameters for composable security analysis."""
    epsilon_sec: float  # Secrecy parameter
    epsilon_cor: float  # Correctness parameter
    epsilon_pa: float   # Privacy amplification failure prob
    epsilon_pe: float   # Parameter estimation error
    epsilon_ec: float   # Error correction failure prob

    @property
    def total_epsilon(self) -> float:
        """Total security parameter."""
        return self.epsilon_sec + self.epsilon_cor

    @classmethod
    def standard(cls) -> 'ComposableSecurityParams':
        """Standard security level (2^-64)."""
        return cls(
            epsilon_sec=2**-64,
            epsilon_cor=2**-64,
            epsilon_pa=2**-64,
            epsilon_pe=2**-64,
            epsilon_ec=2**-64
        )

    @classmethod
    def high_security(cls) -> 'ComposableSecurityParams':
        """High security level (2^-100)."""
        return cls(
            epsilon_sec=2**-100,
            epsilon_cor=2**-100,
            epsilon_pa=2**-100,
            epsilon_pe=2**-100,
            epsilon_ec=2**-100
        )

class ComposableKeyRate:
    """Calculate composable secure key rates."""

    def __init__(self, params: ComposableSecurityParams):
        self.params = params

    def finite_key_length(self, n: int, qber: float,
                         sifting_efficiency: float = 0.5) -> Dict:
        """
        Calculate composable secure key length.

        Accounts for:
        - Parameter estimation cost
        - Error correction leakage
        - Privacy amplification
        - Finite-size effects
        """
        h = InformationTheory.binary_entropy

        # Number of sifted bits
        n_sift = int(n * sifting_efficiency)

        # Bits used for parameter estimation
        n_pe = int(n_sift * 0.1)  # 10% for PE
        n_key = n_sift - n_pe

        # Parameter estimation confidence interval
        delta_pe = np.sqrt(np.log(1/self.params.epsilon_pe) / (2 * n_pe))
        qber_upper = min(0.5, qber + delta_pe)

        # Check if still secure
        if qber_upper >= 0.11:
            return {'key_length': 0, 'reason': 'QBER too high'}

        # Asymptotic term (bits of secret per sifted bit)
        r_asymp = 1 - h(qber_upper)

        # Error correction leakage
        f_ec = 1.16  # Efficiency factor
        leak_ec = f_ec * h(qber_upper) * n_key

        # Eve's information
        chi_E = h(qber_upper) * n_key

        # Privacy amplification cost
        pa_cost = 2 * np.log2(1/self.params.epsilon_pa)

        # Finite-size corrections
        finite_corr = np.sqrt(n_key) * 2 * np.log2(1/self.params.epsilon_sec)

        # Final key length
        key_length = n_key * r_asymp - leak_ec - chi_E - pa_cost - finite_corr

        return {
            'key_length': max(0, int(key_length)),
            'n_signals': n,
            'n_sifted': n_sift,
            'n_for_key': n_key,
            'effective_rate': key_length / n if n > 0 else 0,
            'qber_observed': qber,
            'qber_upper_bound': qber_upper,
            'corrections': {
                'error_correction': leak_ec,
                'privacy_amplification': pa_cost,
                'finite_size': finite_corr
            }
        }

    def minimum_block_size(self, target_rate: float,
                          qber: float) -> int:
        """Find minimum block size for target key rate."""
        for n in [10**i for i in range(4, 10)]:
            result = self.finite_key_length(n, qber)
            if result['effective_rate'] >= target_rate:
                return n
        return -1  # Not achievable
```

## Summary

QKD security proofs provide the mathematical foundation for information-theoretic security:

- **Security definition**: Key indistinguishable from uniform and independent of Eve
- **Key rate formulas**: R = I(A:B) - χ(A:E) bounds achievable secret key
- **Attack hierarchy**: Individual → Collective → Coherent (general)
- **Composability**: Security that composes with other protocols
- **Finite-size effects**: Real implementations need statistical corrections
- **Implementation gap**: Theory must be complemented by careful engineering

Understanding these foundations is essential for properly deploying QKD systems with meaningful security guarantees.
