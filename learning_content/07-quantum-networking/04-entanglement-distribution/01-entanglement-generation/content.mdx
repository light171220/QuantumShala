# Entanglement Generation

## Introduction

Quantum entanglement is the cornerstone of quantum networking, enabling secure communication, distributed quantum computing, and quantum sensing. Before entangled states can be distributed across a network, they must first be created. This lesson explores the physical mechanisms and technologies used to generate entangled quantum states.

## Physical Mechanisms for Entanglement

Entanglement arises from quantum mechanical interactions that correlate the states of two or more particles:


## Spontaneous Parametric Down-Conversion (SPDC)

SPDC is the most widely used method for generating entangled photon pairs. A pump photon interacts with a nonlinear crystal and spontaneously splits into two lower-energy photons (signal and idler) that are entangled.

### SPDC Physics

```
┌─────────────────────────────────────────────────────────────────────┐
│                     SPDC PROCESS                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   Pump photon        Nonlinear          Signal + Idler              │
│   (ωp, kp)           Crystal            (ωs, ωi)                    │
│                      (BBO, PPLN)                                    │
│     ═══════>     ╔═══════════════╗     ════════>                   │
│        λ=405nm   ║     χ⁽²⁾      ║        λ=810nm (×2)             │
│                  ╚═══════════════╝                                  │
│                                                                     │
│   Energy Conservation:    ωp = ωs + ωi                              │
│   Momentum Conservation:  kp = ks + ki  (phase matching)            │
│                                                                     │
│   Type-I:  Signal and idler have SAME polarization                  │
│   Type-II: Signal and idler have ORTHOGONAL polarization            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### Polarization Entanglement via SPDC

```python
import numpy as np
from dataclasses import dataclass
from typing import Tuple, List, Optional
from enum import Enum

class SPDCType(Enum):
    """Type of SPDC process."""
    TYPE_I = "type_i"      # Same polarization (e.g., both |H⟩)
    TYPE_II = "type_ii"    # Orthogonal polarization (|H⟩|V⟩)

class PhaseMatchingGeometry(Enum):
    """Crystal phase matching geometry."""
    COLINEAR = "colinear"      # Signal, idler along pump direction
    NONCOLINEAR = "noncolinear"  # Signal, idler at angles to pump

@dataclass
class PhotonPair:
    """Represents an entangled photon pair."""
    wavelength_signal: float  # nm
    wavelength_idler: float   # nm
    polarization_state: np.ndarray  # 4-element state vector
    generation_time: float    # relative time

    def fidelity_to_bell(self, bell_state: str = "phi+") -> float:
        """Calculate fidelity to ideal Bell state."""
        bell_states = {
            "phi+": np.array([1, 0, 0, 1]) / np.sqrt(2),
            "phi-": np.array([1, 0, 0, -1]) / np.sqrt(2),
            "psi+": np.array([0, 1, 1, 0]) / np.sqrt(2),
            "psi-": np.array([0, 1, -1, 0]) / np.sqrt(2)
        }
        target = bell_states[bell_state]
        return np.abs(np.vdot(target, self.polarization_state))**2

class SPDCSource:
    """
    Simulates Spontaneous Parametric Down-Conversion source.

    Generates polarization-entangled photon pairs using
    Type-II SPDC in nonlinear crystals.
    """

    def __init__(
        self,
        pump_wavelength: float = 405.0,  # nm
        pump_power: float = 100.0,        # mW
        crystal_type: str = "BBO",
        spdc_type: SPDCType = SPDCType.TYPE_II,
        collection_efficiency: float = 0.1
    ):
        self.pump_wavelength = pump_wavelength
        self.pump_power = pump_power
        self.crystal_type = crystal_type
        self.spdc_type = spdc_type
        self.collection_efficiency = collection_efficiency

        # Crystal parameters
        self.crystal_params = {
            "BBO": {"efficiency": 1e-12, "bandwidth": 5.0},   # β-Barium Borate
            "PPLN": {"efficiency": 1e-10, "bandwidth": 0.5},  # Periodically-Poled LiNbO3
            "PPKTP": {"efficiency": 5e-11, "bandwidth": 2.0}  # Periodically-Poled KTP
        }

    def calculate_pair_rate(self) -> float:
        """
        Calculate photon pair generation rate.

        Rate ∝ P_pump × η_crystal × η_collection²
        """
        params = self.crystal_params[self.crystal_type]

        # Approximate pair rate (pairs/second)
        # Real rate depends on focusing, crystal length, phase matching
        base_rate = (self.pump_power * 1e-3 *  # Convert to W
                    params["efficiency"] * 1e15)  # Normalize

        # Collection efficiency applies to both photons
        detected_rate = base_rate * self.collection_efficiency**2

        return detected_rate

    def generate_pair(
        self,
        noise_level: float = 0.01,
        timing_jitter: float = 100e-12  # 100 ps
    ) -> PhotonPair:
        """
        Generate one entangled photon pair.

        For Type-II SPDC, creates state approximately:
        |ψ⟩ = (|H⟩|V⟩ + e^(iφ)|V⟩|H⟩) / √2
        """
        params = self.crystal_params[self.crystal_type]

        # Calculate wavelengths (degenerate case: both 2×pump)
        lambda_signal = 2 * self.pump_wavelength + np.random.normal(0, params["bandwidth"])
        lambda_idler = 2 * self.pump_wavelength + np.random.normal(0, params["bandwidth"])

        # Generate polarization state based on SPDC type
        if self.spdc_type == SPDCType.TYPE_II:
            # |HV⟩ + e^(iφ)|VH⟩ / √2
            phase = np.random.uniform(0, 0.1)  # Small random phase
            state = np.array([0, 1, np.exp(1j * phase), 0]) / np.sqrt(2)
        else:
            # Type-I: |HH⟩ + |VV⟩ / √2 (using two crystals)
            phase = np.random.uniform(0, 0.1)
            state = np.array([1, 0, 0, np.exp(1j * phase)]) / np.sqrt(2)

        # Add depolarizing noise
        if noise_level > 0:
            noise = np.random.normal(0, noise_level, 4) + \
                   1j * np.random.normal(0, noise_level, 4)
            state = state + noise
            state = state / np.linalg.norm(state)

        # Timing
        t = np.random.exponential(1 / self.calculate_pair_rate())
        t += np.random.normal(0, timing_jitter)

        return PhotonPair(
            wavelength_signal=lambda_signal,
            wavelength_idler=lambda_idler,
            polarization_state=state,
            generation_time=t
        )

    def generate_pairs(
        self,
        duration: float = 1.0,  # seconds
        max_pairs: int = 1000
    ) -> List[PhotonPair]:
        """Generate pairs over specified duration."""
        pairs = []
        current_time = 0

        while current_time < duration and len(pairs) < max_pairs:
            pair = self.generate_pair()
            pair.generation_time += current_time
            current_time = pair.generation_time

            if current_time < duration:
                pairs.append(pair)

        return pairs

# Example usage
source = SPDCSource(
    pump_wavelength=405.0,
    pump_power=100.0,
    crystal_type="PPKTP",
    spdc_type=SPDCType.TYPE_II
)

print(f"SPDC Source Configuration:")
print(f"  Pump: {source.pump_wavelength} nm, {source.pump_power} mW")
print(f"  Crystal: {source.crystal_type}")
print(f"  Pair rate: {source.calculate_pair_rate():.0f} pairs/s")

# Generate sample pair
pair = source.generate_pair()
print(f"\nGenerated pair:")
print(f"  Signal: {pair.wavelength_signal:.1f} nm")
print(f"  Idler: {pair.wavelength_idler:.1f} nm")
print(f"  Fidelity to |ψ⁺⟩: {pair.fidelity_to_bell('psi+'):.4f}")
```

## Quantum Dot Entangled Photon Sources

Quantum dots (QDs) are semiconductor nanostructures that can emit entangled photon pairs through the biexciton-exciton cascade.

```
┌─────────────────────────────────────────────────────────────────────┐
│              BIEXCITON CASCADE IN QUANTUM DOT                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Energy                                                             │
│    ↑                                                                │
│    │     ┌───────────┐                                              │
│    │     │   |XX⟩    │  Biexciton (2 e-h pairs)                     │
│    │     └─────┬─────┘                                              │
│    │           │                                                    │
│    │     ┌─────┴─────┐                                              │
│    │    /            \   γ₁ (H or V polarized)                      │
│    │   ↓              ↓                                             │
│    │ ┌───┐          ┌───┐                                           │
│    │ │|X⟩│          │|Y⟩│  Exciton states                           │
│    │ │ H │          │ V │  (fine structure splitting Δ)             │
│    │ └─┬─┘          └─┬─┘                                           │
│    │   │              │                                             │
│    │   ↓              ↓   γ₂ (H or V polarized)                     │
│    │   └──────┬───────┘                                             │
│    │     ┌────┴────┐                                                │
│    │     │   |0⟩   │  Ground state                                  │
│    │     └─────────┘                                                │
│    └─────────────────────────────────────────────────────────────>  │
│                                                                     │
│  Ideal case (Δ=0): |ψ⟩ = (|HH⟩ + |VV⟩)/√2                          │
│  With splitting:   Degraded entanglement fidelity                   │
└─────────────────────────────────────────────────────────────────────┘
```

### Quantum Dot Source Simulation

```python
@dataclass
class QuantumDotParameters:
    """Physical parameters of a quantum dot."""
    fine_structure_splitting: float = 0.0  # μeV (ideally 0)
    radiative_lifetime: float = 1.0        # ns
    biexciton_binding: float = 2.0         # meV
    emission_wavelength: float = 780.0     # nm
    temperature: float = 4.0               # Kelvin

class QuantumDotSource:
    """
    Simulates entangled photon emission from quantum dot.

    Uses biexciton-exciton cascade to generate
    polarization-entangled photon pairs.
    """

    def __init__(self, params: QuantumDotParameters):
        self.params = params
        self.hbar = 6.582e-4  # eV·ns

    def calculate_entanglement_fidelity(self) -> float:
        """
        Calculate entanglement fidelity based on fine structure splitting.

        Fine structure splitting causes which-path information:
        Δ > ℏ/τ leads to reduced entanglement
        """
        # Convert fine structure to angular frequency
        delta_omega = self.params.fine_structure_splitting * 1e-6 / self.hbar  # rad/ns

        # Time uncertainty from radiative lifetime
        tau = self.params.radiative_lifetime

        # Fidelity reduction from splitting
        # F ≈ (1 + exp(-Δ²τ²/2)) / 2
        phase_avg = np.exp(-0.5 * (delta_omega * tau)**2)
        fidelity = 0.5 * (1 + phase_avg)

        return fidelity

    def generate_cascade(self) -> Tuple[PhotonPair, dict]:
        """
        Generate entangled pair via biexciton cascade.

        Returns photon pair and timing information.
        """
        # Emission times (exponential decay)
        t_xx = np.random.exponential(self.params.radiative_lifetime)  # XX → X
        t_x = np.random.exponential(self.params.radiative_lifetime)   # X → 0

        # Calculate wavelengths
        # XX photon has slightly lower energy due to biexciton binding
        e_xx = 1240 / self.params.emission_wavelength  # eV
        e_x = e_xx + self.params.biexciton_binding * 1e-3  # Add binding energy

        lambda_xx = 1240 / e_xx
        lambda_x = 1240 / e_x

        # Phase accumulated during cascade (from fine structure)
        delta_e = self.params.fine_structure_splitting * 1e-6  # eV
        phase = delta_e * t_x / self.hbar

        # Generate state with phase from fine structure
        # Ideal: (|HH⟩ + |VV⟩)/√2
        # With phase: (|HH⟩ + e^(iφ)|VV⟩)/√2
        state = np.array([
            1 / np.sqrt(2),           # |HH⟩
            0,                         # |HV⟩
            0,                         # |VH⟩
            np.exp(1j * phase) / np.sqrt(2)  # |VV⟩
        ])

        pair = PhotonPair(
            wavelength_signal=lambda_xx,
            wavelength_idler=lambda_x,
            polarization_state=state,
            generation_time=t_xx
        )

        timing = {
            "t_biexciton": t_xx,
            "t_exciton": t_x,
            "total_cascade_time": t_xx + t_x,
            "phase_accumulated": phase
        }

        return pair, timing

    def characterize_source(self, n_samples: int = 1000) -> dict:
        """Characterize source through multiple emissions."""
        fidelities = []
        cascade_times = []

        for _ in range(n_samples):
            pair, timing = self.generate_cascade()
            fidelities.append(pair.fidelity_to_bell("phi+"))
            cascade_times.append(timing["total_cascade_time"])

        return {
            "theoretical_fidelity": self.calculate_entanglement_fidelity(),
            "measured_fidelity": np.mean(fidelities),
            "fidelity_std": np.std(fidelities),
            "mean_cascade_time": np.mean(cascade_times),
            "max_emission_rate": 1 / np.mean(cascade_times)
        }

# Compare ideal vs real quantum dot
print("Quantum Dot Entanglement Source Analysis")
print("=" * 50)

# Ideal QD (zero fine structure splitting)
ideal_params = QuantumDotParameters(fine_structure_splitting=0.0)
ideal_source = QuantumDotSource(ideal_params)
ideal_stats = ideal_source.characterize_source()

print(f"\nIdeal QD (Δ = 0 μeV):")
print(f"  Theoretical fidelity: {ideal_stats['theoretical_fidelity']:.4f}")
print(f"  Simulated fidelity: {ideal_stats['measured_fidelity']:.4f}")

# Realistic QD with fine structure splitting
real_params = QuantumDotParameters(fine_structure_splitting=5.0)  # 5 μeV
real_source = QuantumDotSource(real_params)
real_stats = real_source.characterize_source()

print(f"\nReal QD (Δ = 5 μeV):")
print(f"  Theoretical fidelity: {real_stats['theoretical_fidelity']:.4f}")
print(f"  Simulated fidelity: {real_stats['measured_fidelity']:.4f}")
print(f"  Max rate: {real_stats['max_emission_rate']/1e6:.1f} MHz")
```

## Atomic Ensemble Entanglement

Atomic ensembles can generate entanglement through collective excitations, particularly useful for quantum memories.

```
┌─────────────────────────────────────────────────────────────────────┐
│              DLCZ PROTOCOL FOR ATOMIC ENTANGLEMENT                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Ensemble A              Ensemble B                                 │
│  ┌─────────┐             ┌─────────┐                               │
│  │ ● ● ● ● │             │ ● ● ● ● │                               │
│  │ ● ● ● ● │   Write     │ ● ● ● ● │                               │
│  │ ● ● ● ● │   pulse     │ ● ● ● ● │                               │
│  └────┬────┘     ↓       └────┬────┘                               │
│       │     ═════════════     │                                     │
│       │         BSM           │                                     │
│       ↓     ┌───────┐         ↓                                     │
│    Stokes  │  50:50  │    Stokes                                   │
│    photon  │   BS    │    photon                                   │
│            └────┬────┘                                              │
│                 ↓                                                   │
│           Detection                                                 │
│     (heralds entanglement)                                          │
│                                                                     │
│  After detection: Ensembles share entangled spin wave               │
│  |ψ⟩ = (|1_A⟩|0_B⟩ + e^(iφ)|0_A⟩|1_B⟩) / √2                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### DLCZ Protocol Implementation

```python
class AtomicEnsemble:
    """
    Simulates atomic ensemble for DLCZ-type entanglement.

    Based on collective atomic excitations that emit
    photons correlated with stored spin waves.
    """

    def __init__(
        self,
        n_atoms: int = 1000000,
        optical_depth: float = 100.0,
        spin_coherence_time: float = 100e-6,  # 100 μs
        read_efficiency: float = 0.5
    ):
        self.n_atoms = n_atoms
        self.optical_depth = optical_depth
        self.spin_coherence_time = spin_coherence_time
        self.read_efficiency = read_efficiency
        self.has_excitation = False
        self.excitation_time = None

    def write_pulse(self, pulse_probability: float = 0.01) -> Optional[float]:
        """
        Apply write pulse to create probabilistic excitation.

        Returns photon emission time if Stokes photon generated.
        """
        # Probabilistic Raman scattering creates correlated
        # Stokes photon + spin wave
        if np.random.random() < pulse_probability:
            self.has_excitation = True
            self.excitation_time = 0.0  # Reset clock
            return np.random.exponential(1e-9)  # ~ns emission time
        return None

    def read_excitation(self, delay: float) -> Tuple[bool, float]:
        """
        Read out stored spin wave after delay.

        Returns (success, fidelity) based on decoherence.
        """
        if not self.has_excitation:
            return False, 0.0

        # Decay of spin coherence
        coherence = np.exp(-delay / self.spin_coherence_time)

        # Probabilistic readout
        if np.random.random() < self.read_efficiency * coherence:
            self.has_excitation = False
            return True, coherence

        return False, 0.0

class DLCZEntanglement:
    """
    Implements DLCZ protocol for heralded entanglement.

    Two remote atomic ensembles become entangled when
    their emitted photons interfere at a beam splitter.
    """

    def __init__(
        self,
        ensemble_a: AtomicEnsemble,
        ensemble_b: AtomicEnsemble,
        path_length_a: float = 10.0,  # km
        path_length_b: float = 10.0,  # km
        detector_efficiency: float = 0.9
    ):
        self.ensemble_a = ensemble_a
        self.ensemble_b = ensemble_b
        self.path_length_a = path_length_a
        self.path_length_b = path_length_b
        self.detector_efficiency = detector_efficiency
        self.fiber_loss_db_km = 0.2  # dB/km

    def attempt_entanglement(
        self,
        write_prob: float = 0.01
    ) -> Tuple[bool, Optional[dict]]:
        """
        Attempt to create heralded entanglement.

        Success requires exactly one photon detection
        after beam splitter (which-path erasure).
        """
        # Send write pulses to both ensembles
        photon_a = self.ensemble_a.write_pulse(write_prob)
        photon_b = self.ensemble_b.write_pulse(write_prob)

        # Check for single excitation condition
        if photon_a is None and photon_b is None:
            return False, {"reason": "no_excitation"}

        if photon_a is not None and photon_b is not None:
            # Double excitation - destroys entanglement
            self.ensemble_a.has_excitation = False
            self.ensemble_b.has_excitation = False
            return False, {"reason": "double_excitation"}

        # Single excitation - calculate transmission
        trans_a = 10**(-self.fiber_loss_db_km * self.path_length_a / 10)
        trans_b = 10**(-self.fiber_loss_db_km * self.path_length_b / 10)

        # Beam splitter and detection
        if photon_a is not None:
            detect_prob = trans_a * self.detector_efficiency * 0.5
        else:
            detect_prob = trans_b * self.detector_efficiency * 0.5

        if np.random.random() < detect_prob:
            # Successful heralding!
            # Ensembles are now entangled
            return True, {
                "reason": "success",
                "which_detector": np.random.choice(["D1", "D2"]),
                "entangled_state": "|1_A⟩|0_B⟩ + |0_A⟩|1_B⟩"
            }

        # Photon lost
        self.ensemble_a.has_excitation = False
        self.ensemble_b.has_excitation = False
        return False, {"reason": "photon_lost"}

    def run_protocol(
        self,
        max_attempts: int = 10000,
        write_prob: float = 0.01
    ) -> dict:
        """Run multiple entanglement attempts."""
        results = {
            "attempts": 0,
            "successes": 0,
            "no_excitation": 0,
            "double_excitation": 0,
            "photon_lost": 0
        }

        for _ in range(max_attempts):
            results["attempts"] += 1
            success, info = self.attempt_entanglement(write_prob)

            if success:
                results["successes"] += 1
            else:
                results[info["reason"]] += 1

        results["success_rate"] = results["successes"] / results["attempts"]
        results["entanglement_rate"] = (results["successes"] /
                                         (results["attempts"] * 1e-6))  # per μs

        return results

# Simulate DLCZ protocol
ensemble_a = AtomicEnsemble(n_atoms=1000000, optical_depth=100)
ensemble_b = AtomicEnsemble(n_atoms=1000000, optical_depth=100)

dlcz = DLCZEntanglement(
    ensemble_a, ensemble_b,
    path_length_a=5.0,
    path_length_b=5.0,
    detector_efficiency=0.9
)

print("DLCZ Protocol Simulation")
print("=" * 50)
results = dlcz.run_protocol(max_attempts=10000, write_prob=0.01)
print(f"Total attempts: {results['attempts']}")
print(f"Successful entanglements: {results['successes']}")
print(f"Success rate: {results['success_rate']*100:.3f}%")
print(f"\nFailure breakdown:")
print(f"  No excitation: {results['no_excitation']}")
print(f"  Double excitation: {results['double_excitation']}")
print(f"  Photon lost: {results['photon_lost']}")
```

## Entanglement Quality Metrics

Characterizing entanglement quality is essential for quantum networking applications.

```
┌─────────────────────────────────────────────────────────────────────┐
│                 ENTANGLEMENT QUALITY METRICS                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Fidelity:        F = ⟨ψ_ideal|ρ|ψ_ideal⟩                          │
│                   Overlap with target Bell state                    │
│                   F > 0.5 needed for any advantage                  │
│                                                                     │
│  Concurrence:     C = max(0, λ₁ - λ₂ - λ₃ - λ₄)                    │
│                   C = 0: separable, C = 1: maximally entangled     │
│                                                                     │
│  Bell Violation:  S = E(a,b) - E(a,b') + E(a',b) + E(a',b')        │
│                   Classical: S ≤ 2, Quantum: S ≤ 2√2               │
│                                                                     │
│  Purity:          γ = Tr(ρ²)                                        │
│                   γ = 1: pure state, γ = 0.25: maximally mixed     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### Quality Metrics Implementation

```python
class EntanglementMetrics:
    """
    Calculate various entanglement quality metrics.

    Works with two-qubit density matrices or state vectors.
    """

    def __init__(self, state: np.ndarray):
        """Initialize with state vector or density matrix."""
        if state.ndim == 1:
            # Convert state vector to density matrix
            self.rho = np.outer(state, np.conj(state))
        else:
            self.rho = state

        # Pauli matrices
        self.I = np.eye(2)
        self.X = np.array([[0, 1], [1, 0]])
        self.Y = np.array([[0, -1j], [1j, 0]])
        self.Z = np.array([[1, 0], [0, -1]])

    def fidelity_to_bell(self, bell_state: str = "phi+") -> float:
        """Calculate fidelity to specific Bell state."""
        bell_states = {
            "phi+": np.array([1, 0, 0, 1]) / np.sqrt(2),
            "phi-": np.array([1, 0, 0, -1]) / np.sqrt(2),
            "psi+": np.array([0, 1, 1, 0]) / np.sqrt(2),
            "psi-": np.array([0, 1, -1, 0]) / np.sqrt(2)
        }
        target = bell_states[bell_state]
        target_dm = np.outer(target, np.conj(target))

        # Fidelity for pure target state
        return np.real(np.trace(target_dm @ self.rho))

    def purity(self) -> float:
        """Calculate purity Tr(ρ²)."""
        return np.real(np.trace(self.rho @ self.rho))

    def concurrence(self) -> float:
        """
        Calculate concurrence for two-qubit state.

        C = max(0, λ₁ - λ₂ - λ₃ - λ₄)
        where λᵢ are sorted eigenvalues of R = √(√ρ ρ̃ √ρ)
        """
        # Spin-flip matrix
        sigma_y_tensor = np.kron(self.Y, self.Y)

        # Spin-flipped density matrix
        rho_tilde = sigma_y_tensor @ np.conj(self.rho) @ sigma_y_tensor

        # Calculate R matrix
        sqrt_rho = self._matrix_sqrt(self.rho)
        R = self._matrix_sqrt(sqrt_rho @ rho_tilde @ sqrt_rho)

        # Eigenvalues in decreasing order
        eigenvalues = np.sort(np.real(np.linalg.eigvals(R)))[::-1]

        return max(0, eigenvalues[0] - eigenvalues[1] -
                   eigenvalues[2] - eigenvalues[3])

    def _matrix_sqrt(self, M: np.ndarray) -> np.ndarray:
        """Compute matrix square root."""
        eigenvalues, eigenvectors = np.linalg.eig(M)
        sqrt_eigenvalues = np.sqrt(np.maximum(eigenvalues, 0))
        return eigenvectors @ np.diag(sqrt_eigenvalues) @ np.linalg.inv(eigenvectors)

    def bell_parameter(self) -> float:
        """
        Calculate CHSH Bell parameter S.

        Optimal measurements for maximally entangled states:
        Alice: Z, X
        Bob: (Z+X)/√2, (Z-X)/√2
        """
        # Optimal measurement directions
        a1 = np.array([0, 0, 1])   # Z
        a2 = np.array([1, 0, 0])   # X
        b1 = np.array([1, 0, 1]) / np.sqrt(2)   # (X+Z)/√2
        b2 = np.array([1, 0, -1]) / np.sqrt(2)  # (X-Z)/√2

        def expectation(a, b):
            """Calculate ⟨σₐ ⊗ σ_b⟩."""
            sigma_a = a[0]*self.X + a[1]*self.Y + a[2]*self.Z
            sigma_b = b[0]*self.X + b[1]*self.Y + b[2]*self.Z
            observable = np.kron(sigma_a, sigma_b)
            return np.real(np.trace(observable @ self.rho))

        S = (expectation(a1, b1) - expectation(a1, b2) +
             expectation(a2, b1) + expectation(a2, b2))

        return S

    def full_analysis(self) -> dict:
        """Complete entanglement analysis."""
        return {
            "fidelity_phi+": self.fidelity_to_bell("phi+"),
            "fidelity_psi+": self.fidelity_to_bell("psi+"),
            "purity": self.purity(),
            "concurrence": self.concurrence(),
            "bell_parameter": self.bell_parameter(),
            "bell_violation": self.bell_parameter() > 2
        }
```

## Summary

Entanglement generation is a foundational capability for quantum networks:

```
┌─────────────────────────────────────────────────────────────────────┐
│            SOURCE COMPARISON SUMMARY                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Source Type    Rate        Fidelity    Wavelength   Scalability   │
│  ─────────────────────────────────────────────────────────────────  │
│  SPDC           ~MHz        >0.95       Tunable      High          │
│  Quantum Dots   ~GHz        >0.90       Fixed        Moderate      │
│  Atomic         ~kHz        >0.98       Fixed        Low           │
│  NV Centers     ~kHz        >0.95       637nm        Low           │
│                                                                     │
│  Key Tradeoffs:                                                     │
│  • Rate vs Fidelity (higher rate often means lower quality)        │
│  • Wavelength (telecom vs visible determines fiber compatibility)  │
│  • Probabilistic vs Deterministic (heralding helps)                │
│  • Room temp vs Cryogenic (affects deployment)                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

The choice of entanglement source depends on the specific quantum networking application, with SPDC currently dominating for photonic networks and atomic systems preferred for memory-based architectures.
